异步和性能（Async & Performance） ​

一、事件循环基础 ​

  • 调度模型：调用栈、任务队列（macro/micro）
  • 微任务优先级：Promise.then、MutationObserver、queueMicrotask
js
console.log(1)
Promise.resolve().then(() => console.log(2))
console.log(3)
// 输出：1 3 2

二、异步抽象演进 ​

  • 回调 -> Promise -> async/await
  • 错误模型对齐：try/catch 与 Promise 的拒绝处理
js
async function main() {
  try {
    const res = await fetch(url)
    const data = await res.json()
    return data
  } catch (e) {
    // 错误处理
  }
}

三、性能要点 ​

  • 批处理与节流：requestAnimationFrame、requestIdleCallback、节流/防抖
  • 避免长任务：切片执行（setTimeout/MessageChannel）
  • 数据结构与算法层面的优化（复杂度优先于微优化）

四、与 TypeScript 的关系 ​

  • TS在 async/await 的编译产物与 lib 选择上影响运行时兼容性
  • 为异步 API 提供精确类型（返回 Promise<T>、抛错场景）能大幅提升可维护性

练习（含答案） ​

简单 ​

  1. 事件循环顺序题
js
console.log('A')
setTimeout(() => console.log('B'), 0)
Promise.resolve().then(() => console.log('C'))
console.log('D')

答案：A D C B

  • 同步：A、D 先执行
  • 微任务：Promise.then -> C
  • 宏任务：setTimeout -> B
  1. 微任务优先级题
js
console.log(1)
queueMicrotask(() => console.log(2))
Promise.resolve().then(() => console.log(3))
console.log(4)

答案：1 4 2 3（不同实现都遵循“微任务在当前任务末尾”且按入队顺序处理）

挑战 ​

  1. async/await 混合题
js
async function t(){
  console.log('x')
  await Promise.resolve()
  console.log('y')
}
console.log('a')
t().then(() => console.log('z'))
console.log('b')

答案：a x b y z

  • 同步：a、调用 t() 执行到第一个 await 前，输出 x；随后返回 pending Promise
  • 同步继续：b
  • 微任务：await 的后续 y，随后 then 的回调 z
  1. 错误传播与 finally
js
Promise.reject('e')
  .catch(e => { console.log('caught', e); return 'ok' })
  .finally(() => console.log('fin'))
  .then(v => console.log('then', v))

答案：caught e -> fin -> then ok

  • catch 处理后链路恢复为 fulfilled('ok')，finally 不改变值
  1. 并发合并（all/any/allSettled）
js
const a = Promise.resolve(1)
const b = new Promise(res => setTimeout(() => res(2), 10))
const c = Promise.reject('x')

Promise.all([a,b]).then(v => console.log('all', v))
Promise.any([a,c]).then(v => console.log('any', v))
Promise.allSettled([a,c]).then(v => console.log('settled', v))

答案：

  • all -> ['all', [1,2]]（全部成功才 resolve）
  • any -> ['any', 1]（任一成功即 resolve；都失败才 reject）
  • allSettled -> ['settled', [{status:'fulfilled', value:1}, {status:'rejected', reason:'x'}]]

进阶挑战（含答案） ​

  1. 宏/微任务交错
js
setTimeout(() => console.log('T1'))
Promise.resolve().then(() => {
  console.log('M1')
  setTimeout(() => console.log('T2'))
  Promise.resolve().then(() => console.log('M2'))
})
console.log('S')

答案：S -> M1 -> M2 -> T1 -> T2（同一轮任务末尾先清空微任务，再进入下一轮宏任务）

  1. finally 的返回值不生效
js
Promise.resolve('a')
  .finally(() => 'b')
  .then(v => console.log(v))

答案：输出 'a'。finally 的返回值不会替换链路的 settled 值，仅在抛错时将错误继续向后传播。
