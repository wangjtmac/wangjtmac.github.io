第1章：TypeScript 简介 ​

章节概述 ​

本章介绍了 TypeScript 的核心价值和主要优势，重点讨论了类型安全（type safety）的概念。通过对比 JavaScript 和 TypeScript 在错误处理方面的差异，展示了 TypeScript 如何在编写代码时就能捕获错误，而不是等到运行时才发现问题。

关键概念 ​

  • 类型安全（Type Safety）：使用类型来防止程序执行无效操作
  • 编译时错误检测：在代码编写阶段就发现潜在问题
  • 运行时（run time）与编译时（compile time）：代码执行时与代码编译时的区别
  • 类型层面（type level）与值层面（value level）：在设计程序时考虑类型与实际值的关系

代码示例 ​

JavaScript 中的问题示例 ​

javascript
3 + []; // 评估为字符串 "3"
let obj = {};
obj.foo; // 评估为 undefined

function a(b) {
  return b / 2;
}
a("z"); // 评估为 NaN

TypeScript 的错误提示 ​

typescript
3 + []; // 错误 TS2365: 操作符 '+' 不能应用于类型 '3' 和 'never[]'

let obj = {};
obj.foo; // 错误 TS2339: 属性 'foo' 不存在于类型 '{}' 上

function a(b: number) {
  return b / 2;
}
a("z"); // 错误 TS2345: 类型 '"z"' 的参数不能赋值给类型 'number' 的参数

TypeScript 的优势 ​

  1. 编译时错误检测：在开发阶段就能发现潜在问题，而不是等到运行时
  2. 代码自文档化：类型注解作为文档，帮助理解代码意图
  3. 更好的开发工具支持：智能提示、自动补全、重构工具等
  4. 渐进式采用：可以逐步将 JavaScript 代码转换为 TypeScript

学习要点 ​

  1. TypeScript 的核心价值在于在编译时捕获错误，而不是等到运行时
  2. JavaScript 尝试在运行时处理错误，但这往往导致问题被忽视或延迟发现
  3. TypeScript 通过类型系统帮助开发者在编写代码时就发现潜在问题
  4. 类型安全是指使用类型来防止程序执行无效操作，如：
    • 将数字与数组相加
    • 访问对象上不存在的属性
    • 用错误类型的参数调用函数
    • 调用对象上不存在的方法
    • 导入最近被移动的模块
  5. TypeScript 不仅能捕获错误，还能在你编写代码时就提供错误信息，而不需要依赖单元测试或冒烟测试
  6. 使用 TypeScript 会改变你编写代码的方式，让你先在类型层面设计程序，再填充具体的值

实践练习 ​

typescript
// 练习：修复以下 TypeScript 错误

// 1. 修复类型错误
let value = 5;
let items = ["a", "b", "c"];
let result = value + items; // 错误：不能将数字与数组相加

// 修复方法1：将数组转换为字符串
let result1 = value + items.join('');

// 修复方法2：将数字添加到数组中
let result2 = [value, ...items];

// 2. 修复对象属性访问错误
type User = {
  name: string;
  age: number;
};
let user = { name: "Alice", age: 30 };
console.log(user.email); // 错误：属性 'email' 不存在于类型 'User' 上

// 修复方法：添加可选属性或检查属性是否存在
type UserWithEmail = {
  name: string;
  age: number;
  email?: string;
};
let userWithEmail: UserWithEmail = { name: "Alice", age: 30 };
console.log(userWithEmail.email); // 现在是安全的，可能为 undefined

// 3. 修复函数参数类型错误
function calculateArea(width: number, height: number) {
  return width * height;
}
calculateArea("10", 20); // 错误：类型 'string' 的参数不能赋值给类型 'number' 的参数

// 修复方法：转换参数类型
calculateArea(Number("10"), 20); // 正确：将字符串转换为数字

总结 ​

TypeScript 通过静态类型系统提供了编译时的类型检查，帮助开发者在开发阶段就发现并修复潜在的错误。这不仅提高了代码质量，还能减少运行时错误，提升开发效率。通过学习 TypeScript，我们可以编写更安全、更可维护的代码。
