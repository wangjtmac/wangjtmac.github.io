第3章：类型系统 ​

章节概述 ​

本章深入探讨了 TypeScript 的类型系统，介绍了 TypeScript 中的基本类型、特殊类型、复合类型以及类型运算符等内容。通过详细的示例和解释，帮助读者全面理解 TypeScript 的类型系统及其应用。

基本类型 ​

TypeScript 包含以下基本类型：

  • any：任意类型，跳过类型检查（尽量避免使用）
  • unknown：类型安全的 any，需要类型断言或类型收窄才能使用
  • boolean：布尔值，true 或 false
  • number：数值，包括整数和浮点数
  • bigint：大整数，用于表示大于 2^53 - 1 的整数
  • string：字符串
  • symbol：唯一标识符
  • object：非原始类型的对象

示例 ​

typescript
// 基本类型示例
let isActive: boolean = true;
let count: number = 42;
let bigNumber: bigint = 100n;
let name: string = "TypeScript";
let id: symbol = Symbol("id");
let userData: object = { name: "Alice", age: 30 };

特殊类型 ​

  • void：表示函数没有返回值
  • null：表示值为空
  • undefined：表示值未定义
  • never：表示永远不会发生的值（如函数永远不会正常返回）

示例 ​

typescript
// 特殊类型示例
function logMessage(message: string): void {
  console.log(message);
}

function throwError(message: string): never {
  throw new Error(message);
}

function infiniteLoop(): never {
  while (true) {}
}

数组和元组 ​

数组 ​

TypeScript 中的数组可以使用两种方式定义：

typescript
// 方式1：类型[]
let numbers: number[] = [1, 2, 3, 4, 5];

// 方式2：Array<类型>
let strings: Array<string> = ["a", "b", "c"];

元组 ​

元组是固定长度的数组，每个位置有特定的类型：

typescript
// 元组示例
let tuple: [string, number] = ["hello", 42];
let [text, count] = tuple; // 解构赋值

对象类型 ​

TypeScript 中的对象类型可以描述对象的结构：

typescript
// 对象类型
type Person = {
  name: string;
  age: number;
  address?: string; // 可选属性
  readonly id: number; // 只读属性
};

let user: Person = {
  name: "Bob",
  age: 25,
  id: 1,
};

枚举类型 ​

枚举允许定义一组命名常量：

typescript
// 枚举示例
enum Direction {
  Up,
  Down,
  Left,
  Right,
}

let move: Direction = Direction.Up;

类型别名和接口 ​

类型别名 ​

使用 type 关键字创建类型别名：

typescript
type UserID = string | number;
type Point = { x: number; y: number };

接口 ​

使用 interface 关键字定义接口：

typescript
interface User {
  name: string;
  age: number;
}

interface Employee extends User {
  employeeId: string;
}

联合类型和交叉类型 ​

联合类型 ​

联合类型表示值可以是多种类型之一：

typescript
// 联合类型示例
type ID = string | number;
let id: ID = "abc123";
id = 456; // 也是有效的

交叉类型 ​

交叉类型将多个类型合并为一个类型：

typescript
// 交叉类型示例
type Person = { name: string };
type Employee = { employeeId: number };
type EmployeePerson = Person & Employee;

const worker: EmployeePerson = {
  name: "Alice",
  employeeId: 123,
};

类型断言和类型守卫 ​

类型断言 ​

类型断言用于告诉编译器值的类型：

typescript
// 类型断言示例
let someValue: unknown = "this is a string";
let strLength: number = (someValue as string).length;
// 或者
let strLength2: number = (<string>someValue).length; // 旧语法

类型守卫 ​

类型守卫用于在运行时检查类型：

typescript
// 类型守卫示例
function isString(value: any): value is string {
  return typeof value === "string";
}

function processValue(value: string | number) {
  if (isString(value)) {
    // 在这个块中，value 的类型为 string
    console.log(value.toUpperCase());
  } else {
    // 在这个块中，value 的类型为 number
    console.log(value.toFixed(2));
  }
}

字面量类型 ​

字面量类型表示具体的值：

typescript
// 字面量类型示例
type Direction = "up" | "down" | "left" | "right";
let move: Direction = "up"; // 只能是这四个值之一

type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;
let roll: DiceRoll = 3; // 只能是 1-6 之间的整数

索引类型 ​

索引类型允许捕获对象属性名和值的类型：

typescript
// 索引类型示例
type Dictionary<T> = {
  [key: string]: T;
};

const numberDict: Dictionary<number> = {
  one: 1,
  two: 2,
  three: 3,
};

映射类型 ​

映射类型可以基于现有类型创建新类型：

typescript
// 映射类型示例
type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};

type ReadonlyPerson = Readonly<Person>;

条件类型 ​

条件类型基于条件表达式选择类型：

typescript
// 条件类型示例
type TypeName<T> = T extends string
  ? "string"
  : T extends number
  ? "number"
  : T extends boolean
  ? "boolean"
  : "object";

type T0 = TypeName<string>; // "string"
type T1 = TypeName<number>; // "number"
type T2 = TypeName<boolean>; // "boolean"
type T3 = TypeName<{}>; // "object"

内置工具类型 ​

TypeScript 提供了一些实用的工具类型：

typescript
// 内置工具类型示例
type ReadOnly = Readonly<Person>; // 所有属性都是只读的
type Partial = Partial<Person>; // 所有属性都是可选的
type Pick = Pick<Person, "name" | "age">; // 只选择特定的属性
type Omit = Omit<Person, "address">; // 排除特定的属性
type Record = Record<"x" | "y", number>; // 创建一个对象类型，键是特定的字符串，值是指定的类型

实践练习 ​

typescript
// 练习1：创建一个通用的安全类型转换函数
function safeParseJSON<T>(json: string, fallback: T): T {
  try {
    return JSON.parse(json) as T;
  } catch (e) {
    return fallback;
  }
}

const config = safeParseJSON<{ apiUrl: string }>(
  '{"apiUrl": "https://api.example.com"}',
  { apiUrl: "https://default-api.example.com" }
);

// 练习2：实现一个类型安全的事件发射器
type EventMap = {
  click: { x: number; y: number };
  focus: undefined;
  input: { value: string };
};

class TypedEventEmitter<Events extends Record<string, any>> {
  private listeners: {
    [E in keyof Events]?: Array<(data: Events[E]) => void>;
  } = {};

  on<E extends keyof Events>(event: E, listener: (data: Events[E]) => void): void {
    if (!this.listeners[event]) {
      this.listeners[event] = [];
    }
    this.listeners[event]?.push(listener);
  }

  emit<E extends keyof Events>(event: E, data: Events[E]): void {
    this.listeners[event]?.forEach((listener) => listener(data));
  }
}

const emitter = new TypedEventEmitter<EventMap>();
emitter.on("click", ({ x, y }) => console.log(`Clicked at ${x}, ${y}`));
emitter.emit("click", { x: 10, y: 20 });

学习要点 ​

  1. TypeScript 的类型系统是结构化的，而不是名义化的，意味着类型兼容性基于类型的结构，而不是名称
  2. any 类型应该谨慎使用，因为它会绕过类型检查，可能导致运行时错误
  3. unknown 是一个类型安全的替代方案，要求在使用前进行类型检查
  4. 联合类型和交叉类型是组合类型的强大工具，可以创建复杂的类型结构
  5. 类型守卫和断言可以帮助 TypeScript 编译器更好地理解代码中的类型
  6. 字面量类型可以用来创建更精确的类型定义，限制值的范围
  7. TypeScript 的高级类型功能（如映射类型、条件类型）可以帮助创建更灵活和可重用的类型

总结 ​

TypeScript 的类型系统提供了丰富的工具来描述各种数据结构和函数签名。通过掌握这些类型功能，开发者可以创建更安全、更可维护的代码，减少运行时错误，并提高代码的可读性和自文档化程度。本章介绍的类型概念是 TypeScript 编程的基础，为后续章节中的函数、类和接口等高级功能奠定了基础。
