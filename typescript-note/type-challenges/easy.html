1. Pick ​

实现内置的 Pick<T, K> 类型，从类型 T 中选择出属性 K。

typescript
interface Todo {
  title: string
  description: string
  completed: boolean
}

type TodoPreview = MyPick<Todo, 'title' | 'completed'>

const todo: TodoPreview = {
    title: 'Clean room',
    completed: false,
}

2. Readonly ​

实现内置的 Readonly<T> 类型，将 T 的所有属性设为只读。

typescript
interface Todo {
  title: string
  description: string
}

const todo: MyReadonly<Todo> = {
  title: "Hey",
  description: "foobar"
}

todo.title = "Hello" // Error: cannot reassign a readonly property
todo.description = "barFoo" // Error: cannot reassign a readonly property

3. Tuple to Object ​

实现 TupleToObject<T>：将只读元组的每个元素作为键和值，生成对象类型。

ts
const tuple = ['tesla', 'model 3', 'model X', 'model Y'] as const

type result = TupleToObject<typeof tuple>
// 期望：{ tesla: 'tesla'; 'model 3': 'model 3'; 'model X': 'model X'; 'model Y': 'model Y' }

4. First of Array ​

实现 First<T>：获取数组/元组的第一个元素类型；为空返回 never。

ts
type head1 = First<[3, 2, 1]> // 3
type head2 = First<[]> // never

5. Length of Tuple ​

实现 Length<T>：返回只读元组的长度。

ts
type len1 = Length<[1, 2, 3]> // 3
type len2 = Length<[]> // 0

6. If ​

实现 If<C, T, F>：当 C 为 true 时返回 T，否则返回 F。

ts
type A = If<true, 'x', 'y'> // 'x'
type B = If<false, 'x', 'y'> // 'y'

7. Concat ​

实现 Concat<T, U>：拼接两个元组。

ts
type C1 = Concat<[1], [2]> // [1, 2]

8. Includes ​

实现 Includes<T, U>：判断 U 是否存在于元组 T 中。

ts
type I1 = Includes<['Kars', 'Esidisi', 'Wamuu', 'Santana'], 'Kars'> // true

解答 ​

1. MyPick（从 T 中挑选 K 的属性） ​

ts
// 约束：K 必须来自 T 的键集合
export type MyPick<T, K extends keyof T> = {
  [P in K]: T[P]
}

要点说明：

3. TupleToObject ​

ts
export type TupleToObject<T extends readonly PropertyKey[]> = {
  [K in T[number]]: K
}

4. First ​

ts
export type First<T extends readonly unknown[]> = T extends [infer H, ...any] ? H : never

5. Length ​

ts
export type Length<T extends readonly unknown[]> = T['length']

6. If ​

ts
export type If<C extends boolean, T, F> = C extends true ? T : F

7. Concat ​

ts
export type Concat<A extends readonly unknown[], B extends readonly unknown[]> = [...A, ...B]

8. Includes ​

ts
type Equal<X, Y> = (<T>() => T extends X ? 1 : 2) extends <T>() => T extends Y ? 1 : 2 ? true : false
export type Includes<T extends readonly unknown[], U> = T[number] extends never
  ? false
  : true extends ({ [K in keyof T]: Equal<T[K], U> }[number])
    ? true
    : false
  • 使用 keyof 获取对象所有键的联合类型
  • 通过 in 映射类型对 K 中的每个键进行遍历
  • 访问型索引 T[P] 复用原属性的类型（会保留可选/readonly 等修饰符）
  • 使用 K extends keyof T 保障 K 与 T 的键一致，避免误传

测试用例（思路验证）：

ts
interface Todo {
  title: string
  description: string
  completed: boolean
  readonly meta?: { id: number }
}

// 仅挑选 title 与 completed，且保留 meta 的修饰行为不受影响
type TodoPreview = MyPick<Todo, 'title' | 'completed'>

常见陷阱：

  • 忘记添加 K extends keyof T 约束，导致传入任意字符串时报错不明显
  • 误写成 [P in keyof T] 会选出全部键，不是题目要求

2. MyReadonly（将 T 的所有属性设为只读） ​

ts
export type MyReadonly<T> = {
  readonly [P in keyof T]: T[P]
}

要点说明：

  • readonly 映射修饰符为每个已存在的键添加只读标记
  • 默认会保留原有的可选修饰符（?）与嵌套属性的结构（但不会递归）

测试用例（思路验证）：

ts
interface Todo {
  title: string
  description: string
}

const todo: MyReadonly<Todo> = {
  title: 'Hey',
  description: 'foobar',
}
// todo.title = 'Hello' // TS 报错：只读属性不可重新赋值

延伸练习：

  • 实现 Readonly2，仅将部分键设为只读：Readonly2<T, K extends keyof T>
  • 实现 DeepReadonly，对嵌套对象递归设为只读（在 medium/hard 难度会出现）
