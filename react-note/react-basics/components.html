组件基础 ​

组件是 React 应用的核心构建块，它们允许你将 UI 拆分为独立、可重用的部分，并对每个部分进行独立思考。

组件的类型 ​

React 中有两种主要类型的组件：

1. 函数组件 ​

函数组件是最简单的组件形式，它们是接收 props 对象并返回 React 元素的 JavaScript 函数。

jsx
function Welcome(props) {
  return <h1>你好，{props.name}</h1>;
}

使用 TypeScript 时，可以这样定义函数组件：

tsx
import React from 'react';

interface WelcomeProps {
  name: string;
}

const Welcome: React.FC<WelcomeProps> = ({ name }) => {
  return <h1>你好，{name}</h1>;
};

export default Welcome;

2. 类组件 ​

类组件是使用 ES6 类语法定义的组件，它们提供了更多的功能，如状态管理和生命周期方法。

jsx
import React, { Component } from 'react';

class Welcome extends Component {
  render() {
    return <h1>你好，{this.props.name}</h1>;
  }
}

使用 TypeScript 时，可以这样定义类组件：

tsx
import React, { Component } from 'react';

interface WelcomeProps {
  name: string;
}

interface WelcomeState {
  isGreeting: boolean;
}

class Welcome extends Component<WelcomeProps, WelcomeState> {
  constructor(props: WelcomeProps) {
    super(props);
    this.state = {
      isGreeting: true
    };
  }

  render() {
    return <h1>你好，{this.props.name}</h1>;
  }
}

export default Welcome;

组件的组合 ​

组件可以在其输出中引用其他组件，这使得我们可以使用同一组件抽象来构建任意复杂度的 UI。

jsx
function App() {
  return (
    <div>
      <Welcome name="Alice" />
      <Welcome name="Bob" />
      <Welcome name="Charlie" />
    </div>
  );
}

组件的提取 ​

将复杂组件拆分为更小的组件是一个好习惯。

jsx
// 原始组件
function Comment(props) {
  return (
    <div className="comment">
      <div className="user-info">
        <img className="avatar"
          src={props.author.avatarUrl}
          alt={props.author.name}
        />
        <div className="user-name">
          {props.author.name}
        </div>
      </div>
      <div className="comment-text">
        {props.text}
      </div>
      <div className="comment-date">
        {formatDate(props.date)}
      </div>
    </div>
  );
}

// 提取后的组件
function Avatar(props) {
  return (
    <img className="avatar"
      src={props.user.avatarUrl}
      alt={props.user.name}
    />
  );
}

function UserInfo(props) {
  return (
    <div className="user-info">
      <Avatar user={props.user} />
      <div className="user-name">
        {props.user.name}
      </div>
    </div>
  );
}

function Comment(props) {
  return (
    <div className="comment">
      <UserInfo user={props.author} />
      <div className="comment-text">
        {props.text}
      </div>
      <div className="comment-date">
        {formatDate(props.date)}
      </div>
    </div>
  );
}

Props 是只读的 ​

组件无论是函数组件还是类组件，都不能修改自己的 props。所有 React 组件都必须像纯函数一样保护它们的 props 不被更改。

jsx
// 正确的做法
function sum(a, b) {
  return a + b;
}

// 错误的做法
function withdraw(account, amount) {
  account.total -= amount;
}

组件的命名约定 ​

  1. 组件名称必须以大写字母开头：React 将小写字母开头的组件视为 DOM 标签。
  2. 使用语义化的名称：组件名称应该清晰地表达其功能。
  3. 使用 PascalCase：组件名称使用 PascalCase（首字母大写）命名法。
jsx
// 正确的命名
function UserProfile() {
  return <div>...</div>;
}

// 错误的命名
function userProfile() {
  return <div>...</div>;
}

组件的导入和导出 ​

组件可以使用 ES6 模块系统进行导入和导出。

jsx
// 默认导出
export default function Button() {
  return <button>点击我</button>;
}

// 命名导出
export function Alert() {
  return <div className="alert">警告！</div>;
}

// 在另一个文件中导入
import Button, { Alert } from './components';

实践示例：计数器组件 ​

以下是一个简单的计数器组件示例，它展示了函数组件、状态管理和事件处理的基本用法：

tsx
import React, { useState } from 'react';
import './Counter.css';

const Counter: React.FC = () => {
  // 使用 useState Hook 创建一个状态变量和更新函数
  const [count, setCount] = useState(0);

  // 增加计数的处理函数
  const handleIncrement = () => {
    setCount(count + 1);
  };

  // 减少计数的处理函数
  const handleDecrement = () => {
    setCount(count - 1);
  };

  // 重置计数的处理函数
  const handleReset = () => {
    setCount(0);
  };

  return (
    <div className="counter">
      <h2>计数器示例</h2>
      <p>当前计数: <span className={count > 0 ? 'positive' : count < 0 ? 'negative' : ''}>{count}</span></p>
      <div className="buttons">
        <button onClick={handleDecrement}>减少</button>
        <button onClick={handleReset}>重置</button>
        <button onClick={handleIncrement}>增加</button>
      </div>
    </div>
  );
};

export default Counter;

组件设计原则 ​

  1. 单一职责原则：每个组件应该只做一件事。
  2. 可组合性：组件应该易于与其他组件组合。
  3. 可重用性：组件应该可以在多个场景中重用。
  4. 可测试性：组件应该易于测试。
  5. 可维护性：组件应该易于理解和修改。
