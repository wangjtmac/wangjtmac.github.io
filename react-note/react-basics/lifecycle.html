组件生命周期 ​

React 组件的生命周期是指组件从创建到销毁的整个过程。理解生命周期对于正确使用 React 组件至关重要。

类组件生命周期 ​

类组件有完整的生命周期方法，可以分为三个阶段：挂载、更新和卸载。

挂载阶段 ​

组件实例被创建并插入 DOM 的过程：

  1. constructor(props) - 组件被创建时调用
  2. static getDerivedStateFromProps(props, state) - 组件挂载前和接收新 props 时调用
  3. render() - 渲染组件
  4. componentDidMount() - 组件挂载到 DOM 后调用
jsx
class LifecycleDemo extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
    console.log('1. Constructor called');
  }

  static getDerivedStateFromProps(props, state) {
    console.log('2. getDerivedStateFromProps called');
    return null; // 返回 null 表示不更新 state
  }

  componentDidMount() {
    console.log('4. componentDidMount called');
    // 组件挂载后执行的操作，如数据获取、订阅等
  }

  render() {
    console.log('3. Render called');
    return <div>Lifecycle Demo</div>;
  }
}

更新阶段 ​

组件的 props 或 state 发生变化时触发更新：

  1. static getDerivedStateFromProps(props, state) - 接收新 props 时调用
  2. shouldComponentUpdate(nextProps, nextState) - 决定组件是否应该更新
  3. render() - 重新渲染组件
  4. getSnapshotBeforeUpdate(prevProps, prevState) - 在最近一次渲染输出被提交前调用
  5. componentDidUpdate(prevProps, prevState, snapshot) - 组件更新后调用
jsx
class LifecycleDemo extends React.Component {
  shouldComponentUpdate(nextProps, nextState) {
    console.log('2. shouldComponentUpdate called');
    return true; // 返回 true 表示组件应该更新
  }

  getSnapshotBeforeUpdate(prevProps, prevState) {
    console.log('4. getSnapshotBeforeUpdate called');
    return null; // 返回值会作为 componentDidUpdate 的第三个参数
  }

  componentDidUpdate(prevProps, prevState, snapshot) {
    console.log('5. componentDidUpdate called');
    // 组件更新后执行的操作
  }

  render() {
    console.log('3. Render called');
    return <div>Lifecycle Demo</div>;
  }
}

卸载阶段 ​

组件从 DOM 中移除的过程：

  1. componentWillUnmount() - 组件卸载前调用
jsx
class LifecycleDemo extends React.Component {
  componentWillUnmount() {
    console.log('componentWillUnmount called');
    // 清理操作，如取消订阅、清除定时器等
  }

  render() {
    return <div>Lifecycle Demo</div>;
  }
}

错误处理 ​

处理子组件渲染过程中的错误：

  1. static getDerivedStateFromError(error) - 渲染备用 UI
  2. componentDidCatch(error, info) - 记录错误信息
jsx
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error) {
    // 更新 state，下次渲染时显示备用 UI
    return { hasError: true };
  }

  componentDidCatch(error, info) {
    // 记录错误信息
    console.error('Error caught:', error, info);
  }

  render() {
    if (this.state.hasError) {
      return <h1>出错了！</h1>;
    }
    return this.props.children;
  }
}

函数组件生命周期（Hooks） ​

函数组件使用 Hooks 来模拟生命周期行为：

挂载和更新 ​

使用 useEffect Hook 可以模拟 componentDidMount 和 componentDidUpdate：

jsx
import React, { useState, useEffect } from 'react';

function LifecycleDemo() {
  const [count, setCount] = useState(0);

  // 相当于 componentDidMount 和 componentDidUpdate
  useEffect(() => {
    console.log('Component mounted or updated');
    document.title = `点击了 ${count} 次`;
    
    // 返回的函数相当于 componentWillUnmount
    return () => {
      console.log('Component will unmount or update');
      document.title = 'React App';
    };
  }, [count]); // 依赖数组，只有当 count 变化时才执行

  return (
    <div>
      <p>你点击了 {count} 次</p>
      <button onClick={() => setCount(count + 1)}>点击我</button>
    </div>
  );
}

只在挂载时执行 ​

要模拟只在 componentDidMount 执行的行为，可以传递空依赖数组：

jsx
useEffect(() => {
  console.log('Component mounted');
  // 执行只需要在挂载时运行的代码，如初始数据获取
  
  return () => {
    console.log('Component will unmount');
    // 清理代码
  };
}, []); // 空依赖数组表示只在挂载和卸载时执行

在每次渲染后执行 ​

不传递依赖数组会让 effect 在每次渲染后执行：

jsx
useEffect(() => {
  console.log('This runs after every render');
});

生命周期最佳实践 ​

  1. 避免在 componentDidMount 和 componentDidUpdate 中直接设置状态，可能导致无限循环
  2. 在 componentWillUnmount 中清理资源，如定时器、事件监听器和订阅
  3. 使用 shouldComponentUpdate 或 React.PureComponent 优化性能
  4. 在函数组件中正确设置 useEffect 的依赖数组，避免遗漏依赖
  5. 使用错误边界捕获和处理组件树中的 JavaScript 错误
  6. 避免不必要的轮询和自动定时器，特别是在演示组件中，除非确实需要自动更新数据

关于轮询和定时器的注意事项 ​

在 React 组件中使用 setInterval 或其他定时器时需要特别注意：

  1. 确保实际需要轮询：轮询会消耗资源并可能导致不必要的渲染，影响应用性能
  2. 正确清理定时器：在类组件的 componentWillUnmount 或函数组件的 useEffect 清理函数中清除定时器
  3. 使用用户交互触发更新：对于演示类组件，优先使用按钮点击等用户交互来触发状态更新，而不是自动轮询
  4. 考虑替代方案：对于实际需要实时数据的场景，考虑使用 WebSockets 或 Server-Sent Events 代替轮询
jsx
// 不推荐 - 自动轮询更新状态
componentDidMount() {
  this.intervalId = setInterval(() => {
    this.setState({ count: this.state.count + 1 });
  }, 1000);
}

// 推荐 - 用户交互触发更新
handleIncrement = () => {
  this.setState({ count: this.state.count + 1 });
}

render() {
  return (
    <button onClick={this.handleIncrement}>增加计数</button>
  );
}

函数组件中也应遵循相同原则：

jsx
// 不推荐 - 自动轮询
useEffect(() => {
  const intervalId = setInterval(() => {
    setCount(count + 1);
  }, 1000);
  
  return () => clearInterval(intervalId);
}, [count]);

// 推荐 - 用户交互
return (
  <button onClick={() => setCount(count + 1)}>增加计数</button>
);

实践示例：数据获取组件 ​

下面是一个结合生命周期方法的数据获取组件示例：

jsx
import React, { Component } from 'react';

class DataFetcher extends Component {
  constructor(props) {
    super(props);
    this.state = {
      data: null,
      loading: true,
      error: null
    };
  }

  componentDidMount() {
    this.fetchData();
  }

  componentDidUpdate(prevProps) {
    // 如果 URL 改变，重新获取数据
    if (prevProps.url !== this.props.url) {
      this.setState({ loading: true });
      this.fetchData();
    }
  }

  componentWillUnmount() {
    // 如果有需要取消的请求或清理的资源
    if (this.controller) {
      this.controller.abort();
    }
  }

  async fetchData() {
    // 使用 AbortController 以便在组件卸载时取消请求
    this.controller = new AbortController();
    
    try {
      const response = await fetch(this.props.url, {
        signal: this.controller.signal
      });
      
      if (!response.ok) {
        throw new Error(`HTTP error! Status: ${response.status}`);
      }
      
      const data = await response.json();
      this.setState({ data, loading: false, error: null });
    } catch (error) {
      if (error.name !== 'AbortError') {
        this.setState({ error: error.message, loading: false });
      }
    }
  }

  render() {
    const { data, loading, error } = this.state;
    
    if (loading) return <div>加载中...</div>;
    if (error) return <div>错误: {error}</div>;
    if (!data) return null;
    
    return this.props.render(data);
  }
}

// 使用示例
function App() {
  return (
    <DataFetcher
      url="https://jsonplaceholder.typicode.com/posts/1"
      render={data => (
        <div>
          <h1>{data.title}</h1>
          <p>{data.body}</p>
        </div>
      )}
    />
  );
}

函数组件版本：

jsx
import React, { useState, useEffect } from 'react';

function DataFetcher({ url, render }) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const controller = new AbortController();
    
    async function fetchData() {
      try {
        setLoading(true);
        const response = await fetch(url, {
          signal: controller.signal
        });
        
        if (!response.ok) {
          throw new Error(`HTTP error! Status: ${response.status}`);
        }
        
        const result = await response.json();
        setData(result);
        setLoading(false);
      } catch (error) {
        if (error.name !== 'AbortError') {
          setError(error.message);
          setLoading(false);
        }
      }
    }
    
    fetchData();
    
    // 清理函数
    return () => {
      controller.abort();
    };
  }, [url]); // 依赖于 url，当 url 变化时重新获取数据

  if (loading) return <div>加载中...</div>;
  if (error) return <div>错误: {error}</div>;
  if (!data) return null;
  
  return render(data);
}

通过理解组件的生命周期，你可以在正确的时间执行正确的操作，如数据获取、DOM 操作和清理资源，从而构建更健壮的 React 应用。

React Hooks 使用规则 ​

使用 React Hooks 时必须遵循一些重要规则，否则会导致组件行为异常或报错：

1. 只在顶层调用 Hooks ​

规则：只在 React 函数组件或自定义 Hook 的顶层调用 Hooks，不要在循环、条件语句或嵌套函数中调用。

错误示例：

jsx
function Counter() {
  const [count, setCount] = useState(0);
  
  useEffect(() => {
    // 错误：在 useEffect 内部调用 useRef
    const isFirstRender = useRef(true);
    
    if (isFirstRender.current) {
      isFirstRender.current = false;
      return;
    }
    
    console.log(`计数已更新为 ${count}`);
  }, [count]);
  
  return <button onClick={() => setCount(count + 1)}>点击 {count}</button>;
}

正确示例：

jsx
function Counter() {
  const [count, setCount] = useState(0);
  // 正确：在组件顶层调用 useRef
  const isFirstRender = useRef(true);
  
  useEffect(() => {
    if (isFirstRender.current) {
      isFirstRender.current = false;
      return;
    }
    
    console.log(`计数已更新为 ${count}`);
  }, [count]);
  
  return <button onClick={() => setCount(count + 1)}>点击 {count}</button>;
}

2. 只在 React 函数组件中调用 Hooks ​

Hooks 只能在 React 函数组件或自定义 Hook 中调用，不能在普通 JavaScript 函数、类组件或类组件的方法中调用。

3. 自定义 Hook 必须以 "use" 开头命名 ​

这是一个约定，帮助 React 和 linter 识别自定义 Hook，确保 Hook 规则被正确应用。

常见错误与修复 ​

1. 在条件语句中调用 Hook ​

错误：

jsx
function Form({ isEditing }) {
  // 错误：条件语句中调用 Hook
  if (isEditing) {
    const [name, setName] = useState('');
  }
  // ...
}

修复：

jsx
function Form({ isEditing }) {
  // 正确：在顶层调用 Hook
  const [name, setName] = useState(isEditing ? '' : null);
  // 然后在需要时使用条件渲染
  // ...
}

2. 在嵌套函数中创建 Hook ​

错误：

jsx
function Counter() {
  function handleClick() {
    // 错误：在事件处理函数中调用 Hook
    const [count, setCount] = useState(0);
  }
  // ...
}

修复：

jsx
function Counter() {
  // 正确：在顶层调用 Hook
  const [count, setCount] = useState(0);
  
  function handleClick() {
    setCount(count + 1);
  }
  // ...
}

3. 在 useEffect 内部使用 useRef 等 Hook ​

这是一个常见错误，特别是在跟踪组件是否首次渲染时：

错误：

jsx
function Component() {
  const [data, setData] = useState(null);
  
  useEffect(() => {
    // 错误：在 useEffect 内部调用 useRef
    const isFirstRender = useRef(true);
    
    if (isFirstRender.current) {
      isFirstRender.current = false;
      return;
    }
    
    // 处理非首次渲染逻辑
  }, [data]);
  // ...
}

修复：

jsx
function Component() {
  const [data, setData] = useState(null);
  // 正确：在组件顶层声明所有需要的 ref
  const isFirstRender = useRef(true);
  
  useEffect(() => {
    if (isFirstRender.current) {
      isFirstRender.current = false;
      return;
    }
    
    // 处理非首次渲染逻辑
  }, [data]);
  // ...
}

4. 多个相似效果的处理 ​

当需要跟踪多个不同依赖项的首次渲染时，应为每个效果创建单独的 ref：

jsx
function UserProfile() {
  const [userData, setUserData] = useState(null);
  const [preferences, setPreferences] = useState(null);
  
  // 为不同的效果创建单独的 ref
  const isFirstUserDataRender = useRef(true);
  const isFirstPreferencesRender = useRef(true);
  
  useEffect(() => {
    if (isFirstUserDataRender.current) {
      isFirstUserDataRender.current = false;
      return;
    }
    
    console.log('用户数据已更新');
  }, [userData]);
  
  useEffect(() => {
    if (isFirstPreferencesRender.current) {
      isFirstPreferencesRender.current = false;
      return;
    }
    
    console.log('用户偏好已更新');
  }, [preferences]);
  
  // ...
}

遵循这些规则和修复方法，可以避免 React Hooks 相关的常见错误，确保组件行为正常且可预测。
