Props 与 State ​

Props 和 State 是 React 中管理和传递数据的两种主要方式，理解它们的区别和使用方法是掌握 React 的关键。

Props（属性） ​

Props 是从父组件传递给子组件的数据，它们是只读的，子组件不能修改接收到的 props。

Props 的基本用法 ​

jsx
// 父组件
function ParentComponent() {
  return <ChildComponent name="张三" age={25} />;
}

// 子组件
function ChildComponent(props) {
  return (
    <div>
      <p>姓名：{props.name}</p>
      <p>年龄：{props.age}</p>
    </div>
  );
}

使用 TypeScript 时，可以为 props 定义类型：

tsx
interface ChildProps {
  name: string;
  age: number;
}

const ChildComponent: React.FC<ChildProps> = ({ name, age }) => {
  return (
    <div>
      <p>姓名：{name}</p>
      <p>年龄：{age}</p>
    </div>
  );
};

Props 的解构赋值 ​

为了使代码更简洁，我们通常会对 props 进行解构赋值：

jsx
function ChildComponent({ name, age }) {
  return (
    <div>
      <p>姓名：{name}</p>
      <p>年龄：{age}</p>
    </div>
  );
}

默认 Props ​

可以为组件的 props 设置默认值：

jsx
function ChildComponent({ name, age = 18 }) {
  return (
    <div>
      <p>姓名：{name}</p>
      <p>年龄：{age}</p>
    </div>
  );
}

使用 TypeScript 时：

tsx
interface ChildProps {
  name: string;
  age?: number; // 可选属性
}

const ChildComponent: React.FC<ChildProps> = ({ name, age = 18 }) => {
  return (
    <div>
      <p>姓名：{name}</p>
      <p>年龄：{age}</p>
    </div>
  );
};

传递所有 Props ​

有时候我们需要将所有 props 传递给子组件，可以使用展开运算符：

jsx
function WrapperComponent(props) {
  return <ChildComponent {...props} />;
}

子组件（Children） ​

React 组件可以通过 children 属性接收子元素：

jsx
function Container({ children }) {
  return <div className="container">{children}</div>;
}

function App() {
  return (
    <Container>
      <h1>标题</h1>
      <p>内容</p>
    </Container>
  );
}

State（状态） ​

State 是组件内部的数据，它可以被组件自身修改。当 state 发生变化时，组件会重新渲染。

函数组件中的 State ​

在函数组件中，我们使用 useState Hook 来管理状态：

jsx
import React, { useState } from 'react';

function Counter() {
  // 声明一个叫 "count" 的 state 变量，初始值为 0
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>你点击了 {count} 次</p>
      <button onClick={() => setCount(count + 1)}>
        点击我
      </button>
    </div>
  );
}

使用 TypeScript 时：

tsx
import React, { useState } from 'react';

const Counter: React.FC = () => {
  // 明确指定类型
  const [count, setCount] = useState<number>(0);

  return (
    <div>
      <p>你点击了 {count} 次</p>
      <button onClick={() => setCount(count + 1)}>
        点击我
      </button>
    </div>
  );
};

类组件中的 State ​

在类组件中，state 是类的一个属性，通过 this.setState() 方法更新：

jsx
import React, { Component } from 'react';

class Counter extends Component {
  constructor(props) {
    super(props);
    this.state = {
      count: 0
    };
  }

  render() {
    return (
      <div>
        <p>你点击了 {this.state.count} 次</p>
        <button onClick={() => this.setState({ count: this.state.count + 1 })}>
          点击我
        </button>
      </div>
    );
  }
}

使用 TypeScript 时：

tsx
import React, { Component } from 'react';

interface CounterState {
  count: number;
}

class Counter extends Component<{}, CounterState> {
  constructor(props: {}) {
    super(props);
    this.state = {
      count: 0
    };
  }

  render() {
    return (
      <div>
        <p>你点击了 {this.state.count} 次</p>
        <button onClick={() => this.setState({ count: this.state.count + 1 })}>
          点击我
        </button>
      </div>
    );
  }
}

State 更新可能是异步的 ​

React 可能会将多个 setState() 调用合并成一个更新，因此 state 的更新可能是异步的：

jsx
// 错误的方式
this.setState({
  count: this.state.count + 1
});

// 正确的方式（使用函数形式）
this.setState((prevState) => ({
  count: prevState.count + 1
}));

在函数组件中：

jsx
// 错误的方式（在某些情况下可能导致问题）
setCount(count + 1);

// 正确的方式（使用函数形式）
setCount(prevCount => prevCount + 1);

State 更新会被合并 ​

当你调用 setState() 时，React 会将你提供的对象合并到当前的 state：

jsx
// 类组件
this.state = {
  posts: [],
  comments: []
};

// 只更新 comments，posts 不会被影响
this.setState({
  comments: ['新评论']
});

在函数组件中，useState 不会自动合并对象，你需要手动合并：

jsx
const [state, setState] = useState({
  posts: [],
  comments: []
});

// 需要手动合并
setState(prevState => ({
  ...prevState,
  comments: ['新评论']
}));

Props 与 State 的区别 ​

特性PropsState
来源从父组件接收在组件内部定义
可变性只读的可以通过特定方法修改
用途传递数据和回调函数管理组件内部数据
更新当父组件重新渲染时更新当 setState 或 useState 的更新函数被调用时更新

状态提升 ​

当多个组件需要共享状态时，可以将状态提升到它们最近的共同父组件中：

jsx
function TemperatureInput({ temperature, scale, onTemperatureChange }) {
  return (
    <fieldset>
      <legend>输入温度（{scale === 'c' ? '摄氏度' : '华氏度'}）：</legend>
      <input
        value={temperature}
        onChange={(e) => onTemperatureChange(e.target.value)}
      />
    </fieldset>
  );
}

function Calculator() {
  const [temperature, setTemperature] = useState('');
  const [scale, setScale] = useState('c');

  const handleCelsiusChange = (temperature) => {
    setTemperature(temperature);
    setScale('c');
  };

  const handleFahrenheitChange = (temperature) => {
    setTemperature(temperature);
    setScale('f');
  };

  const celsius = scale === 'f' ? tryConvert(temperature, toCelsius) : temperature;
  const fahrenheit = scale === 'c' ? tryConvert(temperature, toFahrenheit) : temperature;

  return (
    <div>
      <TemperatureInput
        scale="c"
        temperature={celsius}
        onTemperatureChange={handleCelsiusChange}
      />
      <TemperatureInput
        scale="f"
        temperature={fahrenheit}
        onTemperatureChange={handleFahrenheitChange}
      />
    </div>
  );
}

实践示例：用户信息卡片 ​

下面是一个结合 Props 和 State 的实践示例，我们将创建一个用户信息卡片组件：

tsx
import React, { useState } from 'react';
import './UserCard.css';

interface UserCardProps {
  name: string;
  title: string;
  avatar: string;
  description?: string;
}

const UserCard: React.FC<UserCardProps> = ({ name, title, avatar, description = '暂无描述' }) => {
  const [isExpanded, setIsExpanded] = useState(false);

  const toggleExpand = () => {
    setIsExpanded(!isExpanded);
  };

  return (
    <div className="user-card">
      <img src={avatar} alt={name} className="user-avatar" />
      <div className="user-info">
        <h3>{name}</h3>
        <p className="user-title">{title}</p>
        {isExpanded && <p className="user-description">{description}</p>}
        <button onClick={toggleExpand}>
          {isExpanded ? '收起' : '查看更多'}
        </button>
      </div>
    </div>
  );
};

export default UserCard;

在上面的例子中：

  • name、title、avatar 和 description 是从父组件传递的 props
  • isExpanded 是组件内部的 state，用于控制描述信息的显示和隐藏
  • 当用户点击按钮时，toggleExpand 函数会更新 isExpanded 状态，从而改变组件的渲染结果

通过这个例子，我们可以看到 Props 和 State 如何协同工作：Props 用于接收外部数据，State 用于管理组件内部状态。
