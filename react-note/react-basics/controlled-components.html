受控组件 ​

在React中，表单元素（如<input>、<textarea>和<select>）通常维护自己的状态，并根据用户输入进行更新。在React的世界里，可变状态通常保存在组件的state中，并且只能通过setState()来更新。我们可以结合这两个特性，使React的state成为"单一数据源"。这种由React控制表单输入值的组件被称为"受控组件"。

受控组件与非受控组件的区别 ​

受控组件 ​

在受控组件中，表单数据由React组件处理。每当表单元素的状态发生变化时，都会触发onChange事件，然后通过setState更新组件的状态。

jsx
function ControlledForm() {
  const [value, setValue] = useState('');

  const handleChange = (event) => {
    setValue(event.target.value);
  };

  const handleSubmit = (event) => {
    event.preventDefault();
    console.log('提交的值:', value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input type="text" value={value} onChange={handleChange} />
      <button type="submit">提交</button>
    </form>
  );
}

非受控组件 ​

在非受控组件中，表单数据由DOM自身处理。我们使用ref来获取表单值，而不是为每个状态更新编写事件处理程序。

jsx
function UncontrolledForm() {
  const inputRef = useRef(null);

  const handleSubmit = (event) => {
    event.preventDefault();
    console.log('提交的值:', inputRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input type="text" defaultValue="" ref={inputRef} />
      <button type="submit">提交</button>
    </form>
  );
}

function Checkbox() {

多选框 ​

jsx
function MultiSelect() {
  const [selectedOptions, setSelectedOptions] = useState(['apple']);

  const handleChange = (e) => {
    const options = Array.from(e.target.selectedOptions, option => option.value);
    setSelectedOptions(options);
  };

  return (
    <div>
      <select multiple value={selectedOptions} onChange={handleChange}>
        <option value="apple">苹果</option>
        <option value="banana">香蕉</option>
        <option value="orange">橙子</option>
        <option value="grape">葡萄</option>
      </select>
      <p>选择的水果: {selectedOptions.join(', ')}</p>
    </div>
  );
}

处理多个输入 ​

对于包含多个输入字段的表单，我们可以给每个元素添加name属性，并让处理函数根据event.target.name的值选择要执行的操作。

jsx
function MultipleInputs() {
  const [formData, setFormData] = useState({
    firstName: '',
    lastName: '',
    email: ''
  });

  const handleChange = (e) => {
    const { name, value } = e.target;
    setFormData(prevData => ({
      ...prevData,
      [name]: value
    }));
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log('表单数据:', formData);
  };

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label>
          名:
          <input
            type="text"
            name="firstName"
            value={formData.firstName}
            onChange={handleChange}
          />
        </label>
      </div>
      <div>
        <label>
          姓:
          <input
            type="text"
            name="lastName"
            value={formData.lastName}
            onChange={handleChange}
          />
        </label>
      </div>
      <div>
        <label>
          邮箱:
          <input
            type="email"
            name="email"
            value={formData.email}
            onChange={handleChange}
          />
        </label>
      </div>
      <button type="submit">提交</button>
    </form>
  );
}

受控组件的优缺点 ​

优点 ​

  1. 即时表单验证：可以在用户输入时立即验证输入内容
  2. 条件禁用提交按钮：可以根据表单状态禁用提交按钮
  3. 强制输入格式：可以强制特定的输入格式
  4. 动态输入：一个输入可以根据另一个输入的变化而变化
  5. 状态可预测：表单的状态始终存储在React组件中，更容易调试和测试

缺点 ​

  1. 更多的样板代码：需要为每个表单字段编写状态和处理函数
  2. 性能考虑：对于非常大的表单，每次按键都会触发重新渲染，可能导致性能问题
  3. 不适合某些场景：文件上传等特殊场景更适合使用非受控组件

受控组件的最佳实践 ​

  1. 使用合适的初始状态：为每个表单字段提供合适的初始值
jsx
// 好的做法
const [value, setValue] = useState('');

// 避免使用undefined作为初始值
const [value, setValue] = useState(undefined); // 不推荐
  1. 处理多个输入时使用计算属性名：
jsx
const handleChange = (e) => {
  const { name, value } = e.target;
  setFormData(prevData => ({
    ...prevData,
    [name]: value // 使用计算属性名
  }));
};
  1. 表单验证：
jsx
function ValidatedForm() {
  const [email, setEmail] = useState('');
  const [error, setError] = useState('');

  const handleChange = (e) => {
    const value = e.target.value;
    setEmail(value);
    
    // 简单的邮箱验证
    if (!/\S+@\S+\.\S+/.test(value) && value) {
      setError('请输入有效的电子邮件地址');
    } else {
      setError('');
    }
  };

  return (
    <div>
      <input
        type="email"
        value={email}
        onChange={handleChange}
        className={error ? 'invalid' : ''}
      />
      {error && <p className="error">{error}</p>}
    </div>
  );
}
  1. 使用自定义钩子抽象表单逻辑：
jsx
function useFormInput(initialValue) {
  const [value, setValue] = useState(initialValue);
  
  const handleChange = (e) => {
    setValue(e.target.value);
  };
  
  return {
    value,
    onChange: handleChange
  };
}

// 使用自定义钩子
function SimpleForm() {
  const name = useFormInput('');
  const email = useFormInput('');
  
  const handleSubmit = (e) => {
    e.preventDefault();
    console.log('提交:', { name: name.value, email: email.value });
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <input type="text" {...name} placeholder="姓名" />
      <input type="email" {...email} placeholder="邮箱" />
      <button type="submit">提交</button>
    </form>
  );
}
  1. 考虑使用表单库：对于复杂表单，考虑使用Formik、React Hook Form等专门的表单库

何时使用受控组件vs非受控组件 ​

使用受控组件的场景 ​

  • 需要即时表单验证
  • 需要有条件地禁用提交按钮
  • 需要强制输入格式
  • 需要多个输入相互依赖
  • 需要动态表单字段

使用非受控组件的场景 ​

  • 简单的表单，不需要即时验证
  • 文件输入字段
  • 集成第三方DOM库
  • 表单非常大且性能成为问题

总结 ​

受控组件是React中处理表单输入的推荐方式，它使表单数据成为React组件状态的一部分，提供了更好的可预测性和可控性。虽然它需要更多的样板代码，但带来的好处通常超过了这个缺点。对于大多数React应用，受控组件应该是处理表单的首选方式。

对于特殊场景，如文件上传或性能敏感的大型表单，可以考虑使用非受控组件。最终，选择哪种方式取决于具体的应用需求和场景。
