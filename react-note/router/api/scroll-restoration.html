ScrollRestoration API ​

ScrollRestoration 是 React Router v6.4+ 提供的一个组件，用于在导航过程中自动管理滚动位置。它可以在页面切换时恢复滚动位置，提供类似浏览器原生导航的用户体验。

基本用法 ​

jsx
import { ScrollRestoration } from 'react-router-dom';

function App() {
  return (
    <>
      <header>
        <nav>{/* 导航链接 */}</nav>
      </header>
      <main>
        <Outlet /> {/* 路由内容 */}
      </main>
      <footer>{/* 页脚内容 */}</footer>
      <ScrollRestoration /> {/* 滚动恢复组件 */}
    </>
  );
}

属性 ​

属性类型默认值描述
getKeyFunctionlocation => location.key自定义函数，用于生成用于保存滚动位置的键
getPositionFunction内部实现自定义函数，用于确定要保存的滚动位置

详细说明 ​

工作原理 ​

ScrollRestoration 组件通过以下方式工作：

  1. 监听导航事件
  2. 在离开当前页面前，保存当前的滚动位置
  3. 在进入新页面或返回到之前的页面时，恢复保存的滚动位置

getKey ​

getKey 属性允许自定义如何生成用于存储滚动位置的键。默认情况下，它使用 location.key，这意味着每个唯一的导航都有自己的滚动位置。

jsx
<ScrollRestoration 
  getKey={(location, matches) => {
    // 使用路径名作为键，相同路径共享滚动位置
    return location.pathname;
  }}
/>

getPosition ​

getPosition 属性允许自定义要保存的滚动位置。默认情况下，它保存窗口的滚动位置。

jsx
<ScrollRestoration 
  getPosition={(location, matches) => {
    // 自定义滚动位置逻辑
    return [0, 0]; // 总是滚动到顶部
  }}
/>

示例 ​

基本示例 ​

jsx
import { Outlet, ScrollRestoration } from 'react-router-dom';

function Layout() {
  return (
    <div>
      <header>
        <h1>我的应用</h1>
        <nav>{/* 导航链接 */}</nav>
      </header>
      <main>
        <Outlet />
      </main>
      <footer>
        <p>© 2023 我的应用</p>
      </footer>
      <ScrollRestoration />
    </div>
  );
}

按路径保存滚动位置 ​

如果你希望相同路径的页面共享滚动位置，可以使用 pathname 作为键：

jsx
import { Outlet, ScrollRestoration } from 'react-router-dom';

function Layout() {
  return (
    <div>
      <Outlet />
      <ScrollRestoration 
        getKey={(location) => {
          return location.pathname;
        }}
      />
    </div>
  );
}

按路由模式保存滚动位置 ​

如果你有动态路由，但希望相同模式的路由共享滚动位置：

jsx
import { Outlet, ScrollRestoration, matchPath } from 'react-router-dom';

function Layout() {
  return (
    <div>
      <Outlet />
      <ScrollRestoration 
        getKey={(location, matches) => {
          // 检查是否匹配产品详情页模式
          const productMatch = matchPath(
            "/products/:productId",
            location.pathname
          );
          
          if (productMatch) {
            // 所有产品详情页共享滚动位置
            return "product-detail";
          }
          
          // 其他页面使用默认键
          return location.key;
        }}
      />
    </div>
  );
}

自定义滚动位置 ​

如果你需要保存特定元素的滚动位置，而不是整个窗口：

jsx
import { Outlet, ScrollRestoration } from 'react-router-dom';
import { useRef, useEffect } from 'react';

function Layout() {
  const mainRef = useRef(null);
  const scrollPositions = useRef({});
  
  useEffect(() => {
    // 在组件卸载时保存当前滚动位置
    return () => {
      if (mainRef.current) {
        const { pathname } = window.location;
        scrollPositions.current[pathname] = mainRef.current.scrollTop;
      }
    };
  }, []);
  
  return (
    <div>
      <header>固定头部</header>
      <main 
        ref={mainRef}
        style={{ 
          height: 'calc(100vh - 100px)', 
          overflow: 'auto' 
        }}
      >
        <Outlet />
      </main>
      <ScrollRestoration 
        getKey={(location) => location.pathname}
        getPosition={(location, matches) => {
          // 如果有保存的位置，则使用它
          const savedPosition = scrollPositions.current[location.pathname];
          if (savedPosition !== undefined) {
            return [0, savedPosition];
          }
          return [0, 0]; // 默认滚动到顶部
        }}
      />
    </div>
  );
}

禁用特定路由的滚动恢复 ​

如果你希望某些路由不恢复滚动位置：

jsx
import { Outlet, ScrollRestoration } from 'react-router-dom';

function Layout() {
  return (
    <div>
      <Outlet />
      <ScrollRestoration 
        getKey={(location) => {
          // 对于特定路径，返回 null 以禁用滚动恢复
          if (location.pathname === '/no-scroll-restoration') {
            return null;
          }
          return location.key;
        }}
      />
    </div>
  );
}

与动画结合使用 ​

当与页面过渡动画结合使用时，可能需要延迟滚动恢复：

jsx
import { Outlet, ScrollRestoration, useLocation } from 'react-router-dom';
import { useState, useEffect } from 'react';
import { CSSTransition, TransitionGroup } from 'react-transition-group';

function Layout() {
  const location = useLocation();
  const [scrollRestoration, setScrollRestoration] = useState(false);
  
  useEffect(() => {
    // 在动画开始时禁用滚动恢复
    setScrollRestoration(false);
    
    // 在动画完成后启用滚动恢复
    const timer = setTimeout(() => {
      setScrollRestoration(true);
    }, 500); // 与动画持续时间相匹配
    
    return () => clearTimeout(timer);
  }, [location]);
  
  return (
    <div>
      <TransitionGroup>
        <CSSTransition 
          key={location.key}
          classNames="page"
          timeout={500}
        >
          <main>
            <Outlet />
          </main>
        </CSSTransition>
      </TransitionGroup>
      
      {scrollRestoration && <ScrollRestoration />}
    </div>
  );
}

注意事项 ​

  1. 组件位置：ScrollRestoration 组件应该放在应用的根布局组件中，通常在 Outlet 组件之后。

  2. 服务器端渲染：在服务器端渲染环境中，ScrollRestoration 会生成一个 <script> 标签，用于在客户端恢复滚动位置。

  3. 与 <Link preventScrollReset> 的交互：如果 Link 组件设置了 preventScrollReset={true}，那么导航到该链接时 ScrollRestoration 不会重置滚动位置。

  4. 滚动行为：ScrollRestoration 模拟浏览器的原生滚动行为，但可能与某些自定义滚动库或动画库冲突。

最佳实践 ​

  1. 单一实例：在应用中只使用一个 ScrollRestoration 组件，通常放在根布局中。

  2. 与动画协调：如果使用页面过渡动画，确保滚动恢复在动画完成后发生。

  3. 考虑嵌套滚动：如果应用有嵌套的可滚动区域，可能需要自定义 getPosition 函数来处理这些区域的滚动位置。

  4. 测试不同设备：在不同设备和浏览器上测试滚动恢复功能，确保一致的用户体验。

相关 API ​

  • Link - 创建导航链接，可以通过 preventScrollReset 属性控制滚动行为
  • useLocation - 获取当前位置信息
  • useNavigationType - 确定当前导航类型（POP、PUSH 或 REPLACE）
