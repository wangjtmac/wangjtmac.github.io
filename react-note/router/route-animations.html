React Router 路由动画实现 ​

在单页应用（SPA）中，路由切换通常是即时的，没有过渡效果，这可能导致用户体验不够流畅。通过添加路由切换动画，可以使页面切换更加平滑自然，提升用户体验。本文档介绍如何在 React Router 中实现路由切换动画。

实现方案 ​

React Router 路由动画主要通过以下技术实现：

  1. React Router - 提供路由功能和位置信息
  2. react-transition-group - 提供动画过渡的组件
  3. CSS 过渡 - 定义具体的动画效果

基本实现步骤 ​

1. 安装依赖 ​

bash
pnpm add react-router-dom react-transition-group
pnpm add @types/react-transition-group -D  # TypeScript 项目需要

2. 创建动画路由组件 ​

tsx
// AnimatedRoutes.tsx
import React, { useRef } from 'react';
import { Routes, useLocation } from 'react-router-dom';
import { TransitionGroup, CSSTransition } from 'react-transition-group';

interface AnimatedRoutesProps {
  children: React.ReactNode;
}

/**
 * 动画路由组件
 * 为路由切换添加平滑的过渡动画效果
 */
const AnimatedRoutes: React.FC<AnimatedRoutesProps> = ({ children }) => {
  const location = useLocation();
  const nodeRef = useRef(null);
  
  return (
    <div className="route-container">
      <TransitionGroup component={null}>
        <CSSTransition
          key={location.key}
          nodeRef={nodeRef}
          timeout={350}
          classNames="page"
        >
          <div ref={nodeRef} className="page">
            <Routes location={location}>
              {children}
            </Routes>
          </div>
        </CSSTransition>
      </TransitionGroup>
    </div>
  );
};

export default AnimatedRoutes;

3. 定义 CSS 动画样式 ​

css
/* 路由过渡动画样式 */
.route-container {
  position: relative;
  width: 100%;
}

.page {
  position: relative;
  width: 100%;
}

/* 页面进入时的初始状态 */
.page-enter {
  opacity: 0;
  transform: translateX(20px);
}

/* 页面进入的过渡状态 */
.page-enter-active {
  opacity: 1;
  transform: translateX(0);
  transition: opacity 350ms ease-in-out, transform 350ms cubic-bezier(0.2, 0.8, 0.4, 1);
}

/* 页面离开时的初始状态 */
.page-exit {
  opacity: 1;
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
}

/* 页面离开的过渡状态 */
.page-exit-active {
  opacity: 0;
  transform: translateX(-20px);
  transition: opacity 350ms ease-in-out, transform 350ms cubic-bezier(0.2, 0.8, 0.4, 1);
}

4. 在应用中使用动画路由 ​

tsx
// App.tsx
import React from 'react';
import { BrowserRouter } from 'react-router-dom';
import AnimatedRoutes from './components/AnimatedRoutes';
import AppRoutes from './routes/AppRoutes';

function App() {
  return (
    <BrowserRouter>
      <AnimatedRoutes>
        {AppRoutes}
      </AnimatedRoutes>
    </BrowserRouter>
  );
}

export default App;

动画原理解析 ​

路由切换动画的逻辑过程 ​

  1. 路由变化触发：

    • 用户点击链接或以其他方式触发路由变化
    • React Router 更新 location 对象
    • AnimatedRoutes 组件通过 useLocation() 钩子获取这个变化

  2. CSSTransition 组件监测变化：

    • 通过 key={location.key} 属性监测路由变化
    • 当 location.key 变化时，启动新的动画过程

  3. 动画类名添加过程：

    对于进入的新页面：

    • 初始状态：添加 page-enter 类
    • 下一帧：添加 page-enter-active 类
    • 动画结束：移除 page-enter 和 page-enter-active 类

    对于离开的旧页面：

    • 初始状态：添加 page-exit 类
    • 下一帧：添加 page-exit-active 类
    • 动画结束：移除 page-exit 和 page-exit-active 类

  4. classNames 与 className 的区别：

    • classNames="page"（CSSTransition 属性）：定义动画类名的前缀
    • className="page"（div 属性）：定义元素的基本样式类名
    • 动画过程中，元素同时拥有基本类名和动态添加的动画类名

React 18+ 兼容性处理 ​

在 React 18 及以上版本中，findDOMNode 方法已被废弃，这会导致旧版本的 react-transition-group 出现问题。解决方案是使用 useRef 和 nodeRef 属性：

jsx
const nodeRef = useRef(null);

<CSSTransition
  nodeRef={nodeRef}  // 传递引用给 CSSTransition
  // 其他属性...
>
  <div ref={nodeRef}>  // 将引用附加到 DOM 元素
    // 内容...
  </div>
</CSSTransition>

动画效果优化 ​

可以通过调整 CSS 属性来优化动画效果：

  1. 使用更自然的动画曲线：

    css
    transition: opacity 350ms ease-in-out, transform 350ms cubic-bezier(0.2, 0.8, 0.4, 1);

  2. 组合多种变换效果：

    css
    transform: scale(0.98) translateX(20px);

  3. 性能优化：

    css
    will-change: transform, opacity;

  4. 确保内容可滚动：

    css
    .main-content {
  overflow-y: auto;
}

常见问题 ​

  1. 动画不生效：

    • 确保 CSSTransition 的 classNames 属性与 CSS 中定义的类名前缀匹配
    • 确保 timeout 值与 CSS 中的过渡时间一致或略长

  2. 页面内容不可见：

    • 避免在 .page 类中使用 position: absolute 和固定高度
    • 确保新旧页面正确定位

  3. 滚动问题：

    • 在内容容器上使用 overflow-y: auto，而不是整个页面
    • 避免在动画元素上使用 overflow: hidden

结论 ​

通过结合 React Router 和 react-transition-group，可以实现流畅的路由切换动画，提升用户体验。关键是理解动画类名的应用过程，并正确配置 CSS 过渡效果。在 React 18+ 中，需要使用 useRef 和 nodeRef 来避免兼容性问题。
