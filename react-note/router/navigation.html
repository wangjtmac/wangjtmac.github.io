React Router 导航 ​

在React Router中，导航是指在不同路由之间切换的方式。React Router提供了多种导航方式，包括声明式导航和编程式导航。本文将详细介绍这些导航方式的使用方法。

声明式导航 ​

Link 组件 ​

Link组件是React Router中最基本的导航组件，它渲染为HTML的<a>标签，但会阻止默认的页面刷新行为，实现客户端导航。

jsx
import { Link } from 'react-router-dom';

function Navigation() {
  return (
    <nav>
      <Link to="/">首页</Link>
      <Link to="/about">关于</Link>
      <Link to="/users">用户列表</Link>
    </nav>
  );
}

Link 属性 ​

  • to: 导航目标，可以是字符串或对象

    jsx
    // 字符串形式
<Link to="/about">关于</Link>

// 对象形式
<Link to={{
  pathname: "/users",
  search: "?sort=name",
  hash: "#user-list",
  state: { fromDashboard: true }
}}>
  用户列表
</Link>

  • replace: 布尔值，如果为true，则导航会替换历史堆栈中的当前条目，而不是添加新条目

    jsx
    <Link to="/login" replace>登录</Link>

  • state: 传递给导航位置的状态

    jsx
    <Link to="/user/123" state={{ fromDashboard: true }}>用户详情</Link>

NavLink 组件 ​

NavLink是一种特殊的Link组件，当它的to属性匹配当前URL时，可以自动添加样式。这对于导航菜单特别有用。

jsx
import { NavLink } from 'react-router-dom';

function Navigation() {
  return (
    <nav>
      <NavLink 
        to="/" 
        className={({ isActive }) => isActive ? 'active' : ''}
      >
        首页
      </NavLink>
      <NavLink 
        to="/about" 
        className={({ isActive }) => isActive ? 'active' : ''}
      >
        关于
      </NavLink>
    </nav>
  );
}

NavLink 属性 ​

  • 继承Link的所有属性
  • className: 可以是字符串或函数，函数接收一个对象，包含isActive属性
  • style: 可以是对象或函数，函数接收一个对象，包含isActive属性
  • end: 布尔值，如果为true，则只有在路径完全匹配时才会应用active样式
jsx
// 使用style属性
<NavLink 
  to="/users" 
  style={({ isActive }) => ({
    color: isActive ? 'red' : 'black',
    fontWeight: isActive ? 'bold' : 'normal'
  })}
>
  用户列表
</NavLink>

// 使用end属性
<NavLink to="/" end>首页</NavLink>

自定义导航链接 ​

你可以创建自己的导航组件，使用useNavigate和useLocation钩子：

jsx
import { useNavigate, useLocation } from 'react-router-dom';

function CustomLink({ to, children, ...props }) {
  const navigate = useNavigate();
  const location = useLocation();
  const isActive = location.pathname === to;
  
  return (
    <a
      href={to}
      onClick={(e) => {
        e.preventDefault();
        navigate(to);
      }}
      className={isActive ? 'active' : ''}
      {...props}
    >
      {children}
    </a>
  );
}

编程式导航 ​

有时候，我们需要在某些事件（如表单提交、按钮点击）之后进行导航，这时就需要使用编程式导航。

useNavigate 钩子 ​

useNavigate钩子返回一个函数，用于编程式导航：

jsx
import { useNavigate } from 'react-router-dom';

function LoginButton() {
  const navigate = useNavigate();
  
  const handleLogin = async (e) => {
    e.preventDefault();
    // 登录逻辑...
    const success = await loginApi();
    
    if (success) {
      // 登录成功后跳转到仪表板
      navigate('/dashboard');
    }
  };
  
  return <button onClick={handleLogin}>登录</button>;
}

navigate 函数参数 ​

  • 第一个参数：导航目标，可以是字符串或对象
  • 第二个参数：选项对象
    • replace: 布尔值，是否替换当前历史记录
    • state: 要传递的状态对象
jsx
// 简单导航
navigate('/dashboard');

// 替换当前历史记录
navigate('/login', { replace: true });

// 传递状态
navigate('/user/123', { state: { fromDashboard: true } });

// 相对导航（向前或向后）
navigate(-1); // 返回上一页
navigate(1);  // 前进一页

useNavigationType 钩子 ​

useNavigationType钩子返回当前导航类型：

jsx
import { useNavigationType } from 'react-router-dom';

function NavigationTypeExample() {
  const navigationType = useNavigationType();
  
  return (
    <div>
      {navigationType === 'POP' && '用户点击了浏览器的前进/后退按钮'}
      {navigationType === 'PUSH' && '这是一个新的导航'}
      {navigationType === 'REPLACE' && '这个页面替换了上一个页面'}
    </div>
  );
}

useLocation 钩子 ​

useLocation钩子返回当前URL的location对象，包含pathname、search、hash和state属性：

jsx
import { useLocation } from 'react-router-dom';

function LocationExample() {
  const location = useLocation();
  
  return (
    <div>
      <p>当前路径: {location.pathname}</p>
      <p>查询参数: {location.search}</p>
      <p>哈希值: {location.hash}</p>
      <p>状态: {JSON.stringify(location.state)}</p>
    </div>
  );
}

导航与表单 ​

在处理表单时，通常需要在提交后进行导航：

jsx
import { Form, useNavigate } from 'react-router-dom';

function ContactForm() {
  const navigate = useNavigate();
  
  const handleSubmit = async (e) => {
    e.preventDefault();
    const formData = new FormData(e.target);
    const contact = Object.fromEntries(formData);
    
    await saveContact(contact);
    navigate('/contacts');
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <label>
        名称: <input name="name" />
      </label>
      <label>
        邮箱: <input name="email" type="email" />
      </label>
      <button type="submit">保存</button>
    </form>
  );
}

最佳实践 ​

  1. 使用Link而不是a标签：使用Link组件而不是原生a标签进行导航，避免页面刷新。

  2. 为活动链接提供明确的视觉反馈：使用NavLink组件为当前活动的导航项提供明确的样式。

  3. 谨慎使用编程式导航：只在必要时（如表单提交后）使用编程式导航，其他情况优先使用声明式导航。

  4. 传递有意义的状态：通过state属性传递有用的上下文信息，但避免传递过大的数据。

  5. 处理导航错误：在编程式导航中添加错误处理，确保用户不会停留在无效状态。

总结 ​

React Router提供了多种导航方式，包括声明式的Link和NavLink组件，以及编程式的useNavigate钩子。选择合适的导航方式可以提升用户体验并简化代码。在大多数情况下，应该优先使用声明式导航，只在特定场景下使用编程式导航。
