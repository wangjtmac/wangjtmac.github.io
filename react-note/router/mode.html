React Router 模式 ​

React Router 提供了几种不同的路由模式，每种模式都有其特定的用途和优势。本文将介绍 React Router v6 中的主要路由模式。

主要路由模式 ​

React Router v6 提供了以下几种主要的路由器组件：

1. BrowserRouter ​

BrowserRouter 是最常用的路由器，它使用 HTML5 的 history API（pushState、replaceState 和 popstate 事件）来保持 UI 与 URL 同步。

jsx
import { BrowserRouter } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      {/* 路由配置 */}
    </BrowserRouter>
  );
}

特点：

  • 使用干净的 URL，没有哈希符号（#）
  • 需要服务器配置，将所有请求重定向到 index.html
  • 适合现代 Web 应用

服务器配置示例：

对于 Nginx：

nginx
location / {
  try_files $uri $uri/ /index.html;
}

对于 Apache：

apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>

2. HashRouter ​

HashRouter 使用 URL 的哈希部分（window.location.hash）来保持 UI 与 URL 同步。

jsx
import { HashRouter } from 'react-router-dom';

function App() {
  return (
    <HashRouter>
      {/* 路由配置 */}
    </HashRouter>
  );
}

特点：

  • URL 中包含哈希符号（#），如 /#/about
  • 不需要特殊的服务器配置
  • 适合静态文件服务器或不支持 HTML5 history API 的旧浏览器
  • 不利于 SEO（搜索引擎优化）

3. MemoryRouter ​

MemoryRouter 将历史记录保存在内存中，不会改变浏览器的 URL。

jsx
import { MemoryRouter } from 'react-router-dom';

function App() {
  return (
    <MemoryRouter>
      {/* 路由配置 */}
    </MemoryRouter>
  );
}

特点：

  • 不会改变浏览器的 URL
  • 适合测试环境或非浏览器环境（如 React Native）
  • 可以通过 initialEntries 和 initialIndex 属性控制初始路由状态

4. StaticRouter ​

StaticRouter 用于服务器端渲染，它从不会改变位置。

jsx
import { StaticRouter } from 'react-router-dom/server';

function App() {
  return (
    <StaticRouter location="/about">
      {/* 路由配置 */}
    </StaticRouter>
  );
}

特点：

  • 专为服务器端渲染设计
  • 通过 location 属性指定当前位置
  • 不会响应历史记录的变化

5. NativeRouter ​

NativeRouter 用于 React Native 应用。

jsx
import { NativeRouter } from 'react-router-native';

function App() {
  return (
    <NativeRouter>
      {/* 路由配置 */}
    </NativeRouter>
  );
}

特点：

  • 专为 React Native 应用设计
  • 使用 React Native 的导航 API

路由器属性 ​

不同的路由器组件支持不同的属性，以下是一些常用属性：

BrowserRouter 属性 ​

jsx
<BrowserRouter
  basename="/app"      // 所有路由的基础路径
  forceRefresh={false} // 是否强制整页刷新
  window={window}      // 要使用的窗口对象
>
  {/* 路由配置 */}
</BrowserRouter>

HashRouter 属性 ​

jsx
<HashRouter
  basename="/app"      // 所有路由的基础路径
  hashType="slash"     // 哈希类型（slash、noslash 或 hashbang）
  window={window}      // 要使用的窗口对象
>
  {/* 路由配置 */}
</HashRouter>

MemoryRouter 属性 ​

jsx
<MemoryRouter
  initialEntries={['/home', '/about']} // 初始路由条目
  initialIndex={1}                     // 初始索引
>
  {/* 路由配置 */}
</MemoryRouter>

选择合适的路由模式 ​

选择合适的路由模式取决于你的应用需求：

  1. BrowserRouter：适合大多数现代 Web 应用，提供干净的 URL，但需要服务器配置。
  2. HashRouter：适合静态文件服务器或不支持 HTML5 history API 的环境，不需要服务器配置，但 URL 不够美观且不利于 SEO。
  3. MemoryRouter：适合测试环境或需要完全控制路由状态的场景。
  4. StaticRouter：专用于服务器端渲染。
  5. NativeRouter：专用于 React Native 应用。

对于大多数 Web 应用，推荐使用 BrowserRouter，它提供了最佳的用户体验和 SEO 支持。
