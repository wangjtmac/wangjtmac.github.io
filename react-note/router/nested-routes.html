React Router 嵌套路由 ​

嵌套路由是React Router的一个强大特性，它允许我们创建复杂的路由结构，实现多层级的页面布局。本文将详细介绍嵌套路由的概念和使用方法。

嵌套路由概念 ​

嵌套路由是指在父路由内部定义子路由，形成层级结构。当URL匹配到子路由时，父路由和子路由的组件都会被渲染，父组件可以作为子组件的容器，提供共享布局。

这种结构非常适合以下场景：

  • 共享布局（如导航栏、侧边栏、页脚等）
  • 多级导航
  • 主内容区域的子页面切换

定义嵌套路由 ​

在React Router v6中，嵌套路由通过在Route组件内部嵌套Route组件来定义：

jsx
import { Routes, Route } from 'react-router-dom';
import Layout from './Layout';
import Home from './Home';
import About from './About';
import Users from './Users';
import UserProfile from './UserProfile';
import UserSettings from './UserSettings';

function App() {
  return (
    <Routes>
      <Route path="/" element={<Layout />}>
        <Route index element={<Home />} />
        <Route path="about" element={<About />} />
        <Route path="users" element={<Users />}>
          <Route path=":userId" element={<UserProfile />} />
          <Route path=":userId/settings" element={<UserSettings />} />
        </Route>
      </Route>
    </Routes>
  );
}

在上面的例子中：

  • / 路径渲染 Layout 组件
  • /about 路径渲染 Layout 和 About 组件
  • /users 路径渲染 Layout 和 Users 组件
  • /users/:userId 路径渲染 Layout、Users 和 UserProfile 组件
  • /users/:userId/settings 路径渲染 Layout、Users 和 UserSettings 组件

Outlet 组件 ​

Outlet组件是嵌套路由的核心，它作为子路由内容的占位符，告诉React Router在哪里渲染匹配的子路由组件。

jsx
import { Outlet } from 'react-router-dom';

function Layout() {
  return (
    <div>
      <header>
        <h1>我的应用</h1>
        <nav>{/* 导航链接 */}</nav>
      </header>
      
      <main>
        {/* 子路由将在这里渲染 */}
        <Outlet />
      </main>
      
      <footer>
        <p>© 2023 我的应用</p>
      </footer>
    </div>
  );
}

同样，在多级嵌套中，每个父路由组件都需要使用Outlet来渲染其子路由：

jsx
function Users() {
  return (
    <div className="users-container">
      <h2>用户管理</h2>
      <div className="users-content">
        <aside className="users-sidebar">
          {/* 用户列表或导航 */}
        </aside>
        <section className="users-detail">
          {/* 用户详情将在这里渲染 */}
          <Outlet />
        </section>
      </div>
    </div>
  );
}

索引路由 ​

索引路由是一种特殊的路由，它在父路由的路径上渲染默认内容。当URL完全匹配父路由路径时，索引路由的组件会被渲染。

jsx
<Route path="users" element={<Users />}>
  <Route index element={<UsersList />} />
  <Route path=":userId" element={<UserProfile />} />
</Route>

在这个例子中：

  • 当URL是/users时，渲染Users和UsersList组件
  • 当URL是/users/123时，渲染Users和UserProfile组件

索引路由使用index属性而不是path属性，表示它匹配父路由的路径。

嵌套路由与布局 ​

嵌套路由最常见的用例是创建共享布局。以下是一个完整的示例，展示如何使用嵌套路由创建一个带有共享布局的应用：

jsx
// App.jsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Layout from './Layout';
import Dashboard from './Dashboard';
import Products from './Products';
import ProductDetail from './ProductDetail';
import Reports from './Reports';
import NotFound from './NotFound';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Layout />}>
          <Route index element={<Dashboard />} />
          <Route path="products" element={<Products />}>
            <Route index element={<p>请选择一个产品</p>} />
            <Route path=":productId" element={<ProductDetail />} />
          </Route>
          <Route path="reports" element={<Reports />} />
          <Route path="*" element={<NotFound />} />
        </Route>
      </Routes>
    </BrowserRouter>
  );
}
jsx
// Layout.jsx
import { Outlet, Link } from 'react-router-dom';

function Layout() {
  return (
    <div className="app-container">
      <header className="app-header">
        <h1>管理系统</h1>
      </header>
      
      <div className="app-body">
        <nav className="app-nav">
          <ul>
            <li><Link to="/">仪表盘</Link></li>
            <li><Link to="/products">产品管理</Link></li>
            <li><Link to="/reports">报表</Link></li>
          </ul>
        </nav>
        
        <main className="app-content">
          <Outlet />
        </main>
      </div>
      
      <footer className="app-footer">
        <p>© 2023 管理系统</p>
      </footer>
    </div>
  );
}

路由上下文 ​

在嵌套路由中，子路由可以访问父路由的上下文信息。例如，在用户详情页面中，可以访问用户ID参数：

jsx
import { useParams } from 'react-router-dom';

function UserProfile() {
  const { userId } = useParams();
  
  return (
    <div>
      <h3>用户详情</h3>
      <p>用户ID: {userId}</p>
      {/* 其他用户信息 */}
    </div>
  );
}

嵌套路由与相对链接 ​

在嵌套路由中，可以使用相对路径来创建链接，这样可以避免硬编码完整路径：

jsx
function UserItem({ user }) {
  return (
    <li>
      {/* 相对于当前路由的链接 */}
      <Link to={user.id}>{user.name}</Link>
      
      {/* 相对于当前路由的设置链接 */}
      <Link to={`${user.id}/settings`}>设置</Link>
    </li>
  );
}

当UserItem组件在/users路由下使用时，上面的链接会指向/users/{user.id}和/users/{user.id}/settings。

Outlet 上下文 ​

Outlet组件可以接收一个context属性，用于向子路由传递数据：

jsx
function Users() {
  const [users, setUsers] = useState([]);
  
  // 获取用户列表...
  
  return (
    <div>
      <h2>用户管理</h2>
      {/* 将users数据传递给子路由 */}
      <Outlet context={{ users }} />
    </div>
  );
}

在子路由中，可以使用useOutletContext钩子获取这些数据：

jsx
import { useOutletContext } from 'react-router-dom';

function UsersList() {
  const { users } = useOutletContext();
  
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}

嵌套路由的最佳实践 ​

  1. 保持路由结构清晰：路由结构应该反映应用的信息架构，使其易于理解和维护。

  2. 适当使用索引路由：为父路由提供默认内容，避免空白页面。

  3. 避免过深的嵌套：过深的嵌套会使URL变得复杂，也会增加组件的复杂性。通常建议不要超过3-4层嵌套。

  4. 使用相对路径：在链接和导航中使用相对路径，使代码更加灵活和可维护。

  5. 合理组织组件文件：按照路由结构组织组件文件，使项目结构更加清晰。

常见问题 ​

子路由不渲染 ​

如果子路由不渲染，请检查以下几点：

  • 确保在父组件中使用了Outlet组件
  • 确保路由路径正确，特别是路径的层级关系
  • 检查是否有路由拦截或重定向

路径匹配问题 ​

React Router使用路径匹配算法来确定哪个路由与当前URL匹配。如果遇到路径匹配问题，可以尝试以下解决方案：

  • 使用end属性确保路径完全匹配
  • 调整路由顺序，更具体的路由应该放在前面
  • 使用通配符路由(*)捕获未匹配的路径

总结 ​

嵌套路由是React Router的一个强大特性，它允许我们创建复杂的路由结构，实现多层级的页面布局。通过合理使用Route嵌套、Outlet组件和索引路由，我们可以构建出结构清晰、用户体验良好的React应用。
