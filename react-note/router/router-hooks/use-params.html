useParams Hook ​

useParams 是 React Router 提供的一个 Hook，用于访问当前 URL 中的动态参数。这些参数是在路由配置中使用冒号（:）语法定义的。

基本用法 ​

jsx
import { useParams } from 'react-router-dom';

function UserProfile() {
  const { userId } = useParams();
  
  return <h1>用户资料: {userId}</h1>;
}

语法 ​

jsx
const params = useParams();

返回值 ​

useParams 返回一个包含当前 URL 匹配的动态参数的对象。如果没有匹配的参数，则返回一个空对象。

示例 ​

基本示例 ​

首先，在路由配置中定义带有参数的路由：

jsx
import { Routes, Route } from 'react-router-dom';
import UserProfile from './UserProfile';

function App() {
  return (
    <Routes>
      <Route path="/users/:userId" element={<UserProfile />} />
    </Routes>
  );
}

然后，在组件中使用 useParams 获取参数：

jsx
import { useParams } from 'react-router-dom';

function UserProfile() {
  const { userId } = useParams();
  
  return (
    <div>
      <h1>用户资料</h1>
      <p>用户 ID: {userId}</p>
    </div>
  );
}

当访问 /users/123 时，userId 的值将是 "123"。

多个参数 ​

你可以在路由中定义多个参数：

jsx
<Route path="/users/:userId/posts/:postId" element={<UserPost />} />

然后在组件中获取所有参数：

jsx
import { useParams } from 'react-router-dom';

function UserPost() {
  const { userId, postId } = useParams();
  
  return (
    <div>
      <h1>用户帖子</h1>
      <p>用户 ID: {userId}</p>
      <p>帖子 ID: {postId}</p>
    </div>
  );
}

参数类型转换 ​

useParams 返回的所有参数都是字符串，如果需要其他类型的值，需要手动转换：

jsx
import { useParams } from 'react-router-dom';

function ProductDetail() {
  const { productId } = useParams();
  
  // 将字符串转换为数字
  const id = parseInt(productId, 10);
  
  return (
    <div>
      <h1>产品详情</h1>
      <p>产品 ID: {id}</p>
      <p>ID 类型: {typeof id}</p>
    </div>
  );
}

可选参数 ​

React Router v6 不直接支持可选参数语法（如 :param?），但你可以通过定义多个路由来实现类似的效果：

jsx
<Routes>
  <Route path="/products" element={<ProductList />} />
  <Route path="/products/:category" element={<ProductList />} />
</Routes>

在组件中，你可以检查参数是否存在：

jsx
import { useParams } from 'react-router-dom';

function ProductList() {
  const { category } = useParams();
  
  return (
    <div>
      <h1>产品列表</h1>
      {category ? (
        <p>分类: {category}</p>
      ) : (
        <p>所有产品</p>
      )}
    </div>
  );
}

与其他 Hooks 结合使用 ​

useParams 通常与其他 Hooks 结合使用，如 useEffect 来加载数据：

jsx
import { useParams } from 'react-router-dom';
import { useState, useEffect } from 'react';

function UserProfile() {
  const { userId } = useParams();
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  
  useEffect(() => {
    async function fetchUser() {
      try {
        setLoading(true);
        const response = await fetch(`https://api.example.com/users/${userId}`);
        if (!response.ok) {
          throw new Error('Failed to fetch user data');
        }
        const data = await response.json();
        setUser(data);
        setError(null);
      } catch (err) {
        setError(err.message);
        setUser(null);
      } finally {
        setLoading(false);
      }
    }
    
    fetchUser();
  }, [userId]); // 当 userId 变化时重新获取数据
  
  if (loading) return <p>加载中...</p>;
  if (error) return <p>错误: {error}</p>;
  if (!user) return <p>未找到用户</p>;
  
  return (
    <div>
      <h1>{user.name}</h1>
      <p>电子邮件: {user.email}</p>
      <p>电话: {user.phone}</p>
    </div>
  );
}

参数编码和解码 ​

URL 参数通常需要进行编码和解码，以处理特殊字符：

jsx
import { useParams } from 'react-router-dom';

function SearchResults() {
  const { query } = useParams();
  
  // 解码参数
  const decodedQuery = decodeURIComponent(query);
  
  return (
    <div>
      <h1>搜索结果</h1>
      <p>搜索词: {decodedQuery}</p>
    </div>
  );
}

在创建链接时，应该对参数进行编码：

jsx
import { Link } from 'react-router-dom';

function SearchForm() {
  const searchTerm = "React & Router";
  
  // 编码参数
  const encodedTerm = encodeURIComponent(searchTerm);
  
  return (
    <Link to={`/search/${encodedTerm}`}>
      搜索 {searchTerm}
    </Link>
  );
}

注意事项 ​

  1. 参数类型：useParams 返回的所有参数都是字符串，需要手动转换为其他类型。

  2. 参数变化：当 URL 参数变化时，组件会重新渲染，但不会重新挂载。如果你需要在参数变化时执行副作用，请将参数添加到依赖数组中。

  3. 未定义的参数：如果路由中没有定义某个参数，useParams 不会包含该参数。

  4. 参数解构：使用对象解构可以方便地获取特定参数，但要注意处理参数不存在的情况。

最佳实践 ​

  1. 参数命名：使用清晰、描述性的参数名称，如 userId 而不是 id。

  2. 参数验证：在使用参数之前进行验证，确保它们具有预期的格式和值。

  3. 错误处理：处理参数不存在或格式不正确的情况，提供适当的错误信息或回退内容。

  4. 参数编码：在创建链接时对参数进行编码，在使用参数时进行解码。
