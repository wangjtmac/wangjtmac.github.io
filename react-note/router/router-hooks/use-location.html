useLocation Hook ​

useLocation 是 React Router 提供的一个 Hook，用于获取当前 URL 的位置信息。它返回一个包含路径、搜索参数、哈希片段和状态的位置对象。

基本用法 ​

jsx
import { useLocation } from 'react-router-dom';

function CurrentLocation() {
  const location = useLocation();
  
  return (
    <div>
      <h1>当前位置信息</h1>
      <p>路径名: {location.pathname}</p>
      <p>搜索参数: {location.search}</p>
      <p>哈希片段: {location.hash}</p>
      <p>状态: {JSON.stringify(location.state)}</p>
    </div>
  );
}

语法 ​

jsx
const location = useLocation();

返回值 ​

useLocation 返回一个位置对象，包含以下属性：

属性类型描述
pathnamestring当前 URL 的路径部分
searchstring当前 URL 的查询字符串部分（包括 ?）
hashstring当前 URL 的哈希片段部分（包括 #）
stateany通过导航传递的状态数据
keystring位置的唯一标识符

示例 ​

基本示例 ​

jsx
import { useLocation } from 'react-router-dom';

function LocationInfo() {
  const location = useLocation();
  
  return (
    <div>
      <h1>位置信息</h1>
      <pre>{JSON.stringify(location, null, 2)}</pre>
    </div>
  );
}

解析查询参数 ​

结合 URLSearchParams 解析查询参数：

jsx
import { useLocation } from 'react-router-dom';

function SearchParamsDemo() {
  const location = useLocation();
  const searchParams = new URLSearchParams(location.search);
  
  const category = searchParams.get('category');
  const sort = searchParams.get('sort');
  
  return (
    <div>
      <h1>查询参数</h1>
      <p>分类: {category || '未指定'}</p>
      <p>排序方式: {sort || '默认'}</p>
    </div>
  );
}

使用状态数据 ​

访问通过导航传递的状态数据：

jsx
import { useLocation } from 'react-router-dom';

function ProductDetail() {
  const location = useLocation();
  const { product } = location.state || {};
  
  if (!product) {
    return <p>没有产品数据</p>;
  }
  
  return (
    <div>
      <h1>{product.name}</h1>
      <p>价格: ¥{product.price}</p>
      <p>描述: {product.description}</p>
    </div>
  );
}

监听位置变化 ​

结合 useEffect 监听位置变化：

jsx
import { useLocation } from 'react-router-dom';
import { useEffect } from 'react';

function LocationTracker() {
  const location = useLocation();
  
  useEffect(() => {
    // 当位置变化时执行
    console.log('位置已变化:', location);
    
    // 可以在这里执行分析跟踪、页面浏览统计等操作
    trackPageView(location.pathname);
    
  }, [location]);
  
  return <div>当前路径: {location.pathname}</div>;
}

// 假设的页面浏览跟踪函数
function trackPageView(path) {
  // 发送页面浏览数据到分析服务
  console.log(`跟踪页面浏览: ${path}`);
}

区分导航来源 ​

使用 location.key 区分导航来源：

jsx
import { useLocation, useEffect } from 'react-router-dom';

function NavigationTracker() {
  const location = useLocation();
  
  useEffect(() => {
    // location.key 在每次导航时都会改变
    // 但在初始加载时可能是 'default'
    if (location.key === 'default') {
      console.log('页面初始加载');
    } else {
      console.log('通过导航到达此页面');
    }
  }, [location]);
  
  return <div>当前路径: {location.pathname}</div>;
}

实现重定向后返回 ​

使用 location.state 实现重定向后返回原页面：

jsx
import { useLocation, useNavigate } from 'react-router-dom';

// 登录页面
function Login() {
  const location = useLocation();
  const navigate = useNavigate();
  const from = location.state?.from || '/';
  
  const handleLogin = () => {
    // 登录逻辑...
    
    // 登录成功后返回原页面
    navigate(from, { replace: true });
  };
  
  return (
    <div>
      <h1>登录</h1>
      <p>登录后将返回: {from}</p>
      <button onClick={handleLogin}>登录</button>
    </div>
  );
}

// 需要身份验证的页面
function ProtectedPage() {
  const location = useLocation();
  const navigate = useNavigate();
  const isAuthenticated = checkAuth(); // 假设的身份验证检查函数
  
  useEffect(() => {
    if (!isAuthenticated) {
      // 重定向到登录页面，并传递当前位置
      navigate('/login', { state: { from: location.pathname } });
    }
  }, [isAuthenticated, navigate, location]);
  
  if (!isAuthenticated) return null;
  
  return <div>受保护的内容</div>;
}

注意事项 ​

  1. 状态持久性：location.state 在页面刷新后会丢失，除非使用 sessionStorage 或 localStorage 进行持久化。

  2. 位置对象是只读的：不要尝试修改 location 对象的属性，它是只读的。如果需要更新位置，应该使用 navigate 函数。

  3. 与 useSearchParams 的区别：如果只需要处理查询参数，可以考虑使用 useSearchParams 钩子，它提供了更便捷的 API。

  4. 位置变化检测：useLocation 返回的对象在每次位置变化时都是一个新对象，即使某些属性（如 pathname）可能没有变化。

最佳实践 ​

  1. 解构位置对象：使用对象解构获取需要的属性，避免重复访问 location 对象。
jsx
const { pathname, search, hash, state } = useLocation();
  1. 处理缺失的状态：始终为 location.state 提供默认值，以防状态丢失。
jsx
const { product = {} } = location.state || {};

  1. 结合其他 Hooks：将 useLocation 与其他 React Router Hooks 结合使用，如 useNavigate 和 useParams。

  2. 避免过度依赖状态：不要过度依赖 location.state 传递关键数据，考虑使用 URL 参数或查询参数代替。

相关 API ​

  • useNavigate - 用于编程式导航
  • useParams - 访问 URL 参数
  • useSearchParams - 处理查询参数
  • Link - 创建导航链接
