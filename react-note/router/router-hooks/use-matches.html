useMatches ​

useMatches 是 React Router v6.4+ 提供的一个 Hook，用于获取当前匹配的路由层次结构。它返回一个数组，包含从根路由到当前路由的所有匹配项。

基本用法 ​

jsx
import { useMatches } from 'react-router-dom'

function MyComponent() {
  const matches = useMatches()
  
  console.log(matches)
  // [
  //   { id: "root", pathname: "/", params: {}, data: null, handle: {} },
  //   { id: "users", pathname: "/users", params: {}, data: [...], handle: {} },
  //   { id: "user", pathname: "/users/123", params: { id: "123" }, data: {...}, handle: {} }
  // ]
  
  return <div>当前匹配的路由数量: {matches.length}</div>
}

返回值结构 ​

每个匹配项包含以下属性：

  • id: 路由的唯一标识符
  • pathname: 匹配的路径名
  • params: 路由参数对象
  • data: loader 函数返回的数据
  • handle: 路由的 handle 对象

实际应用示例 ​

面包屑导航 ​

jsx
import { useMatches, Link } from 'react-router-dom'

function Breadcrumbs() {
  const matches = useMatches()
  
  // 过滤出有面包屑信息的路由
  const crumbs = matches
    .filter(match => match.handle?.crumb)
    .map(match => match.handle.crumb(match))
  
  return (
    <nav aria-label="面包屑">
      <ol className="breadcrumb">
        {crumbs.map((crumb, index) => (
          <li key={index} className="breadcrumb-item">
            {index === crumbs.length - 1 ? (
              <span>{crumb.label}</span>
            ) : (
              <Link to={crumb.to}>{crumb.label}</Link>
            )}
          </li>
        ))}
      </ol>
    </nav>
  )
}

// 路由配置示例
const routes = [
  {
    path: "/",
    element: <Layout />,
    handle: {
      crumb: () => ({ label: "首页", to: "/" })
    },
    children: [
      {
        path: "users",
        element: <Users />,
        handle: {
          crumb: () => ({ label: "用户", to: "/users" })
        },
        children: [
          {
            path: ":userId",
            element: <UserDetail />,
            loader: userLoader,
            handle: {
              crumb: (match) => ({
                label: match.data.user.name,
                to: `/users/${match.params.userId}`
              })
            }
          }
        ]
      }
    ]
  }
]

页面标题生成 ​

jsx
import { useMatches } from 'react-router-dom'
import { useEffect } from 'react'

function usePageTitle() {
  const matches = useMatches()
  
  useEffect(() => {
    // 从匹配的路由中收集标题信息
    const titles = matches
      .filter(match => match.handle?.title)
      .map(match => {
        const titleFn = match.handle.title
        return typeof titleFn === 'function' 
          ? titleFn(match) 
          : titleFn
      })
    
    // 构建完整标题
    const fullTitle = titles.length > 0 
      ? `${titles.join(' - ')} | 我的应用`
      : '我的应用'
    
    document.title = fullTitle
  }, [matches])
}

// 在根组件中使用
function App() {
  usePageTitle()
  return <Outlet />
}

权限检查 ​

jsx
import { useMatches } from 'react-router-dom'
import { useAuth } from './auth'

function useRoutePermissions() {
  const matches = useMatches()
  const { user } = useAuth()
  
  // 检查当前路由链中是否有需要特定权限的路由
  const requiredPermissions = matches
    .filter(match => match.handle?.permissions)
    .flatMap(match => match.handle.permissions)
  
  const hasPermission = requiredPermissions.every(permission =>
    user?.permissions?.includes(permission)
  )
  
  return { hasPermission, requiredPermissions }
}

// 路由配置
const adminRoute = {
  path: "admin",
  element: <AdminPanel />,
  handle: {
    permissions: ['admin']
  }
}

高级用法 ​

获取特定路由的数据 ​

jsx
function useRouteData(routeId) {
  const matches = useMatches()
  
  const match = matches.find(match => match.id === routeId)
  return match?.data
}

// 使用示例
function UserProfile() {
  // 获取父路由的用户数据
  const userData = useRouteData('users')
  
  return <div>用户数据: {JSON.stringify(userData)}</div>
}

路由分析 ​

jsx
function RouteAnalytics() {
  const matches = useMatches()
  
  useEffect(() => {
    // 发送路由访问分析数据
    const routePath = matches.map(match => match.pathname).join(' -> ')
    const routeIds = matches.map(match => match.id)
    
    analytics.track('route_visited', {
      path: routePath,
      routeIds,
      depth: matches.length
    })
  }, [matches])
  
  return null
}

与其他 Hook 的结合 ​

结合 useLocation ​

jsx
import { useMatches, useLocation } from 'react-router-dom'

function useCurrentRoute() {
  const matches = useMatches()
  const location = useLocation()
  
  // 获取当前最深层的匹配路由
  const currentMatch = matches[matches.length - 1]
  
  return {
    currentMatch,
    isExact: currentMatch?.pathname === location.pathname,
    depth: matches.length
  }
}

结合 useNavigation ​

jsx
import { useMatches, useNavigation } from 'react-router-dom'

function LoadingIndicator() {
  const matches = useMatches()
  const navigation = useNavigation()
  
  // 检查是否有任何匹配的路由正在加载
  const isLoading = navigation.state === 'loading'
  const loadingRoute = matches.find(match => 
    match.pathname === navigation.location?.pathname
  )
  
  if (isLoading && loadingRoute) {
    return <div>正在加载 {loadingRoute.handle?.title || '页面'}...</div>
  }
  
  return null
}

注意事项 ​

  1. 性能考虑: useMatches 在每次路由变化时都会重新计算，避免在渲染函数中进行复杂计算

  2. 数据依赖: 返回的 data 属性依赖于 loader 函数，确保 loader 正确返回数据

  3. handle 对象: 充分利用 handle 对象存储路由元数据

  4. 类型安全: 在 TypeScript 中为 handle 对象定义类型

TypeScript 支持 ​

typescript
interface RouteHandle {
  crumb?: (match: any) => { label: string; to: string }
  title?: string | ((match: any) => string)
  permissions?: string[]
}

// 扩展路由类型
declare module 'react-router-dom' {
  interface RouteObject {
    handle?: RouteHandle
  }
}

最佳实践 ​

  1. 统一的 handle 结构: 为所有路由定义一致的 handle 对象结构
  2. 避免过度使用: 只在确实需要访问路由层次结构时使用
  3. 缓存计算结果: 对于复杂的计算，考虑使用 useMemo 缓存结果
  4. 错误处理: 处理 handle 对象可能不存在的情况

useMatches 是一个强大的 Hook，特别适用于需要了解当前路由上下文的场景，如面包屑导航、权限检查和页面标题管理。
