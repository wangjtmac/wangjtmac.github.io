useRouteLoaderData ​

useRouteLoaderData 是 React Router v6.4+ 提供的一个 Hook，用于访问特定路由的 loader 数据，而不仅仅是当前路由的数据。这对于在子组件中访问父路由的数据特别有用。

基本用法 ​

jsx
import { useRouteLoaderData } from 'react-router-dom'

function UserProfile() {
  // 访问 ID 为 "user" 的路由的 loader 数据
  const userData = useRouteLoaderData("user")
  
  if (!userData) {
    return <div>用户数据不可用</div>
  }
  
  return (
    <div>
      <h1>{userData.name}</h1>
      <p>{userData.email}</p>
    </div>
  )
}

路由配置 ​

要使用 useRouteLoaderData，需要为路由指定 id 属性：

jsx
import { createBrowserRouter } from 'react-router-dom'

const router = createBrowserRouter([
  {
    path: "/",
    element: <Layout />,
    children: [
      {
        path: "users/:userId",
        id: "user", // 重要：指定路由 ID
        element: <UserLayout />,
        loader: async ({ params }) => {
          const response = await fetch(`/api/users/${params.userId}`)
          return response.json()
        },
        children: [
          {
            path: "profile",
            element: <UserProfile />
          },
          {
            path: "settings",
            element: <UserSettings />
          }
        ]
      }
    ]
  }
])

实际应用示例 ​

在子路由中访问父路由数据 ​

jsx
// 父路由组件
function UserLayout() {
  const userData = useLoaderData() // 当前路由的数据
  
  return (
    <div>
      <header>
        <h1>{userData.name}的个人中心</h1>
      </header>
      <nav>
        <Link to="profile">个人资料</Link>
        <Link to="settings">设置</Link>
      </nav>
      <main>
        <Outlet />
      </main>
    </div>
  )
}

// 子路由组件
function UserProfile() {
  // 访问父路由的用户数据
  const userData = useRouteLoaderData("user")
  
  return (
    <div>
      <h2>个人资料</h2>
      <p>姓名: {userData.name}</p>
      <p>邮箱: {userData.email}</p>
      <p>注册时间: {userData.createdAt}</p>
    </div>
  )
}

function UserSettings() {
  // 同样可以访问父路由的用户数据
  const userData = useRouteLoaderData("user")
  
  return (
    <div>
      <h2>设置</h2>
      <form>
        <input 
          type="text" 
          defaultValue={userData.name}
          placeholder="姓名"
        />
        <input 
          type="email" 
          defaultValue={userData.email}
          placeholder="邮箱"
        />
      </form>
    </div>
  )
}

跨层级访问数据 ​

jsx
const router = createBrowserRouter([
  {
    path: "/",
    id: "root",
    element: <RootLayout />,
    loader: rootLoader,
    children: [
      {
        path: "organizations/:orgId",
        id: "organization",
        element: <OrganizationLayout />,
        loader: organizationLoader,
        children: [
          {
            path: "projects/:projectId",
            id: "project",
            element: <ProjectLayout />,
            loader: projectLoader,
            children: [
              {
                path: "tasks/:taskId",
                element: <TaskDetail />
              }
            ]
          }
        ]
      }
    ]
  }
])

// 在最深层的组件中访问各级数据
function TaskDetail() {
  const rootData = useRouteLoaderData("root")
  const orgData = useRouteLoaderData("organization")
  const projectData = useRouteLoaderData("project")
  const taskData = useLoaderData() // 当前路由数据
  
  return (
    <div>
      <nav>
        <span>{rootData.siteName}</span>
        <span> / {orgData.name}</span>
        <span> / {projectData.name}</span>
        <span> / {taskData.title}</span>
      </nav>
      
      <div>
        <h1>{taskData.title}</h1>
        <p>所属项目: {projectData.name}</p>
        <p>所属组织: {orgData.name}</p>
      </div>
    </div>
  )
}

条件渲染基于父路由数据 ​

jsx
function UserDashboard() {
  const userData = useRouteLoaderData("user")
  
  // 根据用户权限显示不同内容
  if (userData.role === 'admin') {
    return <AdminDashboard user={userData} />
  }
  
  if (userData.role === 'moderator') {
    return <ModeratorDashboard user={userData} />
  }
  
  return <RegularUserDashboard user={userData} />
}

与其他 Hook 的结合 ​

结合 useParams ​

jsx
function ProjectMember() {
  const { memberId } = useParams()
  const projectData = useRouteLoaderData("project")
  
  // 从项目数据中找到特定成员
  const member = projectData.members.find(m => m.id === memberId)
  
  if (!member) {
    return <div>成员不存在</div>
  }
  
  return (
    <div>
      <h2>{member.name}</h2>
      <p>项目: {projectData.name}</p>
      <p>角色: {member.role}</p>
    </div>
  )
}

结合 useFetcher ​

jsx
function UpdateUserProfile() {
  const userData = useRouteLoaderData("user")
  const fetcher = useFetcher()
  
  const handleSubmit = (formData) => {
    fetcher.submit(formData, {
      method: "patch",
      action: `/users/${userData.id}`
    })
  }
  
  return (
    <fetcher.Form onSubmit={handleSubmit}>
      <input 
        name="name" 
        defaultValue={userData.name}
      />
      <input 
        name="email" 
        defaultValue={userData.email}
      />
      <button type="submit">
        {fetcher.state === 'submitting' ? '保存中...' : '保存'}
      </button>
    </fetcher.Form>
  )
}

错误处理 ​

jsx
function UserProfile() {
  const userData = useRouteLoaderData("user")
  
  // 处理数据不存在的情况
  if (!userData) {
    return (
      <div>
        <h2>数据加载失败</h2>
        <p>无法获取用户数据，请刷新页面重试。</p>
      </div>
    )
  }
  
  // 处理数据结构不完整的情况
  if (!userData.name || !userData.email) {
    return (
      <div>
        <h2>数据不完整</h2>
        <p>用户数据缺少必要信息。</p>
      </div>
    )
  }
  
  return (
    <div>
      <h2>{userData.name}</h2>
      <p>{userData.email}</p>
    </div>
  )
}

TypeScript 支持 ​

typescript
interface User {
  id: string
  name: string
  email: string
  role: 'admin' | 'moderator' | 'user'
}

interface Project {
  id: string
  name: string
  description: string
  members: User[]
}

function UserProfile() {
  const userData = useRouteLoaderData("user") as User
  const projectData = useRouteLoaderData("project") as Project | undefined
  
  return (
    <div>
      <h2>{userData.name}</h2>
      {projectData && (
        <p>当前项目: {projectData.name}</p>
      )}
    </div>
  )
}

注意事项 ​

  1. 路由 ID 必须唯一: 确保每个路由的 id 在整个路由树中是唯一的

  2. 数据可能为 undefined: 如果指定的路由 ID 不存在或该路由没有 loader，返回值将是 undefined

  3. 重新渲染: 当指定路由的数据发生变化时，使用该 Hook 的组件会重新渲染

  4. 性能考虑: 避免在不必要的地方使用此 Hook，因为它会创建对特定路由数据的依赖

最佳实践 ​

  1. 明确的路由 ID: 使用描述性的路由 ID，如 "user"、"project"、"organization"

  2. 类型安全: 在 TypeScript 中为 loader 数据定义明确的类型

  3. 错误处理: 始终检查返回的数据是否存在

  4. 文档化: 在团队中明确哪些路由有 ID，以及它们提供什么数据

useRouteLoaderData 是构建复杂嵌套路由应用的重要工具，它允许子组件轻松访问父路由的数据，避免了 prop drilling 的问题。
