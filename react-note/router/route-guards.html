React Router 路由守卫 ​

路由守卫是控制用户访问权限的重要机制，虽然React Router没有像Vue Router那样内置的守卫概念，但我们可以通过组件组合和条件渲染来实现相同的功能。本文将详细介绍如何在React Router中实现路由守卫和权限控制。

路由守卫概念 ​

路由守卫是一种控制用户是否可以访问特定路由的机制。在 React Router 中，虽然没有内置的"守卫"概念（与 Vue Router 不同），但我们可以通过组件组合和条件渲染实现相同的功能。

路由守卫主要用于：

  1. 身份验证 - 确保只有登录用户才能访问特定页面
  2. 权限控制 - 确保用户具有访问特定资源的权限
  3. 导航确认 - 在用户离开页面前进行确认
  4. 数据预加载 - 在渲染路由前加载必要数据

实现路由守卫的方法 ​

1. 保护路由组件 ​

最常见的方法是创建一个包装组件，用于包装需要保护的路由组件。该组件会检查用户是否满足访问条件，如果不满足则重定向到其他页面（如登录页面）。

jsx
import { Navigate, useLocation } from 'react-router-dom';
import { useAuth } from './auth';

const ProtectedRoute = ({ children, requiredPermission }) => {
  const { isAuthenticated, hasPermission } = useAuth();
  const location = useLocation();

  // 检查用户是否已登录
  if (!isAuthenticated) {
    // 重定向到登录页面，并记住原始请求的URL
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  // 检查用户是否有所需权限
  if (requiredPermission && !hasPermission(requiredPermission)) {
    return <Navigate to="/unauthorized" replace />;
  }

  // 用户通过验证，渲染子组件
  return children;
};

2. 在路由配置中使用 ​

在 React Router v6 中，我们可以在路由配置中使用保护路由组件：

jsx
import { Routes, Route } from 'react-router-dom';
import { ProtectedRoute } from './ProtectedRoute';

function App() {
  return (
    <Routes>
      <Route path="/" element={<MainLayout />}>
        <Route index element={<Home />} />
        <Route path="about" element={<About />} />
        
        {/* 需要登录才能访问的路由 */}
        <Route 
          path="dashboard" 
          element={
            <ProtectedRoute>
              <Dashboard />
            </ProtectedRoute>
          } 
        />
        
        {/* 需要特定权限才能访问的路由 */}
        <Route 
          path="admin" 
          element={
            <ProtectedRoute requiredPermission="admin.access">
              <Admin />
            </ProtectedRoute>
          } 
        />
      </Route>
    </Routes>
  );
}

认证与授权 ​

1. 认证上下文 ​

为了管理用户的认证状态，最好创建一个认证上下文（Context），提供登录、登出和检查权限等功能：

jsx
import { createContext, useContext, useState } from 'react';

const AuthContext = createContext();

export const AuthProvider = ({ children }) => {
  const [user, setUser] = useState(null);
  
  // 登录功能
  const login = async (username, password) => {
    try {
      // 调用 API 进行验证
      const response = await api.login(username, password);
      const userData = response.data;
      
      // 设置用户信息
      setUser(userData);
      
      // 可以在本地存储中保存token
      localStorage.setItem('token', userData.token);
      
      return true;
    } catch (error) {
      console.error('登录失败:', error);
      return false;
    }
  };
  
  // 登出功能
  const logout = () => {
    setUser(null);
    localStorage.removeItem('token');
  };
  
  // 检查权限
  const hasPermission = (permission) => {
    if (!user) return false;
    return user.permissions.includes(permission);
  };
  
  // 检查角色
  const hasRole = (role) => {
    if (!user) return false;
    return user.roles.includes(role);
  };
  
  return (
    <AuthContext.Provider value={{ 
      user, 
      isAuthenticated: !!user, 
      login, 
      logout, 
      hasPermission,
      hasRole
    }}>
      {children}
    </AuthContext.Provider>
  );
};

export const useAuth = () => useContext(AuthContext);

2. 重定向到原始请求页面 ​

当用户尝试访问需要认证的页面时，我们希望在登录后将其重定向回原始请求页面。这可以通过 useLocation 和 Navigate 组件的 state 属性实现：

jsx
// 登录页面组件
import { useLocation, useNavigate } from 'react-router-dom';
import { useAuth } from './auth';

function Login() {
  const { login } = useAuth();
  const navigate = useNavigate();
  const location = useLocation();
  
  // 获取用户原始请求的URL，如果没有则默认为首页
  const from = location.state?.from?.pathname || '/';
  
  const handleSubmit = async (e) => {
    e.preventDefault();
    const formData = new FormData(e.target);
    const username = formData.get('username');
    const password = formData.get('password');
    
    const success = await login(username, password);
    
    if (success) {
      // 登录成功后重定向到原始请求的URL
      navigate(from, { replace: true });
    }
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <h1>登录</h1>
      <input name="username" placeholder="用户名" required />
      <input name="password" type="password" placeholder="密码" required />
      <button type="submit">登录</button>
    </form>
  );
}

权限控制策略 ​

1. 基于角色的访问控制 (RBAC) ​

为用户分配角色（如管理员、编辑者、普通用户），然后基于角色授予访问权限。

jsx
import { useAuth } from './auth';

function AdminPanel() {
  const { hasRole } = useAuth();
  
  if (!hasRole('admin')) {
    return <Navigate to="/unauthorized" replace />;
  }
  
  return (
    <div className="admin-panel">
      {/* 管理员面板内容 */}
    </div>
  );
}

2. 基于权限的访问控制 ​

为用户分配具体权限（如 'users.edit', 'settings.view'），然后检查用户是否拥有特定权限。

jsx
import { useAuth } from './auth';

function UserManagement() {
  const { hasPermission } = useAuth();
  const canViewUsers = hasPermission('users.view');
  const canEditUsers = hasPermission('users.edit');
  const canDeleteUsers = hasPermission('users.delete');
  
  return (
    <div className="user-management">
      <h1>用户管理</h1>
      
      {canViewUsers ? (
        <UserList />
      ) : (
        <p>您没有查看用户的权限</p>
      )}
      
      {canEditUsers && (
        <button className="btn-edit">编辑用户</button>
      )}
      
      {canDeleteUsers && (
        <button className="btn-delete">删除用户</button>
      )}
    </div>
  );
}

3. UI 元素的条件渲染 ​

除了路由级别的权限控制，还可以在 UI 层面进行权限控制：

jsx
function Navigation() {
  const { isAuthenticated, hasRole, hasPermission } = useAuth();
  
  return (
    <nav>
      <ul>
        <li><Link to="/">首页</Link></li>
        <li><Link to="/about">关于</Link></li>
        
        {isAuthenticated && (
          <li><Link to="/dashboard">仪表盘</Link></li>
        )}
        
        {hasRole('admin') && (
          <li><Link to="/admin">管理后台</Link></li>
        )}
        
        {hasPermission('settings.edit') && (
          <li><Link to="/settings">设置</Link></li>
        )}
        
        {isAuthenticated ? (
          <li><button onClick={logout}>登出</button></li>
        ) : (
          <li><Link to="/login">登录</Link></li>
        )}
      </ul>
    </nav>
  );
}

实现导航确认 ​

有时我们需要在用户离开页面前进行确认，例如防止用户意外丢失未保存的表单数据。React Router v6 提供了 usePrompt 和 useBlocker 钩子来实现这一功能。

jsx
import { useCallback, useEffect, useState } from 'react';
import { useBeforeUnload, useBlocker, useNavigate } from 'react-router-dom';

function usePrompt(message, when = true) {
  const navigate = useNavigate();
  
  // 处理浏览器刷新或关闭
  useBeforeUnload(
    useCallback(
      (event) => {
        if (when) {
          event.preventDefault();
          return message;
        }
      },
      [message, when]
    )
  );
  
  // 处理路由变化
  const blocker = useBlocker(
    useCallback(
      ({ currentLocation, nextLocation }) => 
        when && 
        currentLocation.pathname !== nextLocation.pathname,
      [when]
    )
  );
  
  useEffect(() => {
    if (blocker.state === 'blocked') {
      const confirmed = window.confirm(message);
      if (confirmed) {
        blocker.proceed();
      } else {
        blocker.reset();
      }
    }
  }, [blocker, message]);
  
  return null;
}

// 使用示例
function FormWithConfirmation() {
  const [isDirty, setIsDirty] = useState(false);
  
  // 当表单有未保存的更改时显示确认对话框
  usePrompt('您有未保存的更改，确定要离开吗？', isDirty);
  
  return (
    <form>
      <input 
        type="text" 
        onChange={() => setIsDirty(true)} 
      />
      <button 
        type="submit" 
        onClick={() => setIsDirty(false)}
      >
        保存
      </button>
    </form>
  );
}

数据预加载 ​

React Router v6.4+ 引入了新的数据加载API，允许在路由渲染之前加载数据。这可以用于实现另一种形式的路由守卫，确保必要的数据已加载。

jsx
import { 
  createBrowserRouter, 
  RouterProvider,
  defer,
  Await,
  useLoaderData
} from 'react-router-dom';
import { Suspense } from 'react';

// 定义loader函数
async function dashboardLoader({ request }) {
  const { isAuthenticated } = await checkAuth();
  
  // 如果用户未登录，重定向到登录页面
  if (!isAuthenticated) {
    return redirect('/login');
  }
  
  // 加载仪表盘数据
  const dashboardData = fetchDashboardData();
  
  // 使用defer可以先渲染UI，然后等待数据
  return defer({
    dashboardData
  });
}

// 仪表盘组件
function Dashboard() {
  const { dashboardData } = useLoaderData();
  
  return (
    <div>
      <h1>仪表盘</h1>
      <Suspense fallback={<div>加载中...</div>}>
        <Await resolve={dashboardData}>
          {(data) => (
            <DashboardContent data={data} />
          )}
        </Await>
      </Suspense>
    </div>
  );
}

// 创建路由
const router = createBrowserRouter([
  {
    path: '/',
    element: <Layout />,
    children: [
      { index: true, element: <Home /> },
      { 
        path: 'dashboard',
        element: <Dashboard />,
        loader: dashboardLoader
      }
    ]
  }
]);

// 应用入口
function App() {
  return <RouterProvider router={router} />;
}

最佳实践 ​

  1. 分离关注点：将认证逻辑与路由逻辑分开，使用上下文或状态管理库管理认证状态。

  2. 使用自定义钩子：创建自定义钩子如useAuth，使认证功能在组件中易于使用。

  3. 优雅处理未授权访问：不要只是重定向，还应该提供有用的错误消息，并在可能的情况下保留用户的原始请求。

  4. 权限粒度控制：根据应用的复杂性，选择适当的权限控制粒度（角色、权限或两者结合）。

  5. 保持一致性：在整个应用中使用一致的权限检查方法。

  6. 考虑用户体验：在加载状态和错误处理上投入精力，确保用户知道发生了什么。

  7. 安全第一：记住客户端权限检查只是为了用户体验，真正的安全控制应该在服务器端实现。

总结 ​

虽然React Router没有内置的路由守卫机制，但通过组件组合、条件渲染和上下文API，我们可以实现强大而灵活的路由权限控制系统。无论是简单的身份验证还是复杂的基于角色的权限控制，都可以通过本文介绍的方法实现。
