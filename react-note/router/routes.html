React Router 路由 ​

路由是 React Router 的核心概念，它定义了 URL 路径与 UI 组件之间的映射关系。本文将介绍如何在 React Router v6 中配置和使用路由。

基本路由配置 ​

在 React Router v6 中，路由配置主要通过 Routes 和 Route 组件完成：

jsx
import { Routes, Route } from 'react-router-dom';

function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/about" element={<About />} />
      <Route path="/contact" element={<Contact />} />
    </Routes>
  );
}

function Home() {
  return <h1>首页</h1>;
}

function About() {
  return <h1>关于页面</h1>;
}

function Contact() {
  return <h1>联系我们</h1>;
}

路由组件 ​

Routes ​

Routes 组件是所有路由定义的容器，它会匹配当前 URL 并渲染第一个匹配的路由。

jsx
<Routes>
  {/* 路由定义 */}
</Routes>

Route ​

Route 组件定义了 URL 路径与 UI 组件之间的映射关系。

jsx
<Route path="/path" element={<Component />} />

主要属性：

  • path：URL 路径模式
  • element：匹配路径时要渲染的 React 元素
  • caseSensitive：路径匹配是否区分大小写（默认为 false）
  • index：是否为索引路由（默认为 false）

路由匹配规则 ​

React Router v6 使用了新的路径匹配引擎，它有以下特点：

  1. 动态参数：使用 :paramName 语法定义动态参数
  2. 可选参数：不支持可选参数语法，但可以通过多个路由实现
  3. 通配符：使用 * 匹配任意路径
  4. 区分大小写：默认不区分大小写，可通过 caseSensitive 属性修改

动态参数 ​

jsx
<Route path="/users/:userId" element={<UserProfile />} />

在组件中使用 useParams 钩子获取参数：

jsx
import { useParams } from 'react-router-dom';

function UserProfile() {
  const { userId } = useParams();
  return <h1>用户资料: {userId}</h1>;
}

通配符路由 ​

jsx
<Route path="*" element={<NotFound />} />

这个路由会匹配任何未被其他路由匹配的路径，通常用于显示 404 页面。

嵌套路由 ​

React Router v6 支持嵌套路由，可以在父路由内部定义子路由：

jsx
<Routes>
  <Route path="/" element={<Layout />}>
    <Route index element={<Home />} />
    <Route path="about" element={<About />} />
    <Route path="users" element={<Users />}>
      <Route index element={<UsersList />} />
      <Route path=":userId" element={<UserProfile />} />
    </Route>
  </Route>
</Routes>

在父组件中使用 Outlet 组件指定子路由的渲染位置：

jsx
import { Outlet } from 'react-router-dom';

function Layout() {
  return (
    <div>
      <header>网站头部</header>
      <main>
        <Outlet /> {/* 子路由将在这里渲染 */}
      </main>
      <footer>网站底部</footer>
    </div>
  );
}

索引路由 ​

索引路由是在父路由路径完全匹配时渲染的路由，使用 index 属性定义：

jsx
<Route path="users" element={<Users />}>
  <Route index element={<UsersList />} /> {/* 当路径是 "/users" 时渲染 */}
  <Route path=":userId" element={<UserProfile />} /> {/* 当路径是 "/users/:userId" 时渲染 */}
</Route>

相对路径和绝对路径 ​

在 React Router v6 中，嵌套路由的 path 属性可以是相对路径或绝对路径：

  • 相对路径：不以 / 开头，会相对于父路由路径解析
  • 绝对路径：以 / 开头，会从应用根路径解析
jsx
<Route path="users" element={<Users />}> {/* /users */}
  <Route path="profile" element={<Profile />} /> {/* /users/profile */}
  <Route path="/settings" element={<Settings />} /> {/* /settings */}
</Route>

路由配置对象 ​

除了使用 JSX 定义路由外，React Router v6 还支持使用 JavaScript 对象定义路由，然后使用 useRoutes 钩子渲染：

jsx
import { useRoutes } from 'react-router-dom';

function App() {
  const routes = useRoutes([
    {
      path: '/',
      element: <Layout />,
      children: [
        { index: true, element: <Home /> },
        { path: 'about', element: <About /> },
        {
          path: 'users',
          element: <Users />,
          children: [
            { index: true, element: <UsersList /> },
            { path: ':userId', element: <UserProfile /> }
          ]
        },
        { path: '*', element: <NotFound /> }
      ]
    }
  ]);

  return routes;
}

这种方式更加灵活，适合动态生成路由配置的场景。

最佳实践 ​

  1. 路由组织：将路由配置集中在一个文件中，便于管理和维护
  2. 命名规范：使用清晰、一致的路径命名规范，如使用小写字母和连字符
  3. 懒加载：对于大型应用，使用 React.lazy 和 Suspense 实现路由懒加载
  4. 404 处理：始终添加一个通配符路由来处理未匹配的路径
  5. 路由守卫：使用布局路由和 Outlet 实现路由守卫和权限控制

示例：完整的路由配置 ​

jsx
import { Routes, Route, Navigate } from 'react-router-dom';
import { Suspense, lazy } from 'react';
import Layout from './components/Layout';
import Loading from './components/Loading';
import { useAuth } from './hooks/useAuth';

// 懒加载组件
const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));
const UsersList = lazy(() => import('./pages/UsersList'));
const UserProfile = lazy(() => import('./pages/UserProfile'));
const Settings = lazy(() => import('./pages/Settings'));
const NotFound = lazy(() => import('./pages/NotFound'));

// 需要认证的路由
function ProtectedRoute({ children }) {
  const { isAuthenticated } = useAuth();
  
  if (!isAuthenticated) {
    return <Navigate to="/login" replace />;
  }
  
  return children;
}

function AppRoutes() {
  return (
    <Suspense fallback={<Loading />}>
      <Routes>
        <Route path="/" element={<Layout />}>
          <Route index element={<Home />} />
          <Route path="about" element={<About />} />
          <Route path="users">
            <Route index element={<UsersList />} />
            <Route path=":userId" element={<UserProfile />} />
          </Route>
          <Route 
            path="settings" 
            element={
              <ProtectedRoute>
                <Settings />
              </ProtectedRoute>
            } 
          />
          <Route path="*" element={<NotFound />} />
        </Route>
      </Routes>
    </Suspense>
  );
}

export default AppRoutes;
