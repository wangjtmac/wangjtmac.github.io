Hooks 实现

On this page

React Hooks 实现原理 ​

React Hooks 是 React 16.8 引入的特性，它允许在函数组件中使用状态和其他 React 特性。本文将深入探讨 React Hooks 的实现原理。

Hooks 的设计目标 ​

在了解 Hooks 的实现原理之前，我们需要理解它的设计目标：

  1. 在函数组件中复用状态逻辑：解决类组件中难以复用状态逻辑的问题。
  2. 避免复杂的组件层次结构：减少使用高阶组件和 render props 的需求。
  3. 将相关逻辑放在一起：不再将相关逻辑分散在不同的生命周期方法中。
  4. 避免 this 的困扰：不再需要理解 JavaScript 中 this 的工作方式。

Hooks 的基本原理 ​

函数组件与闭包 ​

Hooks 的实现严重依赖于 JavaScript 的闭包机制。当你调用一个 Hook（如 useState）时，它会创建一个闭包，捕获当前组件的状态。

jsx
function Counter() {
  const [count, setCount] = useState(0);
  // 这里的 setCount 是一个闭包，它"记住"了它所属的组件实例
  
  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

组件与 Fiber 节点的关联 ​

每个函数组件在 React 内部都对应一个 Fiber 节点。这个 Fiber 节点存储了组件的状态、副作用等信息。

当一个函数组件执行时，React 会将当前正在渲染的组件与其对应的 Fiber 节点关联起来。这样，当组件内部调用 Hook 时，Hook 就能知道它属于哪个组件。

Hooks 的内部实现 ​

状态的存储 ​

React 在 Fiber 节点上使用一个链表来存储组件的所有状态。每个 Hook 调用都会在这个链表上创建一个节点。

javascript
// Fiber 节点上的 memoizedState 字段指向 Hook 链表的头部
{
  memoizedState: {
    // 第一个 Hook（例如 useState）
    memoizedState: 0, // 存储状态值
    queue: { ... },   // 更新队列
    next: {
      // 第二个 Hook
      memoizedState: false,
      queue: { ... },
      next: {
        // 第三个 Hook
        // ...
      }
    }
  }
}

Hook 的调用顺序 ​

React 依赖于 Hook 的调用顺序来确定哪个状态对应哪个 Hook 调用。这就是为什么 Hooks 必须在组件的顶层调用，而不能在条件语句、循环或嵌套函数中调用。

jsx
// 正确的用法
function Form() {
  const [name, setName] = useState('');
  const [age, setAge] = useState(0);
  
  // ...
}

// 错误的用法
function Form() {
  const [name, setName] = useState('');
  
  if (name !== '') {
    // 这会导致 Hook 的调用顺序在不同渲染之间发生变化
    const [age, setAge] = useState(0);
  }
  
  // ...
}

调度更新 ​

当你调用一个 Hook 的更新函数（如 useState 返回的 setState）时，React 会将这个更新加入到组件的更新队列中，然后安排一个重新渲染。

javascript
function dispatchAction(fiber, queue, action) {
  // 创建一个更新对象
  const update = {
    action,
    next: null
  };
  
  // 将更新添加到队列中
  const pending = queue.pending;
  if (pending === null) {
    update.next = update;
  } else {
    update.next = pending.next;
    pending.next = update;
  }
  queue.pending = update;
  
  // 安排一个更新
  scheduleUpdateOnFiber(fiber);
}

常见 Hooks 的实现 ​

useState ​

useState 是最基本的 Hook，它允许函数组件拥有状态。

javascript
function useState(initialState) {
  // 获取当前正在渲染的组件对应的 Fiber 节点
  const fiber = ReactCurrentOwner.current;
  
  // 获取当前 Hook
  const hook = getHook(fiber);
  
  // 初始化状态或获取之前的状态
  if (hook.memoizedState === undefined) {
    hook.memoizedState = typeof initialState === 'function'
      ? initialState()
      : initialState;
  }
  
  // 创建更新队列
  const queue = hook.queue || (hook.queue = { pending: null });
  
  // 创建 dispatch 函数
  const dispatch = action => dispatchAction(fiber, queue, action);
  
  return [hook.memoizedState, dispatch];
}

useEffect ​

useEffect 允许在函数组件中执行副作用操作。

javascript
function useEffect(create, deps) {
  // 获取当前 Hook
  const hook = getHook();
  
  // 检查依赖项是否发生变化
  const nextDeps = deps === undefined ? null : deps;
  let destroy = undefined;
  
  if (hook.memoizedState !== null) {
    const prevEffect = hook.memoizedState;
    destroy = prevEffect.destroy;
    
    if (nextDeps !== null) {
      const prevDeps = prevEffect.deps;
      if (areHookInputsEqual(nextDeps, prevDeps)) {
        // 依赖项没有变化，跳过这次副作用
        return;
      }
    }
  }
  
  // 记录这个副作用
  hook.memoizedState = {
    create,
    destroy,
    deps: nextDeps,
    next: null
  };
  
  // 将副作用添加到 Fiber 节点的副作用列表中
  pushEffect(HookEffectTag.UnmountPassive | HookEffectTag.MountPassive, create, destroy, nextDeps);
}

useContext ​

useContext 允许组件消费 Context 的值。

javascript
function useContext(Context) {
  // 获取当前 Context 的值
  const value = readContext(Context);
  return value;
}

useReducer ​

useReducer 是 useState 的替代方案，适用于复杂的状态逻辑。

javascript
function useReducer(reducer, initialArg, init) {
  // 获取当前 Hook
  const hook = getHook();
  
  // 初始化状态
  if (hook.memoizedState === undefined) {
    hook.memoizedState = init !== undefined
      ? init(initialArg)
      : initialArg;
  }
  
  // 创建更新队列
  const queue = hook.queue || (hook.queue = {
    pending: null,
    dispatch: null,
    lastRenderedReducer: reducer,
    lastRenderedState: hook.memoizedState
  });
  
  // 创建 dispatch 函数
  const dispatch = queue.dispatch || (queue.dispatch = action => {
    const newState = reducer(hook.memoizedState, action);
    if (newState === hook.memoizedState) {
      // 状态没有变化，不触发更新
      return;
    }
    
    // 更新状态并触发重新渲染
    dispatchAction(fiber, queue, action);
  });
  
  return [hook.memoizedState, dispatch];
}

Hooks 的执行过程 ​

当一个函数组件执行时，React 会按照以下步骤处理 Hooks：

  1. 准备阶段：React 设置当前正在渲染的组件对应的 Fiber 节点，并将 Hook 链表的指针重置到链表头部。
  2. 执行组件：React 执行函数组件的主体代码，每次遇到 Hook 调用时，都会从链表中获取对应的 Hook 状态。
  3. 完成阶段：React 完成组件的渲染，处理副作用（如 useEffect 注册的函数）。

Hooks 的限制与规则 ​

了解 Hooks 的实现原理可以帮助我们理解为什么 Hooks 有以下限制：

  1. 只能在函数组件的顶层调用 Hooks：因为 React 依赖 Hook 的调用顺序来确定状态与 Hook 的对应关系。
  2. 只能在函数组件或自定义 Hook 中调用 Hooks：因为 Hooks 需要访问 React 的内部状态和生命周期。
  3. 自定义 Hook 必须以 "use" 开头：这是一个命名约定，帮助 React 和开发者识别自定义 Hook。

性能考虑 ​

Hooks 的实现在性能上有一些值得注意的点：

  1. 避免不必要的重新渲染：使用 React.memo 和 useMemo/useCallback 可以避免不必要的重新渲染。
  2. 减少 Hook 依赖项：尽量减少 useEffect、useMemo 和 useCallback 的依赖项，以避免不必要的重新计算和副作用执行。
  3. 惰性初始化状态：对于计算成本高的初始状态，可以使用函数形式的初始值。
jsx
// 普通初始化
const [state, setState] = useState(expensiveComputation());

// 惰性初始化
const [state, setState] = useState(() => expensiveComputation());

总结 ​

React Hooks 通过巧妙地利用 JavaScript 的闭包和 React 的 Fiber 架构，实现了在函数组件中使用状态和其他 React 特性的能力。它们的实现依赖于严格的调用顺序和 Fiber 节点上的状态存储。

理解 Hooks 的实现原理不仅有助于正确使用 Hooks，还能帮助我们编写更高效、更可维护的 React 组件。同时，这也为我们创建自定义 Hooks 提供了理论基础，使我们能够更好地复用组件逻辑。
