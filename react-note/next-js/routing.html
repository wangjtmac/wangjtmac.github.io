路由系统 ​

Next.js 提供了基于文件系统的路由，这意味着你可以通过在特定目录中创建文件来定义路由。Next.js 支持两种路由系统：Pages Router 和 App Router。

Pages Router (传统路由) ​

基本路由 ​

在 pages 目录中创建文件会自动创建对应的路由：

pages/
  index.js          → /
  about.js          → /about
  contact.js        → /contact
  blog/
    index.js        → /blog
    first-post.js   → /blog/first-post

动态路由 ​

使用方括号创建动态路由：

pages/
  posts/
    [id].js         → /posts/1, /posts/abc, etc.
    [...slug].js    → /posts/a, /posts/a/b, /posts/a/b/c, etc.

示例：动态路由页面

jsx
// pages/posts/[id].js
import { useRouter } from 'next/router'

export default function Post() {
  const router = useRouter()
  const { id } = router.query

  return <p>Post: {id}</p>
}

嵌套路由 ​

pages/
  blog/
    index.js        → /blog
    [slug].js       → /blog/hello-world
    category/
      [name].js     → /blog/category/react

App Router (新路由系统) ​

Next.js 13 引入了新的 App Router，基于 React Server Components。

文件结构 ​

app/
  layout.js         → 根布局
  page.js           → / 页面
  about/
    page.js         → /about 页面
  blog/
    layout.js       → /blog 布局
    page.js         → /blog 页面
    [slug]/
      page.js       → /blog/[slug] 页面

特殊文件 ​

  • layout.js - 布局组件
  • page.js - 页面组件
  • loading.js - 加载组件
  • error.js - 错误组件
  • not-found.js - 404 页面

示例：根布局

jsx
// app/layout.js
export default function RootLayout({ children }) {
  return (
    <html lang="zh-CN">
      <body>
        <nav>
          <a href="/">首页</a>
          <a href="/about">关于</a>
        </nav>
        <main>{children}</main>
      </body>
    </html>
  )
}

路由导航 ​

Link 组件 ​

jsx
import Link from 'next/link'

export default function Navigation() {
  return (
    <nav>
      <Link href="/">首页</Link>
      <Link href="/about">关于</Link>
      <Link href="/posts/1">文章 1</Link>
    </nav>
  )
}

编程式导航 ​

jsx
import { useRouter } from 'next/router'

export default function LoginButton() {
  const router = useRouter()

  const handleLogin = async () => {
    // 登录逻辑
    await login()
    
    // 导航到仪表板
    router.push('/dashboard')
  }

  return <button onClick={handleLogin}>登录</button>
}

路由参数 ​

查询参数 ​

jsx
// URL: /search?q=react&category=tutorial
import { useRouter } from 'next/router'

export default function Search() {
  const router = useRouter()
  const { q, category } = router.query

  return (
    <div>
      <p>搜索词: {q}</p>
      <p>分类: {category}</p>
    </div>
  )
}

jsx
// pages/posts/[id].js
// URL: /posts/123

export async function getStaticProps({ params }) {
  const post = await fetchPost(params.id)

  const posts = await fetchPosts()
  const paths = posts.map(post => ({

路由组 ​

App Router 支持路由组，用于组织路由而不影响 URL：

app/
  (marketing)/
    about/
      page.js       → /about
    contact/
      page.js       → /contact
  (shop)/
    products/
      page.js       → /products
    cart/
      page.js       → /cart

并行路由 ​

App Router 支持并行路由，可以同时渲染多个页面：

app/
  @analytics/
    page.js
  @team/
    page.js
  layout.js
  page.js
jsx
// app/layout.js
export default function Layout({ children, analytics, team }) {
  return (
    <>
      {children}
      {analytics}
      {team}
    </>
  )
}

拦截路由 ​

拦截路由允许你在当前布局中显示来自应用程序其他部分的路由：

app/
  @modal/
    (.)photo/
      [id]/
        page.js
  photo/
    [id]/
      page.js

最佳实践 ​

  1. 使用 App Router：对于新项目，推荐使用 App Router
  2. 合理组织文件：使用文件夹组织相关页面
  3. 使用 Link 组件：避免使用 <a> 标签进行内部导航
  4. 预取优化：Link 组件会自动预取可见链接
  5. 错误处理：为每个路由段提供错误边界
