useReducer Hook ​

useReducer 是 React 提供的一个 Hook，用于处理复杂的状态逻辑。它是 useState 的替代方案，特别适合那些包含多个子值或者下一个状态依赖于前一个状态的复杂状态逻辑。

基本语法 ​

jsx
const [state, dispatch] = useReducer(reducer, initialState, init);

参数 ​

  • reducer: 一个函数，接收当前状态和动作，返回新的状态。格式为 (state, action) => newState
  • initialState: 初始状态
  • init (可选): 惰性初始化函数，用于计算初始状态

返回值 ​

useReducer 返回一个数组，包含两个元素：

  1. 当前的状态值（state）
  2. 派发动作的函数（dispatch）

工作原理 ​

useReducer 的工作原理类似于 Redux：

  1. 定义一个 reducer 函数，接收当前状态和动作，返回新的状态
  2. 使用 useReducer 初始化状态，获取当前状态和 dispatch 函数
  3. 通过调用 dispatch 函数并传入动作对象来触发状态更新
  4. React 调用 reducer 函数计算新的状态，并重新渲染组件

基本示例 ​

下面是一个简单的计数器示例：

jsx
import React, { useReducer } from 'react';

// 定义 reducer 函数
function counterReducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    case 'reset':
      return { count: 0 };
    default:
      return state;
  }
}

function Counter() {
  // 使用 useReducer 初始化状态
  const [state, dispatch] = useReducer(counterReducer, { count: 0 });

  return (
    <div>
      <p>当前计数: {state.count}</p>
      <button onClick={() => dispatch({ type: 'decrement' })}>减少</button>
      <button onClick={() => dispatch({ type: 'reset' })}>重置</button>
      <button onClick={() => dispatch({ type: 'increment' })}>增加</button>
    </div>
  );
}

复杂示例：待办事项列表 ​

下面是一个更复杂的待办事项列表示例：

jsx
import React, { useReducer, useState } from 'react';

// 定义 reducer 函数
function todoReducer(state, action) {
  switch (action.type) {
    case 'add':
      return {
        ...state,
        todos: [...state.todos, { 
          id: state.nextId, 
          text: action.payload, 
          completed: false 
        }],
        nextId: state.nextId + 1
      };
    case 'toggle':
      return {
        ...state,
        todos: state.todos.map(todo => 
          todo.id === action.payload 
            ? { ...todo, completed: !todo.completed } 
            : todo
        )
      };
    case 'delete':
      return {
        ...state,
        todos: state.todos.filter(todo => todo.id !== action.payload)
      };
    default:
      return state;
  }
}

function TodoList() {
  // 使用 useReducer 初始化状态
  const [state, dispatch] = useReducer(todoReducer, { 
    todos: [], 
    nextId: 1 
  });
  
  const [text, setText] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault();
    if (text.trim()) {
      dispatch({ type: 'add', payload: text });
      setText('');
    }
  };

  return (
    <div>
      <form onSubmit={handleSubmit}>
        <input
          value={text}
          onChange={(e) => setText(e.target.value)}
          placeholder="添加新任务"
        />
        <button type="submit">添加</button>
      </form>
      
      <ul>
        {state.todos.map(todo => (
          <li key={todo.id}>
            <span
              style={{ 
                textDecoration: todo.completed ? 'line-through' : 'none',
                cursor: 'pointer'
              }}
              onClick={() => dispatch({ type: 'toggle', payload: todo.id })}
            >
              {todo.text}
            </span>
            <button 
              onClick={() => dispatch({ type: 'delete', payload: todo.id })}
            >
              删除
            </button>
          </li>
        ))}
      </ul>
    </div>
  );
}

惰性初始化 ​

如果初始状态的计算比较复杂，可以使用惰性初始化来优化性能：

jsx
// 初始化函数
function init(initialCount) {
  return { count: initialCount };
}

function Counter({ initialCount }) {
  const [state, dispatch] = useReducer(counterReducer, initialCount, init);
  
  return (
    <div>
      <p>当前计数: {state.count}</p>
      <button onClick={() => dispatch({ type: 'decrement' })}>减少</button>
      <button onClick={() => dispatch({ type: 'reset' })}>重置</button>
      <button onClick={() => dispatch({ type: 'increment' })}>增加</button>
    </div>
  );
}

使用 TypeScript ​

在 TypeScript 中使用 useReducer 可以提供更好的类型安全：

tsx
// 定义状态和动作的类型
interface State {
  count: number;
}

type Action = 
  | { type: 'increment' }
  | { type: 'decrement' }
  | { type: 'reset' }
  | { type: 'set'; payload: number };

// 定义 reducer 函数
function counterReducer(state: State, action: Action): State {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    case 'reset':
      return { count: 0 };
    case 'set':
      return { count: action.payload };
    default:
      return state;
  }
}

function Counter() {
  const [state, dispatch] = useReducer(counterReducer, { count: 0 });
  
  return (
    <div>
      <p>当前计数: {state.count}</p>
      <button onClick={() => dispatch({ type: 'decrement' })}>减少</button>
      <button onClick={() => dispatch({ type: 'reset' })}>重置</button>
      <button onClick={() => dispatch({ type: 'increment' })}>增加</button>
      <button onClick={() => dispatch({ type: 'set', payload: 100 })}>设置为100</button>
    </div>
  );
}

useReducer 与 Context API 结合 ​

useReducer 与 Context API 结合可以实现类似 Redux 的全局状态管理：

jsx
import React, { createContext, useContext, useReducer } from 'react';

// 创建 Context
const TodoContext = createContext();

// 定义 reducer 函数
function todoReducer(state, action) {
  // 同上...
}

// 创建 Provider 组件
function TodoProvider({ children }) {
  const [state, dispatch] = useReducer(todoReducer, { todos: [], nextId: 1 });
  
  return (
    <TodoContext.Provider value={{ state, dispatch }}>
      {children}
    </TodoContext.Provider>
  );
}

// 创建自定义 Hook
function useTodo() {
  const context = useContext(TodoContext);
  if (!context) {
    throw new Error('useTodo must be used within a TodoProvider');
  }
  return context;
}

// 使用自定义 Hook 的组件
function TodoList() {
  const { state, dispatch } = useTodo();
  
  // 使用 state 和 dispatch...
}

// 应用组件
function App() {
  return (
    <TodoProvider>
      <TodoList />
      <OtherComponent />
    </TodoProvider>
  );
}

何时使用 useReducer ​

以下情况适合使用 useReducer：

  1. 复杂的状态逻辑 - 当状态包含多个子值或者状态之间有依赖关系时
  2. 多种更新方式 - 当状态有多种不同的更新方式时
  3. 状态转换 - 当状态的更新可以被清晰地表达为不同类型的行为时
  4. 深层组件树 - 当需要在深层组件树中更新状态时（结合 Context API）

useReducer 与 Redux 的比较 ​

useReducer 可以看作是 Redux 的简化版，适用于组件级别的状态管理。与 Redux 相比：

  • 优势：

    • 不需要额外的库
    • 更轻量级，适合中小型应用
    • 与 React 紧密集成

  • 劣势：

    • 没有中间件支持
    • 没有开发工具支持
    • 没有内置的状态持久化
    • 状态是局部的，而非全局的

最佳实践 ​

  1. 将 reducer 函数放在组件外部 - 这样可以提高可读性和可测试性
  2. 使用 TypeScript 定义状态和动作类型 - 提供更好的类型安全
  3. 使用 action creator 函数 - 封装 action 创建逻辑，提高可维护性
  4. 结合 Context API 实现全局状态管理 - 对于需要在多个组件间共享的状态
  5. 使用惰性初始化 - 对于复杂的初始状态计算

总结 ​

useReducer 是 React 中用于处理复杂状态逻辑的强大工具。它提供了一种更加结构化和可预测的方式来管理状态，特别适合那些包含多个子值或者下一个状态依赖于前一个状态的复杂状态逻辑。通过将状态更新逻辑集中在 reducer 函数中，useReducer 使得状态变化更加清晰和可维护。
