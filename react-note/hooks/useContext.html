useContext Hook ​

useContext 是 React 提供的一个 Hook，用于在组件树中共享数据，而无需通过 props 层层传递。它提供了一种无需为每层组件手动添加 props，就能在组件树间进行数据传递的方法。

基本语法 ​

jsx
const value = useContext(MyContext);

参数 ​

  • MyContext：一个由 React.createContext 创建的上下文对象。它不保持信息，而是作为信息的载体，声明了可以从组件获取或者给组件提供信息。

返回值 ​

  • 返回传递给上下文 Provider 的当前上下文值。这个值是只读的，如果上下文发生变化，React 会自动重新渲染读取该上下文的组件。

基本用法 ​

首先需要通过 createContext 创建一个上下文，然后使用创建的上下文组件包裹需要访问该上下文的组件，并传递值。被包裹的组件在任何层级都可以通过 useContext 获取上下文的值。

React 18 版本示例 ​

jsx
import React, { useContext, useState } from 'react';

// 创建上下文
const ThemeContext = React.createContext({
  theme: 'light',
  setTheme: () => {}
});

// 子组件
const Child = () => {
  // 获取上下文
  const themeContext = useContext(ThemeContext);
  
  const styles = {
    backgroundColor: themeContext.theme === 'light' ? 'white' : 'black',
    border: '1px solid red',
    width: 100 + 'px',
    height: 100 + 'px',
    color: themeContext.theme === 'light' ? 'black' : 'white'
  };
  
  return (
    <div>
      <div style={styles}>
        child
      </div>
    </div>
  );
};

// 父组件
const Parent = () => {
  // 获取上下文
  const themeContext = useContext(ThemeContext);
  
  const styles = {
    backgroundColor: themeContext.theme === 'light' ? 'white' : 'black',
    border: '1px solid red',
    width: 100 + 'px',
    height: 100 + 'px',
    color: themeContext.theme === 'light' ? 'black' : 'white'
  };
  
  return (
    <div>
      <div style={styles}>
        Parent
      </div>
      <Child />
    </div>
  );
};

// 应用组件
function App() {
  const [theme, setTheme] = useState('light');
  
  return (
    <div>
      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
        切换主题
      </button>
      <ThemeContext.Provider value={{ theme, setTheme }}>
        <Parent />
      </ThemeContext.Provider>
    </div>
  );
}

export default App;

注意事项 ​

Provider 的 value 属性 ​

使用上下文时，传递的属性必须为 value：

jsx
// ❌ 错误：prop 应该是 "value"
<ThemeContext theme={theme}>
  <Button />
</ThemeContext>

// ✅ 正确：传递 value 作为 prop
<ThemeContext value={theme}>
  <Button />
</ThemeContext>

使用多个上下文 ​

如果需要使用多个上下文，可以嵌套使用：

jsx
const ThemeContext = React.createContext({ theme: 'light' });
const UserContext = React.createContext({ user: null });

function App() {
  return (
    <ThemeContext.Provider value={{ theme: 'light' }}>
      <UserContext.Provider value={{ user: { name: 'John' } }}>
        <Content />
      </UserContext.Provider>
    </ThemeContext.Provider>
  );
}

function Content() {
  const theme = useContext(ThemeContext);
  const user = useContext(UserContext);
  
  return (
    <div>
      <p>Current theme: {theme.theme}</p>
      <p>Current user: {user.user.name}</p>
    </div>
  );
}

需要注意的是，如果使用相同的上下文多次，后面的会覆盖前面的：

jsx
const ThemeContext = React.createContext({ theme: 'light' });

function App() {
  return (
    <ThemeContext.Provider value={{ theme: 'light' }}>
      <ThemeContext.Provider value={{ theme: 'dark' }}> {/* 覆盖了上面的值 */}
        <Content />
      </ThemeContext.Provider>
    </ThemeContext.Provider>
  );
}

性能优化 ​

当上下文的值发生变化时，所有消费该上下文的组件都会重新渲染。为了避免不必要的渲染，可以采取以下措施：

  1. 拆分上下文：将不同的状态放在不同的上下文中，这样当某个状态变化时，只有使用该状态的组件会重新渲染。

  2. 使用 React.memo：结合 React.memo 可以避免子组件在上下文变化但实际使用的值没有变化时重新渲染。

  3. 使用 useMemo 优化上下文值：使用 useMemo 来记忆上下文值，只有当依赖项变化时才创建新的值。

jsx
function App() {
  const [theme, setTheme] = useState('light');
  const [user, setUser] = useState({ name: 'John' });
  
  // 使用 useMemo 优化上下文值
  const themeContextValue = useMemo(() => {
    return { theme, setTheme };
  }, [theme]);
  
  const userContextValue = useMemo(() => {
    return { user, setUser };
  }, [user]);
  
  return (
    <ThemeContext.Provider value={themeContextValue}>
      <UserContext.Provider value={userContextValue}>
        <Content />
      </UserContext.Provider>
    </ThemeContext.Provider>
  );
}

实际应用场景 ​

useContext 适用于以下场景：

  1. 主题切换：全局主题管理，如暗黑模式/亮色模式切换。
  2. 用户认证：在整个应用中共享用户登录状态和信息。
  3. 多语言支持：在应用中共享当前语言设置。
  4. 全局状态管理：作为简单的全局状态管理解决方案，对于复杂应用可能需要结合 useReducer 或使用 Redux 等状态管理库。
  5. 路由信息共享：共享当前路由信息，如当前页面、参数等。

最佳实践 ​

  1. 为上下文创建自定义 Hook：封装上下文的使用逻辑，提供更清晰的 API。
jsx
function useTheme() {
  const context = useContext(ThemeContext);
  if (context === undefined) {
    throw new Error('useTheme must be used within a ThemeProvider');
  }
  return context;
}

  1. 合理拆分上下文：不要将所有状态都放在一个上下文中，按照关注点分离原则拆分。

  2. 提供默认值：为上下文提供有意义的默认值，使组件在没有 Provider 包裹时也能正常工作。

  3. 考虑性能影响：上下文变化会导致所有消费该上下文的组件重新渲染，合理设计上下文结构和使用记忆化技术。
