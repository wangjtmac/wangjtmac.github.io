useId Hook ​

useId 是 React 18 引入的一个 Hook，用于生成唯一且稳定的 ID，主要用于可访问性（Accessibility）属性。它解决了在服务器端渲染（SSR）和客户端渲染之间保持 ID 一致性的问题。

基本语法 ​

jsx
const id = useId();

参数 ​

useId 不接受任何参数。

返回值 ​

返回一个唯一且稳定的字符串 ID，在组件的整个生命周期内保持不变。

使用场景 ​

useId 主要用于以下场景：

  1. 生成 HTML 元素的唯一 ID，特别是用于可访问性属性（如 aria-* 属性）
  2. 连接相关的 HTML 元素（如 <label> 和 <input>）
  3. 在服务器端渲染（SSR）和客户端渲染之间保持 ID 一致性

注意：useId 不是用来生成列表的 key 的，列表的 key 应该来自你的数据。

基本示例 ​

jsx
import React, { useId } from 'react';

function FormField() {
  const id = useId();
  
  return (
    <div>
      <label htmlFor={id}>用户名</label>
      <input id={id} type="text" />
    </div>
  );
}

在上面的例子中，useId 生成一个唯一的 ID，用于连接 label 和 input 元素，使得点击标签时可以自动聚焦到输入框。

生成多个相关 ID ​

如果你需要在同一个组件中为多个元素生成相关的 ID，可以在 useId 的基础上添加后缀：

jsx
import React, { useId } from 'react';

function ComplexForm() {
  const id = useId();
  const nameId = `${id}-name`;
  const emailId = `${id}-email`;
  const passwordId = `${id}-password`;
  
  return (
    <form>
      <div>
        <label htmlFor={nameId}>姓名</label>
        <input id={nameId} type="text" />
      </div>
      
      <div>
        <label htmlFor={emailId}>邮箱</label>
        <input id={emailId} type="email" />
      </div>
      
      <div>
        <label htmlFor={passwordId}>密码</label>
        <input id={passwordId} type="password" />
      </div>
    </form>
  );
}

实际应用示例 ​

1. 可访问性增强的表单组件 ​

jsx
import React, { useId } from 'react';

function AccessibleFormField({ label, type = 'text', error, ...props }) {
  const id = useId();
  const inputId = `${id}-input`;
  const errorId = `${id}-error`;
  
  return (
    <div>
      <label htmlFor={inputId}>{label}</label>
      
      <input
        id={inputId}
        type={type}
        aria-invalid={!!error}
        aria-describedby={error ? errorId : undefined}
        {...props}
      />
      
      {error && (
        <div id={errorId} className="error-message" role="alert">
          {error}
        </div>
      )}
    </div>
  );
}

2. 模态对话框 ​

jsx
import React, { useId } from 'react';

function Modal({ title, isOpen, onClose, children }) {
  const id = useId();
  const titleId = `${id}-title`;
  const descriptionId = `${id}-description`;
  
  if (!isOpen) return null;
  
  return (
    <div
      role="dialog"
      aria-modal="true"
      aria-labelledby={titleId}
      aria-describedby={descriptionId}
      className="modal-overlay"
    >
      <div className="modal-content">
        <div className="modal-header">
          <h2 id={titleId}>{title}</h2>
          <button
            onClick={onClose}
            aria-label="关闭"
            className="close-button"
          >
            ×
          </button>
        </div>
        
        <div id={descriptionId} className="modal-body">
          {children}
        </div>
      </div>
    </div>
  );
}

// 使用示例
function App() {
  const [isModalOpen, setIsModalOpen] = React.useState(false);
  
  return (
    <div>
      <button onClick={() => setIsModalOpen(true)}>
        打开模态框
      </button>
      
      <Modal
        title="重要通知"
        isOpen={isModalOpen}
        onClose={() => setIsModalOpen(false)}
      >
        <p>这是一个模态对话框的内容。</p>
        <button onClick={() => setIsModalOpen(false)}>
          确认
        </button>
      </Modal>
    </div>
  );
}

3. 折叠面板 ​

jsx
import React, { useId, useState } from 'react';

function Accordion({ title, children }) {
  const [isExpanded, setIsExpanded] = useState(false);
  const id = useId();
  const headingId = `${id}-heading`;
  const contentId = `${id}-content`;
  
  return (
    <div className="accordion">
      <h3 id={headingId}>
        <button
          aria-expanded={isExpanded}
          aria-controls={contentId}
          onClick={() => setIsExpanded(!isExpanded)}
          className="accordion-trigger"
        >
          {title}
          <span className="accordion-icon">
            {isExpanded ? '▼' : '▶'}
          </span>
        </button>
      </h3>
      
      <div
        id={contentId}
        role="region"
        aria-labelledby={headingId}
        hidden={!isExpanded}
        className="accordion-content"
      >
        {children}
      </div>
    </div>
  );
}

// 使用示例
function FAQSection() {
  return (
    <div>
      <h2>常见问题</h2>
      
      <Accordion title="什么是 React？">

        <p>Hooks 是 React 16.8 引入的新特性，它允许你在不编写类组件的情况下使用状态和其他 React 特性。</p>
      </Accordion>
      
      <Accordion title="什么是 useId？">
        <p>useId 是 React 18 引入的一个 Hook，用于生成唯一且稳定的 ID，主要用于可访问性属性。</p>
      </Accordion>
    </div>
  );
}

useId vs 随机 ID ​

你可能会想知道为什么不直接使用 Math.random() 或 UUID 库生成随机 ID。以下是使用 useId 的优势：

  1. 服务器和客户端一致性 - useId 在服务器端渲染（SSR）和客户端渲染之间生成相同的 ID，避免了水合（hydration）不匹配的问题
  2. 稳定性 - 即使组件重新渲染，useId 也会返回相同的 ID
  3. 无需外部依赖 - 不需要引入额外的 UUID 库
  4. 性能优化 - React 内部优化了 ID 的生成，避免了不必要的计算

注意事项 ​

1. 不要用于列表的 key ​

useId 不应该用于生成列表项的 key。列表的 key 应该来自你的数据，通常是数据的唯一标识符。

jsx
// ❌ 错误：使用 useId 生成列表的 key
function WrongList({ items }) {
  return (
    <ul>
      {items.map(item => {
        const id = useId(); // 错误用法
        return <li key={id}>{item.name}</li>;
      })}
    </ul>
  );
}

// ✅ 正确：使用数据中的唯一标识符作为 key
function CorrectList({ items }) {
  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}

2. ID 格式 ​

useId 生成的 ID 包含 : 字符（例如 r1:0），这是 React 内部使用的格式。虽然这种格式在 HTML 中是有效的，但在某些情况下（如 CSS 选择器）可能需要注意。

如果你需要在 CSS 选择器中使用这些 ID，可以考虑添加前缀：

jsx
const id = useId();
const safeId = `my-prefix-${id}`;

3. 同构渲染 ​

在同构渲染（服务器端渲染 + 客户端水合）的应用中，确保服务器和客户端使用相同版本的 React，否则可能会导致 ID 不匹配。

最佳实践 ​

1. 用于可访问性属性 ​

useId 最适合用于可访问性属性，如 htmlFor、aria-labelledby、aria-describedby 等。

jsx
function AccessibleInput() {
  const id = useId();
  
  return (
    <>
      <label htmlFor={id}>电子邮箱</label>
      <input id={id} type="email" />
    </>
  );
}

2. 为组件内的多个元素生成相关 ID ​

当需要为组件内的多个元素生成 ID 时，使用同一个 useId 调用并添加后缀，而不是多次调用 useId。

jsx
// ✅ 正确：使用一个 useId 调用并添加后缀
function GoodExample() {
  const id = useId();
  
  return (
    <div>
      <label htmlFor={`${id}-name`}>姓名</label>
      <input id={`${id}-name`} type="text" />
      
      <label htmlFor={`${id}-email`}>邮箱</label>
      <input id={`${id}-email`} type="email" />
    </div>
  );
}

// ❌ 不推荐：多次调用 useId
function BadExample() {
  const nameId = useId();
  const emailId = useId();
  
  return (
    <div>
      <label htmlFor={nameId}>姓名</label>
      <input id={nameId} type="text" />
      
      <label htmlFor={emailId}>邮箱</label>
      <input id={emailId} type="email" />
    </div>
  );
}

3. 在可重用组件中使用 ​

在可重用组件中使用 useId，确保即使同一个组件被多次使用，每个实例也有唯一的 ID。

jsx
function FormField({ label, type = 'text', ...props }) {
  const id = useId();
  
  return (
    <div>
      <label htmlFor={id}>{label}</label>
      <input id={id} type={type} {...props} />
    </div>
  );
}

// 使用多个实例
function Form() {
  return (
    <form>
      <FormField label="用户名" />
      <FormField label="邮箱" type="email" />
      <FormField label="密码" type="password" />
    </form>
  );
}

浏览器兼容性 ​

useId 是 React 18 引入的新特性，需要使用 React 18 或更高版本。它在所有支持 React 18 的浏览器中都能正常工作。

总结 ​

useId 是 React 18 引入的一个非常实用的 Hook，它解决了在服务器端渲染和客户端渲染之间保持 ID 一致性的问题。它主要用于生成可访问性属性的唯一 ID，特别适合用于连接相关的 HTML 元素，如 <label> 和 <input>。

通过使用 useId，你可以确保你的 React 应用在服务器端渲染和客户端渲染之间保持一致，同时提高应用的可访问性。记住，useId 不是用来生成列表的 key 的，列表的 key 应该来自你的数据。
