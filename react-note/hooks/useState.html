useState Hook ​

useState 是一个 React Hook，允许函数组件在内部管理状态。

基本概念 ​

组件通常需要根据交互更改屏幕上显示的内容，例如点击某个按钮更改值，或者输入文本框中的内容，这些值被称为状态值（state）。

语法 ​

jsx
const [state, setState] = useState(initialState);

参数 ​

  • initialState：状态的初始值，可以是任何类型的值（数字、字符串、布尔值、对象、数组等）。

返回值 ​

useState 返回一个数组，包含两个元素：

  1. 当前的状态值（state）
  2. 更新状态的函数（setState）

使用规则 ​

  • useState 只能在函数组件的顶层或自定义 Hook 中调用，不能在循环、条件语句或嵌套函数中调用。
  • 在严格模式下，React 会两次调用初始化函数，以帮助你找到意外的不纯性。这只是开发时的行为，不影响生产环境。

基本用法 ​

添加一个状态 ​

jsx
const Card = () => {
  let [index, setIndex] = useState(0);
  let [name, setName] = useState('小满');
  let [arr, setArr] = useState([1, 2, 3]);
}

按照惯例使用数组解构来命名状态变量，例如 [index, setIndex]。

更新状态 ​

要更新屏幕上的内容，调用 setState 函数并传入新的状态值：

jsx
function changeName() {
  setName('大满');
}

调用 setState 函数会触发组件的重新渲染。

完整示例 ​

基本数据类型 ​

jsx
import { useState } from "react";

function App() {
  let [name, setName] = useState('小满'); // 数字、布尔值、null、undefined 都可以直接赋值
  
  const handleClick = () => {
    setName('大满');
  }
  
  return (
    <>
      <button onClick={handleClick}>更改名称</button>
      <div>{name}</div>
    </>
  );
}

export default App;

复杂数据类型 ​

数组操作 ​

在React中，你需要将数组视为只读的，不可以直接修改原数组。下面是常见数组操作的参考：

避免使用 (会改变原始数组)推荐使用 (会返回一个新数组）
添加元素：push, unshiftconcat, [...arr] 展开语法
删除元素：pop, shift, splicefilter, slice
替换元素：splice, arr[i] = ...map
排序：reverse, sort先将数组复制一份
数组新增数据 ​
jsx
import { useState } from "react";

function App() {
  let [arr, setArr] = useState([1, 2, 3]);
  
  const handleClick = () => {
    setArr([...arr, 4]); // 末尾新增，使用扩展运算符
    // setArr([0, ...arr]); // 头部新增，使用扩展运算符
  }
  
  return (
    <>
      <button onClick={handleClick}>更改值</button>
      <div>{arr.join(', ')}</div>
    </>
  );
}

export default App;

指定位置插入元素 ​
jsx
import { useState } from "react";

function App() {
  let [arr, setArr] = useState([1, 2, 3]);
  
  const handleClick = () => {
    let startIndex = 0;
    let endIndex = 2;
    setArr([
      ...arr.slice(startIndex, endIndex),
      2.5,
      ...arr.slice(endIndex)
    ]);
  }
  
  return (
    <>
      <button onClick={handleClick}>更改值</button>
      <div>{arr.join(', ')}</div>
    </>
  );
}

export default App;
排序和旋转 ​
jsx
import { useState } from "react";

function App() {
  let [arr, setArr] = useState([1, 2, 3]);
  
  const handleClick = () => {
    let newList = [...arr].map(v => v + 1); // 拷贝到新数组
    newList.sort((a, b) => b - a);
    // newList.reverse(); // 旋转
    setArr(newList);
  }
  
  return (
    <>
      <button onClick={handleClick}>更改值</button>
      <div>{arr.join(', ')}</div>
    </>
  );
}

export default App;

对象操作 ​

useState 可以接受一个函数作为参数，这个函数只会在组件初始化时执行一次：

jsx
import { useState } from "react";

function App() {
  let [obj, setObject] = useState(() => {
    const date = new Date().getFullYear() + '-' + (new Date().getMonth() + 1) + '-' + new Date().getDate();
    return {
      date,
      name: '小满',
      age: 25
    };
  });
  
  const handleClick = () => {
    setObject({
      ...obj,
      name: '大满'
    });
    // 另一种写法
    // setObject(Object.assign({}, obj, { age: 26 }));
  }
  
  return (
    <>
      <button onClick={handleClick}>更改值</button>
      <div>日期：{obj.date}</div>
      <div>姓名：{obj.name}</div>
      <div>年龄：{obj.age}</div>
    </>
  );
}

export default App;

useState 更新机制 ​

异步更新 ​

useState 的更新函数是异步的：

jsx
import { useState } from "react";

function App() {
  let [index, setIndex] = useState(0);
  
  const handleClick = () => {
    setIndex(index + 1);
    console.log(index); // 输出的是更新前的值
  }
  
  return (
    <>
      <h1>Index: {index}</h1>
      <button onClick={handleClick}>更改值</button>
    </>
  );
}

export default App;

批处理机制 ​

当多次以相同的操作更新状态时，React 会进行比较，如果值相同，则会屏蔽后续的更新行为，这是一种内置的"防抖"功能：

jsx
import { useState } from "react";

function App() {
  let [index, setIndex] = useState(0);
  
  const handleClick = () => {
    setIndex(index + 1); // 1
    setIndex(index + 1); // 还是 1，因为 index 的值没有变化
    setIndex(index + 1); // 还是 1，同理
  }
  
  return (
    <>
      <h1>Index: {index}</h1>
      <button onClick={handleClick}>更改值</button>
    </>
  );
}

export default App;

要解决这个问题，可以使用函数式更新：

jsx
import { useState } from "react";

function App() {
  let [index, setIndex] = useState(0);
  
  const handleClick = () => {
    setIndex(prevIndex => prevIndex + 1); // 1
    setIndex(prevIndex => prevIndex + 1); // 2
    setIndex(prevIndex => prevIndex + 1); // 3
  }
  
  return (
    <>
      <h1>Index: {index}</h1>
      <button onClick={handleClick}>更改值</button>
    </>
  );
}

export default App;

函数式更新的工作方式：

  1. prevIndex => prevIndex + 1 接收 0 作为当前状态，返回 1 作为下一个状态。
  2. prevIndex => prevIndex + 1 接收 1 作为当前状态，返回 2 作为下一个状态。
  3. prevIndex => prevIndex + 1 接收 2 作为当前状态，返回 3 作为下一个状态。

按照惯例，通常将状态参数命名为 prevState 或状态变量名称的第一个字母（如 prevIndex）。

最佳实践 ​

  1. 保持状态最小化：只存储必要的状态，能从现有状态计算出的值不需要单独存储。
  2. 避免冗余状态：如果某些状态可以从 props 或其他状态派生，则不需要在 useState 中存储。
  3. 避免深层嵌套状态：复杂的状态结构可能导致更新困难，考虑使用 useReducer 或状态管理库。
  4. 使用函数式更新：当新状态依赖于之前的状态时，使用函数式更新可以避免闭包陷阱。
  5. 延迟初始化：对于计算成本高的初始状态，使用函数形式可以避免每次渲染都重新计算。
