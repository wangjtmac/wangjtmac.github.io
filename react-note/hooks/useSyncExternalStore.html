useSyncExternalStore Hook ​

useSyncExternalStore 是 React 18 引入的一个 Hook，主要用于订阅外部数据源，并确保在并发渲染特性（如 Suspense 和 Time Slicing）下数据的一致性。

基本语法 ​

jsx
const state = useSyncExternalStore(subscribe, getSnapshot, getServerSnapshot);

参数 ​

  • subscribe: 一个函数，接收一个回调函数作为参数，当存储值变化时调用该回调函数，并返回一个清理函数
  • getSnapshot: 一个函数，返回存储的当前值
  • getServerSnapshot (可选): 一个函数，返回服务器渲染期间使用的存储值

返回值 ​

返回存储的当前值。

工作原理 ​

useSyncExternalStore 的工作原理如下：

  1. 组件渲染时，调用 getSnapshot 获取当前状态
  2. 通过 subscribe 订阅状态变化
  3. 当状态变化时，组件会重新渲染
  4. 在组件卸载时，自动取消订阅

这个 Hook 的特殊之处在于它能确保在并发渲染期间数据的一致性，避免出现撕裂（tearing）问题。

基本示例 ​

jsx
import { useSyncExternalStore } from 'react';

// 创建一个简单的 store
const createStore = (initialState) => {
  let state = initialState;
  const listeners = new Set();
  
  const subscribe = (listener) => {
    listeners.add(listener);
    return () => listeners.delete(listener);
  };
  
  const getSnapshot = () => state;
  
  const setState = (newState) => {
    state = newState;
    listeners.forEach(listener => listener());
  };
  
  return {
    subscribe,
    getSnapshot,
    setState
  };
};

// 创建一个计数器 store
const counterStore = createStore(0);

function Counter() {
  // 使用 useSyncExternalStore 订阅 store
  const count = useSyncExternalStore(
    counterStore.subscribe,
    counterStore.getSnapshot
  );
  
  return (
    <div>
      <p>计数: {count}</p>
      <button onClick={() => counterStore.setState(count + 1)}>
        增加
      </button>
    </div>
  );
}

使用场景 ​

1. 订阅第三方状态管理库 ​

jsx
import { useSyncExternalStore } from 'react';
import { createStore } from 'redux';

// 创建 Redux store
const reduxStore = createStore((state = 0, action) => {
  switch (action.type) {
    case 'INCREMENT':
      return state + 1;
    case 'DECREMENT':
      return state - 1;
    default:
      return state;
  }
});

// 自定义 Hook 连接 Redux
function useReduxStore(selector) {
  return useSyncExternalStore(
    reduxStore.subscribe,
    () => selector(reduxStore.getState()),
    () => selector(reduxStore.getState())
  );
}

function ReduxCounter() {
  const count = useReduxStore(state => state);
  
  return (
    <div>
      <p>Redux 计数: {count}</p>
      <button onClick={() => reduxStore.dispatch({ type: 'INCREMENT' })}>
        增加
      </button>
      <button onClick={() => reduxStore.dispatch({ type: 'DECREMENT' })}>
        减少
      </button>
    </div>
  );
}

2. 订阅浏览器 API ​

jsx
import { useSyncExternalStore } from 'react';

// 创建订阅窗口大小变化的 Hook
function useWindowSize() {
  const getSnapshot = () => ({
    width: window.innerWidth,
    height: window.innerHeight
  });
  
  const subscribe = (callback) => {
    window.addEventListener('resize', callback);
    return () => window.removeEventListener('resize', callback);
  };
  
  return useSyncExternalStore(subscribe, getSnapshot);
}

function WindowSizeDisplay() {
  const { width, height } = useWindowSize();
  
  return (
    <div>
      <p>窗口宽度: {width}px</p>
      <p>窗口高度: {height}px</p>
    </div>
  );
}

3. 订阅 WebSocket ​

jsx
import { useSyncExternalStore } from 'react';

// 创建 WebSocket store
function createWebSocketStore(url) {
  let data = null;
  let socket = null;
  const listeners = new Set();
  
  const connect = () => {
    if (socket) return;
    
    socket = new WebSocket(url);
    
    socket.addEventListener('message', (event) => {
      data = JSON.parse(event.data);
      listeners.forEach(listener => listener());
    });
    
    socket.addEventListener('close', () => {
      socket = null;
    });
  };
  
  const subscribe = (listener) => {
    connect();
    listeners.add(listener);
    
    return () => {
      listeners.delete(listener);
      if (listeners.size === 0) {
        socket?.close();
        socket = null;
      }
    };
  };
  
  const getSnapshot = () => data;
  
  return {
    subscribe,
    getSnapshot
  };
}

// 使用 WebSocket store
const priceStore = createWebSocketStore('wss://example.com/prices');

function StockPriceDisplay() {
  const price = useSyncExternalStore(
    priceStore.subscribe,
    priceStore.getSnapshot
  );
  
  if (!price) return <div>加载中...</div>;
  
  return (
    <div>
      <h2>实时价格</h2>
      <p>{price.symbol}: {price.value}</p>
      <p>更新时间: {new Date(price.timestamp).toLocaleTimeString()}</p>
    </div>
  );
}

服务端渲染支持 ​

useSyncExternalStore 提供了对服务端渲染的支持，通过第三个参数 getServerSnapshot：

jsx
function useWindowSize() {
  const getSnapshot = () => ({
    width: window.innerWidth,
    height: window.innerHeight
  });
  
  const subscribe = (callback) => {
    window.addEventListener('resize', callback);
    return () => window.removeEventListener('resize', callback);
  };
  
  // 服务端渲染时使用的默认值
  const getServerSnapshot = () => ({
    width: 1024,
    height: 768
  });
  
  return useSyncExternalStore(subscribe, getSnapshot, getServerSnapshot);
}

与其他状态管理方案的比较 ​

特性useSyncExternalStoreuseStateuseReducer第三方库 (Redux等)

最佳实践 ​

  1. 分离关注点 - 将数据存储逻辑与组件渲染逻辑分离
  2. 避免频繁更新 - 确保外部存储不会触发过于频繁的更新
  3. 提供服务端渲染支持 - 始终提供 getServerSnapshot 函数以支持 SSR
  4. 优化订阅函数 - 确保订阅函数仅在必要时添加和移除监听器
  5. 使用选择器 - 只订阅组件实际需要的数据部分

注意事项 ​

  1. useSyncExternalStore 主要用于库作者，而不是应用开发者
  2. 对于大多数应用场景，使用 useState、useReducer 或成熟的状态管理库更合适
  3. 当使用 React 18 的并发特性时，如果需要订阅外部数据源，应该优先考虑使用 useSyncExternalStore
  4. 确保 getSnapshot 返回的是不可变数据，或者在每次状态变化时返回新的引用

总结 ​

useSyncExternalStore 是 React 18 引入的一个专门用于安全订阅外部数据源的 Hook。它解决了在并发渲染模式下可能出现的数据不一致问题，特别适用于需要与外部系统集成的场景。虽然大多数应用开发者可能不需要直接使用它，但了解它的工作原理对于构建可靠的 React 应用仍然很有价值。
