useMemo Hook ​

useMemo 是 React 的一个 Hook，它允许你在组件重新渲染之间"记忆"计算结果。当依赖项没有变化时，useMemo 会返回上一次计算的结果，而不是重新执行计算函数。这对于避免在每次渲染时进行昂贵的计算特别有用。

基本语法 ​

jsx
const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);

参数 ​

  • 计算函数: 第一个参数是一个函数，返回需要被记忆的值
  • 依赖项数组: 第二个参数是依赖项数组，只有当依赖项变化时，计算函数才会重新执行

返回值 ​

返回计算函数的执行结果，该结果会被记忆，只有当依赖项变化时才会重新计算。

使用场景 ​

1. 避免昂贵的计算 ​

当组件中有计算量大的操作时，可以使用 useMemo 避免在每次渲染时都重新计算。

jsx
import React, { useState, useMemo } from 'react';

function ExpensiveCalculation({ number }) {
  // 使用 useMemo 记忆计算结果
  const factorial = useMemo(() => {
    console.log('计算阶乘...');
    let result = 1;
    for (let i = 1; i <= number; i++) {
      result *= i;
    }
    return result;
  }, [number]); // 只有当 number 变化时才会重新计算

  return <div>阶乘结果: {factorial}</div>;
}

function App() {
  const [number, setNumber] = useState(5);
  const [count, setCount] = useState(0);

  return (
    <div>
      <input 
        type="number" 
        value={number} 
        onChange={e => setNumber(parseInt(e.target.value))} 
      />
      <ExpensiveCalculation number={number} />
      <button onClick={() => setCount(count + 1)}>点击计数: {count}</button>
    </div>
  );
}

在上面的例子中，即使点击按钮增加计数导致 App 组件重新渲染，阶乘计算也不会重新执行，因为 number 没有变化。

2. 保持引用相等性 ​

useMemo 还可以用来保持对象引用的稳定性，这对于避免不必要的子组件重新渲染特别有用。

jsx
import React, { useState, useMemo } from 'react';

// 使用 React.memo 包裹子组件，只有当 props 变化时才会重新渲染
const ChildComponent = React.memo(({ user }) => {
  console.log('子组件渲染');
  return <div>{user.name} - {user.age}岁</div>;
});

function ParentComponent() {
  const [name, setName] = useState('张三');
  const [age, setAge] = useState(25);
  const [count, setCount] = useState(0);
  
  // 不使用 useMemo - 每次渲染都会创建新的对象引用
  // const user = { name, age };
  
  // 使用 useMemo - 只有当 name 或 age 变化时才会创建新的对象引用
  const user = useMemo(() => {
    return { name, age };
  }, [name, age]);
  
  return (
    <div>
      <input value={name} onChange={e => setName(e.target.value)} />
      <input type="number" value={age} onChange={e => setAge(parseInt(e.target.value))} />
      <ChildComponent user={user} />
      <button onClick={() => setCount(count + 1)}>点击计数: {count}</button>
    </div>
  );
}

在这个例子中，如果不使用 useMemo，每次点击按钮增加计数时，ParentComponent 重新渲染都会创建一个新的 user 对象，导致 ChildComponent 也重新渲染，即使 name 和 age 没有变化。使用 useMemo 后，只有当 name 或 age 变化时才会创建新的 user 对象，从而避免了不必要的子组件重新渲染。

实际应用示例 ​

复杂数据过滤和排序 ​

jsx
import React, { useState, useMemo } from 'react';

function ProductList() {
  const [products, setProducts] = useState([
    { id: 1, name: '手机', price: 2999, category: '电子产品' },
    { id: 2, name: '笔记本', price: 5999, category: '电子产品' },
    { id: 3, name: '书籍', price: 39, category: '图书' },
    { id: 4, name: '耳机', price: 599, category: '电子产品' },
    { id: 5, name: '杯子', price: 29, category: '日用品' }
  ]);
  
  const [category, setCategory] = useState('全部');
  const [sortBy, setSortBy] = useState('默认');
  const [searchTerm, setSearchTerm] = useState('');
  
  // 使用 useMemo 记忆过滤和排序的结果
  const filteredAndSortedProducts = useMemo(() => {
    console.log('过滤和排序产品...');
    
    // 先过滤
    let result = products;
    
    if (category !== '全部') {
      result = result.filter(product => product.category === category);
    }
    
    if (searchTerm) {
      result = result.filter(product => 
        product.name.toLowerCase().includes(searchTerm.toLowerCase())
      );
    }
    
    // 再排序
    if (sortBy === '价格升序') {
      result = [...result].sort((a, b) => a.price - b.price);
    } else if (sortBy === '价格降序') {
      result = [...result].sort((a, b) => b.price - a.price);
    }
    
    return result;
  }, [products, category, sortBy, searchTerm]);
  
  return (
    <div>
      <div>
        <input 
          type="text" 
          placeholder="搜索产品" 
          value={searchTerm}
          onChange={e => setSearchTerm(e.target.value)}
        />
        
        <select value={category} onChange={e => setCategory(e.target.value)}>
          <option value="全部">全部分类</option>
          <option value="电子产品">电子产品</option>
          <option value="图书">图书</option>
          <option value="日用品">日用品</option>
        </select>
        
        <select value={sortBy} onChange={e => setSortBy(e.target.value)}>
          <option value="默认">默认排序</option>
          <option value="价格升序">价格升序</option>
          <option value="价格降序">价格降序</option>
        </select>
      </div>
      
      <ul>
        {filteredAndSortedProducts.map(product => (
          <li key={product.id}>
            {product.name} - ¥{product.price} ({product.category})
          </li>
        ))}
      </ul>
    </div>
  );
}

计算派生状态 ​

jsx
import React, { useState, useMemo } from 'react';

function Dashboard() {
  const [sales, setSales] = useState([
    { id: 1, amount: 100, date: '2023-01-15' },
    { id: 2, amount: 200, date: '2023-01-20' },
    { id: 3, amount: 150, date: '2023-02-05' },
    { id: 4, amount: 300, date: '2023-02-15' },
    { id: 5, amount: 250, date: '2023-03-10' }
  ]);
  
  // 使用 useMemo 计算统计数据
  const statistics = useMemo(() => {
    console.log('计算统计数据...');
    
    // 计算总销售额
    const totalSales = sales.reduce((sum, sale) => sum + sale.amount, 0);
    
    // 按月份分组
    const monthlySales = sales.reduce((acc, sale) => {
      const month = sale.date.substring(0, 7); // 提取年-月
      if (!acc[month]) {
        acc[month] = 0;
      }
      acc[month] += sale.amount;
      return acc;
    }, {});
    
    // 计算平均销售额
    const averageSale = totalSales / sales.length;
    
    // 找出最高销售额
    const maxSale = Math.max(...sales.map(sale => sale.amount));
    
    return {
      totalSales,
      monthlySales,
      averageSale,
      maxSale
    };
  }, [sales]);
  
  return (
    <div>
      <h2>销售统计</h2>
      <p>总销售额: ¥{statistics.totalSales}</p>
      <p>平均销售额: ¥{statistics.averageSale.toFixed(2)}</p>
      <p>最高销售额: ¥{statistics.maxSale}</p>
      
      <h3>月度销售</h3>
      <ul>
        {Object.entries(statistics.monthlySales).map(([month, amount]) => (
          <li key={month}>
            {month}: ¥{amount}
          </li>
        ))}
      </ul>
      
      {/* 添加新销售记录的表单 */}
    </div>
  );
}

useMemo vs useCallback ​

useMemo 和 useCallback 都是用于性能优化的 Hook，但它们有一些关键区别：

  • useMemo 记忆一个计算结果（可以是任何值）
  • useCallback 记忆一个函数
  • useCallback(fn, deps) 等同于 useMemo(() => fn, deps)
jsx
// 使用 useCallback 记忆函数
const memoizedCallback = useCallback(() => {
  doSomething(a, b);
}, [a, b]);

// 等同于使用 useMemo
const memoizedCallback = useMemo(() => {
  return () => {
    doSomething(a, b);
  };
}, [a, b]);

注意事项 ​

  1. 不要过度使用：只对计算量大的操作或需要保持引用稳定性的对象使用 useMemo，因为它也会增加内存使用。

  2. 依赖项要完整：确保依赖数组包含所有创建函数中使用的变量，否则可能会导致过时的值。

  3. 不要在 useMemo 中执行副作用：useMemo 的创建函数应该是纯函数，不应该有副作用。对于副作用，应该使用 useEffect。

  4. 延迟初始化：useMemo 也可以用于延迟初始化复杂对象。

jsx
// 延迟初始化复杂对象
const complexObject = useMemo(() => {
  return createComplexObject();
}, []);

性能优化建议 ​

  1. 测量后优化：在应用 useMemo 之前，先确认是否真的存在性能问题。不必要的优化可能会使代码更复杂而没有明显的性能提升。

  2. 考虑替代方案：有时，将计算逻辑移到组件外部或使用状态提升可能比使用 useMemo 更合适。

  3. 结合 React.memo：当传递对象给使用 React.memo 包裹的子组件时，使用 useMemo 可以获得最大的性能提升。

总结 ​

useMemo 是一个强大的性能优化工具，它可以帮助你避免不必要的计算和渲染。正确使用 useMemo 可以显著提高 React 应用的性能，特别是在处理计算密集型操作或需要保持引用稳定性的场景中。

然而，像所有的优化工具一样，useMemo 应该谨慎使用，只在真正需要的地方应用，避免过度优化导致代码复杂性增加而没有带来明显的性能提升。
