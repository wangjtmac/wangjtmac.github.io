useLayoutEffect Hook ​

useLayoutEffect 是 React 提供的一个 Hook，其函数签名与 useEffect 完全相同，但它会在所有 DOM 变更之后同步调用。它可以用来读取 DOM 布局并同步触发重渲染。

基本语法 ​

jsx
useLayoutEffect(setup, dependencies);

参数 ​

  • setup: 包含副作用逻辑的函数，可以返回一个清理函数
  • dependencies: 依赖项数组，当数组中的值发生变化时，副作用函数会重新执行

返回值 ​

useLayoutEffect 没有返回值。

useLayoutEffect vs useEffect ​

useLayoutEffect 和 useEffect 的区别主要在于执行时机：

特性useLayoutEffectuseEffect
执行时机DOM 更新后同步执行DOM 更新后异步执行
浏览器绘制在浏览器绘制之前执行在浏览器绘制之后执行
阻塞渲染会阻塞浏览器绘制不会阻塞浏览器绘制
适用场景需要在视觉上同步的 DOM 操作大多数副作用

执行顺序 ​

React 组件更新的执行顺序如下：

  1. React 更新 DOM
  2. useLayoutEffect 执行
  3. 浏览器绘制屏幕
  4. useEffect 执行

基本示例 ​

jsx
import React, { useState, useLayoutEffect, useEffect } from 'react';

function Example() {
  const [count, setCount] = useState(0);
  
  // 在 DOM 更新后同步执行
  useLayoutEffect(() => {
    console.log('useLayoutEffect 执行');
    // 这里的代码会在浏览器绘制之前执行
    // 如果在这里修改 DOM，用户不会看到中间状态
  }, [count]);
  
  // 在浏览器绘制后异步执行
  useEffect(() => {
    console.log('useEffect 执行');
    // 这里的代码会在浏览器绘制之后执行
  }, [count]);
  
  return (
    <div>
      <p>当前计数: {count}</p>
      <button onClick={() => setCount(count + 1)}>增加</button>
    </div>
  );
}

使用场景 ​

1. 测量 DOM 节点 ​

jsx
import React, { useState, useLayoutEffect, useRef } from 'react';

function MeasureExample() {
  const [height, setHeight] = useState(0);
  const ref = useRef(null);
  
  useLayoutEffect(() => {
    if (ref.current) {
      // 在 DOM 更新后立即测量高度
      const measuredHeight = ref.current.getBoundingClientRect().height;
      setHeight(measuredHeight);
    }
  }, []);
  
  return (
    <div>
      <div ref={ref}>
        一些内容，可能高度不确定...
      </div>
      <p>上面元素的高度是: {height}px</p>
    </div>
  );
}

2. 防止闪烁 ​

jsx
import React, { useState, useLayoutEffect } from 'react';

function NoFlickerExample() {
  const [position, setPosition] = useState({ left: 0, top: 0 });
  
  // 点击时更新位置
  const handleClick = () => {
    setPosition({ left: Math.random() * 400, top: Math.random() * 400 });
  };
  
  // 使用 useLayoutEffect 确保位置更新不会闪烁
  useLayoutEffect(() => {
    // 如果使用 useEffect，可能会看到元素先在旧位置闪现一下
    const element = document.getElementById('moving-element');
    if (element) {
      element.style.left = `${position.left}px`;
      element.style.top = `${position.top}px`;
    }
  }, [position]);
  
  return (
    <div style={{ position: 'relative', height: '500px', border: '1px solid black' }}>
      <div
        id="moving-element"
        style={{
          position: 'absolute',
          width: '50px',
          height: '50px',
          backgroundColor: 'red',
          cursor: 'pointer'
        }}
        onClick={handleClick}
      />
      <p>点击红色方块移动它</p>
    </div>
  );
}

3. 动画初始化 ​

jsx
import React, { useState, useLayoutEffect, useRef } from 'react';

function AnimationExample() {
  const [show, setShow] = useState(false);
  const boxRef = useRef(null);
  
  // 切换显示状态
  const toggleShow = () => {
    setShow(!show);
  };
  
  useLayoutEffect(() => {
    if (show && boxRef.current) {
      // 初始化动画前设置起始状态
      const box = boxRef.current;
      box.style.opacity = '0';
      box.style.transform = 'scale(0.8)';
      
      // 强制浏览器重排
      box.getBoundingClientRect();
      
      // 设置过渡效果
      box.style.transition = 'opacity 500ms, transform 500ms';
      box.style.opacity = '1';
      box.style.transform = 'scale(1)';
    }
  }, [show]);
  
  return (
    <div>
      <button onClick={toggleShow}>
        {show ? '隐藏' : '显示'}
      </button>
      
      {show && (
        <div
          ref={boxRef}
          style={{
            width: '200px',
            height: '200px',
            backgroundColor: 'blue',
            margin: '20px'
          }}
        />
      )}
    </div>
  );
}

性能考虑 ​

由于 useLayoutEffect 会在浏览器绘制之前同步执行，如果其中的操作耗时较长，可能会导致页面渲染延迟，影响用户体验。因此，应该遵循以下原则：

  1. 默认使用 useEffect，除非确实需要在绘制之前进行 DOM 操作
  2. 在 useLayoutEffect 中保持操作简洁高效
  3. 避免在 useLayoutEffect 中执行复杂计算或网络请求

服务端渲染注意事项 ​

在服务端渲染环境中，useLayoutEffect 和 useEffect 都不会在服务端执行。React 会在客户端首次渲染时执行这些 effect。

如果你的代码依赖于 useLayoutEffect 在服务端渲染时可能会导致警告。为了解决这个问题，可以：

  1. 将逻辑移到 useEffect 中（如果可能的话）
  2. 使用条件渲染，确保包含 useLayoutEffect 的组件只在客户端渲染
  3. 创建一个自定义 Hook，根据环境使用不同的 effect
jsx
import { useEffect, useLayoutEffect } from 'react';

// 根据环境选择合适的 effect
const useIsomorphicLayoutEffect = typeof window !== 'undefined' 
  ? useLayoutEffect 
  : useEffect;

// 使用自定义 Hook
function MyComponent() {
  useIsomorphicLayoutEffect(() => {
    // 这里的代码在客户端会使用 useLayoutEffect
    // 在服务端会使用 useEffect（实际上不会执行）
  }, []);
  
  return <div>...</div>;
}

常见陷阱 ​

1. 无限循环 ​

如果在 useLayoutEffect 中修改了依赖项中的状态，可能会导致无限循环：

jsx
// 错误示例
function BadExample() {
  const [width, setWidth] = useState(0);
  
  useLayoutEffect(() => {
    // 这会导致无限循环！
    setWidth(document.body.clientWidth);
  }, [width]); // width 变化会触发 effect，effect 又会改变 width
  
  return <div>宽度: {width}</div>;
}

// 正确示例
function GoodExample() {
  const [width, setWidth] = useState(0);
  
  useLayoutEffect(() => {
    setWidth(document.body.clientWidth);
  }, []); // 只在挂载时执行一次
  
  return <div>宽度: {width}</div>;
}

2. 过度使用 ​

过度使用 useLayoutEffect 可能会导致性能问题：

jsx
// 不好的做法
function SlowComponent() {
  useLayoutEffect(() => {
    // 复杂计算或不必要的 DOM 操作
    for (let i = 0; i < 10000; i++) {
      // 一些耗时操作...
    }
  });
  
  return <div>...</div>;
}

// 更好的做法
function FastComponent() {
  useEffect(() => {
    // 将非必要的同步操作移到 useEffect 中
    for (let i = 0; i < 10000; i++) {
      // 一些耗时操作...
    }
  });
  
  return <div>...</div>;
}

最佳实践 ​

  1. 默认使用 useEffect - 除非确实需要在绘制之前进行 DOM 操作，否则应该使用 useEffect

  2. 针对视觉更新使用 useLayoutEffect - 当需要在用户看到更新之前进行 DOM 测量或修改时使用 useLayoutEffect

  3. 保持简洁 - 在 useLayoutEffect 中保持操作简洁高效，避免复杂计算

  4. 考虑服务端渲染 - 在服务端渲染环境中，使用条件渲染或自定义 Hook 来避免警告

  5. 清理副作用 - 记得返回清理函数以避免内存泄漏

jsx
useLayoutEffect(() => {
  // 设置某些东西
  const handler = () => {
    // 处理某些事件
  };
  window.addEventListener('resize', handler);
  
  // 返回清理函数
  return () => {
    window.removeEventListener('resize', handler);
  };
}, []);

总结 ​

useLayoutEffect 是一个强大的 Hook，用于在 DOM 更新后、浏览器绘制前同步执行代码。它特别适用于需要在视觉上同步的 DOM 操作，如测量布局、防止闪烁和初始化动画。

然而，由于其同步特性，useLayoutEffect 可能会阻塞浏览器绘制，影响性能。因此，除非确实需要在绘制之前进行 DOM 操作，否则应该使用 useEffect。

通过理解 useLayoutEffect 和 useEffect 的区别，以及它们各自的适用场景，你可以更好地优化 React 应用的性能和用户体验。
