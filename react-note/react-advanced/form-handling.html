表单处理 ​

表单是 Web 应用中收集用户输入的重要方式。在 React 中，表单处理有一些特殊的考虑，因为 HTML 表单元素天生就保持一些内部状态。本文将介绍 React 中处理表单的各种方法和最佳实践。

受控组件与非受控组件 ​

React 中处理表单的主要方式有两种：受控组件和非受控组件。

受控组件 ​

在受控组件中，表单数据由 React 组件的 state 处理。每当表单字段发生变化时，都会调用处理函数来更新组件状态。

jsx
import React, { useState } from 'react';

function ControlledForm() {
  const [formData, setFormData] = useState({
    username: '',
    email: ''
  });
  
  const handleChange = (e) => {
    const { name, value } = e.target;
    setFormData({
      ...formData,
      [name]: value
    });
  };
  
  const handleSubmit = (e) => {
    e.preventDefault();
    console.log('表单数据:', formData);
    // 可以在这里发送数据到服务器
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label htmlFor="username">用户名:</label>
        <input
          type="text"
          id="username"
          name="username"
          value={formData.username}
          onChange={handleChange}
        />
      </div>
      
      <div>
        <label htmlFor="email">电子邮件:</label>
        <input
          type="email"
          id="email"
          name="email"
          value={formData.email}
          onChange={handleChange}
        />
      </div>
      
      <button type="submit">提交</button>
    </form>
  );
}

受控组件的特点 ​

  1. 单一数据源：表单数据存储在组件的 state 中
  2. 即时验证：可以在用户输入时进行验证
  3. 条件禁用：可以根据表单状态禁用提交按钮
  4. 格式化输入：可以在用户输入时格式化或过滤输入值

非受控组件 ​

非受控组件中，表单数据由 DOM 本身处理。我们使用 ref 来获取表单值，而不是为每个状态更新编写事件处理程序。

jsx
import React, { useRef } from 'react';

function UncontrolledForm() {
  const usernameRef = useRef(null);
  const emailRef = useRef(null);
  
  const handleSubmit = (e) => {
    e.preventDefault();
    const formData = {
      username: usernameRef.current.value,
      email: emailRef.current.value
    };
    console.log('表单数据:', formData);
    // 可以在这里发送数据到服务器
  };
  
  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label htmlFor="username">用户名:</label>
        <input
          type="text"
          id="username"
          name="username"
          ref={usernameRef}
          defaultValue="默认用户名"
        />
      </div>
      
      <div>
        <label htmlFor="email">电子邮件:</label>
        <input
          type="email"
          id="email"
          name="email"
          ref={emailRef}
        />
      </div>
      
      <button type="submit">提交</button>
    </form>
  );
}

非受控组件的特点 ​

  1. 更少的代码：不需要为每个输入字段编写事件处理程序
  2. 更容易集成：与非 React 代码集成更容易
  3. 适用于特定场景：如文件输入等特殊表单元素

表单验证 ​

表单验证是确保用户输入有效数据的重要步骤。在 React 中，可以实现客户端验证来提供即时反馈。

基本表单验证 ​

jsx
const [formData, setFormData] = useState({
  username: '',
  email: '',
  password: ''
});

const [errors, setErrors] = useState({
  username: '',
  email: '',
  password: ''
});

const validateForm = () => {
  let isValid = true;
  const newErrors = {
    username: '',
    email: '',
    password: ''
  };
  
  // 验证用户名
  if (!formData.username.trim()) {
    newErrors.username = '用户名是必填的';
    isValid = false;
  } else if (formData.username.length < 3) {
    newErrors.username = '用户名至少需要3个字符';
    isValid = false;
  }
  
  // 验证电子邮件
  if (!formData.email.trim()) {
    newErrors.email = '电子邮件是必填的';
    isValid = false;
  } else if (!/\S+@\S+\.\S+/.test(formData.email)) {
    newErrors.email = '请输入有效的电子邮件地址';
    isValid = false;
  }
  
  // 验证密码
  if (!formData.password) {
    newErrors.password = '密码是必填的';
    isValid = false;
  } else if (formData.password.length < 6) {
    newErrors.password = '密码至少需要6个字符';
    isValid = false;
  }
  
  setErrors(newErrors);
  return isValid;
};

const handleSubmit = (e) => {
  e.preventDefault();
  if (validateForm()) {
    console.log('表单提交成功:', formData);
    // 提交表单
  }
};

实时验证 ​

可以在用户输入时进行验证，提供即时反馈：

jsx
const handleChange = (e) => {
  const { name, value } = e.target;
  setFormData({
    ...formData,
    [name]: value
  });
  
  // 实时验证
  let errorMessage = '';
  
  switch (name) {
    case 'username':
      if (value && value.length < 3) {
        errorMessage = '用户名至少需要3个字符';
      }
      break;
    case 'email':
      if (value && !/\S+@\S+\.\S+/.test(value)) {
        errorMessage = '请输入有效的电子邮件地址';
      }
      break;
    case 'password':
      if (value && value.length < 6) {
        errorMessage = '密码至少需要6个字符';
      }
      break;
    default:
      break;
  }
  
  setErrors({
    ...errors,
    [name]: errorMessage
  });
};

处理多个输入字段 ​

对于包含多个输入字段的表单，可以使用一个事件处理函数处理所有字段：

const [formData, setFormData] = useState({

  lastName: '',

const handleChange = (e) => {
  const { name, value, type, checked } = e.target;

  onChange={handleChange}

  name="lastName"
  value={formData.lastName}
  onChange={handleChange}
/>
// ... 其他字段

动态表单字段 ​

有时需要创建动态表单，允许用户添加或删除表单字段：

jsx
const [fields, setFields] = useState([
  { id: 1, value: '' }
]);

const addField = () => {
  const newId = fields.length > 0 
    ? Math.max(...fields.map(field => field.id)) + 1 
    : 1;
  
  setFields([...fields, { id: newId, value: '' }]);
};

const removeField = (id) => {
  setFields(fields.filter(field => field.id !== id));
};

const updateField = (id, value) => {
  setFields(
    fields.map(field => 
      field.id === id ? { ...field, value } : field
    )
  );
};

// 渲染动态字段
{fields.map(field => (
  <div key={field.id}>
    <input
      value={field.value}
      onChange={(e) => updateField(field.id, e.target.value)}
    />
    <button type="button" onClick={() => removeField(field.id)}>
      删除
    </button>
  </div>
))}
<button type="button" onClick={addField}>
  添加字段
</button>

表单提交 ​

表单提交通常涉及以下步骤：

  1. 阻止默认提交行为
  2. 验证表单数据
  3. 处理提交状态（提交中、成功、失败）
  4. 提交数据（通常是通过 API 调用）
jsx
const [formData, setFormData] = useState({
  // 表单字段
});
const [isSubmitting, setIsSubmitting] = useState(false);
const [submitSuccess, setSubmitSuccess] = useState(false);
const [submitError, setSubmitError] = useState('');

const handleSubmit = async (e) => {
  e.preventDefault();
  
  // 验证表单
  if (!validateForm()) {
    return;
  }
  
  // 设置提交状态
  setIsSubmitting(true);
  setSubmitError('');
  
  try {
    // 发送数据到服务器
    const response = await fetch('/api/submit', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(formData)
    });
    
    if (!response.ok) {
      throw new Error('提交失败');
    }
    
    const data = await response.json();
    console.log('提交成功:', data);
    
    // 设置成功状态
    setSubmitSuccess(true);
    
    // 可选：重置表单
    resetForm();
  } catch (error) {
    console.error('提交错误:', error);
    setSubmitError(error.message || '提交过程中发生错误');
  } finally {
    setIsSubmitting(false);
  }
};

// 在表单中显示状态
<form onSubmit={handleSubmit}>
  {/* 表单字段 */}
  
  <button type="submit" disabled={isSubmitting}>
    {isSubmitting ? '提交中...' : '提交'}
  </button>
  
  {submitSuccess && (
    <div className="success-message">表单提交成功！</div>
  )}
  
  {submitError && (
    <div className="error-message">{submitError}</div>
  )}
</form>

表单库 ​

对于复杂的表单处理，可以使用专门的表单库来简化开发。以下是一些流行的 React 表单库：

Formik ​

Formik 是一个流行的表单库，它简化了表单处理、验证、错误处理等过程。

jsx
import { Formik, Form, Field, ErrorMessage } from 'formik';
import * as Yup from 'yup';

// 定义验证模式
const validationSchema = Yup.object({
  username: Yup.string()
    .min(3, '用户名至少需要3个字符')
    .required('用户名是必填的'),
  email: Yup.string()
    .email('无效的电子邮件地址')
    .required('电子邮件是必填的'),
  password: Yup.string()
    .min(6, '密码至少需要6个字符')
    .required('密码是必填的')
});

function FormikForm() {
  return (
    <Formik
      initialValues={{ username: '', email: '', password: '' }}
      validationSchema={validationSchema}
      onSubmit={(values, { setSubmitting }) => {
        setTimeout(() => {
          alert(JSON.stringify(values, null, 2));
          setSubmitting(false);
        }, 400);
      }}
    >
      {({ isSubmitting }) => (
        <Form>
          <div>
            <label htmlFor="username">用户名</label>
            <Field name="username" type="text" />
            <ErrorMessage name="username" component="div" className="error" />
          </div>
          
          <div>
            <label htmlFor="email">电子邮件</label>
            <Field name="email" type="email" />
            <ErrorMessage name="email" component="div" className="error" />
          </div>
          
          <div>
            <label htmlFor="password">密码</label>
            <Field name="password" type="password" />
            <ErrorMessage name="password" component="div" className="error" />
          </div>
          
          <button type="submit" disabled={isSubmitting}>
            {isSubmitting ? '提交中...' : '提交'}
          </button>
        </Form>
      )}
    </Formik>
  );
}

React Hook Form ​

React Hook Form 是一个轻量级的表单库，专注于性能和开发体验。

jsx
import { useForm } from 'react-hook-form';

function HookForm() {
  const { register, handleSubmit, formState: { errors } } = useForm();
  
  const onSubmit = data => {
    console.log(data);
  };
  
  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <div>
        <label htmlFor="username">用户名</label>
        <input
          id="username"
          {...register('username', { 
            required: '用户名是必填的',
            minLength: { value: 3, message: '用户名至少需要3个字符' }
          })}
        />
        {errors.username && <p className="error">{errors.username.message}</p>}
      </div>
      
      <div>
        <label htmlFor="email">电子邮件</label>
        <input
          id="email"
          {...register('email', { 
            required: '电子邮件是必填的',
            pattern: { 
              value: /\S+@\S+\.\S+/,
              message: '请输入有效的电子邮件地址'
            }
          })}
        />
        {errors.email && <p className="error">{errors.email.message}</p>}
      </div>
      
      <button type="submit">提交</button>
    </form>
  );
}

表单可访问性 ​

确保表单可访问对所有用户都很重要：

  1. 使用语义化标签：使用 <label>、<fieldset>、<legend> 等标签
  2. 关联标签和输入：使用 htmlFor 属性将标签与输入关联
  3. 提供清晰的错误消息：确保错误消息清晰且与相关字段关联
  4. 键盘导航：确保表单可以通过键盘完全导航
  5. ARIA 属性：使用适当的 ARIA 属性增强可访问性
jsx
<div>
  <label htmlFor="username">用户名</label>
  <input
    id="username"
    name="username"
    aria-required="true"
    aria-invalid={!!errors.username}
    aria-describedby={errors.username ? 'username-error' : undefined}
    value={formData.username}
    onChange={handleChange}
  />
  {errors.username && (
    <div id="username-error" className="error" role="alert">
      {errors.username}
    </div>
  )}
</div>

最佳实践 ​

  1. 选择合适的方法：对于大多数情况，使用受控组件；对于文件上传等特殊场景，考虑非受控组件
  2. 保持组件的单一职责：将复杂表单拆分为更小的组件
  3. 实现客户端验证：提供即时反馈，但始终在服务器端进行最终验证
  4. 处理提交状态：显示加载状态、成功消息和错误消息
  5. 考虑使用表单库：对于复杂表单，考虑使用 Formik、React Hook Form 等库
  6. 确保可访问性：使用语义化标签和适当的 ARIA 属性
  7. 提供清晰的用户反馈：对于错误和成功状态提供清晰的视觉反馈
  8. 优化性能：避免不必要的重新渲染，特别是对于大型表单

总结 ​

React 中的表单处理提供了灵活的方式来收集和验证用户输入。通过使用受控组件或非受控组件，可以根据具体需求选择合适的方法。对于复杂表单，可以考虑使用专门的表单库来简化开发过程。无论选择哪种方法，都应该注重表单的可访问性和用户体验。
