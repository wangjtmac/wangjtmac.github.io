错误边界 ​

在 React 应用程序中，错误处理一直是一个重要的话题。在 React 16 中引入的错误边界（Error Boundaries）提供了一种优雅的方式来捕获和处理 React 组件树中的 JavaScript 错误，防止整个应用崩溃，并提供降级 UI 以改善用户体验。

什么是错误边界？ ​

错误边界是一种特殊的 React 组件，它可以捕获子组件树中任何位置的 JavaScript 错误，记录这些错误，并显示一个备用 UI，而不是让整个组件树崩溃。错误边界在以下情况下捕获错误：

  1. 渲染期间
  2. 生命周期方法中
  3. 构造函数中

如何创建错误边界 ​

错误边界是通过定义 static getDerivedStateFromError() 和/或 componentDidCatch() 生命周期方法的类组件来创建的：

jsx
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error) {
    // 更新 state，下次渲染时显示降级 UI
    return { hasError: true };
  }

  componentDidCatch(error, errorInfo) {
    // 可以将错误日志上报给服务器
    console.error('错误边界捕获到错误:', error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      // 渲染降级 UI
      return <h1>出错了！</h1>;
    }

    return this.props.children;
  }
}

然后，你可以像使用普通组件一样使用错误边界：

jsx
<ErrorBoundary>
  <MyComponent />
</ErrorBoundary>

错误边界的工作原理 ​

错误边界通过两个重要的生命周期方法工作：

1. static getDerivedStateFromError() ​

这个方法在渲染阶段被调用，因此不允许有副作用。它接收捕获到的错误作为参数，并返回一个值以更新状态。

jsx
static getDerivedStateFromError(error) {
  // 返回一个值以更新状态
  return { hasError: true };
}

2. componentDidCatch() ​

这个方法在提交阶段被调用，因此允许执行副作用。通常用于记录错误信息。

jsx
componentDidCatch(error, errorInfo) {
  // 记录错误信息
  logErrorToMyService(error, errorInfo);
}

参数 errorInfo 包含 componentStack 属性，其中包含了导致错误的组件堆栈信息。

错误边界的使用场景 ​

1. 应用程序的顶层 ​

在应用程序的顶层使用错误边界可以捕获整个应用中的错误：

jsx
<ErrorBoundary>
  <App />
</ErrorBoundary>

2. 路由级别 ​

在每个路由组件周围使用错误边界，可以防止一个路由中的错误影响其他路由：

jsx
<Router>
  <Switch>
    <Route path="/profile">
      <ErrorBoundary>
        <Profile />
      </ErrorBoundary>
    </Route>
    <Route path="/dashboard">
      <ErrorBoundary>
        <Dashboard />
      </ErrorBoundary>
    </Route>
  </Switch>
</Router>

3. 重要组件周围 ​

在重要或复杂的组件周围使用错误边界，可以隔离这些组件中的错误：

jsx
<div>
  <Header />
  <ErrorBoundary>
    <ComplexChart data={data} />
  </ErrorBoundary>
  <Footer />
</div>

错误边界的高级用法 ​

1. 带有重置功能的错误边界 ​

有时候，我们希望用户能够在错误发生后尝试重新加载或重置组件：

jsx
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
    this.resetErrorBoundary = this.resetErrorBoundary.bind(this);
  }

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  componentDidCatch(error, errorInfo) {
    console.error('错误边界捕获到错误:', error, errorInfo);
  }

  resetErrorBoundary() {
    this.setState({ hasError: false });
  }

  render() {
    if (this.state.hasError) {
      return (
        <div>
          <h1>出错了！</h1>
          <button onClick={this.resetErrorBoundary}>重试</button>
        </div>
      );
    }

    return this.props.children;
  }
}

2. 带有错误详情的错误边界 ​

在开发环境中，显示错误详情可能会很有帮助：

jsx
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, error: null, errorInfo: null };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  componentDidCatch(error, errorInfo) {
    this.setState({ errorInfo });
    console.error('错误边界捕获到错误:', error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return (
        <div>
          <h1>出错了！</h1>
          <p>{this.state.error && this.state.error.toString()}</p>
          <details style={{ whiteSpace: 'pre-wrap' }}>
            {this.state.errorInfo && this.state.errorInfo.componentStack}
          </details>
        </div>
      );
    }

    return this.props.children;
  }
}

3. 根据环境显示不同的 UI ​

在生产环境中，我们通常希望显示用户友好的错误消息，而在开发环境中显示详细的错误信息：

jsx
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, error: null };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  componentDidCatch(error, errorInfo) {
    console.error('错误边界捕获到错误:', error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      if (process.env.NODE_ENV === 'development') {
        return (
          <div>
            <h1>开发环境错误</h1>
            <p>{this.state.error && this.state.error.toString()}</p>
          </div>
        );
      }
      
      return (
        <div>
          <h1>抱歉，出现了一些问题</h1>
          <p>我们正在努力修复这个问题。请稍后再试。</p>
        </div>
      );
    }

    return this.props.children;
  }
}

错误边界的限制 ​

错误边界无法捕获以下类型的错误：

  1. 事件处理器中的错误：事件处理器中的错误不会影响渲染，因此不会被错误边界捕获。应该使用 try/catch 来处理这些错误。
jsx
function Button() {
  const handleClick = () => {
    try {
      // 可能会抛出错误的代码
      someFunction();
    } catch (error) {
      console.error('事件处理器中的错误:', error);
    }
  };

  return <button onClick={handleClick}>点击</button>;
}
  1. 异步代码中的错误：在 setTimeout、requestAnimationFrame、Promise 等异步代码中的错误不会被错误边界自动捕获。
jsx
function AsyncComponent() {
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        // 异步操作
        const response = await fetch('/api/data');
        const data = await response.json();
        // ...
      } catch (error) {
        setError(error);
      }
    };

    fetchData();
  }, []);

  // 如果有错误，手动抛出，这样错误边界就能捕获
  if (error) {
    throw error;
  }

  return <div>数据加载中...</div>;
}

  1. 服务端渲染期间的错误：在服务端渲染过程中，错误边界可能无法正常工作。

  2. 错误边界自身抛出的错误：如果错误边界组件本身抛出错误，错误将传播到最近的上层错误边界。

错误边界与其他错误处理机制的比较 ​

错误边界 vs try/catch ​

  • 错误边界：用于捕获 React 组件树中的渲染错误
  • try/catch：用于命令式代码和事件处理器中的错误

错误边界 vs 全局错误处理 ​

  • 错误边界：提供组件级别的错误隔离和恢复
  • window.onerror 或 window.addEventListener('error')：捕获所有 JavaScript 错误，但不提供组件级别的隔离

最佳实践 ​

  1. 分层使用错误边界：在应用的不同层级使用错误边界，以便隔离错误并提供适当的降级 UI。

  2. 记录错误：使用 componentDidCatch 记录错误信息，以便调试和改进应用。

jsx
componentDidCatch(error, errorInfo) {
  // 发送错误到你的错误跟踪服务
  logErrorToMyService(error, errorInfo);
}

  1. 提供恢复机制：当可能时，提供让用户恢复或重试的方式。

  2. 为异步错误提供处理：使用 try/catch 捕获异步错误，然后通过状态管理将其传递给错误边界。

  3. 不要过度使用：错误边界应该用于捕获意外错误，而不是作为常规控制流的一部分。

开源错误边界库 ​

虽然可以自己编写错误边界组件，但也有一些优秀的开源库可以使用：

  1. react-error-boundary：提供了一个可重用的错误边界组件，带有重置功能和 FallbackComponent 属性。
jsx
import { ErrorBoundary } from 'react-error-boundary';

function ErrorFallback({ error, resetErrorBoundary }) {
  return (
    <div>
      <h2>出错了！</h2>
      <p>{error.message}</p>
      <button onClick={resetErrorBoundary}>重试</button>
    </div>
  );
}

// 使用
<ErrorBoundary
  FallbackComponent={ErrorFallback}
  onReset={() => {
    // 重置应用状态
  }}
>
  <MyComponent />
</ErrorBoundary>
  1. react-error-overlay：提供了一个开发环境中的错误覆盖层，显示详细的错误信息和堆栈跟踪。

总结 ​

错误边界是 React 应用中处理错误的强大工具，它们可以：

  • 捕获子组件树中的渲染错误
  • 提供降级 UI，防止整个应用崩溃
  • 记录错误信息以便调试
  • 提供恢复机制

通过合理使用错误边界，可以大大提高 React 应用的稳定性和用户体验。记住，错误边界是处理意外错误的安全网，而不是常规错误处理的替代品。对于事件处理器和异步代码中的错误，仍然需要使用传统的 try/catch 语句。
