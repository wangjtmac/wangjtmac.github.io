Portals ​

Portal 提供了一种将子节点渲染到存在于父组件以外的 DOM 节点的优秀方案。

什么是 Portal？ ​

Portal 是 React 16 引入的一个新特性，它允许我们将子节点渲染到父组件 DOM 层次结构之外的 DOM 节点中。

jsx
ReactDOM.createPortal(child, container)

第一个参数 child 是任何可渲染的 React 子元素，例如一个元素、字符串或片段。第二个参数 container 是一个 DOM 元素。

为什么需要 Portal？ ​

在某些情况下，我们需要将子组件渲染到父组件 DOM 层次结构之外的位置，例如：

  1. 模态对话框（Modal）
  2. 悬浮卡片（Tooltip）
  3. 浮动菜单（Floating Menu）
  4. 通知提示（Notification）

这些 UI 元素通常需要"打破"父组件的容器限制，以避免 CSS 属性如 overflow: hidden 或 z-index 的影响。

基本用法 ​

下面是一个创建模态对话框的简单示例：

jsx
import React from 'react';
import ReactDOM from 'react-dom';

function Modal({ children, isOpen }) {
  if (!isOpen) return null;
  
  return ReactDOM.createPortal(
    <div className="modal-overlay">
      <div className="modal-content">
        {children}
      </div>
    </div>,
    document.getElementById('modal-root') // 一个在 body 下的 DOM 节点
  );
}

function App() {
  const [isModalOpen, setIsModalOpen] = React.useState(false);
  
  return (
    <div className="app">
      <h1>我的应用</h1>
      <button onClick={() => setIsModalOpen(true)}>打开模态框</button>
      
      <Modal isOpen={isModalOpen}>
        <h2>模态框标题</h2>
        <p>这是模态框的内容</p>
        <button onClick={() => setIsModalOpen(false)}>关闭</button>
      </Modal>
    </div>
  );
}

在 HTML 文件中，需要添加一个额外的 DOM 节点作为 portal 的目标：

html
<div id="root"></div>
<div id="modal-root"></div>

Portal 的事件冒泡 ​

尽管 Portal 可以将子节点渲染到 DOM 树的任何位置，但在 React 的事件系统中，Portal 中的事件仍然会按照 React 组件树进行冒泡。这意味着，即使 Portal 的子节点在 DOM 结构上不是父组件的后代，从 React 的角度看，它仍然是父组件的子节点。

jsx
function Parent() {
  const [count, setCount] = React.useState(0);
  
  const handleClick = () => {
    // 即使按钮在 Portal 中，这个处理函数仍然会被调用
    setCount(count + 1);
  };
  
  return (
    <div onClick={handleClick}>
      <p>点击次数: {count}</p>
      
      {ReactDOM.createPortal(
        <button>在 Portal 中点击我</button>,
        document.getElementById('portal-root')
      )}
    </div>
  );
}

这个特性使得 Portal 非常适合实现模态框、提示框等需要捕获事件的组件。

使用 Portal 的最佳实践 ​

1. 创建专门的容器 ​

为 Portal 创建专门的容器元素，而不是使用已有的 DOM 元素：

html
<div id="root"></div>
<div id="portal-container"></div>

2. 在组件挂载时创建容器 ​

如果需要动态创建 Portal 容器，可以在组件挂载时创建：

jsx
function DynamicPortal({ children }) {
  const [container] = React.useState(() => {
    // 创建一个新的 div 元素
    const div = document.createElement('div');
    div.className = 'dynamic-portal-container';
    return div;
  });
  
  React.useEffect(() => {
    // 在组件挂载时将容器添加到 body
    document.body.appendChild(container);
    
    // 在组件卸载时移除容器
    return () => {
      document.body.removeChild(container);
    };
  }, [container]);
  
  return ReactDOM.createPortal(children, container);
}

3. 处理无障碍访问 ​

确保 Portal 内容对屏幕阅读器等辅助技术可访问：

jsx
function AccessibleModal({ isOpen, onClose, title, children }) {
  if (!isOpen) return null;
  
  // 捕获 ESC 键
  React.useEffect(() => {
    const handleKeyDown = (e) => {
      if (e.key === 'Escape') onClose();
    };
    
    document.addEventListener('keydown', handleKeyDown);
    return () => document.removeEventListener('keydown', handleKeyDown);
  }, [onClose]);
  
  return ReactDOM.createPortal(
    <div 
      className="modal-overlay" 
      onClick={onClose}
      role="dialog"
      aria-modal="true"
      aria-labelledby="modal-title"
    >
      <div 
        className="modal-content" 
        onClick={e => e.stopPropagation()}
      >
        <h2 id="modal-title">{title}</h2>
        {children}
        <button onClick={onClose}>关闭</button>
      </div>
    </div>,
    document.getElementById('modal-root')
  );
}

Portal 的实际应用场景 ​

1. 模态对话框 ​

jsx
function Modal({ isOpen, onClose, children }) {
  if (!isOpen) return null;
  
  return ReactDOM.createPortal(
    <div className="modal-backdrop" onClick={onClose}>
      <div className="modal" onClick={e => e.stopPropagation()}>
        {children}
      </div>
    </div>,
    document.getElementById('modal-root')
  );
}

2. 工具提示 ​

jsx
function Tooltip({ children, text, isVisible }) {
  const [tooltipElement] = React.useState(() => document.createElement('div'));
  
  React.useEffect(() => {
    document.body.appendChild(tooltipElement);
    return () => document.body.removeChild(tooltipElement);
  }, [tooltipElement]);
  
  return (
    <>
      <div className="tooltip-trigger">{children}</div>
      {isVisible && ReactDOM.createPortal(
        <div className="tooltip">{text}</div>,
        tooltipElement
      )}
    </>
  );
}

3. 通知系统 ​

jsx
function NotificationSystem() {
  const [notifications, setNotifications] = React.useState([]);
  
  const addNotification = (message, type = 'info') => {
    const id = Date.now();
    setNotifications(prev => [...prev, { id, message, type }]);
    
    // 自动移除通知
    setTimeout(() => {
      setNotifications(prev => prev.filter(notification => notification.id !== id));
    }, 3000);
  };
  
  return ReactDOM.createPortal(
    <div className="notification-container">
      {notifications.map(notification => (
        <div key={notification.id} className={`notification ${notification.type}`}>
          {notification.message}
        </div>
      ))}
    </div>,
    document.getElementById('notification-root')
  );
}

总结 ​

Portal 是 React 中一个强大的功能，它解决了在复杂 UI 中将组件渲染到 DOM 树不同位置的问题。通过 Portal，我们可以创建更灵活的组件，如模态框、工具提示和通知系统，同时保持 React 组件的组合性和事件系统的完整性。
