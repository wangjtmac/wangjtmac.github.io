Context API ​

Context API 是 React 提供的一种在组件树中共享数据的方式，无需通过 props 层层传递。它主要用于解决 props 深层传递（prop drilling）的问题，特别适合于共享全局数据，如当前用户信息、主题、语言设置等。

基本概念 ​

Context API 由三个主要部分组成：

  1. Context 对象：通过 React.createContext() 创建
  2. Provider 组件：提供数据的组件，使用 Context.Provider
  3. Consumer 组件：消费数据的组件，可以使用 Context.Consumer 或 useContext Hook

创建和使用 Context ​

创建 Context ​

jsx
// 创建一个 Context 对象，默认值为 'light'
const ThemeContext = React.createContext('light');

默认值只有在没有匹配到 Provider 时才会使用，通常用于测试组件或在组件树顶层没有 Provider 的情况。

提供数据（Provider） ​

jsx
function App() {
  const [theme, setTheme] = useState('light');
  
  return (
    <ThemeContext.Provider value={theme}>
      <Toolbar />
      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
        切换主题
      </button>
    </ThemeContext.Provider>
  );
}

Provider 接受一个 value 属性，可以传递任何类型的数据。当 value 变化时，所有消费该 Context 的组件都会重新渲染。

消费数据（Consumer） ​

有多种方式可以消费 Context 数据：

1. 使用 useContext Hook（函数组件） ​

jsx
function ThemedButton() {
  const theme = useContext(ThemeContext);
  
  return (
    <button className={theme}>
      按照当前主题样式渲染
    </button>
  );
}

这是最简洁的方式，推荐在函数组件中使用。

2. 使用 Context.Consumer（函数组件） ​

jsx
function ThemedButton() {
  return (
    <ThemeContext.Consumer>
      {theme => (
        <button className={theme}>
          按照当前主题样式渲染
        </button>
      )}
    </ThemeContext.Consumer>
  );
}

这种方式使用了 render props 模式，适用于需要消费多个 Context 的情况。

3. 使用 contextType（类组件） ​

jsx
class ThemedButton extends React.Component {
  static contextType = ThemeContext;
  
  render() {
    const theme = this.context;
    return (
      <button className={theme}>
        按照当前主题样式渲染
      </button>
    );
  }
}

这种方式只能用于类组件，且一个组件只能使用一个 Context。

多个 Context 的使用 ​

当需要使用多个 Context 时，可以嵌套 Provider：

jsx
function App() {
  return (
    <ThemeContext.Provider value="dark">
      <UserContext.Provider value={{ name: '张三' }}>
        <Layout />
      </UserContext.Provider>
    </ThemeContext.Provider>
  );
}

在函数组件中消费多个 Context：

jsx
function Content() {
  const theme = useContext(ThemeContext);
  const user = useContext(UserContext);
  
  return (
    <div className={theme}>
      <p>当前用户：{user.name}</p>
    </div>
  );
}

Context 的更新 ​

Context 的值可以是任何 JavaScript 值，包括函数。通过传递函数，可以让子组件更新 Context：

jsx
function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  
  const toggleTheme = () => {
    setTheme(prevTheme => (prevTheme === 'light' ? 'dark' : 'light'));
  };
  
  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

然后在子组件中：

jsx
function ThemedButton() {
  const { theme, toggleTheme } = useContext(ThemeContext);
  
  return (
    <button 
      className={theme} 
      onClick={toggleTheme}
    >
      切换主题
    </button>
  );
}

性能优化 ​

Context 的一个潜在问题是，当 Provider 的 value 变化时，所有消费该 Context 的组件都会重新渲染，即使它们只使用了 Context 的一部分。

拆分 Context ​

将不同类型的数据放在不同的 Context 中：

jsx
const ThemeContext = createContext();
const UserContext = createContext();

function App() {
  const [theme, setTheme] = useState('light');
  const [user, setUser] = useState({ name: '张三' });
  
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      <UserContext.Provider value={{ user, setUser }}>
        <Main />
      </UserContext.Provider>
    </ThemeContext.Provider>
  );
}

使用 memo 优化 ​

结合 React.memo 避免不必要的重新渲染：

jsx
const ThemedButton = React.memo(function ThemedButton() {
  const { theme } = useContext(ThemeContext);
  return <button className={theme}>按钮</button>;
});

稳定的 Context 值 ​

使用 useMemo 确保 Context 值的稳定性：

jsx
function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  
  const themeValue = useMemo(() => ({
    theme,
    toggleTheme: () => setTheme(t => (t === 'light' ? 'dark' : 'light'))
  }), [theme]);
  
  return (
    <ThemeContext.Provider value={themeValue}>
      {children}
    </ThemeContext.Provider>
  );
}

最佳实践 ​

  1. 创建自定义 Hook：封装 Context 的使用逻辑
jsx
function useTheme() {
  const context = useContext(ThemeContext);
  if (context === undefined) {
    throw new Error('useTheme must be used within a ThemeProvider');
  }
  return context;
}
  1. 创建专用的 Provider 组件：封装 Context 的创建和提供逻辑
jsx
function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  
  const value = useMemo(() => ({
    theme,
    toggleTheme: () => setTheme(t => (t === 'light' ? 'dark' : 'light'))
  }), [theme]);
  
  return (
    <ThemeContext.Provider value={value}>
      {children}
    </ThemeContext.Provider>
  );
}

  1. 合理组织 Context 层次结构：避免过深的嵌套

  2. 不要过度使用 Context：对于局部状态，仍然使用 props 或组件内部的 state

  3. 考虑使用 Context + useReducer：处理复杂状态逻辑

jsx
const initialState = { theme: 'light', user: null };

function reducer(state, action) {
  switch (action.type) {
    case 'TOGGLE_THEME':
      return { ...state, theme: state.theme === 'light' ? 'dark' : 'light' };
    case 'SET_USER':
      return { ...state, user: action.payload };
    default:
      throw new Error();
  }
}

function AppProvider({ children }) {
  const [state, dispatch] = useReducer(reducer, initialState);
  
  const value = useMemo(() => ({ state, dispatch }), [state]);
  
  return (
    <AppContext.Provider value={value}>
      {children}
    </AppContext.Provider>
  );
}

何时使用 Context ​

Context 适合用于：

  • 主题设置
  • 用户信息
  • 语言设置
  • 路由数据
  • 全局消息通知
  • 权限控制

不适合用于：

  • 频繁变化的数据
  • 可以通过 props 简单传递的数据
  • 组件库的内部实现细节

Context 与状态管理库的对比 ​

对于简单到中等复杂度的应用，Context + useReducer 可以替代 Redux 等状态管理库。但对于大型应用，专业的状态管理库仍然有其优势：

  • 开发者工具支持
  • 中间件系统
  • 时间旅行调试
  • 状态持久化
  • 更成熟的生态系统

总结 ​

Context API 是 React 提供的一种强大的组件间通信机制，适合于共享全局数据。合理使用 Context 可以简化组件树中的数据流，避免 props 层层传递的问题。但也要注意性能问题和使用场景，不要过度使用 Context。
