Refs 和 DOM ​

在 React 的世界中，我们通常使用声明式编程来操作 UI，但有时我们需要直接访问 DOM 元素或组件实例。Refs 提供了一种方式，允许我们在 React 的声明式编程模型之外，直接访问 DOM 节点或 React 元素。

什么是 Refs ​

Refs 是 React 提供的一个特殊属性，允许我们获取对 DOM 节点或 React 元素实例的引用。与 props 不同，Refs 不会触发组件重新渲染，它们主要用于：

  1. 管理焦点、文本选择或媒体播放
  2. 触发命令式动画
  3. 集成第三方 DOM 库
  4. 存储和访问组件内的可变值

创建和使用 Refs ​

在函数组件中使用 useRef ​

在函数组件中，我们使用 useRef Hook 来创建和管理 Refs：

jsx
import React, { useRef } from 'react';

function TextInputWithFocusButton() {
  // 创建一个 ref 对象
  const inputRef = useRef(null);
  
  const focusInput = () => {
    // 使用 current 属性访问 DOM 节点
    inputRef.current.focus();
  };
  
  return (
    <>
      <input ref={inputRef} type="text" />
      <button onClick={focusInput}>聚焦输入框</button>
    </>
  );
}

useRef 返回一个可变的 ref 对象，其 .current 属性被初始化为传入的参数。返回的对象在组件的整个生命周期内保持不变。

在类组件中使用 createRef ​

在类组件中，我们可以使用 React.createRef() 来创建 Refs：

jsx
class TextInputWithFocusButton extends React.Component {
  constructor(props) {
    super(props);
    // 创建一个 ref 对象
    this.inputRef = React.createRef();
  }
  
  focusInput = () => {
    // 使用 current 属性访问 DOM 节点
    this.inputRef.current.focus();
  };
  
  render() {
    return (
      <>
        <input ref={this.inputRef} type="text" />
        <button onClick={this.focusInput}>聚焦输入框</button>
      </>
    );
  }
}

回调 Refs ​

除了使用 useRef 和 createRef，React 还提供了一种更灵活的设置 refs 的方式，称为"回调 refs"：

jsx
function CustomTextInput() {
  let inputRef = null;
  
  const setInputRef = (element) => {
    inputRef = element;
  };
  
  const focusInput = () => {
    if (inputRef) {
      inputRef.focus();
    }
  };
  
  return (
    <>
      <input ref={setInputRef} type="text" />
      <button onClick={focusInput}>聚焦输入框</button>
    </>
  );
}

回调 refs 在某些情况下更为灵活，比如需要根据某些条件动态设置 refs 时。

Refs 的常见使用场景 ​

1. 管理焦点 ​

最常见的 Refs 用例是管理输入框的焦点：

jsx
function FocusExample() {
  const inputRef = useRef(null);
  
  const focusInput = () => {
    inputRef.current.focus();
  };
  
  return (
    <>
      <input ref={inputRef} type="text" />
      <button onClick={focusInput}>聚焦输入框</button>
    </>
  );
}

2. 测量 DOM 元素 ​

Refs 可以用来获取 DOM 元素的尺寸或位置：

jsx
function MeasureExample() {
  const divRef = useRef(null);
  const [dimensions, setDimensions] = useState({ width: 0, height: 0 });
  
  useEffect(() => {
    if (divRef.current) {
      const { width, height } = divRef.current.getBoundingClientRect();
      setDimensions({ width, height });
    }
  }, []);
  
  return (
    <>
      <div ref={divRef} style={{ width: '100%', height: '100px', background: 'lightblue' }}>
        测量我的尺寸
      </div>
      <p>宽度: {dimensions.width}px, 高度: {dimensions.height}px</p>
    </>
  );
}

3. 媒体控制 ​

Refs 可以用来控制视频或音频播放：

jsx
function VideoPlayer() {
  const videoRef = useRef(null);
  
  const playVideo = () => {
    videoRef.current.play();
  };
  
  const pauseVideo = () => {
    videoRef.current.pause();
  };
  
  return (
    <>
      <video ref={videoRef} src="video.mp4" />
      <button onClick={playVideo}>播放</button>
      <button onClick={pauseVideo}>暂停</button>
    </>
  );
}

4. 集成第三方 DOM 库 ​

当需要与第三方 DOM 库集成时，Refs 非常有用：

jsx
function MapComponent() {
  const mapRef = useRef(null);
  
  useEffect(() => {
    // 假设我们使用 Leaflet 地图库
    if (mapRef.current) {
      const map = L.map(mapRef.current).setView([51.505, -0.09], 13);
      L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png').addTo(map);
    }
  }, []);
  
  return <div ref={mapRef} style={{ height: '400px' }} />;
}

5. 存储前一个值 ​

Refs 可以用来在渲染之间存储值，而不触发重新渲染：

jsx
function Counter() {
  const [count, setCount] = useState(0);
  const prevCountRef = useRef(0);
  
  useEffect(() => {
    prevCountRef.current = count;
  }, [count]);
  
  const prevCount = prevCountRef.current;
  
  return (
    <div>
      <p>当前值: {count}, 前一个值: {prevCount}</p>
      <button onClick={() => setCount(count + 1)}>增加</button>
    </div>
  );
}

Ref 转发（Forwarding Refs） ​

默认情况下，Refs 不能传递给子组件。如果你需要将 ref 传递给子组件，可以使用 forwardRef API：

jsx
// 子组件使用 forwardRef 接收父组件传递的 ref
const FancyButton = React.forwardRef((props, ref) => (
  <button ref={ref} className="fancy-button">
    {props.children}
  </button>
));

// 父组件
function App() {
  const buttonRef = useRef(null);
  
  const handleClick = () => {
    buttonRef.current.focus();
  };
  
  return (
    <>
      <FancyButton ref={buttonRef}>Click me!</FancyButton>
      <button onClick={handleClick}>聚焦 FancyButton</button>
    </>
  );
}

Ref 转发是一种将 ref 自动通过组件传递给其子组件的技术，对于高阶组件和可重用的组件库特别有用。

useImperativeHandle ​

当使用 forwardRef 时，有时我们不想暴露整个 DOM 节点，而只想暴露特定的方法。useImperativeHandle 可以让我们自定义暴露给父组件的实例值：

jsx
const FancyInput = React.forwardRef((props, ref) => {
  const inputRef = useRef(null);
  
  // 自定义暴露给父组件的实例值
  useImperativeHandle(ref, () => ({
    focus: () => {
      inputRef.current.focus();
    },
    reset: () => {
      inputRef.current.value = '';
    }
  }));
  
  return <input ref={inputRef} />;
});

// 父组件
function App() {
  const fancyInputRef = useRef(null);
  
  const handleClick = () => {
    fancyInputRef.current.focus();
  };
  
  const handleReset = () => {
    fancyInputRef.current.reset();
  };
  
  return (
    <>
      <FancyInput ref={fancyInputRef} />
      <button onClick={handleClick}>聚焦</button>
      <button onClick={handleReset}>重置</button>
    </>
  );
}

Refs 的注意事项 ​

  1. 避免过度使用 Refs：大多数情况下，应该优先考虑声明式解决方案。只有在必要时才使用 Refs。

  2. 不要在渲染期间读取或写入 refs.current：React 的渲染阶段应该是纯粹的，而 refs.current 的读写是副作用，应该在事件处理函数或生命周期方法中进行。

  3. Refs 和函数式更新：当使用 refs 存储可变值时，要小心不要创建闭包陷阱。考虑使用函数式更新或 useCallback 来避免这些问题。

  4. Refs 不会触发重新渲染：如果你需要在值变化时重新渲染组件，应该使用 state 而不是 refs。

何时使用 Refs ​

Refs 应该在以下情况下使用：

  1. 管理焦点、文本选择或媒体播放
  2. 触发命令式动画
  3. 集成第三方 DOM 库
  4. 测量 DOM 节点的尺寸或位置
  5. 在渲染之间存储可变值，而不触发重新渲染

而在以下情况下应避免使用 Refs：

  1. 可以通过声明式实现的任何事情
  2. 过度操作 DOM
  3. 创建过于命令式的代码结构

总结 ​

Refs 提供了一种在 React 的声明式编程模型之外直接访问 DOM 节点或 React 元素的方式。虽然在大多数情况下应该优先使用声明式解决方案，但 Refs 在特定场景下非常有用，如管理焦点、集成第三方库或测量 DOM 元素。

通过合理使用 Refs，我们可以在保持 React 声明式特性的同时，处理那些需要直接 DOM 访问的边缘情况。
