CSS Modules ​

CSS Modules是一种通过构建工具将CSS类名局部作用域化的技术，它通过自动生成唯一的类名来避免样式冲突，使得CSS能够像模块一样被导入和使用。

基本概念 ​

CSS Modules的核心理念是将CSS的作用域限制在组件级别，通过自动生成唯一的类名来避免全局命名冲突，同时保留原生CSS的所有功能和优势。

主要特点 ​

  • 局部作用域：类名自动转换为唯一标识符，避免全局命名冲突
  • 显式依赖：样式需要显式导入才能使用，避免隐式依赖
  • 组合复用：支持通过composes关键字组合和复用样式
  • 与现有工具兼容：可与SASS、Less、PostCSS等工具结合使用

工作原理 ​

CSS Modules通过构建工具（如Webpack、Vite等）处理以.module.css结尾的CSS文件，将其中的类名转换为唯一标识符，并生成一个JavaScript对象，将原始类名映射到转换后的唯一类名。

转换过程 ​

  1. 创建.module.css文件并定义样式
  2. 构建工具识别并处理该文件
  3. 类名被转换为唯一标识符（通常基于文件路径、类名和哈希值）
  4. 生成一个将原始类名映射到转换后类名的JavaScript对象
  5. 组件导入并使用该对象应用样式

示例转换 ​

原始CSS文件：

css
/* Button.module.css */
.button {
  color: red;
}

转换后的CSS：

css
.Button_button_1a2b3c {
  color: red;
}

JavaScript中的使用：

jsx
import styles from './Button.module.css';
// styles = { button: "Button_button_1a2b3c" }

<button className={styles.button}>Click me</button>
// 渲染为 <button class="Button_button_1a2b3c">Click me</button>

基本用法 ​

创建和使用CSS Module ​

  1. 创建以.module.css结尾的CSS文件：
css
/* Button.module.css */
.button {
  background-color: #007bff;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}

.primary {
  background-color: #007bff;
}

.secondary {
  background-color: #6c757d;
}
  1. 在组件中导入并使用：
jsx
import React from 'react';
import styles from './Button.module.css';

function Button({ children, onClick, variant = 'primary' }) {
  return (
    <button 
      className={`${styles.button} ${styles[variant]}`} 
      onClick={onClick}
    >
      {children}
    </button>
  );
}

export default Button;

组合多个类名 ​

可以使用模板字符串、条件表达式或类名合并库（如classnames）组合多个类名：

jsx
// 使用模板字符串
<div className={`${styles.container} ${styles.active}`}></div>

// 使用条件表达式
<div className={`${styles.button} ${isActive ? styles.active : ''}`}></div>

// 使用classnames库
import classNames from 'classnames';
<div className={classNames(styles.button, { [styles.active]: isActive })}></div>

高级用法 ​

组合样式（Composition） ​

CSS Modules支持通过composes关键字组合和复用样式：

css
/* colors.module.css */
.primary {
  color: #007bff;
}

/* Button.module.css */
.button {
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}

.primaryButton {
  composes: button;
  composes: primary from './colors.module.css';
  background-color: #007bff;
}

使用组合后的样式：

jsx
import styles from './Button.module.css';

<button className={styles.primaryButton}>Primary Button</button>
// 渲染为 <button class="Button_button_1a2b3c Colors_primary_x1y2z3 Button_primaryButton_7u8i9o">Primary Button</button>

全局样式 ​

CSS Modules默认将所有类名局部作用域化，但有时需要定义全局样式，可以使用:global语法：

css
/* global.module.css */
:global(.title) {
  font-size: 24px;
  font-weight: bold;
}

.container {
  padding: 20px;
}

:global {
  .body {
    margin: 0;
    font-family: sans-serif;
  }
  
  .link {
    color: blue;
  }
}

这样，.title、.body和.link将作为全局类名使用，而.container仍然是局部作用域的。

使用变量 ​

CSS Modules可以与CSS变量（自定义属性）结合使用：

css
/* variables.module.css */
.container {
  --primary-color: #007bff;
  --secondary-color: #6c757d;
  --spacing: 10px;
}

.button {
  background-color: var(--primary-color);
  padding: var(--spacing);
  margin: var(--spacing);
}

与CSS预处理器结合 ​

CSS Modules可以与SASS、Less等CSS预处理器结合使用：

scss
/* Button.module.scss */
$primary-color: #007bff;
$border-radius: 4px;

.button {
  background-color: $primary-color;
  border-radius: $border-radius;
  padding: 10px 20px;
  
  &:hover {
    opacity: 0.8;
  }
  
  &.large {
    font-size: 18px;
    padding: 15px 30px;
  }
}

配置方法 ​

Webpack配置 ​

在Webpack中配置CSS Modules：

javascript
// webpack.config.js
module.exports = {
  // ...
  module: {
    rules: [
      {
        test: /\.module\.css$/,
        use: [
          'style-loader',
          {
            loader: 'css-loader',
            options: {
              modules: {
                localIdentName: '[name]__[local]--[hash:base64:5]',
              },
            },
          },
        ],
      },
      // 处理非CSS Module的普通CSS文件
      {
        test: /\.css$/,
        exclude: /\.module\.css$/,
        use: ['style-loader', 'css-loader'],
      },
    ],
  },
  // ...
};

Create React App ​

Create React App默认支持CSS Modules，只需将CSS文件命名为[name].module.css即可。

Vite配置 ​

Vite也默认支持CSS Modules，同样只需将CSS文件命名为[name].module.css。如果需要自定义配置：

javascript
// vite.config.js
export default {
  css: {
    modules: {
      // 自定义生成的类名格式
      generateScopedName: '[name]__[local]--[hash:base64:5]',
    },
  },
};

Next.js配置 ​

Next.js也原生支持CSS Modules，只需将CSS文件命名为[name].module.css即可。

TypeScript集成 ​

要在TypeScript中获得CSS Modules的类型支持，可以使用以下方法：

方法一：声明模块 ​

创建一个类型声明文件：

typescript
// css-modules.d.ts
declare module '*.module.css' {
  const classes: { [key: string]: string };
  export default classes;
}

方法二：使用自动生成的类型定义 ​

使用typescript-plugin-css-modules插件：

bash
npm install --save-dev typescript-plugin-css-modules

然后在tsconfig.json中配置：

json
{
  "compilerOptions": {
    "plugins": [
      { "name": "typescript-plugin-css-modules" }
    ]
  }
}

或者使用@teamsupercell/typings-for-css-modules-loader：

javascript
// webpack.config.js
module.exports = {
  // ...
  module: {
    rules: [
      {
        test: /\.module\.css$/,
        use: [
          'style-loader',
          {
            loader: 'css-loader',
            options: {
              modules: true,
            },
          },
          '@teamsupercell/typings-for-css-modules-loader',
        ],
      },
    ],
  },
  // ...
};

最佳实践 ​

文件组织 ​

将CSS Module与对应的组件文件放在一起：

components/
  Button/
    Button.jsx
    Button.module.css
    Button.test.js
  Card/
    Card.jsx
    Card.module.css
    Card.test.js

命名约定 ​

  • 使用语义化的类名，反映元素的用途而非外观
  • 使用camelCase或kebab-case命名，保持一致性
  • 避免过于通用的类名，如.container、.wrapper等

共享样式 ​

对于需要在多个组件之间共享的样式，可以：

  1. 创建一个共享的CSS Module文件：
css
/* shared.module.css */
.flexCenter {
  display: flex;
  align-items: center;
  justify-content: center;
}

.textEllipsis {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
  1. 使用composes引入共享样式：
css
/* Button.module.css */
.button {
  composes: flexCenter from '../shared.module.css';
  /* 其他样式 */
}

主题和变量 ​

使用CSS变量实现主题：

css
/* theme.css (全局CSS) */
:root {
  --primary-color: #007bff;
  --secondary-color: #6c757d;
  --font-family: 'Roboto', sans-serif;
  --spacing-unit: 8px;
}

/* 暗色主题 */
.dark-theme {
  --primary-color: #0d6efd;
  --secondary-color: #adb5bd;
}

在CSS Module中使用这些变量：

css
/* Button.module.css */
.button {
  background-color: var(--primary-color);
  font-family: var(--font-family);
  padding: calc(var(--spacing-unit) * 2);
}

性能优化 ​

  • 避免过度使用composes，过多的类组合可能导致HTML元素上的类名过多
  • 将全局样式和重置样式放在单独的全局CSS文件中，而不是CSS Module中
  • 考虑使用CSS提取插件（如mini-css-extract-plugin）将CSS提取到单独的文件中，以便并行加载

常见问题与解决方案 ​

动态类名 ​

处理动态类名的几种方法：

jsx
// 方法1：模板字符串
<div className={`${styles.button} ${isActive ? styles.active : ''}`}></div>

// 方法2：数组join
<div className={[styles.button, isActive && styles.active].filter(Boolean).join(' ')}></div>

// 方法3：使用classnames库
import classNames from 'classnames';
<div className={classNames(styles.button, { [styles.active]: isActive })}></div>

处理外部组件 ​

为外部组件（如第三方库）提供CSS Module类名：

jsx
import { Button } from 'some-ui-library';
import styles from './CustomButton.module.css';

// 假设Button组件接受className属性
<Button className={styles.customButton}>Click me</Button>

处理媒体查询 ​

CSS Module中的媒体查询与普通CSS相同：

css
/* Button.module.css */
.button {
  padding: 10px;
  font-size: 14px;
}

@media (min-width: 768px) {
  .button {
    padding: 15px;
    font-size: 16px;
  }
}

总结 ​

CSS Modules是一种强大的CSS组织方案，它通过局部作用域解决了全局命名冲突的问题，同时保留了原生CSS的所有功能和优势。它特别适合中大型项目，能够提供良好的样式模块化和可维护性。

通过合理使用CSS Modules的各种特性，如组合、全局样式和与预处理器的结合，可以构建出结构清晰、易于维护的样式系统，为React应用提供可靠的样式解决方案。
