CSS-in-JS ​

CSS-in-JS是一种将样式直接写在JavaScript中的方法，使组件完全自包含，包括其样式定义。这种方法使得样式可以与组件逻辑紧密结合，实现真正的组件封装。

基本概念 ​

CSS-in-JS是一种设计理念而非具体实现，它的核心思想是将CSS样式直接在JavaScript中编写，通常与组件定义放在一起。这种方法打破了传统的"关注点分离"原则，转而采用"关注点内聚"的思想，使组件成为真正独立的单元。

主要特点 ​

  • 组件级样式：样式与组件紧密绑定，增强组件的自包含性
  • 动态样式：可以基于组件props、state或主题动态生成样式
  • 避免全局命名冲突：自动生成唯一的类名，避免样式冲突
  • 类型安全：支持TypeScript，提供类型检查和自动完成
  • 主题支持：内置对主题的支持，便于实现全局主题系统

主要实现库 ​

styled-components ​

styled-components是最流行的CSS-in-JS库之一，它使用标签模板字符串语法创建样式化组件。

基本用法 ​

jsx
import styled from 'styled-components';

// 创建一个样式化的按钮组件
const Button = styled.button`
  background-color: #007bff;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
  
  &:hover {
    background-color: #0056b3;
  }
`;

// 使用样式化组件
function App() {
  return <Button>Click me</Button>;
}

基于props的动态样式 ​

jsx
const Button = styled.button`
  background-color: ${props => props.primary ? '#007bff' : 'gray'};
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
  
  &:hover {
    background-color: ${props => props.primary ? '#0056b3' : 'darkgray'};
  }
`;

function App() {
  return (
    <div>
      <Button primary>Primary Button</Button>
      <Button>Secondary Button</Button>
    </div>
  );
}

Emotion ​

Emotion是另一个流行的CSS-in-JS库，它提供了多种样式编写方式，包括标签模板字符串和对象语法。

基本用法（jsx pragma方式） ​

jsx
/** @jsx jsx */
import { css, jsx } from '@emotion/react';

const button = css`
  background-color: #007bff;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
  
  &:hover {
    background-color: #0056b3;
  }
`;

function Button({ children }) {
  return <button css={button}>{children}</button>;
}

styled API（类似styled-components） ​

jsx
import styled from '@emotion/styled';

const Button = styled.button`
  background-color: ${props => props.primary ? '#007bff' : 'gray'};
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
`;

function App() {
  return <Button primary>Click me</Button>;
}

其他实现 ​

  • JSS：一个低级别的CSS-in-JS库，使用对象语法定义样式
  • Linaria：零运行时CSS-in-JS库，在构建时提取CSS
  • Stitches：高性能、类型安全的CSS-in-JS库，专注于组件变体
  • vanilla-extract：零运行时、类型安全的CSS-in-JS，在构建时提取CSS

工作原理 ​

CSS-in-JS库通常通过以下方式工作：

  1. 样式定义：开发者使用JavaScript编写CSS样式（通常使用模板字符串或对象语法）
  2. 样式处理：库解析样式定义，生成唯一的类名和对应的CSS规则
  3. 样式注入：将生成的CSS规则注入到DOM中（通常是在<head>中创建<style>标签）
  4. 类名应用：将生成的唯一类名应用到组件上

运行时 vs 构建时 ​

CSS-in-JS库可以分为两类：

  1. 运行时库（如styled-components、Emotion）：

    • 在浏览器运行时解析和注入样式
    • 支持完全动态的样式生成
    • 可能会增加JavaScript运行时开销

  2. 构建时库（如Linaria、vanilla-extract）：

    • 在构建过程中提取CSS到静态文件
    • 减少或消除运行时开销
    • 动态样式能力可能有限

高级用法 ​

主题系统 ​

大多数CSS-in-JS库支持主题系统，允许在整个应用中共享设计变量。

styled-components的主题 ​

jsx
import { ThemeProvider, styled } from 'styled-components';

// 定义主题
const theme = {
  colors: {
    primary: '#007bff',
    secondary: '#6c757d',
    danger: '#dc3545',
  },
  spacing: {
    small: '8px',
    medium: '16px',
    large: '24px',
  },
  fontSizes: {
    small: '12px',
    medium: '16px',
    large: '20px',
  },
};

// 使用主题中的值
const Button = styled.button`
  background-color: ${props => props.theme.colors.primary};
  padding: ${props => props.theme.spacing.medium};
  font-size: ${props => props.theme.fontSizes.medium};
  color: white;
  border: none;
  border-radius: 4px;
`;

// 提供主题
function App() {
  return (
    <ThemeProvider theme={theme}>
      <Button>Themed Button</Button>
    </ThemeProvider>
  );
}

全局样式 ​

虽然CSS-in-JS主要关注组件级样式，但也支持定义全局样式。

styled-components的全局样式 ​

jsx
import { createGlobalStyle } from 'styled-components';

const GlobalStyle = createGlobalStyle`
  * {
    box-sizing: border-box;
    margin: 0;
    padding: 0;
  }
  
  body {
    font-family: 'Roboto', sans-serif;
    line-height: 1.5;
    color: #333;
  }
  
  a {
    color: #007bff;
    text-decoration: none;
  }
`;

function App() {
  return (
    <>
      <GlobalStyle />
      <div>App content</div>
    </>
  );
}

服务端渲染（SSR）支持 ​

大多数CSS-in-JS库支持服务端渲染，但实现方式可能不同。

styled-components的SSR ​

jsx
import { renderToString } from 'react-dom/server';
import { ServerStyleSheet } from 'styled-components';
import App from './App';

function renderApp() {
  const sheet = new ServerStyleSheet();
  try {
    // 捕获样式
    const html = renderToString(sheet.collectStyles(<App />));
    // 提取样式标签
    const styleTags = sheet.getStyleTags();
    
    return `
      <!DOCTYPE html>
      <html>
        <head>
          <title>My App</title>
          ${styleTags}
        </head>
        <body>
          <div id="root">${html}</div>
        </body>
      </html>
    `;
  } finally {
    sheet.seal();
  }
}

Emotion的SSR ​

jsx
import { renderToString } from 'react-dom/server';
import createEmotionServer from '@emotion/server/create-instance';
import { CacheProvider } from '@emotion/react';
import createCache from '@emotion/cache';
import App from './App';

function renderApp() {
  const cache = createCache({ key: 'css' });
  const { extractCritical } = createEmotionServer(cache);
  
  const html = renderToString(
    <CacheProvider value={cache}>
      <App />
    </CacheProvider>
  );
  
  const { css, ids } = extractCritical(html);
  
  return `
    <!DOCTYPE html>
    <html>
      <head>
        <title>My App</title>
        <style data-emotion="${cache.key} ${ids.join(' ')}">${css}</style>
      </head>
      <body>
        <div id="root">${html}</div>
      </body>
    </html>
  `;
}

性能考虑 ​

运行时开销 ​

运行时CSS-in-JS库（如styled-components）在客户端解析和注入样式，可能会增加JavaScript执行时间和内存使用。这在大型应用或低性能设备上可能会导致性能问题。

优化策略 ​

  1. 使用构建时CSS-in-JS库：考虑使用Linaria或vanilla-extract等构建时库，它们在构建时提取CSS，减少或消除运行时开销。

  2. 启用SSR：服务端渲染可以减少客户端的样式处理工作，提高首屏渲染性能。

  3. 使用Babel插件：styled-components和Emotion都提供了Babel插件，可以优化样式处理：

bash
# 安装styled-components的Babel插件
npm install --save-dev babel-plugin-styled-components

# 安装Emotion的Babel插件
npm install --save-dev @emotion/babel-plugin

配置Babel：

json
// .babelrc
{
  "plugins": [
    "babel-plugin-styled-components",
    "@emotion/babel-plugin"
  ]
}

  1. 避免过度使用props：过多依赖props生成样式可能导致不必要的重新渲染。

  2. 使用组件记忆化：对于使用动态样式的组件，考虑使用React.memo或useMemo减少不必要的重新计算。

TypeScript集成 ​

CSS-in-JS库通常提供良好的TypeScript支持。

styled-components的TypeScript集成 ​

tsx
import styled from 'styled-components';

// 定义props类型
interface ButtonProps {
  primary?: boolean;
  size?: 'small' | 'medium' | 'large';
}

// 使用TypeScript泛型
const Button = styled.button<ButtonProps>`
  background-color: ${props => props.primary ? '#007bff' : 'gray'};
  color: white;
  padding: ${props => {
    switch (props.size) {
      case 'small': return '5px 10px';
      case 'large': return '15px 30px';
      default: return '10px 20px';
    }
  }};
  border: none;
  border-radius: 4px;
  font-size: ${props => props.size === 'large' ? '18px' : '14px'};
`;

// 使用带类型的组件
function App() {
  return (
    <div>
      <Button>Default Button</Button>
      <Button primary>Primary Button</Button>
      <Button size="large" primary>Large Primary Button</Button>
    </div>
  );
}

// components/Box.js

const Box = styled.div`

  ${props => props.width && `width: ${props.width};`}
  ${props => props.height && `height: ${props.height};`}
`;

export default Box;

  colors: {
    text: '#212529',
    background: '#ffffff',
    // 其他颜色...

// ThemeProvider.js
import { useState } from 'react';
import { ThemeProvider as StyledThemeProvider } from 'styled-components';
import { lightTheme, darkTheme } from './themes';

export const ThemeContext = React.createContext({
  theme: 'light',
  toggleTheme: () => {},
});

export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  
  const toggleTheme = () => {
    setTheme(prevTheme => prevTheme === 'light' ? 'dark' : 'light');
  };
  
  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      <StyledThemeProvider theme={theme === 'light' ? lightTheme : darkTheme}>
        {children}
      </StyledThemeProvider>
    </ThemeContext.Provider>
  );
}

性能最佳实践 ​

  1. 避免内联样式定义：不要在渲染方法中创建样式化组件，这会导致每次渲染都创建新的组件。
jsx
// 错误：在渲染方法中创建样式化组件
function Component() {
  // 这会在每次渲染时创建新的样式化组件
  const Button = styled.button`
    background-color: blue;
    color: white;
  `;
  
  return <Button>Click me</Button>;
}

// 正确：在组件外部创建样式化组件
const Button = styled.button`
  background-color: blue;
  color: white;
`;

function Component() {
  return <Button>Click me</Button>;
}
  1. 使用memoization：对于依赖props生成样式的组件，考虑使用memoization减少不必要的重新渲染。
jsx
import React, { useMemo } from 'react';
import styled from 'styled-components';

function ComplexComponent({ color, size, ...props }) {
  // 使用useMemo缓存样式化组件
  const StyledComponent = useMemo(() => {
    return styled.div`
      color: ${color};
      font-size: ${size}px;
      /* 其他复杂样式 */
    `;
  }, [color, size]);
  
  return <StyledComponent {...props} />;
}

export default React.memo(ComplexComponent);

常见问题与解决方案 ​

样式闪烁（FOUC） ​

在使用CSS-in-JS时，可能会出现样式闪烁（Flash of Unstyled Content）问题，特别是在客户端渲染时。

解决方案：

  • 使用服务端渲染（SSR）
  • 确保正确配置了SSR的样式提取
  • 考虑使用构建时CSS-in-JS库

与第三方组件集成 ​

有时需要将CSS-in-JS与第三方组件库集成。

解决方案：

  • 使用样式覆盖：
jsx
import { Button as MUIButton } from '@material-ui/core';
import styled from 'styled-components';

const StyledMUIButton = styled(MUIButton)`
  && {
    background-color: #007bff;
    color: white;
    
    &:hover {
      background-color: #0056b3;
    }
  }
`;

测试样式化组件 ​

测试样式化组件可能需要特殊处理。

解决方案：

  • 使用jest-styled-components库增强Jest测试：
jsx
import React from 'react';
import { render } from '@testing-library/react';
import 'jest-styled-components';
import Button from './Button';

test('Button renders correctly', () => {
  const { container } = render(<Button primary />);
  expect(container.firstChild).toHaveStyleRule('background-color', '#007bff');
});

结论 ​

CSS-in-JS是一种强大的样式解决方案，特别适合需要高度动态样式、主题支持和组件封装的React应用。它通过将样式与组件逻辑紧密结合，提供了更好的开发体验和组件可重用性。

虽然CSS-in-JS可能带来一些运行时开销，但通过正确的优化策略和选择合适的库，这些问题可以得到有效缓解。对于需要构建复杂UI组件库或设计系统的团队，CSS-in-JS是一个值得考虑的选择。

随着前端技术的不断发展，CSS-in-JS库也在不断改进，如零运行时CSS-in-JS库的出现，为开发者提供了更多选择，使CSS-in-JS成为现代React应用样式管理的有力工具。
