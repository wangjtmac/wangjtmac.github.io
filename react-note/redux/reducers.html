核心概念 ​

什么是 Reducer？ ​

Reducer 是一个纯函数，接收当前的 state 和一个 action 作为参数，返回新的 state。它指定了应用状态的变化如何响应 actions 并发送到 store。

typescript
(state, action) => newState

Reducer 的特性 ​

  1. 纯函数：相同的输入总是产生相同的输出，没有副作用
  2. 不可变性：不直接修改 state，而是返回一个新的 state
  3. 单一职责：每个 reducer 只负责管理全局 state 中的一部分

基本结构 ​

一个典型的 reducer 函数结构如下：

typescript
const initialState = { /* 初始状态 */ };

function exampleReducer(state = initialState, action) {
  switch (action.type) {
    case 'ACTION_TYPE_1':
      return {
        ...state,
        // 根据 action 更新状态
      };
    case 'ACTION_TYPE_2':
      return {
        ...state,
        // 根据 action 更新状态
      };
    default:
      return state;
  }
}

Reducer 组合 ​

随着应用变得复杂，我们可以将 reducer 拆分为多个小的 reducer，每个负责状态树的一部分，然后使用 combineReducers 函数将它们组合起来：

typescript
import { combineReducers } from 'redux';
import todosReducer from './todosReducer';
import filterReducer from './filterReducer';

const rootReducer = combineReducers({
  todos: todosReducer,
  filter: filterReducer
});

export default rootReducer;

实践应用 ​

在我们的 Todo 应用中，我们实现了以下 Reducer：

1. Todo Reducer ​

typescript
import { ADD_TODO, TOGGLE_TODO, REMOVE_TODO } from '../actions/types';

const initialState = [];

const todoReducer = (state = initialState, action) => {
  switch (action.type) {
    case ADD_TODO:
      return [
        ...state,
        {
          id: Date.now(),
          text: action.payload.text,
          completed: false
        }
      ];
    
    case TOGGLE_TODO:
      return state.map(todo => 
        todo.id === action.payload.id 
          ? { ...todo, completed: !todo.completed } 
          : todo
      );
    
    case REMOVE_TODO:
      return state.filter(todo => todo.id !== action.payload.id);
    
    default:
      return state;
  }
};

export default todoReducer;

2. Filter Reducer ​

typescript
import { SET_FILTER, FilterTypes } from '../actions/types';

const initialState = FilterTypes.ALL;

const filterReducer = (state = initialState, action) => {
  switch (action.type) {
    case SET_FILTER:
      return action.payload.filter;
    
    default:
      return state;
  }
};

export default filterReducer;

最佳实践 ​

  1. 保持 Reducer 纯净：不要在 reducer 中执行 API 调用、生成随机值或修改传入的参数
  2. 使用不可变更新模式：使用展开运算符、map、filter 等方法创建新对象和数组
  3. 拆分 Reducer：将大型 reducer 拆分为更小、更专注的 reducer
  4. 使用 combineReducers：组合多个 reducer 以管理复杂的状态树
  5. 初始化默认状态：为每个 reducer 提供默认的初始状态
  6. 处理未知 Action：始终包含 default 分支，返回当前 state

个人总结与心得 ​

Reducer 是 Redux 中最核心的概念之一，它定义了应用状态如何变化。通过遵循纯函数和不可变性原则，Reducer 使状态变化变得可预测和可追踪。将 Reducer 拆分为更小的、专注于特定领域的 Reducer 可以使代码更加模块化和可维护。
