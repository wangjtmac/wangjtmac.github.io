Redux 中间件 ​

Redux 中间件提供了一个在 action 被发送到 reducer 之前拦截和处理 action 的机制。中间件是 Redux 处理副作用（如异步操作、日志记录、错误报告等）的主要方式。

中间件概念 ​

什么是中间件？ ​

中间件是一个函数，它可以：

  1. 拦截 dispatched actions
  2. 执行副作用（如 API 调用）
  3. 修改、延迟或停止 actions
  4. 访问 store 的 state

中间件的执行流程 ​

Action → Middleware 1 → Middleware 2 → ... → Reducer → New State

常用中间件 ​

1. redux-thunk ​

允许 action creators 返回函数而不是普通对象。

安装：

bash
npm install redux-thunk

配置：

jsx
import { createStore, applyMiddleware } from 'redux'
import thunk from 'redux-thunk'
import rootReducer from './reducers'

const store = createStore(
  rootReducer,
  applyMiddleware(thunk)
)

使用示例：

jsx
// Action Creators
const fetchUser = (userId) => {
  return async (dispatch, getState) => {
    dispatch({ type: 'FETCH_USER_START' })
    
    try {
      const response = await fetch(`/api/users/${userId}`)
      const user = await response.json()
      dispatch({ type: 'FETCH_USER_SUCCESS', payload: user })
    } catch (error) {
      dispatch({ type: 'FETCH_USER_ERROR', payload: error.message })
    }
  }
}

// 在组件中使用
dispatch(fetchUser(123))

2. redux-saga ​

使用 ES6 Generator 函数处理副作用的中间件。

安装：

bash
npm install redux-saga

配置：

jsx
import { createStore, applyMiddleware } from 'redux'
import createSagaMiddleware from 'redux-saga'
import rootReducer from './reducers'
import rootSaga from './sagas'

const sagaMiddleware = createSagaMiddleware()

const store = createStore(
  rootReducer,
  applyMiddleware(sagaMiddleware)
)

sagaMiddleware.run(rootSaga)

Saga 示例：

jsx
// sagas/userSaga.js
import { call, put, takeEvery } from 'redux-saga/effects'

function* fetchUser(action) {
  try {
    yield put({ type: 'FETCH_USER_START' })
    const user = yield call(fetch, `/api/users/${action.payload.userId}`)
    const userData = yield call([user, 'json'])
    yield put({ type: 'FETCH_USER_SUCCESS', payload: userData })
  } catch (error) {
    yield put({ type: 'FETCH_USER_ERROR', payload: error.message })
  }
}

function* watchFetchUser() {
  yield takeEvery('FETCH_USER_REQUEST', fetchUser)
}

export default function* rootSaga() {
  yield watchFetchUser()
}

记录每个 action 和状态变化的中间件。

安装：

bash

使用：

import { createStore, applyMiddleware } from 'redux'

const store = createStore(
  rootReducer,
  applyMiddleware(logger)
)

自定义中间件 ​

基本结构 ​

中间件是一个三层嵌套的函数：

jsx
const customMiddleware = (store) => (next) => (action) => {
  // 中间件逻辑
  console.log('dispatching', action)
  
  // 调用下一个中间件或 reducer
  const result = next(action)
  
  console.log('next state', store.getState())
  return result
}

日志中间件示例 ​

jsx
const loggerMiddleware = (store) => (next) => (action) => {
  console.group(action.type)
  console.info('dispatching', action)
  console.log('prev state', store.getState())
  
  const result = next(action)
  
  console.log('next state', store.getState())
  console.groupEnd()
  
  return result
}

错误处理中间件 ​

jsx
const errorMiddleware = (store) => (next) => (action) => {
  try {
    return next(action)
  } catch (error) {
    console.error('Redux error:', error)
    
    // 发送错误到监控服务
    if (typeof window !== 'undefined' && window.Sentry) {
      window.Sentry.captureException(error)
    }
    
    // 可以 dispatch 一个错误 action
    store.dispatch({
      type: 'GLOBAL_ERROR',
      payload: error.message
    })
    
    throw error
  }
}

API 中间件 ​

jsx
const apiMiddleware = (store) => (next) => (action) => {
  // 只处理 API actions
  if (!action.meta || !action.meta.api) {
    return next(action)
  }

  const { url, method = 'GET', data } = action.meta.api
  const { type } = action

  // Dispatch 开始 action
  next({ type: `${type}_START` })

  // 执行 API 调用
  return fetch(url, {
    method,
    headers: {
      'Content-Type': 'application/json',
    },
    body: data ? JSON.stringify(data) : undefined,
  })
    .then(response => response.json())
    .then(data => {
      // Dispatch 成功 action
      next({
        type: `${type}_SUCCESS`,
        payload: data
      })
    })
    .catch(error => {
      // Dispatch 失败 action
      next({
        type: `${type}_FAILURE`,
        payload: error.message
      })
    })
}

// 使用示例
dispatch({
  type: 'FETCH_USERS',
  meta: {
    api: {
      url: '/api/users',
      method: 'GET'
    }
  }
})

Redux Toolkit 中的中间件 ​

默认中间件 ​

Redux Toolkit 的 configureStore 默认包含：

  • redux-thunk
  • 开发环境的序列化检查中间件
  • 开发环境的不可变性检查中间件
jsx
import { configureStore } from '@reduxjs/toolkit'

const store = configureStore({
  reducer: {
    // reducers
  },
  // 自动包含 thunk 和开发工具
})

添加自定义中间件 ​

jsx
import { configureStore } from '@reduxjs/toolkit'
import logger from 'redux-logger'

const store = configureStore({
  reducer: {
    // reducers
  },
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware()
      .concat(logger)
      .concat(customMiddleware),
})

禁用默认中间件 ​

jsx
const store = configureStore({
  reducer: {
    // reducers
  },
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware({
      thunk: false,
      serializableCheck: false,
      immutableCheck: false,
    }),
})

中间件组合 ​

中间件执行顺序 ​

中间件按照添加的顺序执行：

jsx
const store = createStore(
  reducer,
  applyMiddleware(
    middleware1, // 第一个执行
    middleware2, // 第二个执行
    middleware3  // 第三个执行
  )
)

条件中间件 ​

jsx
const conditionalMiddleware = (condition) => (store) => (next) => (action) => {
  if (condition(action, store.getState())) {
    // 执行特定逻辑
    console.log('Condition met for action:', action.type)
  }
  return next(action)
}

// 使用
const isDevMode = process.env.NODE_ENV === 'development'
const devOnlyMiddleware = conditionalMiddleware(
  (action, state) => isDevMode && action.type.includes('DEBUG')
)

异步中间件对比 ​

Redux Thunk vs Redux Saga ​

特性Redux ThunkRedux Saga
学习曲线简单复杂
功能基础异步处理强大的副作用管理
测试较难测试易于测试
取消操作不支持支持
并发控制有限强大
包大小小大

选择建议 ​

  • 简单项目：使用 Redux Thunk
  • 复杂异步流程：使用 Redux Saga
  • 现代项目：考虑 RTK Query

最佳实践 ​

  1. 保持中间件简单：每个中间件只做一件事
  2. 错误处理：在中间件中处理错误
  3. 性能考虑：避免在中间件中进行昂贵的操作
  4. 开发工具：在开发环境使用日志中间件
  5. 测试：为自定义中间件编写测试
  6. 文档：为复杂中间件编写文档

调试中间件 ​

使用 Redux DevTools ​

jsx
import { configureStore } from '@reduxjs/toolkit'

const store = configureStore({
  reducer: rootReducer,
  devTools: process.env.NODE_ENV !== 'production',
})

自定义调试中间件 ​

jsx
const debugMiddleware = (store) => (next) => (action) => {
  if (process.env.NODE_ENV === 'development') {
    console.log('Action:', action)
    console.log('State before:', store.getState())
  }
  
  const result = next(action)
  
  if (process.env.NODE_ENV === 'development') {
    console.log('State after:', store.getState())
  }
  
  return result
}

中间件是 Redux 生态系统的重要组成部分，它们提供了强大的扩展能力，让你可以优雅地处理副作用和增强 Redux 的功能。
