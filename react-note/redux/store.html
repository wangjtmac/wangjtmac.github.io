Redux Store ​

核心概念 ​

什么是 Store？ ​

Store 是存储应用状态的对象。在 Redux 应用中，只有一个单一的 store。Store 有以下职责：

  1. 存储应用的状态（state）
  2. 通过 getState() 方法访问状态
  3. 通过 dispatch(action) 方法更新状态
  4. 通过 subscribe(listener) 方法注册监听器
  5. 通过 subscribe(listener) 返回的函数注销监听器

创建 Store ​

使用 Redux 的 createStore 函数创建 store：

javascript
import { createStore } from 'redux';
import rootReducer from './reducers';

const store = createStore(rootReducer);

createStore 函数接受三个参数：

  1. reducer（必需）：一个返回下一个状态树的函数
  2. preloadedState（可选）：初始状态
  3. enhancer（可选）：store 增强器，如应用中间件

Store 方法 ​

getState() ​

返回应用当前的状态树：

javascript
const state = store.getState();
console.log(state);

dispatch(action) ​

分发 action 是改变状态的唯一方法：

javascript
store.dispatch({ type: 'ADD_TODO', payload: { text: '学习 Redux' } });

subscribe(listener) ​

添加一个变化监听器，每当 dispatch action 时都会执行：

javascript
const unsubscribe = store.subscribe(() => {
  console.log('状态更新了:', store.getState());
});

// 稍后取消订阅
unsubscribe();

replaceReducer(nextReducer) ​

替换 store 当前使用的 reducer，这对于代码分割和动态加载 reducer 很有用：

javascript
import nextReducer from './nextReducer';
store.replaceReducer(nextReducer);

Store 增强器 ​

Store 增强器是一个函数，用于增强 store 的功能。最常用的增强器是 applyMiddleware，它允许你使用中间件来扩展 Redux：

javascript
import { createStore, applyMiddleware } from 'redux';
import thunk from 'redux-thunk';
import rootReducer from './reducers';

const store = createStore(
  rootReducer,
  applyMiddleware(thunk)
);

Redux DevTools ​

Redux DevTools 是一个强大的开发工具，可以让你实时查看每个 action 如何改变状态，甚至可以"时间旅行"到之前的状态：

javascript
import { createStore } from 'redux';
import rootReducer from './reducers';

const store = createStore(
  rootReducer,
  window.__REDUX_DEVTOOLS_EXTENSION__ && window.__REDUX_DEVTOOLS_EXTENSION__()
);

实践应用 ​

在我们的 Todo 应用中，我们创建了一个 store 来管理应用的状态：

javascript
// src/store/index.js
import { createStore } from 'redux';
import rootReducer from './reducers';

const configureStore = () => {
  // 启用 Redux DevTools 扩展
  const devTools = 
    window.__REDUX_DEVTOOLS_EXTENSION__ && 
    window.__REDUX_DEVTOOLS_EXTENSION__();
  
  // 创建 Store
  const store = createStore(rootReducer, devTools);
  
  return store;
};

export default configureStore;

然后在应用的入口文件中使用 Provider 组件将 store 提供给整个应用：

javascript
// src/App.js
import React from 'react';
import { Provider } from 'react-redux';
import configureStore from './store';
import TodoApp from './components/TodoApp';

const store = configureStore();

const App = () => (
  <Provider store={store}>
    <TodoApp />
  </Provider>
);

export default App;

最佳实践 ​

  1. 单一 Store：Redux 应用应该只有一个 store
  2. 使用 Redux DevTools：它可以帮助你调试和理解状态变化
  3. 配置 Store 的函数：创建一个函数来配置 store，而不是直接导出 store 实例
  4. 初始状态：考虑从服务器或本地存储加载初始状态
  5. 状态范式化：对于复杂的状态，考虑使用范式化的结构
  6. 状态选择器：创建选择器函数来访问状态，而不是直接访问 state 对象

与其他知识点的关联 ​

  • Action：通过 dispatch 方法将 action 发送到 store
  • Reducer：store 将 action 传递给 reducer，并使用 reducer 返回的新状态
  • React-Redux：Provider 组件使 React 组件可以访问 Redux store
  • 中间件：扩展 Redux 的功能，如处理异步操作

个人总结与心得 ​

Store 是 Redux 应用的核心，它将 Action 和 Reducer 连接起来，形成完整的数据流。虽然 Store 的 API 非常简单，但它提供了强大的功能，使我们能够以可预测的方式管理应用状态。

在实践中，我发现使用 Redux DevTools 非常有帮助，它让我可以清晰地看到每个 action 如何改变状态，这对于调试和理解应用的行为非常重要。同时，将 store 的配置封装在一个函数中也是一个好习惯，它使得 store 的创建更加灵活，便于测试和扩展。
