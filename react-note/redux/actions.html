核心概念 ​

什么是 Action？ ​

Action 是把数据从应用传到 store 的有效载荷（payload）。它们是 store 数据的唯一来源。Action 本质上是普通的 JavaScript 对象，用于描述发生了什么事情。

Action 的结构 ​

一个典型的 Action 对象包含以下属性：

typescript
{
  type: 'ADD_TODO',
  payload: {
    text: '学习 Redux',
    id: 1
  }
}
  • type：字符串常量，表示 Action 的类型，是必须的属性
  • payload：包含 Action 相关数据的对象，是可选的属性

Action 类型常量 ​

通常，我们会将 Action 类型定义为常量，以避免拼写错误和重复定义：

typescript
// actionTypes.js
export const ADD_TODO = 'ADD_TODO';
export const TOGGLE_TODO = 'TOGGLE_TODO';
export const SET_FILTER = 'SET_FILTER';

Action 创建函数 ​

Action 创建函数是生成 Action 的函数，使 Action 创建过程更加便捷和一致：

typescript
// actions.js
import { ADD_TODO, TOGGLE_TODO } from './actionTypes';

export const addTodo = (text) => ({
  type: ADD_TODO,
  payload: { text }
});

export const toggleTodo = (id) => ({
  type: TOGGLE_TODO,
  payload: { id }
});

实践应用 ​

在我们的 Todo 应用中，我们实现了以下 Action：

1. Action 类型定义 ​

typescript
// types.ts
export const ADD_TODO = 'ADD_TODO';
export const TOGGLE_TODO = 'TOGGLE_TODO';
export const REMOVE_TODO = 'REMOVE_TODO';
export const SET_FILTER = 'SET_FILTER';

export enum FilterTypes {
  ALL = 'ALL',
  COMPLETED = 'COMPLETED',
  ACTIVE = 'ACTIVE'
}

2. Action 创建函数 ​

typescript
// todoActions.ts
import { ADD_TODO, TOGGLE_TODO, REMOVE_TODO } from './types';

export const addTodo = (text) => ({
  type: ADD_TODO,
  payload: { text }
});

export const toggleTodo = (id) => ({
  type: TOGGLE_TODO,
  payload: { id }
});

export const removeTodo = (id) => ({
  type: REMOVE_TODO,
  payload: { id }
});

// filterActions.ts
import { SET_FILTER, FilterTypes } from './types';

export const setFilter = (filter) => ({
  type: SET_FILTER,
  payload: { filter }
});

最佳实践 ​

  1. 使用 Action 类型常量：避免拼写错误和重复定义
  2. 使用 Action 创建函数：保持 Action 创建的一致性
  3. 将相关 Action 分组：按功能或领域组织 Action

个人总结与心得 ​

Action 是 Redux 数据流的起点，设计良好的 Action 可以使应用状态变化更加清晰和可预测。通过定义明确的 Action 类型和创建函数，我们可以更好地组织和管理应用的状态变化。
