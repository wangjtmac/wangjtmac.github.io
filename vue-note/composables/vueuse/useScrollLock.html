useScrollLock ​

锁定元素的滚动。

useScrollLock 是 VueUse 库中的一个组合式函数，它提供了一种简单且响应式的方式来锁定或解锁指定 HTML 元素的滚动行为。这在需要防止页面内容滚动（例如当打开模态框、侧边抽屉或全屏图片预览时）时非常有用。它通过操作元素的 CSS overflow 属性来阻止滚动，并在 iOS 设备上提供额外的触摸事件阻止机制，以确保在不同平台上都能有效锁定滚动。

基本用法 ​

ts
import { useScrollLock } from '@vueuse/core'
import { ref } from 'vue'

const body = ref(document.body)
const isLocked = useScrollLock(body)

// 锁定滚动
isLocked.value = true

// 解锁滚动
isLocked.value = false

// 示例：在模板中通过按钮切换滚动锁定状态
// <template>
//   <button @click="isLocked = !isLocked">
//     {{ isLocked ? '解锁滚动' : '锁定滚动' }}
//   </button>
//   <div style="height: 2000px; background: linear-gradient(to bottom, #eee, #ccc);">
//     页面内容
//   </div>
// </template>

你可以将 isLocked 作为一个响应式引用，通过设置其 true 或 false 来控制元素的滚动锁定状态。

类型定义 ​

ts
import type { Fn } from '@vueuse/shared'
import type { MaybeRefOrGetter } from 'vue'

declare function useScrollLock(
  element: MaybeRefOrGetter<HTMLElement | SVGElement | Window | Document | null | undefined>,
  initialState?: boolean,
): import("vue").ComputedRef<boolean>

参数 ​

useScrollLock 函数接受以下两个参数：

  1. element: MaybeRefOrGetter<HTMLElement | SVGElement | Window | Document | null | undefined>：

    • 类型：MaybeRefOrGetter<HTMLElement | SVGElement | Window | Document | null | undefined>
    • 描述：要锁定滚动的目标元素。可以是 DOM 元素、Vue ref、Window 对象或 Document 对象。通常，你会将其设置为 document.body 或一个特定的可滚动容器。

  2. initialState?: boolean：可选，默认为 false。表示滚动锁定的初始状态。如果设置为 true，则在函数初始化时立即锁定滚动。

返回值 ​

useScrollLock 函数返回一个 ComputedRef<boolean> 类型的响应式引用：

  • ComputedRef<boolean>：一个计算属性，其值为 true 表示滚动已锁定，false 表示滚动未锁定。你可以通过设置此 ref 的值来控制锁定状态。

工作原理 ​

useScrollLock 的实现主要通过以下几种方式来锁定滚动：

  1. CSS overflow: hidden：当 isLocked.value 设置为 true 时，函数会将目标元素的 style.overflow 属性设置为 hidden。这会阻止元素内部内容的滚动。在解锁时，它会将 overflow 恢复到原始值。
  2. WeakMap 存储初始 overflow：为了在解锁时能够正确恢复元素的原始 overflow 样式，useScrollLock 使用 WeakMap 来存储每个元素的初始 overflow 值。这样可以避免覆盖用户或 CSS 规则设置的原始样式。
  3. iOS 触摸事件阻止：在 iOS 设备上，仅设置 overflow: hidden 可能不足以完全阻止滚动（特别是当内容超出视口时，Safari 可能会触发弹性滚动）。因此，useScrollLock 会额外监听 touchmove 事件，并通过调用 event.preventDefault() 来阻止默认的触摸滚动行为。为了避免阻止用户在可滚动区域内的滚动，它会检查触摸事件的目标元素及其父级是否具有 overflow: scroll 或 overflow: auto 样式。
  4. tryOnScopeDispose 清理：当组件或作用域被销毁时，tryOnScopeDispose 会自动调用 unlock 函数，确保滚动锁定被正确解除，避免副作用残留。

使用场景 ​

  • 模态框/弹窗：当显示全屏模态框时，阻止背景页面的滚动，提升用户体验。
  • 侧边抽屉/导航菜单：当侧边菜单展开时，防止主内容区域滚动。
  • 图片查看器：当用户查看全屏图片或画廊时，锁定背景滚动。
  • 拖拽操作：在某些拖拽场景下，可能需要暂时禁用页面的整体滚动。
  • 游戏或交互式应用：在特定的交互状态下，需要完全控制滚动行为。

注意事项 ​

  • 目标元素：通常建议将 document.body 作为 element 参数来锁定整个页面的滚动。如果你只希望锁定某个特定容器的滚动，请确保该容器有明确的高度和 overflow 属性。
  • iOS 兼容性：iOS 上的滚动锁定行为较为特殊，useScrollLock 已对此进行了处理。但仍需在真实 iOS 设备上进行测试，以确保效果符合预期。
  • 样式覆盖：如果你在 CSS 中为目标元素设置了 !important 的 overflow 规则，useScrollLock 可能无法有效覆盖。在这种情况下，你需要调整 CSS 优先级或寻找其他锁定方法。
  • 用户体验：在锁定滚动时，要考虑给用户提供清晰的视觉反馈，表明当前页面处于锁定状态，以及如何解除锁定。避免长时间锁定滚动而导致用户困惑。
  • 可访问性：确保滚动锁定不会影响屏幕阅读器用户或键盘导航的可访问性。通常，模态框等元素应该捕获焦点并限制 Tab 键导航。

实际应用示例 ​

以下是一个简单的 Vue 组件，演示如何使用 useScrollLock 来在模态框打开时锁定 body 的滚动，并在模态框关闭时解锁。

vue
<template>
  <div class="demo-container">
    <h2>useScrollLock 演示</h2>

    <p class="note">
      点击下面的按钮打开模态框。观察背景滚动条是否被锁定。
    </p>

    <button @click="showModal = true" class="open-modal-button">
      打开模态框
    </button>

    <div class="long-content">
      <p v-for="i in 30" :key="i">
        这是背景内容，用于演示滚动。行 {{ i }}
      </p>
    </div>

    <!-- 模态框 -->
    <Transition name="modal-fade">
      <div v-if="showModal" class="modal-overlay">
        <div class="modal-content">
          <h3>模态框标题</h3>
          <p>这是模态框的内容。当模态框打开时，背景滚动应该被锁定。</p>
          <div class="modal-long-text">
            <p v-for="i in 10" :key="i">模态框内部可滚动内容 {{ i }}</p>
          </div>
          <button @click="showModal = false" class="close-modal-button">
            关闭模态框
          </button>
        </div>
      </div>
    </Transition>
  </div>
</template>

<script setup>
import { ref, watch } from 'vue'
import { useScrollLock } from '@vueuse/core'

const showModal = ref(false)

// 锁定 document.body 的滚动
const isBodyLocked = useScrollLock(document.body)

// 监听 showModal 的变化来控制滚动锁定状态
watch(showModal, (newVal) => {
  isBodyLocked.value = newVal
})
</script>
