useElementHover ​

响应式地跟踪 DOM 元素的悬停状态。

useElementHover 是 VueUse 库中的一个组合式函数，它提供了一种简单而响应式的方式来判断一个 DOM 元素是否被鼠标悬停。这对于创建交互式 UI 组件、显示工具提示或在用户与元素交互时触发特定行为非常有用。

它能够监听 mouseenter 和 mouseleave 事件，并返回一个表示悬停状态的布尔值 isHovered。

基本用法 ​

你可以将任何一个 HTMLElement 引用传递给 useElementHover，它将返回一个 ref，指示该元素当前是否被鼠标悬停。

vue
<template>
  <div>
    <h2>useElementHover 演示</h2>
    <div class="hover-area" ref="el">
      将鼠标悬停在我上面
    </div>
    <p>元素是否被悬停: <strong>{{ isHovered }}</strong></p>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useElementHover } from '@vueuse/core'

const el = ref(null)
const isHovered = useElementHover(el)
</script>

类型定义 ​

ts
import type { TimerHandle } from '@vueuse/shared'
import type { MaybeRefOrGetter, ShallowRef } from 'vue'
import type { ConfigurableWindow } from '../_configurable'
import type { MaybeComputedElementRef } from '../unrefElement'
import { computed, shallowRef } from 'vue'
import { defaultWindow } from '../_configurable'
import { onElementRemoval } from '../onElementRemoval'
import { unrefElement } from '../unrefElement'
import { useEventListener } from '../useEventListener'

export interface UseElementHoverOptions extends ConfigurableWindow {
  /**
   * 进入延迟 (毫秒)。
   * @default 0
   */
  delayEnter?: number
  /**
   * 离开延迟 (毫秒)。
   * @default 0
   */
  delayLeave?: number
  /**
   * 当元素从 DOM 中移除时是否触发离开事件。
   * @default false
   */
  triggerOnRemoval?: boolean
}

export declare function useElementHover(
  el: MaybeRefOrGetter<EventTarget | null | undefined>,
  options?: UseElementHoverOptions
): ShallowRef<boolean>

参数 ​

useElementHover 函数接受两个参数：

  1. el: MaybeRefOrGetter<EventTarget | null | undefined>：

    • 必需。要监听悬停状态的 DOM 元素、组件实例、ref 或 getter。通常是一个模板引用（ref）。

  2. options?: UseElementHoverOptions：

    • 可选。一个配置对象，可以包含以下属性：
      • delayEnter?: number：可选，默认为 0。当鼠标进入元素时，在实际将 isHovered 设置为 true 之前的延迟时间（毫秒）。这可以用于避免在鼠标快速掠过元素时频繁触发悬停状态。
      • delayLeave?: number：可选，默认为 0。当鼠标离开元素时，在实际将 isHovered 设置为 false 之前的延迟时间（毫秒）。这可以用于保持悬停状态，即使鼠标短暂离开了元素，例如在弹出菜单中。
      • triggerOnRemoval?: boolean：可选，默认为 false。一个布尔值。如果设置为 true，当元素从 DOM 中移除时，isHovered 会自动设置为 false。这在处理动态添加/移除的元素时很有用。
      • window?: Window (来自 ConfigurableWindow)：可选。允许你指定一个自定义的 Window 对象，通常用于测试环境或非浏览器环境。

返回值 ​

useElementHover 函数返回一个 ShallowRef<boolean>，表示被监听元素当前的悬停状态。

  • isHovered: ShallowRef<boolean>：一个浅层响应式引用。如果鼠标当前悬停在 el 元素上，则为 true，否则为 false。

工作原理 ​

useElementHover 的核心工作原理是利用了事件监听和可选的延迟机制：

  1. 事件监听：它通过 useEventListener 分别监听目标元素（el）的 mouseenter 和 mouseleave 事件。
    • 当 mouseenter 事件触发时，表示鼠标进入元素区域。
    • 当 mouseleave 事件触发时，表示鼠标离开元素区域。
  2. 延迟处理：
    • 如果设置了 delayEnter 或 delayLeave，函数会使用 setTimeout 来引入延迟。这意味着 isHovered 的状态不会立即改变，而是在延迟时间后才更新。在延迟期间，如果鼠标行为（进入/离开）发生反转，之前的定时器会被清除，避免不必要的触发。
  3. 状态更新：在经过可选的延迟后，isHovered 这个 shallowRef 的值会被更新为 true（进入）或 false（离开）。由于 shallowRef 只会追踪其 .value 的变化，这对于频繁更新的布尔值是高效的。
  4. 元素移除处理：如果 triggerOnRemoval 设置为 true，它会利用 onElementRemoval 钩子，当目标元素从 DOM 中移除时，强制将 isHovered 设置为 false，确保状态的正确性。

使用场景 ​

  • 工具提示 (Tooltips)：当鼠标悬停在某个图标或文本上时，显示一个包含更多信息的工具提示。
  • 导航菜单：创建在鼠标悬停时展开的子菜单。
  • 图片特效：当鼠标悬停在图片上时，显示叠加层、放大图片或播放动画。
  • 按钮交互：为按钮添加自定义的悬停效果，如背景色变化、阴影等。
  • 卡片组件：在鼠标悬停在卡片上时，显示额外操作按钮或高亮卡片。
  • 拖放交互：在拖放过程中，高亮显示可放置区域。

注意事项 ​

  • useTemplateRef vs ref：在 Vue 组合式 API 中，如果 el 是一个通过 <template ref="el"> 方式获取的模板引用，你需要确保 el.value 能够正确地指向 DOM 元素。直接使用 ref(null) 并将其绑定到 ref 属性是标准的做法。对于 VueUse 提供的 useTemplateRef，它会处理模板引用的响应式更新。
  • 性能：useElementHover 本身性能开销很小，因为它主要依赖于原生的 DOM 事件。延迟选项可以进一步优化用户体验，避免过度敏感的触发。
  • pointer-events CSS 属性：如果元素或其父元素设置了 pointer-events: none，鼠标事件将不会在该元素上触发，从而 useElementHover 也不会起作用。
  • 嵌套元素：mouseenter 和 mouseleave 事件是不冒泡的，这意味着它们只会在直接进入或离开监听元素时触发，而不会因为进入或离开其子元素而重复触发。这通常是期望的行为。

实际应用示例 ​

以下是一个结合了 delayEnter 和 delayLeave 选项的 useElementHover 演示，展示了如何创建一个带有延迟效果的工具提示。

vue
<template>
  <div class="demo-container">
    <h2>useElementHover 实际应用</h2>

    <p class="note">
      将鼠标悬停在下面的卡片上，注意工具提示的出现和消失会有一个短暂的延迟。
    </p>

    <div class="card-wrapper">
      <div class="card" ref="cardRef">
        <h3>悬停我查看详情</h3>
        <p>这是卡片内容。</p>
      </div>

      <div v-if="isHovered" class="tooltip" :class="{ 'tooltip-visible': isHovered }">
        这是一个带有延迟的工具提示！
      </div>
    </div>

    <div class="controls">
      <label>
        进入延迟 (ms):
        <input type="number" v-model.number="delayEnter" />
      </label>
      <label>
        离开延迟 (ms):
        <input type="number" v-model.number="delayLeave" />
      </label>
    </div>

    <p class="status">卡片悬停状态: <strong>{{ isHovered }}</strong></p>

    <div class="code-section">
      <h3>代码示例</h3>
      <pre><code>

const cardRef = ref(null)

const delayLeave = ref(500) // 默认离开延迟 500ms

    </div>
  </div>
</template>
