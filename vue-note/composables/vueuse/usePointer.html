usePointer ​

响应式地获取统一的指针（鼠标、触摸、笔）状态。

usePointer 是 VueUse 库中的一个组合式函数，它提供了一种统一且响应式的方式来跟踪各种指针设备（包括鼠标、触摸屏和手写笔）的状态。它封装了 Pointer Events API，允许你获取指针的坐标、压力、倾斜等详细信息，从而简化了跨设备和输入类型的交互逻辑。

基本用法 ​

ts
import { usePointer } from '@vueuse/core'

const { x, y, pressure, pointerType, isInside } = usePointer()

// x, y: 指针在页面上的坐标
// pressure: 指针的压力（0到1之间，0表示没有压力）
// pointerType: 指针类型 ('mouse', 'pen', 'touch')
// isInside: 布尔值，指示指针是否在监听目标元素内部

// 示例：在模板中显示指针位置和类型
// <template>
//   <p>X: {{ x }}</p>
//   <p>Y: {{ y }}</p>
//   <p>类型: {{ pointerType }}</p>
//   <p>是否在内部: {{ isInside }}</p>
// </template>

默认情况下，usePointer 监听 window 上的指针事件。你可以通过 target 选项指定一个特定的 DOM 元素作为监听目标。

类型定义 ​

ts
import type { MaybeRef, Ref } from 'vue'
import type { ConfigurableWindow } from '../_configurable'
import type { PointerType, Position } from '../types'

export interface UsePointerState extends Position {
  pressure: number
  pointerId: number
  tiltX: number
  tiltY: number
  width: number
  height: number
  twist: number
  pointerType: PointerType | null
}

export interface UsePointerOptions extends ConfigurableWindow {
  /**
   * 要监听的指针类型。
   *
   * @default ['mouse', 'touch', 'pen']
   */
  pointerTypes?: PointerType[]

  /**
   * 初始值。
   */
  initialValue?: MaybeRef<Partial<UsePointerState>>

  /**
   * 监听事件的目标元素。默认为 `window`。
   *
   * @default window
   */
  target?: MaybeRef<EventTarget | null | undefined> | Document | Window
}

export interface UsePointerReturn {
  x: Ref<number>
  y: Ref<number>
  pressure: Ref<number>
  pointerId: Ref<number>
  tiltX: Ref<number>
  tiltY: Ref<number>
  width: Ref<number>
  height: Ref<number>
  twist: Ref<number>
  pointerType: Ref<PointerType | null>
  isInside: Ref<boolean>
}

declare function usePointer(
  options?: UsePointerOptions,
): UsePointerReturn

参数 ​

usePointer 函数接受一个可选的 options 对象，可以包含以下属性：

  • pointerTypes?: PointerType[]：可选，默认为 ['mouse', 'touch', 'pen']。一个字符串数组，指定要监听的指针类型。PointerType 可以是 'mouse' (鼠标), 'touch' (触摸), 或 'pen' (手写笔)。
  • initialValue?: MaybeRef<Partial<UsePointerState>>：可选。指针状态的初始值。例如，可以设置 initialValue: { x: 100, y: 100 }。
  • target?: MaybeRef<EventTarget | null | undefined> | Document | Window：可选，默认为 defaultWindow (通常是全局 window 对象)。指定监听指针事件的目标元素。可以是 HTMLElement、Document、Window 或 Vue ref 对象。
  • window?: Window (来自 ConfigurableWindow)：可选。允许你指定一个自定义的 Window 对象，通常用于测试环境或 SSR。

返回值 ​

usePointer 函数返回一个包含多个响应式引用的对象，用于提供详细的指针状态信息：

  • x: Ref<number>：指针在页面上的水平坐标。
  • y: Ref<number>：指针在页面上的垂直坐标。
  • pressure: Ref<number>：指针的压力值，通常在 0 (无压力) 到 1 (最大压力) 之间。仅在支持压力的设备（如某些触摸屏或手写笔）上有效。
  • pointerId: Ref<number>：当前指针事件的唯一标识符。在多点触控或多笔输入场景下很有用。
  • tiltX: Ref<number>：指针与 Y 轴的平面夹角，范围从 -90 度到 90 度。仅在支持倾斜的设备上有效。
  • tiltY: Ref<number>：指针与 X 轴的平面夹角，范围从 -90 度到 90 度。仅在支持倾斜的设备上有效。
  • width: Ref<number>：接触区域的宽度（椭圆的长轴）。仅在支持接触区域的设备上有效。
  • height: Ref<number>：接触区域的高度（椭圆的短轴）。仅在支持接触区域的设备上有效。
  • twist: Ref<number>：Z 轴旋转量，范围从 0 到 359 度。仅在支持旋转的设备上有效。
  • pointerType: Ref<PointerType | null>：事件的指针类型，可以是 'mouse', 'pen', 'touch', 或 null (如果未检测到)。
  • isInside: Ref<boolean>：一个布尔值，指示指针是否在监听目标元素内部 (true) 或外部 (false)。

工作原理 ​

usePointer 的实现主要依赖于浏览器的 Pointer Events API。它通过 useEventListener 监听 pointerdown、pointermove 和 pointerup 事件，并在 handler 函数中处理这些事件：

  1. 事件监听：在指定的 target 元素上（默认为 window），注册 pointerdown (指针按下)、pointermove (指针移动) 和 pointerup (指针抬起) 事件的监听器。同时监听 pointerleave 事件来更新 isInside 状态。
  2. 数据提取：当指针事件触发时，事件对象 (PointerEvent) 包含了丰富的指针信息。usePointer 使用 objectPick 工具函数从 PointerEvent 中提取所需的属性（如 x, y, pressure, pointerType 等），并将其赋值给响应式状态 state.value。
  3. 类型过滤：通过 pointerTypes 选项，函数可以根据指针类型过滤事件，只处理感兴趣的类型。
  4. 响应式暴露：利用 Vue 的 toRefs 函数，将内部的响应式状态对象转换为一组单独的 ref 引用，从而可以在组件中直接解构和使用。

使用场景 ​

  • 统一处理各种输入：无需区分鼠标、触摸或笔事件，一套逻辑即可处理所有指针输入。
  • 拖拽和手势识别：获取精确的指针位置和压力信息，用于实现复杂的拖拽、缩放或旋转手势。
  • 绘图应用：利用压力和倾斜数据，实现更真实的笔刷效果。
  • 游戏开发：实现更灵活、更具表现力的用户输入控制。
  • 自定义控件：创建对鼠标、触摸和笔输入都有良好响应的自定义 UI 控件。

注意事项 ​

  • 浏览器兼容性：Pointer Events API 在现代浏览器中得到了广泛支持，但仍需注意目标用户的浏览器版本。旧版浏览器可能需要 Polyfill。
  • 性能：与所有事件监听器一样，频繁的指针事件（尤其是 pointermove）可能会对性能产生影响。通常建议在需要时才使用此函数，并在组件卸载时确保监听器被清理。
  • 设备特性：并非所有指针设备都支持所有特性（例如压力、倾斜、接触区域尺寸）。在使用这些特性之前，最好进行能力检测或提供回退方案。
  • CSS touch-action：在使用触摸事件时，可以考虑使用 CSS touch-action 属性来控制浏览器默认的触摸行为（如滚动、缩放），以避免与你的自定义手势冲突。

实际应用示例 ​

以下是一个简单的 Vue 组件，演示如何使用 usePointer 跟踪鼠标/触摸/笔的各种状态，并实时显示相关数据：

vue
<template>
  <div class="demo-container">
    <h2>usePointer 演示</h2>

    <p class="note">
      在下面的区域内移动鼠标、触摸屏幕或使用手写笔（如果您的设备支持）。
    </p>

    <div class="pointer-area" ref="targetRef">
      <p><b>{{ isInside ? '指针在区域内' : '指针在区域外' }}</b></p>
      <p>页面 X: {{ x.toFixed(2) }}</p>
      <p>页面 Y: {{ y.toFixed(2) }}</p>
      <p v-if="pressure !== 0">压力: {{ pressure.toFixed(4) }}</p>
      <p v-if="pointerId !== 0">指针 ID: {{ pointerId }}</p>
      <p>指针类型: {{ pointerType || '未知' }}</p>
      <p v-if="tiltX !== 0 || tiltY !== 0">倾斜: X({{ tiltX.toFixed(2) }}°), Y({{ tiltY.toFixed(2) }}°)</p>
      <p v-if="width !== 0 || height !== 0">接触尺寸: 宽({{ width.toFixed(2)}}), 高({{ height.toFixed(2) }})</p>
      <p v-if="twist !== 0">旋转: {{ twist.toFixed(2) }}°</p>
    </div>

    <div class="code-section">
      <h3>代码示例</h3>

</template>
