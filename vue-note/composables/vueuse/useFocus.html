useFocus ​

响应式地跟踪或设置 DOM 元素的焦点状态。

useFocus 是 VueUse 库中的一个组合式函数，它提供了一种响应式的方式来跟踪或设置 DOM 元素的焦点状态。你可以用它来监听元素何时获得焦点或失去焦点，也可以通过修改其返回值来程序化地控制元素的焦点。

这个函数对于构建可访问的用户界面、自定义输入行为或在特定条件下自动聚焦元素非常有用。

基本用法 ​

你可以将任何一个 HTMLElement 引用传递给 useFocus，它将返回一个 focused 的 WritableComputedRef，指示该元素当前是否被聚焦。通过修改 focused 的值，你可以控制元素的焦点状态。

vue
<template>
  <div>
    <h2>useFocus 演示</h2>
    <p>
      <button ref="buttonRef" @click="focusedInput = !focusedInput">
        点击我来聚焦/失焦输入框
      </button>
    </p>
    <input ref="inputRef" type="text" placeholder="这是一个输入框" />
    <p>输入框当前是否被聚焦: <strong>{{ focusedInput }}</strong></p>
    <p>按钮当前是否被聚焦: <strong>{{ focusedButton }}</strong></p>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useFocus } from '@vueuse/core'

const inputRef = ref(null)
const buttonRef = ref(null)

const { focused: focusedInput } = useFocus(inputRef)
const { focused: focusedButton } = useFocus(buttonRef)
</script>

类型定义 ​

ts
import type { WritableComputedRef } from 'vue'
import type { ConfigurableWindow } from '../_configurable'
import type { MaybeElementRef } from '../unrefElement'

export interface UseFocusOptions extends ConfigurableWindow {
  /**
   * 初始值。如果设置为 true，则会在目标元素上设置焦点。
   * @default false
   */
  initialValue?: boolean
  /**
   * 模拟 CSS 的 :focus-visible 行为。
   * 如果为 true，只有当焦点是通过键盘导航而不是鼠标点击时，`focused` 才会变为 true。
   * @default false
   */
  focusVisible?: boolean
  /**
   * 在聚焦元素时阻止页面滚动到该元素。
   * @default false
   */
  preventScroll?: boolean
}

export interface UseFocusReturn {
  /**
   * 如果读取为 true，则元素具有焦点。如果读取为 false，则元素没有焦点。
   * 如果设置为 true，则元素将被聚焦。如果设置为 false，元素将失去焦点。
   */
  focused: WritableComputedRef<boolean>
}

declare function useFocus(
  target: MaybeElementRef,
  options?: UseFocusOptions
): UseFocusReturn

参数 ​

useFocus 函数接受两个参数：

  1. target: MaybeElementRef：

    • 必需。要监听或设置焦点状态的 DOM 元素、组件实例、ref 或 getter。通常是一个模板引用（ref）。

  2. options?: UseFocusOptions：

    • 可选。一个配置对象，可以包含以下属性：
      • initialValue?: boolean：可选，默认为 false。一个布尔值。如果设置为 true，在函数初始化时（通常是组件挂载后）会立即尝试将焦点设置到目标元素上。
      • focusVisible?: boolean：可选，默认为 false。一个布尔值。如果设置为 true，focused 状态的变化会模仿 CSS 的 :focus-visible 伪类行为。这意味着只有当焦点是通过键盘（例如 Tab 键）导航到元素时，focused 才会变为 true。如果焦点是通过鼠标点击设置的，focused 将保持 false。这对于实现更细粒度的焦点样式控制很有用。
      • preventScroll?: boolean：可选，默认为 false。一个布尔值。当通过设置 focused.value = true 来程序化地聚焦元素时，如果设置为 true，浏览器将不会自动滚动页面以使该元素可见。这在你希望手动控制滚动行为时很有用。
      • window?: Window (来自 ConfigurableWindow)：可选。允许你指定一个自定义的 Window 对象，通常用于测试环境或非浏览器环境。

返回值 ​

useFocus 函数返回一个包含以下属性的对象：

  • focused: WritableComputedRef<boolean>：一个可写的计算属性。读取时，它指示目标元素是否当前被聚焦。写入时，如果设置为 true，目标元素将获得焦点；如果设置为 false，目标元素将失去焦点。

工作原理 ​

useFocus 的核心工作原理是结合了事件监听和 Vue 的响应式系统，以及对 DOM focus() 和 blur() 方法的封装：

  1. 目标元素解析：通过 unrefElement(target) 将传入的 target 转换为实际的 DOM 元素引用。这确保了无论是 ref、响应式对象还是原始 DOM 元素，都能被正确处理。
  2. 事件监听：
    • 它使用 useEventListener 监听目标元素的 focus 和 blur 事件。当这些事件触发时，内部的 innerFocused shallowRef 会相应地更新为 true 或 false。
    • 如果 focusVisible 选项为 true，focus 事件的回调会额外检查触发事件的元素是否匹配 :focus-visible 伪类，以实现仅在键盘导航时才更新 focused 状态的逻辑。
  3. 双向绑定 focused：
    • focused 是一个 WritableComputedRef。它的 get 方法直接返回 innerFocused.value，反映当前的焦点状态。
    • 它的 set 方法则实现了对焦点状态的程序化控制：当 focused 被设置为 true 时，它会调用目标元素的 focus() 方法（并根据 preventScroll 选项控制滚动行为）；当设置为 false 时，则调用 blur() 方法。
  4. 初始值设置：通过一个 watch 监听 targetElement 的变化，并在其首次可用时（immediate: true）将 focused.value 设置为 initialValue，从而在组件挂载后自动设置初始焦点。

使用场景 ​

  • 自动聚焦输入框：在页面加载或某个组件显示时，自动将焦点设置到特定的输入框，方便用户立即输入。
  • 自定义焦点管理：在复杂的表单或交互中，根据用户操作（例如，点击"下一步"按钮后，将焦点移到下一个表单字段）。
  • 可访问性 (Accessibility)：确保使用键盘导航的用户能够清晰地知道当前焦点所在位置，并通过 focusVisible 选项增强视觉反馈。
  • 模态框/弹窗：在模态框打开时将焦点捕获在模态框内部，并在关闭时将焦点返回到触发元素。
  • 搜索功能：点击搜索图标后，自动聚焦搜索输入框。
  • 游戏开发：在基于 DOM 的游戏中，控制玩家控制的元素的焦点。

注意事项 ​

  • 可聚焦元素：只有可聚焦的 DOM 元素（如 <input>, <button>, <select>, <textarea>, 具有 tabindex 属性的元素，或某些链接元素）才能响应 focus() 和 blur() 方法。如果你尝试聚焦一个不可聚焦的元素，它可能不会有任何效果。
  • preventScroll 选项：在使用 focused.value = true 强制聚焦时，preventScroll 很有用。但要注意，如果元素不在视口内，用户可能看不到焦点的变化。
  • focusVisible 选项：这个选项依赖于浏览器对 :focus-visible 伪类的支持。它主要用于视觉增强，不会改变实际的焦点行为。
  • DOM 元素生命周期：确保在 useFocus 作用的元素被挂载到 DOM 之后才进行操作，尤其是在 initialValue 为 true 的情况下。Vue 的模板引用会在组件挂载后才被赋值。
  • 性能：useFocus 自身性能开销很小，因为它主要处理 DOM 焦点的原生行为。

实际应用示例 ​

以下是一个完整的 Vue 组件，演示了如何使用 useFocus 来管理多个元素的焦点状态，并通过按钮来切换它们的焦点，同时展示了 initialValue 和 focusVisible 的效果。

vue
<template>
  <div class="demo-container">
    <h2>useFocus 实际应用</h2>

    <p class="note">
      点击下面的按钮来切换不同元素的焦点。注意输入框默认是聚焦的。
      尝试使用 Tab 键在元素之间切换焦点，观察 `:focus-visible` 的行为。
    </p>

    <div class="elements-wrapper">

        这是一个可聚焦的段落。

        placeholder="这是一个默认聚焦的输入框"

<script setup lang="ts">
import { ref } from 'vue'
import { useFocus } from '@vueuse/core'

const paragraphRef = ref<HTMLElement | null>(null)
const inputRef = ref<HTMLInputElement | null>(null)
const buttonRef = ref<HTMLButtonElement | null>(null)

// 跟踪段落的焦点状态，不模拟 :focus-visible (默认行为)
const { focused: focusedParagraph } = useFocus(paragraphRef, { focusVisible: false })

// 跟踪输入框的焦点状态，并设置初始聚焦
const { focused: focusedInput } = useFocus(inputRef, { initialValue: true })

// 跟踪按钮的焦点状态，模拟 :focus-visible
const { focused: focusedButton } = useFocus(buttonRef, { focusVisible: true })

const focusParagraph = () => {
  focusedParagraph.value = !focusedParagraph.value
}
const focusInput = () => {
  focusedInput.value = !focusedInput.value
}
const focusButton = () => {
  focusedButton.value = !focusedButton.value
}
</script>
