useMousePressed ​

响应式鼠标按下状态

useMousePressed 是 VueUse 库中的一个传感器类组合式函数，用于跟踪鼠标（或触摸）按钮的按下和释放状态。它能响应 mousedown 和 touchstart 事件来表示按下，并在 mouseup、mouseleave、touchend 和 touchcancel 事件时表示释放。这个函数非常适合需要检测用户是否按住鼠标或触摸屏幕的场景。

基本用法 ​

ts
import { useMousePressed } from '@vueuse/core'

const { pressed, sourceType } = useMousePressed()

// pressed 是一个布尔值，表示鼠标是否被按下
// sourceType 是触发事件的源类型，可以是 'mouse' 或 'touch'

默认情况下，useMousePressed 同时监听鼠标和触摸事件。如果只想监听鼠标事件，可以将 touch 选项设置为 false：

ts
import { useMousePressed } from '@vueuse/core'

const { pressed } = useMousePressed({ touch: false })

如果只想在特定元素上捕获 mousedown 和 touchstart 事件，可以通过传入元素的 ref 来指定 target：

vue
<script setup>
import { ref } from 'vue'
import { useMousePressed } from '@vueuse/core'

const el = ref(null)

const { pressed } = useMousePressed({ target: el })
</script>

<template>
  <div ref="el">
    只有点击此元素才会触发状态更新。
  </div>
</template>

配置选项 (MousePressedOptions) ​

MousePressedOptions 接口继承自 ConfigurableWindow，包含以下选项：

选项类型默认值描述
touchbooleantrue是否监听触摸事件 (touchstart, touchend, touchcancel)。设置为 false 则只响应鼠标事件。
dragbooleantrue是否监听拖拽事件 (dragstart, drop, dragend)。当拖拽开始时，pressed 状态为 true。
capturebooleanfalse是否在事件捕获阶段添加事件监听器。
initialValuebooleanfalsepressed 状态的初始值。
targetMaybeComputedElementRefwindow监听事件的目标元素。如果未指定，默认为 window。
onPressed(event: MouseEvent | TouchEvent | DragEvent) => voidundefined鼠标/触摸按下时调用的回调函数。
onReleased(event: MouseEvent | TouchEvent | DragEvent) => voidundefined鼠标/触摸释放时调用的回调函数。
windowWindowdefaultWindow要使用的窗口对象，通常不需要修改。

返回值 ​

useMousePressed 函数返回一个包含以下属性的对象：

属性类型描述
pressedRef<boolean>一个响应式布尔值，表示鼠标或触摸是否处于按下状态。
sourceTypeRef<UseMouseSourceType>触发当前按下/释放事件的源类型，可以是 'mouse'（鼠标事件）或 'touch'（触摸事件），在释放时为 null。

工作原理 ​

useMousePressed 通过 useEventListener 监听指定 target 元素（默认为 window）上的以下事件：

  • 按下事件：mousedown (鼠标)、touchstart (触摸)、dragstart (拖拽)
  • 释放事件：mouseup (鼠标)、mouseleave (鼠标离开窗口)、touchend (触摸结束)、touchcancel (触摸取消)、drop (拖拽释放)、dragend (拖拽结束)

当检测到按下事件时，pressed 引用会被设置为 true，sourceType 会更新为相应的类型（'mouse' 或 'touch'）。当检测到释放事件时，pressed 会被设置为 false，sourceType 会被重置为 null。

通过监听这些事件，useMousePressed 提供了一个简洁的响应式 API 来管理鼠标（或触摸）的按下状态。

源码实现 ​

typescript
import type { ConfigurableWindow } from '../_configurable'
import type { MaybeComputedElementRef } from '../unrefElement'
import type { UseMouseSourceType } from '../useMouse'
import { computed, shallowRef } from 'vue'
import { defaultWindow } from '../_configurable'
import { unrefElement } from '../unrefElement'
import { useEventListener } from '../useEventListener'

export interface MousePressedOptions extends ConfigurableWindow {
  /**
   * Listen to `touchstart` `touchend` events
   *
   * @default true
   */
  touch?: boolean

  /**
   * Listen to `dragstart` `drop` and `dragend` events
   *
   * @default true
   */
  drag?: boolean

  /**
   * Add event listerners with the `capture` option set to `true`
   * (see [MDN](https://developer.mozilla.org/en-US/docs/Web/API/EventTarget/addEventListener#capture))
   *
   * @default false
   */
  capture?: boolean

  /**
   * Initial values
   *
   * @default false
   */
  initialValue?: boolean

  /**
   * Element target to be capture the click
   */
  target?: MaybeComputedElementRef

  /**
   * Callback to be called when the mouse is pressed
   *
   * @param event
   */
  onPressed?: (event: MouseEvent | TouchEvent | DragEvent) => void

  /**
   * Callback to be called when the mouse is released
   *
   * @param event
   */
  onReleased?: (event: MouseEvent | TouchEvent | DragEvent) => void
}

/**
 * Reactive mouse pressing state.
 *
 * @see https://vueuse.org/useMousePressed
 * @param options
 */
export function useMousePressed(options: MousePressedOptions = {}) {
  const {
    touch = true,
    drag = true,
    capture = false,
    initialValue = false,
    window = defaultWindow,
  } = options

  const pressed = shallowRef(initialValue)
  const sourceType = shallowRef<UseMouseSourceType>(null)

  if (!window) {
    return {
      pressed,
      sourceType,
    }
  }

  const onPressed = (srcType: UseMouseSourceType) => (event: MouseEvent | TouchEvent | DragEvent) => {
    pressed.value = true
    sourceType.value = srcType
    options.onPressed?.(event)
  }
  const onReleased = (event: MouseEvent | TouchEvent | DragEvent) => {
    pressed.value = false
    sourceType.value = null
    options.onReleased?.(event)
  }

  const target = computed(() => unrefElement(options.target) || window)

  const listenerOptions = { passive: true, capture }
  useEventListener<MouseEvent>(target, 'mousedown', onPressed('mouse'), listenerOptions)

  useEventListener<MouseEvent>(window, 'mouseleave', onReleased, listenerOptions)
  useEventListener<MouseEvent>(window, 'mouseup', onReleased, listenerOptions)

  if (drag) {
    useEventListener<DragEvent>(target, 'dragstart', onPressed('mouse'), listenerOptions)

    useEventListener<DragEvent>(window, 'drop', onReleased, listenerOptions)
    useEventListener<DragEvent>(window, 'dragend', onReleased, listenerOptions)
  }

  if (touch) {
    useEventListener<TouchEvent>(target, 'touchstart', onPressed('touch'), listenerOptions)

    useEventListener<TouchEvent>(window, 'touchend', onReleased, listenerOptions)
    useEventListener<TouchEvent>(window, 'touchcancel', onReleased, listenerOptions)
  }

  return {
    pressed,
    sourceType,
  }
}

export type UseMousePressedReturn = ReturnType<typeof useMousePressed>

使用场景 ​

useMousePressed 在以下场景中非常有用：

  1. 按钮交互：创建自定义按钮效果，例如在按钮被按住时改变样式。
  2. 拖拽操作：作为拖拽功能的基础，检测用户何时开始按住元素进行拖拽。
  3. 游戏控制：在简单的基于鼠标/触摸的游戏中，检测玩家的输入按键状态。
  4. 长按事件：结合 setTimeout 或 useTimeout 来实现长按事件的检测。
  5. 画图应用：在画布上绘制时，检测鼠标是否持续按下。
  6. 辅助功能：为某些用户提供通过按键保持状态的交互方式。

注意事项 ​

  1. 事件冒泡与捕获：默认情况下，事件在冒泡阶段被监听。如果需要更早地捕获事件（例如阻止默认行为），可以将 capture 选项设置为 true。
  2. 多设备兼容性：在同时支持鼠标和触摸的设备上（如带有触摸屏的笔记本），要确保行为符合预期。根据需求，可能需要通过 touch 选项禁用触摸事件。
  3. 拖拽事件：当 drag 选项启用时，dragstart 也会触发 pressed 状态，并且 drop 和 dragend 会触发释放。这对于实现完整的拖拽功能很有帮助。
  4. mouseleave 事件：当鼠标离开监听目标（通常是窗口）时，即使鼠标按钮仍然按下，pressed 状态也会被重置为 false。这在某些场景下可能需要注意。
  5. sourceType 的使用：sourceType 属性可以帮助区分是鼠标事件还是触摸事件触发了状态变化，这在需要针对不同输入设备提供不同反馈时很有用。

与其他函数的对比 ​

函数描述区别
useMousePressed跟踪鼠标/触摸的按下状态仅关注鼠标/触摸按钮的按下和释放状态，不提供鼠标位置信息。
useMouse跟踪鼠标/触摸的当前位置和源类型提供鼠标/触摸的实时坐标和事件源信息。
useLongPress监听元素的长按事件专门用于检测元素是否被"长按"，通常会有一个按下的持续时间阈值。而 useMousePressed 只是提供瞬时的按下状态。

示例2：模拟长按效果 ​

这个示例结合 useMousePressed 和 setTimeout 来模拟一个长按事件。

vue
<template>
  <div class="long-press-container">
    <h2>模拟长按按钮</h2>
    <p>长按下面的按钮（至少 {{ longPressDuration / 1000 }} 秒）</p>
    <button 
      class="long-press-button"
      :class="{ 'is-pressing': isPressing, 'is-long-pressed': isLongPressed }"
      @mousedown="startPress"
      @mouseup="endPress"
      @mouseleave="endPress"
      @touchstart.prevent="startPress"
      @touchend.prevent="endPress"
      @touchcancel.prevent="endPress"
    >
      {{ buttonText }}
    </button>
    <p class="feedback-text">
      {{ feedbackMessage }}
    </p>
  </div>
</template>

<script setup>
import { ref, computed } from 'vue'
import { useMousePressed } from '@vueuse/core'

const longPressDuration = 2000 // 2秒
const isPressing = ref(false)
const isLongPressed = ref(false)
let pressTimeout = null

// 简单地使用 useMousePressed 来跟踪按下状态，尽管在此示例中手动管理事件监听器更直接
// const { pressed } = useMousePressed()

function startPress() {
  isPressing.value = true
  isLongPressed.value = false
  clearTimeout(pressTimeout)
  pressTimeout = setTimeout(() => {
    if (isPressing.value) { // 确认仍然按着
      isLongPressed.value = true
    }
  }, longPressDuration)
}

function endPress() {
  isPressing.value = false
  clearTimeout(pressTimeout)
  // 如果是长按，可能需要执行某些操作
  if (isLongPressed.value) {
    // console.log("执行长按操作！")
  }
  // 状态在长按操作完成后或释放时重置
  setTimeout(() => { // 延迟重置，以便用户看到状态
    isLongPressed.value = false
  }, 300) 
}

const buttonText = computed(() => {
  if (isLongPressed.value) return '长按成功！'
  if (isPressing.value) return '按住中...'
  return '按我'
})

const feedbackMessage = computed(() => {
  if (isLongPressed.value) return '您已成功长按按钮！'
  if (isPressing.value) return `请继续按住 ${((longPressDuration - (Date.now() - (pressTimeout._idleStart + pressTimeout._idleTimeout))) / 1000).toFixed(1)} 秒...`
  return ''
})

</script>
