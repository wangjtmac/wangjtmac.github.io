VueUse - 状态管理函数 ​

状态管理是前端开发中的核心需求，VueUse提供了多种状态管理相关的组合式函数，帮助我们更方便地管理应用状态。

useStorage / useLocalStorage / useSessionStorage ​

这组函数提供了在浏览器存储（localStorage或sessionStorage）中持久化响应式数据的能力。

useLocalStorage ​

useLocalStorage是最常用的本地存储函数，它将响应式状态与localStorage同步。

基本信息 ​

  • 库: VueUse
  • 类别: 状态管理/浏览器
  • 版本: 10.5.0

功能描述 ​

将响应式状态与localStorage同步，当状态变化时自动更新localStorage，当页面加载时从localStorage恢复状态。

基本语法 ​

js
const state = useLocalStorage(key, defaultValue, options?)

参数选项 ​

参数名类型默认值说明
keystring必填localStorage中的键名
defaultValueTundefined当localStorage中不存在该键时的默认值
optionsUseStorageOptions{}配置选项

配置选项 ​

选项名类型默认值说明
serializerSerializerJSON自定义序列化器
deepbooleantrue是否深度监听对象变化
listenToStorageChangesbooleantrue是否监听其他标签页的storage事件
shallowbooleanfalse是否使用shallowRef创建响应式引用
windowWindowglobalThis.window指定Window对象
eventFilterEventFilterundefined事件过滤器
onError(error: unknown) => voidconsole.error错误处理函数

返回值 ​

返回一个响应式引用（ref），其值与localStorage同步。

使用示例 ​

vue
<script setup>
import { useLocalStorage } from '@vueuse/core'

// 基本用法
const name = useLocalStorage('name', 'John Doe')

// 存储对象
const user = useLocalStorage('user', { name: 'John', age: 25 })

// 修改值（自动同步到localStorage）
function updateName() {
  name.value = 'Jane Doe'
}

function updateUser() {
  user.value.age = 26 // 深度监听，会自动同步
  // 或者整体替换
  user.value = { name: 'Jane', age: 26 }
}

// 使用自定义序列化器
const state = useLocalStorage(
  'custom-state',
  { complex: 'data' },
  {
    serializer: {
      read: (v) => v ? JSON.parse(v) : null,
      write: (v) => JSON.stringify(v)
    }
  }
)
</script>

<template>
  <div>
    <h2>Local Storage Demo</h2>
    <p>Name: {{ name }}</p>
    <p>User: {{ user.name }}, {{ user.age }}</p>
    <button @click="updateName">Update Name</button>
    <button @click="updateUser">Update User</button>
  </div>
</template>

实际应用场景 ​

  1. 用户偏好设置：保存用户的主题、语言等偏好设置
  2. 表单数据暂存：防止用户刷新页面丢失表单数据
  3. 应用状态持久化：保存应用的部分状态，如购物车内容
  4. 用户认证信息：存储JWT令牌等认证信息（非敏感信息）

思考与见解 ​

  1. 优雅的持久化：相比手动操作localStorage，useLocalStorage提供了更优雅的API，自动处理序列化和反序列化。
  2. 响应式集成：与Vue的响应式系统无缝集成，使用体验一致。
  3. 跨标签页同步：默认监听storage事件，实现跨标签页的数据同步。
  4. 类型安全：完善的TypeScript类型定义，提供良好的类型推导。
  5. 默认值设计：当localStorage中不存在数据时，使用默认值，避免了手动判断的麻烦。

优缺点分析 ​

优点:

  • 简化了localStorage的使用，无需手动序列化/反序列化
  • 响应式数据自动同步到localStorage
  • 支持复杂数据类型（对象、数组等）
  • 支持跨标签页同步
  • 完善的TypeScript支持

缺点:

  • 对大型对象频繁更新可能影响性能
  • localStorage有存储大小限制（通常为5MB）
  • 所有数据都以明文形式存储，不适合敏感信息

与其他库对比 ​

  • Pinia持久化插件：更适合全局状态管理，而useLocalStorage更适合组件级别的状态持久化
  • Vuex-persistedstate：类似功能，但专为Vuex设计，useLocalStorage更灵活且与组合式API一致
  • Vue-Hooks-Plus的useLocalStorageState：功能类似，但VueUse的实现更成熟，选项更丰富

useSessionStorage ​

useSessionStorage与useLocalStorage用法完全相同，区别在于它使用sessionStorage存储数据，这意味着数据只在当前会话期间有效，关闭标签页后数据会被清除。

vue
<script setup>
import { useSessionStorage } from '@vueuse/core'

// 会话级别的临时数据
const tempData = useSessionStorage('temp-data', { step: 1, data: {} })
</script>

useStorage ​

useStorage是一个更通用的函数，允许你选择存储类型（localStorage或sessionStorage）。

vue
<script setup>
import { useStorage } from '@vueuse/core'

// 使用localStorage
const localData = useStorage('local-data', { count: 0 }, localStorage)

// 使用sessionStorage
const sessionData = useStorage('session-data', { count: 0 }, sessionStorage)
</script>

实现原理 ​

useStorage系列函数的核心实现原理如下：

  1. 创建一个响应式引用（ref）
  2. 初始化时，尝试从存储中读取数据，如果不存在则使用默认值
  3. 使用watch监听响应式引用的变化，当值变化时更新存储
  4. 监听storage事件，当其他标签页更新存储时同步更新本地的响应式引用

简化的实现示例：

js
function useStorage(key, defaultValue, storage = localStorage) {
  // 尝试从存储中读取数据
  const rawValue = storage.getItem(key)
  const data = rawValue ? JSON.parse(rawValue) : defaultValue
  
  // 创建响应式引用
  const state = ref(data)
  
  // 监听变化并更新存储
  watch(
    state,
    (newValue) => {
      if (newValue === null || newValue === undefined)
        storage.removeItem(key)
      else
        storage.setItem(key, JSON.stringify(newValue))
    },
    { deep: true }
  )
  
  // 监听storage事件（跨标签页同步）
  window.addEventListener('storage', (e) => {
    if (e.key === key && e.storageArea === storage) {
      state.value = e.newValue ? JSON.parse(e.newValue) : defaultValue
    }
  })
  
  return state
}

最佳实践 ​

  1. 合理使用localStorage和sessionStorage

    • localStorage：持久性数据，如用户偏好设置
    • sessionStorage：会话级临时数据，如向导步骤状态

  2. 避免存储敏感信息

    • 不要存储密码、信用卡信息等敏感数据
    • 对于认证令牌，考虑使用httpOnly cookie

  3. 处理存储限制

    • localStorage通常限制为5MB
    • 对于大型数据，考虑分片存储或使用IndexedDB

  4. 错误处理

    • 在隐私模式或存储已满的情况下，localStorage操作可能失败
    • 使用try/catch或useStorage的onError选项处理异常

  5. 类型安全

    • 利用TypeScript泛型确保类型安全
    • 示例：const count = useLocalStorage<number>('count', 0)

  6. 自定义序列化

    • 对于特殊数据类型（如Date、Map、Set），使用自定义序列化器
    • 对于敏感数据，可以实现加密序列化器

高级用法 ​

1. 与响应式对象结合 ​

vue
<script setup>
import { reactive, toRefs } from 'vue'
import { useLocalStorage } from '@vueuse/core'

const state = reactive({
  name: useLocalStorage('user-name', ''),
  theme: useLocalStorage('user-theme', 'light'),
  language: useLocalStorage('user-language', 'en')
})

// 解构并保持响应性
const { name, theme, language } = toRefs(state)
</script>

2. 自定义序列化器（加密示例） ​

vue
<script setup>
import { useLocalStorage } from '@vueuse/core'
import CryptoJS from 'crypto-js'

const SECRET_KEY = 'your-secret-key'

const encryptedData = useLocalStorage(
  'sensitive-data',
  { username: 'user', notes: 'private notes' },
  {
    serializer: {
      read: (v) => {
        if (!v) return null
        const bytes = CryptoJS.AES.decrypt(v, SECRET_KEY)
        return JSON.parse(bytes.toString(CryptoJS.enc.Utf8))
      },
      write: (v) => {
        return CryptoJS.AES.encrypt(
          JSON.stringify(v),
          SECRET_KEY
        ).toString()
      }
    }
  }
)
</script>

3. 条件性存储更新 ​

vue
<script setup>
import { useLocalStorage } from '@vueuse/core'
import { debounce } from 'lodash-es'

const debouncedData = useLocalStorage(
  'debounced-data',
  { text: '' },
  {
    eventFilter: debounce(500), // 500ms防抖后才更新localStorage
  }
)
</script>

总结 ​

VueUse的存储相关函数（useStorage、useLocalStorage、useSessionStorage）提供了一种优雅且强大的方式来持久化Vue应用中的状态。它们与Vue的响应式系统无缝集成，大大简化了本地存储的使用，同时提供了丰富的配置选项和类型安全保障。

在实际项目中，这些函数可以用于各种场景，从用户偏好设置到表单数据暂存，从应用状态持久化到跨标签页通信，极大地提升了开发效率和用户体验。
