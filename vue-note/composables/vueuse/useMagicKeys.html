useMagicKeys ​

响应式地跟踪按键状态，支持魔法般的组合键。

useMagicKeys 是 VueUse 库中的一个强大组合式函数，它允许你以响应式的方式监听键盘按键和组合键（快捷键）。无论是单个按键、多个按键的组合，还是修饰键（如 Shift, Ctrl, Alt）的组合，它都能轻松处理。

基本用法 ​

你可以通过解构 useMagicKeys() 的返回值来获取你感兴趣的按键的响应式引用。这些引用是 Ref<boolean> 类型，表示该按键当前是否被按下。

vue
<template>
  <div class="demo-container">
    <h2>useMagicKeys 基本用法</h2>
    <p class="note">尝试按下并释放键盘上的 `Shift` 键、`Space` 键和 `A` 键，观察下方状态变化。</p>

    <div class="key-status-box">
      <p>Shift 键: <strong :class="{ 'is-pressed': shift }">{{ shift ? '按下' : '未按下' }}</strong></p>
      <p>Space 键: <strong :class="{ 'is-pressed': space }">{{ space ? '按下' : '未按下' }}</strong></p>
      <p>A 键: <strong :class="{ 'is-pressed': a }">{{ a ? '按下' : '未按下' }}</strong></p>
    </div>

    <div class="combinations">
      <h3>组合键示例</h3>
      <p>Shift + A: <strong :class="{ 'is-pressed': shiftA }">{{ shiftA ? '按下' : '未按下' }}</strong></p>
      <p>Ctrl + S: <strong :class="{ 'is-pressed': ctrlS }">{{ ctrlS ? '按下' : '未按下' }}</strong></p>
    </div>

    <div class="current-keys">
      <h3>当前按下的所有键 (current Set)</h3>
      <p>{{ Array.from(current).join(', ') || '无' }}</p>
    </div>
  </div>
</template>

<script setup>
import { useMagicKeys } from '@vueuse/core'
import { watch } from 'vue'

// 获取单个按键的响应式引用
const { shift, space, a } = useMagicKeys()

// 监听 Space 键的按下和释放
watch(space, (isPressed) => {
  if (isPressed) {
    console.log('Space 键已被按下')
  } else {
    console.log('Space 键已被释放')
  }
})

// 获取组合键的响应式引用
// 可以使用 '+' 或 '_' 连接键名来表示组合键
const { 'Shift+A': shiftA, 'Ctrl+S': ctrlS } = useMagicKeys() // 或者 const keys = useMagicKeys(); const shiftA = keys['Shift+A'];

// 监听 Shift + A 组合键
watch(shiftA, (isPressed) => {
  if (isPressed) {
    console.log('Shift + A 组合键已被按下')
  } else {
    console.log('Shift + A 组合键已被释放')
  }
})

// 获取当前按下的所有键 (一个 Set)
const { current } = useMagicKeys()
</script>

类型定义 ​

ts
import type { ComputedRef, MaybeRefOrGetter } from 'vue'

export interface UseMagicKeysOptions<Reactive extends boolean> {
  /**
   * 返回一个响应式对象而不是 Ref 对象的对象
   * @default false
   */
  reactive?: Reactive
  /**
   * 监听事件的目标
   * @default window
   */
  target?: MaybeRefOrGetter<EventTarget>
  /**
   * 键的别名映射，所有键都应该是小写
   * { target: keycode }
   * @example { ctrl: "control" }
   * @default <预定义映射>
   */
  aliasMap?: Record<string, string>
  /**
   * 注册被动监听器
   * @default true
   */
  passive?: boolean
  /**
   * keydown/keyup 事件的自定义事件处理器。
   * 当你想应用自定义逻辑时很有用。
   * 当使用 `e.preventDefault()` 时，你需要将 `passive: false` 传递给 useMagicKeys()。
   */
  onEventFired?: (e: KeyboardEvent) => void | boolean
}

export interface MagicKeysInternal {
  /**
   * 当前按下的键的 Set 集合，存储原始键代码。
   * @see https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key
   */
  current: Set<string>
}

export type UseMagicKeysReturn<Reactive extends boolean>
  = Readonly<
    Omit<Reactive extends true
      ? Record<string, boolean>
      : Record<string, ComputedRef<boolean>>, keyof MagicKeysInternal>
      & MagicKeysInternal
  >

declare function useMagicKeys(options?: UseMagicKeysOptions<false>): UseMagicKeysReturn<false>
declare function useMagicKeys(options: UseMagicKeysOptions<true>): UseMagicKeysReturn<true>
declare function useMagicKeys(options: UseMagicKeysOptions<boolean>): any

参数 ​

useMagicKeys 函数接受一个可选的 options 对象作为参数，类型为 UseMagicKeysOptions，包含以下属性：

  • reactive?: boolean：
    • 可选，默认为 false。如果设置为 true，则返回一个响应式对象，其属性值是布尔类型；如果为 false，则返回一个包含 ComputedRef<boolean> 类型的对象。在模板中直接使用时，设置为 true 更方便。
  • target?: MaybeRefOrGetter<EventTarget>：
    • 可选，默认为 window。监听键盘事件的目标元素。可以是 window、document 或任何其他 DOM 元素。
  • aliasMap?: Record<string, string>：
    • 可选，默认为 VueUse 预定义的一些别名映射。一个对象，用于为键设置别名。例如 { ctrl: "control" } 可以让你使用 ctrl 来代替 control。所有别名和目标键都应该是小写。
  • passive?: boolean：
    • 可选，默认为 true。是否注册为被动事件监听器。如果你的 onEventFired 回调中会调用 e.preventDefault()，则需要将其设置为 false。
  • onEventFired?: (e: KeyboardEvent) => void | boolean：
    • 可选。一个自定义事件处理器，会在 keydown 和 keyup 事件触发时被调用。你可以在此函数中执行自定义逻辑，例如阻止默认行为（需要将 passive 设置为 false）。返回 false 将阻止内部的键状态更新（不推荐）。

返回值 ​

useMagicKeys 函数返回一个只读对象，其类型为 UseMagicKeysReturn，包含以下属性：

  • 动态属性：对于你通过解构或属性访问的每个键（例如 shift、space、Ctrl+A 等），都会返回一个响应式引用（如果 reactive 为 false，则为 ComputedRef<boolean>；如果 reactive 为 true，则为 boolean）。这些属性名对应于你请求的键名或组合键名。
  • current: Set<string>：一个 Set 对象，包含了当前所有被按下的键的原始 key 值（例如 control、shift、a 等）。你可以用它来检查任意键的组合状态，或者进行更复杂的逻辑判断。

工作原理 ​

useMagicKeys 通过 useEventListener 监听 keydown 和 keyup 事件来实现其功能：

  1. 事件监听：它在指定的目标（默认为 window）上监听 keydown 和 keyup 事件。
  2. 键状态更新：当 keydown 事件触发时，相应的键被添加到内部的 current Set 中，并且对应的响应式引用（或布尔值，取决于 reactive 选项）被设置为 true。当 keyup 事件触发时，键从 current Set 中移除，对应的引用设置为 false。
  3. 组合键处理：当访问 keys['Key1+Key2'] 这样的组合键时，useMagicKeys 会动态创建一个计算属性（或直接的布尔值），该属性会检查 Key1 和 Key2 是否同时被按下。这使得组合键的使用非常直观和灵活。
  4. 修饰键的特殊处理：对于像 Shift 和 Meta (Cmd/Windows 键) 这样的修饰键，useMagicKeys 会处理它们释放时可能存在的特殊行为（例如在 macOS 上 Meta 键释放不触发 keyup）。它通过内部追踪机制确保组合键在修饰键释放后正确重置。
  5. onEventFired 回调：允许开发者在按键事件被 useMagicKeys 处理之前或之后插入自定义逻辑。
  6. blur 和 focus 事件处理：为了在页面失去焦点或重新获得焦点时正确重置内部状态，useMagicKeys 也监听了 blur 和 focus 事件，并在这些事件发生时清空当前按下的键状态。

使用场景 ​

  • 快捷键/热键：为应用程序添加自定义快捷键，例如 Ctrl+S 保存、Shift+N 新建、Space 播放/暂停等。
  • 游戏控制：在基于 Web 的游戏中，处理复杂的键盘输入以控制角色或游戏行为。
  • 无障碍性：提供键盘导航选项，改善用户体验。
  • 表单输入增强：例如，在文本输入框中，按下 Enter 键提交表单，而按下 Shift+Enter 插入新行。
  • 调试工具：在开发过程中，使用特定的组合键来触发调试信息或开发者工具。
  • 多媒体播放器：通过键盘控制播放、暂停、快进、快退等。

注意事项 ​

  • 键名大小写：尽管 JavaScript 的 KeyboardEvent.key 属性值可能包含大写字母（如 Shift），但在 useMagicKeys 中，建议使用小写形式来访问键（例如 shift）。组合键中的键名也应使用小写，例如 Shift+A 对应的属性名是 shift_a 或 shift+a。
  • 组合键分隔符：组合键可以使用 + 或 _ 作为分隔符，例如 Ctrl+Alt+Delete 或 Ctrl_Alt_Delete。
  • e.preventDefault()：如果你在 onEventFired 回调中调用 e.preventDefault() 来阻止浏览器默认行为（例如 Ctrl+S 阻止保存弹窗），请务必将 useMagicKeys 的 passive 选项设置为 false，否则 preventDefault 将无效。
  • 性能：虽然 useMagicKeys 经过优化，但监听大量的键和复杂的组合可能会有轻微的性能开销。在大多数应用中，这不会成为问题。
  • 输入框中的行为：默认情况下，useMagicKeys 会在全局监听按键事件。如果你不希望在 <input>, <textarea> 或其他可编辑元素中触发快捷键，你需要结合 useActiveElement 等函数进行条件判断，如官方文档示例所示。

实际应用示例 ​

以下是一个更复杂的 useMagicKeys 示例，展示了如何结合 whenever 和 useActiveElement 来创建更智能的快捷键：

vue
<template>
  <div class="demo-container">
    <h2>useMagicKeys 实际应用：智能快捷键</h2>

    <p class="note">
      这个演示展示了如何创建只有在特定条件下才生效的快捷键。
      尝试在输入框外按下 `Tab` 键，然后在一个输入框内按下 `Tab` 键进行对比。
      同时，也展示了一个 `Ctrl+S` 的自定义保存快捷键。
    </p>

    <div class="input-section">
      <label for="input1">输入框 1:</label>
      <input id="input1" type="text" placeholder="在这里输入一些文本" />
    </div>

    <div class="input-section">
      <label for="input2">输入框 2:</label>
      <textarea id="input2" placeholder="在这里输入多行文本"></textarea>
    </div>

    <div class="status-box">
      <p>按下 Tab (仅输入框外生效): <strong :class="{ 'is-pressed': tabOutsideInput }">{{ tabOutsideInput ? '是' : '否' }}</strong></p>
      <p>按下 Ctrl + S: <strong :class="{ 'is-pressed': ctrlS }">{{ ctrlS ? '是' : '否' }}</strong></p>
      <p>当前按下的键: {{ Array.from(current).join(', ') || '无' }}</p>
    </div>
  </div>
</template>

<script setup>
import { ref, computed } from 'vue'
import { useMagicKeys, useActiveElement, whenever } from '@vueuse/core'
import { logicAnd } from '@vueuse/math'

const { tab, 'Ctrl+S': ctrlS, current } = useMagicKeys({ passive: false }) // passive: false 用于允许 preventDefault
const activeElement = useActiveElement()

// 判断当前焦点是否不在输入框或文本域中
const notUsingInput = computed(() => {
  const tagName = activeElement.value?.tagName
  return tagName !== 'INPUT' && tagName !== 'TEXTAREA'
})

// 组合条件：Tab 键按下 且 不在输入框内
const tabOutsideInput = logicAnd(tab, notUsingInput)

whenever(tabOutsideInput, () => {
  console.log('在输入框外按下了 Tab 键！')
  // 可以在这里执行一些自定义逻辑
})

whenever(ctrlS, () => {
  console.log('按下了 Ctrl + S 快捷键，执行保存操作！')
  alert('执行保存操作！(Ctrl+S)')
  // 在这里添加你的保存逻辑
})

</script>
