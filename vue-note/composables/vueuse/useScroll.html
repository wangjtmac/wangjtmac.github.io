useScroll ​

响应式地获取滚动位置和状态。

useScroll 是 VueUse 库中的一个组合式函数，它提供了一种简单且响应式的方式来跟踪指定元素或整个文档的滚动位置和状态。这个函数不仅返回水平和垂直滚动坐标 (x, y)，还提供了滚动状态 (isScrolling)、到达状态 (arrivedState - 是否滚动到顶部、底部、左侧或右侧) 和滚动方向 (directions) 等详细信息。它还允许你通过设置 x 和 y 的值来控制滚动行为，并支持平滑滚动。

基本用法 ​

ts
import { useScroll } from '@vueuse/core'
import { ref } from 'vue'

const scrollTarget = ref(null)

// 监听整个窗口的滚动
const { x, y, isScrolling, arrivedState, directions } = useScroll(window)

// 监听特定元素的滚动
const { 
  x: elX, 
  y: elY, 
  isScrolling: elIsScrolling, 
  arrivedState: elArrivedState, 
  directions: elDirections 
} = useScroll(scrollTarget)

// x, y: 当前水平/垂直滚动位置
// isScrolling: 是否正在滚动
// arrivedState: { left, right, top, bottom } - 是否到达边缘
// directions: { left, right, top, bottom } - 滚动方向

你可以将 scrollTarget ref 绑定到任何可滚动的 HTML 元素上，或者直接传入 window 或 document 来监听整个页面的滚动。

类型定义 ​

ts
import type { MaybeRefOrGetter } from 'vue'
import type { ConfigurableWindow } from '../_configurable'

export interface UseScrollOptions extends ConfigurableWindow {
  /**
   * 滚动事件的节流时间（毫秒），默认为 0（禁用节流）。
   *
   * @default 0
   */
  throttle?: number

  /**
   * 滚动结束时的检查时间。
   * 当配置了 `throttle` 时，此配置将设置为 (`throttle` + `idle`)。
   *
   * @default 200
   */
  idle?: number

  /**
   * 抵达状态的偏移量（像素）。
   * 允许在距离边缘一定像素时即视为抵达。
   */
  offset?: {
    left?: number
    right?: number
    top?: number
    bottom?: number
  }

  /**
   * 使用 MutationObserver 监视特定的 DOM 变化，
   * 例如属性修改、子节点添加或删除，或子树变化。
   * 默认 { mutation: false }。
   *
   * @default { mutation: false }
   */
  observe?: boolean | {
    mutation?: boolean
  }

  /**
   * 滚动时触发的回调函数。
   */
  onScroll?: (e: Event) => void

  /**
   * 滚动结束时触发的回调函数。
   */
  onStop?: (e: Event) => void

  /**
   * 滚动事件的事件监听器选项。
   *
   * @default {capture: false, passive: true}
   */
  eventListenerOptions?: boolean | AddEventListenerOptions

  /**
   * 可选指定滚动行为为 `auto` (默认，非平滑滚动) 或 `smooth` (平滑滚动)，
   * 当更改 `x` 或 `y` ref 时生效。
   *
   * @default 'auto'
   */
  behavior?: MaybeRefOrGetter<ScrollBehavior>

  /**
   * 错误回调。
   *
   * 默认将错误记录到 `console.error`。
   */
  onError?: (error: unknown) => void
}

export interface UseScrollReturn {
  x: import("vue").ComputedRef<number>
  y: import("vue").ComputedRef<number>
  isScrolling: import("vue").ShallowRef<boolean>
  arrivedState: {
    left: boolean
    right: boolean
    top: boolean
    bottom: boolean
  }
  directions: {
    left: boolean
    right: boolean
    top: boolean
    bottom: boolean
  }
  measure: () => void
}

declare function useScroll(
  element: MaybeRefOrGetter<HTMLElement | SVGElement | Window | Document | null | undefined>,
  options?: UseScrollOptions,
): UseScrollReturn

参数 ​

useScroll 函数接受以下两个参数：

  1. element: MaybeRefOrGetter<HTMLElement | SVGElement | Window | Document | null | undefined>：

    • 类型：MaybeRefOrGetter<HTMLElement | SVGElement | Window | Document | null | undefined>
    • 描述：要监听滚动事件的容器元素。可以是 DOM 元素、Vue ref、Window 对象或 Document 对象。

  2. options?: UseScrollOptions：可选的配置对象，可以包含以下属性：

    • throttle?: number：可选，默认为 0。滚动事件的节流时间（毫秒）。设置为大于 0 的值可以限制 onScroll 事件的触发频率，从而优化性能。
    • idle?: number：可选，默认为 200 毫秒。滚动结束后，多长时间（毫秒）才将 isScrolling 设置为 false 并触发 onStop。当 throttle 配置时，此配置将设置为 (throttle + idle)。
    • offset?: { left?: number; right?: number; top?: number; bottom?: number }：可选。一个对象，用于设置抵达状态的偏移量（像素）。例如，offset: { bottom: 100 } 意味着当滚动条距离底部 100px 以内时，arrivedState.bottom 就会变为 true。
    • observe?: boolean | { mutation?: boolean }：可选，默认为 { mutation: false }。是否使用 MutationObserver 来监视 DOM 变化。设置为 true 会监听元素子树的变化（添加/删除子节点），这在动态内容加载导致滚动条大小变化时很有用。如果只关心属性变化，可以设置为 { mutation: true }。
    • onScroll?: (e: Event) => void：可选。滚动事件触发时执行的回调函数。
    • onStop?: (e: Event) => void：可选。滚动停止时执行的回调函数。
    • eventListenerOptions?: boolean | AddEventListenerOptions：可选，默认为 { capture: false, passive: true }。传递给底层 addEventListener 的选项。passive: true 通常用于提高滚动性能。
    • behavior?: MaybeRefOrGetter<ScrollBehavior>：可选，默认为 'auto'。当通过设置 x 或 y 来控制滚动位置时，指定滚动的行为。可以是 'auto' (即时滚动) 或 'smooth' (平滑滚动)。也可以是一个响应式引用，以便动态改变。
    • onError?: (error: unknown) => void：可选。在内部操作发生错误时调用的回调函数。默认情况下会将错误记录到 console.error。
    • window?: Window (来自 ConfigurableWindow)：可选。允许你指定一个自定义的 Window 对象，通常用于测试环境或 SSR（服务端渲染）环境。

返回值 ​

useScroll 函数返回一个包含以下属性的响应式对象：

  • x: ComputedRef<number>：当前水平滚动位置的响应式引用。可以通过设置其值来控制水平滚动。
  • y: ComputedRef<number>：当前垂直滚动位置的响应式引用。可以通过设置其值来控制垂直滚动。
  • isScrolling: ShallowRef<boolean>：一个浅层响应式引用，指示元素当前是否正在滚动。在滚动开始时变为 true，在滚动结束（经过 idle 时间后）变为 false。
  • arrivedState: { left: boolean; right: boolean; top: boolean; bottom: boolean }：一个响应式对象，包含四个布尔值，指示滚动是否已到达容器的左侧、右侧、顶部或底部边缘。考虑了 offset 选项。
  • directions: { left: boolean; right: boolean; top: boolean; bottom: boolean }：一个响应式对象，包含四个布尔值，指示当前的滚动方向。例如，如果正在向下滚动，directions.bottom 为 true。
  • measure: () => void：一个函数，调用它可以手动强制重新计算滚动位置和 arrivedState。这在动态内容变化（例如添加或删除列表项导致滚动条大小变化）后非常有用，因为这些变化可能不会自动触发滚动事件。建议在 DOM 更新后（例如 nextTick 中）调用此函数。

工作原理 ​

useScroll 的核心工作原理是监听滚动容器的 scroll 事件，并结合节流、防抖和 MutationObserver 来提供精确和高效的滚动状态：

  1. 事件监听：通过 useEventListener 在目标元素上添加 scroll 事件监听器。
  2. 节流/防抖：为了防止在快速滚动时频繁更新状态导致性能问题，它会根据 throttle 选项对 scroll 事件进行节流处理。同时，isScrolling 状态的更新和 onStop 回调的触发会使用 throttle + idle 的时间进行防抖。
  3. 位置计算：在每次滚动事件触发时，它会读取滚动容器的 scrollLeft 和 scrollTop 属性，并更新 x 和 y 的内部值。
  4. 抵达状态 (arrivedState)：通过比较 scrollLeft/scrollTop 与容器的 clientWidth/clientHeight 和 scrollWidth/scrollHeight，结合 offset 选项，精确判断是否到达了滚动容器的各个边缘。考虑了 ARRIVED_STATE_THRESHOLD_PIXELS 来解决浏览器浮点数计算的精度问题。
  5. 滚动方向 (directions)：通过比较当前滚动位置与上一次滚动位置，来判断当前的滚动方向。
  6. MutationObserver (可选)：如果 observe.mutation 选项为 true，它会创建一个 MutationObserver 来监听滚动容器的子节点变化，当内容发生变化可能影响滚动条时，会触发 measure() 来重新计算状态。
  7. scrollTo 实现：x 和 y 的 setter 内部会调用元素的 scrollTo 方法来控制滚动位置，并支持 behavior 选项。

使用场景 ​

  • 滚动加载：与 useInfiniteScroll 配合使用，实现更精细的无限滚动控制。
  • 导航指示器：根据滚动位置高亮显示当前所在的章节导航。
  • 返回顶部/底部按钮：根据滚动位置的 arrivedState 来显示或隐藏返回顶部/底部的按钮。
  • 视差滚动：根据滚动位置改变元素样式，创建视差效果。
  • 图片懒加载：结合 useElementVisibility，在图片进入视口时加载。
  • 视频播放进度：跟踪视频播放器的滚动进度。
  • 动态调整布局：根据滚动方向或位置调整页面布局（例如，滚动时隐藏顶部导航栏）。

注意事项 ​

  • 滚动容器：确保你监听的元素确实是可滚动的（即内容超出了其可见区域，并且设置了 overflow: auto 或 overflow: scroll）。对于 window 或 document，它们默认是可滚动的。
  • 性能优化：对于高频滚动的场景，合理设置 throttle 和 idle 选项是关键，以平衡响应性和性能。如果不需要实时精确的滚动位置，可以增大节流时间。
  • 初始状态：在组件挂载时，useScroll 会自动初始化其状态。但在某些情况下，如果 DOM 在挂载后立即发生变化（例如异步内容加载），可能需要手动调用 measure() 来更新状态。
  • 滚动条样式：不同浏览器对滚动条的渲染方式可能有所不同，可能需要自定义样式以保持一致的用户体验。
  • 嵌套滚动：在处理嵌套滚动容器时，需要仔细考虑 element 参数的传递，以确保监听正确的滚动区域。

以下是一个简单的 Vue 组件，演示如何使用 useScroll 监听一个 div 元素的滚动，并显示其滚动位置、滚动状态和抵达状态。同时，演示了如何通过设置 x 和 y 来控制滚动。

<template>
  <div class="demo-container">
    <h2>useScroll 演示</h2>

    <p class="note">
      在下面的方框中滚动，观察滚动位置、状态和方向的变化。
    </p>

    <div class="controls">
      <button @click="y = 0">滚动到顶部</button>
      <button @click="y += 100">向下滚动 100px</button>
      <button @click="y = scrollContainer?.scrollHeight">滚动到底部</button>
      <button @click="x = 0">滚动到最左</button>
      <button @click="x += 100">向右滚动 100px</button>
      <button @click="x = scrollContainer?.scrollWidth">滚动到最右</button>
      <button @click="smoothScroll = !smoothScroll">
        {{ smoothScroll ? '切换为自动滚动' : '切换为平滑滚动' }}

      <p>平滑滚动: <b>{{ smoothScroll ? '开启' : '关闭' }}</b></p>

      <div class="content">

        </div>

      <ul>

      </ul>
