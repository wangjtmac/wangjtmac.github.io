VueUse - 浏览器（Browser） ​

浏览器（Browser）分类下的函数提供了一系列用于与浏览器 API 交互的工具，帮助开发者更便捷地访问和使用浏览器功能。

函数列表 ​

浏览器 API ​

  • useBluetooth - 使用蓝牙 API
  • useBreakpoints - 响应式断点检测
  • useBroadcastChannel - 使用 BroadcastChannel API
  • useClipboard - 访问剪贴板
  • useCookies - 操作 Cookie
  • useColorMode - 颜色模式（亮色/暗色）
  • useCssVar - 操作 CSS 变量
  • useDark - 暗色模式
  • useEventListener - 添加事件监听器
  • useEyeDropper - 使用吸管工具选择颜色
  • useFileDialog - 打开文件选择对话框
  • useFullscreen - 全屏模式
  • useGamepad - 使用游戏手柄 API
  • useImage - 加载图片
  • useMediaControls - 媒体控制
  • useMediaQuery - 媒体查询
  • useMemory - 获取内存使用情况
  • usePermission - 请求权限
  • usePreferredColorScheme - 获取用户偏好的颜色方案
  • usePreferredDark - 检测用户是否偏好深色模式
  • usePreferredLanguages - 获取用户偏好语言
  • useScreenOrientation - 屏幕方向
  • useScreenSafeArea - 安全区域
  • useShare - 使用 Web Share API
  • useTitle - 设置页面标题
  • useUrlSearchParams - 操作 URL 查询参数
  • useUserMedia - 访问用户媒体设备
  • useVibrate - 使用振动 API
  • useWakeLock - 防止设备休眠
  • useWebNotification - 使用 Web 通知 API
  • useWebWorker - 使用 Web Worker

窗口相关 ​

  • useActiveElement - 获取当前活动元素
  • useDocumentVisibility - 检测文档可见性
  • useDraggable - 使元素可拖动
  • useEventSource - 使用 EventSource API
  • useFavicon - 设置网站图标
  • useWindowFocus - 检测窗口是否获得焦点
  • useWindowScroll - 获取和设置窗口滚动位置
  • useWindowSize - 获取窗口尺寸

useBreakpoints ​

响应式断点检测。

vue
<template>
  <div>
    <h3>响应式断点示例</h3>

    <div class="breakpoints-info">
      <div class="current-breakpoint">
        当前断点: <span class="highlight">{{ currentBreakpoint }}</span>
      </div>

      <div class="breakpoints-status">
        <div class="breakpoint" :class="{ active: breakpoints.sm }">
          sm (小于 640px): {{ breakpoints.sm ? "是" : "否" }}
        </div>
        <div class="breakpoint" :class="{ active: breakpoints.md }">
          md (小于 768px): {{ breakpoints.md ? "是" : "否" }}
        </div>
        <div class="breakpoint" :class="{ active: breakpoints.lg }">
          lg (小于 1024px): {{ breakpoints.lg ? "是" : "否" }}
        </div>
        <div class="breakpoint" :class="{ active: breakpoints.xl }">
          xl (小于 1280px): {{ breakpoints.xl ? "是" : "否" }}
        </div>
        <div class="breakpoint" :class="{ active: breakpoints['2xl'] }">
          2xl (小于 1536px): {{ breakpoints["2xl"] ? "是" : "否" }}
        </div>
      </div>
    </div>

    <div class="responsive-demo">
      <div
        class="box"
        :class="{
          'box-sm': breakpoints.sm && !breakpoints.md,
          'box-md': breakpoints.md && !breakpoints.lg,
          'box-lg': breakpoints.lg && !breakpoints.xl,
          'box-xl': breakpoints.xl && !breakpoints['2xl'],
          'box-2xl': breakpoints['2xl'],
        }"
      >
        响应式盒子
      </div>
    </div>
  </div>
</template>

<script setup>
import { computed } from "vue";
import { useBreakpoints, breakpointsTailwind } from "@vueuse/core";

const breakpoints = useBreakpoints(breakpointsTailwind);

const currentBreakpoint = computed(() => {
  if (breakpoints["2xl"].value) return "2xl";
  if (breakpoints.xl.value) return "xl";
  if (breakpoints.lg.value) return "lg";
  if (breakpoints.md.value) return "md";
  if (breakpoints.sm.value) return "sm";
  return "xs";
});
</script>

useClipboard ​

访问剪贴板。

vue
<template>
  <div>
    <h3>剪贴板示例</h3>

    <div class="clipboard-section">
      <h4>复制到剪贴板</h4>
      <div class="input-group">
        <input v-model="text" placeholder="输入要复制的文本" />
        <button @click="copy(text)">复制</button>
      </div>
      <div v-if="copied" class="success-message">复制成功！</div>
    </div>

    <div class="clipboard-section">
      <h4>从剪贴板粘贴</h4>
      <div class="input-group">
        <input
          v-model="pastedText"
          placeholder="粘贴的文本将显示在这里"
          readonly
        />
        <button @click="paste">粘贴</button>
      </div>
    </div>

    <div class="clipboard-section">
      <h4>剪贴板历史</h4>
      <div class="history-list">
        <div
          v-for="(item, index) in clipboardHistory"
          :key="index"
          class="history-item"
        >
          <div class="history-text">{{ item }}</div>
          <button @click="copy(item)">重新复制</button>
        </div>
        <div v-if="clipboardHistory.length === 0" class="empty-history">
          暂无历史记录
        </div>
      </div>
    </div>
  </div>
</template>

<script setup>
import { ref } from "vue";
import { useClipboard } from "@vueuse/core";

const text = ref("");
const pastedText = ref("");
const clipboardHistory = ref([]);

const { copy, copied, paste } = useClipboard({ copiedDuring: 1500 });

// 监听粘贴事件
const handlePaste = async () => {
  try {
    const text = await paste();
    pastedText.value = text;
  } catch (error) {
    console.error("无法访问剪贴板:", error);
  }
};

// 重写复制方法，添加历史记录功能
const copyWithHistory = async (text) => {
  await copy(text);
  if (text && !clipboardHistory.value.includes(text)) {
    clipboardHistory.value.unshift(text);
    // 限制历史记录数量
    if (clipboardHistory.value.length > 5) {
      clipboardHistory.value.pop();
    }
  }
};
</script>

useTitle ​

设置页面标题。

vue
<template>
  <div>
    <h3>页面标题示例</h3>

    <div class="title-section">
      <div class="input-group">
        <input v-model="title" placeholder="输入页面标题" />
        <button @click="updateTitle">更新标题</button>
      </div>

      <div class="title-info">
        <p>
          当前页面标题: <strong>{{ currentTitle }}</strong>
        </p>
      </div>

      <div class="title-options">
        <label>
          <input type="checkbox" v-model="includeAppName" />
          包含应用名称
        </label>
      </div>
    </div>
  </div>
</template>

<script setup>
import { ref, computed, watch } from "vue";
import { useTitle } from "@vueuse/core";

const appName = "VueUse 演示";
const title = ref("");
const includeAppName = ref(true);

const titleToSet = computed(() => {
  if (!title.value) return appName;
  return includeAppName.value ? `${title.value} | ${appName}` : title.value;
});

const currentTitle = useTitle(titleToSet);

const updateTitle = () => {
  // useTitle 已经自动更新了标题，这里不需要额外操作
  // 可以添加一些额外的逻辑，比如保存标题历史等
};

// 监听标题变化
watch(currentTitle, (newTitle) => {
  console.log("页面标题已更新:", newTitle);
});
</script>

应用场景 ​

浏览器相关函数在以下场景特别有用：

  1. 响应式设计：使用断点检测实现响应式布局
  2. 主题切换：实现亮色/暗色模式切换
  3. 用户体验增强：使用剪贴板、通知等功能提升用户体验
  4. 媒体处理：访问用户摄像头、麦克风等设备
  5. 设备适配：根据屏幕方向、安全区域等调整界面
  6. 权限管理：请求和检查用户权限
  7. 本地存储：使用 Cookie、LocalStorage 等存储数据
  8. 页面交互：全屏模式、拖拽功能等

注意事项 ​

  1. 浏览器 API 的兼容性各不相同，使用前应检查目标浏览器是否支持
  2. 某些 API（如摄像头、麦克风访问）需要用户授权，应处理权限被拒绝的情况
  3. 在服务端渲染环境中，浏览器相关函数可能需要特殊处理
  4. 使用剪贴板、通知等功能时，应遵循用户体验最佳实践，避免过度打扰用户
  5. 响应式设计应考虑各种设备和屏幕尺寸，进行充分测试
