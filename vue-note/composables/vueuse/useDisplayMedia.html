useDisplayMedia ​

响应式地实现 mediaDevices.getDisplayMedia 屏幕共享。

useDisplayMedia 是 VueUse 库中的一个组合式函数，它提供了一种简单且响应式的方式来访问 Web API MediaDevices.getDisplayMedia()。 这个 API 允许你请求用户选择共享其屏幕的某个部分（整个屏幕、特定应用程序窗口或浏览器标签页），并获取一个包含视频和可选音频轨道的 MediaStream。这对于实现屏幕录制、在线演示、视频会议中的屏幕共享等功能非常有用。

基本用法 ​

通过调用 start() 函数可以开始屏幕共享，并通过 stream 引用获取共享的媒体流，然后将其绑定到 <video> 元素进行预览。

vue
<template>
  <div>
    <h2>useDisplayMedia 演示</h2>
    <div>
      <button @click="enabled = !enabled">
        {{ enabled ? '停止共享' : '开始共享我的屏幕' }}
      </button>
    </div>
    <div class="video-container">
      <video
        ref="videoRef"
        muted
        autoplay
        controls
        class="shared-screen-video"
      />
    </div>
    <p class="status-text">状态: {{ enabled ? '正在共享' : '未共享' }}</p>
    <p v-if="!isSupported" class="unsupported-text">您的浏览器不支持 `getDisplayMedia` API。</p>
  </div>
</template>

<script setup>
import { ref, watchEffect } from 'vue'
import { useDisplayMedia } from '@vueuse/core'

const videoRef = ref(null)
const { stream, enabled, isSupported } = useDisplayMedia()

watchEffect(() => {
  if (videoRef.value && stream.value) {
    videoRef.value.srcObject = stream.value
  }
})
</script>

当用户接受共享请求后，stream 引用将包含一个 MediaStream 对象，你可以将其赋值给 <video> 元素的 srcObject 属性来播放屏幕内容。通过设置 enabled.value = false 或调用 stop() 函数可以停止共享。

类型定义 ​

ts
import type { MaybeRef } from 'vue'
import type { ConfigurableNavigator } from '../_configurable'
import { shallowRef, watch } from 'vue'
import { defaultNavigator } from '../_configurable'
import { useEventListener } from '../useEventListener'
import { useSupported } from '../useSupported'

export interface UseDisplayMediaOptions extends ConfigurableNavigator {
  /**
   * 如果流已启用
   * @default false
   */
  enabled?: MaybeRef<boolean>

  /**
   * 视频媒体约束，如果为 `true` 则使用默认约束，可以传入 `MediaTrackConstraints` 对象自定义
   */
  video?: boolean | MediaTrackConstraints | undefined
  /**
   * 音频媒体约束，如果为 `true` 则使用默认约束，可以传入 `MediaTrackConstraints` 对象自定义
   */
  audio?: boolean | MediaTrackConstraints | undefined
}

export interface UseDisplayMediaReturn {
  isSupported: import("vue").ComputedRef<boolean>
  stream: import("vue").ShallowRef<MediaStream | undefined>
  start: () => Promise<MediaStream | undefined>
  stop: () => void
  enabled: import("vue").Ref<boolean>
}

declare function useDisplayMedia(
  options?: UseDisplayMediaOptions,
): UseDisplayMediaReturn

参数 ​

useDisplayMedia 函数接受一个可选的 options 对象，可以包含以下属性：

  • enabled?: MaybeRef<boolean>：可选，默认为 false。一个布尔值或其 ref，指示媒体流是否应该被启用。当此值变为 true 时，会自动尝试开始共享；当变为 false 时，会自动停止共享。
  • video?: boolean | MediaTrackConstraints | undefined：可选。用于视频轨道的媒体约束。可以是一个布尔值（true 表示请求视频轨道，使用默认约束），也可以是一个 MediaTrackConstraints 对象来指定更详细的视频设置（如分辨率、帧率等）。undefined 表示不请求视频。
  • audio?: boolean | MediaTrackConstraints | undefined：可选。用于音频轨道的媒体约束。可以是一个布尔值（true 表示请求音频轨道，使用默认约束），也可以是一个 MediaTrackConstraints 对象来指定更详细的音频设置。undefined 表示不请求音频。
  • navigator?: Navigator (来自 ConfigurableNavigator)：可选。允许你指定一个自定义的 Navigator 对象，通常用于测试环境或非浏览器环境。

返回值 ​

useDisplayMedia 函数返回一个包含以下属性的对象：

  • isSupported: ComputedRef<boolean>：一个计算属性，指示当前浏览器环境是否支持 mediaDevices.getDisplayMedia API。如果支持则为 true，否则为 false。
  • stream: ShallowRef<MediaStream | undefined>：一个浅层响应式引用，包含当前活动的 MediaStream 对象。在开始共享之前或停止共享之后为 undefined。当用户选择共享并授权后，它会包含屏幕共享的媒体流。
  • start: () => Promise<MediaStream | undefined>：一个异步函数，调用它可以请求开始屏幕共享。它会返回一个 Promise，解析为 MediaStream 对象（如果用户授权并成功获取流）或 undefined。
  • stop: () => void：一个函数，调用它可以停止当前的屏幕共享。
  • enabled: Ref<boolean>：一个响应式引用，其值与 options 中的 enabled 属性同步。你可以通过直接修改 enabled.value 来控制屏幕共享的开启和关闭。

工作原理 ​

useDisplayMedia 的核心工作原理是封装了 Navigator.mediaDevices.getDisplayMedia() API，并结合 Vue 的响应式系统进行状态管理：

  1. 能力检测：内部使用 useSupported 来判断 navigator.mediaDevices.getDisplayMedia 是否存在，从而确定浏览器是否支持此功能。
  2. 约束配置：将 video 和 audio 选项转换为 MediaStreamConstraints 对象，用于传递给 getDisplayMedia。
  3. 媒体流获取：当 start() 函数被调用或 enabled 变为 true 时，如果浏览器支持此 API，它会调用 navigator.mediaDevices.getDisplayMedia(constraint)。这个方法会触发浏览器原生界面，提示用户选择要共享的屏幕内容并请求权限。
  4. 媒体流管理：成功获取到 MediaStream 后，将其赋值给 stream 响应式引用。同时，为 stream 中的每个 MediaStreamTrack（视频轨道和音频轨道）添加 ended 事件监听器。当某个轨道结束（例如用户停止共享或浏览器端自动停止）时，会调用内部的 stop() 函数来清理资源。
  5. 停止共享：stop() 函数会遍历 MediaStream 中的所有轨道，并调用它们的 stop() 方法来释放资源，同时将 stream 置为 undefined，并更新 enabled 状态。
  6. 响应式同步：通过 watch 监听 enabled 状态的变化，自动触发 _start() 或 _stop() 函数，从而实现了声明式的屏幕共享控制。shallowRef 被用于 stream，因为 MediaStream 对象本身的变化不频繁，只需浅层响应即可。

使用场景 ​

  • 在线会议和演示：实现屏幕共享功能，允许用户在视频会议中展示其屏幕内容。
  • 远程协助和支持：远程技术支持人员可以通过屏幕共享查看用户的问题并进行指导。
  • 在线教育和教学：教师可以共享屏幕进行教学演示或代码演示。
  • 屏幕录制应用：结合其他 Web API（如 MediaRecorder），可以实现基于浏览器的屏幕录制工具。
  • 游戏直播：在一些简单的基于 Web 的游戏直播平台中，实现游戏画面的共享。

注意事项 ​

  • HTTPS 要求：mediaDevices.getDisplayMedia() API 必须在安全的上下文（HTTPS）中运行。在 http://localhost 上进行开发通常不受此限制，但在部署到生产环境时务必使用 HTTPS。
  • 用户权限：每次调用 getDisplayMedia() 都会触发浏览器权限请求，用户必须手动选择要共享的屏幕、窗口或标签页，并显式授权。无法在代码中绕过此用户交互。
  • 用户停止共享：用户可以通过浏览器提供的 UI 控件随时停止共享（例如，浏览器地址栏旁边的共享指示器）。当用户停止共享时，stream 中的轨道会触发 ended 事件，useDisplayMedia 会自动更新其状态并清理资源。
  • 错误处理：在调用 start() 时，如果用户拒绝权限或发生其他错误（例如设备不可用），Promise 会被拒绝。你需要在调用 start() 时处理这些错误。
  • 浏览器兼容性：虽然 getDisplayMedia() 在现代浏览器中得到了广泛支持，但仍需注意目标用户的浏览器版本。可以利用 isSupported 引用来提供优雅的降级方案。
  • 性能影响：屏幕共享可能会消耗较多的 CPU 和网络资源，特别是在高分辨率或高帧率下。在设计应用时应考虑到这一点，并根据需要提供质量调节选项。

<script setup>
import { ref, watchEffect } from 'vue'
import { useDisplayMedia } from '@vueuse/core'

const videoRef = ref(null)
const { stream, enabled, isSupported, start, stop } = useDisplayMedia({
  video: true, // 请求视频轨道
  audio: false, // 不请求音频轨道，根据需要可设为 true
})

// watchEffect 响应式地将媒体流绑定到 video 元素
watchEffect(() => {
  if (videoRef.value && stream.value) {
    videoRef.value.srcObject = stream.value
  } else if (videoRef.value) {
    // 当 stream.value 变为 null/undefined 时，清理 video 元素的 srcObject
    videoRef.value.srcObject = null
  }
})

// 切换屏幕共享状态的函数
async function toggleScreenSharing() {
  if (enabled.value) {
    // 如果当前正在共享，则停止共享
    stop()
  } else {
    // 如果当前未共享，则尝试开始共享
    try {
      await start()
    } catch (error) {
      // 捕获并处理屏幕共享启动过程中的错误
      console.error('屏幕共享失败:', error)
      alert('无法开始屏幕共享。请确保您已授予权限或浏览器支持此功能。')
    }
  }
}
</script>
