useFps ​

响应式地跟踪并显示每秒帧数 (FPS)。

useFps 是 VueUse 库中的一个组合式函数，它提供了一种简单的方式来实时计算和显示页面的每秒帧数 (Frames Per Second)。这对于监控 Web 应用程序的性能、调试动画卡顿或评估渲染效率非常有用。

它利用 requestAnimationFrame API 来优化性能计数，确保 FPS 的计算与浏览器的渲染周期同步。

基本用法 ​

你可以直接调用 useFps() 函数来获取一个响应式的 fps 引用，该引用会实时更新当前的帧率。

vue
<template>
  <div>
    <h2>useFps 演示</h2>
    <p>当前 FPS: <strong>{{ fps }}</strong></p>
  </div>
</template>

<script setup>
import { useFps } from '@vueuse/core'

const fps = useFps()
</script>

<style scoped>
p {
  font-size: 1.5em;
  font-weight: bold;
  color: #34495e;
}

p strong {
  color: #42b883;
}
</style>

类型定义 ​

ts
import type { ShallowRef } from 'vue'

export interface UseFpsOptions {
  /**
   * 每隔 x 帧计算一次 FPS。
   * @default 10
   */
  every?: number
}

export declare function useFps(
  options?: UseFpsOptions
): ShallowRef<number>

参数 ​

useFps 函数接受一个可选的 options 对象，可以包含以下属性：

  1. options?: UseFpsOptions：
    • 可选。一个配置对象，可以包含以下属性：
      • every?: number：可选，默认为 10。一个数字。指定每隔多少帧计算一次 FPS。例如，如果设置为 10，则每隔 10 个动画帧计算一次 FPS。这有助于平滑 FPS 的显示，避免过于频繁的更新和波动。

返回值 ​

useFps 函数返回一个 ShallowRef<number>，表示当前的每秒帧数。

  • fps: ShallowRef<number>：一个浅层响应式引用。其值表示当前的 FPS。如果浏览器不支持 performance API，或者在非浏览器环境中运行，fps 的值将为 0。

工作原理 ​

useFps 的核心工作原理是利用了 requestAnimationFrame 和 performance.now() 来精确计算帧率：

  1. requestAnimationFrame 循环：它使用 useRafFn 创建一个 requestAnimationFrame 循环。这意味着每当浏览器准备好进行下一次重绘时，它都会执行一次回调函数。
  2. 帧计数与时间戳：
    • 在每次 requestAnimationFrame 回调中，ticks 计数器会增加。ticks 用于记录自上次 FPS 计算以来渲染了多少帧。
    • last 变量存储上一次计算 FPS 的时间戳，通过 performance.now() 获取高精度的时间。
  3. FPS 计算：
    • 当 ticks 达到 every 选项指定的值时（例如，每 10 帧），就会触发一次 FPS 计算。
    • now 获取当前时间戳。
    • diff = now - last 计算出在 every 帧内所经过的总毫秒数。
    • fps.value = Math.round(1000 / (diff / ticks)) 是 FPS 计算的核心公式：
      • diff / ticks 得到平均每帧所花费的毫秒数。
      • 1000 / (平均每帧毫秒数) 得到每秒的帧数。
      • Math.round() 对结果进行四舍五入，使其更易读。
  4. 重置：计算完成后，last 会更新为当前时间 now，ticks 重置为 0，为下一次计算做准备。
  5. 兼容性处理：在函数开始时，会检查 typeof performance === 'undefined'。如果 performance API 不可用（例如在一些旧版浏览器或非浏览器环境中），fps 会直接返回 shallowRef(0)，表示不支持 FPS 追踪。

使用场景 ​

  • 性能监控：在开发或测试阶段，实时监控应用的帧率，以便发现和解决性能瓶颈，特别是涉及复杂动画、大量 DOM 操作或重型计算的场景。
  • 游戏开发：在基于 Web 的游戏中，显示当前游戏的帧率，帮助开发者优化游戏流畅度。
  • 动画调试：在调试动画效果时，通过观察 FPS 变化来判断动画是否流畅，是否有掉帧现象。
  • 用户体验衡量：虽然不能直接代表用户体验，但高且稳定的 FPS 通常意味着更流畅的用户界面和更好的用户体验。
  • 响应式布局测试：在不同设备和屏幕尺寸下测试页面的渲染性能。

注意事项 ​

  • requestAnimationFrame：useFps 依赖于 requestAnimationFrame，因此它只在浏览器环境中工作，并且在页面处于后台时（例如，用户切换到其他标签页），requestAnimationFrame 会被浏览器暂停，导致 FPS 显示为 0 或不更新。
  • performance.now()：performance.now() 提供了一个高精度的当前时间戳，比 Date.now() 更适合用于性能测量。
  • every 选项：调整 every 值可以影响 FPS 更新的平滑度。较小的值会使 FPS 波动更大，但响应更及时；较大的值会使 FPS 显示更稳定，但响应会略有延迟。默认值 10 通常是一个很好的平衡点。
  • 系统负载：显示的 FPS 值会受到浏览器、操作系统、硬件以及当前运行的其他应用程序的影响。它反映的是客户端的实际渲染性能。

实际应用示例 ​

以下是一个简单的 Vue 组件，演示了如何在页面上显示实时的 FPS，并允许用户调整 every 选项来观察其对 FPS 显示稳定性的影响。

vue
<template>
  <div class="demo-container">
    <h2>useFps 演示：实时帧率监控</h2>

    <p class="note">
      实时显示页面当前的帧率 (FPS)。尝试进行一些页面操作，例如滚动或鼠标移动，观察 FPS 的变化。
    </p>

    <div class="fps-display">
      当前 FPS: <span class="fps-value">{{ fps }}</span>
    </div>

    <div class="controls">
      <label for="every-frames">
        每隔
        <input
          id="every-frames"
          type="number"
          v-model.number="everyFrames"
          min="1"
          max="60"
        />
        帧更新一次 FPS
      </label>
    </div>

    <div class="code-section">
      <h3>代码示例</h3>
      <pre><code>
import { ref } from 'vue'
import { useFps } from '@vueuse/core'

const everyFrames = ref(10) // 默认每10帧更新一次
const fps = useFps({ every: everyFrames })
      </code></pre>
    </div>
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue'
import { useFps } from '@vueuse/core'

const everyFrames = ref(10) // 默认每10帧更新一次

// 将 everyFrames ref 传递给 useFps 的 options
const fps = useFps({ every: everyFrames })
</script>
