useMouseInElement ​

响应式地跟踪鼠标在指定元素内的位置。

useMouseInElement 是 VueUse 库中的一个组合式函数，用于跟踪鼠标相对于特定 HTML 元素的位置。它不仅返回鼠标在页面上的绝对坐标，还计算出鼠标在目标元素内部的相对坐标，并指示鼠标是否在元素外部。这在需要实现基于元素内鼠标位置的交互（如自定义工具提示、拖拽功能、元素内部热点区域检测等）时非常有用。

基本用法 ​

ts
import { ref } from 'vue'
import { useMouseInElement } from '@vueuse/core'

const target = ref(null)
const { x, y, elementX, elementY, elementPositionX, elementPositionY, elementHeight, elementWidth, isOutside } = useMouseInElement(target)

// x, y: 鼠标在页面上的绝对坐标
// elementX, elementY: 鼠标在目标元素内部的相对坐标（左上角为0,0）
// elementPositionX, elementPositionY: 目标元素在页面上的绝对位置（左上角）
// elementHeight, elementWidth: 目标元素的宽度和高度
// isOutside: 布尔值，指示鼠标是否在元素外部

通过将一个 ref 绑定到你的目标元素上，useMouseInElement 会自动监听该元素及其内部的鼠标事件，并提供实时的鼠标位置信息。

类型定义 ​

ts
import type { MaybeElementRef } from '../unrefElement'
import type { UseMouseOptions } from '../useMouse'
import type { Ref } from 'vue'

export interface MouseInElementOptions extends UseMouseOptions {
  /**
   * 当光标在目标元素外部时是否处理鼠标事件。
   * 启用后，即使在元素边界之外，鼠标位置也会继续被跟踪。
   *
   * @default true
   */
  handleOutside?: boolean

  /**
   * 是否监听窗口 resize 事件以更新元素位置和大小。
   *
   * @default true
   */
  windowResize?: boolean

  /**
   * 是否监听窗口 scroll 事件以更新元素位置和大小。
   *
   * @default true
   */
  windowScroll?: boolean
}

export interface UseMouseInElementReturn {
  x: Ref<number>
  y: Ref<number>
  sourceType: Ref<'mouse' | 'touch' | null>
  elementX: Ref<number>
  elementY: Ref<number>
  elementPositionX: Ref<number>
  elementPositionY: Ref<number>
  elementHeight: Ref<number>
  elementWidth: Ref<number>
  isOutside: Ref<boolean>
  stop: () => void
}

declare function useMouseInElement(
  target?: MaybeElementRef,
  options?: MouseInElementOptions,
): UseMouseInElementReturn

参数 ​

useMouseInElement 函数接受以下两个参数：

  1. target?: MaybeElementRef：

    • 类型：MaybeElementRef (可以是 HTMLElement、SVGElement、Vue ref 对象或 null/undefined)
    • 描述：要跟踪鼠标在其内部位置的目标元素。如果未提供，默认是 document.body。

  2. options?: MouseInElementOptions：可选的配置对象，可以包含以下属性：

    • handleOutside?: boolean：可选，默认为 true。如果设置为 true，即使鼠标在目标元素外部，elementX 和 elementY 也会继续根据鼠标的页面坐标相对于元素计算。如果设置为 false，当鼠标在元素外部时，elementX 和 elementY 将不再更新。
    • windowResize?: boolean：可选，默认为 true。如果设置为 true，函数会监听窗口的 resize 事件，当窗口大小改变时，会重新计算目标元素的位置和尺寸。这有助于确保 elementPositionX、elementPositionY、elementWidth、elementHeight 的准确性。
    • windowScroll?: boolean：可选，默认为 true。如果设置为 true，函数会监听窗口的 scroll 事件，当页面滚动时，会重新计算目标元素的位置。这对于在滚动页面中保持鼠标在元素内位置的准确性很重要。
    • type?: UseMouseCoordType | UseMouseEventExtractor (来自 UseMouseOptions)：可选。指定鼠标坐标的类型（'page' 或 'client'）或自定义提取器。默认为 'page'。
    • initialValue?: { x: number; y: number } (来自 UseMouseOptions)：可选。鼠标坐标的初始值。
    • restoreOnUnmounted?: boolean (来自 UseMouseOptions)：可选，默认为 false。是否在组件卸载时将鼠标坐标重置为 initialValue。

返回值 ​

useMouseInElement 函数返回一个包含多个响应式引用的对象，用于提供详细的鼠标位置信息：

  • x: Ref<number>：鼠标在页面（或客户端，取决于 type 选项）上的水平坐标。
  • y: Ref<number>：鼠标在页面（或客户端，取决于 type 选项）上的垂直坐标。
  • sourceType: Ref<'mouse' | 'touch' | null>：事件的来源类型（'mouse' 或 'touch'）。
  • elementX: Ref<number>：鼠标在目标元素内部的水平坐标，相对于元素左上角 (0,0)。
  • elementY: Ref<number>：鼠标在目标元素内部的垂直坐标，相对于元素左上角 (0,0)。
  • elementPositionX: Ref<number>：目标元素在页面上的绝对水平位置（左上角）。
  • elementPositionY: Ref<number>：目标元素在页面上的绝对垂直位置（左上角）。
  • elementHeight: Ref<number>：目标元素的高度。
  • elementWidth: Ref<number>：目标元素的宽度。
  • isOutside: Ref<boolean>：一个布尔值，true 表示鼠标在目标元素外部，false 表示鼠标在目标元素内部。
  • stop: () => void：一个函数，调用它会停止所有内部事件监听器，从而停止跟踪鼠标位置。

工作原理 ​

useMouseInElement 的核心工作原理结合了 useMouse 和 getBoundingClientRect() 方法：

  1. useMouse 集成：它内部使用了 useMouse 来获取鼠标在整个页面上的全局坐标 (x, y)。
  2. 元素尺寸和位置获取：通过 target 元素的 getBoundingClientRect() 方法，获取到目标元素在视口中的位置 (left, top) 和尺寸 (width, height)。这些信息会根据 window.pageXOffset 和 window.pageYOffset（如果 type 是 'page'）进行调整，以获取相对于文档的绝对位置。
  3. 计算相对坐标：elementX 和 elementY 是通过将全局鼠标坐标 (x, y) 减去元素的绝对位置 (elementPositionX, elementPositionY) 来计算的。这样就得到了鼠标在元素内部的相对坐标。
  4. 判断 isOutside：通过比较 elementX 和 elementY 是否在元素 width 和 height 的范围内来判断鼠标是否在元素外部。
  5. 响应式更新：利用 useResizeObserver 监听目标元素本身的尺寸变化，useMutationObserver 监听元素样式或类名变化（这可能影响布局），以及 useEventListener 监听 scroll 和 resize 事件，确保当元素位置或尺寸发生变化时，update 函数被调用以重新计算所有相关坐标和状态。
  6. handleOutside 选项：根据此选项，决定当鼠标移出元素时是否继续更新 elementX 和 elementY。

使用场景 ​

  • 自定义工具提示（Tooltip）：根据鼠标在元素内的精确位置显示工具提示。
  • 图片放大镜效果：当鼠标在图片上移动时，在局部区域显示放大后的图像。
  • 拖拽功能：判断鼠标是否在可拖拽元素内部开始拖拽，并计算拖拽偏移量。
  • 游戏开发：实现基于鼠标在特定游戏区域内位置的交互。
  • 数据可视化：在图表上显示鼠标悬停点的数据信息。
  • UI 组件的精确交互：例如，在进度条上显示鼠标当前位置对应的数值。

注意事项 ​

  • 性能考量：虽然 useMouseInElement 内部进行了优化，但频繁的鼠标移动事件仍可能对性能产生轻微影响。在不需要高精度或持续跟踪的场景下，可以考虑在适当的时机停止监听 (stop() 方法)。
  • 目标元素：确保你传递给 useMouseInElement 的 target 是一个实际存在的 DOM 元素或 Vue ref。如果目标元素在渲染时不存在，函数将无法正确工作。
  • CSS 变换：如果目标元素应用了 CSS transform (如 scale, rotate 等)，getBoundingClientRect() 可能会返回变换后的尺寸和位置，这可能导致计算出的 elementX 和 elementY 与视觉上的期望有所偏差。在这种情况下，可能需要额外的计算来调整。
  • 滚动容器：如果目标元素在一个可滚动的容器内，并且你希望鼠标位置始终相对于该容器计算，确保 windowScroll 选项生效，或者考虑监听容器的滚动事件。
