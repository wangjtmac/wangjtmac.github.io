UniApp 组件系统 ​

本文介绍 UniApp 组件系统的基础知识。如需了解更详细的组件用法、扩展组件(uni-ui)以及实战案例，请参阅 组件详解。

组件基础 ​

UniApp 组件系统基于 Vue 组件系统，同时兼容小程序组件规范，提供了丰富的内置组件和扩展能力。

页面结构 ​

UniApp 页面采用 Vue 单文件组件的结构：

vue
<template>
  <view class="container">
    <!-- 页面内容 -->
  </view>
</template>

<script>
export default {
  data() {
    return {
      // 数据
    }
  },
  // 生命周期
  onLoad() {
    // 页面加载时执行
  },
  onShow() {
    // 页面显示时执行
  },
  methods: {
    // 方法
  }
}
</script>

<style>
/* 样式 */
</style>

内置组件分类 ​

UniApp 提供了丰富的内置组件，可以分为以下几类：

视图容器组件 ​

view ​

最基础的视图容器，类似于 HTML 中的 div。

vue
<view class="container">
  <view class="item">项目1</view>
  <view class="item">项目2</view>
</view>

scroll-view ​

可滚动的视图区域，用于展示超出容器的内容。

vue
<scroll-view scroll-y="true" class="scroll-container" @scroll="onScroll">
  <view class="scroll-item" v-for="(item, index) in items" :key="index">
    {{ item.name }}
  </view>
</scroll-view>

常用属性：

  • scroll-x: 允许横向滚动
  • scroll-y: 允许纵向滚动
  • scroll-top: 设置竖向滚动条位置
  • scroll-left: 设置横向滚动条位置
  • scroll-into-view: 滚动到指定元素

swiper ​

滑块视图容器，用于实现轮播图等效果。

vue
<swiper class="banner" indicator-dots autoplay circular :interval="3000" :duration="500">
  <swiper-item v-for="(item, index) in banners" :key="index">
    <image class="banner-image" :src="item.image" mode="aspectFill"></image>
  </swiper-item>
</swiper>

常用属性：

  • indicator-dots: 是否显示指示点
  • autoplay: 是否自动切换
  • circular: 是否循环播放
  • interval: 自动切换时间间隔
  • duration: 滑动动画时长

基础内容组件 ​

text ​

文本组件，用于显示文本内容。

vue
<text class="title" selectable>可选择的文本内容</text>

常用属性：

  • selectable: 文本是否可选
  • space: 显示连续空格
  • decode: 是否解码

rich-text ​

富文本组件，用于显示 HTML 内容。

vue
<rich-text :nodes="htmlContent"></rich-text>

progress ​

进度条组件。

vue
<progress :percent="80" show-info stroke-width="3" />

表单组件 ​

button ​

按钮组件。

vue
<button type="primary" @tap="handleTap">主按钮</button>
<button type="default" size="mini">次按钮</button>

常用属性：

  • type: 按钮类型（primary, default, warn）
  • size: 按钮大小（default, mini）
  • plain: 是否镂空
  • loading: 是否显示加载图标

input ​

输入框组件。

vue
<input type="text" v-model="inputValue" placeholder="请输入内容" @input="onInput" />

常用属性：

  • type: 输入类型（text, number, idcard, digit）
  • password: 是否是密码类型
  • placeholder: 输入提示
  • focus: 是否自动聚焦

checkbox & radio ​

多选框和单选框。

vue
<checkbox-group @change="checkboxChange">
  <label v-for="(item, index) in items" :key="index">
    <checkbox :value="item.value" :checked="item.checked" />
    {{item.name}}
  </label>
</checkbox-group>

<radio-group @change="radioChange">
  <label v-for="(item, index) in items" :key="index">
    <radio :value="item.value" :checked="item.checked" />
    {{item.name}}
  </label>
</radio-group>

picker ​

选择器组件，支持普通选择器、多列选择器、时间选择器、日期选择器等。

vue
<picker @change="bindPickerChange" :value="index" :range="array">
  <view class="picker">
    当前选择: {{array[index]}}
  </view>
</picker>

媒体组件 ​

image ​

图片组件。

vue
<image src="/static/logo.png" mode="aspectFit"></image>

常用属性：

  • src: 图片路径
  • mode: 图片裁剪、缩放模式
    • aspectFit: 保持纵横比缩放图片，使图片完整显示
    • aspectFill: 保持纵横比缩放图片，只保证图片的短边能完全显示出来
    • widthFix: 宽度不变，高度自动变化，保持原图宽高比不变

video ​

视频组件。

vue
<video src="https://example.com/video.mp4" controls></video>

导航组件 ​

navigator ​

页面链接组件。

vue
<navigator url="/pages/detail/detail?id=1" hover-class="navigator-hover">
  跳转到详情页
</navigator>

常用属性：

  • url: 跳转页面路径
  • open-type: 跳转方式
    • navigate: 对应 uni.navigateTo
    • redirect: 对应 uni.redirectTo
    • switchTab: 对应 uni.switchTab
    • reLaunch: 对应 uni.reLaunch
    • navigateBack: 对应 uni.navigateBack

样式与布局 ​

尺寸单位 ​

UniApp 支持 rpx 响应式尺寸单位，可以根据屏幕宽度进行自适应。

  • 规定屏幕宽为750rpx
  • 1rpx = 屏幕宽度 / 750
css
.container {
  width: 750rpx;
  height: 200rpx;
  padding: 20rpx;
}

Flex 布局 ​

UniApp 完全支持 CSS3 Flex 布局，推荐使用 Flex 布局实现复杂的页面结构。

css
.flex-container {
  display: flex;
  flex-direction: row;
  justify-content: space-between;
  align-items: center;
}

.flex-item {
  flex: 1;
}

条件编译样式 ​

可以使用条件编译为不同平台编写特定样式：

css
/* #ifdef APP-PLUS */
.app-only {
  color: red;
}
/* #endif */

/* #ifdef MP-WEIXIN */
.wx-only {
  color: green;
}
/* #endif */

组件通信 ​

父子组件通信 ​

  1. Props 向下传递：
vue
<!-- 父组件 -->
<template>
  <child-component :message="parentMessage" @update="handleUpdate"></child-component>
</template>

<!-- 子组件 -->
<script>
export default {
  props: {
    message: {
      type: String,
      required: true
    }
  },
  methods: {
    sendToParent() {
      this.$emit('update', 'data from child')
    }
  }
}
</script>
  1. 事件向上传递：子组件通过 $emit 触发事件，父组件监听该事件。

跨组件通信 ​

  1. 事件总线：
js
// 创建事件总线
Vue.prototype.$bus = new Vue()

// 组件A发送事件
this.$bus.$emit('custom-event', { data: 'value' })

// 组件B接收事件
this.$bus.$on('custom-event', (data) => {
  console.log(data)
})
  1. Vuex 状态管理：适用于复杂应用的状态管理。

自定义组件 ​

创建组件 ​

在 UniApp 中创建自定义组件与 Vue 基本一致：

  1. 创建组件文件 components/my-component.vue：
vue
<template>
  <view class="my-component">
    <text>{{ title }}</text>
    <slot></slot>
  </view>
</template>

<script>
export default {
  name: 'MyComponent',
  props: {
    title: {
      type: String,
      default: '默认标题'
    }
  },
  data() {
    return {
      // 组件内部数据
    }
  },
  methods: {
    // 组件方法
  }
}
</script>

<style>
.my-component {
  padding: 20rpx;
  background-color: #f5f5f5;
}
</style>
  1. 在页面中使用组件：
vue
<template>
  <view>
    <my-component title="自定义标题">
      <text>这是插槽内容</text>
    </my-component>
  </view>
</template>

<script>
import MyComponent from '@/components/my-component.vue'

export default {
  components: {
    MyComponent
  }
}
</script>

easycom 组件规范 ​

UniApp 提供了 easycom 组件规范，可以免去组件的引用和注册步骤：

  1. 将组件放在 components 目录下
  2. 组件命名符合 组件名/组件名.vue 的规范

然后可以直接在页面中使用：

vue
<template>
  <view>
    <my-component title="自定义标题"></my-component>
  </view>
</template>

<script>
export default {
  // 无需引入和注册
}
</script>

可以在 pages.json 中配置 easycom：

json
"easycom": {
  "autoscan": true,
  "custom": {
    "^u-(.*)": "@/uview-ui/components/u-$1/u-$1.vue",
    "^uni-(.*)": "@dcloudio/uni-ui/lib/uni-$1/uni-$1.vue"
  }
}

实践心得 ​

  1. 组件复用：

    • 将常用的 UI 结构封装为组件，提高代码复用率
    • 合理使用插槽，增强组件的灵活性
    • 组件粒度要适中，既不过大也不过小

  2. 性能优化：

    • 避免组件过度嵌套，减少渲染层级
    • 合理使用 v-if 和 v-show，减少不必要的渲染
    • 大列表使用 scroll-view 并实现虚拟列表

  3. 样式隔离：

    • 组件样式默认不隔离，注意避免样式冲突
    • 可以使用 scoped 或命名空间避免样式污染

  4. 平台差异：

    • 使用条件编译处理不同平台的组件差异
    • 了解各平台的组件限制，避免使用不支持的特性

实战案例 ​

在我们的电商应用中，我们实现了以下自定义组件：

  1. 商品卡片组件：
vue
<!-- components/product-card/product-card.vue -->
<template>
  <view class="product-card" @tap="viewProduct">
    <image class="product-image" :src="product.image" mode="aspectFill"></image>
    <view class="product-info">
      <text class="product-name">{{ product.name }}</text>
      <text class="product-price">¥{{ product.price.toFixed(2) }}</text>
    </view>
  </view>
</template>

<script>
export default {
  name: 'ProductCard',
  props: {
    product: {
      type: Object,
      required: true
    }
  },
  methods: {
    viewProduct() {
      uni.navigateTo({
        url: `/pages/product/detail?id=${this.product.id}`
      })
    }
  }
}
</script>

<style>
.product-card {
  width: 345rpx;
  margin: 10rpx;
  background-color: #fff;
  border-radius: 10rpx;
  overflow: hidden;
  box-shadow: 0 2rpx 10rpx rgba(0, 0, 0, 0.1);
}

.product-image {
  width: 100%;
  height: 345rpx;
}

.product-info {
  padding: 16rpx;
}

.product-name {
  font-size: 28rpx;
  color: #333;
  line-height: 40rpx;
  height: 80rpx;
  overflow: hidden;
  text-overflow: ellipsis;
  display: -webkit-box;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
}

.product-price {
  font-size: 32rpx;
  color: #ff6700;
  margin-top: 10rpx;
  font-weight: bold;
}
</style>
  1. 使用自定义组件：
vue
<!-- pages/index/index.vue -->
<template>
  <view class="container">
    <view class="product-list">
      <product-card 
        v-for="(item, index) in products" 
        :key="index" 
        :product="item"
      ></product-card>
    </view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      products: [
        { id: 1, name: '商品1', price: 99.99, image: '/static/product1.jpg' },
        { id: 2, name: '商品2', price: 199.99, image: '/static/product2.jpg' }
      ]
    }
  }
}
</script>

<style>
.product-list {
  display: flex;
  flex-wrap: wrap;
  padding: 10rpx;
}
</style>

总结 ​

UniApp 的组件系统继承了 Vue 的组件化思想，同时兼容小程序的组件规范，提供了丰富的内置组件和灵活的扩展能力。通过合理使用组件，可以提高代码的复用性和可维护性，提升开发效率。

在实际开发中，应当充分利用 UniApp 提供的组件能力，合理设计组件结构，处理好组件间的通信，并注意不同平台的兼容性问题。同时，熟练掌握样式和布局技巧，可以实现更加美观和响应式的界面。
