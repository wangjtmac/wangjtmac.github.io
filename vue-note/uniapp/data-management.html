UniApp 数据管理 ​

数据管理概述 ​

在 UniApp 应用开发中，随着应用规模的扩大，组件间的状态共享和数据传递变得越来越复杂。为了解决这个问题，我们需要引入状态管理工具。UniApp 完全兼容 Vue 的状态管理方案，主要包括 Vuex 和 Pinia 两种。

  • Vuex：Vue 2 官方推荐的状态管理模式，集中式存储和管理应用的所有组件状态。
  • Pinia：Vue 3 官方推荐的状态管理库，提供更简单、更灵活的 API，更好的 TypeScript 支持。

Vuex 状态管理 ​

Vuex 基本概念 ​

Vuex 的核心概念包括：

  1. State：状态，即数据源
  2. Getter：从 state 派生的状态
  3. Mutation：修改状态的同步方法
  4. Action：可包含异步操作的方法，提交 mutation
  5. Module：将 store 分割成模块，每个模块拥有自己的 state、getter、mutation 和 action

在 UniApp 中集成 Vuex ​

  1. 首先安装 Vuex：
bash
# Vue 2 项目使用 Vuex 3.x
npm install vuex@3 --save

# Vue 3 项目使用 Vuex 4.x
npm install vuex@4 --save
  1. 创建 store 目录结构：
├── store
│   ├── index.js          # 组装模块并导出 store
│   ├── state.js          # 根级别的 state
│   ├── getters.js        # 根级别的 getter
│   ├── mutations.js      # 根级别的 mutation
│   ├── actions.js        # 根级别的 action
│   └── modules
│       ├── user.js       # 用户模块
│       └── cart.js       # 购物车模块
  1. 在 store/index.js 中创建 store：
js
import Vue from 'vue'
import Vuex from 'vuex'
import state from './state'
import getters from './getters'
import mutations from './mutations'
import actions from './actions'
import user from './modules/user'
import cart from './modules/cart'

Vue.use(Vuex)

const store = new Vuex.Store({
  state,
  getters,
  mutations,
  actions,
  modules: {
    user,
    cart
  }
})

export default store
  1. 在 main.js 中挂载 store：
js
import Vue from 'vue'
import App from './App'
import store from './store'

Vue.config.productionTip = false

App.mpType = 'app'

const app = new Vue({
  store,
  ...App
})
app.$mount()

Vuex 使用示例 ​

  1. 定义 state：
js

  isLoading: false,

  1. 定义 getter：

// store/mutations.js

    state.networkType = type

  1. 定义 action：
js
// store/actions.js
export default {
  checkNetworkStatus({ commit }) {
    return new Promise((resolve) => {
      uni.getNetworkType({
        success: (res) => {
          commit('SET_NETWORK_TYPE', res.networkType)
          resolve(res.networkType)
        }
      })
    })
  },
  showLoading({ commit }) {
    commit('SET_LOADING', true)
  },
  hideLoading({ commit }) {
    commit('SET_LOADING', false)
  }
}
  1. 定义模块：
js
// store/modules/user.js
export default {
  namespaced: true,
  state: {
    userInfo: null,
    token: uni.getStorageSync('token') || ''
  },
  getters: {
    isLoggedIn: state => !!state.token
  },
  mutations: {
    SET_TOKEN(state, token) {
      state.token = token
      uni.setStorageSync('token', token)
    },
    SET_USER_INFO(state, userInfo) {
      state.userInfo = userInfo
    },
    CLEAR_USER(state) {
      state.token = ''
      state.userInfo = null
      uni.removeStorageSync('token')
    }
  },
  actions: {
    // 登录
    login({ commit }, userInfo) {
      return new Promise((resolve, reject) => {
        // 模拟登录请求
        setTimeout(() => {
          const token = 'mock-token-' + Date.now()
          commit('SET_TOKEN', token)
          commit('SET_USER_INFO', userInfo)
          resolve(token)
        }, 1000)
      })
    },
    // 退出登录
    logout({ commit }) {
      commit('CLEAR_USER')
    },
    // 获取用户信息
    getUserInfo({ commit, state }) {
      return new Promise((resolve, reject) => {
        if (state.userInfo) {
          resolve(state.userInfo)
        } else {
          // 模拟获取用户信息
          setTimeout(() => {
            const userInfo = {
              id: 1,
              name: '测试用户',
              avatar: '/static/avatar.png'
            }
            commit('SET_USER_INFO', userInfo)
            resolve(userInfo)
          }, 1000)
        }
      })
    }
  }
}
  1. 在组件中使用：
vue
<template>
  <view class="container">
    <view v-if="isLoading" class="loading">加载中...</view>
    <view v-else>
      <view v-if="isLoggedIn">
        <text>欢迎，{{ userInfo.name }}</text>
        <button @tap="handleLogout">退出登录</button>
      </view>
      <view v-else>
        <button @tap="handleLogin">登录</button>
      </view>
    </view>
  </view>
</template>

<script>
import { mapState, mapGetters, mapActions } from 'vuex'

export default {
  computed: {
    ...mapState({
      isLoading: state => state.isLoading,
      userInfo: state => state.user.userInfo
    }),
    ...mapGetters({
      isConnected: 'isConnected',
      isLoggedIn: 'user/isLoggedIn'
    })
  },
  onLoad() {
    this.checkNetworkStatus()
    if (this.isLoggedIn) {
      this.getUserInfo()
    }
  },
  methods: {
    ...mapActions({
      checkNetworkStatus: 'checkNetworkStatus',
      showLoading: 'showLoading',
      hideLoading: 'hideLoading',
      login: 'user/login',
      logout: 'user/logout',
      getUserInfo: 'user/getUserInfo'
    }),
    async handleLogin() {
      this.showLoading()
      try {
        await this.login({ name: '测试用户', password: '123456' })
        uni.showToast({
          title: '登录成功',
          icon: 'success'
        })
      } catch (error) {
        uni.showToast({
          title: '登录失败',
          icon: 'none'
        })
      } finally {
        this.hideLoading()
      }
    },
    async handleLogout() {
      this.logout()
      uni.showToast({
        title: '已退出登录',
        icon: 'success'
      })
    }
  }
}
</script>

Pinia 状态管理 ​

Pinia 是 Vue 3 的官方状态管理库，相比 Vuex，它提供了更简单的 API、更好的 TypeScript 支持，并且不再有 mutations 的概念。

Pinia 基本概念 ​

Pinia 的核心概念包括：

  1. State：状态，即数据源
  2. Getters：类似于 Vue 的计算属性，可以从 state 派生出状态
  3. Actions：可以包含业务逻辑和异步操作

在 UniApp 中集成 Pinia ​

  1. 首先安装 Pinia：
bash
npm install pinia --save
  1. 创建 store 目录结构：
├── store
│   ├── index.js          # 导出 pinia 实例
│   ├── modules
│       ├── user.js       # 用户模块
│       └── cart.js       # 购物车模块
  1. 在 store/index.js 中创建 pinia：
js
import { createPinia } from 'pinia'

const pinia = createPinia()

export default pinia
  1. 在 main.js 中挂载 pinia：
js
import { createSSRApp } from 'vue'
import App from './App.vue'
import pinia from './store'

export function createApp() {
  const app = createSSRApp(App)
  app.use(pinia)
  return {
    app
  }
}

Vuex 与 Pinia 对比 ​

特性VuexPinia
Vue 版本兼容Vue 2 和 Vue 3主要为 Vue 3 设计，但也兼容 Vue 2
API 复杂度相对复杂简单直观
TypeScript 支持有限支持完全支持，类型推断更好
模块化需要手动配置天然模块化设计
开发工具Vue Devtools 支持Vue Devtools 支持，且提供更好的调试体验
代码结构需要定义 mutations没有 mutations，直接修改 state
异步操作通过 actions直接在 actions 中处理
体积相对较大更小巧

实践心得 ​

  1. 选择合适的状态管理工具：

    • 对于 Vue 2 项目，可以使用 Vuex
    • 对于 Vue 3 项目，推荐使用 Pinia
    • 小型项目可以考虑使用 Vue 的 provide/inject 或简单的事件总线

  2. 状态管理最佳实践：

    • 只将需要共享的状态放入 store
    • 合理划分模块，避免单个 store 过于臃肿
    • 使用持久化插件保存关键状态

  3. 性能优化：

    • 避免在 getters 中进行复杂计算
    • 合理使用计算属性缓存派生状态
    • 避免直接存储大量数据，考虑按需加载

  4. 调试技巧：

    • 安装 Vue Devtools 进行状态调试
    • 在开发环境启用严格模式，捕获不规范的状态修改
    • 使用日志插件记录状态变化

实战案例 ​

电商应用的购物车模块 ​

以下是一个使用 Pinia 实现的购物车模块示例：

js
// store/modules/cart.js
import { defineStore } from 'pinia'

export const useCartStore = defineStore('cart', {
  state: () => ({
    items: uni.getStorageSync('cart_items') || [],
    couponCode: ''
  }),
  
  getters: {
    // 购物车商品数量
    count: (state) => state.items.length,
    
    // 购物车是否为空
    isEmpty: (state) => state.items.length === 0,
    
    // 购物车商品总价
    totalPrice: (state) => {
      return state.items.reduce((total, item) => {
        return total + (item.price * item.quantity)
      }, 0)
    },
    
    // 应用优惠后的价格
    finalPrice: (state, getters) => {
      // 假设优惠码提供 10% 折扣
      if (state.couponCode) {
        return getters.totalPrice * 0.9
      }
      return getters.totalPrice
    }
  },
  
  actions: {
    // 添加商品到购物车
    addItem(product, quantity = 1) {
      const existingItem = this.items.find(item => item.id === product.id)
      
      if (existingItem) {
        // 如果商品已存在，增加数量
        existingItem.quantity += quantity
      } else {
        // 添加新商品
        this.items.push({
          id: product.id,
          name: product.name,
          price: product.price,
          image: product.image,
          quantity
        })
      }
      
      // 保存到本地存储
      this.saveCart()
      
      // 显示提示
      uni.showToast({
        title: '已加入购物车',
        icon: 'success'
      })
    },
    
    // 从购物车移除商品
    removeItem(productId) {
      const index = this.items.findIndex(item => item.id === productId)
      if (index !== -1) {
        this.items.splice(index, 1)
        this.saveCart()
      }
    },
    
    // 更新商品数量
    updateItemQuantity(productId, quantity) {
      const item = this.items.find(item => item.id === productId)
      if (item) {
        item.quantity = quantity
        this.saveCart()
      }
    },
    
    // 清空购物车
    clearCart() {
      this.items = []
      this.saveCart()
    },
    
    // 应用优惠码
    applyCoupon(code) {
      // 这里可以添加优惠码验证逻辑
      this.couponCode = code
    },
    
    // 保存购物车到本地存储
    saveCart() {
      uni.setStorageSync('cart_items', this.items)
    }
  }
})

在购物车页面中使用： ​

vue
<template>
  <view class="cart-page">
    <view v-if="cartStore.isEmpty" class="empty-cart">
      <image src="/static/empty-cart.png" mode="aspectFit"></image>
      <text>购物车空空如也</text>
      <button @tap="goToHome" type="primary">去购物</button>
    </view>
    <view v-else class="cart-content">
      <!-- 购物车商品列表 -->
      <view class="cart-list">
        <view 
          v-for="(item, index) in cartStore.items" 
          :key="item.id" 
          class="cart-item"
        >
          <image class="item-image" :src="item.image" mode="aspectFill"></image>
          <view class="item-info">
            <text class="item-name">{{ item.name }}</text>
            <text class="item-price">¥{{ item.price.toFixed(2) }}</text>
          </view>
          <view class="item-quantity">
            <button 
              size="mini" 
              @tap="decreaseQuantity(item)"
              :disabled="item.quantity <= 1"
            >-</button>
            <text>{{ item.quantity }}</text>
            <button size="mini" @tap="increaseQuantity(item)">+</button>
          </view>
          <button 
            class="remove-btn" 
            size="mini" 
            type="warn"
            @tap="removeItem(item.id)"
          >删除</button>
        </view>
      </view>
      
      <!-- 优惠码 -->
      <view class="coupon-section">
        <input 
          class="coupon-input" 
          v-model="couponCode" 
          placeholder="请输入优惠码" 
        />
        <button 
          size="mini" 
          @tap="applyCoupon"
        >应用</button>
      </view>
      
      <!-- 价格汇总 -->
      <view class="price-summary">
        <view class="price-row">
          <text>商品总价</text>
          <text>¥{{ cartStore.totalPrice.toFixed(2) }}</text>
        </view>
        <view v-if="cartStore.couponCode" class="price-row discount">
          <text>优惠折扣</text>
          <text>-¥{{ (cartStore.totalPrice - cartStore.finalPrice).toFixed(2) }}</text>
        </view>
        <view class="price-row final">
          <text>应付金额</text>
          <text>¥{{ cartStore.finalPrice.toFixed(2) }}</text>
        </view>
      </view>
      
      <!-- 结算按钮 -->
      <view class="checkout-section">
        <button type="primary" @tap="checkout">去结算</button>
      </view>
    </view>
  </view>
</template>

<script setup>
import { ref } from 'vue'
import { useCartStore } from '@/store/modules/cart'

// 获取购物车 store
const cartStore = useCartStore()

// 优惠码输入
const couponCode = ref('')

// 去首页
const goToHome = () => {
  uni.switchTab({
    url: '/pages/index/index'
  })
}

// 增加商品数量
const increaseQuantity = (item) => {
  cartStore.updateItemQuantity(item.id, item.quantity + 1)
}

// 减少商品数量
const decreaseQuantity = (item) => {
  if (item.quantity > 1) {
    cartStore.updateItemQuantity(item.id, item.quantity - 1)
  }
}

// 移除商品
const removeItem = (productId) => {
  uni.showModal({
    title: '提示',
    content: '确定要移除该商品吗？',
    success: (res) => {
      if (res.confirm) {
        cartStore.removeItem(productId)
      }
    }
  })
}

// 应用优惠码
const applyCoupon = () => {
  if (!couponCode.value) {
    uni.showToast({
      title: '请输入优惠码',
      icon: 'none'
    })
    return
  }
  
  cartStore.applyCoupon(couponCode.value)
  uni.showToast({
    title: '优惠码已应用',
    icon: 'success'
  })
}

// 结算
const checkout = () => {
  uni.navigateTo({
    url: '/pages/order/checkout'
  })
}
</script>

总结 ​

UniApp 中的数据管理方案完全兼容 Vue 生态，可以根据项目的 Vue 版本选择 Vuex 或 Pinia。对于新项目，特别是基于 Vue 3 的项目，推荐使用 Pinia 作为状态管理工具，它提供了更简洁的 API 和更好的 TypeScript 支持。

在实际开发中，合理划分状态模块，只将需要共享的状态放入 store，可以提高代码的可维护性和性能。同时，利用本地存储实现状态持久化，可以提升用户体验。

通过本文的学习，我们掌握了在 UniApp 中使用 Vuex 和 Pinia 进行状态管理的方法，以及一些实用的最佳实践。这些知识将帮助我们构建更加复杂和可维护的 UniApp 应用。
