UniApp 页面与路由 ​

页面配置 ​

在 UniApp 中，页面配置主要通过 pages.json 文件进行管理，这是整个应用的核心配置文件之一。

pages 数组 ​

pages 数组定义应用包含的所有页面，数组中第一项表示应用的启动页。

json
"pages": [
  {
    "path": "pages/index/index",
    "style": {
      "navigationBarTitleText": "首页"
    }
  },
  {
    "path": "pages/category/category",
    "style": {
      "navigationBarTitleText": "分类"
    }
  }
]

每个页面对象包含：

  • path: 页面路径，相对于 src 目录
  • style: 页面样式配置，可以设置导航栏标题、背景色等

tabBar 配置 ​

tabBar 用于配置底部导航栏，常用于主要功能模块的切换。

json
"tabBar": {
  "color": "#7A7E83",
  "selectedColor": "#3cc51f",
  "borderStyle": "black",
  "backgroundColor": "#ffffff",
  "list": [
    {
      "pagePath": "pages/index/index",
      "iconPath": "static/tabbar/home.png",
      "selectedIconPath": "static/tabbar/home-active.png",
      "text": "首页"
    },
    {
      "pagePath": "pages/category/category",
      "iconPath": "static/tabbar/category.png",
      "selectedIconPath": "static/tabbar/category-active.png",
      "text": "分类"
    }
  ]
}

tabBar 配置包含：

  • color: 文字默认颜色
  • selectedColor: 文字选中时的颜色
  • borderStyle: 上边框的颜色，仅支持 black/white
  • backgroundColor: 背景色
  • list: 导航项目数组，最少2个，最多5个

globalStyle ​

globalStyle 用于设置应用的全局样式，会作用于每一个页面。

json
"globalStyle": {
  "navigationBarTextStyle": "black",
  "navigationBarTitleText": "应用名称",
  "navigationBarBackgroundColor": "#F8F8F8",
  "backgroundColor": "#F8F8F8"
}

页面跳转方式 ​

UniApp 提供了多种页面跳转方式，适用于不同的场景：

1. 普通页面跳转 - navigateTo ​

保留当前页面，跳转到应用内的某个页面。

js
uni.navigateTo({
  url: '/pages/product/detail?id=1'
})

适用场景：

  • 需要返回到当前页面的场景
  • 页面层级较少的场景

2. 重定向页面 - redirectTo ​

关闭当前页面，跳转到应用内的某个页面。

js
uni.redirectTo({
  url: '/pages/login/login'
})

适用场景：

  • 不需要返回到当前页面的场景
  • 避免页面栈过深的场景

3. Tab 切换 - switchTab ​

跳转到 tabBar 页面，并关闭其他所有非 tabBar 页面。

js
uni.switchTab({
  url: '/pages/index/index'
})

适用场景：

  • 切换到 tabBar 页面
  • 重置应用状态

4. 返回页面 - navigateBack ​

关闭当前页面，返回上一页面或多级页面。

js
uni.navigateBack({
  delta: 1 // 返回的页面数，如果 delta 大于现有页面数，则返回到首页
})

适用场景：

  • 完成操作后返回
  • 取消操作返回

5. 重启应用 - reLaunch ​

关闭所有页面，打开到应用内的某个页面。

js
uni.reLaunch({
  url: '/pages/index/index'
})

适用场景：

  • 需要重置整个应用状态
  • 登录成功后跳转到主页

页面传参 ​

URL 传参 ​

在跳转时通过 URL 携带参数：

js
uni.navigateTo({
  url: '/pages/product/detail?id=1&name=product'
})

在目标页面的 onLoad 生命周期中接收参数：

js
export default {
  onLoad(options) {
    const id = options.id
    const name = options.name
    console.log('接收到的参数：', id, name)
  }
}

事件总线通信 ​

UniApp 提供了事件机制用于页面间通信：

js
// 发送事件
uni.$emit('updateCart', { count: 5 })

// 接收事件
uni.$on('updateCart', function(data) {
  console.log('购物车更新：', data.count)
})

// 移除事件监听
uni.$off('updateCart')

页面栈操作 ​

获取当前页面栈，实现页面间数据传递：

js
const pages = getCurrentPages()
const currentPage = pages[pages.length - 1]
const prevPage = pages[pages.length - 2]

// 操作上一页数据
if (prevPage) {
  prevPage.$vm.refreshData()
}

实践心得 ​

  1. 页面结构设计：

    • 将常用功能放在 tabBar 中，最多不超过5个
    • 将相关页面放在同一个目录下，便于管理
    • 遵循"扁平化"原则，避免过深的页面层级

  2. 页面跳转选择：

    • 需要返回的场景使用 navigateTo
    • 登录、注册等流程使用 redirectTo
    • 主要功能模块切换使用 switchTab
    • 避免过多的页面堆栈，导致内存占用过大

  3. 参数传递：

    • 简单数据使用 URL 参数传递
    • 复杂数据可以使用事件总线或全局状态管理
    • 大量数据可以考虑本地存储

  4. 注意事项：

    • tabBar 页面不支持传参，需要使用其他方式共享数据
    • 页面栈最多10层，超出后无法再使用 navigateTo
    • 小程序端对页面大小有限制，页面文件过大可能导致加载缓慢

实战案例 ​

在我们的电商应用中，我们实现了以下页面导航结构：

  1. tabBar 页面：

    • 首页：展示商品推荐和活动
    • 分类：展示商品分类
    • 购物车：展示已添加商品
    • 我的：用户中心

  2. 页面跳转逻辑：

    • 从首页/分类点击商品，使用 navigateTo 跳转到商品详情页
    • 从商品详情页添加商品到购物车后，可以使用 switchTab 跳转到购物车页面
    • 未登录用户访问"我的"页面中的订单等功能时，使用 navigateTo 跳转到登录页
    • 登录成功后，使用 navigateBack 返回之前的页面

  3. 参数传递示例：

    js
    // 跳转到商品详情页
uni.navigateTo({
  url: `/pages/product/detail?id=${product.id}&from=home`
})

// 商品详情页接收参数
export default {
  data() {
    return {
      productId: '',
      fromPage: ''
    }
  },
  onLoad(options) {
    this.productId = options.id
    this.fromPage = options.from
    this.loadProductDetail()
  },
  methods: {
    loadProductDetail() {
      // 根据 productId 加载商品详情
      console.log(`加载商品 ${this.productId} 的详情`)
    }
  }
}

总结 ​

UniApp 的页面路由系统设计合理，既保留了 Vue 的开发体验，又兼容了各平台的特性。通过 pages.json 配置，我们可以轻松管理页面和导航栏。不同的页面跳转方式满足了不同的业务场景需求，页面传参方式也比较灵活。

在实际开发中，合理规划页面结构和跳转逻辑，可以提升用户体验和应用性能。同时，了解各平台的限制和差异，可以避免在跨平台开发中遇到兼容性问题。
