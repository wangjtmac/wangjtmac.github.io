UniApp 性能优化 ​

性能优化是移动应用开发中的关键环节，对于跨平台应用尤为重要。本文将介绍 UniApp 应用中常见的性能优化策略，包括首屏加载优化、长列表优化和图片优化等方面。

4. 减少初始数据量 ​

  • 采用分页加载而非一次性加载全部数据
  • 初始只加载必要的数据，其他数据按需加载
  • 使用骨架屏提升用户体验
vue
<template>
  <view class="container">
    <!-- 骨架屏 -->
    <view v-if="loading" class="skeleton">
      <view class="skeleton-header"></view>
      <view class="skeleton-content">
        <view class="skeleton-item" v-for="i in 5" :key="i"></view>
      </view>
    </view>
    
    <!-- 实际内容 -->
    <view v-else>
      <view class="header">{{ title }}</view>
      <view class="content">
        <view class="item" v-for="(item, index) in list" :key="index">
          {{ item.name }}
        </view>
      </view>
    </view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      loading: true,
      title: '',
      list: [],
      page: 1,
      pageSize: 10
    };
  },
  onLoad() {
    this.loadInitialData();
  },
  methods: {
    async loadInitialData() {
      try {
        const res = await this.$api.getData({
          page: this.page,
          pageSize: this.pageSize
        });
        
        this.title = res.title;
        this.list = res.list;
      } catch (err) {
        console.error(err);
      } finally {
        this.loading = false;
      }
    }
  }
}
</script>

长列表优化 ​

长列表是移动应用中常见的性能瓶颈，特别是在低端设备上。

1. 虚拟列表 ​

对于大量数据的列表，可以使用虚拟列表技术，只渲染可视区域的内容：

vue
<template>
  <view class="list-container">
    <view class="list" :style="{ height: totalHeight + 'px' }">
      <view 
        v-for="item in visibleItems" 
        :key="item.id"
        class="list-item"
        :style="{ transform: `translateY(${item.top}px)` }"
      >
        {{ item.content }}
      </view>
    </view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      allItems: [], // 所有数据
      visibleItems: [], // 可见数据
      itemHeight: 60, // 每项高度
      screenHeight: 0, // 屏幕高度
      scrollTop: 0, // 滚动位置
      bufferSize: 5 // 缓冲区大小
    };
  },
  computed: {
    totalHeight() {
      return this.allItems.length * this.itemHeight;
    }
  },
  onLoad() {
    // 生成测试数据
    this.allItems = Array.from({ length: 1000 }, (_, i) => ({
      id: i,
      content: `Item ${i}`,
      top: i * this.itemHeight
    }));
    
    // 获取屏幕高度
    const systemInfo = uni.getSystemInfoSync();
    this.screenHeight = systemInfo.windowHeight;
    
    // 初始化可见项
    this.updateVisibleItems(0);
  },
  onPageScroll(e) {
    this.scrollTop = e.scrollTop;
    this.updateVisibleItems(e.scrollTop);
  },
  methods: {
    updateVisibleItems(scrollTop) {
      // 计算可见范围
      const startIndex = Math.max(0, Math.floor(scrollTop / this.itemHeight) - this.bufferSize);
      const endIndex = Math.min(
        this.allItems.length,
        Math.ceil((scrollTop + this.screenHeight) / this.itemHeight) + this.bufferSize
      );
      
      // 更新可见项
      this.visibleItems = this.allItems.slice(startIndex, endIndex);
    }
  }
};
</script>

<style>
.list-container {
  position: relative;
  width: 100%;
}

.list {
  position: relative;
  width: 100%;
}

.list-item {
  position: absolute;
  width: 100%;
  height: 60px;
  line-height: 60px;
  padding: 0 20px;
  border-bottom: 1px solid #eee;
}
</style>

2. 分页加载 ​

结合下拉刷新和上拉加载实现分页加载：

vue
<template>
  <view class="page">
    <view class="list">
      <view 
        class="item" 
        v-for="(item, index) in list" 
        :key="index"
      >
        {{ item.name }}
      </view>
    </view>
    
    <!-- 加载更多 -->
    <view class="loading-more" v-if="hasMore">
      {{ loadingMore ? '加载中...' : '上拉加载更多' }}
    </view>
    <view class="no-more" v-else>没有更多数据了</view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      list: [],
      page: 1,
      pageSize: 20,
      hasMore: true,
      loadingMore: false,
      refreshing: false
    };
  },
  onLoad() {
    this.loadData();
  },
  onPullDownRefresh() {
    this.refreshing = true;
    this.page = 1;
    this.loadData(true);
  },
  onReachBottom() {
    if (this.hasMore && !this.loadingMore) {
      this.loadingMore = true;
      this.page++;
      this.loadData();
    }
  },
  methods: {
    async loadData(isRefresh = false) {
      try {
        const res = await this.$api.getList({
          page: this.page,
          pageSize: this.pageSize
        });
        
        if (isRefresh) {
          this.list = res.list;
        } else {
          this.list = [...this.list, ...res.list];
        }
        
        // 判断是否还有更多数据
        this.hasMore = res.list.length === this.pageSize;
      } catch (err) {
        console.error(err);
      } finally {
        this.loadingMore = false;
        this.refreshing = false;
        
        if (isRefresh) {
          uni.stopPullDownRefresh();
        }
      }
    }
  }
};
</script>

3. 避免频繁更新 ​

  • 使用 nextTick 合并多次更新
  • 减少不必要的响应式数据
  • 使用 v-once 渲染静态内容
vue
<template>
  <view class="container">
    <!-- 静态内容使用 v-once -->
    <view class="static-content" v-once>
      <text>{{ staticText }}</text>
    </view>
    
    <!-- 动态内容 -->
    <view class="dynamic-content">
      <text>{{ dynamicText }}</text>
    </view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      staticText: '这是静态内容，只渲染一次',
      dynamicText: '初始动态内容'
    };
  },
  methods: {
    updateContent() {
      // 一次性更新多个值
      this.$nextTick(() => {
        this.dynamicText = '更新后的动态内容';
        // 其他更新...
      });
    }
  }
};
</script>

图片优化 ​

图片是移动应用中最占用资源的元素之一，优化图片可以显著提升应用性能。

1. 图片懒加载 ​

vue
<template>
  <view class="image-list">
    <view 
      class="image-item" 
      v-for="(item, index) in images" 
      :key="index"
    >
      <image 
        :src="item.loaded ? item.url : defaultImage" 
        :data-index="index"
        :data-url="item.url"
        @load="onImageLoad"
        lazy-load
      ></image>
    </view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      defaultImage: '/static/images/placeholder.png',
      images: [
        { url: '/static/images/image1.jpg', loaded: false },
        { url: '/static/images/image2.jpg', loaded: false },
        // ...更多图片
      ]
    };
  },
  methods: {
    onImageLoad(e) {
      const { index } = e.currentTarget.dataset;
      this.images[index].loaded = true;
    }
  }
};
</script>

2. 图片压缩和格式选择 ​

  • 使用 WebP 格式替代 JPG/PNG（注意兼容性）
  • 根据屏幕尺寸和分辨率提供不同尺寸的图片
  • 使用图片压缩工具减小图片体积
vue
<template>
  <view class="image-container">
    <!-- 根据平台和屏幕尺寸选择不同格式和尺寸的图片 -->
    <image :src="optimizedImageSrc" mode="widthFix"></image>
  </view>
</template>

<script>
export default {
  data() {
    return {
      imageId: 'product-123'
    };
  },
  computed: {
    optimizedImageSrc() {
      const platform = uni.getSystemInfoSync().platform;
      const pixelRatio = uni.getSystemInfoSync().pixelRatio || 1;
      const width = uni.getSystemInfoSync().windowWidth;
      
      // 计算需要的图片宽度
      const targetWidth = Math.round(width * pixelRatio);
      
      // 根据平台选择最佳格式
      let format = 'jpg';
      if (platform === 'android') {
        format = 'webp'; // Android 支持 WebP
      }
      
      // 返回优化后的图片 URL
      return `https://example.com/images/${this.imageId}_${targetWidth}w.${format}`;
    }
  }
};
</script>

3. 图片预加载 ​

对于轮播图等重要图片，可以提前预加载：

js
// 图片预加载函数
function preloadImages(urls) {
  if (!Array.isArray(urls)) {
    urls = [urls];
  }
  
  const promises = urls.map(url => {
    return new Promise((resolve, reject) => {
      uni.getImageInfo({
        src: url,
        success: () => {
          console.log('预加载图片成功:', url);
          resolve(url);
        },
        fail: (err) => {
          console.error('预加载图片失败:', url, err);
          reject(err);
        }
      });
    });
  });
  
  return Promise.all(promises);
}

// 使用
export default {
  data() {
    return {
      bannerImages: [
        'https://example.com/banner1.jpg',
        'https://example.com/banner2.jpg',
        'https://example.com/banner3.jpg'
      ],
      imagesLoaded: false
    };
  },
  onLoad() {
    // 预加载轮播图
    preloadImages(this.bannerImages).then(() => {
      this.imagesLoaded = true;
    }).catch(err => {
      console.error('轮播图预加载失败', err);
    });
  }
};

2. 避免频繁 GC ​

  • 避免在循环中创建大量临时对象
  • 合理使用对象池模式
  • 避免闭包导致的内存泄漏
js
// 不良示例
function processItems(items) {
  for (let i = 0; i < items.length; i++) {
    const temp = { ... }; // 每次循环创建新对象
    // 处理逻辑
  }
}

// 优化后
function processItems(items) {
  const temp = {}; // 复用对象
  for (let i = 0; i < items.length; i++) {
    // 重置 temp 对象属性
    Object.keys(temp).forEach(key => {
      delete temp[key];
    });
    
    // 设置新属性
    // ...
    
    // 处理逻辑
  }
}

总结 ​

UniApp 应用的性能优化是一个系统工程，需要从多个方面入手：

  1. 首屏加载优化：减少初始加载资源，使用骨架屏，预加载关键资源
  2. 长列表优化：使用虚拟列表，分页加载，减少不必要的更新
  3. 图片优化：懒加载，合理选择格式和尺寸，使用 CDN
  4. 代码优化：减少不必要的计算，避免频繁 GC，使用 worker 处理耗时任务

通过综合应用这些优化策略，可以显著提升 UniApp 应用的性能和用户体验。在实际开发中，应根据具体场景和需求，选择合适的优化方案。
