UniApp 生命周期 ​

生命周期概述 ​

UniApp 的生命周期分为应用生命周期和页面生命周期两部分，它们分别控制应用和页面的不同阶段的行为。理解这些生命周期钩子函数对于开发高效的应用至关重要。

应用生命周期 ​

应用生命周期是指 App.vue 中定义的生命周期函数，用于监听应用的打开、关闭等事件。

应用生命周期函数 ​

函数名说明触发时机平台差异
onLaunch应用初始化当应用第一次启动时触发-
onShow应用显示应用启动或从后台进入前台时触发-
onHide应用隐藏应用从前台进入后台时触发-
onError应用错误应用发生 js 错误时触发-
onUniNViewMessage应用收到原生页面消息监听 nvue 页面发送的消息App
onUnhandledRejection未处理的 Promise 拒绝当 Promise 被 reject 且没有 reject 处理器时触发-
onPageNotFound页面不存在应用要打开的页面不存在时触发-
onThemeChange系统主题变化系统主题发生变化时触发-

应用生命周期示例 ​

vue
<!-- App.vue -->
<script>
export default {
  onLaunch: function(options) {
    console.log('App onLaunch')
    console.log('启动参数:', options)
    // 可以在这里进行应用初始化、版本检查、更新提示等操作
  },
  onShow: function(options) {
    console.log('App onShow')
    console.log('显示参数:', options)
    // 可以在这里处理应用从后台切换到前台时的逻辑
    // 例如：恢复音频播放、刷新数据等
  },
  onHide: function() {
    console.log('App onHide')
    // 可以在这里处理应用从前台切换到后台时的逻辑
    // 例如：暂停音频播放、保存数据等
  },
  onError: function(err) {
    console.log('App onError')
    console.error(err)
    // 可以在这里进行错误日志上报
  },
  onPageNotFound: function(options) {
    console.log('App onPageNotFound')
    console.log(options)
    // 可以在这里处理页面不存在的情况
    // 例如：跳转到首页或错误页
    uni.navigateTo({
      url: '/pages/error/404'
    })
  }
}
</script>

页面生命周期 ​

页面生命周期是指在 pages 目录下的 vue 文件中定义的生命周期函数，用于监听页面的加载、显示、隐藏等事件。

页面生命周期函数 ​

函数名说明触发时机平台差异
onLoad页面加载页面加载时触发，只触发一次-
onShow页面显示页面显示或从后台切换到前台时触发-
onReady页面初次渲染完成页面初次渲染完成时触发，只触发一次-
onHide页面隐藏页面隐藏或从前台切换到后台时触发-
onUnload页面卸载页面卸载时触发-
onPullDownRefresh下拉刷新用户下拉刷新时触发-
onReachBottom上拉触底用户上拉触底时触发-
onTabItemTap点击 Tab 时触发点击 Tab 时触发，参数为 Object微信小程序、百度小程序、H5、App（自定义组件模式）
onShareAppMessage用户点击右上角分享用户点击右上角分享时触发微信小程序、百度小程序、字节跳动小程序、支付宝小程序
onPageScroll页面滚动页面滚动时触发-
onResize页面尺寸变化页面尺寸变化时触发App、微信小程序
onNavigationBarButtonTap原生标题栏按钮点击事件原生标题栏按钮点击时触发App、H5
onBackPress页面返回页面返回时触发App、H5
onNavigationBarSearchInputChanged原生标题栏搜索输入框内容变化事件原生标题栏搜索输入框内容变化时触发App、H5
onNavigationBarSearchInputConfirmed原生标题栏搜索输入框搜索事件原生标题栏搜索输入框搜索时触发App、H5
onNavigationBarSearchInputClicked原生标题栏搜索输入框点击事件原生标题栏搜索输入框点击时触发App、H5

  onShow() {
    console.log('页面 onShow')
    // 可以在这里刷新页面数据
  },
  onReady() {
    console.log('页面 onReady')
    // 可以在这里获取页面元素、执行动画等
  },
  onHide() {
    console.log('页面 onHide')
    // 可以在这里暂停页面相关操作
  },
  onUnload() {
    console.log('页面 onUnload')
    // 可以在这里清理页面资源
  },
  onPullDownRefresh() {
    console.log('页面 onPullDownRefresh')
    // 下拉刷新逻辑
    setTimeout(() => {
      // 刷新完成后停止下拉刷新动画

    }, 1000)
  },
  onReachBottom() {

    // 上拉加载更多逻辑

    // console.log('页面滚动', e.scrollTop)

    return {
      title: this.title,
      path: '/pages/index/index',
      imageUrl: '/static/share.png'
    }
  },
  onBackPress(options) {
    // 返回 true 阻止默认返回行为
    console.log('用户点击了返回', options)
    // 如果需要自定义返回行为，返回 true
    // return true
  }
}
</script>

组件生命周期 ​

UniApp 组件生命周期与 Vue 组件的生命周期一致，主要包括以下钩子函数：

Vue2 组件生命周期 ​

函数名说明触发时机
beforeCreate组件实例创建前在实例初始化之后，数据观测 (data observer) 和 event/watcher 事件配置之前被调用
created组件实例创建后在实例创建完成后被立即调用，此时已完成数据观测、属性和方法的运算，但尚未开始 DOM 渲染
beforeMount组件挂载前在挂载开始之前被调用，相关的 render 函数首次被调用
mounted组件挂载后在实例挂载完成后被调用，此时可以操作 DOM
beforeUpdate数据更新前数据更新时调用，发生在虚拟 DOM 重新渲染和打补丁之前
updated数据更新后数据更新后调用，此时 DOM 已经更新
beforeDestroy组件销毁前实例销毁之前调用，此时实例仍然可用
destroyed组件销毁后实例销毁后调用，此时所有的事件监听器已被移除，子实例也已被销毁

Vue3 组件生命周期（组合式 API） ​

如果使用 Vue3 的组合式 API，生命周期钩子函数名称有所变化：

生命周期执行顺序 ​

在一个完整的 UniApp 应用中，生命周期的执行顺序如下：

  1. 应用启动：

    • App.vue 中的 onLaunch
    • App.vue 中的 onShow

  2. 首页加载：

    • 页面的 onLoad
    • 页面的 onShow
    • 页面组件的 beforeCreate
    • 页面组件的 created
    • 页面组件的 beforeMount
    • 页面的 onReady
    • 页面组件的 mounted

  3. 页面跳转：

    • 当前页面的 onHide
    • 新页面的 onLoad
    • 新页面的 onShow
    • 新页面组件的 beforeCreate
    • 新页面组件的 created
    • 新页面组件的 beforeMount
    • 新页面的 onReady
    • 新页面组件的 mounted

  4. 页面返回：

    • 当前页面的 onUnload
    • 当前页面组件的 beforeDestroy
    • 当前页面组件的 destroyed
    • 返回页面的 onShow

  5. 应用切换到后台：

    • 页面的 onHide
    • App.vue 中的 onHide

  6. 应用从后台切换到前台：

    • App.vue 中的 onShow
    • 页面的 onShow

实践心得 ​

  1. 合理使用生命周期函数：

    • onLoad: 用于获取页面参数、初始化数据
    • onShow: 用于刷新页面数据、恢复页面状态
    • onReady: 用于获取页面元素、执行需要 DOM 的操作
    • onHide/onUnload: 用于保存页面状态、清理资源

  2. 避免生命周期函数中的常见错误：

    • 不要在 onPageScroll 中执行耗时操作，会影响滚动性能
    • 不要在生命周期函数中使用 async/await，可能导致执行顺序混乱
    • 不要在 onLoad 中直接操作 DOM，此时 DOM 可能尚未渲染完成

  3. 页面数据加载策略：

    • 简单数据：直接在 onLoad 中加载
    • 复杂数据：在 onLoad 中显示加载状态，然后异步加载数据
    • 需要刷新的数据：在 onShow 中加载，确保每次显示页面都是最新数据

  4. 组件生命周期与页面生命周期的结合：

    • 页面组件的 mounted 在页面的 onReady 之前执行
    • 如果需要在组件中获取页面参数，可以通过 props 传递或使用全局状态管理

  5. 平台差异处理：

    • 使用条件编译处理不同平台的生命周期差异
    • 某些生命周期函数（如 onShareAppMessage）仅在特定平台可用

实战案例 ​

商品详情页生命周期管理 ​

vue
<template>
  <view class="product-detail">
    <view v-if="loading" class="loading">
      <text>加载中...</text>
    </view>
    <block v-else>
      <swiper class="product-swiper" indicator-dots autoplay circular>
        <swiper-item v-for="(item, index) in product.images" :key="index">
          <image :src="item" mode="aspectFill"></image>
        </swiper-item>
      </swiper>
      <view class="product-info">
        <text class="product-name">{{ product.name }}</text>
        <text class="product-price">¥{{ product.price }}</text>
        <text class="product-desc">{{ product.description }}</text>
      </view>
      <view class="product-actions">
        <button @tap="addToCart">加入购物车</button>
        <button type="primary" @tap="buyNow">立即购买</button>
      </view>
    </block>
  </view>
</template>

<script>
export default {
  data() {
    return {
      productId: '',
      product: {},
      loading: true,
      viewCount: 0
    }
  },
  onLoad(options) {
    // 获取页面参数
    this.productId = options.id
    console.log('商品ID:', this.productId)
    
    // 加载商品数据
    this.loadProductDetail()
  },
  onShow() {
    // 更新浏览记录
    this.updateViewHistory()
    
    // 增加浏览次数
    this.viewCount++
    console.log('浏览次数:', this.viewCount)
  },
  onReady() {
    // 设置导航栏标题
    uni.setNavigationBarTitle({
      title: this.product.name || '商品详情'
    })
  },
  onHide() {
    // 保存页面状态
    this.savePageState()
  },
  onUnload() {
    // 清理资源
    console.log('页面卸载，清理资源')
    // 例如取消未完成的网络请求等
  },
  onPullDownRefresh() {
    // 下拉刷新，重新加载商品数据
    this.loadProductDetail(true)
  },
  onShareAppMessage() {
    // 自定义分享内容
    return {
      title: this.product.name,
      path: `/pages/product/detail?id=${this.productId}`,
      imageUrl: this.product.images ? this.product.images[0] : ''
    }
  },
  methods: {
    // 加载商品详情
    async loadProductDetail(isRefresh = false) {
      this.loading = true
      
      try {
        // 模拟API请求
        await new Promise(resolve => setTimeout(resolve, 1000))
        
        // 假设这是从API获取的数据
        this.product = {
          id: this.productId,
          name: '示例商品',
          price: 99.99,
          description: '这是一个示例商品描述',
          images: [
            '/static/product1.jpg',
            '/static/product2.jpg'
          ]
        }
        
        // 更新页面标题
        if (this.product.name) {
          uni.setNavigationBarTitle({
            title: this.product.name
          })
        }
      } catch (error) {
        console.error('加载商品详情失败', error)
        uni.showToast({
          title: '加载失败',
          icon: 'none'
        })
      } finally {
        this.loading = false
        
        // 如果是下拉刷新，停止下拉刷新动画
        if (isRefresh) {
          uni.stopPullDownRefresh()
        }
      }
    },
    
    // 更新浏览记录
    updateViewHistory() {
      try {
        // 获取浏览历史
        const historyString = uni.getStorageSync('viewHistory') || '[]'
        const history = JSON.parse(historyString)
        
        // 添加当前商品到浏览历史
        const exists = history.findIndex(item => item.id === this.productId)
        if (exists >= 0) {
          // 已存在，移到最前面
          history.splice(exists, 1)
        }
        
        // 添加到历史记录开头
        history.unshift({
          id: this.productId,
          time: Date.now()
        })
        
        // 只保留最近20条记录
        const recentHistory = history.slice(0, 20)
        
        // 保存回本地存储
        uni.setStorageSync('viewHistory', JSON.stringify(recentHistory))
      } catch (e) {
        console.error('更新浏览历史失败', e)
      }
    },
    
    // 保存页面状态
    savePageState() {
      // 可以保存滚动位置等状态
      const scrollPosition = 0 // 假设这是获取到的滚动位置
      uni.setStorageSync(`product_${this.productId}_scroll`, scrollPosition)
    },
    
    // 加入购物车
    addToCart() {
      uni.showToast({
        title: '已加入购物车',
        icon: 'success'
      })
    },
    
    // 立即购买
    buyNow() {
      uni.navigateTo({
        url: `/pages/order/confirm?productId=${this.productId}`
      })
    }
  }
}
</script>

总结 ​

UniApp 的生命周期系统继承了 Vue 的生命周期，并结合了小程序的页面生命周期，形成了一套完整的应用、页面和组件生命周期机制。通过合理利用这些生命周期函数，我们可以在适当的时机执行相应的操作，提升应用的性能和用户体验。

在实际开发中，应当深入理解各个生命周期函数的触发时机和使用场景，避免常见的生命周期使用错误，并针对不同平台的差异进行适当的处理。通过这些最佳实践，可以开发出更加稳定、高效的 UniApp 应用。
