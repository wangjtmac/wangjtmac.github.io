Vue2 vs Vue3 ​

Vue 3 是 Vue.js 的最新主要版本，相比 Vue 2 带来了许多重大改进。本文将对比 Vue 2 和 Vue 3 的主要区别，帮助你理解 Vue 的演进方向。

核心架构变化 ​

响应式系统 ​

Vue 2:

  • 使用 Object.defineProperty 实现响应式
  • 无法检测对象属性的添加和删除
  • 无法检测数组索引和长度的变化
  • 需要使用 Vue.set 和 Vue.delete 来处理这些限制
js
// Vue 2 响应式示例
const data = {
  message: "Hello",
};

Object.defineProperty(data, "message", {
  get() {
    // 依赖收集
    return value;
  },
  set(newValue) {
    // 触发更新
    value = newValue;
  },
});

Vue 3:

  • 使用 ES6 的 Proxy 重写响应式系统
  • 可以检测对象属性的添加和删除
  • 可以检测数组索引和长度的变化
  • 性能更好，内存占用更低
js
// Vue 3 响应式示例
const data = {
  message: "Hello",
};

const proxy = new Proxy(data, {
  get(target, key) {
    // 依赖收集
    return target[key];
  },
  set(target, key, value) {
    // 触发更新
    target[key] = value;
    return true;
  },
});

API 风格 ​

Vue 2:

  • 主要使用选项式 API (Options API)
  • 通过 data, methods, computed 等选项组织代码
  • 随着组件复杂度增加，相关逻辑分散在不同选项中
js
// Vue 2 选项式 API
export default {
  data() {
    return {
      count: 0,
    };
  },
  methods: {
    increment() {
      this.count++;
    },
  },
  computed: {
    doubleCount() {
      return this.count * 2;
    },
  },
};

Vue 3:

  • 引入组合式 API (Composition API)
  • 可以按照逻辑关注点组织代码
  • 更好的代码复用和组织
  • 仍然支持选项式 API
js
// Vue 3 组合式 API
import { ref, computed } from "vue";

export default {
  setup() {
    const count = ref(0);

    function increment() {
      count.value++;
    }

    const doubleCount = computed(() => count.value * 2);

    return {
      count,
      increment,
      doubleCount,
    };
  },
};

性能提升 ​

Vue 3:

  • 重写虚拟 DOM 实现，渲染性能提升
  • 编译优化：
    • 静态树提升
    • 静态属性提升
    • 基于 Proxy 的响应式系统
  • 更小的包体积，通过 tree-shaking 优化
  • 更好的 TypeScript 支持

新特性 ​

多根节点组件 (Fragments) ​

Vue 2:

  • 组件模板必须有一个单一根节点
html
<!-- Vue 2 -->
<template>
  <div>
    <header>...</header>
    <main>...</main>
    <footer>...</footer>
  </div>
</template>

Vue 3:

  • 支持多根节点组件
html
<!-- Vue 3 -->
<template>
  <header>...</header>
  <main>...</main>
  <footer>...</footer>
</template>

Teleport ​

Vue 3 引入了 <teleport> 组件，允许将内容渲染到 DOM 树的不同位置：

html
<teleport to="body">
  <div class="modal">
    <!-- 模态框内容 -->
  </div>
</teleport>

Suspense ​

Vue 3 引入了 <suspense> 组件，用于处理异步组件和异步数据：

html
<suspense>
  <template #default>
    <async-component />
  </template>
  <template #fallback>
    <loading-spinner />
  </template>
</suspense>

组合式函数 (Composables) ​

Vue 3 的组合式 API 使得创建可复用的逻辑变得更加简单：

js
// useCounter.js
import { ref, computed } from "vue";

export function useCounter() {
  const count = ref(0);

  function increment() {
    count.value++;
  }

  const doubleCount = computed(() => count.value * 2);

  return {
    count,
    increment,
    doubleCount,
  };
}

// 在组件中使用
import { useCounter } from "./useCounter";

export default {
  setup() {
    const { count, increment, doubleCount } = useCounter();

    return {
      count,
      increment,
      doubleCount,
    };
  },
};

生态系统变化 ​

Vue 3 的发布也带动了整个生态系统的更新：

  • Vue Router 4: 适配 Vue 3，API 改进
  • Vuex 4: 适配 Vue 3，保持与 Vuex 3 相同的 API
  • Pinia: 新一代状态管理库，被视为 Vuex 5 的雏形
  • Element Plus: Element UI 的 Vue 3 版本
  • Vant 3+: Vant 的 Vue 3 版本

迁移考虑 ​

从 Vue 2 迁移到 Vue 3 时需要考虑的因素：

  1. 破坏性变更：Vue 3 有一些 API 变更和移除
  2. 构建工具：可能需要更新 Webpack、Babel 等配置
  3. 依赖库：确保使用的第三方库支持 Vue 3
  4. 浏览器支持：Vue 3 不支持 IE11

Vue 官方提供了迁移指南来帮助开发者平滑迁移。

总结 ​

Vue 3 相比 Vue 2 有以下主要优势：

  1. 更好的性能：重写的虚拟 DOM 和响应式系统
  2. 更小的包体积：通过 tree-shaking 优化
  3. 更好的 TypeScript 支持
  4. 组合式 API：提供更灵活的代码组织方式
  5. 新特性：Fragments、Teleport、Suspense 等

但 Vue 2 仍然是一个成熟稳定的框架，有着庞大的生态系统和社区支持。选择哪个版本取决于项目需求、团队熟悉度和浏览器兼容性要求。
