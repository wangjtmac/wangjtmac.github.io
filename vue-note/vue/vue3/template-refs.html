Vue3 组合式 API 中的模板引用 ​

在 Vue 中，模板引用（Template Refs）允许我们直接访问底层 DOM 元素或子组件实例。在 Vue 3 的组合式 API 中，模板引用的工作方式与 Vue 2 有所不同，本文将详细介绍如何在组合式 API 中使用模板引用。

基本概念 ​

模板引用是通过 ref 属性来实现的，它允许我们获取对特定 DOM 元素或组件实例的引用。在组合式 API 中，模板引用与响应式系统中的 ref 结合使用，使得获取元素引用变得更加简单和直观。

在组合式 API 中使用模板引用 ​

基本用法 ​

在 <script setup> 中使用模板引用非常简单：

vue
<script setup>
import { ref, onMounted } from "vue";

// 声明一个同名的 ref
const inputRef = ref(null);

onMounted(() => {
  // 在挂载后，可以访问到 DOM 元素
  inputRef.value.focus();
});
</script>

<template>
  <!-- 将 ref 属性绑定到模板中的元素 -->
  <input ref="inputRef" />
</template>

上面的代码中，我们创建了一个名为 inputRef 的 ref，并将其绑定到 <input> 元素上。当组件挂载后，inputRef.value 将包含对该 DOM 元素的引用，我们可以直接操作它。

在 v-for 中使用模板引用 ​

当在 v-for 中使用模板引用时，引用的值将是一个数组，包含所有对应的元素：

vue
<script setup>
import { ref, onMounted } from "vue";

// 声明一个 ref 来存储多个元素引用
const itemRefs = ref([]);

onMounted(() => {
  // itemRefs.value 是一个数组，包含所有元素引用
  itemRefs.value.forEach((item) => {
    console.log(item.textContent);
  });
});
</script>

<template>
  <ul>
    <li v-for="(item, index) in ['A', 'B', 'C']" :key="index" ref="itemRefs">
      {{ item }}
    </li>
  </ul>
</template>

函数型 ref ​

我们也可以传递一个函数给 ref 属性，这个函数会在每次元素被绑定或解绑时调用：

vue
<script setup>
import { ref, onMounted } from "vue";

const itemRefs = ref([]);

function setItemRef(el) {
  if (el) {
    itemRefs.value.push(el);
  }
}

onMounted(() => {
  console.log(itemRefs.value.length); // 输出元素数量
});
</script>

<template>
  <ul>
    <li v-for="(item, index) in ['A', 'B', 'C']" :key="index" :ref="setItemRef">
      {{ item }}
    </li>
  </ul>
</template>

这种方法在处理动态列表时特别有用，因为它可以让我们完全控制如何管理元素引用。

组件引用 ​

模板引用不仅可以用于 DOM 元素，还可以用于子组件：

vue
<script setup>
import { ref, onMounted } from "vue";
import ChildComponent from "./ChildComponent.vue";

const childRef = ref(null);

onMounted(() => {
  // 访问子组件的公开方法或属性
  childRef.value.someMethod();
});
</script>

<template>
  <ChildComponent ref="childRef" />
</template>

在 Vue 3 中，子组件需要使用 defineExpose 宏来显式暴露其内部属性和方法：

vue
<!-- ChildComponent.vue -->
<script setup>
import { ref } from "vue";

const count = ref(0);

function increment() {
  count.value++;
}

// 显式暴露属性和方法给父组件
defineExpose({
  count,
  increment,
  someMethod: () => {
    console.log("Called from parent");
  },
});
</script>

如果不使用 defineExpose，则子组件的内部状态对父组件是完全不可见的，这有助于更好地封装组件。

在标准组件选项中使用模板引用 ​

如果不使用 <script setup>，而是使用标准的组件选项 API 和 setup 函数，模板引用的使用方式略有不同：

vue
<script>
import { ref, onMounted } from "vue";

export default {
  setup() {
    const inputRef = ref(null);

    onMounted(() => {
      inputRef.value.focus();
    });

    return {
      inputRef,
    };
  },
};
</script>

<template>
  <input ref="inputRef" />
</template>

访问模板引用的时机 ​

重要的是要理解，模板引用只有在组件挂载后才能访问。在 setup 或 <script setup> 的顶层代码中，模板引用的值为 null，因为组件还没有被挂载：

vue
<script setup>
import { ref, onMounted } from "vue";

const inputRef = ref(null);

// 这里 inputRef.value 为 null
console.log(inputRef.value); // null

onMounted(() => {
  // 在挂载后，可以访问到 DOM 元素
  console.log(inputRef.value); // <input>
  inputRef.value.focus();
});
</script>

<template>
  <input ref="inputRef" />
</template>

因此，任何需要访问模板引用的代码都应该放在 onMounted 或其他生命周期钩子中。

使用 watchEffect 监听模板引用的变化 ​

我们可以使用 watchEffect 来监听模板引用的变化，并在引用更新时执行相应的操作：

vue
<script setup>
import { ref, watchEffect } from "vue";

const inputRef = ref(null);

watchEffect(
  () => {
    if (inputRef.value) {
      // 当 inputRef.value 不为 null 时执行
      inputRef.value.focus();
    }
  },
  { flush: "post" }
); // 确保在 DOM 更新后执行
</script>

<template>
  <input ref="inputRef" />
</template>

注意 { flush: 'post' } 选项，它确保 watchEffect 在 DOM 更新后执行，这样我们就能访问到更新后的 DOM 元素。

模板引用与异步组件 ​

当使用异步组件时，组件可能在初始渲染时不可用。在这种情况下，我们可以使用 watchEffect 或条件检查来确保在组件可用时才访问模板引用：

vue
<script setup>
import { ref, watchEffect } from "vue";
import { defineAsyncComponent } from "vue";

const AsyncComponent = defineAsyncComponent(() =>
  import("./AsyncComponent.vue")
);

const asyncCompRef = ref(null);

watchEffect(
  () => {
    if (asyncCompRef.value) {
      // 当异步组件加载完成并挂载后执行
      console.log("Async component is ready");
    }
  },
  { flush: "post" }
);
</script>

<template>
  <AsyncComponent ref="asyncCompRef" />
</template>

模板引用与组合式函数 ​

我们可以将模板引用的逻辑封装到组合式函数中，以便在多个组件中复用：

js
// useElementFocus.js
import { ref, onMounted } from "vue";

export function useElementFocus() {
  const elementRef = ref(null);

  onMounted(() => {
    if (elementRef.value) {
      elementRef.value.focus();
    }
  });

  return elementRef;
}

然后在组件中使用：

vue
<script setup>
import { useElementFocus } from "./useElementFocus";

const inputRef = useElementFocus();
</script>

<template>
  <input ref="inputRef" />
</template>

模板引用与 TypeScript ​

在 TypeScript 项目中，我们可以为模板引用添加类型标注：

vue
<script setup lang="ts">
import { ref, onMounted } from "vue";

// 为 DOM 元素添加类型
const inputRef = ref<HTMLInputElement | null>(null);

onMounted(() => {
  if (inputRef.value) {
    // TypeScript 现在知道这是一个 HTMLInputElement
    inputRef.value.focus();
  }
});

// 为组件添加类型
import MyComponent from "./MyComponent.vue";
const myComponentRef = ref<InstanceType<typeof MyComponent> | null>(null);

onMounted(() => {
  if (myComponentRef.value) {
    // 访问组件的公开方法
    myComponentRef.value.someMethod();
  }
});
</script>

<template>
  <input ref="inputRef" />
  <MyComponent ref="myComponentRef" />
</template>

最佳实践 ​

1. 谨慎使用模板引用 ​

模板引用允许我们直接操作 DOM，这可能会绕过 Vue 的响应式系统。因此，应该谨慎使用模板引用，只在必要时才使用它们，例如：

  • 管理焦点、选择或滚动位置
  • 触发命令式动画
  • 集成第三方 DOM 库

2. 在正确的生命周期阶段访问模板引用 ​

始终在 onMounted 或其他适当的生命周期钩子中访问模板引用，确保组件已经挂载：

vue
<script setup>
import { ref, onMounted } from "vue";

const chartRef = ref(null);

onMounted(() => {
  // 在这里初始化图表库
  if (chartRef.value) {
    initChart(chartRef.value);
  }
});
</script>

<template>
  <div ref="chartRef" class="chart-container"></div>
</template>

3. 使用函数型 ref 处理动态列表 ​

当处理动态列表时，使用函数型 ref 可以更灵活地管理元素引用：

vue
<script setup>
import { ref, onBeforeUpdate, onUpdated } from "vue";

const itemRefs = ref([]);

// 在更新前清空引用数组
onBeforeUpdate(() => {
  itemRefs.value = [];
});

function setItemRef(el) {
  if (el) {
    itemRefs.value.push(el);
  }
}

onUpdated(() => {
  // 在更新后处理引用
  console.log(itemRefs.value.length);
});
</script>

<template>
  <ul>
    <li v-for="(item, index) in items" :key="index" :ref="setItemRef">
      {{ item }}
    </li>
  </ul>
</template>

4. 明确暴露子组件的 API ​

在子组件中使用 defineExpose 明确暴露需要给父组件访问的属性和方法，避免暴露过多的内部实现细节：

vue
<script setup>
import { ref } from "vue";

const count = ref(0);
const internalState = ref("hidden"); // 内部状态，不暴露

function increment() {
  count.value++;
}

// 只暴露必要的 API
defineExpose({
  count,
  increment,
});
</script>

总结 ​

Vue 3 组合式 API 中的模板引用提供了一种简洁而强大的方式来访问底层 DOM 元素和子组件实例。通过将模板引用与响应式系统结合，Vue 3 使得直接操作 DOM 变得更加自然和直观。

虽然模板引用很有用，但应该谨慎使用，避免过度依赖直接 DOM 操作，而是尽可能地利用 Vue 的声明式渲染和响应式系统。在必要时使用模板引用，可以帮助我们构建更加复杂和交互丰富的应用程序。
