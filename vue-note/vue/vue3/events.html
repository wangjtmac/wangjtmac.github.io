自定义事件 ​

在 Vue 中，自定义事件是组件间通信的重要方式，特别是用于子组件向父组件传递数据和状态变化。本文档详细介绍 Vue 3 中自定义事件的使用方法、验证和最佳实践。

事件声明 ​

使用 <script setup> ​

在使用 <script setup> 的组件中，可以使用 defineEmits 宏来声明组件可以触发的事件：

vue
<script setup>
// 声明触发的事件
const emit = defineEmits(["change", "update", "delete"]);

// 触发事件
function handleChange() {
  emit("change", "新值");
}
</script>

不使用 <script setup> ​

在不使用 <script setup> 的组件中，可以通过 emits 选项来声明事件：

vue
<script>
export default {
  emits: ["change", "update", "delete"],
  setup(props, { emit }) {
    function handleChange() {
      emit("change", "新值");
    }

    return {
      handleChange,
    };
  },
};
</script>

选项式 API ​

在选项式 API 中，也可以通过 emits 选项来声明事件：

vue
<script>
export default {
  emits: ["change", "update", "delete"],
  methods: {
    handleChange() {
      this.$emit("change", "新值");
    },
  },
};
</script>

事件验证 ​

与 props 验证类似，事件也可以被验证。可以使用对象语法来定义事件，其中事件的值可以是一个验证函数，该函数接收传递给 $emit 调用的参数，并返回一个布尔值来表示事件参数是否有效。

基本验证 ​

vue
<script setup>
const emit = defineEmits({
  // 无验证
  click: null,

  // 验证 submit 事件
  submit: (payload) => {
    if (!payload) {
      console.warn("Missing submit event payload");
      return false;
    }
    if (!payload.email) {
      console.warn("Missing email in submit event payload");
      return false;
    }
    return true;
  },
});

function submitForm(email) {
  emit("submit", { email });
}
</script>

TypeScript 验证 ​

使用 TypeScript 时，可以使用类型注解来定义事件：

vue
<script setup lang="ts">
const emit = defineEmits<{
  (e: "change", id: number): void;
  (e: "update", value: string): void;
  (e: "delete", id: number, permanent: boolean): void;
}>();

// 使用方式
emit("change", 1);
emit("update", "new value");
emit("delete", 2, true);
</script>

监听事件 ​

父组件可以使用 v-on 或其缩写 @ 来监听子组件触发的事件：

vue
<!-- 父组件 -->
<template>
  <div>
    <ChildComponent
      @change="handleChange"
      @update="handleUpdate"
      @delete="handleDelete"
    />
  </div>
</template>

<script setup>
import { ref } from "vue";
import ChildComponent from "./ChildComponent.vue";

function handleChange(newValue) {
  console.log("Value changed:", newValue);
}

function handleUpdate(value) {
  console.log("Update received:", value);
}

function handleDelete(id, permanent) {
  console.log(`Deleting item ${id}, permanent: ${permanent}`);
}
</script>

事件参数 ​

事件可以传递多个参数：

vue
<script setup>
const emit = defineEmits(["delete", "update"]);

function deleteItem(id) {
  // 传递一个参数
  emit("delete", id);
}

function updateItem(id, newValue, timestamp) {
  // 传递多个参数
  emit("update", id, newValue, timestamp);
}
</script>

父组件接收参数：

vue
<template>
  <ChildComponent @delete="handleDelete" @update="handleUpdate" />
</template>

<script setup>
function handleDelete(id) {
  console.log(`删除 ID 为 ${id} 的项目`);
}

function handleUpdate(id, newValue, timestamp) {
  console.log(`更新项目 ${id}:`, newValue);
  console.log(`更新时间: ${timestamp}`);
}
</script>

v-model 与自定义事件 ​

v-model 是 Vue 中实现双向绑定的语法糖，它结合了属性绑定和事件处理。在 Vue 3 中，v-model 在组件上使用时，默认使用 modelValue 作为 prop，并使用 update:modelValue 作为事件。

基本用法 ​

vue
<!-- 子组件 -->
<template>
  <input
    :value="modelValue"
    @input="$emit('update:modelValue', $event.target.value)"
  />
</template>

<script setup>
defineProps(["modelValue"]);
defineEmits(["update:modelValue"]);
</script>

父组件使用：

vue
<template>
  <ChildComponent v-model="searchText" />
  <!-- 等同于 -->
  <ChildComponent
    :modelValue="searchText"
    @update:modelValue="(newValue) => (searchText = newValue)"
  />
</template>

<script setup>
import { ref } from "vue";
import ChildComponent from "./ChildComponent.vue";

const searchText = ref("");
</script>

多个 v-model 绑定 ​

Vue 3 允许在同一个组件上使用多个 v-model 绑定：

vue
<!-- 子组件 -->
<template>
  <div>
    <input
      :value="firstName"
      @input="$emit('update:firstName', $event.target.value)"
    />
    <input
      :value="lastName"
      @input="$emit('update:lastName', $event.target.value)"
    />
  </div>
</template>

<script setup>
defineProps({
  firstName: String,
  lastName: String,
});
defineEmits(["update:firstName", "update:lastName"]);
</script>

父组件：

vue
<template>
  <UserName v-model:first-name="firstName" v-model:last-name="lastName" />
</template>

<script setup>
import { ref } from "vue";
import UserName from "./UserName.vue";

const firstName = ref("张");
const lastName = ref("三");
</script>

v-model 修饰符 ​

可以通过 modelModifiers prop 在组件内部访问 v-model 的修饰符：

vue
<!-- 子组件 -->
<template>
  <input type="text" :value="modelValue" @input="emitValue" />
</template>

<script setup>
const props = defineProps({
  modelValue: String,
  modelModifiers: { default: () => ({}) },
});
const emit = defineEmits(["update:modelValue"]);

function emitValue(e) {
  let value = e.target.value;
  // 如果使用了 capitalize 修饰符
  if (props.modelModifiers.capitalize) {
    value = value.charAt(0).toUpperCase() + value.slice(1);
  }
  emit("update:modelValue", value);
}
</script>

父组件使用：

vue
<template>
  <MyInput v-model.capitalize="myText" />
</template>

对于带参数的 v-model，修饰符 prop 的名称将是 arg + "Modifiers"：

vue
<template>
  <MyComponent v-model:title.capitalize="myText" />
</template>

组件接收：

vue
<script setup>
const props = defineProps(["title", "titleModifiers"]);
defineEmits(["update:title"]);
console.log(props.titleModifiers); // { capitalize: true }
</script>

事件命名风格 ​

与组件和 prop 类似，事件名也提供了自动的大小写转换。如果在子组件中触发一个以 camelCase (驼峰式) 命名的事件，你可以在父组件中使用 kebab-case (短横线分隔式) 来监听：

vue
<!-- 子组件 -->
<script setup>
const emit = defineEmits(["updateData"]);

emit("updateData", {
  /* ... */
}); // 驼峰式命名
</script>

<!-- 父组件 -->
<ChildComponent @update-data="handleUpdate" />
<!-- 短横线分隔式命名 -->

但在模板中，推荐使用 kebab-case 事件名，这样更加符合 HTML 属性的命名风格。

事件冒泡和原生事件 ​

默认情况下，组件触发的自定义事件不会冒泡。要监听组件根元素的原生事件，可以使用 .native 修饰符（Vue 2）或在 Vue 3 中使用 v-on 绑定到组件的根元素。

在 Vue 3 中，.native 修饰符已被移除。相反，所有未在 emits 选项中声明的事件监听器现在都会作为原生事件监听器添加到组件的根元素中。

vue
<!-- 如果 focus 没有在子组件的 emits 中声明 -->
<MyComponent @focus="onFocus" />

若不希望事件监听器继承到组件的根元素，可以在 emits 中显式声明：

vue
<script setup>
// 这将防止 focus 事件被添加到组件的根元素
defineEmits(["focus"]);
</script>

事件总线（Event Bus） ​

在 Vue 3 中，官方推荐的通信方式是使用 props 和自定义事件进行父子组件通信，使用 provide/inject 进行深层级组件通信，以及使用状态管理库（如 Vuex 或 Pinia）进行复杂的组件间通信。

然而，在简单的场景中，你也可以使用第三方库创建事件总线，或使用 Vue 3 中的 mitt 库：

js
// eventBus.js
import mitt from "mitt";
export const emitter = mitt();

使用事件总线：

vue
<script setup>
import { onMounted, onUnmounted } from "vue";
import { emitter } from "./eventBus";

// 监听事件
onMounted(() => {
  emitter.on("custom-event", handleCustomEvent);
});

// 移除事件监听器
onUnmounted(() => {
  emitter.off("custom-event", handleCustomEvent);
});

function handleCustomEvent(data) {
  console.log("Received custom event:", data);
}

// 触发事件
function triggerEvent() {
  emitter.emit("custom-event", { message: "Hello from component!" });
}
</script>

最佳实践 ​

事件命名 ​

为事件选择有意义的名称，通常使用动词或动词短语：

js
// 好的事件名
"update";
"delete";
"submit-form";
"change-status";

// 避免的事件名
"event";
"callback";
"action";

明确事件的用途和参数 ​

在文档或注释中明确说明事件的用途和参数：

vue
<script setup>
/**
 * 定义组件可触发的事件
 * - change: 当选中项改变时触发，参数为新的选中项 ID
 * - select: 当用户点击某项时触发，参数为该项的数据和索引
 * - load: 当数据加载完成时触发，参数为加载的数据列表
 */
const emit = defineEmits(["change", "select", "load"]);
</script>

避免过度使用事件 ​

事件应该用于表示重要的状态变化或用户交互。避免为每个小变化都触发事件：

vue
<script setup>
// 不好的做法：过多的微小事件
const emit = defineEmits([
  "mouseEnter",
  "mouseLeave",
  "focusChange",
  "valueUpdate",
  "borderColorChange",
  // ...更多小事件
]);

// 好的做法：合并相关事件
const emit = defineEmits([
  "stateChange", // 包含状态变化的所有细节
  "interaction", // 包含用户交互的细节
]);

function updateState(newState) {
  // 发送完整的状态变化
  emit("stateChange", {
    value: newState.value,
    focused: newState.focused,
    hovered: newState.hovered,
  });
}
</script>

事件验证 ​

为关键事件添加验证函数，确保数据格式正确：

vue
<script setup>
const emit = defineEmits({
  submit: (payload) => {
    if (!payload) return false;
    if (!payload.email || !payload.password) return false;
    if (
      typeof payload.email !== "string" ||
      typeof payload.password !== "string"
    )
      return false;
    if (payload.email.trim() === "" || payload.password.trim() === "")
      return false;
    return true;
  },
});

function submitForm(formData) {
  emit("submit", {
    email: formData.email,
    password: formData.password,
  });
}
</script>

处理事件与更新状态分离 ​

在父组件中，最好将事件处理和状态更新分开：

vue
<template>
  <ChildComponent @update="handleUpdate" />
</template>

<script setup>
import { ref } from "vue";
import ChildComponent from "./ChildComponent.vue";

const data = ref(null);

// 好的做法：分离事件处理和状态更新
function handleUpdate(newData) {
  // 事件处理逻辑
  console.log("Received update:", newData);
  validateData(newData);

  // 状态更新
  updateState(newData);
}

function validateData(data) {
  // 验证数据
}

function updateState(newData) {
  // 更新状态
  data.value = newData;
}
</script>

总结 ​

自定义事件是 Vue 中实现组件通信的关键机制，特别是对于子组件向父组件传递信息的情况。本文档涵盖了以下内容：

  1. 事件声明：使用 defineEmits 或 emits 选项声明组件可触发的事件
  2. 事件验证：通过验证函数确保事件参数的正确性
  3. 事件监听：在父组件中使用 v-on 或 @ 监听子组件事件
  4. v-model：理解 v-model 在组件上的工作原理及其与事件的关系
  5. 最佳实践：事件命名、参数传递、验证和处理的最佳实践

通过合理使用自定义事件，可以建立清晰、可维护的组件间通信机制，使应用架构更加健壮和灵活。
