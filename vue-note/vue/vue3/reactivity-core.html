Vue3 响应式核心 ​

Vue 3 的响应式系统是其最强大的特性之一，它允许我们创建能够自动跟踪依赖并在数据变化时更新视图的响应式状态。本文将详细介绍 Vue 3 响应式系统的核心 API 和使用方法。

响应式 API 概览 ​

Vue 3 提供了一系列响应式 API，主要分为以下几类：

  1. 响应式状态创建：ref、reactive、readonly 等
  2. 响应式状态读取与修改：.value、解构、展开等
  3. 响应式工具函数：isRef、unref、toRef、toRefs 等
  4. 计算属性与监听：computed、watch、watchEffect 等

创建响应式状态 ​

ref ​

ref 函数用于创建一个包含单一值的响应式引用，它可以包含任何类型的值：

js
import { ref } from "vue";

// 创建一个包含基本类型的响应式引用
const count = ref(0);

// 访问和修改值需要使用 .value 属性
console.log(count.value); // 0
count.value++;
console.log(count.value); // 1

// 创建一个包含对象的响应式引用
const user = ref({
  name: "John",
  age: 30,
});

// 修改对象属性也需要使用 .value
user.value.age = 31;

在模板中使用 ref 时，不需要使用 .value，Vue 会自动解包：

vue
<template>
  <div>{{ count }}</div>
  <!-- 不需要 .value -->
</template>

<script setup>
import { ref } from "vue";

const count = ref(0);
</script>

reactive ​

reactive 函数用于创建一个响应式对象或数组：

js
import { reactive } from "vue";

const state = reactive({
  count: 0,
  user: {
    name: "John",
    age: 30,
  },
  items: ["apple", "banana"],
});

// 直接修改属性即可触发更新
state.count++;
state.user.age = 31;
state.items.push("orange");

reactive 与 ref 的区别：

  1. reactive 只能用于对象类型（对象、数组、Map、Set 等）
  2. reactive 创建的对象不需要使用 .value 访问
  3. reactive 是深层响应式的，会递归地将所有嵌套属性转换为响应式
  4. reactive 不能用于基本类型（字符串、数字、布尔值等）

readonly ​

readonly 函数用于创建一个只读的响应式对象，它接收一个响应式对象（ref 或 reactive）作为参数，并返回一个只读的代理：

js
import { ref, reactive, readonly } from "vue";

const count = ref(0);
const readonlyCount = readonly(count);

// 这会触发警告
readonlyCount.value++; // 警告: Set operation on key "value" failed: target is readonly.

const state = reactive({ count: 0 });
const readonlyState = readonly(state);

// 这会触发警告
readonlyState.count++; // 警告: Set operation on key "count" failed: target is readonly.

readonly 对象是深度只读的，所有嵌套属性也都是只读的。

响应式状态的使用 ​

解构响应式对象 ​

当我们解构一个 reactive 对象时，解构出的属性会失去响应性：

js
import { reactive } from "vue";

const state = reactive({
  count: 0,
  user: {
    name: "John",
  },
});

// 解构后，count 和 user 失去响应性
const { count, user } = state;

// 这些修改不会触发更新
count++;
user.name = "Jane";

为了保持响应性，我们可以使用 toRefs 函数：

js
import { reactive, toRefs } from "vue";

const state = reactive({
  count: 0,
  user: {
    name: "John",
  },
});

// 使用 toRefs 将响应式对象的所有属性转换为 ref
const { count, user } = toRefs(state);

// 现在这些修改会触发更新
count.value++;
user.value.name = "Jane";

响应式工具函数 ​

Vue 3 提供了一系列工具函数来处理响应式对象：

isRef ​

检查一个值是否为 ref 对象：

js
import { ref, isRef } from "vue";

const foo = ref(1);
const bar = 1;

console.log(isRef(foo)); // true
console.log(isRef(bar)); // false

unref ​

如果参数是 ref，则返回其 .value，否则返回参数本身：

js
import { ref, unref } from "vue";

const foo = ref(1);
const bar = 1;

console.log(unref(foo)); // 1
console.log(unref(bar)); // 1

toRef ​

为响应式对象的属性创建一个 ref：

js
import { reactive, toRef } from "vue";

const state = reactive({
  count: 0,
});

const countRef = toRef(state, "count");

// 修改 ref 会更新原始对象
countRef.value++;
console.log(state.count); // 1

// 修改原始对象也会更新 ref
state.count++;
console.log(countRef.value); // 2

toRefs ​

将响应式对象转换为普通对象，其中每个属性都是指向原始对象相应属性的 ref：

js
import { reactive, toRefs } from "vue";

const state = reactive({
  foo: 1,
  bar: 2,
});

const stateAsRefs = toRefs(state);

// stateAsRefs 的类型: { foo: Ref<number>, bar: Ref<number> }
stateAsRefs.foo.value++; // 修改 ref 会更新原始对象
console.log(state.foo); // 2

state.bar++; // 修改原始对象也会更新 ref
console.log(stateAsRefs.bar.value); // 3

计算属性与监听 ​

computed ​

computed 函数用于创建计算属性，它接收一个 getter 函数，并返回一个只读的响应式 ref 对象：

js
import { ref, computed } from "vue";

const count = ref(0);

// 创建一个计算属性
const doubleCount = computed(() => count.value * 2);

console.log(doubleCount.value); // 0

count.value++; // 修改依赖项
console.log(doubleCount.value); // 2

也可以创建可写的计算属性：

js
import { ref, computed } from "vue";

const firstName = ref("John");
const lastName = ref("Doe");

// 创建可写的计算属性
const fullName = computed({
  get: () => `${firstName.value} ${lastName.value}`,
  set: (newValue) => {
    [firstName.value, lastName.value] = newValue.split(" ");
  },
});

console.log(fullName.value); // "John Doe"

fullName.value = "Jane Smith"; // 触发 setter
console.log(firstName.value); // "Jane"
console.log(lastName.value); // "Smith"

watch ​

watch 函数用于监听一个或多个响应式数据源，并在数据源变化时执行回调函数：

js
import { ref, watch } from "vue";

const count = ref(0);

// 监听单个 ref
watch(count, (newValue, oldValue) => {
  console.log(`count changed from ${oldValue} to ${newValue}`);
});

count.value++; // 输出: count changed from 0 to 1

监听多个数据源：

js
import { ref, watch } from "vue";

const firstName = ref("John");
const lastName = ref("Doe");

// 监听多个数据源
watch(
  [firstName, lastName],
  ([newFirstName, newLastName], [oldFirstName, oldLastName]) => {
    console.log(
      `Name changed from ${oldFirstName} ${oldLastName} to ${newFirstName} ${newLastName}`
    );
  }
);

firstName.value = "Jane"; // 输出: Name changed from John Doe to Jane Doe

监听响应式对象的属性：

js
import { reactive, watch } from "vue";

const state = reactive({
  count: 0,
  user: {
    name: "John",
  },
});

// 使用 getter 函数监听嵌套属性
watch(
  () => state.user.name,
  (newName, oldName) => {
    console.log(`Name changed from ${oldName} to ${newName}`);
  }
);

state.user.name = "Jane"; // 输出: Name changed from John to Jane

watchEffect ​

watchEffect 函数会立即执行一个函数，并自动跟踪其依赖项，当依赖项变化时重新执行该函数：

js
import { ref, watchEffect } from "vue";

const count = ref(0);
const name = ref("John");

// watchEffect 会立即执行，并跟踪依赖项
const stop = watchEffect(() => {
  console.log(`Count: ${count.value}, Name: ${name.value}`);
});
// 输出: Count: 0, Name: John

count.value++; // 输出: Count: 1, Name: John
name.value = "Jane"; // 输出: Count: 1, Name: Jane

// 停止监听
stop();

// 这些修改不会触发回调
count.value++;
name.value = "Jack";

watchEffect 与 watch 的区别：

  1. watchEffect 会立即执行，而 watch 默认是惰性的
  2. watchEffect 自动跟踪依赖项，而 watch 需要明确指定监听的数据源
  3. watchEffect 不提供旧值，只能访问新值

响应式系统的注意事项 ​

响应式对象的限制 ​

  1. 对于 reactive 对象，Vue 不能跟踪属性的添加或删除，可以使用 Vue.set 或展开运算符创建新对象
  2. 当将 reactive 对象解构或展开时，会失去响应性
  3. ref 在模板中自动解包，但在 JavaScript 中需要使用 .value

性能优化 ​

  1. 避免在大型响应式对象中存储不需要响应式的数据
  2. 使用 shallowRef 和 shallowReactive 创建浅层响应式对象，减少深层转换的开销
  3. 使用 markRaw 标记不需要转换为响应式的对象
js
import { reactive, markRaw } from "vue";

const data = markRaw({
  // 这个对象不会被转换为响应式
  largeData: new Array(10000),
});

const state = reactive({
  data, // data 保持非响应式
});

总结 ​

Vue 3 的响应式系统提供了强大而灵活的 API，使我们能够创建高效的响应式应用。通过合理使用 ref、reactive、computed、watch 等 API，我们可以构建出复杂的响应式逻辑，同时保持代码的清晰和可维护性。

在实际开发中，建议遵循以下原则：

  1. 对于基本类型的响应式状态，使用 ref
  2. 对于对象类型的响应式状态，使用 reactive
  3. 使用 computed 创建派生状态
  4. 使用 watch 或 watchEffect 处理副作用
  5. 使用组合式函数封装和复用响应式逻辑
