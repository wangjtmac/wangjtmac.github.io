Provide / Inject ​

依赖注入（Provide / Inject）是 Vue 中用于跨多层组件传递数据的方式，特别适用于深层组件嵌套的情况。本文档详细介绍 Vue 3 中依赖注入的使用方法和最佳实践。

基本概念 ​

在大型应用中，组件树可能非常深，若要从顶层组件向深层子组件传递数据，仅靠 props 逐级传递会变得非常繁琐。这种情况下，我们可以使用 provide 和 inject 这对 API 来实现依赖注入。

  • provide：允许一个祖先组件向其所有后代组件注入一个依赖，不管组件层次有多深
  • inject：允许一个组件接收其祖先组件提供的依赖

基础用法 ​

使用组合式 API ​

在组合式 API 中，使用 provide 和 inject 函数来实现依赖注入：

vue
<!-- 提供方：祖先组件 -->
<script setup>
import { provide, ref } from "vue";

// 提供静态值
provide("message", "这是一条消息");

// 提供响应式数据
const count = ref(0);
provide("counter", count);

// 提供方法
const incrementCount = () => {
  count.value++;
};
provide("increment", incrementCount);
</script>

在后代组件中注入并使用：

vue
<!-- 注入方：后代组件 -->
<script setup>
import { inject } from "vue";

// 注入值
const message = inject("message");

// 注入响应式数据
const counter = inject("counter");

// 注入方法
const increment = inject("increment");
</script>

<template>
  <div>
    <p>消息: {{ message }}</p>
    <p>计数: {{ counter }}</p>
    <button @click="increment">增加</button>
  </div>
</template>

使用选项式 API ​

在选项式 API 中，可以使用 provide 和 inject 选项：

js
// 提供方：祖先组件
export default {
  data() {
    return {
      message: "这是一条消息",
      count: 0,
    };
  },
  provide() {
    return {
      message: this.message, // 非响应式
      count: this.count, // 非响应式
      // 若需要响应式，需要提供计算属性或使用组合式 API
      counterObj: () => this.count, // 方法形式保持响应性
      increment: this.incrementCount, // 提供方法
    };
  },
  methods: {
    incrementCount() {
      this.count++;
    },
  },
};

在后代组件中注入：

js
// 注入方：后代组件
export default {
  inject: ["message", "count", "counterObj", "increment"],
  mounted() {
    console.log(this.message); // '这是一条消息'
    console.log(this.count); // 0（非响应式）
    console.log(this.counterObj()); // 0（通过调用方法获取最新值）
    this.increment(); // 调用注入的方法
  },
};

处理响应性 ​

组合式 API 中的响应性传递 ​

在组合式 API 中，提供的响应式状态会自动保持响应性：

vue
<script setup>
import { ref, provide } from "vue";

const count = ref(0);
provide("count", count); // 直接提供 ref

const increment = () => {
  count.value++;
};
provide("increment", increment);
</script>

子组件注入后会保持响应性：

vue
<script setup>
import { inject } from "vue";

const count = inject("count"); // 保持响应性
const increment = inject("increment");
</script>

<template>
  <div>
    <p>当前计数: {{ count }}</p>
    <button @click="increment">增加</button>
  </div>
</template>

选项式 API 中的响应性传递 ​

在选项式 API 中，需要使用计算属性来保持响应性：

js
// 提供方：祖先组件
import { computed } from "vue";

export default {
  data() {
    return {
      count: 0,
    };
  },
  provide() {
    return {
      // 使用计算属性保持响应性
      count: computed(() => this.count),
      increment: () => this.count++,
    };
  },
};

注入方需要访问 .value 属性（因为注入的是 computed 值）：

js
// 注入方：后代组件
export default {
  inject: ["count", "increment"],
  mounted() {
    console.log(this.count.value); // 访问 .value
  },
};

注入的默认值 ​

当提供的值可能不存在时，可以为 inject 指定默认值：

vue
<script setup>
import { inject } from "vue";

// 使用第二个参数作为默认值
const message = inject("message", "默认消息");

// 使用工厂函数作为默认值（用于创建非原始值或需要计算的值）
const theme = inject("theme", () => ({ primary: "#3a5ccc" }));
</script>

应用层 Provide ​

除了在组件中提供依赖，还可以在整个应用层面提供依赖，这样应用内的所有组件都可以注入这些依赖：

js
// main.js
import { createApp } from "vue";
import App from "./App.vue";
import "./index.css";

const app = createApp(App);

// 提供应用层级的依赖
app.provide("appName", "我的 Vue 应用");
app.provide("apiBaseUrl", "https://api.example.com");

app.mount("#app");

所有组件都可以注入这些值：

vue
<script setup>
import { inject } from "vue";

const appName = inject("appName");
const apiBaseUrl = inject("apiBaseUrl");
</script>

<template>
  <div>
    <p>应用名称: {{ appName }}</p>
    <p>API 基础 URL: {{ apiBaseUrl }}</p>
  </div>
</template>

使用 Symbol 作为 Key ​

为了避免潜在的命名冲突，可以使用 Symbol 作为注入名的 key：

js
// keys.js
export const countKey = Symbol("count");
export const messageKey = Symbol("message");

提供方：

vue
<script setup>
import { provide, ref } from "vue";
import { countKey, messageKey } from "./keys";

const count = ref(0);
provide(countKey, count);
provide(messageKey, "这是一条消息");
</script>

注入方：

vue
<script setup>
import { inject } from "vue";
import { countKey, messageKey } from "./keys";

const count = inject(countKey);
const message = inject(messageKey);
</script>

只读的注入值 ​

有时我们希望提供的值可以被后代组件使用，但不希望它们被修改。这时可以使用 readonly 函数来包装提供的值：

vue
<script setup>
import { ref, provide, readonly } from "vue";

const count = ref(0);
// 提供一个只读版本
provide("count", readonly(count));

// 修改应该由提供方控制
function increment() {
  count.value++;
}
provide("increment", increment);
</script>

修改注入值的最佳实践 ​

虽然可以在注入值的组件中直接修改注入的状态，但这样做会使数据流向变得难以理解和调试。更好的做法是在提供方组件中声明并提供修改数据的方法：

vue
<!-- 提供方组件 -->
<script setup>
import { provide, ref } from "vue";

const location = ref("北京");

function updateLocation(newLocation) {
  location.value = newLocation;
}

provide("location", {
  location,
  updateLocation,
});
</script>

注入方组件：

vue
<!-- 注入方组件 -->
<script setup>
import { inject } from "vue";

const { location, updateLocation } = inject("location");

function changeLocation() {
  updateLocation("上海"); // 通过提供的方法修改状态
}
</script>

与组合式函数结合 ​

Provide/Inject 可以很好地与组合式函数结合，创建可复用的逻辑：

js
// useTheme.js
import { ref, provide, inject } from "vue";

const themeSymbol = Symbol();

export function provideTheme() {
  const theme = ref("light");

  function toggleTheme() {
    theme.value = theme.value === "light" ? "dark" : "light";
  }

  provide(themeSymbol, {
    theme,
    toggleTheme,
  });
}

export function useTheme() {
  const theme = inject(themeSymbol);
  if (!theme) {
    throw new Error("useTheme() must be used after provideTheme()");
  }

  return theme;
}

使用这些组合式函数：

vue
<!-- 根组件 -->
<script setup>
import { provideTheme } from "./composables/useTheme";

// 提供主题
provideTheme();
</script>

<!-- 子组件 -->
<script setup>
import { useTheme } from "./composables/useTheme";

// 使用主题
const { theme, toggleTheme } = useTheme();
</script>

<template>
  <div :class="theme">
    <p>当前主题: {{ theme }}</p>
    <button @click="toggleTheme">切换主题</button>
  </div>
</template>

使用场景 ​

1. 主题配置 ​

在应用中实现全局的主题配置：

vue
<!-- ThemeProvider.vue -->
<script setup>
import { ref, provide } from "vue";

const theme = ref({
  primaryColor: "#3498db",
  secondaryColor: "#2ecc71",
  textColor: "#333",
  backgroundColor: "#fff",
});

// 提供切换深色/浅色模式的功能
function toggleDarkMode() {
  if (theme.value.backgroundColor === "#fff") {
    theme.value = {
      primaryColor: "#3498db",
      secondaryColor: "#2ecc71",
      textColor: "#f0f0f0",
      backgroundColor: "#222",
    };
  } else {
    theme.value = {
      primaryColor: "#3498db",
      secondaryColor: "#2ecc71",
      textColor: "#333",
      backgroundColor: "#fff",
    };
  }
}

provide("theme", {
  theme,
  toggleDarkMode,
});
</script>

<template>
  <slot></slot>
</template>

在任意深度的后代组件中使用主题：

vue
<script setup>
import { inject } from "vue";

const { theme, toggleDarkMode } = inject("theme");
</script>

<template>
  <div
    :style="{
      color: theme.textColor,
      backgroundColor: theme.backgroundColor,
    }"
  >
    <h2 :style="{ color: theme.primaryColor }">标题</h2>
    <p>一些文本内容</p>
    <button @click="toggleDarkMode">切换深色模式</button>
  </div>
</template>

2. 全局状态管理 ​

对于中小型应用，可以使用 provide/inject 代替 Vuex 或 Pinia 进行简单的全局状态管理：

vue
<!-- StateProvider.vue -->
<script setup>
import { reactive, provide } from "vue";

const state = reactive({
  user: null,
  isLoggedIn: false,
  notifications: [],
});

// 提供状态修改方法
function login(userData) {
  state.user = userData;
  state.isLoggedIn = true;
}

function logout() {
  state.user = null;
  state.isLoggedIn = false;
}

function addNotification(notification) {
  state.notifications.push(notification);
}

provide("globalState", {
  state,
  login,
  logout,
  addNotification,
});
</script>

<template>
  <slot></slot>
</template>

在应用中使用：

vue
<script setup>
import { inject } from "vue";

const { state, login, logout } = inject("globalState");
</script>

<template>
  <div>
    <p v-if="state.isLoggedIn">欢迎, {{ state.user.name }}</p>
    <button v-if="state.isLoggedIn" @click="logout">登出</button>
    <button v-else @click="() => login({ id: 1, name: '张三' })">登录</button>
  </div>
</template>

3. 国际化 (i18n) ​

实现简单的国际化功能：

vue
<!-- I18nProvider.vue -->
<script setup>
import { ref, provide } from "vue";

const locales = {
  zh: {
    greeting: "你好",
    welcome: "欢迎来到我们的网站",
    buttonText: "点击这里",
  },
  en: {
    greeting: "Hello",
    welcome: "Welcome to our website",
    buttonText: "Click here",
  },
};

const currentLocale = ref("zh");
const messages = ref(locales[currentLocale.value]);

function changeLocale(locale) {
  if (locales[locale]) {
    currentLocale.value = locale;
    messages.value = locales[locale];
  }
}

provide("i18n", {
  locale: currentLocale,
  messages,
  changeLocale,
});
</script>

<template>
  <slot></slot>
</template>

在组件中使用：

vue
<script setup>
import { inject } from "vue";

const { locale, messages, changeLocale } = inject("i18n");
</script>

<template>
  <div>
    <h1>{{ messages.greeting }}</h1>
    <p>{{ messages.welcome }}</p>
    <button>{{ messages.buttonText }}</button>

    <div>
      <button @click="() => changeLocale('zh')">中文</button>
      <button @click="() => changeLocale('en')">English</button>
    </div>
  </div>
</template>

最佳实践 ​

1. 何时使用 provide/inject ​

  • 适合使用的场景：

    • 需要在多级嵌套组件间传递数据
    • 组件库开发，提供主题、配置等
    • 插件开发，提供全局服务和状态
    • 简单的全局状态管理

  • 不适合使用的场景：

    • 父子组件直接通信（使用 props 和 emit 更清晰）
    • 复杂的全局状态管理（考虑使用 Pinia 或 Vuex）
    • 当数据流向需要非常清晰可追踪时

2. 使用类型系统增强依赖注入 ​

使用 TypeScript 时，可以为 provide 和 inject 提供类型支持：

ts
// types.ts
import { InjectionKey } from "vue";

export interface UserInfo {
  id: number;
  name: string;
  email: string;
}

export interface UserState {
  user: UserInfo | null;
  loading: boolean;
  error: Error | null;
}

export interface UserActions {
  fetchUser: (id: number) => Promise<void>;
  updateUser: (user: UserInfo) => Promise<void>;
  clearUser: () => void;
}

export interface UserStore {
  state: UserState;
  actions: UserActions;
}

// 创建 Symbol 注入键
export const UserKey: InjectionKey<UserStore> = Symbol("user");

提供方：

vue
<script setup lang="ts">
import { provide, reactive } from "vue";
import { UserKey, UserInfo, UserState, UserActions, UserStore } from "./types";

const state = reactive<UserState>({
  user: null,
  loading: false,
  error: null,
});

const actions: UserActions = {
  async fetchUser(id: number) {
    state.loading = true;
    try {
      // 模拟 API 调用
      const response = await fetch(`/api/users/${id}`);
      state.user = await response.json();
    } catch (error) {
      state.error = error as Error;
    } finally {
      state.loading = false;
    }
  },
  async updateUser(user: UserInfo) {
    // 实现更新逻辑
  },
  clearUser() {
    state.user = null;
  },
};

const userStore: UserStore = {
  state,
  actions,
};

provide(UserKey, userStore);
</script>

注入方：

vue
<script setup lang="ts">
import { inject } from "vue";
import { UserKey } from "./types";

// TypeScript 会自动推断类型
const userStore = inject(UserKey);

// 如果可能为 undefined，需要提供默认值或进行检查
if (!userStore) {
  throw new Error("UserStore was not provided");
}

const { state, actions } = userStore;

// 使用注入的值
function loadUser(id: number) {
  actions.fetchUser(id);
}
</script>

3. 组织复杂的 provide/inject 结构 ​

对于大型应用，可以创建专门的 provider 组件来组织依赖：

vue
<!-- AppProviders.vue -->
<script setup>
import ThemeProvider from "./providers/ThemeProvider.vue";
import I18nProvider from "./providers/I18nProvider.vue";
import AuthProvider from "./providers/AuthProvider.vue";
import ApiProvider from "./providers/ApiProvider.vue";
</script>

<template>
  <!-- 嵌套多个 provider，从外到内 -->
  <ApiProvider>
    <AuthProvider>
      <ThemeProvider>
        <I18nProvider>
          <slot></slot>
        </I18nProvider>
      </ThemeProvider>
    </AuthProvider>
  </ApiProvider>
</template>

然后在应用入口中使用：

vue
<!-- App.vue -->
<script setup>
import AppProviders from "./providers/AppProviders.vue";
import AppContent from "./AppContent.vue";
</script>

<template>
  <AppProviders>
    <AppContent />
  </AppProviders>
</template>

4. 避免直接修改注入的状态 ​

始终通过提供方提供的方法来修改状态，这样可以保持状态变化的可追踪性：

vue
<!-- 不好的做法 -->
<script setup>
import { inject } from "vue";

const count = inject("count");
// 直接修改注入的状态
function increment() {
  count.value++; // 避免这样做
}
</script>

<!-- 好的做法 -->
<script setup>
import { inject } from "vue";

const { count, increment } = inject("counter");
// 使用提供的方法修改状态
</script>

5. 设计清晰的 API 界面 ​

设计提供的依赖时，考虑如何创建清晰的 API 界面：

vue
<script setup>
import { reactive, provide } from "vue";

// 定义状态
const userState = reactive({
  data: null,
  loading: false,
  error: null,
});

// 定义 API
const userApi = {
  // 查询方法
  getUser: () => userState.data,
  isLoading: () => userState.loading,
  getError: () => userState.error,

  // 动作方法
  fetchUser: async (id) => {
    userState.loading = true;
    try {
      // 实现逻辑...
    } catch (error) {
      userState.error = error;
    } finally {
      userState.loading = false;
    }
  },
  clearUser: () => {
    userState.data = null;
  },
};

// 提供清晰分离的 API
provide("userApi", userApi);
</script>

总结 ​

Provide / Inject 是 Vue 中实现跨组件数据传递的强大机制，特别适合处理组件深层嵌套的情况。主要内容包括：

  1. 基本概念：provide 在祖先组件中提供数据，inject 在后代组件中接收数据
  2. 响应性传递：Vue 3 中可以直接传递响应式状态，保持响应性
  3. 默认值和 Symbol：提供默认值和使用 Symbol 作为键来避免冲突
  4. 只读与修改模式：通过 readonly 提供只读数据，通过方法控制修改
  5. 与组合式 API 结合：创建功能强大的可复用逻辑
  6. 使用场景：主题配置、简单状态管理、国际化等
  7. 最佳实践：合理设计 API、避免直接修改、结合类型系统等

当正确使用时，Provide / Inject 可以大大简化组件间的数据传递，使代码更清晰、更易于维护。
