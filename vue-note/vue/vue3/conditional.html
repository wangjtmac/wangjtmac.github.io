条件渲染 ​

在 Vue 中，我们可以使用 v-if 指令和相关指令来控制元素的显示与隐藏。Vue 提供了多种方式来实现条件渲染，以满足不同的场景需求。

v-if 指令 ​

v-if 指令用于条件性地渲染一块内容。这块内容只会在指令的表达式返回真值时被渲染。

vue
<template>
  <div>
    <h1 v-if="awesome">Vue 很棒！</h1>
  </div>
</template>

<script setup>
import { ref } from "vue";

const awesome = ref(true);
</script>

v-else ​

你可以使用 v-else 指令来表示 v-if 的"else 块"：

vue
<template>
  <div>
    <h1 v-if="awesome">Vue 很棒！</h1>
    <h1 v-else>哦，不是很棒</h1>
  </div>
</template>

<script setup>
import { ref } from "vue";

const awesome = ref(false);
</script>

v-else 元素必须紧跟在带 v-if 或者 v-else-if 的元素之后，否则它将不会被识别。

v-else-if ​

v-else-if 提供的是相应于 v-if 的"else if 块"，可以连续使用：

vue
<template>
  <div>
    <h1 v-if="type === 'A'">A</h1>
    <h1 v-else-if="type === 'B'">B</h1>
    <h1 v-else-if="type === 'C'">C</h1>
    <h1 v-else>非 A/B/C</h1>
  </div>
</template>

<script setup>
import { ref } from "vue";

const type = ref("B");
</script>

和 v-else 类似，v-else-if 也必须紧跟在带 v-if 或者 v-else-if 的元素之后。

在 <template> 元素上使用 v-if ​

因为 v-if 是一个指令，所以必须将它附加到一个元素上。但如果想切换多个元素呢？此时可以把一个 <template> 元素当做不可见的包裹元素，并在上面使用 v-if。最终的渲染结果将不包含 <template> 元素。

vue
<template>
  <div>
    <template v-if="showUserInfo">
      <h1>用户信息</h1>
      <p>姓名：{{ user.name }}</p>
      <p>年龄：{{ user.age }}</p>
      <p>地址：{{ user.address }}</p>
    </template>
  </div>
</template>

<script setup>
import { ref, reactive } from "vue";

const showUserInfo = ref(true);
const user = reactive({
  name: "张三",
  age: 30,
  address: "北京市朝阳区",
});
</script>

v-show ​

另一个用于条件性展示元素的指令是 v-show。用法大致一样：

vue
<template>
  <div>
    <h1 v-show="showTitle">标题</h1>
  </div>
</template>

<script setup>
import { ref } from "vue";

const showTitle = ref(true);
</script>

不同的是带有 v-show 的元素始终会被渲染并保留在 DOM 中。v-show 只是简单地切换元素的 CSS 属性 display。

v-if vs v-show ​

v-if 是"真正"的条件渲染，因为它会确保在切换过程中，条件块内的事件监听器和子组件适当地被销毁和重建。

v-if 也是惰性的：如果在初始渲染时条件为假，则什么也不做——直到条件第一次变为真时，才会开始渲染条件块。

相比之下，v-show 就简单得多——不管初始条件是什么，元素总是会被渲染，并且只是简单地基于 CSS 进行切换。

一般来说，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。

性能对比 ​

特性v-ifv-show
初始渲染开销低（条件为假时不渲染）高（总是渲染）
切换开销高（销毁/重建组件和事件监听器）低（只切换 CSS display 属性）
适用场景条件很少改变的情况频繁切换的情况

v-if 与 v-for 一起使用 ​

WARNING

同时使用 v-if 和 v-for 是不推荐的，因为这样二者的优先级不明显。

当 v-if 与 v-for 一起使用时，v-if 会先执行。

vue
<template>
  <div>
    <!-- 这样做会遍历整个列表，然后检查 isActive 属性 -->
    <li v-for="user in users" v-if="user.isActive" :key="user.id">
      {{ user.name }}
    </li>
  </div>
</template>

这种写法会遍历整个列表，然后检查每个用户的 isActive 属性。

更好的做法是使用计算属性先过滤数据：

vue
<template>
  <div>
    <li v-for="user in activeUsers" :key="user.id">
      {{ user.name }}
    </li>
  </div>
</template>

<script setup>
import { ref, computed } from "vue";

const users = ref([
  { id: 1, name: "张三", isActive: true },
  { id: 2, name: "李四", isActive: false },
  { id: 3, name: "王五", isActive: true },
]);

const activeUsers = computed(() => {
  return users.value.filter((user) => user.isActive);
});
</script>

这种方式有以下优点：

  1. 过滤后的列表只会在 users 数组发生相关变化时才被重新运算，过滤更高效
  2. 渲染时只遍历活跃用户，渲染更高效
  3. 逻辑更清晰，易于理解和维护

如果你想要有条件地跳过循环的执行，可以将 v-if 置于外层元素 (或 <template>) 上：

vue
<template>
  <div>
    <template v-if="hasActiveUsers">
      <li v-for="user in users" v-if="user.isActive" :key="user.id">
        {{ user.name }}
      </li>
    </template>
    <p v-else>没有活跃用户</p>
  </div>
</template>

<script setup>
import { ref, computed } from "vue";

const users = ref([
  { id: 1, name: "张三", isActive: true },
  { id: 2, name: "李四", isActive: false },
  { id: 3, name: "王五", isActive: true },
]);

const hasActiveUsers = computed(() => {
  return users.value.some((user) => user.isActive);
});
</script>

条件渲染的最佳实践 ​

使用 key 管理可复用的元素 ​

Vue 会尽可能高效地渲染元素，通常会复用已有元素而不是从头开始渲染。这么做除了使 Vue 变得非常快之外，还有一些好处。例如，如果你允许用户在不同的登录方式之间切换：

vue
<template>
  <div>
    <template v-if="loginType === 'username'">
      <label>用户名</label>
      <input placeholder="输入用户名" />
    </template>
    <template v-else>
      <label>邮箱</label>
      <input placeholder="输入邮箱" />
    </template>
    <button @click="toggleLoginType">切换登录方式</button>
  </div>
</template>

<script setup>
import { ref } from "vue";

const loginType = ref("username");

function toggleLoginType() {
  loginType.value = loginType.value === "username" ? "email" : "username";
}
</script>

在上面的代码中，切换 loginType 不会清除用户已经输入的内容。因为两个模板使用了相同的元素，<input> 不会被替换掉——仅仅是替换了它的 placeholder。

这样也不总是符合实际需求，所以 Vue 提供了一种方式来表达"这两个元素是完全独立的，不要复用它们"。只需添加一个具有唯一值的 key 属性即可：

vue
<template>
  <div>
    <template v-if="loginType === 'username'">
      <label>用户名</label>
      <input placeholder="输入用户名" key="username-input" />
    </template>
    <template v-else>
      <label>邮箱</label>
      <input placeholder="输入邮箱" key="email-input" />
    </template>
    <button @click="toggleLoginType">切换登录方式</button>
  </div>
</template>

现在，每次切换时，输入框都将被重新渲染。

使用计算属性简化条件逻辑 ​

当条件逻辑变得复杂时，可以使用计算属性来简化模板：

vue
<template>
  <div>
    <div :class="statusClass">{{ statusText }}</div>
  </div>
</template>

<script setup>
import { ref, computed } from "vue";

const status = ref("pending"); // 'pending', 'success', 'error'

const statusClass = computed(() => {
  return {
    "status-pending": status.value === "pending",
    "status-success": status.value === "success",
    "status-error": status.value === "error",
  };
});

const statusText = computed(() => {
  switch (status.value) {
    case "pending":
      return "处理中...";
    case "success":
      return "操作成功！";
    case "error":
      return "操作失败！";
    default:
      return "";
  }
});
</script>

<style>
.status-pending {
  color: blue;
}
.status-success {
  color: green;
}
.status-error {
  color: red;
}
</style>

使用组件封装条件逻辑 ​

对于复杂的条件渲染逻辑，可以考虑封装成组件：

vue
<!-- StatusMessage.vue -->
<template>
  <div :class="statusClass">{{ statusText }}</div>
</template>

<script setup>
import { computed } from "vue";

const props = defineProps({
  status: {
    type: String,
    required: true,
    validator: (value) => ["pending", "success", "error"].includes(value),
  },
});

const statusClass = computed(() => `status-${props.status}`);

const statusText = computed(() => {
  switch (props.status) {
    case "pending":
      return "处理中...";
    case "success":
      return "操作成功！";
    case "error":
      return "操作失败！";
    default:
      return "";
  }
});
</script>

<style scoped>
.status-pending {
  color: blue;
}
.status-success {
  color: green;
}
.status-error {
  color: red;
}
</style>

使用组件：

vue
<template>
  <div>
    <StatusMessage :status="currentStatus" />
    <button @click="changeStatus">更改状态</button>
  </div>
</template>

<script setup>
import { ref } from "vue";
import StatusMessage from "./StatusMessage.vue";

const currentStatus = ref("pending");

function changeStatus() {
  const statuses = ["pending", "success", "error"];
  const currentIndex = statuses.indexOf(currentStatus.value);
  const nextIndex = (currentIndex + 1) % statuses.length;
  currentStatus.value = statuses[nextIndex];
}
</script>

总结 ​

Vue 提供了多种条件渲染的方式：

  1. v-if / v-else / v-else-if：

    • 真正的条件渲染，条件为假时不会渲染元素
    • 适用于条件很少改变的情况
    • 可以在 <template> 上使用，包裹多个元素

  2. v-show：

    • 基于 CSS 的条件显示，元素总是会被渲染
    • 只是切换元素的 display 属性
    • 适用于需要频繁切换的情况

  3. 最佳实践：

    • 使用 key 属性管理可复用的元素
    • 避免同时使用 v-if 和 v-for
    • 使用计算属性简化条件逻辑
    • 考虑将复杂的条件渲染逻辑封装为组件 </rewritten_file>
