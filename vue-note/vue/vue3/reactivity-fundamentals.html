响应式基础 ​

Vue3 的响应式系统是其核心特性之一，它允许我们声明式地处理状态变化。Vue3 使用 ES6 的 Proxy 重写了响应式系统，提供了更强大和灵活的响应式能力。

声明响应式状态 ​

在 Vue3 中，我们可以使用reactive和ref两个 API 来创建响应式状态。

reactive ​

reactive函数接收一个普通对象，返回一个响应式的代理对象（Proxy）：

js
import { reactive } from "vue";

const state = reactive({
  count: 0,
  name: "Vue",
});

// 修改状态
state.count++;

reactive转换是"深层"的——它会影响所有嵌套属性。当我们访问或修改响应式对象的属性时，Vue 能够跟踪依赖并在属性被修改时触发更新。

reactive 的局限性 ​

  1. 只对对象类型有效：对原始类型（如 String、Number、Boolean）无效。
  2. 不能解构：解构响应式对象会使其失去响应性。
  3. 替换整个对象：如果你将一个响应式对象替换为一个新对象，引用将会改变，旧对象的响应性将会丢失。
js
// 这些操作会使响应性丢失
let { count } = state; // count不再是响应式的
state = { count: 2 }; // 原始引用被替换，响应性丢失

ref ​

为了解决reactive的局限性，Vue3 提供了ref函数，它可以将任何值（包括原始类型）包装为响应式对象：

js
import { ref } from "vue";

const count = ref(0);
const message = ref("Hello");
const isActive = ref(true);
const array = ref([1, 2, 3]);
const object = ref({ name: "Vue" });

// 访问或修改ref的值需要使用.value
console.log(count.value); // 0
count.value++;
console.log(count.value); // 1

ref创建的是一个包含.value属性的对象，我们需要通过.value来访问或修改其值。

ref 的自动解包 ​

在模板中使用 ref 时，不需要使用.value，Vue 会自动解包：

html
<template>
  <div>
    <p>Count: {{ count }}</p>
    <button @click="count++">Increment</button>
  </div>
</template>

<script setup>
  import { ref } from "vue";

  const count = ref(0);
</script>

当 ref 作为响应式对象的属性被访问或修改时，也会自动解包：

js
const state = reactive({
  count: ref(0),
});

// 不需要使用.value
console.log(state.count); // 0
state.count++; // 等同于state.count.value++

但是，当 ref 从数组或普通对象中被访问时，不会自动解包：

js
const array = [ref(0)];
// 这里需要.value
console.log(array[0].value);

const object = { foo: ref(0) };
// 这里需要.value
console.log(object.foo.value);

响应式状态解构 ​

使用 toRefs 和 toRef ​

如果我们想要解构响应式对象，同时保持其响应性，可以使用toRefs函数：

js
import { reactive, toRefs } from "vue";

const state = reactive({
  count: 0,
  name: "Vue",
});

// 将响应式对象的所有属性转换为ref
const { count, name } = toRefs(state);

// count和name现在是ref，并且与原始对象保持同步
console.log(count.value); // 0
count.value++;
console.log(state.count); // 1

如果只需要一个属性的 ref，可以使用toRef：

js
import { reactive, toRef } from "vue";

const state = reactive({
  count: 0,
  name: "Vue",
});

// 只将count属性转换为ref
const count = toRef(state, "count");

// count是ref，并且与原始对象保持同步
console.log(count.value); // 0
count.value++;
console.log(state.count); // 1

使用结构的替代方案 ​

除了使用toRefs和toRef，我们还可以通过以下方式处理响应式状态：

  1. 直接在模板中使用：在模板中可以直接使用响应式对象的属性，无需解构。
html
<template>
  <div>
    <p>Count: {{ state.count }}</p>
    <p>Name: {{ state.name }}</p>
  </div>
</template>

<script setup>
  import { reactive } from "vue";

  const state = reactive({
    count: 0,
    name: "Vue",
  });
</script>
  1. 使用计算属性：通过计算属性返回响应式状态的值。
js
import { reactive, computed } from "vue";

const state = reactive({
  firstName: "John",
  lastName: "Doe",
});

// 使用计算属性获取和设置响应式状态
const fullName = computed({
  get: () => `${state.firstName} ${state.lastName}`,
  set: (value) => {
    const names = value.split(" ");
    state.firstName = names[0];
    state.lastName = names[1];
  },
});

响应式工具函数 ​

Vue3 提供了一系列响应式工具函数，帮助我们更好地处理响应式状态。

isRef ​

检查一个值是否为 ref 对象：

js
import { ref, isRef } from "vue";

const count = ref(0);
console.log(isRef(count)); // true
console.log(isRef(0)); // false

unref ​

如果参数是 ref，则返回其.value，否则返回参数本身：

js
import { ref, unref } from "vue";

const count = ref(0);
console.log(unref(count)); // 0
console.log(unref(1)); // 1

isProxy ​

检查一个对象是否是由reactive或readonly创建的代理：

js
import { reactive, isProxy } from "vue";

const state = reactive({});
console.log(isProxy(state)); // true

isReactive ​

检查一个对象是否是由reactive创建的响应式代理：

js
import { reactive, isReactive } from "vue";

const state = reactive({});
console.log(isReactive(state)); // true

isReadonly ​

检查一个对象是否是由readonly创建的只读代理：

js
import { readonly, isReadonly } from "vue";

const state = readonly({});
console.log(isReadonly(state)); // true

响应式状态的最佳实践 ​

何时使用 ref vs. reactive ​

  • 使用 ref：

    • 当需要响应式的原始类型值（string, number, boolean）
    • 当需要将响应式状态传递给函数或从函数返回
    • 当需要保持对响应式状态的引用，即使它被重新赋值

  • 使用 reactive：

    • 当需要一个响应式对象，且不需要解构或传递给函数
    • 当需要嵌套的响应式对象结构
    • 当性能是关键考虑因素（reactive 比多个 ref 稍微高效）

避免响应式丢失的情况 ​

  1. 不要解构 reactive 对象：除非使用toRefs或toRef。
  2. 不要直接替换整个 reactive 对象：修改其属性，而不是替换整个对象。
  3. 在函数间传递时小心：传递 ref 或使用toRefs转换后的对象。

响应式状态的组织 ​

对于复杂组件，可以将相关的响应式状态组织在一起：

js
// 使用reactive组织相关状态
const userData = reactive({
  name: "John",
  age: 30,
  address: {
    city: "New York",
    country: "USA",
  },
});

// 或者使用组合式函数组织相关逻辑和状态
function useUser() {
  const name = ref("John");
  const age = ref(30);

  function updateName(newName) {
    name.value = newName;
  }

  return {
    name,
    age,
    updateName,
  };
}

const { name, age, updateName } = useUser();

深入理解响应式原理 ​

Proxy 的工作原理 ​

Vue3 的响应式系统基于 ES6 的 Proxy，它可以拦截对象的各种操作：

js
const original = { count: 0 };
const proxy = new Proxy(original, {
  get(target, key) {
    console.log(`Getting ${key}`);
    return target[key];
  },
  set(target, key, value) {
    console.log(`Setting ${key} to ${value}`);
    target[key] = value;
    return true;
  },
});

proxy.count; // 输出: Getting count
proxy.count = 1; // 输出: Setting count to 1

Vue3 的reactive函数创建的代理更加复杂，它不仅跟踪属性的访问和修改，还处理嵌套对象、集合类型（如 Map、Set）等。

依赖追踪 ​

Vue3 使用"依赖追踪"来确定哪些代码依赖于哪些响应式状态：

  1. 当一个组件渲染时，Vue 会跟踪在渲染过程中访问的所有响应式状态。
  2. 当这些状态变化时，Vue 知道需要重新渲染哪些组件。

这种精确的依赖追踪使得 Vue3 的性能得到了显著提升。

小结 ​

Vue3 的响应式系统是其最强大的特性之一，通过reactive和ref API，我们可以创建响应式状态，并在状态变化时自动更新视图。理解这些响应式 API 的使用和原理，是掌握 Vue3 的关键。在实际开发中，我们需要根据具体情况选择合适的 API，并注意避免响应式丢失的情况。
