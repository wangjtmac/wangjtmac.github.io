事件处理 ​

Vue 提供了 v-on 指令（简写为 @）来监听 DOM 事件并在触发时执行 JavaScript 代码。通过事件处理系统，Vue 让我们能够轻松地与用户交互并响应用户的操作。

监听事件 ​

我们可以使用 v-on 指令（通常简写为 @）来监听 DOM 事件：

vue
<template>
  <div>
    <!-- 完整语法 -->
    <button v-on:click="counter++">增加 1</button>

    <!-- 缩写语法 -->
    <button @click="counter++">增加 1</button>

    <p>按钮被点击了 {{ counter }} 次。</p>
  </div>
</template>

<script setup>
import { ref } from "vue";

const counter = ref(0);
</script>

事件处理方法 ​

对于更复杂的逻辑，直接把 JavaScript 代码写在模板中是不可行的。因此 v-on 也可以接受一个方法名或函数声明：

vue
<template>
  <div>
    <!-- 绑定事件处理方法 -->
    <button @click="greet">问候</button>

    <!-- 内联调用方法 -->
    <button @click="say('你好')">说你好</button>
    <button @click="say('再见')">说再见</button>
  </div>
</template>

<script setup>
import { ref } from "vue";

const name = ref("Vue.js");

function greet() {
  alert(`你好，${name.value}!`);
}

function say(message) {
  alert(message);
}
</script>

内联事件处理器 ​

有时需要在内联事件处理器中访问原生 DOM 事件。你可以向该处理器方法传入特殊的 $event 变量，或者使用内联箭头函数：

vue
<template>
  <div>
    <!-- 使用特殊的 $event 变量 -->
    <button @click="warn('表单不能提交', $event)">提交</button>

    <!-- 使用内联箭头函数 -->
    <button @click="(event) => warn('表单不能提交', event)">提交</button>
  </div>
</template>

<script setup>
function warn(message, event) {
  // 现在可以访问原生事件
  if (event) {
    event.preventDefault();
    console.log(event.target.tagName);
  }
  alert(message);
}
</script>

事件修饰符 ​

在处理事件时，我们经常需要调用 event.preventDefault() 或 event.stopPropagation()。尽管我们可以在方法中轻松实现这点，但更好的方式是让方法只关注数据逻辑而不去处理 DOM 事件细节。

为解决这一问题，Vue 提供了事件修饰符。修饰符是以点开头的指令后缀：

  • .stop - 调用 event.stopPropagation()，阻止事件冒泡
  • .prevent - 调用 event.preventDefault()，阻止默认事件
  • .capture - 在捕获模式添加事件监听器
  • .self - 只当事件是从侦听器绑定的元素本身触发时才触发回调
  • .once - 只触发一次回调
  • .passive - 以 { passive: true } 模式添加侦听器
vue
<template>
  <div>
    <!-- 阻止单击事件继续传播 -->
    <a @click.stop="doThis">点击</a>

    <!-- 提交事件不再重载页面 -->
    <form @submit.prevent="onSubmit">
      <button type="submit">提交</button>
    </form>

    <!-- 修饰符可以串联 -->
    <a @click.stop.prevent="doThat">点击</a>

    <!-- 只有修饰符 -->
    <form @submit.prevent>
      <button type="submit">提交</button>
    </form>

    <!-- 只当在 div 元素本身触发时才触发处理函数 -->
    <div @click.self="doSomething">...</div>

    <!-- 点击事件最多被触发一次 -->
    <button @click.once="doThis">点击</button>

    <!-- 滚动事件的默认行为 (即滚动行为) 将立即发生 -->
    <!-- 而不会等待 `onScroll` 完成 -->
    <div @scroll.passive="onScroll">...</div>
  </div>
</template>

<script setup>
function doThis() {
  console.log("doThis");
}

function doThat() {
  console.log("doThat");
}

function doSomething() {
  console.log("doSomething");
}

function onSubmit() {
  console.log("表单提交");
}

function onScroll() {
  console.log("滚动中");
}
</script>

注意

使用修饰符时需要注意顺序，因为相关代码是以相同顺序生成的。因此：

  • @click.prevent.self 会阻止所有的点击
  • @click.self.prevent 只会阻止元素自身的点击

提示

不要将 .passive 和 .prevent 一起使用，因为 .passive 会告诉浏览器你不想阻止事件的默认行为，而 .prevent 正是要阻止默认行为。

按键修饰符 ​

在监听键盘事件时，我们经常需要检查特定的按键。Vue 允许在 v-on 或 @ 监听按键事件时添加按键修饰符：

vue
<template>
  <div>
    <!-- 只有在 key 是 Enter 时调用 submit -->
    <input @keyup.enter="submit" />

    <!-- 只有在 key 是 Delete 或 Backspace 时调用 clearInput -->
    <input @keyup.delete="clearInput" />

    <!-- 只有在同时按下 Ctrl + C 时调用 onCopy -->
    <input @keyup.ctrl.c="onCopy" />

    <!-- 即使同时按下 Alt 或 Shift 也会触发 -->
    <button @click.ctrl="onClick">A</button>

    <!-- 有且只有 Ctrl 被按下时才触发 -->
    <button @click.ctrl.exact="onCtrlClick">A</button>

    <!-- 没有任何系统修饰符被按下时才触发 -->
    <button @click.exact="onClick">A</button>
  </div>
</template>

<script setup>
function submit() {
  console.log("提交");
}

function clearInput(event) {
  event.target.value = "";
  console.log("清除输入");
}

function onCopy() {
  console.log("复制");
}

function onClick() {
  console.log("点击");
}

function onCtrlClick() {
  console.log("Ctrl + 点击");
}
</script>

以下是常用的按键修饰符别名：

  • .enter
  • .tab
  • .delete (捕获"删除"和"退格"键)
  • .esc
  • .space
  • .up
  • .down
  • .left
  • .right

Vue 还提供了以下系统修饰键：

  • .ctrl
  • .alt
  • .shift
  • .meta (Windows 键或 macOS 上的 Command 键)

鼠标按钮修饰符 ​

Vue 提供了以下鼠标按钮修饰符：

  • .left - 左键
  • .right - 右键
  • .middle - 中键
vue
<template>
  <div>
    <!-- 只有点击鼠标右键时才触发 -->
    <button @click.right="onRightClick">右键点击</button>
  </div>
</template>

<script setup>
function onRightClick() {
  console.log("鼠标右键点击");
}
</script>

事件委托与性能 ​

在处理大量重复元素的事件时，通常会考虑使用事件委托模式来提高性能。Vue 的事件系统会自动做一些优化，因此在大多数情况下，你不需要手动优化事件处理以提高性能。

但是，如果确实需要自定义事件委托，可以使用以下方式：

vue
<template>
  <div>
    <!-- 列表外部绑定单个事件处理器 -->
    <ul @click="handleItemClick">
      <li v-for="item in items" :key="item.id" :data-id="item.id">
        {{ item.text }}
      </li>
    </ul>
  </div>
</template>

<script setup>
import { ref } from "vue";

const items = ref([
  { id: 1, text: "项目 1" },
  { id: 2, text: "项目 2" },
  { id: 3, text: "项目 3" },
  // ... 可能有成百上千个项目
]);

function handleItemClick(event) {
  // 使用事件委托，检查点击的是否为列表项
  if (event.target.tagName === "LI") {
    const id = Number(event.target.getAttribute("data-id"));
    console.log(`点击了项目 ID: ${id}`);
    // 查找对应的项目并执行操作
    const item = items.value.find((item) => item.id === id);
    if (item) {
      console.log(`点击了: ${item.text}`);
    }
  }
}
</script>

使用 v-model 处理表单事件 ​

对于表单元素，你可能经常需要在用户输入时更新数据。手动绑定 value 和事件处理程序可能很繁琐：

vue
<template>
  <div>
    <input :value="text" @input="text = $event.target.value" />
    <p>输入的文本: {{ text }}</p>
  </div>
</template>

<script setup>
import { ref } from "vue";

const text = ref("");
</script>

为了简化这种常见模式，Vue 提供了 v-model 指令，它是语法糖，负责监听用户的输入事件以更新数据，并特别处理一些极端场景：

vue
<template>
  <div>
    <input v-model="text" />
    <p>输入的文本: {{ text }}</p>
  </div>
</template>

<script setup>
import { ref } from "vue";

const text = ref("");
</script>

v-model 将根据不同的表单元素类型自动选择正确的方式来更新元素：

  • 文本类型的 <input> 和 <textarea> 元素使用 value 属性和 input 事件
  • <input type="checkbox"> 和 <input type="radio"> 使用 checked 属性和 change 事件
  • <select> 使用 value 属性和 change 事件

自定义组件事件 ​

在父子组件通信中，事件是一种重要的通信方式。子组件可以通过发射事件来通知父组件某些操作或状态变化，而父组件则可以监听这些事件。

在子组件中发射事件 ​

在子组件中，可以通过 defineEmits() 宏来声明需要发射的事件，然后使用返回的函数来发射事件：

vue
<!-- ChildComponent.vue -->
<template>
  <div>
    <button @click="increment">增加</button>
  </div>
</template>

<script setup>
const emit = defineEmits(["increment", "decrement"]);

function increment() {
  emit("increment", 1); // 发射事件，传递参数
}
</script>

在父组件中监听事件 ​

父组件可以使用 v-on（或 @）来监听子组件的事件：

vue
<!-- ParentComponent.vue -->
<template>
  <div>
    <p>计数: {{ count }}</p>
    <ChildComponent @increment="handleIncrement" @decrement="handleDecrement" />
  </div>
</template>

<script setup>
import { ref } from "vue";
import ChildComponent from "./ChildComponent.vue";

const count = ref(0);

function handleIncrement(value) {
  count.value += value;
}

function handleDecrement(value) {
  count.value -= value;
}
</script>

带有验证的事件 ​

与 props 类型验证类似，发射的事件也可以进行验证：

vue
<script setup>
const emit = defineEmits({
  // 没有验证
  click: null,

  // 验证 submit 事件
  submit: (payload) => {
    if (payload && typeof payload === "object" && "email" in payload) {
      return true;
    } else {
      console.warn("Invalid submit event payload!");
      return false;
    }
  },
});

function submitForm(email) {
  emit("submit", { email }); // 发射带有验证的事件
}
</script>

最佳实践 ​

事件命名 ​

与组件和 prop 一样，事件的名字也提供了自动的大小写转换。如果在子组件中触发一个以 camelCase (驼峰式) 命名的事件，你可以在父组件中使用 kebab-case (短横线分隔式) 来监听：

vue
<!-- 子组件 -->
<script setup>
const emit = defineEmits(["myEvent"]);

emit("myEvent"); // 驼峰式命名
</script>

<!-- 父组件 -->
<template>
  <ChildComponent @my-event="handleEvent" />
  <!-- 短横线分隔式命名 -->
</template>

与 props 的命名一样，在模板中使用 kebab-case 是推荐的方式，但在 JavaScript 代码中使用 camelCase 命名。

避免在模板中放置太多逻辑 ​

尽管在模板中可以直接使用内联事件处理器，但当逻辑变得复杂时，最好将逻辑移到方法或计算属性中：

vue
<template>
  <!-- 不好：模板中包含复杂逻辑 -->
  <button
    @click="
      if (isValid) {
        submitForm();
        trackEvent('form_submitted');
        resetForm();
      } else {
        showError();
      }
    "
  >
    提交
  </button>

  <!-- 好：逻辑放在方法中 -->
  <button @click="handleSubmit">提交</button>
</template>

<script setup>
import { ref } from "vue";

const isValid = ref(true);

function handleSubmit() {
  if (isValid.value) {
    submitForm();
    trackEvent("form_submitted");
    resetForm();
  } else {
    showError();
  }
}

function submitForm() {
  // 提交表单的逻辑
}

function trackEvent(event) {
  // 跟踪事件的逻辑
}

function resetForm() {
  // 重置表单的逻辑
}

function showError() {
  // 显示错误的逻辑
}
</script>

使用事件处理程序的几种方式对比 ​

Vue 中有多种方式来处理事件，每种方式都有其适用场景：

  1. 内联事件处理器：适用于简单操作

    vue
    <button @click="count++">增加</button>

  2. 方法事件处理器：适用于复杂逻辑

    vue
    <button @click="increment">增加</button>

  3. 带参数的事件处理器：适用于需要传递额外数据的情况

    vue
    <button @click="incrementBy(5)">增加 5</button>

  4. 使用 $event 的事件处理器：适用于需要访问原生 DOM 事件的情况

    vue
    <button @click="increment($event, 5)">增加 5</button>

总结 ​

Vue 的事件处理系统非常灵活和强大，它允许你：

  1. 使用 v-on 指令（或简写 @）来监听 DOM 事件
  2. 使用事件修饰符（如 .stop、.prevent 等）来处理常见的 DOM 事件操作
  3. 使用按键修饰符和鼠标按钮修饰符来监听特定的键盘和鼠标事件
  4. 在组件之间通过自定义事件进行通信
  5. 通过事件验证确保发射的事件符合预期

遵循这些最佳实践，你可以构建出交互性强、可维护的 Vue 应用程序。
