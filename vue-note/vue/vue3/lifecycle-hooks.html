生命周期钩子 ​

每个 Vue 组件实例在创建时都会经历一系列的初始化步骤，比如设置数据监听、编译模板、挂载实例到 DOM、以及在数据变化时更新 DOM。在此过程中，Vue 会在特定阶段调用生命周期钩子，让开发者有机会在特定阶段添加自己的代码。

生命周期图示 ​

下面是 Vue 组件生命周期的图示，展示了各个生命周期钩子在组件创建、更新和销毁过程中的调用时机：

创建阶段                更新阶段                  卸载阶段
   ↓                      ↓                       ↓
setup()              ────────────►            ────────────►
   ↓                      ↓                       ↓
onBeforeMount()    onBeforeUpdate()          onBeforeUnmount()
   ↓                      ↓                       ↓
onMounted()        onUpdated()              onUnmounted()

注册生命周期钩子 ​

组合式 API ​

在组合式 API 中，我们可以使用直接导入的 on* 函数注册生命周期钩子：

vue
<script setup>
import { onMounted, onUpdated, onUnmounted } from "vue";

onMounted(() => {
  console.log("组件已挂载！");
});

onUpdated(() => {
  console.log("组件已更新！");
});

onUnmounted(() => {
  console.log("组件已卸载！");
});
</script>

选项式 API ​

在选项式 API 中，我们可以直接在组件选项中设置相应的生命周期钩子：

vue
<script>
export default {
  beforeCreate() {
    console.log("组件即将创建");
  },
  created() {
    console.log("组件已创建");
  },
  beforeMount() {
    console.log("组件即将挂载");
  },
  mounted() {
    console.log("组件已挂载");
  },
  beforeUpdate() {
    console.log("组件即将更新");
  },
  updated() {
    console.log("组件已更新");
  },
  beforeUnmount() {
    console.log("组件即将卸载");
  },
  unmounted() {
    console.log("组件已卸载");
  },
};
</script>

详细的生命周期钩子 ​

创建阶段 ​

setup() [组合式 API] ​

  • 时机：组件创建过程中最先执行，在 beforeCreate 和 created 之前执行
  • 使用场景：setup 函数是组合式 API 的入口，用于定义响应式数据、方法和生命周期钩子
  • 示例：
vue
<script setup>
// setup 函数的代码会在组件创建过程中最先执行
import { ref } from "vue";

const count = ref(0);
</script>

beforeCreate [选项式 API] ​

  • 时机：实例初始化之后，数据观测 (data) 和事件配置之前调用
  • 使用场景：这个阶段实例刚刚创建，还不能访问到 data 中的数据和 methods 中的方法
  • 示例：
js
export default {
  beforeCreate() {
    console.log("beforeCreate", this.$data); // undefined
  },
};

created [选项式 API] ​

  • 时机：实例创建完成后立即调用
  • 使用场景：此时可以访问数据和方法，可以进行 API 调用，但尚未挂载到 DOM，无法访问 $el
  • 示例：
js
export default {
  created() {
    console.log("created", this.$data); // 可以访问数据
    // 可以在此处发起 API 请求
    this.fetchData();
  },
  methods: {
    fetchData() {
      // 发起 API 请求的逻辑
    },
  },
};

挂载阶段 ​

onBeforeMount() [组合式 API] / beforeMount [选项式 API] ​

  • 时机：组件被挂载到 DOM 之前调用
  • 使用场景：此时模板已编译，但还没有渲染到页面上
  • 示例：
vue
<script setup>
import { onBeforeMount } from "vue";

onBeforeMount(() => {
  console.log("组件即将挂载");
  console.log(document.querySelector("#app")); // null，DOM尚未挂载
});
</script>

选项式 API：

js
export default {
  beforeMount() {
    console.log("beforeMount");
    console.log(this.$el); // 此时 $el 已经创建，但尚未插入文档
  },
};

onMounted() [组合式 API] / mounted [选项式 API] ​

  • 时机：组件挂载到 DOM 后调用
  • 使用场景：最常用的生命周期钩子之一，可以访问 DOM，执行需要 DOM 的操作，如初始化第三方库、发送请求等
  • 示例：
vue
<script setup>
import { onMounted, ref } from "vue";

const chartRef = ref(null);

onMounted(() => {
  console.log("组件已挂载");
  // 访问 DOM
  console.log(document.querySelector("#app"));

  // 初始化图表库
  if (chartRef.value) {
    initChart(chartRef.value);
  }

  // 发送 API 请求
  fetchData();
});

function initChart(element) {
  // 初始化图表的逻辑
}

function fetchData() {
  // 获取数据的逻辑
}
</script>

<template>
  <div ref="chartRef"></div>
</template>

选项式 API：

js
export default {
  mounted() {
    console.log("mounted");
    console.log(this.$el); // 可以访问 DOM

    // 初始化第三方库
    if (this.$refs.chart) {
      this.initChart(this.$refs.chart);
    }
  },
  methods: {
    initChart(element) {
      // 初始化图表的逻辑
    },
  },
};

更新阶段 ​

onBeforeUpdate() [组合式 API] / beforeUpdate [选项式 API] ​

  • 时机：数据更新后，DOM 重新渲染之前调用
  • 使用场景：适合在更新前访问现有的 DOM 状态，如记录更新前的滚动位置等
  • 示例：
vue
<script setup>
import { onBeforeUpdate, ref } from "vue";

const count = ref(0);
const scrollPosition = ref(0);

onBeforeUpdate(() => {
  // 记录更新前的滚动位置
  scrollPosition.value = document.documentElement.scrollTop;
});
</script>

选项式 API：

js
export default {
  data() {
    return {
      count: 0,
      scrollPosition: 0,
    };
  },
  beforeUpdate() {
    // 记录更新前的滚动位置
    this.scrollPosition = document.documentElement.scrollTop;
  },
};

onUpdated() [组合式 API] / updated [选项式 API] ​

  • 时机：DOM 重新渲染后调用
  • 使用场景：可以执行依赖于更新后的 DOM 的操作，如恢复滚动位置等
  • 注意：避免在此钩子中修改组件状态，可能导致无限更新循环
  • 示例：
vue
<script setup>
import { onUpdated, ref } from "vue";

const count = ref(0);
let previousScrollPosition = 0;

onBeforeUpdate(() => {
  previousScrollPosition = document.documentElement.scrollTop;
});

onUpdated(() => {
  console.log("组件已更新");

  // 恢复滚动位置
  document.documentElement.scrollTop = previousScrollPosition;

  // 更新后 DOM 相关操作
  updateDomDependentState();
});

function updateDomDependentState() {
  // 更新依赖于 DOM 的状态
}
</script>

选项式 API：

js
export default {
  data() {
    return {
      count: 0,
    };
  },
  updated() {
    console.log("updated");

    // 执行依赖于更新后的 DOM 的操作
    this.updateDomDependentState();
  },
  methods: {
    updateDomDependentState() {
      // 更新依赖于 DOM 的状态
    },
  },
};

卸载阶段 ​

onBeforeUnmount() [组合式 API] / beforeUnmount [选项式 API] ​

  • 时机：组件卸载前调用
  • 使用场景：适合清理计时器、事件监听器等资源，避免内存泄漏
  • 示例：
vue
<script setup>
import { onBeforeUnmount, onMounted } from "vue";

let intervalId = null;

onMounted(() => {
  // 设置定时器
  intervalId = setInterval(() => {
    console.log("定时器运行中...");
  }, 1000);

  // 添加事件监听器
  window.addEventListener("resize", handleResize);
});

onBeforeUnmount(() => {
  // 清理定时器
  if (intervalId) {
    clearInterval(intervalId);
    intervalId = null;
  }

  // 移除事件监听器
  window.removeEventListener("resize", handleResize);
});

function handleResize() {
  console.log("窗口大小改变了");
}
</script>

选项式 API：

js
export default {
  data() {
    return {
      intervalId: null,
    };
  },
  mounted() {
    // 设置定时器
    this.intervalId = setInterval(() => {
      console.log("定时器运行中...");
    }, 1000);

    // 添加事件监听器
    window.addEventListener("resize", this.handleResize);
  },
  beforeUnmount() {
    // 清理定时器
    if (this.intervalId) {
      clearInterval(this.intervalId);
      this.intervalId = null;
    }

    // 移除事件监听器
    window.removeEventListener("resize", this.handleResize);
  },
  methods: {
    handleResize() {
      console.log("窗口大小改变了");
    },
  },
};

onUnmounted() [组合式 API] / unmounted [选项式 API] ​

  • 时机：组件卸载后调用
  • 使用场景：执行最终的清理操作，此时所有的子组件也都已经被卸载
  • 示例：
vue
<script setup>
import { onUnmounted } from "vue";

onUnmounted(() => {
  console.log("组件已卸载");

  // 可以执行额外的清理操作
  cleanupResources();
});

function cleanupResources() {
  // 清理资源的逻辑
}
</script>

选项式 API：

js
export default {
  unmounted() {
    console.log("组件已卸载");

    // 可以执行额外的清理操作
    this.cleanupResources();
  },
  methods: {
    cleanupResources() {
      // 清理资源的逻辑
    },
  },
};

特殊的生命周期钩子 ​

errorCaptured ​

  • 时机：当捕获到后代组件传递的错误时调用
  • 使用场景：用于捕获所有后代组件的错误，实现错误边界
  • 示例：
vue
<script setup>
import { onErrorCaptured, ref } from "vue";

const error = ref(null);

onErrorCaptured((err, instance, info) => {
  // 处理错误
  error.value = {
    message: err.message,
    component: instance.$.type.name,
    info,
  };

  // 返回 false 可以阻止错误继续传播
  return false;
});
</script>

<template>
  <div>
    <div v-if="error" class="error-boundary">
      <h2>出错了</h2>
      <p>{{ error.message }}</p>
      <p>组件: {{ error.component }}</p>
    </div>
    <slot v-else></slot>
  </div>
</template>

选项式 API：

js
export default {
  data() {
    return {
      error: null,
    };
  },
  errorCaptured(err, instance, info) {
    // 处理错误
    this.error = {
      message: err.message,
      component: instance.$options.name,
      info,
    };

    // 返回 false 可以阻止错误继续传播
    return false;
  },
};

renderTracked 和 renderTriggered [仅在开发模式下可用] ​

  • 时机：分别在依赖项跟踪和触发重新渲染时调用
  • 使用场景：用于调试组件的渲染性能
  • 示例：
vue
<script setup>
import { onRenderTracked, onRenderTriggered } from "vue";

// 仅在开发模式下工作
onRenderTracked((event) => {
  console.log("依赖跟踪：", event);
});

onRenderTriggered((event) => {
  console.log("触发重新渲染：", event);
});
</script>

选项式 API：

js
export default {
  renderTracked(event) {
    console.log("依赖跟踪：", event);
  },
  renderTriggered(event) {
    console.log("触发重新渲染：", event);
  },
};

activated 和 deactivated ​

  • 时机：被 <keep-alive> 缓存的组件激活和停用时调用
  • 使用场景：适合需要在组件被缓存时保持某些状态，或在激活时恢复状态的场景
  • 示例：
vue
<script setup>
import { onActivated, onDeactivated, ref } from "vue";

const scrollPosition = ref(0);

onActivated(() => {
  console.log("组件被激活了");

  // 恢复滚动位置
  document.documentElement.scrollTop = scrollPosition.value;
});

onDeactivated(() => {
  console.log("组件被停用了");

  // 保存滚动位置
  scrollPosition.value = document.documentElement.scrollTop;
});
</script>

选项式 API：

js
export default {
  data() {
    return {
      scrollPosition: 0,
    };
  },
  activated() {
    console.log("组件被激活了");

    // 恢复滚动位置
    document.documentElement.scrollTop = this.scrollPosition;
  },
  deactivated() {
    console.log("组件被停用了");

    // 保存滚动位置
    this.scrollPosition = document.documentElement.scrollTop;
  },
};

生命周期钩子最佳实践 ​

数据获取的最佳时机 ​

根据数据获取的性质和用途，可以选择不同的生命周期钩子：

  1. onMounted：最常用的数据获取时机，组件已挂载到 DOM，可以显示加载状态

    vue
    <script setup>
import { ref, onMounted } from "vue";

const posts = ref([]);
const loading = ref(true);
const error = ref(null);

onMounted(async () => {
  try {
    const response = await fetch("https://api.example.com/posts");
    posts.value = await response.json();
  } catch (err) {
    error.value = err.message;
  } finally {
    loading.value = false;
  }
});
</script>

<template>
  <div v-if="loading">加载中...</div>
  <div v-else-if="error">出错了: {{ error }}</div>
  <ul v-else>
    <li v-for="post in posts" :key="post.id">{{ post.title }}</li>
  </ul>
</template>

  2. setup/created：对于不依赖 DOM 的数据获取，可以更早进行，减少用户等待时间

    vue
    <script setup>
import { ref } from "vue";

const userInfo = ref(null);
const loading = ref(true);

// 直接在 setup 中获取数据
fetchUserInfo();

async function fetchUserInfo() {
  try {
    const response = await fetch("https://api.example.com/user");
    userInfo.value = await response.json();
  } catch (error) {
    console.error(error);
  } finally {
    loading.value = false;
  }
}
</script>

资源清理和内存管理 ​

在 onBeforeUnmount/beforeUnmount 中清理资源，避免内存泄漏：

vue
<script setup>
import { onMounted, onBeforeUnmount } from "vue";
import { initChart, destroyChart } from "chart-library";

let chart = null;

onMounted(() => {
  chart = initChart("#chart-container", {
    // 图表配置
  });
});

onBeforeUnmount(() => {
  if (chart) {
    destroyChart(chart);
    chart = null;
  }
});
</script>

使用生命周期钩子进行日志记录和调试 ​

在开发阶段，可以使用生命周期钩子来调试组件的生命周期：

vue
<script setup>
import {
  onBeforeMount,
  onMounted,
  onBeforeUpdate,
  onUpdated,
  onBeforeUnmount,
  onUnmounted,
} from "vue";

const componentName = "MyComponent";

onBeforeMount(() => console.log(`${componentName} - onBeforeMount`));
onMounted(() => console.log(`${componentName} - onMounted`));
onBeforeUpdate(() => console.log(`${componentName} - onBeforeUpdate`));
onUpdated(() => console.log(`${componentName} - onUpdated`));
onBeforeUnmount(() => console.log(`${componentName} - onBeforeUnmount`));
onUnmounted(() => console.log(`${componentName} - onUnmounted`));
</script>

在更新钩子中避免无限循环 ​

在 onUpdated/updated 钩子中修改数据时要格外小心，以避免无限循环：

vue
<script setup>
import { ref, onUpdated, nextTick } from "vue";

const count = ref(0);
let isUpdating = false;

onUpdated(async () => {
  // 防止无限循环
  if (isUpdating) return;

  // 需要更新数据的情况
  if (needToUpdateData()) {
    isUpdating = true;

    // 使用 nextTick 等待 DOM 更新完成
    await nextTick();

    // 安全地更新数据
    count.value++;

    isUpdating = false;
  }
});

function needToUpdateData() {
  // 判断是否需要更新数据的逻辑
  return false; // 通常返回 false，避免循环
}
</script>

总结 ​

  • Vue 提供了丰富的生命周期钩子，帮助开发者在组件不同阶段执行代码
  • 组合式 API 使用 onXXX 函数注册生命周期钩子，选项式 API 直接在选项对象中定义
  • 创建阶段的钩子包括 setup（组合式 API 特有）、beforeCreate 和 created
  • 挂载阶段的钩子有 onBeforeMount/beforeMount 和 onMounted/mounted
  • 更新阶段的钩子有 onBeforeUpdate/beforeUpdate 和 onUpdated/updated
  • 卸载阶段的钩子有 onBeforeUnmount/beforeUnmount 和 onUnmounted/unmounted
  • 特殊钩子包括 errorCaptured（错误处理）、renderTracked/renderTriggered（调试用）、activated/deactivated（与 keep-alive 配合使用）
  • 合理使用生命周期钩子可以优化数据获取、资源管理和组件性能 </rewritten_file>
