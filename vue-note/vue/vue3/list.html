列表渲染 ​

在 Vue 中，我们可以使用 v-for 指令基于一个数组来渲染一个列表。v-for 指令需要使用 item in items 形式的特殊语法，其中 items 是源数据数组，而 item 则是被迭代的数组元素的别名。

基本用法 ​

遍历数组 ​

vue
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      {{ item.message }}
    </li>
  </ul>
</template>

<script setup>
import { ref } from "vue";

const items = ref([
  { id: 1, message: "苹果" },
  { id: 2, message: "香蕉" },
  { id: 3, message: "橙子" },
]);
</script>

在 v-for 块中，我们可以访问所有父作用域的 property。v-for 还支持一个可选的第二个参数，即当前项的索引。

vue
<template>
  <ul>
    <li v-for="(item, index) in items" :key="item.id">
      {{ index }} - {{ item.message }}
    </li>
  </ul>
</template>

你也可以使用 of 替代 in 作为分隔符，因为它更接近 JavaScript 的迭代器语法：

vue
<template>
  <ul>
    <li v-for="item of items" :key="item.id">
      {{ item.message }}
    </li>
  </ul>
</template>

遍历对象 ​

你也可以用 v-for 来遍历一个对象的 property：

vue
<template>
  <ul>
    <li v-for="(value, key, index) in myObject" :key="key">
      {{ index }}. {{ key }}: {{ value }}
    </li>
  </ul>
</template>

<script setup>
import { reactive } from "vue";

const myObject = reactive({
  title: "Vue 列表渲染",
  author: "张三",
  publishedAt: "2023-01-01",
});
</script>

在遍历对象时，会按 Object.keys() 的结果遍历，但是不能保证它的结果在不同的 JavaScript 引擎下都一致。

遍历数字范围 ​

v-for 也可以接受整数。在这种情况下，它会把模板重复对应次数。

vue
<template>
  <ul>
    <li v-for="n in 10" :key="n">{{ n }}</li>
  </ul>
</template>

注意此处 n 的初值是从 1 开始而不是 0。

使用 <template> 的 v-for ​

类似于 v-if，你也可以利用带有 v-for 的 <template> 来循环渲染一段包含多个元素的内容。比如：

vue
<template>
  <ul>
    <template v-for="item in items" :key="item.id">
      <li>{{ item.name }}</li>
      <li class="divider"></li>
    </template>
  </ul>
</template>

v-for 与 v-if 一起使用 ​

WARNING

注意不要在同一个元素上同时使用 v-if 和 v-for。

当它们处于同一节点，v-if 的优先级比 v-for 更高，这意味着 v-if 将没有权限访问 v-for 里的变量：

vue
<!-- 错误写法 -->
<template>
  <li v-for="item in items" v-if="item.visible" :key="item.id">
    {{ item.message }}
  </li>
</template>

可以用 <template> 标签包装 v-for 的元素来解决这个问题：

vue
<!-- 正确写法 -->
<template>
  <template v-for="item in items" :key="item.id">
    <li v-if="item.visible">
      {{ item.message }}
    </li>
  </template>
</template>

或者，更好的做法是使用计算属性过滤列表：

vue
<template>
  <li v-for="item in visibleItems" :key="item.id">
    {{ item.message }}
  </li>
</template>

<script setup>
import { ref, computed } from "vue";

const items = ref([
  { id: 1, message: "苹果", visible: true },
  { id: 2, message: "香蕉", visible: false },
  { id: 3, message: "橙子", visible: true },
]);

const visibleItems = computed(() => {
  return items.value.filter((item) => item.visible);
});
</script>

维护状态 ​

当 Vue 正在更新使用 v-for 渲染的元素列表时，它默认使用"就地更新"的策略。如果数据项的顺序被改变，Vue 将不会移动 DOM 元素来匹配数据项的顺序，而是就地更新每个元素，并且确保它们在每个索引位置正确渲染。

这个默认的模式是高效的，但是只适用于不依赖子组件状态或临时 DOM 状态 (例如：表单输入值) 的列表渲染输出。

为了给 Vue 一个提示，以便它能跟踪每个节点的身份，从而重用和重新排序现有元素，你需要为每项提供一个唯一的 key 属性：

vue
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      {{ item.message }}
    </li>
  </ul>
</template>

建议尽可能在使用 v-for 时提供 key attribute，除非遍历输出的 DOM 内容非常简单，或者是刻意依赖默认行为以获取性能上的提升。

因为它是 Vue 识别节点的一个通用机制，key 并不仅与 v-for 特别关联，我们将在之后的章节中看到其他用例。

TIP

不要使用对象或数组之类的非基本类型值作为 v-for 的 key。请用字符串或数值类型的值。

数组更新检测 ​

变更方法 ​

Vue 能够侦听响应式数组的变更方法，并在它们被调用时触发相关的更新。这些变更方法包括：

  • push()
  • pop()
  • shift()
  • unshift()
  • splice()
  • sort()
  • reverse()
vue
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      {{ item.message }}
    </li>
  </ul>
  <button @click="addItem">添加项目</button>
  <button @click="removeItem">移除项目</button>
</template>

<script setup>
import { ref } from "vue";

const items = ref([
  { id: 1, message: "苹果" },
  { id: 2, message: "香蕉" },
]);

function addItem() {
  items.value.push({ id: items.value.length + 1, message: "新水果" });
}

function removeItem() {
  items.value.pop();
}
</script>

替换数组 ​

变更方法，顾名思义，会变更调用了这些方法的原始数组。相比之下，也有非变更方法，例如 filter()、concat() 和 slice()。它们不会变更原始数组，而总是返回一个新数组。当使用非变更方法时，我们可以用新数组替换旧数组：

vue
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      {{ item.message }}
    </li>
  </ul>
  <button @click="filterItems">只显示苹果</button>
</template>

<script setup>
import { ref } from "vue";

const items = ref([
  { id: 1, message: "苹果" },
  { id: 2, message: "香蕉" },
  { id: 3, message: "苹果" },
]);

function filterItems() {
  items.value = items.value.filter((item) => item.message === "苹果");
}
</script>

你可能认为这将导致 Vue 丢弃现有 DOM 并重新渲染整个列表。幸运的是，事实并非如此。Vue 为了使得 DOM 元素得到最大范围的重用，实现了一些智能启发式方法，所以用一个含有相同元素的数组去替换原来的数组是非常高效的操作。

显示过滤/排序后的结果 ​

有时，我们想要显示一个数组经过过滤或排序后的版本，而不实际变更或重置原始数据。在这种情况下，你可以创建一个计算属性，来返回过滤或排序后的数组。

vue
<template>
  <ul>
    <li v-for="item in filteredItems" :key="item.id">
      {{ item.message }}
    </li>
  </ul>
  <button @click="toggleFilter">
    {{ showAll ? "只显示苹果" : "显示全部" }}
  </button>
</template>

<script setup>
import { ref, computed } from "vue";

const showAll = ref(true);
const items = ref([
  { id: 1, message: "苹果" },
  { id: 2, message: "香蕉" },
  { id: 3, message: "苹果" },
  { id: 4, message: "橙子" },
]);

const filteredItems = computed(() => {
  return showAll.value
    ? items.value
    : items.value.filter((item) => item.message === "苹果");
});

function toggleFilter() {
  showAll.value = !showAll.value;
}
</script>

在计算属性不适用的情况下 (例如，在嵌套的 v-for 循环中)，你可以使用一个方法：

vue
<template>
  <ul>
    <li v-for="item in filterItems(items)" :key="item.id">
      {{ item.message }}
    </li>
  </ul>
</template>

<script setup>
import { ref } from "vue";

const items = ref([
  { id: 1, message: "苹果" },
  { id: 2, message: "香蕉" },
  { id: 3, message: "苹果" },
  { id: 4, message: "橙子" },
]);

function filterItems(items) {
  return items.filter((item) => item.message === "苹果");
}
</script>

在 v-for 里使用范围值 ​

v-for 也可以接受整数。在这种情况下，它会把模板重复对应次数。

vue
<template>
  <div>
    <span v-for="n in 10" :key="n">{{ n }} </span>
  </div>
</template>

注意此处 n 的初始值是从 1 开始而不是 0。

在 <template> 中使用 v-for ​

类似于 v-if，你也可以利用带有 v-for 的 <template> 来循环渲染一段包含多个元素的内容。比如：

vue
<template>
  <ul>
    <template v-for="item in items" :key="item.id">
      <li>{{ item.message }}</li>
      <li class="divider" role="separator"></li>
    </template>
  </ul>
</template>

组件上使用 v-for ​

在自定义组件上，你可以像在任何普通元素上一样使用 v-for：

vue
<template>
  <div>
    <my-component
      v-for="(item, index) in items"
      :key="item.id"
      :item="item"
      :index="index"
    ></my-component>
  </div>
</template>

然而，这不会自动将任何数据传递到组件里，因为组件有自己独立的作用域。为了把迭代数据传递到组件里，我们要使用 props：

vue
<!-- MyComponent.vue -->
<template>
  <li>{{ item.message }} - {{ index }}</li>
</template>

<script setup>
defineProps(["item", "index"]);
</script>

使用该组件：

vue
<template>
  <ul>
    <my-component
      v-for="(item, index) in items"
      :key="item.id"
      :item="item"
      :index="index"
    ></my-component>
  </ul>
</template>

<script setup>
import { ref } from "vue";
import MyComponent from "./MyComponent.vue";

const items = ref([
  { id: 1, message: "苹果" },
  { id: 2, message: "香蕉" },
  { id: 3, message: "橙子" },
]);
</script>

列表渲染的最佳实践 ​

使用 key ​

在使用 v-for 时，始终为列表项提供一个唯一的 key 属性：

vue
<template>
  <div>
    <div v-for="item in items" :key="item.id">
      <!-- 内容 -->
    </div>
  </div>
</template>

这有助于 Vue 跟踪每个节点的身份，从而重用和重新排序现有元素，使得列表渲染更加高效。

避免直接修改数组索引 ​

在 Vue 3 中，即使直接修改数组索引也能触发视图更新，但为了代码的可读性和一致性，建议使用数组变更方法或创建新数组：

vue
<template>
  <div>
    <div v-for="(item, index) in items" :key="item.id">
      {{ item.message }}
      <button @click="updateItem(index)">更新</button>
    </div>
  </div>
</template>

<script setup>
import { ref } from "vue";

const items = ref([
  { id: 1, message: "苹果" },
  { id: 2, message: "香蕉" },
  { id: 3, message: "橙子" },
]);

function updateItem(index) {
  // 推荐方式
  const newItems = [...items.value];
  newItems[index] = {
    ...newItems[index],
    message: newItems[index].message + " (已更新)",
  };
  items.value = newItems;

  // 或者使用 splice
  // items.value.splice(index, 1, { ...items.value[index], message: items.value[index].message + ' (已更新)' })

  // 不推荐（虽然在 Vue 3 中也能工作）
  // items.value[index].message = items.value[index].message + ' (已更新)'
}
</script>

使用计算属性处理过滤和排序 ​

当你需要展示经过过滤或排序的列表时，应该使用计算属性而不是在模板中使用复杂表达式：

vue
<template>
  <div>
    <input v-model="searchQuery" placeholder="搜索水果..." />
    <ul>
      <li v-for="item in filteredItems" :key="item.id">
        {{ item.message }}
      </li>
    </ul>
  </div>
</template>

<script setup>
import { ref, computed } from "vue";

const searchQuery = ref("");
const items = ref([
  { id: 1, message: "苹果" },
  { id: 2, message: "香蕉" },
  { id: 3, message: "橙子" },
  { id: 4, message: "葡萄" },
]);

const filteredItems = computed(() => {
  const query = searchQuery.value.toLowerCase();
  return items.value.filter((item) =>
    item.message.toLowerCase().includes(query)
  );
});
</script>

处理空列表 ​

当列表可能为空时，提供一个友好的提示：

vue
<template>
  <div>
    <ul v-if="items.length">
      <li v-for="item in items" :key="item.id">
        {{ item.message }}
      </li>
    </ul>
    <p v-else>暂无数据</p>
  </div>
</template>

列表分组 ​

使用嵌套的 v-for 可以实现列表分组：

vue
<template>
  <div>
    <div v-for="(group, groupName) in groupedItems" :key="groupName">
      <h3>{{ groupName }}</h3>
      <ul>
        <li v-for="item in group" :key="item.id">
          {{ item.message }}
        </li>
      </ul>
    </div>
  </div>
</template>

<script setup>
import { reactive, computed } from "vue";

const items = reactive([
  { id: 1, message: "苹果", category: "水果" },
  { id: 2, message: "香蕉", category: "水果" },
  { id: 3, message: "胡萝卜", category: "蔬菜" },
  { id: 4, message: "西红柿", category: "蔬菜" },
]);

const groupedItems = computed(() => {
  const groups = {};
  items.forEach((item) => {
    if (!groups[item.category]) {
      groups[item.category] = [];
    }
    groups[item.category].push(item);
  });
  return groups;
});
</script>

总结 ​

Vue 的列表渲染功能通过 v-for 指令提供，它可以：

  1. 遍历数组：v-for="item in items"
  2. 遍历对象：v-for="(value, key, index) in object"
  3. 遍历数字范围：v-for="n in 10"

为了高效的列表渲染，记住以下几点：

  1. 始终为列表项提供唯一的 key 属性
  2. 避免同时使用 v-if 和 v-for
  3. 使用计算属性处理过滤和排序逻辑
  4. 使用数组变更方法或创建新数组来更新列表
  5. 在组件上使用 v-for 时，通过 props 传递数据 </rewritten_file>
