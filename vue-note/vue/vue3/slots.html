插槽 ​

插槽（Slots）是 Vue 中一种强大的内容分发机制，允许组件接收并渲染传入的内容，使组件更加灵活和可复用。本文档详细介绍 Vue 3 中插槽的使用方法和最佳实践。

插槽基础 ​

基本用法 ​

插槽允许父组件将内容传递给子组件的模板中。子组件需要使用 <slot> 元素来指定内容的放置位置：

vue
<!-- 子组件：Button.vue -->
<template>
  <button class="custom-button">
    <slot></slot>
    <!-- 插槽出口 -->
  </button>
</template>

<style scoped>
.custom-button {
  padding: 10px 20px;
  background-color: #4caf50;
  color: white;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}
</style>

父组件使用子组件时，可以在子组件标签内传入内容：

vue
<!-- 父组件 -->
<template>
  <div>
    <Button>点击提交</Button>
    <Button>
      <span class="icon">✓</span>
      确认
    </Button>
  </div>
</template>

<script setup>
import Button from "./Button.vue";
</script>

默认内容 ​

当父组件没有提供内容时，可以在 <slot> 中设置默认内容：

vue
<!-- 子组件：Button.vue -->
<template>
  <button class="custom-button">
    <slot>默认文本</slot>
  </button>
</template>

父组件使用：

vue
<template>
  <div>
    <Button>自定义文本</Button>
    <!-- 显示"自定义文本" -->
    <Button></Button>
    <!-- 显示"默认文本" -->
  </div>
</template>

具名插槽 ​

当需要多个插槽时，可以使用具名插槽来区分不同的插槽位置。

声明具名插槽 ​

在子组件中使用 name 属性为插槽命名：

vue
<!-- 子组件：BaseLayout.vue -->
<template>
  <div class="container">
    <header>
      <slot name="header"></slot>
    </header>

    <main>
      <slot></slot>
      <!-- 默认插槽，等同于 name="default" -->
    </main>

    <footer>
      <slot name="footer"></slot>
    </footer>
  </div>
</template>

<style scoped>
.container {
  display: grid;
  grid-template-rows: auto 1fr auto;
  height: 100vh;
}
</style>

使用具名插槽 ​

在父组件中，使用 v-slot 指令（缩写为 #）来指定内容要插入的具名插槽：

vue
<template>
  <BaseLayout>
    <template v-slot:header>
      <h1>网站标题</h1>
    </template>

    <template v-slot:default>
      <!-- 或直接省略 v-slot:default -->
      <p>主要内容区域</p>
    </template>

    <template v-slot:footer>
      <p>版权信息 © 2023</p>
    </template>
  </BaseLayout>
</template>

使用缩写形式 # 更加简洁：

vue
<template>
  <BaseLayout>
    <template #header>
      <h1>网站标题</h1>
    </template>

    <!-- 默认插槽内容可以直接放在组件标签内 -->
    <p>主要内容区域</p>

    <template #footer>
      <p>版权信息 © 2023</p>
    </template>
  </BaseLayout>
</template>

作用域插槽 ​

作用域插槽允许子组件向父组件传递数据，让父组件可以访问子组件内部的数据。

基本使用 ​

在子组件中，通过 v-bind 将数据传递给插槽：

vue
<!-- 子组件：UserList.vue -->
<template>
  <ul>
    <li v-for="(user, index) in users" :key="user.id">
      <slot :user="user" :index="index"></slot>
    </li>
  </ul>
</template>

<script setup>
import { ref } from "vue";

const users = ref([
  { id: 1, name: "张三", online: true },
  { id: 2, name: "李四", online: false },
  { id: 3, name: "王五", online: true },
]);
</script>

在父组件中，通过 v-slot 的值来访问子组件传递的数据：

vue
<template>
  <UserList>
    <!-- slotProps 可以是任何名称 -->
    <template v-slot="slotProps">
      <span :class="{ online: slotProps.user.online }">
        {{ slotProps.user.name }}
      </span>
      <small>索引: {{ slotProps.index }}</small>
    </template>
  </UserList>
</template>

使用解构语法更加清晰：

vue
<template>
  <UserList>
    <template v-slot="{ user, index }">
      <span :class="{ online: user.online }">
        {{ user.name }}
      </span>
      <small>索引: {{ index }}</small>
    </template>
  </UserList>
</template>

具名作用域插槽 ​

具名插槽和作用域插槽可以结合使用：

vue
<!-- 子组件：TaskList.vue -->
<template>
  <div class="task-list">
    <header>
      <slot name="header" :count="tasks.length"></slot>
    </header>

    <ul>
      <li v-for="(task, index) in tasks" :key="task.id">
        <slot name="task" :task="task" :index="index"></slot>
      </li>
    </ul>

    <footer>
      <slot
        name="footer"
        :completed="completedCount"
        :total="tasks.length"
      ></slot>
    </footer>
  </div>
</template>

<script setup>
import { ref, computed } from "vue";

const tasks = ref([
  { id: 1, text: "学习 Vue", completed: true },
  { id: 2, text: "创建组件", completed: false },
  { id: 3, text: "使用插槽", completed: false },
]);

const completedCount = computed(() => {
  return tasks.value.filter((task) => task.completed).length;
});
</script>

在父组件中使用：

vue
<template>
  <TaskList>
    <template #header="{ count }">
      <h2>任务列表 ({{ count }})</h2>
    </template>

    <template #task="{ task, index }">
      <div class="task-item" :class="{ completed: task.completed }">
        <input type="checkbox" v-model="task.completed" />
        <span>{{ index + 1 }}. {{ task.text }}</span>
      </div>
    </template>

    <template #footer="{ completed, total }">
      <p>完成进度: {{ completed }}/{{ total }}</p>
    </template>
  </TaskList>
</template>

插槽的高级用法 ​

动态插槽名 ​

插槽名可以是动态的，使用 v-slot:[动态名称] 或 #[动态名称]：

vue
<template>
  <BaseLayout>
    <template #[currentSlot]>
      <!-- 根据 currentSlot 变量的值动态确定插入的插槽 -->
      当前插槽内容
    </template>
  </BaseLayout>
</template>

<script setup>
import { ref } from "vue";

const currentSlot = ref("header"); // 可以动态改变：'header', 'footer' 等
</script>

插槽传递（转发插槽） ​

可以将从父组件接收的插槽内容再传递到子组件，这种技术称为插槽传递或转发插槽：

vue
<!-- 中间组件：Wrapper.vue -->
<template>
  <div class="wrapper">
    <BaseLayout>
      <template #header>
        <!-- 转发 header 插槽 -->
        <slot name="header"></slot>
      </template>

      <template #default>
        <!-- 转发默认插槽 -->
        <slot></slot>
      </template>

      <template #footer>
        <!-- 转发 footer 插槽 -->
        <slot name="footer"></slot>
      </template>
    </BaseLayout>
  </div>
</template>

使用这个中间组件：

vue
<template>
  <Wrapper>
    <template #header>
      <h1>标题</h1>
    </template>

    <p>主要内容</p>

    <template #footer>
      <p>页脚</p>
    </template>
  </Wrapper>
</template>

渲染作用域 ​

在插槽内容中，只能访问父组件的数据，除非使用作用域插槽从子组件传递数据：

vue
<template>
  <ChildComponent>
    {{ parentData }}
    <!-- 可以访问父组件的数据 -->
    {{ childData }}
    <!-- 不能访问子组件的数据，除非通过作用域插槽传递 -->
  </ChildComponent>
</template>

<script setup>
import { ref } from "vue";
import ChildComponent from "./ChildComponent.vue";

const parentData = ref("父组件数据");
</script>

插槽实践与模式 ​

组件库开发中的插槽应用 ​

在设计可复用组件时，插槽是提供灵活性的关键。以下是常见的插槽使用模式：

1. 内容占位符插槽 ​

为组件的主要内容区域提供插槽：

vue
<!-- Card.vue -->
<template>
  <div class="card">
    <div class="card-header">
      <slot name="header">
        <h3>默认标题</h3>
      </slot>
    </div>

    <div class="card-body">
      <slot></slot>
    </div>

    <div class="card-footer">
      <slot name="footer"></slot>
    </div>
  </div>
</template>

2. 定制化渲染插槽 ​

允许父组件完全控制每个项的渲染方式：

vue
<!-- DataTable.vue -->
<template>
  <table>
    <thead>
      <tr>
        <th v-for="column in columns" :key="column.key">
          <slot name="header" :column="column">
            {{ column.title }}
          </slot>
        </th>
      </tr>
    </thead>

    <tbody>
      <tr v-for="(row, rowIndex) in data" :key="rowIndex">
        <td v-for="column in columns" :key="column.key">
          <slot :name="column.key" :row="row" :index="rowIndex">
            {{ row[column.key] }}
          </slot>
        </td>
      </tr>
    </tbody>
  </table>
</template>

使用方式：

vue
<DataTable :columns="columns" :data="userData">
  <template #header="{ column }">
    <strong>{{ column.title.toUpperCase() }}</strong>
  </template>
  
  <template #email="{ row }">
    <a :href="`mailto:${row.email}`">{{ row.email }}</a>
  </template>
  
  <template #status="{ row }">
    <StatusBadge :type="row.status" />
  </template>
</DataTable>

3. 空状态插槽 ​

为空数据状态提供自定义内容：

vue
<!-- List.vue -->
<template>
  <div class="list-container">
    <template v-if="items.length">
      <div v-for="(item, index) in items" :key="item.id" class="list-item">
        <slot :item="item" :index="index"></slot>
      </div>
    </template>
    <template v-else>
      <div class="empty-state">
        <slot name="empty">
          <p>暂无数据</p>
        </slot>
      </div>
    </template>
  </div>
</template>

插槽的性能考虑 ​

插槽内容是在父组件的环境中编译的，但在子组件中渲染。这种机制有一些性能影响：

  1. 懒渲染：对于条件渲染的插槽，如果条件为假，插槽内容不会被渲染
  2. 组件更新：当父组件更新时，插槽内容可能需要重新渲染

优化策略：

vue
<!-- 使用 v-once 对不需要更新的插槽内容进行优化 -->
<template v-once #static-content>
  <div class="static-content">这部分内容只渲染一次</div>
</template>

<!-- 对于大型插槽内容，可以使用组件封装 -->
<template #large-content>
  <LargeContentComponent />
</template>

最佳实践 ​

1. 为插槽提供默认内容 ​

始终为关键插槽提供合理的默认内容，使组件在没有传入内容时也能正常工作：

vue
<template>
  <div class="alert">
    <div class="alert-icon">
      <slot name="icon">
        <InfoIcon />
        <!-- 默认图标 -->
      </slot>
    </div>

    <div class="alert-content">
      <slot>
        <p>默认提示信息</p>
        <!-- 默认内容 -->
      </slot>
    </div>
  </div>
</template>

2. 在文档中明确描述插槽 ​

在组件中使用注释或文档清晰说明每个插槽的用途和可用的作用域属性：

vue
<template>
  <!-- 
    头部插槽
    @slot header - 自定义头部内容
    @binding {number} itemCount - 项目总数
  -->
  <slot name="header" :itemCount="items.length">
    <h2>项目列表 ({{ items.length }})</h2>
  </slot>
</template>

3. 合理规划插槽的粒度 ​

  • 过少插槽：限制了组件的灵活性
  • 过多插槽：增加了组件的复杂性和使用难度

找到合适的平衡点，为需要定制的区域提供插槽，其他部分使用 props 配置。

4. 命名一致性 ​

在整个应用或组件库中保持插槽命名的一致性：

vue
<!-- 所有列表组件使用相同的插槽名命名约定 -->
<template>
  <div class="list">
    <div class="list-header">
      <slot name="header"></slot>
    </div>

    <div class="list-body">
      <slot name="item" v-for="item in items" :item="item"></slot>
    </div>

    <div class="list-footer">
      <slot name="footer"></slot>
    </div>
  </div>
</template>

5. 避免过度依赖作用域插槽 ​

作用域插槽功能强大，但也会增加复杂性和降低可读性。只在真正需要访问子组件数据的情况下使用作用域插槽。

总结 ​

插槽是 Vue 组件系统中实现内容分发的核心机制，它允许组件更加灵活和可复用。主要内容包括：

  1. 基本插槽：在组件中预留内容位置
  2. 具名插槽：通过命名来区分多个插槽位置
  3. 作用域插槽：允许子组件向插槽内容传递数据
  4. 高级技巧：动态插槽名、插槽传递等
  5. 最佳实践：默认内容、文档、命名一致性等

合理使用插槽可以大幅提高组件的可复用性和灵活性，同时保持组件的易用性和可维护性。
