uView Plus 实践指南 ​

本文档提供了uView Plus的实践指南，包括从uView UI迁移到uView Plus的步骤、常见问题解决方案以及最佳实践。

从uView UI迁移到uView Plus ​

如果你正在使用uView UI（Vue2版本）并计划迁移到uView Plus（Vue3版本），以下是迁移步骤和注意事项：

1. 依赖包替换 ​

首先需要卸载uView UI并安装uView Plus：

bash
# 卸载uView UI
npm uninstall uview-ui
# 或
yarn remove uview-ui
# 或
pnpm remove uview-ui

# 安装uView Plus
npm install uview-plus
# 或
yarn add uview-plus
# 或
pnpm add uview-plus

2. 配置文件更新 ​

main.js 更新 ​

js
// 旧版 (uView UI)
import uView from 'uview-ui'
Vue.use(uView)

// 新版 (uView Plus)
import { createSSRApp } from 'vue'
import App from './App'
import uviewPlus from 'uview-plus'

export function createApp() {
  const app = createSSRApp(App)
  app.use(uviewPlus)
  return { app }
}

App.vue 样式导入更新 ​

vue
<!-- 旧版 (uView UI) -->
<style lang="scss">
@import "uview-ui/index.scss";
</style>

<!-- 新版 (uView Plus) -->
<style lang="scss">
@import "uview-plus/index.scss";
</style>

uni.scss 主题变量导入更新 ​

scss
/* 旧版 (uView UI) */
@import "uview-ui/theme.scss";

/* 新版 (uView Plus) */
@import "uview-plus/theme.scss";

pages.json easycom配置更新 ​

json
// 旧版 (uView UI)
{
  "easycom": {
    "autoscan": true,
    "custom": {
      "^u-(.*)": "uview-ui/components/u-$1/u-$1.vue"
    }
  }
}

// 新版 (uView Plus)
{
  "easycom": {
    "autoscan": true,
    "custom": {
      "^u-(.*)": "uview-plus/components/u-$1/u-$1.vue"
    }
  }
}

3. Vue2到Vue3的语法迁移 ​

除了更换组件库外，还需要将Vue2的语法迁移到Vue3：

选项式API到组合式API ​

vue
<!-- 旧版 (Vue2 选项式API) -->
<script>
export default {
  data() {
    return {
      keyword: ''
    }
  },
  methods: {
    onSearch() {
      // 处理搜索
    }
  }
}
</script>

<!-- 新版 (Vue3 组合式API) -->
<script setup>
import { ref } from 'vue'

const keyword = ref('')
const onSearch = () => {
  // 处理搜索
}
</script>

生命周期钩子变更 ​

js
// Vue2 生命周期
beforeCreate
created
beforeMount
mounted
beforeUpdate
updated
beforeDestroy
destroyed

// Vue3 生命周期
// beforeCreate和created被setup()替代
onBeforeMount
onMounted
onBeforeUpdate
onUpdated
onBeforeUnmount // 替代beforeDestroy
onUnmounted // 替代destroyed

常见问题解决方案 ​

1. Vue.filter is not a function ​

问题：

Vue.filter is not a function

原因： Vue3中移除了Vue.filter API

解决方案： 使用全局方法或本地计算属性替代过滤器

2. Sass变量未定义问题 ​

问题：

Undefined variable: $u-border-color

原因： 缺少uView Plus主题变量的正确导入

解决方案： 确保在uni.scss中正确导入theme.scss

scss
@import 'uview-plus/theme.scss';

3. 组件解析失败问题 ​

问题：

Failed to resolve component: u-search

原因： easycom配置错误，组件无法自动导入

解决方案： 检查pages.json中的easycom配置

json
{
  "easycom": {
    "autoscan": true,
    "custom": {
      "^u-(.*)": "uview-plus/components/u-$1/u-$1.vue"
    }
  }
}

4. 特定组件缺失问题 ​

问题： 某些组件在uView Plus中不存在或实现方式不同

解决方案： 自定义实现缺失的组件，例如u-section组件：

vue
<template>
  <view class="section-header">
    <text class="section-title">{{ title }}</text>
    <view class="section-more" @click="handleMore" v-if="showMore">
      <text class="section-right">{{ moreText }}</text>
      <u-icon name="arrow-right" size="28rpx"></u-icon>
    </view>
  </view>
</template>

<script setup>
import { defineProps, defineEmits } from 'vue'

const props = defineProps({
  title: {
    type: String,
    default: '标题'
  },
  showMore: {
    type: Boolean,
    default: true
  },
  moreText: {
    type: String,
    default: '更多'
  }
})

const emit = defineEmits(['more'])

const handleMore = () => {
  emit('more')
}
</script>

<style lang="scss">
.section-header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  margin-bottom: 20rpx;
}

.section-title {
  font-size: 32rpx;
  font-weight: bold;
  color: #333;
}

.section-more {
  display: flex;
  align-items: center;
  cursor: pointer;
}

.section-right {
  font-size: 24rpx;
  color: #909399;
  margin-right: 6rpx;
}
</style>

5. Sass警告问题 ​

问题： 使用uView Plus时可能会遇到以下Sass相关警告：

Deprecation Warning [legacy-js-api]: The legacy JS API is deprecated and will be removed in Dart Sass 2.0.0.
More info: https://sass-lang.com/d/legacy-js-api

Deprecation Warning [import]: Sass @import rules are deprecated and will be removed in Dart Sass 3.0.0.
More info and automated migrator: https://sass-lang.com/d/import

   ╷
16 │ @import 'uview-plus/theme.scss';
   │         ^^^^^^^^^^^^^^^^^^^^^^^
   ╵
    node_modules/uview-plus/components/u-loading-icon/u-loading-icon.vue 16:9  root stylesheet

原因：

  1. uView Plus使用了即将在Dart Sass 2.0.0中移除的旧版JS API
  2. uView Plus使用了@import规则，该规则将在Dart Sass 3.0.0中被移除，推荐使用@use和@forward替代

解决方案：

  1. 短期解决方案：这些只是警告，不会影响功能，可以暂时忽略
  2. 中期解决方案：降级Sass版本
    bash
    npm install sass@1.32.13 --save-dev
# 或
yarn add sass@1.32.13 --dev
# 或
pnpm add sass@1.32.13 -D
  3. 长期解决方案：创建自定义样式文件，使用@use替代@import
    scss
    /* 创建custom-theme.scss */
@use 'uview-plus/theme' as *;
    然后在uni.scss中引入：
    scss
    @use './custom-theme';

注意：长期解决方案可能需要更多的自定义工作，因为uView Plus内部的样式文件也使用了@import语法。如果您的项目对这些警告特别敏感，可能需要考虑fork一份uView Plus代码并进行现代化改造。

6. 组件前缀警告问题 ​

问题：

[Vue warn]: Failed to resolve component: up-loading-icon

原因： uView Plus内部存在组件命名转换机制，将组件名从u-前缀转换为up-前缀，但在某些情况下会出现解析错误。

解决方案：

  1. 使用正确的前缀：始终使用u-前缀，而不是up-前缀

    vue
    <!-- 错误用法 -->
<up-loading-icon></up-loading-icon>

<!-- 正确用法 -->
<u-loading-icon></u-loading-icon>

  2. 抑制警告：在main.js中添加代码，抑制特定的组件解析警告

    js
    // main.js
export function createApp() {
  const app = createSSRApp(App);
  
  app.use(uviewPlus);
  
  // 修复组件前缀警告问题
  // 过滤掉up-前缀相关的警告
  app.config.warnHandler = (msg, instance, trace) => {
    if (msg.includes('Failed to resolve component: up-')) {
      return
    }
    console.warn(msg, instance, trace)
  }
  
  return { app };
}

  3. 更新easycom配置：确保pages.json中的easycom配置使用了正确的组件路径

    json
    {
  "easycom": {
    "autoscan": true,
    "custom": {
      "^u-(.*)": "uview-plus/components/u-$1/u-$1.vue"
    }
  }
}

注意：这个警告通常不会影响功能，但为了避免控制台警告，建议使用上述解决方案。第2种方法（抑制警告）是最有效的，因为它不需要修改uView Plus的源代码。

调试技巧 ​

1. 组件测试方法 ​

逐步测试组件可用性：

vue
<template>
  <view>
    <!-- 1. 先测试基础组件 -->
    <u-button type="primary" text="测试"></u-button>
    
    <!-- 2. 再添加复杂组件 -->
    <u-search v-model="keyword"></u-search>
  </view>
</template>

2. easycom路径调试 ​

在控制台检查组件路径：

js
console.log(require.resolve('uview-plus/components/u-button/u-button.vue'))

3. 手动导入备用方案 ​

如果easycom失败，可以手动导入：

vue
<script setup>
// 手动导入组件
import UButton from 'uview-plus/components/u-button/u-button.vue'
</script>

<template>
  <UButton text="按钮"></UButton>
</template>

最佳实践 ​

  1. 版本选择：Vue3项目必须使用uview-plus，不要使用uview-ui
  2. 配置顺序：先配置样式导入，再配置组件注册，最后配置easycom
  3. 调试策略：遇到问题先简化代码，确认基础功能正常后再逐步添加
  4. 组件检查：使用前先检查组件文件是否存在，避免使用缺失的组件
  5. 备用方案：对于关键组件，准备手动导入的备用方案
  6. 类型检查：利用TypeScript的类型检查功能，确保组件属性使用正确
  7. 按需引入：对于较大的项目，考虑按需引入组件以减小打包体积
  8. 自定义主题：根据项目需求自定义主题变量，保持UI风格一致性

总结 ​

从uView UI迁移到uView Plus需要注意依赖包替换、配置文件更新以及Vue2到Vue3的语法迁移。在使用过程中，可能会遇到各种问题，如组件解析失败、特定组件缺失等，可以通过本文提供的解决方案进行处理。

通过遵循最佳实践和使用调试技巧，可以更顺利地使用uView Plus开发高质量的UniApp Vue3应用。
