React 18 新特性 ​

并发特性 ​

Concurrent Rendering ​

React 18 引入了并发渲染，允许 React 准备多个版本的 UI，并选择最优的版本进行渲染。

jsx
import { startTransition } from 'react'

function SearchResults() {
  const [query, setQuery] = useState('')
  const [results, setResults] = useState([])

  const handleChange = (e) => {
    // 紧急更新：立即执行
    setQuery(e.target.value)

    // 过渡更新：可以被打断
    startTransition(() => {
      setResults(searchResults(e.target.value))
    })
  }

  return (
    <div>
      <input
        type="text"
        value={query}
        onChange={handleChange}
        placeholder="搜索..."
      />
      <ResultsList results={results} />
    </div>
  )
}

useTransition Hook ​

useTransition Hook 用于标记非紧急的状态更新。

jsx
import { useTransition } from 'react'

function TabContainer() {
  const [isPending, startTransition] = useTransition()
  const [tab, setTab] = useState('about')

  function selectTab(nextTab) {
    startTransition(() => {
      setTab(nextTab)
    })
  }

  return (
    <>
      <TabButton
        isActive={tab === 'about'}
        onClick={() => selectTab('about')}
      >
        关于
      </TabButton>
      <TabButton
        isActive={tab === 'posts'}
        onClick={() => selectTab('posts')}
      >
        文章
      </TabButton>
      <TabButton
        isActive={tab === 'contact'}
        onClick={() => selectTab('contact')}
      >
        联系
      </TabButton>
      <hr />
      {isPending && <Spinner />}
      <Suspense fallback={<Spinner />}>
        <TabContent tab={tab} />
      </Suspense>
    </>
  )
}

useDeferredValue Hook ​

useDeferredValue 用于延迟更新某些值，提高性能。

jsx
import { useDeferredValue } from 'react'

function SearchResults({ query }) {
  const deferredQuery = useDeferredValue(query)

  return (
    <div>
      <input value={query} onChange={e => setQuery(e.target.value)} />
      <ResultsList query={deferredQuery} />
    </div>
  )
}

function ProfilePage() {

新的 Hooks ​

useId Hook ​

useId 用于生成唯一的 ID，特别适用于无障碍访问。

jsx
import { useId } from 'react'

function PasswordFields() {
  const passwordHintId = useId()

  return (
    <>
      <input
        type="password"
        aria-describedby={passwordHintId}
      />
      <p id={passwordHintId}>
        密码应该包含至少 8 个字符
      </p>
    </>
  )
}

useSyncExternalStore Hook ​

useSyncExternalStore 用于订阅外部数据源。

jsx
import { useSyncExternalStore } from 'react'

function useOnlineStatus() {
  const isOnline = useSyncExternalStore(subscribe, getSnapshot)

  return isOnline
}

function subscribe(callback) {
  window.addEventListener('online', callback)
  window.addEventListener('offline', callback)
  return () => {
    window.removeEventListener('online', callback)
    window.removeEventListener('offline', callback)
  }
}

function getSnapshot() {
  return navigator.onLine
}

function StatusBar() {
  const isOnline = useOnlineStatus()

  return <h1>{isOnline ? '✅ 在线' : '❌ 离线'}</h1>
}

useInsertionEffect Hook ​

useInsertionEffect 用于在 DOM 变更前执行副作用。

jsx
import { useInsertionEffect } from 'react'

function useCSS(rule) {
  useInsertionEffect(() => {
    const style = document.createElement('style')
    style.textContent = rule
    document.head.appendChild(style)

    return () => {
      document.head.removeChild(style)
    }
  }, [rule])
}

function StyledComponent() {
  useCSS(`
    .styled {
      background-color: #007bff;
      color: white;
      padding: 10px 20px;
    }
  `)

  return <div className="styled">样式组件</div>
}

自动批处理 ​

自动批处理示例 ​

React 18 自动批处理更多的状态更新，减少渲染次数。

jsx
function handleClick() {
  // React 18: 这些更新会被批处理为一次重新渲染
  setCount(c => c + 1)
  setFlag(f => !f)
  // React 17: 这些更新会被批处理为两次重新渲染
}

在异步操作中自动批处理 ​

jsx
function fetchData() {
  fetch('/api/data').then(data => {
    // React 18: 这些更新会被批处理
    setData(data)
    setLoading(false)
  })
}

禁用自动批处理 ​

jsx
import { flushSync } from 'react-dom'

function handleClick() {
  // 立即更新，不批处理
  flushSync(() => {
    setCount(c => c + 1)
  })
  // 立即更新，不批处理
  flushSync(() => {
    setFlag(f => !f)
  })
}

严格模式变化 ​

双重调用组件 ​

React 18 的严格模式会在开发模式下双重调用组件，以检测副作用。

jsx
function UserProfile({ userId }) {
  const [user, setUser] = useState(null)

  useEffect(() => {
    let ignore = false

    async function fetchUser() {
      const data = await fetchUser(userId)
      if (!ignore) {
        setUser(data)
      }
    }

    fetchUser()

    return () => {
      ignore = true
    }
  }, [userId])

  if (!user) {
    return <div>加载中...</div>
  }

  return <div>{user.name}</div>
}

检测不安全的副作用 ​

jsx
function Component() {
  const [count, setCount] = useState(0)

  // 不安全的副作用：在渲染期间修改状态
  // React 18 严格模式会检测到这个问题
  if (count > 0) {
    setCount(0)
  }

  return <div>Count: {count}</div>
}

新的客户端 API ​

createRoot API ​

React 18 使用新的 createRoot API 替代 render。

jsx
// React 17
import { render } from 'react-dom'
import App from './App'

const container = document.getElementById('root')
render(<App />, container)

// React 18
import { createRoot } from 'react-dom/client'
import App from './App'

const container = document.getElementById('root')
const root = createRoot(container)
root.render(<App />)

hydrateRoot API ​

React 18 使用新的 hydrateRoot API 替代 hydrate。

jsx
// React 17
import { hydrate } from 'react-dom'
import App from './App'

hydrate(<App />, document.getElementById('root'))

// React 18
import { hydrateRoot } from 'react-dom/client'
import App from './App'

hydrateRoot(document.getElementById('root'), <App />)

renderToPipeableStream API ​

React 18 支持流式渲染，用于服务端渲染。

jsx
import { renderToPipeableStream } from 'react-dom/server'

app.get('/', (req, res) => {
  const { pipe } = renderToPipeableStream(
    <App />,
    {
      bootstrapScripts: ['/client.js'],
      onShellReady() {
        res.statusCode = 200
        res.setHeader('Content-type', 'text/html')
        pipe(res)
      },
      onShellError(error) {
        res.statusCode = 500
        res.send('<!doctype html><p>Error loading app...</p>')
      }
    }
  )
})

React 18 面试题 ​

1. React 18 有哪些新特性？ ​

答案：

React 18 的新特性：

  1. 并发特性:

    • Concurrent Rendering
    • startTransition API
    • useTransition Hook
    • useDeferredValue Hook

  2. Suspense 改进:

    • 支持服务端渲染
    • 支持数据获取
    • 支持列表渲染

  3. 新的 Hooks:

    • useId: 生成唯一 ID
    • useSyncExternalStore: 订阅外部数据源
    • useInsertionEffect: 在 DOM 变更前执行副作用

  4. 自动批处理:

    • 自动批处理更多的状态更新
    • 在异步操作中自动批处理

  5. 严格模式变化:

    • 双重调用组件
    • 检测不安全的副作用

  6. 新的客户端 API:

    • createRoot API
    • hydrateRoot API
    • renderToPipeableStream API

2. 什么是并发渲染？ ​

答案：

并发渲染是 React 18 引入的新特性：

  1. 概念:

    • 允许 React 准备多个版本的 UI
    • 可以被打断和恢复
    • 优先处理高优先级的更新

  2. 优势:

    • 提高应用性能
    • 改善用户体验
    • 避免界面卡顿

  3. 使用场景:

    • 搜索输入
    • 列表过滤
    • 大数据渲染

  4. 示例:

    jsx
    import { startTransition } from 'react'

function handleChange(e) {
  // 紧急更新：立即执行
  setInput(e.target.value)

  // 过渡更新：可以被打断
  startTransition(() => {
    setResults(search(e.target.value))
  })
}

3. useTransition 和 useDeferredValue 的区别是什么？ ​

答案：

主要区别：

  1. 用途:

    • useTransition: 标记非紧急的状态更新
    • useDeferredValue: 延迟更新某些值

  2. 使用场景:

    • useTransition: 适用于状态更新
    • useDeferredValue: 适用于派生值

  3. 示例:

    jsx
    // useTransition
const [isPending, startTransition] = useTransition()

function handleChange(e) {
  startTransition(() => {
    setResults(search(e.target.value))
  })
}

// useDeferredValue
const deferredQuery = useDeferredValue(query)

return <ResultsList query={deferredQuery} />

4. React 18 的自动批处理是如何工作的？ ​

答案：

自动批处理的工作原理：

  1. 概念:

    • React 18 自动批处理更多的状态更新
    • 减少不必要的渲染次数

  2. 批处理场景:

    jsx
    function handleClick() {
  // 这些更新会被批处理为一次重新渲染
  setCount(c => c + 1)
  setFlag(f => !f)
}

function fetchData() {
  fetch('/api/data').then(data => {
    // 这些更新也会被批处理
    setData(data)
    setLoading(false)
  })
}

  3. 禁用自动批处理:

    jsx
    import { flushSync } from 'react-dom'

function handleClick() {
  flushSync(() => {
    setCount(c => c + 1)
  })
  flushSync(() => {
    setFlag(f => !f)
  })
}

5. React 18 的严格模式有什么变化？ ​

答案：

严格模式的变化：

  1. 双重调用组件:

    • 开发模式下双重调用组件
    • 用于检测副作用

  2. 检测不安全的副作用:

    jsx
    function Component() {
  useEffect(() => {
    // 不安全的副作用
    document.title = 'Title'
  })

  return <div>Component</div>
}

  3. 正确处理副作用:

    jsx
    function Component() {
  useEffect(() => {

      if (!ignore) {
        setData(data)
      }
    }

   fetchData()

    return () => {
      ignore = true
    }
  }, [])

  return <div>Component</div>
}
