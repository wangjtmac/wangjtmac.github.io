Hooks 基础 ​

1. 什么是 Hooks？ Hooks 是 React 16.8 引入的新特性，允许你在不编写类组件的情况下使用 state 和其他 React 特性。

2. Hooks 的优势？

  • 简化代码：避免类组件的复杂性
  • 逻辑复用：通过自定义 Hooks 复用逻辑
  • 更好的组织：将相关逻辑组织在一起
  • 减少样板代码：减少类组件的样板代码

3. Hooks 的使用规则？

  • 只在函数组件中使用：Hooks 只能在函数组件或自定义 Hooks 中使用
  • 只在顶层调用：不要在循环、条件或嵌套函数中调用 Hooks
  • 自定义 Hooks：自定义 Hooks 必须以 "use" 开头

4. 为什么需要 Hooks？

  • 类组件的复杂性：类组件的 this 绑定和生命周期方法复杂
  • 逻辑复用困难：在类组件中复用逻辑困难
  • 组件拆分问题：相关逻辑被迫拆分到不同的生命周期方法中

useState Hook ​

1. useState 的基本用法？

jsx
import { useState } from 'react'

function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  )
}

2. useState 的初始值？

jsx
// 基本类型
const [count, setCount] = useState(0)
const [name, setName] = useState('John')
const [isActive, setIsActive] = useState(false)

// 对象
const [user, setUser] = useState({ name: 'John', age: 30 })

// 数组
const [items, setItems] = useState([1, 2, 3])

// 惰性初始化
const [state, setState] = useState(() => {
  const initialState = someExpensiveComputation(props)
  return initialState
})

3. useState 的更新方式？

jsx
// 直接更新
const [count, setCount] = useState(0)
setCount(count + 1)

// 函数式更新
setCount(prevCount => prevCount + 1)

// 对象更新
const [user, setUser] = useState({ name: 'John', age: 30 })
setUser({ ...user, age: user.age + 1 })

// 数组更新
const [items, setItems] = useState([1, 2, 3])
setItems([...items, 4])
setItems(items.filter(item => item !== 2))

4. useState 的常见问题？

jsx
// 错误：直接修改 state
const [items, setItems] = useState([1, 2, 3])
items.push(4)

// 正确：创建新数组
const [items, setItems] = useState([1, 2, 3])
setItems([...items, 4])

// 错误：基于旧 state 更新
const [count, setCount] = useState(0)
setCount(count + 1)
setCount(count + 1)

// 正确：使用函数式更新
const [count, setCount] = useState(0)
setCount(prevCount => prevCount + 1)
setCount(prevCount => prevCount + 1)

useEffect Hook ​

1. useEffect 的基本用法？

jsx
import { useState, useEffect } from 'react'

function Example() {
  const [count, setCount] = useState(0)

  useEffect(() => {
    document.title = `You clicked ${count} times`
  })

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  )
}

2. useEffect 的依赖数组？

jsx
// 每次渲染后执行
useEffect(() => {
  console.log('Rendered')
})

// 只在组件挂载时执行
useEffect(() => {
  console.log('Mounted')
}, [])

// 当 count 变化时执行
useEffect(() => {
  console.log('Count changed:', count)
}, [count])

// 当 count 或 name 变化时执行
useEffect(() => {
  console.log('Count or name changed')
}, [count, name])

3. useEffect 的清理函数？

jsx
useEffect(() => {
  const subscription = props.source.subscribe()
  return () => {
    subscription.unsubscribe()
  }
}, [props.source])

4. useEffect 的常见使用场景？

jsx
// 数据获取
useEffect(() => {
  const fetchData = async () => {
    const response = await fetch('/api/data')
    const data = await response.json()
    setData(data)
  }
  fetchData()
}, [])

// 订阅
useEffect(() => {
  const subscription = props.source.subscribe()
  return () => {
    subscription.unsubscribe()
  }
}, [props.source])

// 定时器
useEffect(() => {
  const timer = setInterval(() => {
    console.log('Timer')
  }, 1000)
  return () => {
    clearInterval(timer)
  }
}, [])

// DOM 操作
useEffect(() => {
  const element = document.getElementById('my-element')
  if (element) {
    element.style.color = 'red'
  }
}, [])

5. useEffect 的常见问题？

jsx
// 错误：忘记依赖数组
useEffect(() => {
  console.log(count)
})

// 正确：添加依赖数组
useEffect(() => {
  console.log(count)
}, [count])

// 错误：依赖数组不完整
useEffect(() => {
  console.log(count, name)
}, [count])

// 正确：添加所有依赖
useEffect(() => {
  console.log(count, name)
}, [count, name])

// 错误：无限循环
useEffect(() => {
  setCount(count + 1)
}, [count])

// 正确：避免无限循环
useEffect(() => {
  const timer = setInterval(() => {
    setCount(prevCount => prevCount + 1)
  }, 1000)
  return () => {
    clearInterval(timer)
  }
}, [])

useContext Hook ​

1. useContext 的基本用法？

jsx
import { useContext } from 'react'

const ThemeContext = React.createContext('light')

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Toolbar />
    </ThemeContext.Provider>
  )
}

function Toolbar() {
  return (
    <div>
      <ThemedButton />
    </div>
  )
}

function ThemedButton() {
  const theme = useContext(ThemeContext)
  return <button className={theme}>I am styled by theme context!</button>
}

2. useContext 的优势？

  • 简化代码：避免使用 Consumer 组件
  • 更易读：代码更简洁易读
  • 减少嵌套：避免多层嵌套的 Consumer

3. useContext 的使用场景？

  • 主题切换：全局主题管理
  • 用户信息：全局用户信息
  • 国际化：全局语言设置
  • 配置管理：全局配置管理

4. useContext 的注意事项？

  • 性能优化：Context 值变化时，所有消费者都会重新渲染
  • 避免过度使用：不要将所有状态都放在 Context 中
  • 拆分 Context：将不同类型的 Context 拆分开

useReducer Hook ​

1. useReducer 的基本用法？

jsx
import { useReducer } from 'react'

const initialState = { count: 0 }

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 }
    case 'decrement':
      return { count: state.count - 1 }
    default:
      throw new Error()
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState)

  return (
    <div>
      Count: {state.count}
      <button onClick={() => dispatch({ type: 'decrement' })}>-</button>
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>
    </div>
  )
}

2. useReducer 与 useState 的区别？

  • 复杂状态：useReducer 适合复杂的状态逻辑
  • 下一个状态依赖前一个状态：useReducer 更适合这种情况
  • 可测试：reducer 函数更容易测试
  • 可预测：reducer 是纯函数，状态变化可预测

3. useReducer 的初始化？

jsx
// 惰性初始化
const [state, dispatch] = useReducer(reducer, initialArg, init)

function init(initialArg) {
  return { value: initialArg }
}

function Counter({ initialCount }) {
  const [state, dispatch] = useReducer(reducer, initialCount, init)
}

4. useReducer 的使用场景？

  • 复杂状态逻辑：状态更新逻辑复杂
  • 多个子值：状态包含多个子值
  • 下一个状态依赖前一个状态：状态更新依赖前一个状态

useCallback Hook ​

1. useCallback 的基本用法？

jsx
import { useCallback } from 'react'

function Parent() {
  const [count, setCount] = useState(0)

  const handleClick = useCallback(() => {
    setCount(count + 1)
  }, [count])

  return <Child onClick={handleClick} />
}

2. useCallback 的作用？

  • 性能优化：避免在每次渲染时创建新函数
  • 减少子组件渲染：避免子组件不必要的重新渲染
  • 依赖追踪：只有依赖变化时才创建新函数

3. useCallback 的使用场景？

jsx
// 传递给子组件
function Parent() {
  const handleClick = useCallback(() => {
    console.log('Clicked')
  }, [])

  return <Child onClick={handleClick} />
}

// 作为 useEffect 的依赖
function Component() {
  const fetchData = useCallback(async () => {
    const response = await fetch('/api/data')
    const data = await response.json()
    setData(data)
  }, [])

  useEffect(() => {
    fetchData()
  }, [fetchData])
}

4. useCallback 的注意事项？

  • 不要过度使用：只有在性能问题时才使用
  • 正确的依赖：确保依赖数组正确
  • 与 React.memo 配合：与 React.memo 配合使用效果更好

useMemo Hook ​

1. useMemo 的基本用法？

jsx
import { useMemo } from 'react'

function ExpensiveComponent({ a, b }) {
  const result = useMemo(() => {
    return expensiveCalculation(a, b)
  }, [a, b])

  return <div>Result: {result}</div>
}

2. useMemo 的作用？

  • 性能优化：避免在每次渲染时重新计算
  • 缓存结果：只有依赖变化时才重新计算
  • 减少计算开销：减少昂贵的计算

3. useMemo 的使用场景？

jsx
// 昂贵的计算
function Component({ data }) {
  const sortedData = useMemo(() => {
    return data.sort((a, b) => a.value - b.value)
  }, [data])

  return <ul>{sortedData.map(item => <li key={item.id}>{item.name}</li>)}</ul>
}

// 复杂的对象
function Component({ items }) {
  const total = useMemo(() => {
    return items.reduce((sum, item) => sum + item.price, 0)
  }, [items])

  return <div>Total: {total}</div>
}

4. useMemo 的注意事项？

  • 不要过度使用：只有在性能问题时才使用
  • 正确的依赖：确保依赖数组正确
  • 与 useCallback 配合：与 useCallback 配合使用效果更好

useRef Hook ​

1. useRef 的基本用法？

jsx
import { useRef } from 'react'

function TextInputWithFocusButton() {
  const inputEl = useRef(null)

  const onButtonClick = () => {
    inputEl.current.focus()
  }

  return (
    <>
      <input ref={inputEl} type="text" />
      <button onClick={onButtonClick}>Focus the input</button>
    </>
  )
}

2. useRef 的作用？

  • 访问 DOM：访问 DOM 元素或组件实例
  • 保存可变值：保存不会触发重新渲染的可变值
  • 定时器：保存定时器 ID
  • 上一次的值：保存上一次的 props 或 state

3. useRef 的使用场景？

jsx
// 访问 DOM
function Component() {
  const inputRef = useRef(null)

  const handleClick = () => {
    inputRef.current.focus()
  }

  return <input ref={inputRef} />
}

// 保存可变值
function Component() {
  const timerRef = useRef(null)

  const startTimer = () => {
    timerRef.current = setInterval(() => {
      console.log('Timer')
    }, 1000)
  }

  const stopTimer = () => {
    clearInterval(timerRef.current)
  }

  return (
    <div>
      <button onClick={startTimer}>Start</button>
      <button onClick={stopTimer}>Stop</button>
    </div>
  )
}

// 保存上一次的值
function Component({ value }) {
  const prevValueRef = useRef()

  useEffect(() => {
    prevValueRef.current = value
  }, [value])

  const prevValue = prevValueRef.current

  return <div>Current: {value}, Previous: {prevValue}</div>
}

自定义 Hooks ​

1. 什么是自定义 Hooks？ 自定义 Hooks 是一个函数，其名称以 "use" 开头，可以调用其他 Hooks。

2. 自定义 Hooks 的基本用法？

jsx
function useWindowSize() {
  const [size, setSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight
  })

  useEffect(() => {
    const handleResize = () => {
      setSize({
        width: window.innerWidth,
        height: window.innerHeight
      })
    }

    window.addEventListener('resize', handleResize)
    return () => {
      window.removeEventListener('resize', handleResize)
    }
  }, [])

  return size
}

function Component() {
  const size = useWindowSize()

  return (
    <div>
      Width: {size.width}, Height: {size.height}
    </div>
  )
}

3. 自定义 Hooks 的优势？

  • 逻辑复用：可以在多个组件中复用逻辑
  • 代码组织：将相关逻辑组织在一起
  • 测试友好：自定义 Hooks 易于测试

4. 自定义 Hooks 的常见模式？

jsx
// 数据获取
function useFetch(url) {
  const [data, setData] = useState(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)

  useEffect(() => {
    const fetchData = async () => {
      try {
        setLoading(true)
        const response = await fetch(url)
        const data = await response.json()
        setData(data)
      } catch (err) {
        setError(err)
      } finally {
        setLoading(false)
      }
    }
    fetchData()
  }, [url])

  return { data, loading, error }
}

// 表单处理
function useForm(initialValues) {
  const [values, setValues] = useState(initialValues)

  const handleChange = (event) => {
    const { name, value } = event.target
    setValues({
      ...values,
      [name]: value
    })
  }

  const handleSubmit = (callback) => (event) => {
    event.preventDefault()
    callback(values)
  }

  return { values, handleChange, handleSubmit }
}

// 本地存储
function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key)
      return item ? JSON.parse(item) : initialValue
    } catch (error) {
      return initialValue
    }
  })

  const setValue = (value) => {
    try {
      setStoredValue(value)
      window.localStorage.setItem(key, JSON.stringify(value))
    } catch (error) {
      console.error(error)
    }
  }

  return [storedValue, setValue]
}

Hooks 的最佳实践 ​

1. Hooks 的使用原则？

  • 只在顶层调用：不要在循环、条件或嵌套函数中调用 Hooks
  • 只在函数组件中使用：Hooks 只能在函数组件或自定义 Hooks 中使用
  • 自定义 Hooks 以 use 开头：自定义 Hooks 必须以 "use" 开头

2. Hooks 的性能优化？

  • 使用 useCallback：避免在每次渲染时创建新函数
  • 使用 useMemo：避免在每次渲染时重新计算
  • 使用 React.memo：避免子组件不必要的重新渲染
  • 正确的依赖：确保依赖数组正确

3. Hooks 的常见错误？

jsx
// 错误：在条件中调用 Hooks
function Component({ condition }) {
  if (condition) {
    const [count, setCount] = useState(0)
  }
}

// 正确：始终调用 Hooks
function Component({ condition }) {
  const [count, setCount] = useState(0)

  if (condition) {
    return <div>{count}</div>
  }
  return null
}

// 错误：在循环中调用 Hooks
function Component({ items }) {
  items.forEach(item => {
    const [value, setValue] = useState(item)
  })
}

// 正确：使用数组或对象
function Component({ items }) {
  const [values, setValues] = useState(items)
}

4. Hooks 的调试技巧？

  • 使用 React DevTools：React DevTools 可以查看 Hooks 的状态
  • 使用 eslint-plugin-react-hooks：检查 Hooks 的使用是否正确
  • 使用 console.log：在 Hooks 中使用 console.log 调试
  • 使用自定义 Hooks：创建自定义 Hooks 用于调试
