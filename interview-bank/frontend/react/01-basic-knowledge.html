React 基础知识 ​

React 的基础知识相关面试题。

React 的基本概念 ​

1. 什么是 React？ React 是由 Facebook 开发的一个用于构建用户界面的 JavaScript 库。它采用组件化的开发模式，通过虚拟 DOM 提高渲染性能，支持单向数据流。

2. React 的特点有哪些？

  • 组件化：将 UI 拆分为可复用的组件
  • 虚拟 DOM：通过虚拟 DOM 减少真实 DOM 操作，提高性能
  • 单向数据流：数据从父组件流向子组件
  • 声明式编程：通过声明式的代码描述 UI
  • JSX：使用类似 HTML 的语法编写组件
  • 跨平台：React Native 可以用于移动端开发

3. React 与 Vue 的区别？

  • 设计理念：React 是函数式编程思想，Vue 是响应式编程思想
  • 模板语法：React 使用 JSX，Vue 使用模板语法
  • 状态管理：React 使用 setState，Vue 使用响应式系统
  • 组件通信：React 使用 props 和回调，Vue 使用 props、events、provide/inject
  • 学习曲线：Vue 更容易上手，React 需要掌握更多概念
  • 生态：React 生态更丰富，Vue 生态更集中

4. React 与 Angular 的区别？

  • 复杂度：Angular 是完整的框架，React 是库
  • 学习曲线：Angular 学习曲线较陡，React 相对简单
  • 数据绑定：Angular 支持双向数据绑定，React 是单向数据流
  • 依赖注入：Angular 内置依赖注入，React 需要第三方库
  • 性能：React 虚拟 DOM 性能更好，Angular 使用变更检测

React 的核心概念 ​

1. 什么是 JSX？ JSX 是 JavaScript 的语法扩展，允许在 JavaScript 中编写类似 HTML 的代码。JSX 最终会被 Babel 编译为 React.createElement() 调用。

2. JSX 的特点？

  • 类名：使用 className 代替 class
  • 样式：使用 style= 对象语法
  • 注释：使用 {/* */} 语法
  • 表达式：使用 {} 包裹 JavaScript 表达式
  • 自闭合：自闭合标签必须以 /> 结尾

3. JSX 的基本用法？

jsx
// 元素
const element = <h1>Hello, world!</h1>

// 表达式
const name = 'John'
const element = <h1>Hello, {name}!</h1>

// 属性
const element = <img src={user.avatarUrl} alt={user.name} />

// 子元素
const element = (
  <div>
    <h1>Hello</h1>
    <p>World</p>
  </div>
)

4. 什么是虚拟 DOM？ 虚拟 DOM 是真实 DOM 的 JavaScript 对象表示。React 通过对比新旧虚拟 DOM 的差异，只更新需要变化的部分，从而提高渲染性能。

5. 虚拟 DOM 的优势？

  • 性能优化：减少真实 DOM 操作
  • 跨平台：可以在不同平台上渲染
  • 批量更新：可以批量更新 DOM
  • Diff 算法：高效的 Diff 算法减少更新

6. 什么是组件？ 组件是 React 中可复用的 UI 单元，接受 props 作为输入，返回 JSX 作为输出。

7. 组件的类型？

  • 函数组件：使用函数定义的组件
  • 类组件：使用 ES6 类定义的组件

8. 函数组件与类组件的区别？

  • 语法：函数组件使用函数，类组件使用类
  • 状态：类组件有 state，函数组件通过 Hooks 使用 state
  • 生命周期：类组件有生命周期方法，函数组件通过 Hooks 实现类似功能
  • 性能：函数组件性能更好，React 18 后推荐使用函数组件

9. 什么是 Props？ Props 是组件的输入，父组件通过 props 向子组件传递数据。

10. Props 的特点？

  • 只读：子组件不能修改 props
  • 单向数据流：数据从父组件流向子组件
  • 类型检查：可以使用 PropTypes 进行类型检查

11. 什么是 State？ State 是组件的内部状态，组件可以修改自己的 state。

12. State 的特点？

  • 可变：组件可以修改自己的 state
  • 私有：state 是组件私有的
  • 异步更新：setState 是异步的

React 的安装与使用 ​

1. 如何创建 React 项目？

  • Create React App：npx create-react-app my-app
  • Vite：npm create vite@latest my-app -- --template react
  • Next.js：npx create-next-app my-app

2. 如何在现有项目中使用 React？

html
<!DOCTYPE html>
<html>
  <head>
    <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
    <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
  </head>
  <body>
    <div id="root"></div>
    <script type="text/babel">
      const root = ReactDOM.createRoot(document.getElementById('root'))
      root.render(<h1>Hello, world!</h1>)
    </script>
  </body>
</html>

3. React 的渲染方式？

jsx
// React 18
import { createRoot } from 'react-dom/client'
import App from './App'

const root = createRoot(document.getElementById('root'))
root.render(<App />)

// React 17 及之前
import { render } from 'react-dom'
import App from './App'

render(<App />, document.getElementById('root'))

React 的事件处理 ​

1. React 事件的特点？

  • 合成事件：React 使用合成事件系统，封装了原生事件
  • 事件委托：React 使用事件委托，将事件绑定到根元素
  • 驼峰命名：事件名使用驼峰命名，如 onClick、onChange
  • 阻止默认行为：使用 event.preventDefault() 而不是 return false

2. 如何绑定事件？

jsx
function Button() {
  const handleClick = (event) => {
    console.log('Button clicked', event)
  }

  return <button onClick={handleClick}>Click me</button>
}

3. 如何传递参数？

jsx
function Button() {
  const handleClick = (id, event) => {
    console.log('Button clicked', id, event)
  }

  return <button onClick={(event) => handleClick(1, event)}>Click me</button>
}

4. 事件对象的特点？

  • 合成事件：event 是合成事件对象
  • 兼容性：解决了浏览器兼容性问题
  • 性能：使用对象池提高性能
  • 原生事件：通过 event.nativeEvent 访问原生事件

React 的条件渲染 ​

1. 如何实现条件渲染？

jsx
// 使用 if 语句
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>
  }
  return <h1>Please sign up.</h1>
}

// 使用三元运算符
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please sign up.</h1>}
    </div>
  )
}

// 使用 && 运算符
function WarningBanner({ warn }) {
  if (!warn) {
    return null
  }
  return (
    <div className="warning">
      Warning!
    </div>
  )
}

2. 如何阻止组件渲染？

jsx
// 返回 null
function WarningBanner({ warn }) {
  if (!warn) {
    return null
  }
  return <div className="warning">Warning!</div>
}

React 的列表渲染 ​

1. 如何渲染列表？

jsx
function NumberList({ numbers }) {
  const listItems = numbers.map((number) =>
    <li key={number.toString()}>
      {number}
    </li>
  )

  return (
    <ul>{listItems}</ul>
  )
}

2. 为什么需要 key？

  • 标识元素：key 帮助 React 识别哪些元素发生了变化
  • 性能优化：key 可以提高列表渲染性能
  • 避免问题：没有 key 可能导致状态错乱

3. key 的使用注意事项？

  • 唯一性：key 必须在兄弟元素中唯一
  • 稳定性：key 应该是稳定的，不要使用索引
  • 推荐值：使用数据中的唯一 ID 作为 key

4. 如何使用索引作为 key？

jsx
// 不推荐：使用索引作为 key
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map((todo, index) =>
        <li key={index}>
          {todo.text}
        </li>
      )}
    </ul>
  )
}

// 推荐：使用唯一 ID 作为 key
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map(todo =>
        <li key={todo.id}>
          {todo.text}
        </li>
      )}
    </ul>
  )
}

React 的表单处理 ​

1. 什么是受控组件？ 受控组件是指表单元素的值由 React 的 state 控制，通过 onChange 事件更新 state。

2. 受控组件的用法？

jsx
function NameForm() {
  const [value, setValue] = useState('')

  const handleChange = (event) => {
    setValue(event.target.value)
  }

  return (
    <form>
      <label>
        Name:
        <input type="text" value={value} onChange={handleChange} />
      </label>
      <input type="submit" value="Submit" />
    </form>
  )
}

3. 什么是非受控组件？ 非受控组件是指表单元素的值由 DOM 控制，通过 ref 获取值。

4. 非受控组件的用法？

jsx
function NameForm() {
  const inputRef = useRef(null)

  const handleSubmit = (event) => {
    alert('A name was submitted: ' + inputRef.current.value)
    event.preventDefault()
  }

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name:
        <input type="text" ref={inputRef} />
      </label>
      <input type="submit" value="Submit" />
    </form>
  )
}

5. 受控组件与非受控组件的区别？

  • 数据来源：受控组件数据来自 state，非受控组件数据来自 DOM
  • 更新方式：受控组件通过 onChange 更新，非受控组件通过 ref 获取
  • 适用场景：受控组件适合复杂表单，非受控组件适合简单表单

React 的 Refs ​

1. 什么是 Refs？ Refs 提供了一种访问 DOM 元素或 React 元素的方式。

2. 如何创建 Refs？

jsx
// 使用 useRef Hook
function TextInputWithFocusButton() {
  const inputEl = useRef(null)

  const onButtonClick = () => {
    inputEl.current.focus()
  }

  return (
    <>
      <input ref={inputEl} type="text" />
      <button onClick={onButtonClick}>Focus the input</button>
    </>
  )
}

// 使用 createRef（类组件）
class CustomTextInput extends React.Component {
  constructor(props) {
    super(props)
    this.textInput = React.createRef()
  }

  focusTextInput = () => {
    this.textInput.current.focus()
  }

  render() {
    return (
      <div>
        <input
          type="text"
          ref={this.textInput}
        />
        <button onClick={this.focusTextInput}>
          Focus the text input
        </button>
      </div>
    )
  }
}

3. Refs 的使用场景？

  • DOM 操作：需要直接操作 DOM 元素
  • 焦点管理：管理元素的焦点
  • 动画：使用第三方动画库
  • 媒体播放：控制媒体元素的播放

4. Refs 的注意事项？

  • 不要过度使用：优先使用 props 和 state
  • 避免在渲染中访问：不要在渲染函数中访问 ref
  • 清理 Refs：在组件卸载时清理 ref

React 的常见问题 ​

1. 为什么不能直接修改 state？

  • 不可变数据：React 使用不可变数据，直接修改 state 不会触发重新渲染
  • 性能优化：React 通过对比新旧 state 判断是否需要更新
  • 调试：不可变数据便于调试和时间旅行

2. setState 是异步的吗？ 是的，setState 是异步的。React 会批量更新 state 以提高性能。

3. 如何在 setState 后获取最新的 state？

jsx
// 使用回调函数
this.setState((prevState, props) => ({
  counter: prevState.counter + props.increment
}))

// 使用 useEffect Hook
function Counter() {
  const [count, setCount] = useState(0)

  useEffect(() => {
    console.log('Count updated:', count)
  }, [count])

  return <button onClick={() => setCount(count + 1)}>Increment</button>
}

4. 为什么建议使用函数式 setState？

jsx
// 不推荐：基于旧 state 更新
this.setState({ count: this.state.count + 1 })

// 推荐：使用函数式更新
this.setState(prevState => ({ count: prevState.count + 1 }))

5. 如何避免无限循环？

  • 依赖数组：在 useEffect 中正确设置依赖数组
  • 条件判断：在事件处理中添加条件判断
  • 避免在渲染中调用：不要在渲染函数中调用 setState

React 的最佳实践 ​

1. 组件的设计原则？

  • 单一职责：每个组件只做一件事
  • 可复用性：设计可复用的组件
  • 可测试性：组件应该易于测试
  • 可维护性：保持代码简洁和可读

2. 命名规范？

  • 组件名：使用 PascalCase，如 MyComponent
  • 文件名：与组件名相同，如 MyComponent.jsx
  • 事件处理：使用 handle 前缀，如 handleClick
  • 布尔值：使用 is、has、should 前缀，如 isVisible

3. 代码组织？

  • 按功能组织：将相关组件放在同一目录
  • 按类型组织：将组件、hooks、utils 分开
  • 使用 index 文件：简化导入路径

4. 性能优化？

  • 避免不必要的渲染：使用 React.memo、useMemo、useCallback
  • 代码分割：使用 React.lazy 和 Suspense
  • 虚拟列表：对于长列表使用虚拟滚动
  • 避免内联函数：避免在渲染中创建新函数
