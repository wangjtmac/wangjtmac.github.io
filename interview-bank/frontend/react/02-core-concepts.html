JSX 深入理解 ​

1. JSX 的本质是什么？ JSX 是 JavaScript 的语法扩展，它会被 Babel 编译为 React.createElement() 调用。

2. JSX 的编译过程？

jsx
// JSX
const element = <h1 className="greeting">Hello, world!</h1>

// 编译后
const element = React.createElement(
  'h1',
  { className: 'greeting' },
  'Hello, world!'
)

3. JSX 的注意事项？

  • 必须有一个根元素：JSX 表达式必须有一个根元素
  • 使用 Fragment：可以使用 <React.Fragment> 或 <> 包裹多个元素
  • 表达式：使用 {} 包裹 JavaScript 表达式
  • 注释：使用 {/* */} 语法
  • 驼峰命名：属性名使用驼峰命名

4. 如何在 JSX 中使用 JavaScript 表达式？

jsx
// 变量
const name = 'John'
const element = <h1>Hello, {name}!</h1>

// 函数调用
function formatName(user) {
  return user.firstName + ' ' + user.lastName
}
const element = <h1>Hello, {formatName(user)}!</h1>

// 三元运算符
const element = <div>{isLoggedIn ? <UserGreeting /> : <GuestGreeting />}</div>

// 逻辑与运算符
const element = <div>{unreadMessages.length > 0 && <h2>You have {unreadMessages.length} unread messages.</h2>}</div>

5. JSX 中的属性？

jsx
// 字符串字面量
const element = <div tabIndex="0"></div>

// JavaScript 表达式
const element = <img src={user.avatarUrl}></img>

// 默认为 true
const element = <input type="checkbox" defaultChecked={true} />

// 展开属性
const props = { firstName: 'Ben', lastName: 'Hector' }
const element = <Greeting {...props} />

虚拟 DOM 深入理解 ​

1. 虚拟 DOM 的工作原理？

  • React 创建虚拟 DOM 树
  • 当状态变化时，创建新的虚拟 DOM 树
  • 对比新旧虚拟 DOM 树的差异（Diff 算法）
  • 只更新真实 DOM 中有差异的部分

2. React 的 Diff 算法？

  • 同层比较：只比较同一层级的节点
  • 类型不同：直接替换整个节点
  • 类型相同：递归比较子节点
  • key 优化：使用 key 优化列表渲染

3. 虚拟 DOM 的优势？

  • 性能优化：减少真实 DOM 操作
  • 跨平台：可以在不同平台上渲染
  • 批量更新：可以批量更新 DOM
  • 可预测性：状态变化可以预测

4. 虚拟 DOM 的劣势？

  • 内存占用：虚拟 DOM 占用额外内存
  • 首次渲染：首次渲染较慢
  • 简单场景：对于简单应用，虚拟 DOM 优势不明显

组件深入理解 ​

1. 函数组件的特点？

  • 简单：语法简单，易于理解
  • 无状态：早期版本没有 state，现在通过 Hooks 支持
  • 性能：性能更好，React 18 后推荐使用
  • Hooks：可以使用 Hooks 实现复杂功能

2. 类组件的特点？

  • 完整功能：支持 state、生命周期等完整功能
  • this 绑定：需要注意 this 的绑定问题
  • 复杂：代码相对复杂
  • 逐步淘汰：React 18 后推荐使用函数组件

3. 函数组件与类组件的选择？

  • 推荐函数组件：React 18 后推荐使用函数组件
  • 需要生命周期：可以使用 Hooks 替代
  • 需要 this：函数组件没有 this
  • 性能要求：函数组件性能更好

4. 组件的渲染过程？

  • 初始化：创建组件实例
  • 渲染：调用 render 方法或函数组件
  • 更新：props 或 state 变化时重新渲染
  • 卸载：组件从 DOM 中移除

Props 深入理解 ​

1. Props 的传递方式？

jsx
// 父组件
function Parent() {
  return <Child name="John" age={30} />
}

// 子组件
function Child(props) {
  return <div>Name: {props.name}, Age: {props.age}</div>
}

// 解构赋值
function Child({ name, age }) {
  return <div>Name: {name}, Age: {age}</div>
}

2. Props 的类型检查？

jsx
import PropTypes from 'prop-types'

function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>
}

Greeting.propTypes = {
  name: PropTypes.string.isRequired
}

Greeting.defaultProps = {
  name: 'Stranger'
}

3. Props 的只读特性？

  • 不可修改：子组件不能修改 props
  • 单向数据流：数据从父组件流向子组件
  • 数据同步：如果需要修改，通过回调函数

4. Props 的传递技巧？

  • 解构赋值：使用解构简化代码
  • 默认值：使用 defaultProps 设置默认值
  • 展开运算符：使用 ...props 展开属性
  • children：使用 props.children 传递子元素

State 深入理解 ​

1. State 的使用方式？

jsx
// 函数组件
function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  )
}

// 类组件
class Counter extends React.Component {
  constructor(props) {
    super(props)
    this.state = { count: 0 }
  }

  increment = () => {
    this.setState({ count: this.state.count + 1 })
  }

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={this.increment}>Increment</button>
      </div>
    )
  }
}

2. State 的更新规则？

  • 不要直接修改：不要直接修改 state
  • 使用 setState：使用 setState 或 setState 的函数形式
  • 异步更新：setState 是异步的
  • 合并更新：setState 会合并更新

3. State 的更新陷阱？

jsx
// 错误：直接修改 state
this.state.count = this.state.count + 1

// 错误：基于旧 state 更新
this.setState({ count: this.state.count + 1 })

// 正确：使用函数式更新
this.setState(prevState => ({ count: prevState.count + 1 }))

4. State 的设计原则？

  • 最小化 state：只保留必要的 state
  • 避免冗余：避免从 props 或其他 state 派生 state
  • 组件化：将相关 state 放在同一个组件中

事件处理深入理解 ​

1. 合成事件的特点？

  • 跨浏览器：解决了浏览器兼容性问题
  • 事件委托：使用事件委托提高性能
  • 对象池：使用对象池减少内存分配
  • 阻止默认：使用 event.preventDefault() 阻止默认行为

2. 事件处理的最佳实践？

jsx
// 使用箭头函数
function Button() {
  const handleClick = () => {
    console.log('Button clicked')
  }

  return <button onClick={handleClick}>Click me</button>
}

// 传递参数
function Button() {
  const handleClick = (id) => {
    console.log('Button', id, 'clicked')
  }

  return <button onClick={() => handleClick(1)}>Click me</button>
}

// 使用 useCallback 优化
function Button({ onClick }) {
  const handleClick = useCallback(() => {
    onClick()
  }, [onClick])

  return <button onClick={handleClick}>Click me</button>
}

3. 事件对象的属性？

  • bubbles：事件是否冒泡
  • cancelable：事件是否可取消
  • currentTarget：当前事件处理程序所在的元素
  • defaultPrevented：是否调用了 preventDefault()
  • eventPhase：事件所处的阶段
  • isTrusted：事件是否由用户操作触发
  • nativeEvent：原生事件对象
  • target：触发事件的元素
  • timeStamp：事件触发的时间戳
  • type：事件类型

条件渲染深入理解 ​

1. 条件渲染的方式？

jsx
// 使用 if 语句
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>
  }
  return <h1>Please sign up.</h1>
}

// 使用三元运算符
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <h1>Please sign up.</h1>}
    </div>
  )
}

// 使用 && 运算符
function Mailbox({ unreadMessages }) {
  return (
    <div>
      <h1>Hello!</h1>
      {unreadMessages.length > 0 &&
        <h2>
          You have {unreadMessages.length} unread messages.
        </h2>
      }
    </div>
  )
}

// 使用 IIFE
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {(() => {
        if (isLoggedIn) {
          return <h1>Welcome back!</h1>
        } else {
          return <h1>Please sign up.</h1>
        }
      })()}
    </div>
  )
}

2. 阻止组件渲染？

jsx
// 返回 null
function WarningBanner({ warn }) {
  if (!warn) {
    return null
  }
  return <div className="warning">Warning!</div>
}

// 使用 React.Fragment
function Table({ data }) {
  if (!data || data.length === 0) {
    return <React.Fragment />
  }
  return (
    <table>
      {data.map(row => <TableRow key={row.id} data={row} />)}
    </table>
  )
}

列表渲染深入理解 ​

1. 列表渲染的方式？

jsx
// 使用 map
function NumberList({ numbers }) {
  return (
    <ul>
      {numbers.map((number) =>
        <li key={number.toString()}>
          {number}
        </li>
      )}
    </ul>
  )
}

// 使用 for 循环
function NumberList({ numbers }) {
  const listItems = []
  for (let i = 0; i < numbers.length; i++) {
    listItems.push(<li key={numbers[i]}>{numbers[i]}</li>)
  }
  return <ul>{listItems}</ul>
}

2. Key 的作用？

  • 标识元素：key 帮助 React 识别哪些元素发生了变化
  • 性能优化：key 可以提高列表渲染性能
  • 避免问题：没有 key 可能导致状态错乱

3. Key 的使用原则？

  • 唯一性：key 必须在兄弟元素中唯一
  • 稳定性：key 应该是稳定的，不要使用索引
  • 推荐值：使用数据中的唯一 ID 作为 key
  • 避免使用索引：除非列表是静态的

4. Key 的常见错误？

jsx
// 错误：使用索引作为 key
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map((todo, index) =>
        <li key={index}>
          {todo.text}
        </li>
      )}
    </ul>
  )
}

// 正确：使用唯一 ID 作为 key
function TodoList({ todos }) {
  return (
    <ul>
      {todos.map(todo =>
        <li key={todo.id}>
          {todo.text}
        </li>
      )}
    </ul>
  )
}

表单处理深入理解 ​

1. 受控组件的完整实现？

jsx
function NameForm() {
  const [value, setValue] = useState('')

  const handleChange = (event) => {
    setValue(event.target.value)
  }

  const handleSubmit = (event) => {
    alert('A name was submitted: ' + value)
    event.preventDefault()
  }

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name:
        <input type="text" value={value} onChange={handleChange} />
      </label>
      <input type="submit" value="Submit" />
    </form>
  )
}

2. 多个输入框的处理？

jsx
function Reservation() {
  const [formData, setFormData] = useState({
    isGoing: true,
    numberOfGuests: 2
  })

  const handleInputChange = (event) => {
    const target = event.target
    const value = target.type === 'checkbox' ? target.checked : target.value
    const name = target.name

    setFormData({
      ...formData,
      [name]: value
    })
  }

  return (
    <form>
      <label>
        Is going:
        <input
          name="isGoing"
          type="checkbox"
          checked={formData.isGoing}
          onChange={handleInputChange}
        />
      </label>
      <br />
      <label>
        Number of guests:
        <input
          name="numberOfGuests"
          type="number"
          value={formData.numberOfGuests}
          onChange={handleInputChange}
        />
      </label>
    </form>
  )
}

3. 非受控组件的完整实现？

jsx
function NameForm() {
  const inputRef = useRef(null)

  const handleSubmit = (event) => {
    alert('A name was submitted: ' + inputRef.current.value)
    event.preventDefault()
  }

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Name:
        <input type="text" ref={inputRef} />
      </label>
      <input type="submit" value="Submit" />

  )
}

Refs 深入理解 ​

1. Refs 的使用场景？

  • DOM 操作：需要直接操作 DOM 元素
  • 焦点管理：管理元素的焦点
  • 动画：使用第三方动画库
  • 媒体播放：控制媒体元素的播放

2. Refs 的使用方式？

jsx
// 使用 useRef Hook
function TextInputWithFocusButton() {
  const inputEl = useRef(null)

  const onButtonClick = () => {
    inputEl.current.focus()
  }

  return (
    <>
      <input ref={inputEl} type="text" />
      <button onClick={onButtonClick}>Focus the input</button>
    </>
  )
}

// 使用 createRef（类组件）
class CustomTextInput extends React.Component {
  constructor(props) {
    super(props)
    this.textInput = React.createRef()
  }

  focusTextInput = () => {
    this.textInput.current.focus()
  }

  render() {
    return (
      <div>
        <input type="text" ref={this.textInput} />
        <button onClick={this.focusTextInput}>
          Focus the text input
        </button>
      </div>
    )
  }
}

// 回调 Refs
function CustomTextInput(props) {
  return (
    <div>
      <input ref={props.inputRef} />
    </div>
  )
}

class Parent extends React.Component {
  render() {
    return (
      <CustomTextInput
        inputRef={el => this.inputElement = el}
      />
    )
  }
}

3. Refs 的注意事项？

  • 不要过度使用：优先使用 props 和 state
  • 避免在渲染中访问：不要在渲染函数中访问 ref
  • 清理 Refs：在组件卸载时清理 ref
  • 不要在函数组件中使用 this：函数组件没有 this

Fragment 深入理解 ​

1. Fragment 的作用？ Fragment 允许你将子元素分组，而不会向 DOM 添加额外的节点。

2. Fragment 的使用方式？

jsx
// 使用 React.Fragment
class Columns extends React.Component {
  render() {
    return (
      <React.Fragment>
        <td>Hello</td>
        <td>World</td>
      </React.Fragment>
    )
  }
}

// 使用简写语法
class Columns extends React.Component {
  render() {
    return (
      <>
        <td>Hello</td>
        <td>World</td>
      </>
    )
  }
}

3. Fragment 的 key？

jsx
function Glossary(props) {
  return (
    <dl>
      {props.items.map(item => (
        <React.Fragment key={item.id}>
          <dt>{item.term}</dt>
          <dd>{item.description}</dd>
        </React.Fragment>
      ))}
    </dl>
  )
}

4. Fragment 的使用场景？

  • 返回多个元素：组件需要返回多个元素
  • 表格元素：在表格中使用多个 td 元素
  • 避免额外 div：避免添加额外的 DOM 节点
