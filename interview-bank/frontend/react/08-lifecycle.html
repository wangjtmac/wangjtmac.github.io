React 生命周期 ​

类组件生命周期 ​

生命周期方法分类 ​

React 类组件的生命周期方法可以分为三个阶段：

  1. 挂载阶段 (Mounting): 组件被创建并插入到 DOM 中
  2. 更新阶段 (Updating): 组件的 props 或 state 发生变化
  3. 卸载阶段 (Unmounting): 组件从 DOM 中移除

挂载阶段 ​

挂载阶段按顺序执行以下方法：

jsx
class LifecycleDemo extends React.Component {
  // 1. constructor - 构造函数
  constructor(props) {
    super(props)
    console.log('1. constructor')
    this.state = { count: 0 }
  }

  // 2. static getDerivedStateFromProps - 从 props 派生 state
  static getDerivedStateFromProps(props, state) {
    console.log('2. getDerivedStateFromProps')
    return null
  }

  // 3. render - 渲染
  render() {
    console.log('3. render')
    return <div>Count: {this.state.count}</div>
  }

  // 4. componentDidMount - 组件挂载完成
  componentDidMount() {
    console.log('4. componentDidMount')
    // 可以在这里进行 DOM 操作、发起网络请求
  }
}

更新阶段 ​

更新阶段按顺序执行以下方法：

jsx
class LifecycleDemo extends React.Component {
  // 1. static getDerivedStateFromProps - 从 props 派生 state
  static getDerivedStateFromProps(props, state) {
    console.log('1. getDerivedStateFromProps')
    return null
  }

  // 2. shouldComponentUpdate - 是否应该更新
  shouldComponentUpdate(nextProps, nextState) {
    console.log('2. shouldComponentUpdate')
    // 返回 true 继续更新，返回 false 跳过更新
    return true
  }

  // 3. render - 渲染
  render() {
    console.log('3. render')
    return <div>Count: {this.state.count}</div>
  }

  // 4. getSnapshotBeforeUpdate - 更新前获取快照
  getSnapshotBeforeUpdate(prevProps, prevState) {
    console.log('4. getSnapshotBeforeUpdate')
    return null
  }

  // 5. componentDidUpdate - 组件更新完成
  componentDidUpdate(prevProps, prevState, snapshot) {
    console.log('5. componentDidUpdate')
    // 可以在这里进行 DOM 操作或网络请求
  }
}

卸载阶段 ​

卸载阶段执行以下方法：

jsx
class LifecycleDemo extends React.Component {
  // componentWillUnmount - 组件卸载前
  componentWillUnmount() {
    console.log('componentWillUnmount')
    // 清理工作：清除定时器、取消网络请求、移除事件监听
  }
}

已废弃的生命周期方法 ​

以下生命周期方法已被废弃：

jsx
// UNSAFE_componentWillMount - 挂载前
UNSAFE_componentWillMount() {
  console.log('componentWillMount')
}

// UNSAFE_componentWillReceiveProps - 接收新 props 前
UNSAFE_componentWillReceiveProps(nextProps) {
  console.log('componentWillReceiveProps')
}

// UNSAFE_componentWillUpdate - 更新前
UNSAFE_componentWillUpdate(nextProps, nextState) {
  console.log('componentWillUpdate')
}

函数组件生命周期 ​

useEffect Hook ​

函数组件使用 useEffect Hook 来处理副作用，可以模拟类组件的生命周期方法：

jsx
import { useState, useEffect } from 'react'

function LifecycleDemo() {
  const [count, setCount] = useState(0)

  // 模拟 componentDidMount 和 componentDidUpdate
  useEffect(() => {
    console.log('组件挂载或更新')
  })

  // 模拟 componentDidMount（只在挂载时执行）
  useEffect(() => {
    console.log('组件挂载')
  }, [])

  // 模拟 componentDidUpdate（只在 count 变化时执行）
  useEffect(() => {
    console.log('count 变化:', count)
  }, [count])

  // 模拟 componentWillUnmount
  useEffect(() => {
    console.log('组件挂载')

    return () => {
      console.log('组件卸载')
      // 清理工作
    }
  }, [])

  return <div>Count: {count}</div>
}

useEffect 的使用场景 ​

jsx
function UseEffectExamples() {
  // 1. 每次渲染后执行
  useEffect(() => {
    console.log('每次渲染后执行')
  })

  // 2. 只在挂载时执行（模拟 componentDidMount）
  useEffect(() => {
    console.log('只在挂载时执行')
  }, [])

  // 3. 依赖项变化时执行（模拟 componentDidUpdate）
  const [count, setCount] = useState(0)
  useEffect(() => {
    console.log('count 变化:', count)
  }, [count])

  // 4. 清理副作用（模拟 componentWillUnmount）
  useEffect(() => {
    const timer = setInterval(() => {
      console.log('定时器执行')
    }, 1000)

    return () => {
      clearInterval(timer)
    }
  }, [])

  // 5. 多个依赖项
  const [name, setName] = useState('')
  useEffect(() => {
    console.log('count 或 name 变化:', count, name)
  }, [count, name])

  return <div>Count: {count}</div>
}

生命周期面试题 ​

1. React 组件的生命周期有哪些？ ​

答案：

React 组件的生命周期分为三个阶段：

  1. 挂载阶段 (Mounting):

    • constructor(): 构造函数，初始化 state
    • static getDerivedStateFromProps(): 从 props 派生 state
    • render(): 渲染组件
    • componentDidMount(): 组件挂载完成

  2. 更新阶段 (Updating):

    • static getDerivedStateFromProps(): 从 props 派生 state
    • shouldComponentUpdate(): 是否应该更新
    • render(): 渲染组件
    • getSnapshotBeforeUpdate(): 更新前获取快照
    • componentDidUpdate(): 组件更新完成

  3. 卸载阶段 (Unmounting):

    • componentWillUnmount(): 组件卸载前

2. React 为什么要废弃 componentWillMount、componentWillReceiveProps 和 componentWillUpdate？ ​

答案：

这些方法被废弃的原因：

  1. 不安全的使用场景:

    • 这些方法在渲染过程中被调用，可能导致重复渲染
    • 在这些方法中调用 setState 会导致额外的渲染周期

  2. 异步渲染问题:

    • React 18 引入了并发渲染，这些方法在异步渲染中会导致问题
    • 可能在组件更新前被多次调用

  3. 更好的替代方案:

    • componentWillMount → componentDidMount 或 constructor
    • componentWillReceiveProps → getDerivedStateFromProps 或 componentDidUpdate
    • componentWillUpdate → getSnapshotBeforeUpdate 或 componentDidUpdate

3. useEffect 和 componentDidMount 的区别是什么？ ​

答案：

主要区别：

  1. 执行时机:

    • componentDidMount: 只在组件挂载后执行一次
    • useEffect(() => {}, []): 只在依赖项为空数组时执行一次

  2. 执行次数:

    • componentDidMount: 保证只执行一次
    • useEffect: 依赖项为空数组时只执行一次，但可能在某些情况下执行多次

  3. 清理时机:

    • componentWillUnmount: 在组件卸载前执行
    • useEffect 的返回函数: 在组件卸载前或下次 effect 执行前执行

  4. 使用场景:

    • componentDidMount: 适合在类组件中使用
    • useEffect: 适合在函数组件中使用，更灵活

4. React 18 中 useEffect 的执行时机有什么变化？ ​

答案：

React 18 中 useEffect 的执行时机变化：

  1. 并发渲染:

    • React 18 引入了并发渲染，useEffect 可能会延迟执行
    • 浏览器绘制完成后再执行 useEffect

  2. 优先级调度:

    • 高优先级更新会打断低优先级的 useEffect 执行
    • useEffect 的执行可能被推迟

  3. Strict Mode:

    • 开发模式下，useEffect 会执行两次（挂载和卸载）
    • 用于检测副作用清理是否正确

  4. 解决方案:

    • 使用 useLayoutEffect 代替 useEffect 来同步执行副作用
    • 使用 useInsertionEffect 来在 DOM 变更前执行副作用

5. getDerivedStateFromProps 的使用场景是什么？ ​

答案：

getDerivedStateFromProps 的使用场景：

  1. 根据 props 更新 state:

    jsx
    static getDerivedStateFromProps(props, state) {
  if (props.id !== state.id) {
    return {
      id: props.id,
      data: null
    }
  }
  return null
}

  2. 派生 state:

    jsx
    static getDerivedStateFromProps(props, state) {
  return {
    fullName: `${props.firstName} ${props.lastName}`
  }
}

  3. 注意事项:

    • 这是一个静态方法，不能访问组件实例
    • 必须返回一个对象或 null
    • 不应该在这里执行副作用
    • 应该谨慎使用，避免过度使用

6. shouldComponentUpdate 的使用场景是什么？ ​

答案：

shouldComponentUpdate 的使用场景：

  1. 性能优化:

    jsx
    shouldComponentUpdate(nextProps, nextState) {
  return nextProps.id !== this.props.id ||
         nextState.count !== this.state.count
}

  2. 避免不必要的渲染:

    jsx
    shouldComponentUpdate(nextProps, nextState) {
  // 如果 count 没有变化，不重新渲染
  return nextState.count !== this.state.count
}

  3. 注意事项:

    • 不要在这里调用 setState
    • 不要在这里执行副作用
    • 返回 false 会跳过整个更新过程
    • 考虑使用 React.PureComponent 或 React.memo 代替

7. getSnapshotBeforeUpdate 的使用场景是什么？ ​

答案：

getSnapshotBeforeUpdate 的使用场景：

  1. 获取更新前的 DOM 状态:

    jsx
    getSnapshotBeforeUpdate(prevProps, prevState) {
  if (prevProps.list.length < this.props.list.length) {
    const list = this.listRef.current
    return list.scrollHeight - list.scrollTop
  }
  return null
}

componentDidUpdate(prevProps, prevState, snapshot) {
  if (snapshot !== null) {
    const list = this.listRef.current
    list.scrollTop = list.scrollHeight - snapshot
  }
}

  2. 保存滚动位置:

    jsx
    getSnapshotBeforeUpdate(prevProps) {
  if (prevProps.isScrollingDown !== this.props.isScrollingDown) {
    return this.listRef.current.scrollHeight
  }
  return null
}

  3. 注意事项:

    • 必须返回一个值或 null
    • 返回值会作为 componentDidUpdate 的第三个参数
    • 在 DOM 更新前执行
    • 不应该在这里执行副作用

8. useEffect 和 useLayoutEffect 的区别是什么？ ​

答案：

主要区别：

  1. 执行时机:

    • useEffect: 在浏览器绘制完成后异步执行
    • useLayoutEffect: 在 DOM 变更后、浏览器绘制前同步执行

  2. 使用场景:

    • useEffect: 大多数副作用，如数据获取、订阅、事件监听
    • useLayoutEffect: 需要同步读取 DOM 布局或同步触发重绘的场景

  3. 性能影响:

    • useEffect: 不会阻塞浏览器绘制，性能更好
    • useLayoutEffect: 会阻塞浏览器绘制，可能影响性能

  4. 示例:

    jsx
    // useEffect - 异步执行
useEffect(() => {
  document.title = `Count: ${count}`
}, [count])

// useLayoutEffect - 同步执行
useLayoutEffect(() => {
  const ref = myRef.current
  ref.style.transform = 'translateX(100px)'
}, [])

9. 如何在函数组件中模拟 componentWillUnmount？ ​

答案：

在函数组件中使用 useEffect 的返回函数来模拟 componentWillUnmount：

jsx
function Component() {
  useEffect(() => {
    // 组件挂载时执行
    const timer = setInterval(() => {
      console.log('定时器')
    }, 1000)

    // 组件卸载时执行
    return () => {
      clearInterval(timer)
      console.log('清理定时器')
    }
  }, [])

  return <div>Component</div>
}

清理函数会在以下时机执行：

  • 组件卸载时
  • 下次 effect 执行前（如果依赖项变化）

10. React 生命周期中哪些方法可以执行 setState？ ​

答案：

可以执行 setState 的方法：

  1. 可以执行:

    • componentDidMount: 组件挂载完成后
    • componentDidUpdate: 组件更新完成后
    • componentWillUnmount: 组件卸载前（但不推荐）

  2. 不能执行:

    • constructor: 应该使用 this.state = {} 初始化
    • render: 会导致无限循环
    • shouldComponentUpdate: 会导致无限循环
    • getDerivedStateFromProps: 应该返回派生的 state
    • getSnapshotBeforeUpdate: 不应该在这里执行副作用

  3. 注意事项:

    • 在 componentDidMount 中调用 setState 会触发额外的渲染
    • 在 componentDidUpdate 中调用 setState 需要条件判断，避免无限循环
    • 推荐使用 useEffect 代替 componentDidMount 和 componentDidUpdate

学习建议 ​

  1. 理解生命周期流程: 熟悉类组件的生命周期方法及其执行顺序
  2. 掌握 useEffect: 理解 useEffect 的各种使用场景和依赖项的作用
  3. 性能优化: 使用 shouldComponentUpdate、React.memo、useMemo、useCallback 等优化性能
  4. 清理副作用: 理解如何正确清理副作用，避免内存泄漏
  5. 迁移到 Hooks: 优先使用函数组件和 Hooks，而不是类组件
