React.memo ​

1. 什么是 React.memo？ React.memo 是一个高阶组件，用于避免不必要的重新渲染。

2. React.memo 的基本用法？

jsx
const MyComponent = React.memo(function MyComponent(props) {
  return <div>{props.name}</div>
})

3. React.memo 的比较函数？

jsx
function MyComponent(props) {
  return <div>{props.name}</div>
}

function areEqual(prevProps, nextProps) {
  return prevProps.name === nextProps.name
}

const MemoizedComponent = React.memo(MyComponent, areEqual)

4. React.memo 的使用场景？

  • 纯组件：组件的输出只依赖于 props
  • 频繁渲染：组件频繁重新渲染
  • 性能瓶颈：组件是性能瓶颈

useMemo ​

1. useMemo 的基本用法？

jsx
import { useMemo } from 'react'

function ExpensiveComponent({ a, b }) {
  const result = useMemo(() => {
    return expensiveCalculation(a, b)
  }, [a, b])

  return <div>Result: {result}</div>
}

2. useMemo 的使用场景？

  • 昂贵的计算：避免在每次渲染时重新计算
  • 复杂的对象：避免在每次渲染时创建新对象
  • 依赖优化：只有依赖变化时才重新计算

3. useMemo 的注意事项？

  • 不要过度使用：只有在性能问题时才使用
  • 正确的依赖：确保依赖数组正确
  • 与 useCallback 配合：与 useCallback 配合使用效果更好

useCallback ​

1. useCallback 的基本用法？

jsx
import { useCallback } from 'react'

function Parent() {
  const [count, setCount] = useState(0)

  const handleClick = useCallback(() => {
    setCount(count + 1)
  }, [count])

  return <Child onClick={handleClick} />
}

2. useCallback 的使用场景？

  • 传递给子组件：避免子组件不必要的重新渲染
  • 作为 useEffect 的依赖：避免无限循环
  • 事件处理：优化事件处理函数

3. useCallback 的注意事项？

  • 不要过度使用：只有在性能问题时才使用
  • 正确的依赖：确保依赖数组正确
  • 与 React.memo 配合：与 React.memo 配合使用效果更好

代码分割 ​

1. 什么是代码分割？ 代码分割是指将代码拆分成多个块，按需加载，减少初始加载时间。

2. React.lazy 的基本用法？

jsx
import { lazy, Suspense } from 'react'

const Home = lazy(() => import('./Home'))
const About = lazy(() => import('./About'))

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </Suspense>
  )
}

3. 代码分割的优势？

  • 减少初始加载：减少初始加载体积
  • 按需加载：只在需要时加载对应的代码
  • 提高性能：提高应用的加载性能

4. 代码分割的注意事项？

  • 合理拆分：按路由或功能拆分代码
  • 预加载：预加载可能访问的代码
  • 错误处理：处理加载错误

虚拟列表 ​

1. 什么是虚拟列表？ 虚拟列表是指只渲染可见部分的列表项，提高长列表的渲染性能。

2. react-window 的基本用法？

jsx
import { FixedSizeList as List } from 'react-window'

function Row({ index, style }) {
  return <div style={style}>Row {index}</div>
}

function VirtualList() {
  return (
    <List
      height={600}
      itemCount={1000}
      itemSize={35}
      width={300}
    >
      {Row}
    </List>
  )
}

3. react-virtualized 的基本用法？

jsx
import { List } from 'react-virtualized'

function Row({ index, key, style }) {
  return <div key={key} style={style}>Row {index}</div>
}

function VirtualList() {
  return (
    <List
      width={300}
      height={600}
      rowCount={1000}
      rowHeight={35}
      rowRenderer={Row}
    />
  )
}

4. 虚拟列表的使用场景？

  • 长列表：列表项数量很多
  • 性能瓶颈：列表渲染是性能瓶颈
  • 滚动性能：需要优化滚动性能

图片懒加载 ​

1. 什么是图片懒加载？ 图片懒加载是指只在图片进入视口时才加载图片，减少初始加载时间。

2. react-lazyload-image-component 的基本用法？

jsx
import LazyLoadImage from 'react-lazyload-image-component'

function ImageGallery() {
  const images = [
    'image1.jpg',
    'image2.jpg',
    'image3.jpg'
  ]

  return (
    <div>
      {images.map(src => (
        <LazyLoadImage
          key={src}
          src={src}
          alt="Image"
          effect="blur"
        />
      ))}
    </div>
  )
}

3. 图片懒加载的优势？

  • 减少初始加载：减少初始加载的图片数量
  • 提高性能：提高页面的加载性能
  • 节省带宽：节省用户的带宽

防抖与节流 ​

1. 什么是防抖？ 防抖是指在事件触发后等待一定时间，如果在这段时间内没有再次触发，才执行函数。

2. 什么是节流？ 节流是指在事件触发后，在一定时间内只执行一次函数。

3. lodash 的防抖与节流？

jsx
import { debounce, throttle } from 'lodash'

function SearchInput() {
  const [query, setQuery] = useState('')

  const handleSearch = debounce((value) => {
    console.log('Search:', value)
  }, 300)

  const handleScroll = throttle(() => {
    console.log('Scroll')
  }, 100)

  return (
    <div>
      <input
        value={query}
        onChange={(e) => {
          setQuery(e.target.value)
          handleSearch(e.target.value)
        }}
      />
      <div onScroll={handleScroll}>Scroll me</div>
    </div>
  )
}

4. 自定义防抖与节流？

jsx
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value)

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value)
    }, delay)

    return () => {
      clearTimeout(handler)
    }
  }, [value, delay])

  return debouncedValue
}

function SearchInput() {
  const [query, setQuery] = useState('')
  const debouncedQuery = useDebounce(query, 300)

  useEffect(() => {
    console.log('Search:', debouncedQuery)
  }, [debouncedQuery])

  return <input value={query} onChange={(e) => setQuery(e.target.value)} />
}

避免不必要的渲染 ​

1. 如何避免不必要的渲染？

  • 使用 React.memo：避免组件不必要的重新渲染
  • 使用 useMemo：避免昂贵的计算
  • 使用 useCallback：避免创建新函数
  • 正确的依赖：确保依赖数组正确

2. 如何优化列表渲染？

  • 使用 key：使用唯一且稳定的 key
  • 避免内联函数：避免在渲染中创建新函数
  • 使用虚拟列表：对于长列表使用虚拟列表

3. 如何优化条件渲染？

  • 使用短路运算符：使用 && 运算符而不是三元运算符
  • 避免复杂的条件：避免复杂的条件判断
  • 使用早期返回：使用早期返回避免嵌套

性能优化的最佳实践 ​

1. 性能优化的原则？

  • 测量优先：先测量性能瓶颈，再优化
  • 优化关键路径：优化用户交互的关键路径
  • 避免过度优化：不要过度优化，保持代码可读性
  • 持续优化：持续关注性能，及时优化

2. 性能优化的工具？

  • React DevTools：使用 React DevTools 分析性能
  • Chrome DevTools：使用 Chrome DevTools 分析性能
  • Lighthouse：使用 Lighthouse 分析性能
  • Webpack Bundle Analyzer：分析打包体积

3. 性能优化的注意事项？

  • 不要过早优化：先实现功能，再优化性能
  • 保持可读性：不要为了性能牺牲可读性
  • 测试优化效果：测试优化是否有效
  • 持续关注：持续关注性能指标
