Vue 基础语法 ​

Vue 的基础语法和概念相关面试题。

Vue 的基本概念和特点 ​

1. 什么是 Vue？ Vue 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。

2. Vue 的特点有哪些？

  • 响应式数据绑定：通过 Object.defineProperty 实现数据的响应式
  • 组件化开发：将界面拆分为可复用的组件
  • 虚拟 DOM：提高渲染性能
  • 指令系统：提供 v-if、v-for 等便捷指令
  • 生命周期钩子：提供多个生命周期阶段的钩子函数
  • 轻量级：体积小，性能好
  • 易于集成：可以与其它库无缝集成

3. Vue 与 React、Angular 的区别？

  • Vue：渐进式框架，上手简单，API 友好，文档完善
  • React：基于 JSX，生态丰富，适合大型应用，需要学习更多概念
  • Angular：完整的 MVC 框架，学习曲线较陡，适合企业级应用

Vue 指令的使用 ​

1. 常用的 Vue 指令有哪些？

  • v-if：条件渲染
  • v-else：与 v-if 配合使用
  • v-else-if：多条件渲染
  • v-for：列表渲染
  • v-show：条件显示（通过 CSS 控制）
  • v-bind：绑定属性，简写为 :
  • v-on：绑定事件，简写为 @
  • v-model：双向数据绑定
  • v-pre：跳过编译
  • v-cloak：防止闪烁
  • v-once：只渲染一次

2. v-if 和 v-show 的区别？

  • v-if：通过 DOM 操作添加/移除元素，有更高的切换开销
  • v-show：通过 CSS 的 display 属性控制显示/隐藏，有更高的初始渲染开销
  • 使用场景：频繁切换用 v-show，条件不常改变用 v-if

3. v-for 中为什么要使用 key？

  • key 是 Vue 识别节点的唯一标识，用于优化渲染性能
  • 没有 key 时，Vue 会使用"就地更新"策略，可能导致状态错乱
  • key 应该使用唯一且稳定的值，不建议使用索引

Vue 生命周期钩子 ​

1. Vue 的生命周期分为哪些阶段？

  • 创建阶段：beforeCreate、created
  • 挂载阶段：beforeMount、mounted
  • 更新阶段：beforeUpdate、updated
  • 销毁阶段：beforeDestroy、destroyed
  • Vue 3 新增：beforeUnmount、unmounted

2. beforeCreate 和 created 的区别？

  • beforeCreate：实例刚创建，数据观测和事件配置尚未完成，无法访问 data 和 methods
  • created：实例创建完成，数据观测和事件配置已完成，可以访问 data 和 methods，但 DOM 尚未挂载

3. mounted 和 updated 的注意事项？

  • mounted：DOM 已挂载，可执行 DOM 操作，发起网络请求
  • updated：DOM 已更新，避免在此时修改数据，可能导致无限循环

4. 组件销毁时需要做什么？

  • 清除定时器
  • 取消网络请求
  • 解绑事件监听器
  • 释放第三方库资源

Vue 模板语法 ​

1. Vue 的模板语法包括哪些内容？

  • 插值表达式：""
  • 指令：v- 前缀的特殊属性
  • 计算属性：基于响应式数据计算得出的值
  • 监听器：监听数据变化执行回调
  • 过滤器：对数据进行格式化（Vue 3 已移除）

2. 如何在模板中使用 JavaScript 表达式？ 可以在插值表达式和指令中使用简单的 JavaScript 表达式，如：

vue
{{ count + 1 }}
{{ message.split('').reverse().join('') }}
<div :class="{ active: isActive }">

3. 模板编译的过程是怎样的？

  • 解析模板字符串生成 AST（抽象语法树）
  • 优化 AST
  • 生成渲染函数
  • 执行渲染函数生成虚拟 DOM

计算属性与监听器 ​

1. 什么是计算属性？ 计算属性是基于响应式数据计算得出的属性，具有缓存特性，只有依赖数据变化时才会重新计算。

2. 计算属性与方法的区别？

  • 计算属性：有缓存，依赖数据变化时才重新计算
  • 方法：每次调用都会重新执行
  • 使用场景：需要缓存结果用计算属性，需要实时计算用方法

3. 什么是监听器？ 监听器用于监听数据变化并执行回调函数，适用于需要在数据变化时执行异步或开销较大的操作。

4. 计算属性与监听器的使用场景？

  • 计算属性：当需要基于现有数据生成新数据时
  • 监听器：当需要在数据变化时执行副作用操作时（如网络请求、DOM 操作等）

条件渲染与列表渲染 ​

1. 条件渲染的方式有哪些？

  • v-if：条件渲染元素
  • v-else：与 v-if 配合
  • v-else-if：多条件渲染
  • v-show：条件显示元素

2. 列表渲染的注意事项？

  • 使用 v-for 指令
  • 必须添加唯一的 key 属性
  • 避免在 v-for 中使用 v-if，可能影响性能
  • 可以使用 of 替代 in 作为分隔符

3. 如何遍历对象？

vue
<div v-for="(value, key, index) in object" :key="key">
  {{ index }}. {{ key }}: {{ value }}
</div>

事件处理 ​

1. 如何绑定事件？

  • 使用 v-on 指令，简写为 @
  • 可以传递参数
  • 可以访问事件对象 $event

2. 事件修饰符有哪些？

  • .stop：阻止冒泡
  • .prevent：阻止默认行为
  • .capture：使用捕获模式
  • .self：仅当事件目标是元素本身时触发
  • .once：只触发一次
  • .passive：告诉浏览器不阻止默认行为

3. 按键修饰符有哪些？

  • .enter：回车
  • .tab：制表
  • .delete：删除/退格
  • .esc：退出
  • .space：空格
  • .up：上
  • .down：下
  • .left：左
  • .right：右

表单输入绑定 ​

1. v-model 的原理是什么？ v-model 是语法糖，本质上是 v-bind 绑定 value 属性和 v-on 绑定 input 事件的结合。

2. v-model 可以用在哪些元素上？

  • 文本输入框
  • 文本域
  • 复选框
  • 单选按钮
  • 下拉选择框

3. v-model 的修饰符有哪些？

  • .lazy：在 change 事件而非 input 事件中更新
  • .number：自动将输入值转为数字类型
  • .trim：自动过滤首尾空格
vue
<!-- 懒更新 -->
<input v-model.lazy="message">

<!-- 数字类型 -->
<input v-model.number="age">

<!-- 去除空格 -->
<input v-model.trim="username">

4. 如何自定义 v-model？ 在组件中，通过 model 选项指定 prop 和 event：

vue
// 子组件
export default {
  model: {
    prop: 'value',
    event: 'input'
  },
  props: {
    value: String
  },
  methods: {
    updateValue(event) {
      this.$emit('input', event.target.value)
    }
  }
}

// 父组件
<custom-input v-model="message"></custom-input>

5. 如何处理表单验证？

  • 使用计算属性验证
  • 使用第三方库如 VeeValidate
  • 自定义验证规则
  • 实时验证或提交时验证

6. 复选框和单选按钮的 v-model？

vue
<!-- 复选框 -->
<input type="checkbox" v-model="checked">

<!-- 多个复选框绑定到数组 -->
<input type="checkbox" value="A" v-model="selected">
<input type="checkbox" value="B" v-model="selected">

<!-- 单选按钮 -->
<input type="radio" value="A" v-model="picked">
<input type="radio" value="B" v-model="picked">

7. 下拉选择框的 v-model？

vue
<!-- 单选 -->
<select v-model="selected">
  <option value="A">A</option>
  <option value="B">B</option>
</select>

<!-- 多选 -->
<select v-model="selected" multiple>
  <option value="A">A</option>
  <option value="B">B</option>
</select>

过渡与动画 ​

1. Vue 的过渡系统是什么？ Vue 提供了内置的过渡系统，可以在元素插入、更新、移除时自动应用过渡效果。

2. 过渡的类名有哪些？

  • v-enter：进入过渡的开始状态
  • v-enter-active：进入过渡的激活状态
  • v-enter-to：进入过渡的结束状态
  • v-leave：离开过渡的开始状态
  • v-leave-active：离开过渡的激活状态
  • v-leave-to：离开过渡的结束状态

3. 如何使用过渡？

vue
<template>
  <button @click="show = !show">Toggle</button>
  <transition name="fade">
    <p v-if="show">Hello</p>
  </transition>
</template>

<style>
.fade-enter-active, .fade-leave-active {
  transition: opacity 0.5s;
}
.fade-enter, .fade-leave-to {
  opacity: 0;
}
</style>

4. 过渡的模式有哪些？

  • in-out：新元素先进入，当前元素再离开
  • out-in：当前元素先离开，新元素再进入
  • 默认模式：同时进行
vue
<transition name="fade" mode="out-in">
  <p v-if="show">Hello</p>
</transition>

5. JavaScript 钩子有哪些？

  • before-enter：进入过渡前
  • enter：进入过渡中
  • after-enter：进入过渡后
  • enter-cancelled：进入过渡取消
  • before-leave：离开过渡前
  • leave：离开过渡中
  • after-leave：离开过渡后
  • leave-cancelled：离开过渡取消
vue
<transition
  @before-enter="beforeEnter"
  @enter="enter"
  @after-enter="afterEnter"
  @before-leave="beforeLeave"
  @leave="leave"
  @after-leave="afterLeave"
>
  <p v-if="show">Hello</p>
</transition>

6. 列表过渡如何使用？ 使用 <transition-group> 组件：

vue
<transition-group name="list" tag="ul">
  <li v-for="item in items" :key="item.id">
    {{ item.text }}
  </li>
</transition-group>

<style>
.list-enter-active, .list-leave-active {
  transition: all 0.5s;
}
.list-enter, .list-leave-to {
  opacity: 0;
  transform: translateX(30px);
}
.list-move {
  transition: transform 0.5s;
}
</style>

7. 如何使用动画？

vue
<transition name="bounce" enter-active-class="animated bounce" leave-active-class="animated bounce">
  <p v-if="show">Hello</p>
</transition>

<style>
@keyframes bounce {
  0%, 100% { transform: scale(0); }
  50% { transform: scale(1.5); }
}

.animated {
  animation-duration: 1s;
  animation-fill-mode: both;
}
</style>

8. 过渡和动画的区别？

  • 过渡：基于 CSS 过渡，适用于简单的进入/离开效果
  • 动画：基于 CSS 动画，适用于复杂的效果
  • 性能：过渡性能更好，动画更灵活

9. 如何优化过渡性能？

  • 使用 transform 和 opacity（触发 GPU 加速）
  • 避免使用 left/top（触发重排）
  • 使用 will-change 提示浏览器
css
.fade-enter-active {
  will-change: opacity;
  transition: opacity 0.3s;
}
