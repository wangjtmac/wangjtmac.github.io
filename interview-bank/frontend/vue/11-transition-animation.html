Vue 过渡与动画 ​

Vue 过渡与动画系统相关面试题。

过渡系统基础 ​

1. Vue 的过渡系统是什么？ Vue 提供了内置的过渡系统，可以在元素插入、更新、移除时自动应用过渡效果。Vue 的过渡系统基于 CSS 过渡和动画，同时支持 JavaScript 钩子。

2. 过渡的类名有哪些？

  • v-enter：进入过渡的开始状态
  • v-enter-active：进入过渡的激活状态
  • v-enter-to：进入过渡的结束状态（Vue 2.1.8+）
  • v-leave：离开过渡的开始状态
  • v-leave-active：离开过渡的激活状态
  • v-leave-to：离开过渡的结束状态（Vue 2.1.8+）

3. 过渡的执行顺序？

  1. 进入过渡：v-enter → v-enter-active → v-enter-to
  2. 离开过渡：v-leave → v-leave-active → v-leave-to

4. 如何使用过渡？

vue
<template>
  <div>
    <button @click="show = !show">Toggle</button>
    <transition name="fade">
      <p v-if="show">Hello Vue!</p>
    </transition>
  </div>
</template>

<script>
export default {
  data() {
    return {
      show: true
    }
  }
}
</script>

<style>
.fade-enter-active, .fade-leave-active {
  transition: opacity 0.5s ease;
}
.fade-enter, .fade-leave-to {
  opacity: 0;
}
</style>

过渡模式 ​

1. 过渡的模式有哪些？

  • 默认模式：进入和离开同时进行
  • in-out：新元素先进入，当前元素再离开
  • out-in：当前元素先离开，新元素再进入

2. 如何使用过渡模式？

vue
<transition name="fade" mode="out-in">
  <component :is="currentComponent"></component>
</transition>

3. 过渡模式的使用场景？

  • out-in：适用于组件切换，避免同时显示两个组件
  • in-out：适用于需要平滑过渡的场景
  • 默认模式：适用于简单的进入/离开效果

4. 过渡模式的注意事项？

  • 使用过渡模式时，确保元素有唯一的 key
  • 过渡模式会影响过渡的性能
  • 选择合适的过渡模式可以提高用户体验

JavaScript 钩子 ​

1. JavaScript 钩子有哪些？

  • before-enter：进入过渡前
  • enter：进入过渡中
  • after-enter：进入过渡后
  • enter-cancelled：进入过渡取消
  • before-leave：离开过渡前
  • leave：离开过渡中
  • after-leave：离开过渡后
  • leave-cancelled：离开过渡取消

2. 如何使用 JavaScript 钩子？

vue
<template>
  <transition
    @before-enter="beforeEnter"
    @enter="enter"
    @after-enter="afterEnter"
    @before-leave="beforeLeave"
    @leave="leave"
    @after-leave="afterLeave"
  >
    <p v-if="show">Hello Vue!</p>
  </transition>
</template>

<script>
export default {
  methods: {
    beforeEnter(el) {
      el.style.opacity = 0
    },
    enter(el, done) {
      el.offsetHeight
      el.style.transition = 'opacity 0.5s'
      el.style.opacity = 1
      done()
    },
    afterEnter(el) {
      console.log('Enter complete')
    },
    beforeLeave(el) {
      el.style.opacity = 1
    },
    leave(el, done) {
      el.style.transition = 'opacity 0.5s'
      el.style.opacity = 0
      done()
    },
    afterLeave(el) {
      console.log('Leave complete')
    }
  }
}
</script>

3. JavaScript 钩子的参数？

  • el：过渡的元素
  • done：回调函数，用于通知 Vue 过渡完成

4. JavaScript 钩子的使用场景？

  • 复杂动画：需要使用 JavaScript 实现复杂动画
  • 动画库集成：集成第三方动画库
  • 性能优化：使用 JavaScript 优化动画性能

列表过渡 ​

1. 什么是列表过渡？ 列表过渡是指对列表中的元素进行过渡效果，使用 <transition-group> 组件实现。

2. 如何使用列表过渡？

vue
<template>
  <div>
    <button @click="add">Add</button>
    <button @click="remove">Remove</button>
    <transition-group name="list" tag="ul">
      <li v-for="item in items" :key="item.id" class="list-item">
        {{ item.text }}
      </li>
    </transition-group>
  </div>
</template>

<script>
export default {
  data() {
    return {
      items: [
        { id: 1, text: 'Item 1' },
        { id: 2, text: 'Item 2' },
        { id: 3, text: 'Item 3' }
      ]
    }
  },
  methods: {
    add() {
      const id = this.items.length + 1
      this.items.push({ id, text: `Item ${id}` })
    },
    remove() {
      this.items.pop()
    }
  }
}
</script>

<style>
.list-enter-active, .list-leave-active {
  transition: all 0.5s ease;
}
.list-enter, .list-leave-to {
  opacity: 0;
  transform: translateX(30px);
}
.list-move {
  transition: transform 0.5s ease;
}
</style>

3. 列表过渡的特殊类名？

  • v-move：当元素的位置发生变化时应用

4. 列表过渡的注意事项？

  • 必须为列表项设置唯一的 key
  • 使用 tag 属性指定包裹元素
  • 使用 v-move 类名实现平滑的移动效果

动画 ​

1. 什么是动画？ 动画是基于 CSS @keyframes 的过渡效果，可以实现更复杂的动画效果。

2. 如何使用动画？

vue
<template>
  <transition name="bounce">
    <p v-if="show">Bounce!</p>
  </transition>
</template>

<style>
@keyframes bounce {
  0%, 100% {
    transform: scale(0);
  }
  50% {
    transform: scale(1.5);
  }
}

.bounce-enter-active {
  animation: bounce 0.5s;
}
.bounce-leave-active {
  animation: bounce 0.5s reverse;
}
</style>

3. 如何使用动画库？

vue
<template>
  <transition
    enter-active-class="animated bounceIn"
    leave-active-class="animated bounceOut"
  >
    <p v-if="show">Animate!</p>
  </transition>
</template>

<style>
@import 'animate.css';
</style>

4. 动画的使用场景？

  • 复杂效果：需要实现复杂的动画效果
  • 动画库：使用第三方动画库
  • 性能优化：使用动画优化性能

过渡与动画的性能优化 ​

1. 如何优化过渡性能？

  • 使用 transform 和 opacity：这些属性不会触发重排，性能更好
  • 避免使用 left/top：这些属性会触发重排，性能较差
  • 使用 will-change：提示浏览器优化动画性能
  • 使用 GPU 加速：使用 transform: translateZ(0) 触发 GPU 加速

2. 如何使用 will-change？

css
.fade-enter-active {
  will-change: opacity;
  transition: opacity 0.5s;
}

3. 如何使用 GPU 加速？

css
.fade-enter-active {
  transform: translateZ(0);
  transition: opacity 0.5s;
}

4. 过渡性能的注意事项？

  • 避免在过渡中使用复杂的 CSS 属性
  • 使用 requestAnimationFrame 优化 JavaScript 动画
  • 使用 debounce 或 throttle 优化频繁触发的事件

过渡与动画的高级用法 ​

1. 如何实现可复用的过渡？

vue
<!-- MyTransition.vue -->
<template>
  <transition
    :name="name"
    @before-enter="beforeEnter"
    @enter="enter"
    @after-enter="afterEnter"
  >
    <slot></slot>
  </transition>
</template>

<script>
export default {
  props: {
    name: {
      type: String,
      default: 'fade'
    }
  },
  methods: {
    beforeEnter(el) {
      el.style.opacity = 0
    },
    enter(el, done) {
      el.offsetHeight
      el.style.transition = 'opacity 0.5s'
      el.style.opacity = 1
      done()
    },
    afterEnter(el) {
      console.log('Enter complete')
    }
  }
}
</script>

<!-- 使用 -->
<my-transition name="fade">
  <p v-if="show">Hello Vue!</p>
</my-transition>

2. 如何实现动态过渡？

vue
<template>
  <transition :name="transitionName">
    <p v-if="show">Hello Vue!</p>
  </transition>
</template>

<script>
export default {
  data() {
    return {
      show: true,
      transitionName: 'fade'
    }
  },
  methods: {
    toggle() {
      this.transitionName = this.show ? 'fade-out' : 'fade-in'
      this.show = !this.show
    }
  }
}
</script>

3. 如何实现交错动画？

vue
<template>
  <transition-group name="list" tag="ul">
    <li
      v-for="(item, index) in items"
      :key="item.id"
      :style="{ transitionDelay: `${index * 0.1}s` }"
    >
      {{ item.text }}
    </li>
  </transition-group>
</template>

<style>
.list-enter-active, .list-leave-active {
  transition: all 0.5s ease;
}
.list-enter, .list-leave-to {
  opacity: 0;
  transform: translateX(30px);
}
</style>

4. 如何实现滚动动画？

vue
<template>
  <div v-scroll="handleScroll">
    <transition name="fade">
      <p v-if="isVisible">Scroll to see me!</p>
    </transition>
  </div>
</template>

<script>
export default {
  data() {
    return {
      isVisible: false
    }
  },
  methods: {
    handleScroll(el) {
      const rect = el.getBoundingClientRect()
      this.isVisible = rect.top < window.innerHeight && rect.bottom > 0
    }
  }
}
</script>

过渡与动画的常见问题 ​

1. 过渡不生效的原因？

  • 没有设置过渡的类名
  • 过渡的类名与 CSS 不匹配
  • 过渡的元素没有唯一的 key
  • 过渡的元素被其他元素遮挡

2. 如何调试过渡？

  • 使用 Vue DevTools 查看过渡状态
  • 使用浏览器的开发者工具查看 CSS
  • 使用 console.log 输出过渡信息

3. 过渡的性能问题？

  • 过渡使用了复杂的 CSS 属性
  • 过渡没有使用 GPU 加速
  • 过渡的元素过多

4. 如何解决过渡的性能问题？

  • 使用 transform 和 opacity 代替其他属性
  • 使用 will-change 提示浏览器优化
  • 使用 requestAnimationFrame 优化 JavaScript 动画
  • 减少 DOM 操作

过渡与动画的最佳实践 ​

1. 过渡的使用原则？

  • 保持简单：使用简单的过渡效果
  • 性能优先：使用性能好的 CSS 属性
  • 用户体验：提供良好的用户体验
  • 可维护性：保持代码的可维护性

2. 过渡的命名规范？

  • 使用语义化的名称
  • 使用短横线分隔
  • 避免使用保留字

3. 过渡的代码组织？

  • 将过渡相关的代码放在同一个文件中
  • 使用可复用的过渡组件
  • 使用 CSS 变量管理过渡参数

4. 过渡的测试？

  • 测试过渡的正确性
  • 测试过渡的性能
  • 测试过渡的兼容性
