Vue 响应式系统 ​

Vue 响应式数据相关面试题。

数据响应式原理 ​

1. 什么是数据响应式？ 数据响应式是指当数据发生变化时，视图会自动更新，无需手动操作 DOM。Vue 的响应式系统是其核心特性之一。

2. Vue 2 的响应式系统是如何实现的？ Vue 2 使用 Object.defineProperty 方法对数据对象的属性进行劫持，当属性被访问或修改时，触发相应的 getter 和 setter 函数，从而实现依赖收集和派发更新。

3. Vue 3 的响应式系统是如何实现的？ Vue 3 使用 Proxy 对象对数据进行代理，相比 Object.defineProperty，Proxy 可以监听对象的新增属性、删除属性、数组的变化等更多场景。

Object.defineProperty 与 Proxy ​

1. Object.defineProperty 的实现原理？

javascript
function defineReactive(obj, key, val) {
  Object.defineProperty(obj, key, {
    enumerable: true,
    configurable: true,
    get() {
      // 依赖收集
      if (Dep.target) {
        dep.depend()
      }
      return val
    },
    set(newVal) {
      if (val === newVal) return
      val = newVal
      // 派发更新
      dep.notify()
    }
  })
}

2. Object.defineProperty 的缺点有哪些？

  • 无法监听数组的变化：需要特殊处理数组方法
  • 无法监听对象的新增属性：需要使用 Vue.set 或 this.$set
  • 无法监听对象的删除属性：需要使用 Vue.delete 或 this.$delete
  • 性能开销：需要递归遍历对象的所有属性

3. Proxy 的实现原理？

javascript
function reactive(obj) {
  return new Proxy(obj, {
    get(target, key, receiver) {
      // 依赖收集
      track(target, key)
      return Reflect.get(target, key, receiver)
    },
    set(target, key, value, receiver) {
      const oldValue = target[key]
      const result = Reflect.set(target, key, value, receiver)
      // 派发更新
      if (oldValue !== value) {
        trigger(target, key)
      }
      return result
    }
  })
}

4. Proxy 相比 Object.defineProperty 有哪些优势？

  • 可以监听数组的变化：不需要特殊处理数组方法
  • 可以监听对象的新增属性：自动监听
  • 可以监听对象的删除属性：自动监听
  • 可以监听 Symbol 类型的属性：支持 Symbol 作为键
  • 性能更好：不需要递归遍历所有属性

依赖收集与派发更新 ​

1. 什么是依赖收集？ 依赖收集是指在访问响应式数据时，将当前的 Watcher（观察者）记录到 Dep（依赖收集器）中，以便在数据变化时通知这些 Watcher 进行更新。

2. 什么是派发更新？ 派发更新是指在修改响应式数据时，通知所有依赖该数据的 Watcher 进行更新，从而触发视图的重新渲染。

3. Dep 和 Watcher 的关系？

  • Dep：依赖收集器，每个响应式数据都有一个 Dep 实例，用于存储所有依赖该数据的 Watcher
  • Watcher：观察者，每个组件或计算属性都有一个 Watcher 实例，用于监听数据的变化并执行相应的更新逻辑

4. 依赖收集和派发更新的流程？

  1. 初始化：创建 Dep 和 Watcher 实例
  2. 依赖收集：在访问响应式数据时，将当前的 Watcher 添加到 Dep 中
  3. 数据变化：在修改响应式数据时，触发 setter
  4. 派发更新：setter 中通知 Dep 中的所有 Watcher
  5. 执行更新：Watcher 执行更新逻辑，触发视图的重新渲染

计算属性的实现 ​

1. 计算属性的原理是什么？ 计算属性是基于响应式数据计算得出的属性，具有缓存特性。只有当依赖的响应式数据变化时，才会重新计算。

2. 计算属性的缓存机制？

javascript
class ComputedRef {
  constructor(getter) {
    this._getter = getter
    this._value = undefined
    this._dirty = true
    this._dep = new Dep()
  }
  
  get value() {
    if (this._dirty) {
      this._value = this._getter()
      this._dirty = false
    }
    // 依赖收集
    if (Dep.target) {
      this._dep.depend()
    }
    return this._value
  }
  
  update() {
    this._dirty = true
    // 派发更新
    this._dep.notify()
  }
}

3. 计算属性与方法的区别？

  • 计算属性：有缓存，依赖数据变化时才重新计算
  • 方法：每次调用都会重新执行
  • 使用场景：需要缓存结果用计算属性，需要实时计算用方法

4. 计算属性的注意事项？

  • 不能在计算属性中修改数据：计算属性应该是纯函数
  • 避免在计算属性中执行异步操作：计算属性应该是同步的
  • 合理使用计算属性：对于复杂的计算逻辑，使用计算属性可以提高性能

监听器的实现 ​

1. 监听器的原理是什么？ 监听器用于监听数据变化并执行回调函数。Vue 内部为每个监听器创建一个 Watcher 实例，当数据变化时，Watcher 执行回调函数。

2. 监听器的配置选项？

javascript
watch: {
  // 简单监听
  count(newVal, oldVal) {
    console.log(`count 从 ${oldVal} 变为 ${newVal}`)
  },
  
  // 深度监听
  user: {
    handler(newVal, oldVal) {
      console.log('user 变化了')
    },
    deep: true
  },
  
  // 立即执行
  message: {
    handler(newVal) {
      console.log('message 变化了', newVal)
    },
    immediate: true
  }
}

3. 监听器的使用场景？

  • 数据变化时执行副作用操作：如网络请求、DOM 操作等
  • 异步操作：需要在数据变化时执行异步操作
  • 复杂逻辑：需要在数据变化时执行复杂的逻辑

4. 监听器的注意事项？

  • 避免在监听器中修改被监听的数据：可能导致无限循环
  • 合理使用 deep 选项：深度监听会影响性能
  • 清理监听器：在组件销毁时清理监听器，避免内存泄漏

响应式数据的注意事项 ​

1. Vue 2 中响应式数据的限制？

  • 无法监听数组索引的变化：需要使用 Vue.set 或 this.$set
  • 无法监听对象的新增属性：需要使用 Vue.set 或 this.$set
  • 无法监听对象的删除属性：需要使用 Vue.delete 或 this.$delete

2. 如何在 Vue 2 中解决响应式数据的限制？

javascript
// 新增响应式属性
this.$set(this.obj, 'newProp', value)

// 删除响应式属性
this.$delete(this.obj, 'prop')

// 修改数组索引
this.$set(this.list, index, newValue)

3. Vue 3 中响应式数据的改进？

  • 可以监听数组的变化：不需要特殊处理
  • 可以监听对象的新增属性：自动监听
  • 可以监听对象的删除属性：自动监听
  • 更好的性能：使用 Proxy 提高了性能

4. 响应式数据的性能优化？

  • 避免过度使用响应式数据：不是所有数据都需要是响应式的
  • 合理使用计算属性：利用计算属性的缓存特性
  • 避免在模板中使用复杂表达式：将复杂表达式移到计算属性中
  • 使用 Object.freeze：对于不需要响应式的数据，使用 Object.freeze 冻结对象

Vue 3 的响应式系统 ​

1. Vue 3 的响应式 API 有哪些？

  • ref：用于创建响应式的基本类型数据
  • reactive：用于创建响应式的对象类型数据
  • computed：用于创建计算属性
  • watch：用于监听数据变化
  • watchEffect：用于自动追踪响应式数据的变化
  • readonly：用于创建只读的响应式数据
  • shallowRef：用于创建浅响应式的基本类型数据
  • shallowReactive：用于创建浅响应式的对象类型数据

2. ref 与 reactive 的区别？

  • ref：用于包装基本类型数据，也可以包装对象，访问时需要使用 .value
  • reactive：用于包装对象类型数据，返回的是代理对象，不需要使用 .value
  • 使用场景：基本类型数据使用 ref，对象类型数据使用 reactive

3. watch 与 watchEffect 的区别？

  • watch：需要明确指定监听的数据源，只有当数据源变化时才会执行回调
  • watchEffect：自动追踪回调函数中使用的响应式数据，当这些数据变化时执行回调
  • 执行时机：watch 在组件挂载后执行，watchEffect 在组件挂载前执行

4. 如何在 Vue 3 中使用响应式 API？

vue
<script setup>
import { ref, reactive, computed, watch, watchEffect } from 'vue'

// 使用 ref
const count = ref(0)
count.value++

// 使用 reactive
const user = reactive({ name: '张三', age: 18 })
user.name = '李四'

// 使用 computed
const doubleCount = computed(() => count.value * 2)

// 使用 watch
watch(count, (newValue, oldValue) => {
  console.log(`count 从 ${oldValue} 变为 ${newValue}`)
})

// 使用 watchEffect
watchEffect(() => {
  console.log(`count 的值为 ${count.value}`)
})
</script>
