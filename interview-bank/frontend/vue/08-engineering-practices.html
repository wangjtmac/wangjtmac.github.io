Vue 工程化实践 ​

Vue 工程化与最佳实践相关面试题。

Vue CLI 的使用 ​

1. 什么是 Vue CLI？ Vue CLI 是 Vue.js 官方提供的脚手架工具，用于快速创建 Vue 项目，配置开发环境，构建生产版本等。

2. Vue CLI 的安装方式？

  • npm 安装：npm install -g @vue/cli
  • yarn 安装：yarn global add @vue/cli

3. Vue CLI 的基本命令？

  • 创建项目：vue create project-name
  • 启动开发服务器：npm run serve 或 yarn serve
  • 构建生产版本：npm run build 或 yarn build
  • lint 代码：npm run lint 或 yarn lint
  • 查看版本：vue --version
  • 添加插件：vue add plugin-name

4. Vue CLI 项目的目录结构？

project-name/
├── public/            # 静态资源
├── src/               # 源代码
│   ├── assets/        # 资源文件
│   ├── components/    # 组件
│   ├── views/         # 视图
│   ├── router/        # 路由
│   ├── store/         # 状态管理
│   ├── App.vue        # 根组件
│   └── main.js        # 入口文件
├── .eslintrc.js       # ESLint 配置
├── .gitignore         # Git 忽略文件
├── babel.config.js    # Babel 配置
├── package.json       # 项目配置
└── vue.config.js      # Vue CLI 配置

5. Vue CLI 的配置文件是什么？ Vue CLI 的配置文件是 vue.config.js，用于自定义 Vue CLI 的配置，如代理、打包输出目录、插件配置等。

Vite 的使用 ​

1. 什么是 Vite？ Vite 是 Vue 的作者尤雨溪开发的新一代前端构建工具，基于 ES 模块的原生支持，提供了更快的开发服务器启动速度和热更新速度。

2. Vite 与 Vue CLI 的区别？

  • 开发服务器：Vite 使用原生 ES 模块，启动速度更快，热更新更及时
  • 构建速度：Vite 使用 Rollup 进行生产构建，速度更快
  • 配置：Vite 的配置更简洁，基于 ESM
  • 兼容性：Vite 要求浏览器支持 ES 模块，不支持 IE11
  • 插件系统：Vite 有自己的插件系统，与 Rollup 插件兼容

3. Vite 的安装方式？

  • 创建项目：npm create vite@latest project-name -- --template vue
  • yarn：yarn create vite project-name -- --template vue
  • pnpm：pnpm create vite project-name -- --template vue

4. Vite 的基本命令？

  • 启动开发服务器：npm run dev 或 yarn dev
  • 构建生产版本：npm run build 或 yarn build
  • 预览生产构建：npm run preview 或 yarn preview

5. Vite 的配置文件是什么？ Vite 的配置文件是 vite.config.js 或 vite.config.ts，用于自定义 Vite 的配置，如代理、插件、构建选项等。

构建工具配置 ​

1. 如何配置 Vue CLI 的代理？ 在 vue.config.js 中配置：

javascript
// vue.config.js
module.exports = {
  devServer: {
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true,
        pathRewrite: {
          '^/api': ''
        }
      }
    }
  }
}

2. 如何配置 Vite 的代理？ 在 vite.config.js 中配置：

javascript
// vite.config.js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  }
})

3. 如何配置构建输出目录？

  • Vue CLI：在 vue.config.js 中配置 outputDir
  • Vite：在 vite.config.js 中配置 build.outDir

4. 如何配置别名？

  • Vue CLI：
javascript
// vue.config.js
module.exports = {
  configureWebpack: {
    resolve: {
      alias: {
        '@': path.resolve(__dirname, 'src')
      }
    }
  }
}
  • Vite：
javascript
// vite.config.js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import path from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src')
    }
  }
})

代码规范与 ESLint ​

1. 什么是 ESLint？ ESLint 是一个 JavaScript 代码检查工具，用于识别和报告代码中的模式问题，帮助开发者编写更一致、更高质量的代码。

2. 如何在 Vue 项目中配置 ESLint？

  • Vue CLI：创建项目时选择 ESLint 选项，或使用 vue add eslint 添加
  • Vite：使用 npm create vite@latest 创建项目时选择 ESLint 选项
  • 手动配置：创建 .eslintrc.js 文件，配置 ESLint 规则

3. 常用的 ESLint 规则有哪些？

  • semi：是否使用分号
  • quotes：使用单引号还是双引号
  • indent：缩进空格数
  • no-console：是否允许使用 console
  • no-unused-vars：是否允许未使用的变量
  • no-undef：是否允许未定义的变量

4. 如何集成 Prettier 与 ESLint？

  • 安装依赖：npm install --save-dev prettier eslint-plugin-prettier eslint-config-prettier
  • 配置 ESLint：在 .eslintrc.js 中添加 Prettier 配置
  • 创建 Prettier 配置：创建 .prettierrc 文件，配置 Prettier 规则

5. 如何在 VS Code 中自动修复 ESLint 错误？

  • 安装 ESLint 插件：在 VS Code 中安装 ESLint 插件
  • 配置 VS Code：在 settings.json 中添加自动修复配置
json
{
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  }
}

单元测试 ​

1. 什么是单元测试？ 单元测试是指对软件中的最小可测试单元进行测试，如函数、组件等，以确保它们的行为符合预期。

2. Vue 项目中常用的单元测试框架有哪些？

  • Jest：由 Facebook 开发的 JavaScript 测试框架，功能强大，配置简单
  • Mocha：灵活的 JavaScript 测试框架，需要配合其他库使用
  • Vitest：Vite 官方推荐的测试框架，基于 Vite，速度快

3. 如何在 Vue 项目中配置单元测试？

  • Vue CLI：创建项目时选择单元测试选项，或使用 vue add unit-jest 添加
  • Vite：使用 npm create vite@latest 创建项目时选择单元测试选项，或安装 Vitest

4. 如何编写 Vue 组件的单元测试？

  • 使用 @vue/test-utils：Vue 官方提供的测试工具库
  • 测试组件的渲染：测试组件是否正确渲染
  • 测试组件的 props：测试组件对 props 的响应
  • 测试组件的事件：测试组件是否正确触发事件
  • 测试组件的方法：测试组件的方法是否正确执行

5. 单元测试的最佳实践？

  • 测试覆盖率：保持较高的测试覆盖率，但不要追求 100%
  • 测试命名：使用清晰、语义化的测试命名
  • 测试隔离：每个测试应该独立，不依赖其他测试的状态
  • 测试速度：保持测试运行速度快，便于频繁执行
  • 测试维护：定期更新和维护测试，确保测试与代码同步

端到端测试 ​

1. 什么是端到端测试？ 端到端测试（E2E 测试）是指模拟用户操作，对整个应用进行测试，从用户的角度验证应用的功能是否正常。

2. Vue 项目中常用的端到端测试框架有哪些？

  • Cypress：现代的端到端测试框架，提供了友好的测试界面和强大的 API
  • Puppeteer：由 Google 开发的 Node.js 库，用于控制 Chrome 或 Chromium 浏览器
  • Nightwatch.js：基于 Selenium 的端到端测试框架

3. 如何在 Vue 项目中配置端到端测试？

  • Vue CLI：创建项目时选择 E2E 测试选项，或使用 vue add e2e-cypress 添加
  • Vite：安装 Cypress 或其他测试框架，手动配置

4. 如何编写端到端测试？

  • 测试用户流程：模拟用户的操作流程，如登录、浏览、点击等
  • 测试页面渲染：测试页面是否正确渲染
  • 测试表单提交：测试表单提交是否正常
  • 测试导航：测试路由导航是否正常
  • 测试错误处理：测试应用对错误的处理是否正确

5. 端到端测试的最佳实践？

  • 测试关键流程：优先测试应用的关键流程，如登录、支付等
  • 测试环境：使用与生产环境相似的测试环境
  • 测试数据：使用干净、可预测的测试数据
  • 测试速度：端到端测试较慢，建议在 CI/CD 中执行，而不是本地频繁执行
  • 测试维护：定期更新和维护测试，确保测试与应用同步

持续集成与部署 ​

1. 什么是持续集成（CI）？ 持续集成是指开发者频繁地将代码集成到主干分支，每次集成都会自动运行测试，确保代码的质量和稳定性。

2. 什么是持续部署（CD）？ 持续部署是指在代码通过测试后，自动将代码部署到生产环境，减少手动操作，提高部署效率。

3. 常用的 CI/CD 工具有哪些？

  • GitHub Actions：GitHub 提供的 CI/CD 服务，与 GitHub 仓库集成
  • GitLab CI/CD：GitLab 提供的 CI/CD 服务，与 GitLab 仓库集成
  • Jenkins：开源的 CI/CD 工具，功能强大，可高度定制
  • Travis CI：基于云的 CI 服务，与 GitHub 集成良好
  • CircleCI：基于云的 CI/CD 服务，速度快，配置简单

4. 如何配置 GitHub Actions 用于 Vue 项目？ 创建 .github/workflows/ci.yml 文件：

yaml
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Use Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '16'
    - run: npm ci
    - run: npm run build
    - run: npm run test

5. 如何部署 Vue 应用？

  • 静态网站托管：部署到 Netlify、Vercel、GitHub Pages 等
  • 服务器部署：部署到 Nginx、Apache 等服务器
  • 容器部署：使用 Docker 容器部署
  • 云服务：部署到 AWS、Azure、Google Cloud 等云服务

6. 部署 Vue 应用的最佳实践？

  • 环境变量：使用环境变量管理不同环境的配置
  • 构建优化：使用生产构建，优化打包体积
  • 缓存策略：合理设置缓存策略，提高访问速度
  • CDN 加速：使用 CDN 加速静态资源
  • 监控：部署后监控应用的运行状态，及时发现问题

自定义指令 ​

1. 什么是自定义指令？ 自定义指令是 Vue 提供的一种机制，允许开发者创建自定义的指令，用于在模板中执行底层的 DOM 操作。

2. 如何注册自定义指令？

  • 全局注册：使用 Vue.directive('directive-name', { ... }) 注册
  • 局部注册：在组件的 directives 选项中注册

3. 自定义指令的钩子函数有哪些？

  • bind：只调用一次，指令第一次绑定到元素时调用
  • inserted：被绑定元素插入父节点时调用
  • update：所在组件的 VNode 更新时调用
  • componentUpdated：指令所在组件的 VNode 及其子 VNode 全部更新后调用
  • unbind：只调用一次，指令与元素解绑时调用

4. 如何创建自定义指令？

javascript
// 全局注册
Vue.directive('focus', {
  inserted: function (el) {
    el.focus()
  }
})

// 局部注册
export default {
  directives: {
    focus: {
      inserted: function (el) {
        el.focus()
      }
    }
  }
}

5. 指令钩子函数的参数有哪些？

  • el：指令所绑定的元素
  • binding：一个对象，包含指令的名称、值、表达式、参数、修饰符等
  • vnode：Vue 编译生成的虚拟节点
  • oldVnode：上一个虚拟节点，仅在 update 和 componentUpdated 钩子中可用

6. 如何创建带参数的自定义指令？

javascript
Vue.directive('color', {
  bind: function (el, binding) {
    el.style.color = binding.value
  }
})

// 使用
<p v-color="'red'">Hello</p>

7. 如何创建带修饰符的自定义指令？

javascript
Vue.directive('color', {
  bind: function (el, binding) {
    if (binding.modifiers.bold) {
      el.style.fontWeight = 'bold'
    }
    el.style.color = binding.value
  }
})

// 使用
<p v-color.bold="'red'">Hello</p>

8. 自定义指令的使用场景？

  • DOM 操作：需要直接操作 DOM 的场景
  • 第三方库集成：集成需要直接操作 DOM 的第三方库
  • 权限控制：根据权限控制元素的显示/隐藏
  • 表单验证：自定义表单验证逻辑
  • 自动聚焦：自动聚焦到输入框

自定义插件 ​

1. 什么是自定义插件？ 插件是 Vue 提供的一种机制，允许开发者创建可复用的功能，并在多个 Vue 应用中使用。

2. 如何创建自定义插件？

javascript
// my-plugin.js
const MyPlugin = {
  install(Vue, options) {
    // 添加全局方法或属性
    Vue.myGlobalMethod = function () {
      console.log('Global method')
    }

    // 添加全局资源
    Vue.directive('my-directive', {
      bind(el, binding, vnode, oldVnode) {
        // 指令逻辑
      }
    })

    // 注入组件选项
    Vue.mixin({
      created() {
        // 混入逻辑
      }
    })

    // 添加实例方法
    Vue.prototype.$myMethod = function (methodOptions) {
      // 实例方法逻辑
    }
  }
}

export default MyPlugin

3. 如何使用自定义插件？

javascript
import MyPlugin from './my-plugin'

// 全局安装
Vue.use(MyPlugin, {
  someOption: true
})

// 使用
Vue.myGlobalMethod()
this.$myMethod()

4. 插件的参数是什么？ 插件的 install 方法接收两个参数：

  • Vue：Vue 构造函数
  • options：可选的配置对象

5. 插件的使用场景？

  • 全局功能：需要在全局范围内使用的功能
  • 第三方库集成：集成第三方库到 Vue 应用
  • 全局方法：添加全局方法或属性
  • 全局资源：添加全局指令、过滤器、组件等
  • 混入：向所有组件注入混入

6. 如何创建一个状态管理插件？

javascript
const StorePlugin = {
  install(Vue, options) {
    const store = new Store(options)
    Vue.prototype.$store = store
    Vue.mixin({
      beforeCreate() {
        this.$store = store
      }
    })
  }
}

class Store {
  constructor(options) {
    this.state = options.state || {}
    this.mutations = options.mutations || {}
    this.actions = options.actions || {}
  }

  commit(type, payload) {
    const mutation = this.mutations[type]
    if (mutation) {
      mutation(this.state, payload)
    }
  }

  dispatch(type, payload) {
    const action = this.actions[type]
    if (action) {
      action(this, payload)
    }
  }
}

export default StorePlugin

Vue 测试 ​

1. Vue 单元测试的工具是什么？ Vue 单元测试主要使用 @vue/test-utils，它是 Vue 官方提供的测试工具库，用于测试 Vue 组件。

2. 如何测试 Vue 组件？

javascript
import { mount } from '@vue/test-utils'
import MyComponent from '@/components/MyComponent.vue'

describe('MyComponent', () => {
  it('renders correctly', () => {
    const wrapper = mount(MyComponent)
    expect(wrapper.text()).toContain('Hello')
  })

  it('emits event on click', async () => {
    const wrapper = mount(MyComponent)
    await wrapper.find('button').trigger('click')
    expect(wrapper.emitted('click')).toBeTruthy()
  })
})

3. 如何测试组件的 props？

javascript
it('renders props correctly', () => {
  const message = 'Test Message'
  const wrapper = mount(MyComponent, {
    propsData: { message }
  })
  expect(wrapper.text()).toContain(message)
})

4. 如何测试组件的事件？

javascript
it('emits event with correct payload', async () => {
  const wrapper = mount(MyComponent)
  await wrapper.find('button').trigger('click')
  expect(wrapper.emitted('click')[0]).toEqual(['payload'])
})

5. 如何测试组件的方法？

javascript
it('calls method correctly', () => {
  const wrapper = mount(MyComponent)
  const spy = jest.spyOn(wrapper.vm, 'myMethod')
  wrapper.vm.myMethod()
  expect(spy).toHaveBeenCalled()
})

6. 如何测试组件的插槽？

javascript
it('renders slot content', () => {
  const wrapper = mount(MyComponent, {
    slots: {
      default: '<p>Slot Content</p>'
    }
  })
  expect(wrapper.html()).toContain('Slot Content')
})

7. 如何测试组件的异步操作？

javascript
it('handles async operation', async () => {
  const wrapper = mount(MyComponent)
  await wrapper.vm.fetchData()
  expect(wrapper.vm.data).toBeDefined()
})

8. Vue 测试的最佳实践？

  • 隔离测试：每个测试应该独立，不依赖其他测试
  • 测试覆盖率：保持较高的测试覆盖率
  • 测试命名：使用清晰、语义化的测试命名
  • 测试速度：保持测试运行速度快
  • Mock 外部依赖：使用 mock 模拟外部依赖

Vue DevTools ​

1. 什么是 Vue DevTools？ Vue DevTools 是 Vue 官方提供的浏览器扩展，用于调试 Vue 应用，可以查看组件树、状态、事件等信息。

2. Vue DevTools 的功能有哪些？

  • 组件树：查看应用的组件树结构
  • 组件状态：查看和修改组件的数据和计算属性
  • 事件：查看组件触发的事件
  • Vuex/Pinia：查看和修改状态管理的数据
  • 路由：查看当前路由信息
  • 性能：查看组件渲染性能
  • 时间旅行：回溯和重放状态变化

3. 如何使用 Vue DevTools？

  • 安装扩展：在浏览器中安装 Vue DevTools 扩展
  • 打开面板：按 F12 打开开发者工具，选择 Vue 面板
  • 查看组件：在组件树中选择组件，查看组件信息
  • 修改状态：在组件状态面板中修改数据

4. Vue DevTools 的高级功能？

  • 组件过滤：过滤组件树中的组件
  • 性能分析：分析组件渲染性能
  • 事件追踪：追踪组件触发的事件
  • 状态快照：保存和恢复状态快照
  • 自定义配置：自定义 DevTools 的配置

5. 如何在生产环境禁用 Vue DevTools？

javascript
// Vue 2
Vue.config.devtools = false

// Vue 3
const app = createApp(App)
app.config.devtools = false
