Vue 组件开发 ​

Vue 组件化开发相关面试题。

组件的定义与注册 ​

1. 什么是组件？ 组件是 Vue 中可复用的 UI 单元，是构成页面的基本 building block。每个组件都有自己的模板、数据、方法和生命周期。

2. 组件的注册方式有哪些？

  • 全局注册：使用 Vue.component('component-name', { ... }) 注册，在整个应用中可用
  • 局部注册：在组件的 components 选项中注册，只在当前组件中可用
  • 动态注册：使用 Vue.extend 创建组件构造器，然后实例化

全局注册示例：

javascript
// main.js
import Vue from 'vue'

// 全局注册组件
Vue.component('my-button', {
  template: '<button @click="handleClick">{{ text }}</button>',
  props: {
    text: {
      type: String,
      default: '点击我'
    }
  },
  methods: {
    handleClick() {
      this.$emit('click')
    }
  }
})

// 在任何组件中都可以直接使用
// <my-button text="提交"></my-button>

局部注册示例：

javascript
// ParentComponent.vue
import ChildComponent from './ChildComponent.vue'

export default {
  components: {
    ChildComponent
  },
  template: `
    <div>
      <child-component></child-component>
    </div>
  `
}

// 或者使用 .vue 文件
// ParentComponent.vue
<template>
  <div>
    <child-component></child-component>
  </div>
</template>

<script>
import ChildComponent from './ChildComponent.vue'

export default {
  components: {
    ChildComponent
  }
}
</script>

动态注册（Vue.extend）示例：

javascript
// 1. 使用 Vue.extend 创建组件构造器
const MyComponentConstructor = Vue.extend({
  template: '<div>{{ message }}</div>',
  data() {
    return {
      message: 'Hello from Vue.extend!'
    }
  },
  methods: {
    greet() {
      console.log('Greeting from component!')
    }
  }
})

// 2. 实例化组件
const componentInstance = new MyComponentConstructor({
  data() {
    return {
      message: 'Custom message'
    }
  }
})

// 3. 将组件挂载到 DOM
componentInstance.$mount('#app')
// 或者手动挂载
// componentInstance.$mount()
// document.body.appendChild(componentInstance.$el)

动态创建组件实例示例：

javascript
// 动态创建并挂载 Alert 组件
function createAlert(message, type = 'info') {
  const AlertConstructor = Vue.extend({
    template: `
      <div class="alert" :class="type">
        <slot></slot>
        <button @click="close">关闭</button>
      </div>
    `,
    props: {
      type: {
        type: String,
        default: 'info'
      }
    },
    methods: {
      close() {
        this.$destroy()
        this.$el.remove()
      }
    }
  })
  
  const instance = new AlertConstructor({
    propsData: { type }
  })
  
  instance.$mount()
  document.body.appendChild(instance.$el)
  
  return instance
}

创建可复用的组件工厂示例：

javascript
// 创建模态框组件工厂
const ModalFactory = Vue.extend({
  template: `
    <div class="modal-overlay" v-if="visible">
      <div class="modal">
        <div class="modal-header">
          <h3>{{ title }}</h3>
          <button @click="close">×</button>
        </div>
        <div class="modal-body">
          <slot></slot>
        </div>
        <div class="modal-footer">
          <button @click="confirm">确定</button>
          <button @click="close">取消</button>
        </div>
      </div>
    </div>
  `,
  data() {
    return {
      visible: false,
      title: '提示'
    }
  },
  methods: {
    show() {
      this.visible = true
    },
    close() {
      this.visible = false
    },
    confirm() {
      this.$emit('confirm')
      this.close()
    }
  }
})

// 使用工厂创建模态框实例
function createModal(title) {
  const modal = new ModalFactory({
    propsData: { title }
  })
  modal.$mount()
  document.body.appendChild(modal.$el)
  return modal
}

Toast 提示组件示例：

javascript
// 创建 Toast 提示组件
const ToastConstructor = Vue.extend({
  template: `
    <div class="toast" :class="type" :style="{ top: position + 'px' }">
      {{ message }}
    </div>
  `,
  data() {
    return {
      message: '',
      type: 'info',
      position: 20,
      duration: 3000
    }
  },
  mounted() {
    setTimeout(() => {
      this.close()
    }, this.duration)
  },
  methods: {
    close() {
      this.$el.style.opacity = '0'
      setTimeout(() => {
        this.$destroy()
        this.$el.remove()
      }, 300)
    }
  }
})

// Toast 工具函数
const Toast = {
  show(message, type = 'info') {
    const toast = new ToastConstructor({
      data() {
        return { message, type }
      }
    })
    toast.$mount()
    document.body.appendChild(toast.$el)
    return toast
  },
  success(message) {
    return this.show(message, 'success')
  },
  error(message) {
    return this.show(message, 'error')
  }
}

Vue.extend 的特点：

  • 灵活性：可以在运行时动态创建组件实例
  • 独立性：每个实例都是独立的，有自己的状态
  • 手动控制：需要手动管理实例的生命周期（挂载、销毁）
  • 编程式：适合需要编程式创建组件的场景

使用注意事项：

  • 内存管理：手动创建的实例需要手动销毁，避免内存泄漏
  • DOM 管理：需要手动将组件挂载到 DOM 中
  • 生命周期：手动创建的实例生命周期需要开发者自己控制
  • Vue 3 变化：在 Vue 3 中，Vue.extend 已被废弃，推荐使用 defineComponent 和 createApp

Vue 3 中的替代方案：

javascript
// Vue 3 中使用 createApp
import { createApp, defineComponent } from 'vue'

// 定义组件
const MyComponent = defineComponent({
  template: '<div>{{ message }}</div>',
  data() {
    return {
      message: 'Hello Vue 3'
    }
  }
})

// 创建应用实例
const app = createApp(MyComponent)
const instance = app.mount('#app')

3. 组件命名的规范？

  • 短横线分隔：推荐使用短横线分隔的命名方式，如 my-component
  • PascalCase：在组件定义时可以使用 PascalCase 命名，如 MyComponent
  • 避免保留字：不要使用 HTML 标签名或 Vue 保留字作为组件名
  • 语义化：组件名应该语义化，清晰表达组件的功能

4. 全局注册与局部注册的区别？

  • 全局注册：在任何组件中都可以直接使用，不需要导入
  • 局部注册：只在注册它的组件中可用，需要导入
  • 性能影响：全局注册的组件会增加打包体积，局部注册更加灵活

组件通信方式 ​

1. 组件通信的方式有哪些？

  • Props/Events：父组件通过 props 向子组件传递数据，子组件通过 events 向父组件传递事件
  • $refs：父组件通过 $refs 直接访问子组件的实例
  • $parent/$children：通过 $parent 访问父组件实例，通过 $children 访问子组件实例
  • provide/inject：父组件通过 provide 提供数据，子组件通过 inject 注入数据
  • Vuex：使用状态管理库进行全局状态管理
  • 事件总线：使用 Vue 实例作为事件总线，进行组件间通信
  • $attrs/$listeners：传递未被 props 接收的属性和事件监听器

2. Props 的特点和使用注意事项？

  • 单向数据流：props 是单向数据流，子组件不应该修改 props
  • 类型验证：可以为 props 指定类型，如 String、Number、Boolean、Array、Object、Function 等
  • 默认值：可以为 props 设置默认值
  • 必传验证：可以设置 props 为必填项
  • 对象默认值：对象和数组的默认值需要通过函数返回

3. Events 的使用方式？

  • 子组件触发事件：使用 this.$emit('event-name', payload) 触发事件
  • 父组件监听事件：使用 @event-name="handler" 监听事件
  • 事件修饰符：可以使用 .native 修饰符监听原生事件

4. provide/inject 的使用场景？

  • 深层组件通信：当组件层级较深时，使用 provide/inject 可以避免 props drilling
  • 插件开发：插件可以通过 provide 向应用注入功能
  • 注意事项：provide/inject 不是响应式的，除非传递的是响应式对象

5. 事件总线的实现方式？

javascript
// 创建事件总线
const bus = new Vue()

// 发送事件
bus.$emit('event-name', data)

// 监听事件
bus.$on('event-name', (data) => {
  // 处理事件
})

// 移除事件监听
bus.$off('event-name')

插槽的使用 ​

1. 什么是插槽？ 插槽（Slot）是 Vue 中用于组件内容分发的机制，允许父组件向子组件的指定位置插入内容。

2. 插槽的类型有哪些？

  • 默认插槽：没有名称的插槽，父组件的内容会插入到默认插槽中
  • 具名插槽：有名称的插槽，父组件可以将内容插入到指定名称的插槽中
  • 作用域插槽：子组件可以向父组件传递数据的插槽

3. 默认插槽的使用方式？

vue
<!-- 子组件 -->
<template>
  <div>
    <slot></slot>
  </div>
</template>

<!-- 父组件 -->
<child-component>
  <p>这是插入到默认插槽的内容</p>
</child-component>

4. 具名插槽的使用方式？

vue
<!-- 子组件 -->
<template>
  <div>
    <header>
      <slot name="header"></slot>
    </header>
    <main>
      <slot></slot>
    </main>
    <footer>
      <slot name="footer"></slot>
    </footer>
  </div>
</template>

<!-- 父组件 -->
<child-component>
  <template v-slot:header>
    <h1>页面标题</h1>
  </template>
  <p>这是正文内容</p>
  <template v-slot:footer>
    <p>版权信息</p>
  </template>
</child-component>

5. 作用域插槽的使用方式？

vue
<!-- 子组件 -->
<template>
  <div>
    <slot :user="user" :age="age"></slot>
  </div>
</template>

<script>
export default {
  data() {
    return {
      user: '张三',
      age: 18
    }
  }
}
</script>

<!-- 父组件 -->
<child-component>
  <template v-slot="scope">
    <p>用户名：{{ scope.user }}</p>
    <p>年龄：{{ scope.age }}</p>
  </template>
</child-component>

动态组件 ​

1. 什么是动态组件？ 动态组件是指可以根据数据动态切换的组件，使用 <component :is="componentName"> 语法实现。

2. 动态组件的使用方式？

vue
<template>
  <div>
    <button @click="currentComponent = 'ComponentA'">组件 A</button>
    <button @click="currentComponent = 'ComponentB'">组件 B</button>
    <component :is="currentComponent"></component>
  </div>
</template>

<script>
import ComponentA from './ComponentA.vue'
import ComponentB from './ComponentB.vue'

export default {
  components: {
    ComponentA,
    ComponentB
  },
  data() {
    return {
      currentComponent: 'ComponentA'
    }
  }
}
</script>

3. 动态组件的缓存？ 使用 <keep-alive> 标签包裹动态组件，可以缓存不活跃的组件实例，避免重复创建和销毁：

vue
<keep-alive>
  <component :is="currentComponent"></component>
</keep-alive>

4. <keep-alive> 的属性有哪些？

  • include：只缓存指定的组件
  • exclude：不缓存指定的组件
  • max：最多缓存的组件实例数

异步组件 ​

1. 什么是异步组件？ 异步组件是指在需要时才加载的组件，用于优化应用的初始加载性能。

2. 异步组件的实现方式？

  • 工厂函数：使用返回 Promise 的工厂函数定义异步组件
  • ES 模块动态导入：使用 import() 语法实现异步组件
  • 高级异步组件：使用对象形式定义异步组件，支持加载状态和错误状态

3. 基本异步组件的实现？

javascript
// 工厂函数方式
const AsyncComponent = () => import('./AsyncComponent.vue')

// 注册组件
export default {
  components: {
    AsyncComponent
  }
}

4. 高级异步组件的实现？

javascript
const AsyncComponent = () => ({
  // 异步组件
  component: import('./AsyncComponent.vue'),
  // 加载中状态
  loading: LoadingComponent,
  // 错误状态
  error: ErrorComponent,
  // 加载超时时间
  timeout: 3000
})

5. 异步组件的使用场景？

  • 大型应用：将应用拆分为多个异步组件，减少初始加载时间
  • 路由懒加载：结合 Vue Router 实现路由的懒加载
  • 条件渲染：只在需要时加载特定的组件

组件的生命周期 ​

1. 组件的生命周期钩子有哪些？

  • 创建阶段：beforeCreate、created
  • 挂载阶段：beforeMount、mounted
  • 更新阶段：beforeUpdate、updated
  • 销毁阶段：beforeDestroy、destroyed
  • Vue 3 新增：beforeUnmount、unmounted
  • 其他钩子：activated、deactivated（用于 keep-alive 组件）

2. 组件生命周期的执行顺序？

  • 父组件与子组件：父组件的 beforeMount 钩子执行后，执行子组件的生命周期，最后执行父组件的 mounted 钩子
  • 兄弟组件：按照组件在模板中的顺序执行生命周期钩子

3. activated 和 deactivated 钩子的作用？

  • activated：当 keep-alive 包裹的组件被激活时调用
  • deactivated：当 keep-alive 包裹的组件被停用时调用
  • 使用场景：用于处理 keep-alive 组件的激活和停用状态

组件的性能优化 ​

1. 组件性能优化的方法有哪些？

  • 合理使用 v-if 和 v-show：频繁切换用 v-show，条件不常改变用 v-if
  • 使用 keep-alive：缓存组件实例，避免重复创建和销毁
  • 组件拆分：将大型组件拆分为小型组件，提高复用性和维护性
  • 使用 computed 和 watch：合理使用计算属性和监听器，避免不必要的计算
  • 避免不必要的渲染：使用 shouldComponentUpdate 或 Vue.memo 优化渲染
  • 虚拟滚动：对于长列表，使用虚拟滚动技术减少 DOM 节点
  • 异步组件：使用异步组件减少初始加载时间
  • 事件委托：对于大量子元素的事件，使用事件委托

2. 如何避免组件的重复渲染？

  • 使用 computed：将计算逻辑放在计算属性中，利用缓存
  • 使用 v-once：对于静态内容，使用 v-once 指令
  • 使用 shouldComponentUpdate：在 Vue 2 中，通过 shouldComponentUpdate 钩子优化渲染
  • 使用 Vue.memo：在 Vue 3 中，使用 Vue.memo 包装组件
  • 避免在模板中使用复杂表达式：将复杂表达式移到计算属性中

3. 组件懒加载的实现方式？

  • 异步组件：使用 import() 语法实现组件的懒加载
  • 路由懒加载：结合 Vue Router 实现路由的懒加载
  • 条件懒加载：只在需要时加载特定的组件

4. keep-alive 的工作原理？

  • <keep-alive> 是一个抽象组件，不会渲染到 DOM 中
  • 它会缓存组件的实例和状态
  • 当组件被包裹在 <keep-alive> 中时，会触发 activated 和 deactivated 钩子
  • 可以通过 include 和 exclude 属性控制缓存的组件

混入（Mixins） ​

1. 什么是混入？ 混入（Mixin）是一种分发 Vue 组件中可复用功能的灵活方式。混入对象可以包含任意组件选项，当组件使用混入时，所有混入对象的选项将被"混入"该组件本身的选项。

2. 混入的使用方式？

javascript
// 定义混入
const myMixin = {
  data() {
    return {
      message: 'Hello from mixin'
    }
  },
  methods: {
    greet() {
      console.log(this.message)
    }
  }
}

// 使用混入
export default {
  mixins: [myMixin],
  created() {
    this.greet() // "Hello from mixin"
  }
}

3. 混入的合并策略？

  • data：返回对象的函数进行合并，发生冲突时以组件数据优先
  • methods：对象合并，发生冲突时以组件方法优先
  • computed：对象合并，发生冲突时以组件计算属性优先
  • 生命周期钩子：混入的钩子将在组件自身钩子之前调用
  • components/directives/filters：对象合并，发生冲突时以组件选项优先

4. 全局混入的使用方式？

javascript
// 全局注册混入
Vue.mixin({
  created() {
    console.log('全局混入的 created 钩子')
  }
})

// 所有组件都会应用这个混入

5. 混入的注意事项？

  • 命名冲突：混入和组件中的属性、方法可能发生冲突
  • 隐式依赖：混入可能引入隐式依赖，增加维护难度
  • 复用性：过度使用混入可能导致代码难以理解和维护
  • 替代方案：Vue 3 推荐使用 Composition API 替代混入

函数式组件 ​

1. 什么是函数式组件？ 函数式组件是一个无状态、无实例的组件，它只是一个函数，接收 props 和 context 作为参数，返回 VNode。

2. 函数式组件的使用方式？

javascript
// 函数式组件定义
export default function FunctionalComponent(props, context) {
  return h('div', { class: 'functional' }, props.message)
}

// 或者在 .vue 文件中使用
<template functional>
  <div class="functional">{{ props.message }}</div>
</template>

<script>
export default {
  functional: true,
  props: {
    message: String
  }
}
</script>

3. 函数式组件的 context 参数包含什么？

  • props：传递给组件的 props
  • children：子节点
  • slots：插槽对象
  • scopedSlots：作用域插槽对象
  • data：传递给组件的数据
  • parent：父组件引用
  • root：根节点引用
  • listeners：事件监听器对象

4. 函数式组件的优势？

  • 无状态：没有实例，没有 this，性能更好
  • 无生命周期：没有生命周期钩子，减少开销
  • 渲染性能：渲染速度更快，适合纯展示组件
  • 简单：代码更简单，易于理解

5. 函数式组件的使用场景？

  • 纯展示组件：只负责渲染，不涉及状态管理
  • 高渲染性能要求：需要优化渲染性能的场景
  • 无状态组件：不需要维护状态的组件

渲染函数 ​

1. 什么是渲染函数？ 渲染函数是 Vue 中用于创建虚拟 DOM 的函数，它返回一个 VNode（虚拟节点）。

2. 渲染函数的使用方式？

javascript
export default {
  render(h) {
    return h('div', { class: 'container' }, [
      h('h1', 'Hello'),
      h('p', 'World')
    ])
  }
}

3. h 函数的参数是什么？

  • 第一个参数：标签名或组件
  • 第二个参数：数据对象（属性、props、class、style 等）
  • 第三个参数：子节点（可以是字符串、数组或 VNode）

4. JSX 的使用方式？

javascript
// 安装 @vue/babel-plugin-jsx
import { defineComponent } from 'vue'

export default defineComponent({
  render() {
    return (
      <div class="container">
        <h1>Hello</h1>
        <p>World</p>
      </div>
    )
  }
})

5. 渲染函数和模板的区别？

  • 灵活性：渲染函数更灵活，可以使用完整的 JavaScript 功能
  • 可读性：模板更易读，适合大多数场景
  • 性能：渲染函数性能更好，但差异不大
  • 使用场景：需要高度定制化时使用渲染函数，否则使用模板

组件的高级用法 ​

1. 递归组件的使用方式？

vue
<!-- 递归组件 -->
<template>
  <div>
    <tree-node
      v-for="node in nodes"
      :key="node.id"
      :node="node"
    ></tree-node>
  </div>
</template>

<script>
export default {
  name: 'TreeNode',
  props: {
    node: Object
  }
}
</script>

2. 动态插槽的使用方式？

vue
<!-- 动态插槽名 -->
<template>
  <div>
    <slot :name="dynamicSlotName"></slot>
  </div>
</template>

3. v-model 的自定义实现？

vue
<!-- 自定义 v-model -->
<template>
  <input
    :value="value"
    @input="$emit('input', $event.target.value)"
  />
</template>

<script>
export default {
  props: ['value']
}
</script>

4. 组件的 v-model 默认实现？

javascript
// Vue 2
export default {
  model: {
    prop: 'value',
    event: 'input'
  }
}

// Vue 3
export default {
  props: {
    modelValue: String
  },
  emits: ['update:modelValue'],
  setup(props, { emit }) {
    const updateValue = (event) => {
      emit('update:modelValue', event.target.value)
    }
    return { updateValue }
  }
}
