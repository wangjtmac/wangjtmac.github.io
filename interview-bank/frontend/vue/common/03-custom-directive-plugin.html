Vue 自定义指令与插件 ​

Vue 自定义指令与插件相关面试题。

自定义指令基础 ​

1. 什么是自定义指令？ 自定义指令是 Vue 提供的一种机制，允许开发者创建自定义的指令，用于在模板中执行底层的 DOM 操作。

2. 为什么要使用自定义指令？

  • DOM 操作：需要直接操作 DOM 的场景
  • 第三方库集成：集成需要直接操作 DOM 的第三方库
  • 代码复用：将常用的 DOM 操作封装为指令
  • 模板简化：简化模板代码

3. 自定义指令的注册方式？

  • 全局注册：使用 Vue.directive('directive-name', { ... }) 注册
  • 局部注册：在组件的 directives 选项中注册

4. 全局注册自定义指令？

javascript
// main.js
import Vue from 'vue'

Vue.directive('focus', {
  inserted: function (el) {
    el.focus()
  }
})

5. 局部注册自定义指令？

vue
<template>
  <input v-focus>
</template>

<script>
export default {
  directives: {
    focus: {
      inserted: function (el) {
        el.focus()
      }
    }
  }
}
</script>

自定义指令的钩子函数 ​

1. 自定义指令的钩子函数有哪些？

  • bind：只调用一次，指令第一次绑定到元素时调用
  • inserted：被绑定元素插入父节点时调用
  • update：所在组件的 VNode 更新时调用
  • componentUpdated：指令所在组件的 VNode 及其子 VNode 全部更新后调用
  • unbind：只调用一次，指令与元素解绑时调用

2. 钩子函数的执行顺序？

  1. bind：指令绑定到元素
  2. inserted：元素插入到 DOM
  3. update：组件更新
  4. componentUpdated：组件及其子组件更新完成
  5. unbind：指令与元素解绑

3. 钩子函数的参数？

  • el：指令所绑定的元素
  • binding：一个对象，包含指令的名称、值、表达式、参数、修饰符等
  • vnode：Vue 编译生成的虚拟节点
  • oldVnode：上一个虚拟节点，仅在 update 和 componentUpdated 钩子中可用

4. binding 对象的属性？

  • name：指令名，不包括 v- 前缀
  • value：指令的绑定值
  • oldValue：指令绑定的前一个值
  • expression：字符串形式的指令表达式
  • arg：传给指令的参数
  • modifiers：一个包含修饰符的对象

自定义指令的参数 ​

1. 如何创建带参数的自定义指令？

javascript
Vue.directive('color', {
  bind: function (el, binding) {
    el.style.color = binding.value
  }
})

// 使用
<p v-color="'red'">Hello Vue!</p>

2. 如何创建带参数和修饰符的自定义指令？

javascript
Vue.directive('color', {
  bind: function (el, binding) {
    if (binding.modifiers.bold) {
      el.style.fontWeight = 'bold'
    }
    if (binding.modifiers.italic) {
      el.style.fontStyle = 'italic'
    }
    el.style.color = binding.value
  }
})

// 使用
<p v-color.bold.italic="'red'">Hello Vue!</p>

3. 如何创建带动态参数的自定义指令？

javascript
Vue.directive('color', {
  bind: function (el, binding) {
    el.style[binding.arg] = binding.value
  }
})

// 使用
<p v-color:color="'red'">Hello Vue!</p>
<p v-color:background="'blue'">Hello Vue!</p>

4. 如何创建带多个参数的自定义指令？

javascript
Vue.directive('style', {
  bind: function (el, binding) {
    const styles = binding.value
    Object.keys(styles).forEach(key => {
      el.style[key] = styles[key]
    })
  }
})

// 使用
<p v-style="{ color: 'red', fontSize: '20px' }">Hello Vue!</p>

自定义指令的使用场景 ​

1. 自动聚焦指令？

javascript
Vue.directive('focus', {
  inserted: function (el) {
    el.focus()
  }
})

2. 权限控制指令？

javascript
Vue.directive('permission', {
  inserted: function (el, binding) {
    const permissions = store.getters.permissions
    if (!permissions.includes(binding.value)) {
      el.parentNode && el.parentNode.removeChild(el)
    }
  }
})

// 使用
<button v-permission="'admin'">删除</button>

3. 防抖指令？

javascript
Vue.directive('debounce', {
  inserted: function (el, binding) {
    let timer
    el.addEventListener('click', () => {
      clearTimeout(timer)
      timer = setTimeout(() => {
        binding.value()
      }, binding.arg || 300)
    })
  }
})

// 使用
<button v-debounce:500="handleClick">点击</button>

4. 节流指令？

javascript
Vue.directive('throttle', {
  inserted: function (el, binding) {
    let timer
    let lastTime = 0
    el.addEventListener('click', () => {
      const now = Date.now()
      if (now - lastTime >= (binding.arg || 300)) {
        binding.value()
        lastTime = now
      }
    })
  }
})

// 使用
<button v-throttle:500="handleClick">点击</button>

5. 无限滚动指令？

javascript
Vue.directive('infinite-scroll', {
  inserted: function (el, binding) {
    const callback = binding.value
    const options = {
      root: null,
      rootMargin: '0px',
      threshold: 0.1
    }

    const observer = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          callback()
        }
      })
    }, options)

    observer.observe(el)

    el._observer = observer
  },
  unbind: function (el) {
    if (el._observer) {
      el._observer.disconnect()
    }
  }
})

// 使用
<div v-infinite-scroll="loadMore"></div>

6. 复制到剪贴板指令？

javascript
Vue.directive('copy', {
  bind: function (el, binding) {
    el.addEventListener('click', () => {
      const text = binding.value
      navigator.clipboard.writeText(text).then(() => {
        console.log('复制成功')
      }).catch(err => {
        console.error('复制失败', err)
      })
    })
  }
})

// 使用
<button v-copy="'Hello Vue!'">复制</button>

7. 图片懒加载指令？

javascript
Vue.directive('lazy', {
  inserted: function (el, binding) {
    const src = binding.value
    const observer = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          el.src = src
          observer.unobserve(el)
        }
      })
    })

    observer.observe(el)
    el._observer = observer
  },
  unbind: function (el) {
    if (el._observer) {
      el._observer.disconnect()
    }
  }
})

// 使用
<img v-lazy="imageUrl">

自定义插件基础 ​

1. 什么是自定义插件？ 插件是 Vue 提供的一种机制，允许开发者创建可复用的功能，并在多个 Vue 应用中使用。

2. 为什么要使用自定义插件？

  • 代码复用：将常用的功能封装为插件
  • 全局功能：在全局范围内使用功能
  • 第三方库集成：集成第三方库到 Vue 应用
  • 功能扩展：扩展 Vue 的功能

3. 插件的 install 方法？

javascript
const MyPlugin = {
  install(Vue, options) {
    // 插件逻辑
  }
}

export default MyPlugin

4. 如何使用插件？

javascript
import MyPlugin from './my-plugin'

Vue.use(MyPlugin, {
  someOption: true
})

自定义插件的使用场景 ​

1. 状态管理插件？

javascript
const StorePlugin = {
  install(Vue, options) {
    const store = new Store(options)
    Vue.prototype.$store = store
    Vue.mixin({
      beforeCreate() {
        this.$store = store
      }
    })
  }
}

class Store {
  constructor(options) {
    this.state = options.state || {}
    this.mutations = options.mutations || {}
    this.actions = options.actions || {}
  }

  commit(type, payload) {
    const mutation = this.mutations[type]
    if (mutation) {
      mutation(this.state, payload)
    }
  }

  dispatch(type, payload) {
    const action = this.actions[type]
    if (action) {
      action(this, payload)
    }
  }
}

export default StorePlugin

2. 路由插件？

javascript
const RouterPlugin = {
  install(Vue, options) {
    const router = new Router(options)
    Vue.prototype.$router = router
    Vue.mixin({
      beforeCreate() {
        this.$router = router
      }
    })
  }
}

class Router {
  constructor(options) {
    this.routes = options.routes || []
    this.currentRoute = '/'
    this.init()
  }

  init() {
    window.addEventListener('popstate', () => {
      this.currentRoute = window.location.pathname
    })
  }

  push(path) {
    window.history.pushState({}, '', path)
    this.currentRoute = path
  }
}

export default RouterPlugin

3. HTTP 请求插件？

javascript
const HttpPlugin = {
  install(Vue, options) {
    const http = new Http(options)
    Vue.prototype.$http = http
  }
}

class Http {
  constructor(options) {
    this.baseURL = options.baseURL || ''
    this.headers = options.headers || {}
  }

  get(url, config = {}) {
    return fetch(this.baseURL + url, {
      method: 'GET',
      headers: { ...this.headers, ...config.headers }
    }).then(response => response.json())
  }

  post(url, data, config = {}) {
    return fetch(this.baseURL + url, {
      method: 'POST',
      headers: { ...this.headers, ...config.headers, 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    }).then(response => response.json())
  }
}

export default HttpPlugin

4. 国际化插件？

javascript
const I18nPlugin = {
  install(Vue, options) {
    const i18n = new I18n(options)
    Vue.prototype.$t = i18n.translate.bind(i18n)
    Vue.filter('translate', i18n.translate.bind(i18n))
  }
}

class I18n {
  constructor(options) {
    this.locale = options.locale || 'en'
    this.messages = options.messages || {}
  }

  translate(key) {
    return this.messages[this.locale][key] || key
  }

  setLocale(locale) {
    this.locale = locale
  }
}

export default I18nPlugin

5. 主题切换插件？

javascript
const ThemePlugin = {
  install(Vue, options) {
    const theme = new Theme(options)
    Vue.prototype.$theme = theme
    Vue.mixin({
      created() {
        this.$theme.apply()
      }
    })
  }
}

class Theme {
  constructor(options) {
    this.currentTheme = options.default || 'light'
    this.themes = options.themes || {}
  }

  apply() {
    const theme = this.themes[this.currentTheme]
    if (theme) {
      Object.keys(theme).forEach(key => {
        document.documentElement.style.setProperty(key, theme[key])
      })
    }
  }

  setTheme(themeName) {
    this.currentTheme = themeName
    this.apply()
  }
}

export default ThemePlugin
