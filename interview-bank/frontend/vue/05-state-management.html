Vue 状态管理 ​

Vuex 和 Pinia 相关面试题。

Vuex 基本使用 ​

1. 什么是 Vuex？ Vuex 是 Vue.js 官方的状态管理模式，用于管理应用的全局状态。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态的一致性。

2. Vuex 的安装方式？

  • npm 安装：npm install vuex
  • yarn 安装：yarn add vuex
  • Vue CLI：使用 Vue CLI 创建项目时选择 Vuex 选项

3. Vuex 的基本配置？

javascript
// 导入 Vue 和 Vuex
import Vue from 'vue'
import Vuex from 'vuex'

// 注册 Vuex
Vue.use(Vuex)

// 创建 store 实例
export default new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++
    }
  },
  actions: {
    incrementAsync({ commit }) {
      setTimeout(() => {
        commit('increment')
      }, 1000)
    }
  },
  getters: {
    doubleCount(state) {
      return state.count * 2
    }
  },
  modules: {
    // 模块
  }
})

4. 如何在组件中使用 Vuex？

  • 获取状态：this.$store.state.count
  • 提交 mutation：this.$store.commit('increment')
  • 分发 action：this.$store.dispatch('incrementAsync')
  • 获取 getter：this.$store.getters.doubleCount
  • 辅助函数：mapState、mapMutations、mapActions、mapGetters

5. Vuex 的辅助函数使用？

javascript
import { mapState, mapMutations, mapActions, mapGetters } from 'vuex'

export default {
  computed: {
    ...mapState(['count']),
    ...mapGetters(['doubleCount'])
  },
  methods: {
    ...mapMutations(['increment']),
    ...mapActions(['incrementAsync'])
  }
}

Vuex 的核心概念 ​

1. Vuex 的核心概念有哪些？

  • state：存储应用的状态
  • mutations：修改状态的方法，必须是同步的
  • actions：处理异步操作，提交 mutations
  • getters：计算属性，基于 state 派生新的状态
  • modules：模块化管理状态

2. mutations 与 actions 的区别？

  • mutations：同步操作，直接修改 state，通过 commit 调用
  • actions：异步操作，提交 mutations，通过 dispatch 调用
  • 使用场景：同步操作使用 mutations，异步操作使用 actions

3. getters 的作用是什么？

  • 计算属性：基于 state 派生新的状态
  • 缓存：与组件的计算属性类似，具有缓存特性
  • 复用：可以在多个组件中复用相同的计算逻辑

4. modules 的使用场景？

  • 大型应用：当应用的状态管理变得复杂时，使用模块进行拆分
  • 命名空间：每个模块可以拥有自己的 state、mutations、actions、getters
  • 模块化管理：提高代码的可维护性和可读性

5. 模块的命名空间？

javascript
// 启用命名空间的模块
const userModule = {
  namespaced: true,
  state: { ... },
  mutations: { ... },
  actions: { ... },
  getters: { ... }
}

// 访问命名空间模块的状态
this.$store.state.user.count

// 提交命名空间模块的 mutation
this.$store.commit('user/increment')

// 分发命名空间模块的 action
this.$store.dispatch('user/incrementAsync')

// 获取命名空间模块的 getter
this.$store.getters['user/doubleCount']

Pinia 的使用 ​

1. 什么是 Pinia？ Pinia 是 Vue 3 官方推荐的状态管理库，是 Vuex 的替代品。它提供了更简洁的 API、更好的 TypeScript 支持和更灵活的组合式 API。

2. Pinia 与 Vuex 的区别？

  • API 更简洁：不需要 mutations，直接在 actions 中修改状态
  • 更好的 TypeScript 支持：内置 TypeScript 类型推断
  • 更灵活的组合式 API：支持 Composition API
  • 模块化：每个 store 都是一个独立的模块
  • 无需命名空间：默认支持模块的命名空间

3. Pinia 的安装方式？

  • npm 安装：npm install pinia
  • yarn 安装：yarn add pinia

4. Pinia 的基本配置？

javascript
// main.js
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import App from './App.vue'

const app = createApp(App)
const pinia = createPinia()

app.use(pinia)
app.mount('#app')

5. Pinia 的 store 创建？

javascript
// stores/counter.js
import { defineStore } from 'pinia'

export const useCounterStore = defineStore('counter', {
  state: () => ({
    count: 0
  }),
  getters: {
    doubleCount: (state) => state.count * 2
  },
  actions: {
    increment() {
      this.count++
    },
    incrementAsync() {
      return new Promise((resolve) => {
        setTimeout(() => {
          this.count++
          resolve()
        }, 1000)
      })
    }
  }
})

6. 如何在组件中使用 Pinia？

javascript
import { useCounterStore } from '@/stores/counter'

export default {
  setup() {
    const counterStore = useCounterStore()
    
    return {
      counterStore,
      increment: counterStore.increment,
      doubleCount: counterStore.doubleCount
    }
  }
}

状态管理的最佳实践 ​

1. 状态管理的使用场景？

  • 全局状态：需要在多个组件间共享的状态
  • 复杂状态：状态逻辑复杂，需要集中管理
  • 跨组件通信：组件层级较深，使用 props/emit 不方便
  • 持久化状态：需要持久化存储的状态，如用户登录信息

2. 状态管理的设计原则？

  • 单一数据源：所有状态集中在一个 store 中
  • 状态只读：通过 mutations 或 actions 修改状态，不直接修改
  • 异步操作：异步操作放在 actions 中处理
  • 模块化：使用模块拆分复杂状态
  • 命名规范：使用语义化的命名，清晰表达状态的含义

3. 状态持久化的实现方式？

  • localStorage/sessionStorage：将状态存储在浏览器的存储中
  • cookie：适合存储小量数据
  • IndexedDB：适合存储大量数据
  • 插件：使用 vuex-persist 或 pinia-plugin-persistedstate 等插件

4. 状态管理的性能优化？

  • 按需加载：使用模块的动态导入
  • 缓存：合理使用 getters 的缓存特性
  • 避免过度使用：只管理真正需要全局共享的状态
  • 批量更新：在 mutations 中批量更新状态
  • 使用常量：使用常量定义 mutation 类型，避免拼写错误

5. 如何调试 Vuex/Pinia？

  • Vue DevTools：使用 Vue DevTools 调试状态变化
  • 日志：在 mutations 和 actions 中添加日志
  • 时间旅行：使用 Vue DevTools 的时间旅行功能，回溯状态变化
  • 测试：编写单元测试，确保状态管理的正确性
