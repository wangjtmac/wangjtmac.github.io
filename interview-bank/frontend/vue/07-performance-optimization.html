Vue 性能优化 ​

Vue 性能优化相关面试题。

渲染性能优化 ​

1. 如何优化 Vue 的渲染性能？

  • 合理使用 v-if 和 v-show：频繁切换用 v-show，条件不常改变用 v-if
  • 使用 v-for 的 key：为 v-for 循环的项目添加唯一的 key 属性
  • 避免在模板中使用复杂表达式：将复杂逻辑移到计算属性中
  • 使用 v-once：对于静态内容，使用 v-once 指令
  • 使用 keep-alive：缓存组件实例，避免重复创建和销毁
  • 拆分组件：将大型组件拆分为小型组件，提高复用性和维护性
  • 使用虚拟滚动：对于长列表，使用虚拟滚动技术减少 DOM 节点

2. v-for 和 v-if 为什么不建议一起使用？

  • 性能问题：v-for 的优先级高于 v-if，会先遍历所有项目，然后再进行条件判断
  • 解决方案：使用计算属性过滤需要显示的项目，或在 v-for 外部使用 v-if

3. 虚拟滚动的原理是什么？ 虚拟滚动通过只渲染可视区域内的列表项，而不是整个列表，来减少 DOM 节点的数量，从而提高渲染性能。当用户滚动时，动态计算并渲染可视区域内的列表项。

4. 如何实现虚拟滚动？

  • 使用第三方库：如 vue-virtual-scroller、vue-virtual-scroll-list 等
  • 自定义实现：计算可视区域的高度、列表项的高度、当前滚动位置等，动态渲染可视区域内的列表项

打包优化 ​

1. 如何优化 Vue 应用的打包体积？

  • 按需加载：使用路由懒加载和组件懒加载
  • Tree Shaking：移除未使用的代码
  • 代码分割：将代码分割为多个 chunk，按需加载
  • 压缩代码：使用 terser 压缩 JavaScript 代码
  • 压缩 CSS：使用 cssnano 压缩 CSS 代码
  • 压缩图片：使用 image-webpack-loader 压缩图片
  • 移除 console：在生产环境中移除 console 语句
  • 使用 CDN：将第三方库通过 CDN 引入，减少打包体积

2. 路由懒加载的实现方式？

javascript
const routes = [
  {
    path: '/',
    name: 'Home',
    component: () => import('./views/Home.vue')
  },
  {
    path: '/about',
    name: 'About',
    component: () => import('./views/About.vue')
  }
]

3. 组件懒加载的实现方式？

javascript
export default {
  components: {
    AsyncComponent: () => import('./AsyncComponent.vue')
  }
}

4. Webpack 的代码分割配置？

javascript
// webpack.config.js
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {

首屏加载优化 ​

1. 如何优化 Vue 应用的首屏加载速度？

  • 路由懒加载：使用动态导入分割代码
  • 组件懒加载：按需加载组件
  • CDN 加速：将第三方库通过 CDN 引入
  • 服务端渲染：使用 SSR 提高首屏渲染速度
  • 预渲染：对于静态页面，使用预渲染
  • 减少 HTTP 请求：合并资源文件，使用 sprite 图
  • 压缩资源：压缩 HTML、CSS、JavaScript、图片等
  • 使用 HTTP/2：支持多路复用，减少连接数
  • 使用缓存：合理设置缓存策略

2. 服务端渲染（SSR）的优势？

  • 首屏加载快：服务端直接生成 HTML，减少客户端渲染时间
  • SEO 友好：搜索引擎可以直接抓取服务端生成的 HTML
  • 更好的用户体验：用户可以更快看到页面内容

3. 预渲染的使用场景？

  • 静态页面：如首页、关于我们等内容固定的页面
  • SEO 需求：需要搜索引擎抓取的页面
  • 性能优化：减少客户端渲染时间

4. 如何实现预渲染？

  • 使用 prerender-spa-plugin：在构建时预渲染指定的路由
  • 配置示例：
javascript
// vue.config.js
const PrerenderSPAPlugin = require('prerender-spa-plugin')
const Renderer = PrerenderSPAPlugin.PuppeteerRenderer

module.exports = {
  configureWebpack: {
    plugins: [
      new PrerenderSPAPlugin({
        staticDir: path.join(__dirname, 'dist'),
        routes: ['/', '/about', '/contact'],
        renderer: new Renderer({
          inject: {
            foo: 'bar'
          },
          headless: true,
          renderAfterDocumentEvent: 'render-event'
        })
      })
    ]
  }
}

运行时优化 ​

1. 如何优化 Vue 应用的运行时性能？

  • 合理使用计算属性：利用计算属性的缓存特性
  • 合理使用监听器：避免不必要的监听器
  • 避免深度监听：深度监听会增加性能开销
  • 使用防抖和节流：对于频繁触发的事件，如滚动、输入等
  • 优化事件处理：使用事件委托，避免在多个子元素上绑定相同的事件
  • 减少重排和重绘：避免频繁修改 DOM 样式
  • 使用 requestAnimationFrame：对于动画效果，使用 requestAnimationFrame

2. 防抖和节流的区别？

  • 防抖：在事件触发后等待一段时间再执行，如果在等待期间事件再次触发，则重新计时
  • 节流：在事件触发后立即执行，然后在一段时间内不再执行，无论事件触发多少次
  • 使用场景：防抖适用于输入验证、搜索联想等；节流适用于滚动、拖拽等

3. 如何实现防抖和节流？

javascript
// 防抖
function debounce(func, wait) {
  let timeout
  return function() {
    const context = this
    const args = arguments
    clearTimeout(timeout)
    timeout = setTimeout(() => {
      func.apply(context, args)
    }, wait)
  }
}

// 节流
function throttle(func, wait) {
  let lastTime = 0
  return function() {
    const now = Date.now()
    if (now - lastTime >= wait) {
      func.apply(this, arguments)
      lastTime = now
    }
  }
}

4. 如何减少重排和重绘？

  • 批量修改样式：将多个样式修改合并为一次
  • 使用 class：通过修改 class 来修改样式，而不是直接修改样式属性
  • 使用 DocumentFragment：对于大量 DOM 操作，使用 DocumentFragment
  • 避免频繁读取布局属性：如 offsetTop、offsetLeft 等，会触发重排
  • 使用 CSS transform：transform 不会触发重排，只会触发重绘

内存泄漏的预防 ​

1. 什么是内存泄漏？ 内存泄漏是指程序中已分配的内存没有被及时释放，导致内存使用量不断增加，最终可能导致程序崩溃或系统变慢。

2. Vue 应用中常见的内存泄漏场景有哪些？

  • 定时器未清除：setTimeout、setInterval 等定时器未在组件销毁时清除
  • 事件监听器未移除：addEventListener 添加的事件监听器未在组件销毁时移除
  • 闭包引用：组件销毁后，闭包仍然引用组件的实例或数据
  • 第三方库未清理：使用第三方库时，未在组件销毁时清理资源
  • 无限循环：代码中存在无限循环，导致内存使用量不断增加

3. 如何预防内存泄漏？

  • 清除定时器：在 beforeDestroy 或 unmounted 钩子中清除定时器
  • 移除事件监听器：在 beforeDestroy 或 unmounted 钩子中移除事件监听器
  • 清理第三方库：在 beforeDestroy 或 unmounted 钩子中清理第三方库的资源
  • 避免闭包引用：避免在闭包中引用组件的实例或数据，或在组件销毁时解除引用
  • 使用 WeakMap 和 WeakSet：对于临时引用，使用 WeakMap 和 WeakSet，它们不会阻止垃圾回收

4. 如何检测内存泄漏？

  • 使用浏览器开发者工具：Chrome DevTools 的 Performance 和 Memory 面板
  • 监控内存使用量：使用 performance.memory API 监控内存使用量
  • 定期测试：定期测试应用的内存使用情况，特别是在长时间使用后

网络请求优化 ​

1. 如何优化 Vue 应用的网络请求？

  • 使用 HTTP/2：支持多路复用，减少连接数
  • 使用 HTTPS：提高安全性和性能
  • 合理设置缓存：使用浏览器缓存和服务器缓存
  • 压缩资源：压缩 HTML、CSS、JavaScript、图片等
  • 使用 CDN：将静态资源部署到 CDN，提高访问速度
  • 减少 HTTP 请求：合并资源文件，使用 sprite 图
  • 使用 WebSocket：对于实时通信，使用 WebSocket
  • 批量请求：将多个小请求合并为一个大请求
  • 延迟加载：对于非关键资源，使用延迟加载

2. 如何实现请求缓存？

  • 使用 localStorage/sessionStorage：缓存不经常变化的数据
  • 使用 axios 的缓存：配置 axios 的缓存策略
  • 使用 Service Worker：实现离线缓存和请求拦截
  • 后端缓存：使用 Redis 等缓存服务器缓存数据

3. 如何实现请求防抖？

  • 输入搜索：在用户输入时，等待一段时间再发送请求，避免频繁发送请求
  • 实现方式：使用防抖函数包装请求函数

4. 如何处理网络错误？

  • 错误处理：使用 try/catch 捕获错误，或使用 axios 的错误处理
  • 重试机制：对于网络错误，实现自动重试机制
  • 降级方案：当网络请求失败时，使用本地缓存或默认数据
  • 用户提示：向用户显示友好的错误提示

5. 如何优化 API 响应时间？

  • 后端优化：优化数据库查询、使用缓存、异步处理等
  • 减少数据传输：只返回必要的数据，使用压缩
  • 分页查询：对于大量数据，使用分页查询
  • 预加载：预加载可能需要的数据
  • 使用 GraphQL：按需获取数据，减少冗余数据
