HTML 优化与最佳实践 ​

HTML 优化与最佳实践相关面试题，包括 SEO 优化、无障碍访问、表单验证、实体编码等。

SEO 优化 ​

HTML 层面的 SEO 优化 ​

html
<title>页面标题 - 网站名称</title>
<meta name="description" content="页面描述，150字以内">
<meta name="keywords" content="关键词1,关键词2">

<h1>主标题</h1>
<h2>副标题</h2>

<img src="image.jpg" alt="图片描述">

<a href="url" title="链接描述">链接文字</a>

<nav>
  <a href="/">首页</a>
  <a href="/about">关于</a>
</nav>

<main>
  <article>
    <h1>文章标题</h1>
    <p>文章内容...</p>
  </article>
</main>

优化要点 ​

优化项说明
语义化标签使用正确的 HTML 标签表达内容含义
title 标签每个页面有唯一、描述性的标题
meta description准确描述页面内容
alt 属性图片必须有 alt 描述
heading 层级合理使用 h1-h6，h1 每页仅一个
结构化数据使用 JSON-LD 格式
URL 结构简洁、有意义的 URL

常见面试问题 ​

Q: 什么是语义化 HTML？

语义化 HTML 是指使用正确的标签来表达内容的含义，而不是仅仅使用 div 和 span。好处：

  • 提升 SEO
  • 提升无障碍访问
  • 代码更易维护

Q: 什么是结构化数据？

结构化数据是一种标准格式，用于向搜索引擎提供页面内容的详细信息：

html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "文章标题",
  "author": "作者",
  "datePublished": "2024-01-01"
}
</script>

无障碍访问（ARIA） ​

ARIA 属性 ​

ARIA（Accessible Rich Internet Applications）用于增强 Web 应用的无障碍访问：

属性说明
role定义元素的角色
aria-label元素的标签
aria-labelledby关联标签元素
aria-describedby关联描述元素
aria-hidden对辅助设备隐藏
aria-disabled禁用状态
aria-expanded展开/折叠状态
aria-selected选中状态
aria-live动态内容更新通知

使用示例 ​

html
<button aria-label="关闭对话框">✕</button>

<nav role="navigation" aria-label="主导航">
  <a href="/">首页</a>
</nav>

<div role="alert" aria-live="polite">
  操作成功
</div>

<button aria-expanded="false" aria-controls="menu">
  菜单
</button>

常见面试问题 ​

Q: 什么时候需要使用 ARIA？

  • 原生 HTML 无法表达语义时
  • 自定义交互组件（如自定义下拉框、模态框）
  • 动态内容更新需要通知用户时

Q: ARIA 的第一原则是什么？

原生 HTML 优先：如果原生 HTML 元素能表达语义，就不要使用 ARIA。例如：

  • 使用 <button> 而不是 <div role="button">
  • 使用 <nav> 而不是 <div role="navigation">

HTML 表单验证 ​

HTML5 原生验证 ​

html
<form>
  <input type="text" required>
  <input type="email" required>
  <input type="tel" pattern="[0-9]{11}">
  <input type="number" min="0" max="100">
  <input type="text" minlength="2" maxlength="10">

属性说明
required必填

minlength / maxlength最小/最大长度
type="email"邮箱格式
type="url"URL 格式

JavaScript 验证 API ​

javascript
const input = document.querySelector('input');

input.validity.valid;
input.validity.valueMissing;
input.validity.patternMismatch;

使用 setCustomValidity 方法：

javascript
input.setCustomValidity('自定义错误信息');

常见面试问题 ​

Q: 为什么需要实体编码？

  • 避免与 HTML 标签冲突（如 < 和 >）
  • 显示特殊字符（如版权符号、货币符号）
  • 确保在不同编码下正确显示

Q: 如何处理特殊字符？

  • 使用实体编码
  • 使用 UTF-8 编码
  • 在 JavaScript 中使用 Unicode 转义

属性说明示例

属性说明示例

属性说明值

ARIA 使用示例 ​

自定义按钮 ​

html
<div role="button" tabindex="0" aria-label="关闭对话框">
  ✕
</div>

自定义下拉框 ​

html
<div role="combobox" aria-expanded="false" aria-haspopup="listbox" aria-owns="list1">
  <button aria-label="选择城市">北京 ▼</button>
  <ul role="listbox" id="list1" aria-hidden="true">
    <li role="option" aria-selected="true">北京</li>
    <li role="option">上海</li>
    <li role="option">广州</li>
  </ul>
</div>

自定义模态框 ​

html
<div role="dialog" aria-modal="true" aria-labelledby="dialog-title" aria-describedby="dialog-desc">
  <h2 id="dialog-title">确认删除</h2>
  <p id="dialog-desc">确定要删除这个项目吗？</p>
  <button aria-label="取消">取消</button>
  <button aria-label="确认">确认</button>
</div>

进度条 ​

html
<div role="progressbar" aria-valuenow="50" aria-valuemin="0" aria-valuemax="100" aria-label="加载进度">
  <div style="width: 50%">50%</div>
</div>

标签页 ​

html
<div role="tablist" aria-label="设置标签">
  <button role="tab" aria-selected="true" aria-controls="panel1">基本设置</button>
  <button role="tab" aria-selected="false" aria-controls="panel2">高级设置</button>
</div>

<div role="tabpanel" id="panel1" aria-labelledby="tab1">
  基本设置内容
</div>

<div role="tabpanel" id="panel2" aria-labelledby="tab2" hidden>
  高级设置内容
</div>

ARIA 最佳实践 ​

第一原则：原生 HTML 优先 ​

html
<!-- 推荐 -->
<button>点击</button>

<!-- 不推荐 -->
<div role="button">点击</div>

第二原则：不重复语义 ​

html
<!-- 推荐 -->
<button>点击</button>

<!-- 不推荐 -->
<button role="button">点击</button>

第三原则：所有交互元素可访问 ​

html
<!-- 推荐 -->
<button aria-label="关闭">✕</button>

<!-- 不推荐 -->
<div>✕</div>

第四原则：提供有意义的标签 ​

html
<!-- 推荐 -->
<button aria-label="关闭对话框">✕</button>

<!-- 不推荐 -->
<button aria-label="X">✕</button>

常见面试问题 ​

Q: ARIA 的作用？

  • 增强无障碍访问
  • 帮助屏幕阅读器理解页面
  • 提升用户体验

Q: 什么时候需要使用 ARIA？

  • 原生 HTML 无法表达语义时
  • 自定义交互组件
  • 动态内容更新需要通知用户时

Q: ARIA 的第一原则是什么？

原生 HTML 优先：如果原生 HTML 元素能表达语义，就不要使用 ARIA。

Q: aria-live 的值有什么区别？

值说明
off不通知更新
polite用户空闲时通知
assertive立即通知，打断当前操作

Q: 如何测试 ARIA？

  • 使用屏幕阅读器（如 NVDA、JAWS）
  • 使用浏览器开发者工具的 Accessibility 面板
  • 使用在线工具（如 WAVE、axe）
