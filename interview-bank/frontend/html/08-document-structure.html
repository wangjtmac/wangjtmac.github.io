HTML 文档结构 ​

HTML 文档结构相关面试题，包括 DOCTYPE、html 根元素、head 头部元素、body 主体元素等。

HTML 文档基本结构 ​

完整结构示例 ​

html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <title>页面标题</title>
  <meta name="description" content="页面描述">
  <meta name="keywords" content="关键词1,关键词2">
  <link rel="stylesheet" href="style.css">
  <script src="app.js"></script>
</head>
<body>
  <header>页面头部</header>
  <main>页面主体</main>
  <footer>页面底部</footer>
</body>
</html>

DOCTYPE 声明 ​

DOCTYPE 的作用 ​

<!DOCTYPE> 声明位于 HTML 文档的最前面，告诉浏览器当前 HTML 文档使用的是哪个版本的 HTML 规范。

HTML5 DOCTYPE ​

html
<!DOCTYPE html>

DOCTYPE 的演变 ​

版本DOCTYPE 声明
HTML 4.01 Strict<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
HTML 4.01 Transitional<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
XHTML 1.0 Strict<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
HTML5<!DOCTYPE html>

标准模式与怪异模式 ​

模式说明DOCTYPE
标准模式浏览器按标准规范渲染有 DOCTYPE 声明
怪异模式浏览器模拟旧版行为无 DOCTYPE 或不完整

常见面试问题 ​

Q: 为什么 HTML5 的 DOCTYPE 这么简单？

HTML5 不基于 SGML，不需要引用 DTD，因此 DOCTYPE 声明非常简单。

Q: 标准模式和怪异模式的区别？

  • 标准模式：按 W3C 标准渲染，布局一致
  • 怪异模式：模拟旧版浏览器行为，布局不一致

html 根元素 ​

基本属性 ​

html
<html lang="zh-CN">
<html lang="en">
<html lang="zh-CN" dir="ltr">

常用属性 ​

属性说明示例
lang指定文档语言lang="zh-CN"
dir文本方向dir="ltr"（从左到右）、dir="rtl"（从右到左）

常见面试问题 ​

Q: lang 属性的作用？

  • 帮助搜索引擎识别页面语言
  • 帮助屏幕阅读器正确发音
  • 帮助浏览器选择正确的字体

meta 标签 ​

字符编码 ​

html
<meta charset="UTF-8">

视口设置 ​

html
<meta name="viewport" content="width=device-width, initial-scale=1.0">

兼容模式 ​

html
<meta http-equiv="X-UA-Compatible" content="ie=edge">

SEO 相关 ​

html
<meta name="description" content="页面描述">
<meta name="keywords" content="关键词1,关键词2">
<meta name="author" content="作者">

其他常用 meta ​

html
<!-- 禁止缓存 -->
<meta http-equiv="Cache-Control" content="no-cache">

<!-- 自动刷新 -->
<meta http-equiv="refresh" content="30">

<!-- 主题色（移动端） -->
<meta name="theme-color" content="#ffffff">

<!-- Open Graph 协议 -->
<meta property="og:title" content="标题">
<meta property="og:description" content="描述">
<meta property="og:image" content="图片URL">

title 标签 ​

html
<title>页面标题 - 网站名称</title>

注意事项：

  • 每个页面必须有唯一的 title
  • 长度建议 50-60 个字符
  • 包含关键词，提高 SEO

常见面试问题 ​

Q: meta 标签的 name 和 property 属性的区别？

  • name：标准 meta 属性，如 description、keywords
  • property：用于 Open Graph 协议，如 og:title、og:description

Q: link 标签的 rel 属性有哪些常用值？

值说明
stylesheet引入样式表
icon网站图标
preload预加载资源
prefetch预取资源
canonical规范化 URL

body 主体元素 ​

常用结构标签 ​

html
<body>
  <header>页面头部</header>
  <nav>导航</nav>
  <main>主要内容</main>
  <aside>侧边栏</aside>
  <footer>页面底部</footer>
</body>

常见面试问题 ​

Q: body 标签的常用事件有哪些？

  • onload：页面加载完成
  • onunload：页面卸载
  • onresize：窗口大小改变
  • onscroll：页面滚动

常见面试问题 ​

Q: HTML 文档的加载顺序？

  1. 解析 DOCTYPE 和 html 标签
  2. 解析 head 元素
  3. 加载外部资源（CSS、JS）
  4. 解析 body 元素
  5. 执行 JS（根据 defer/async 决定时机）
  6. 触发 onload 事件

Q: 如何优化 HTML 文档结构？

  • 使用语义化标签
  • 合理使用 meta 标签
  • 优化资源加载顺序
  • 减少 DOM 层级深度
  • 使用 preload/prefetch
