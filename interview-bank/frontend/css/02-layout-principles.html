CSS 布局原理 ​

CSS 布局原理相关面试题，包括 BFC、Flex 布局、position 定位、水平垂直居中、清除浮动、margin 重叠等。

BFC（块级格式化上下文） ​

什么是 BFC ​

BFC（Block Formatting Context）是一个独立的渲染区域，内部的元素不会影响外部的元素。

触发 BFC 的条件 ​

条件说明
float 不为 noneleft、right
position 为 absolute 或 fixed脱离文档流
display 为 inline-block、table-cell、table-caption、flex、grid特殊显示模式
overflow 不为 visiblehidden、auto、scroll
contain 为 layout、content、strictCSS Containment

BFC 的特性 ​

  1. 内部的 Box 垂直方向排列
  2. 计算 BFC 高度时，浮动元素也参与计算
  3. BFC 区域不会与浮动元素重叠
  4. BFC 是独立容器，内外元素互不影响

BFC 应用场景 ​

场景 1：清除浮动

css
.parent {
  overflow: hidden; /* 触发 BFC */
}
.child {
  float: left;
}

场景 2：防止 margin 重叠

css
.container {
  overflow: hidden; /* 触发 BFC，隔离 margin */
}

场景 3：两栏布局

css
.left {
  float: left;
  width: 200px;
}
.right {
  overflow: hidden; /* 触发 BFC，不与浮动元素重叠 */
}

常见面试问题 ​

Q: 什么是 BFC？有什么作用？

BFC 是块级格式化上下文，是一个独立的渲染区域。作用：

  • 清除浮动
  • 防止 margin 重叠
  • 实现自适应两栏布局

Flex 布局 ​

容器属性 ​

属性说明值
display定义 flex 容器flex、inline-flex
flex-direction主轴方向row、row-reverse、column、column-reverse
flex-wrap换行方式nowrap、wrap、wrap-reverse
flex-flowdirection + wrap 简写row nowrap
justify-content主轴对齐flex-start、flex-end、center、space-between、space-around、space-evenly
align-items交叉轴对齐flex-start、flex-end、center、baseline、stretch
align-content多轴对齐同 align-items

项目属性 ​

属性说明
order排列顺序，数值越小越靠前
flex-grow放大比例，默认 0
flex-shrink缩小比例，默认 1
flex-basis初始大小，默认 auto
flexgrow + shrink + basis 简写
align-self单独设置交叉轴对齐

常用布局示例 ​

水平垂直居中

css
.container {
  display: flex;
  justify-content: center;
  align-items: center;
}

两端对齐

css
.container {
  display: flex;
  justify-content: space-between;
}

等分布局

css
.container {
  display: flex;
}
.item {
  flex: 1;
}

水平垂直居中 ​

方法汇总 ​

css
/* 1. Flex 布局（推荐） */
.parent {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* 2. Grid 布局 */
.parent {
  display: grid;
  place-items: center;
}

/* 3. 绝对定位 + transform */
.parent {
  position: relative;
}
.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

/* 4. 绝对定位 + margin: auto */
.parent {
  position: relative;
}
.child {
  position: absolute;
  top: 0;
  right: 0;
  bottom: 0;
  left: 0;
  margin: auto;
}

/* 5. 绝对定位 + 负 margin（需知道子元素宽高） */
.parent {
  position: relative;
}
.child {
  position: absolute;
  top: 50%;
  left: 50%;
  margin-top: -50px;  /* 高度的一半 */
  margin-left: -50px; /* 宽度的一半 */
}

/* 6. 行内元素 */
.parent {
  text-align: center;
  line-height: 200px; /* 父元素高度 */
}

常见面试问题 ​

Q: 如何实现水平垂直居中？

推荐使用 Flex 或 Grid 布局，兼容性好且代码简洁。如果需要兼容老浏览器，可以使用绝对定位方案。

清除浮动 ​

为什么需要清除浮动 ​

浮动元素脱离文档流，父元素无法计算高度，导致高度塌陷。

清除浮动的方法 ​

css
/* 1. 额外标签法 */
.clear {
  clear: both;
}

/* 2. 伪元素法（推荐） */
.clearfix::after {
  content: "";
  display: block;
  clear: both;
}

/* 3. 触发 BFC */
.parent {
  overflow: hidden;
}

/* 4. 父元素也浮动 */
.parent {
  float: left;
}

position 定位 ​

定位类型 ​

值说明是否脱离文档流
static默认值，正常文档流否
relative相对定位，相对于自身原位置否
absolute绝对定位，相对于最近的非 static 祖先是
fixed固定定位，相对于视口是
sticky粘性定位，滚动时切换 relative/fixed否

定位特点 ​

relative（相对定位）

  • 相对于元素自身原位置偏移
  • 不脱离文档流，原位置保留
  • 常用作绝对定位的参照容器

absolute（绝对定位）

  • 相对于最近的非 static 定位的祖先元素
  • 脱离文档流，原位置不保留
  • 若无定位祖先，相对于 <html>

fixed（固定定位）

  • 相对于浏览器视口
  • 脱离文档流
  • 滚动时位置不变

sticky（粘性定位）

  • 在阈值前表现为 relative
  • 滚动到阈值后表现为 fixed
  • 需指定 top/bottom/left/right 之一
css
.header {
  position: sticky;
  top: 0;
}

常见面试问题 ​

Q: absolute 定位相对于谁？

相对于最近的非 static 定位的祖先元素，如果没有则相对于 <html>（初始包含块）。

Q: fixed 定位在移动端有什么问题？

在 iOS 上，fixed 元素在键盘弹出时可能位置异常。解决方案：

  • 使用 absolute 替代
  • 使用 position: sticky
  • 避免在输入框页面使用 fixed

margin 重叠 ​

什么是 margin 重叠 ​

相邻块级元素的上下 margin 会发生重叠，取较大值而非相加。

重叠场景 ​

  1. 相邻兄弟元素
  2. 父元素与第一个/最后一个子元素
  3. 空块级元素

重叠规则 ​

  • 同号 margin 取较大值
  • 异号 margin 取相加值
  • 负 margin 会减小重叠区域

解决方案 ​

css
/* 方案 1：触发 BFC */
.parent {
  overflow: hidden;
}

/* 方案 2：给父元素添加 border */
.parent {
  border-top: 1px solid transparent;
}

/* 方案 3：给父元素添加 padding */
.parent {
  padding-top: 1px;
}

/* 方案 4：使用 flex/grid 布局 */
.parent {
  display: flex;
  flex-direction: column;
}
