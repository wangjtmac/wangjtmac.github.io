浏览器安全 ​

同源策略 ​

同源策略概念 ​

javascript
// 同源策略（Same-Origin Policy）
// 协议、域名、端口都相同

// 示例
// https://www.example.com:443/page1.html
// https://www.example.com:443/page2.html // 同源
// http://www.example.com:443/page3.html // 不同源（协议不同）
// https://api.example.com:443/page4.html // 不同源（域名不同）
// https://www.example.com:8080/page5.html // 不同源（端口不同）

同源策略限制 ​

javascript
// 同源策略限制
// 1. Cookie、LocalStorage、IndexedDB
// 2. DOM 访问
// 3. AJAX 请求

// 示例
// Cookie 限制
document.cookie // 只能读取同源的 Cookie

// DOM 访问限制
const iframe = document.getElementById('iframe')
const iframeWindow = iframe.contentWindow // 不同源会报错

// AJAX 请求限制
fetch('https://api.example.com/data') // 不同源会报错

XSS 攻击 ​

XSS 攻击概念 ​

javascript
// XSS（Cross-Site Scripting）
// 跨站脚本攻击
// 恶意脚本注入到网页

// 示例
// 不安全的输入
const userInput = '<script>alert("XSS")</script>'
element.innerHTML = userInput // 执行恶意脚本

// 安全的输入
const userInput = '<script>alert("XSS")</script>'
element.textContent = userInput // 不执行脚本

XSS 攻击防护 ​

javascript
// XSS 攻击防护
// 1. 输入过滤
// 2. 输出编码
// 3. 使用 CSP（Content Security Policy）

// 输入过滤
function sanitizeInput(input) {
  return input.replace(/<script.*?>.*?<\/script>/gi, '')
}

// 输出编码
function escapeHtml(unsafe) {
  return unsafe
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#039;")
}

// 使用 CSP
// <meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self' 'unsafe-inline'">

CSRF 攻击 ​

CSRF 攻击概念 ​

javascript
// CSRF（Cross-Site Request Forgery）
// 跨站请求伪造
// 利用用户的身份发送请求

// 示例
// 恶意网站
<img src="https://bank.example.com/transfer?to=attacker&amount=10000">

// 用户访问恶意网站后
// 浏览器自动发送请求
// 使用用户的 Cookie

CSRF 攻击防护 ​

javascript
// CSRF 攻击防护
// 1. 使用 CSRF Token
// 2. 验证 Referer
// 3. 使用 SameSite Cookie

// 使用 CSRF Token
// 服务器生成 Token
const csrfToken = generateToken()

// 客户端发送 Token
fetch('/api/transfer', {
  method: 'POST',
  headers: {
    'X-CSRF-Token': csrfToken
  },
  body: JSON.stringify({ to: 'user', amount: 100 })
})

// 验证 Referer
function validateReferer(referer) {
  const allowedDomains = ['https://bank.example.com']
  return allowedDomains.some(domain => referer.startsWith(domain))
}

// 使用 SameSite Cookie
document.cookie = "sessionid=xxx; samesite=strict"

CORS ​

CORS 概念 ​

javascript
// CORS（Cross-Origin Resource Sharing）
// 跨域资源共享
// 允许服务器声明哪些源可以访问资源

// 简单请求
// GET
// HEAD
// POST
// Content-Type: text/plain, multipart/form-data, application/x-www-form-urlencoded

// 预检请求（Preflight）
// OPTIONS
// Content-Type: application/json

CORS 配置 ​

javascript
// 服务器配置 CORS
// Express
app.use((req, res, next) => {
  res.header('Access-Control-Allow-Origin', '*')
  res.header('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE')
  res.header('Access-Control-Allow-Headers', 'Content-Type, Authorization')
  next()
})

// Nginx
add_header 'Access-Control-Allow-Origin' '*';
add_header 'Access-Control-Allow-Methods' 'GET, POST, PUT, DELETE';
add_header 'Access-Control-Allow-Headers' 'Content-Type, Authorization';

CORS 请求 ​

javascript
// CORS 请求
fetch('https://api.example.com/data', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  credentials: 'include', // 包含 Cookie
  body: JSON.stringify({ name: '张三' })
})
  .then(response => response.json())
  .then(data => console.log(data))

面试题 ​

1. 什么是同源策略？ ​

答案：

同源策略的概念：

  1. 定义:

    • 协议、域名、端口都相同
    • 限制跨域访问

  2. 限制:

    • Cookie、LocalStorage、IndexedDB
    • DOM 访问
    • AJAX 请求

  3. 示例:

    javascript
    // https://www.example.com:443/page1.html
// https://www.example.com:443/page2.html // 同源
// http://www.example.com:443/page3.html // 不同源

2. 什么是 XSS 攻击？ ​

答案：

XSS 攻击的概念：

  1. 定义:

    • 跨站脚本攻击
    • 恶意脚本注入到网页

  2. 防护:

    • 输入过滤
    • 输出编码
    • 使用 CSP

  3. 示例:

    javascript
    // 不安全
element.innerHTML = userInput

// 安全
element.textContent = userInput

3. 什么是 CSRF 攻击？ ​

答案：

CSRF 攻击的概念：

  1. 定义:

    • 跨站请求伪造
    • 利用用户的身份发送请求

  2. 防护:

    • 使用 CSRF Token
    • 验证 Referer
    • 使用 SameSite Cookie

  3. 示例:

    javascript
    // 恶意网站
<img src="https://bank.example.com/transfer?to=attacker&amount=10000">

4. 什么是 CORS？ ​

答案：

CORS 的概念：

  1. 定义:

    • 跨域资源共享
    • 允许服务器声明哪些源可以访问资源

  2. 请求类型:

    • 简单请求
    • 预检请求（Preflight）

  3. 配置:

    javascript
    res.header('Access-Control-Allow-Origin', '*')
res.header('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE')

5. 如何防止点击劫持？ ​

答案：

防止点击劫持的方法：

  1. 使用 X-Frame-Options:

    http
    X-Frame-Options: DENY
X-Frame-Options: SAMEORIGIN

  2. 使用 CSP frame-ancestors:

    http
    Content-Security-Policy: frame-ancestors 'self'
