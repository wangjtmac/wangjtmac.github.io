JavaScript 性能优化 ​

JavaScript 性能优化相关面试题，包括内存泄漏、垃圾回收、代码优化、V8 引擎优化等。

内存泄漏 ​

常见内存泄漏场景 ​

场景说明解决方案
全局变量未释放的变量及时释放
闭包未释放的闭包引用及时释放引用
DOM 引用移除的 DOM 仍被引用移除引用
定时器未清除的定时器清除定时器
事件监听器未移除的事件监听器移除监听器

示例 ​

javascript
// 1. 全局变量泄漏
var globalData = { /* 大对象 */ };
// 解决：使用完及时释放
globalData = null;

// 2. 闭包泄漏
function outer() {
  var data = { /* 大对象 */ };
  return function() {
    console.log(data);
  };
}
var fn = outer();
// 解决：使用完及时释放
fn = null;

// 3. DOM 引用泄漏
var element = document.getElementById('element');
document.body.removeChild(element);
// 解决：移除引用
element = null;

// 4. 定时器泄漏
var timer = setInterval(function() {
  console.log('tick');
}, 1000);
// 解决：清除定时器
clearInterval(timer);

// 5. 事件监听器泄漏
function handler() {
  console.log('click');
}
element.addEventListener('click', handler);
// 解决：移除监听器
element.removeEventListener('click', handler);

常见面试问题 ​

Q: 如何检测内存泄漏？

使用 Chrome 开发者工具的 Memory 面板或 Performance 面板。

Q: 如何避免内存泄漏？

  • 及时释放全局变量
  • 及时释放闭包引用
  • 及时移除 DOM 引用
  • 及时清除定时器和事件监听器

垃圾回收 ​

垃圾回收机制 ​

JavaScript 使用自动垃圾回收（GC）管理内存。

回收算法 ​

算法说明
引用计数记录对象被引用次数，为 0 时回收
标记清除从根开始标记可达对象，清除未标记的

可达对象 ​

javascript
// 全局对象
var globalObj = {};

function fn() {
  var localObj = {};
  globalObj.ref = localObj;  // localObj 可达
}

fn();
// fn 执行后，localObj 不可达，可被回收
// globalObj.ref 仍然可达，不会被回收

Q: 垃圾回收的触发时机？

  • 内存不足时
  • 定期触发（具体由引擎决定）

Q: 如何手动触发垃圾回收？

javascript
// Node.js

// 浏览器（需要开启）
window.gc();

代码优化 ​

减少重排重绘 ​

javascript
// 不好的做法
element.style.width = '100px';
element.style.height = '100px';
element.style.background = 'red';

// 好的做法
element.style.cssText = 'width: 100px; height: 100px; background: red;';

使用文档片段 ​

javascript
// 不好的做法
for (let i = 0; i < 100; i++) {
  document.body.appendChild(createElement());
}

// 好的做法
const fragment = document.createDocumentFragment();
for (let i = 0; i < 100; i++) {
  fragment.appendChild(createElement());
}
document.body.appendChild(fragment);

事件委托 ​

javascript
// 不好的做法
const items = document.querySelectorAll('.item');
items.forEach(item => {
  item.addEventListener('click', handler);
});

// 好的做法
document.querySelector('.container').addEventListener('click', function(event) {
  if (event.target.classList.contains('item')) {
    handler(event);
  }
});

防抖节流 ​

javascript
// 防抖
function debounce(fn, delay) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流
function throttle(fn, delay) {
  let lastTime = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastTime >= delay) {
      fn.apply(this, args);
      lastTime = now;
    }
  };
}

常见面试问题 ​

Q: 重排和重绘的区别？

  • 重排（Reflow）：元素尺寸、位置改变，需要重新计算布局
  • 重绘（Repaint）：元素外观改变，不需要重新计算布局

Q: 如何减少重排重绘？

  • 批量修改 DOM
  • 使用文档片段
  • 避免逐条修改样式
  • 使用 transform 和 opacity

V8 引擎优化 ​

隐藏类 ​

javascript
// V8 会将对象分为隐藏类和普通类
// 隐藏类：属性少、变化慢
// 普通类：属性多、变化快

// 优化：保持对象结构稳定
function createUser(name, age) {
  return {
    name: name,
    age: age
  };
}

内联缓存 ​

javascript
// V8 会缓存对象属性访问
function process(obj) {
  const name = obj.name;  // 缓存
  const age = obj.age;    // 缓存
  // ...
}

数组优化 ​

javascript
// 稀疏数组会降低性能
// 优化：使用密集数组
const arr = new Array(1000);
for (let i = 0; i < arr.length; i++) {
  arr[i] = i;
}

Q: V8 的隐藏类是什么？

V8 将对象分为隐藏类和普通类，隐藏类优化了访问速度。

Q: 如何优化对象属性访问？

  • 保持对象结构稳定
  • 使用常量属性名
  • 避免频繁添加删除属性

性能检测 ​

Performance API ​

javascript
// 记录性能
performance.mark('start');

// 执行操作
const result = heavyOperation();

performance.mark('end');
performance.measure('operation', 'start', 'end');

const measure = performance.getEntriesByName('operation')[0];
console.log(measure.duration);  // 执行时间

PerformanceObserver ​

javascript
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    console.log(entry.name, entry.duration);
  }
});

observer.observe({ entryTypes: ['measure'] });

}

Q: 如何优化递归？

javascript
// 不好的做法
function factorial(n) {
  if (n <= 1) return 1;
  return n * factorial(n - 1);
}

// 好的做法（尾递归）
function factorial(n, acc = 1) {
  if (n <= 1) return acc;
  return factorial(n - 1, n * acc);
}
