常见面试问题 ​

Q: typeof null 为什么返回 "object"？

这是 JavaScript 的一个历史遗留问题。在 JavaScript 最初的实现中，值以 32 位单元存储，低 3 位用于类型标签，null 的机器码全为 0，因此类型标签也被识别为对象。

Q: 如何准确判断数组？

javascript
Array.isArray([]);  // true

// 或
Object.prototype.toString.call([]) === '[object Array]';  // true

常见面试问题 ​

Q: 什么是变量提升？

JavaScript 在代码执行前，会将变量和函数声明提升到作用域顶部。

Q: let 和 const 的区别？

  • let：可以重新赋值
  • const：声明后不能重新赋值（但对象属性可以修改）

常见面试问题 ​

Q: == 和 === 的区别？

  • ==：会进行类型转换后比较
  • ===：不会进行类型转换，严格比较
javascript
1 == '1';   // true
1 === '1';  // false

null == undefined;  // true
null === undefined; // false

Q: [] == ![] 为什么是 true？

javascript
![];         // false
[] == false;  // true

深浅拷贝 ​

浅拷贝 ​

只复制对象的第一层属性。

javascript
// 方法 1：Object.assign
const obj = { a: 1, b: { c: 2 } };
const copy = Object.assign({}, obj);

// 方法 2：展开运算符
const copy = { ...obj };

// 方法 3：Array.slice（数组）
const arr = [1, 2, 3];
const copy = arr.slice();

深拷贝 ​

递归复制对象的所有层级。

javascript
// 方法 1：JSON 方法
const obj = { a: 1, b: { c: 2 } };
const copy = JSON.parse(JSON.stringify(obj));

// 方法 2：递归
function deepClone(obj) {
  if (obj === null || typeof obj !== 'object') return obj;
  const clone = Array.isArray(obj) ? [] : {};
  for (let key in obj) {
    if (obj.hasOwnProperty(key)) {
      clone[key] = deepClone(obj[key]);
    }
  }
  return clone;
}

// 方法 3：structuredClone（现代浏览器）
const copy = structuredClone(obj);

常见面试问题 ​

Q: 深拷贝和浅拷贝的区别？

  • 浅拷贝：只复制第一层属性，嵌套对象仍然是引用
  • 深拷贝：递归复制所有层级，完全独立

Q: JSON.parse(JSON.stringify()) 的缺点？

  • 无法复制函数
  • 无法复制 undefined
  • 无法复制 Symbol
  • 循环引用会报错
