JavaScript 跨域与安全 ​

JavaScript 跨域与安全相关面试题，包括同源策略、跨域解决方案、XSS、CSRF 等。

同源策略 ​

什么是同源策略 ​

同源策略（Same-Origin Policy）是浏览器的一种安全机制，限制一个源的文档或脚本如何与另一个源的资源进行交互。

同源判断 ​

协议域名端口
httpexample.com80
httpsexample.com443

只有协议、域名、端口都相同才认为是同源。

常见面试问题 ​

Q: 以下哪些是同源？

  • http://example.com 和 http://example.com/page ✓
  • http://example.com 和 https://example.com ✗
  • http://example.com 和 http://example.com:8080 ✗
  • http://example.com 和 http://www.example.com ✗

跨域解决方案 ​

JSONP ​

javascript
// 前端
function handleResponse(data) {
  console.log(data);
}

const script = document.createElement('script');
script.src = 'http://api.example.com/data?callback=handleResponse';
document.body.appendChild(script);

// 后端
// http://api.example.com/data?callback=handleResponse
// 返回：handleResponse({ name: 'Alice', age: 20 })

CORS ​

javascript
// 前端
fetch('http://api.example.com/data', {
  method: 'GET',
  mode: 'cors',
  headers: {
    'Content-Type': 'application/json'
  }
})
  .then(response => response.json())
  .then(data => console.log(data));

// 后端
// 设置响应头
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type
Access-Control-Allow-Credentials: true

postMessage ​

javascript
// 父页面
const iframe = document.createElement('iframe');
iframe.src = 'http://child.example.com';
document.body.appendChild(iframe);

// 父页面发送消息
iframe.contentWindow.postMessage({ type: 'greeting', data: 'Hello' }, 'http://child.example.com');

// 子页面接收消息
window.addEventListener('message', function(event) {
  if (event.origin === 'http://parent.example.com') {
    console.log(event.data);  // { type: 'greeting', data: 'Hello' }
  }
});

// 子页面发送消息
event.source.postMessage({ type: 'response', data: 'World' }, event.origin);

WebSocket ​

javascript
const socket = new WebSocket('ws://api.example.com');

socket.onopen = function() {
  console.log('WebSocket connected');
};

socket.onmessage = function(event) {
  console.log('Received:', event.data);
};

socket.send('Hello, Server!');

代理服务器 ​

javascript
// 前端
fetch('/api/data')  // 通过代理服务器请求

// 代理服务器（如 Nginx）
location /api/ {
  proxy_pass http://api.example.com/;
}

常见面试问题 ​

Q: CORS 的预检请求是什么？

对于复杂请求（如 PUT、DELETE），浏览器会先发送 OPTIONS 请求，检查服务器是否允许跨域。

Q: JSONP 的缺点？

  • 只支持 GET 请求
  • 存在 XSS 风险
  • 错误处理困难

XSS（跨站脚本攻击） ​

什么是 XSS ​

攻击者向网页中注入恶意脚本，当用户访问时执行恶意代码。

XSS 类型 ​

类型说明示例
存储型 XSS恶意代码存储在服务器评论、留言板
反射型 XSS恶意代码通过 URL 传递http://example.com?name=<script>alert(1)</script>
DOM 型 XSS恶意代码通过 DOM 操作element.innerHTML = userInput

防御措施 ​

javascript
// 1. 输入过滤
function sanitizeInput(input) {
  return input.replace(/<[^>]*>/g, '');
}

// 2. 输出编码
function encodeOutput(output) {
  return output
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#39;');
}

// 3. 使用 textContent 而非 innerHTML
element.textContent = userInput;  // 安全
element.innerHTML = userInput;  // 不安全

// 4. CSP（内容安全策略）
// HTTP 头
Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline';

常见面试问题 ​

Q: 如何防止 XSS 攻击？

  • 输入过滤和编码
  • 使用 textContent 而非 innerHTML
  • 设置 CSP 头
  • 使用 HttpOnly Cookie

CSRF（跨站请求伪造） ​

什么是 CSRF ​

攻击者诱导用户在已登录的网站上执行非预期的操作。

CSRF 攻击流程 ​

1. 用户登录银行网站
2. 攻击者发送恶意链接给用户
3. 用户点击链接
4. 浏览器携带银行的 Cookie 访问恶意网站
5. 银行执行转账操作

防御措施 ​

javascript
// 1. CSRF Token
// 前端
fetch('/transfer', {
  method: 'POST',
  headers: {
    'X-CSRF-Token': getCsrfToken()
  },
  body: JSON.stringify({ amount: 1000 })
});

// 后端
// 验证 Token
if (request.headers['X-CSRF-Token'] !== session.csrfToken) {
  throw new Error('CSRF Token mismatch');
}

// 2. SameSite Cookie
// 设置 Cookie
Set-Cookie: sessionId=xxx; SameSite=Strict; HttpOnly

// 3. 验证 Referer
// 后端验证请求来源
if (request.headers['Referer'] !== 'https://bank.example.com') {
  throw new Error('Invalid Referer');
}

// 4. 双重提交 Cookie
// 设置双重提交 Cookie
Set-Cookie: csrfToken1=xxx; Path=/;
Set-Cookie: csrfToken2=xxx; Path=/;

常见面试问题 ​

Q: CSRF 和 XSS 的区别？

对比项XSSCSRF
攻击目标用户服务器
执行位置用户浏览器服务器
防御重点输出编码Token 验证

Q: SameSite Cookie 的作用？

限制 Cookie 的发送范围，防止 CSRF 攻击。

// 防御措施
// 1. X-Frame-Options
X-Frame-Options: DENY

// 3. JS 防御

  window.top.location = window.self.location;
}

Q: 如何防止点击劫持？

使用 X-Frame-Options 或 CSP 头限制页面被嵌入 iframe。

Q: HTTPS 的作用？

加密数据传输，防止中间人攻击。
