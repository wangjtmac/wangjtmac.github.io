面试题 ​

1. 小程序的网络请求有哪些限制？ ​

参考答案：

小程序的网络请求限制：

  1. 域名限制

    • 必须在微信公众平台配置服务器域名
    • 只能请求配置的域名
    • 不支持IP地址

  2. 协议限制

    • 必须使用HTTPS协议
    • 不支持HTTP协议
    • 支持WebSocket

  3. 并发限制

    • 同时最多10个请求
    • 超过限制会排队等待

  4. 超时限制

    • 默认超时时间60秒
    • 可以在配置中修改

  5. 请求方法限制

    • 支持GET、POST、PUT、DELETE等
    • 不支持所有HTTP方法

2. 小程序的本地存储有哪些限制？ ​

参考答案：

小程序的本地存储限制：

  1. 存储大小限制

    • 单个key最大1MB
    • 总存储空间最大10MB
    • 超过限制会报错

  2. 存储类型限制

    • 支持Object、Array、String、Number、Boolean
    • 自动序列化和反序列化
    • 不支持Function、Symbol等

  3. 存储位置限制

    • 存储在本地
    • 不同小程序隔离
    • 清除缓存会删除

  4. 存储性能限制

    • 同步API会阻塞
    • 异步API不会阻塞
    • 频繁操作影响性能

6. 小程序的登录流程是怎样的？ ​

  1. 获取code

    javascript
    wx.login({
  success(res) {
    if (res.code) {
      // 发送code到服务器
      wx.request({
        url: 'https://example.com/login',
        method: 'POST',
        data: {

        },

  2. 服务器端处理

    • 生成自定义登录态
    • 返回给小程序

  3. 存储登录态

    • 存储到本地
    • 后续请求携带登录态
    • 登录态过期重新登录

  4. 获取用户信息

    javascript
    wx.getUserInfo({
  success(res) {
    console.log(res.userInfo)
  }
})

7. 小程序的支付流程是怎样的？ ​

参考答案：

小程序的支付流程：

  1. 统一下单

    javascript
    wx.request({
  url: 'https://example.com/unifiedorder',
  method: 'POST',
  data: {
    openid: 'openid',
    total_fee: 100
  },
  success(res) {
    const { timeStamp, nonceStr, package, signType, paySign } = res.data
    // 调起支付
    wx.requestPayment({
      timeStamp,
      nonceStr,
      package,
      signType,
      paySign,
      success(res) {
        console.log('支付成功')
      },
      fail(res) {
        console.log('支付失败')
      }
    })
  }
})

  2. 支付回调

    • 支付成功后，微信服务器会通知商户服务器
    • 商户服务器验证签名
    • 更新订单状态

  3. 支付结果查询

    • 可以查询支付结果
    • 确保订单状态正确
