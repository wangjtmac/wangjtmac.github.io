组件化设计原则 ​

组件化设计的原则：

  1. 单一职责：每个组件只负责一个功能
  2. 高内聚低耦合：组件内部高内聚，组件之间低耦合
  3. 可复用性：组件应该易于复用
  4. 可维护性：组件应该易于维护
  5. 可测试性：组件应该易于测试

组件分类 ​

组件的分类：

  1. 基础组件：Button、Input、Modal 等
  2. 业务组件：UserCard、PostList 等
  3. 页面组件：HomePage、AboutPage 等
  4. 布局组件：Header、Footer、Sidebar 等

组件通信 ​

组件通信的方式：

  1. Props：父组件向子组件传递数据
  2. Events：子组件向父组件传递数据
  3. Provide/Inject：跨层级组件通信
  4. Event Bus：使用事件总线通信
  5. Vuex/Redux：使用状态管理通信
  6. Ref：直接访问子组件

组件复用 ​

组件复用的方式：

  1. 插槽：使用插槽实现内容分发
  2. 作用域插槽：使用作用域插槽传递数据
  3. 高阶组件：使用高阶组件增强组件
  4. 组合式函数：使用组合式函数复用逻辑

3. 如何实现组件通信？ ​

答案：

组件通信的方式：

  1. Props：父组件向子组件传递数据
  2. Events：子组件向父组件传递数据
  3. Provide/Inject：跨层级组件通信
  4. Event Bus：使用事件总线通信
  5. Vuex/Redux：使用状态管理通信
  6. Ref：直接访问子组件

示例：

javascript
// Props：父组件向子组件传递数据
const Parent = () => {
  const [message, setMessage] = useState('Hello')
  
  return <Child message={message} />
}

const Child = ({ message }) => {
  return <p>{message}</p>
}

// Events：子组件向父组件传递数据
const Parent = () => {
  const handleMessage = (data) => {
    console.log('Received:', data)
  }
  
  return <Child onUpdate={handleMessage} />
}

const Child = ({ onUpdate }) => {
  const handleClick = () => {
    onUpdate('Hello from child')
  }
  
  return <button onClick={handleClick}>Send</button>
}

// Provide/Inject：跨层级组件通信
const Parent = () => {
  const data = { message: 'Hello' }
  
  return (
    <DataContext.Provider value={data}>
      <Child />
    </DataContext.Provider>
  )
}

const Child = () => {
  return <GrandChild />
}

const GrandChild = () => {
  const data = useContext(DataContext)
  return <p>{data.message}</p>
}

// Event Bus：使用事件总线通信
const eventBus = new EventEmitter()

const ComponentA = () => {
  const handleClick = () => {
    eventBus.emit('update', { message: 'Hello' })
  }
  
  return <button onClick={handleClick}>Send</button>
}

const ComponentB = () => {
  useEffect(() => {
    const handler = (data) => {
      console.log('Received:', data)
    }
    eventBus.on('update', handler)
    
    return () => {
      eventBus.off('update', handler)
    }
  }, [])
  
  return <div>Component B</div>
}

const List = ({ items, renderItem }) => {

// 组合式函数：使用组合式函数复用逻辑
function useWindowSize() {
  const [size, setSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight
  })
  
  useEffect(() => {
    const handleResize = () => {
      setSize({
        width: window.innerWidth,
        height: window.innerHeight
      })
    }
    
    window.addEventListener('resize', handleResize)
    return () => {
      window.removeEventListener('resize', handleResize)
    }
  }, [])
  
  return size
}

const Component = () => {
  const { width, height } = useWindowSize()
  
  return (
    <div>
      Width: {width}, Height: {height}
    </div>
  )

5. 如何设计可复用的组件？ ​

答案：

设计可复用组件的原则：

  1. 单一职责：每个组件只负责一个功能
  2. 可配置性：通过 props 配置组件行为
  3. 插槽支持：支持插槽实现内容分发
  4. 事件回调：通过事件回调与父组件通信
  5. 样式定制：支持样式定制

示例：

javascript
// 可复用的按钮组件
const Button = ({
  type = 'default',
  size = 'medium',
  disabled = false,
  loading = false,
  onClick,
  children,
  className = ''
}) => {
  const handleClick = (e) => {
    if (disabled || loading) return
    onClick?.(e)
  }
  
  const classes = [
    'btn',
    `btn-${type}`,
    `btn-${size}`,
    disabled && 'btn-disabled',
    loading && 'btn-loading',
    className
  ].filter(Boolean).join(' ')
  
  return (
    <button className={classes} onClick={handleClick} disabled={disabled || loading}>
      {loading && <span className="btn-spinner" />}
      {children}
    </button>
  )
}

// 使用
<Button type="primary" size="large" onClick={handleClick}>
  Submit
</Button>

// 可复用的模态框组件
const Modal = ({
  visible = false,
  title = '',
  onClose,
  onConfirm,
  children,
  footer = null
}) => {
  if (!visible) return null
  
  return (
    <div className="modal-overlay" onClick={onClose}>
      <div className="modal" onClick={(e) => e.stopPropagation()}>
        <div className="modal-header">
          <h3>{title}</h3>
          <button className="modal-close" onClick={onClose}>×</button>
        </div>
        <div className="modal-body">
          {children}
        </div>
        <div className="modal-footer">
          {footer || (
            <>
              <Button onClick={onClose}>Cancel</Button>
              <Button type="primary" onClick={onConfirm}>Confirm</Button>
            </>
          )}
        </div>
      </div>
    </div>
  )
}

// 使用
<Modal
  visible={visible}
  title="Confirm"
  onClose={() => setVisible(false)}
  onConfirm={handleConfirm}
>
  <p>Are you sure you want to delete this item?</p>
</Modal>

6. 如何优化组件性能？ ​

  1. React.memo：使用 React.memo 避免不必要的重新渲染
  2. useMemo：使用 useMemo 缓存计算结果
  3. useCallback：使用 useCallback 缓存函数
  4. 虚拟列表：使用虚拟列表优化长列表渲染
  5. 懒加载：使用懒加载减少初始加载时间

示例：

javascript
// React.memo：避免不必要的重新渲染
const ExpensiveComponent = React.memo(({ data }) => {
  return <div>{data}</div>
})

// useMemo：缓存计算结果
const Component = ({ items }) => {
  const total = useMemo(() => {
    return items.reduce((sum, item) => sum + item.value, 0)
  }, [items])
  
  return <div>Total: {total}</div>
}

// useCallback：缓存函数
const Component = ({ onClick }) => {
  const handleClick = useCallback(() => {
    onClick('data')
  }, [onClick])
  
  return <button onClick={handleClick}>Click</button>
}

// 虚拟列表：优化长列表渲染
import { FixedSizeList } from 'react-window'

const List = ({ items }) => {

    <div style={style}>
      {items[index].name}

  )

    <FixedSizeList
      height={400}
      itemCount={items.length}
      itemSize={35}
      width="100%"
    >
      {Row}
    </FixedSizeList>
  )

// 懒加载：减少初始加载时间
const LazyComponent = React.lazy(() => import('./LazyComponent'))

const App = () => {
  return (
    <React.Suspense fallback={<div>Loading...</div>}>
      <LazyComponent />
    </React.Suspense>

}

8. 如何设计组件的状态管理？ ​

答案：

设计组件状态管理的原则：

  1. 本地状态：使用 useState 管理组件内部状态
  2. 全局状态：使用 Redux、Pinia 管理全局状态
  3. 共享状态：使用 Context API 管理共享状态
  4. 状态提升：将状态提升到最近的共同父组件

示例：

javascript
// 本地状态
const Component = () => {
  const [count, setCount] = useState(0)
  
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  )
}

// 全局状态（Redux）
const store = createStore(reducer)

const Component = () => {
  const count = useSelector(state => state.count)
  const dispatch = useDispatch()
  
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => dispatch({ type: 'INCREMENT' })}>Increment</button>
    </div>
  )
}

// 共享状态（Context API）
const CountContext = createContext()

const CountProvider = ({ children }) => {
  const [count, setCount] = useState(0)
  
  return (
    <CountContext.Provider value={{ count, setCount }}>
      {children}
    </CountContext.Provider>
  )
}

const Component = () => {
  const { count, setCount } = useContext(CountContext)
  
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  )
}

// 状态提升
const Parent = () => {
  const [count, setCount] = useState(0)
  
  return (
    <div>
      <ChildA count={count} />
      <ChildB setCount={setCount} />
    </div>
  )
}

const ChildA = ({ count }) => {
  return <p>Count: {count}</p>
}

const ChildB = ({ setCount }) => {
  return <button onClick={() => setCount(prev => prev + 1)}>Increment</button>
}

9. 如何设计组件的样式？ ​

答案：

设计组件样式的方法：

  1. CSS Modules：使用 CSS Modules 避免样式冲突
  2. Styled Components：使用 Styled Components 编写样式
  3. Tailwind CSS：使用 Tailwind CSS 快速构建样式
  4. CSS-in-JS：使用 CSS-in-JS 编写动态样式

示例：

javascript
// CSS Modules
import styles from './Button.module.css'

const Button = ({ type, children }) => {
  return (
    <button className={`${styles.button} ${styles[type]}`}>
      {children}
    </button>
  )
}

// Styled Components
import styled from 'styled-components'

const StyledButton = styled.button`
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  background-color: ${props => props.type === 'primary' ? '#007bff' : '#6c757d'};
  color: white;
  cursor: pointer;
  
  &:hover {
    background-color: ${props => props.type === 'primary' ? '#0056b3' : '#5a6268'};
  }
`

const Button = ({ type, children }) => {
  return <StyledButton type={type}>{children}</StyledButton>
}

// Tailwind CSS
const Button = ({ type, children }) => {
  const typeClasses = {
    primary: 'bg-blue-500 hover:bg-blue-700',
    secondary: 'bg-gray-500 hover:bg-gray-700'
  }
  
  return (
    <button className={`px-5 py-2 rounded text-white cursor-pointer ${typeClasses[type]}`}>
      {children}
    </button>
  )
}

// CSS-in-JS
import { css } from '@emotion/react'

const buttonStyles = (type) => css`
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  background-color: ${type === 'primary' ? '#007bff' : '#6c757d'};
  color: white;
  cursor: pointer;
  
  &:hover {
    background-color: ${type === 'primary' ? '#0056b3' : '#5a6268'};
  }
`

const Button = ({ type, children }) => {
  return <button css={buttonStyles(type)}>{children}</button>
}

答案：

设计组件测试的方法：

  1. 单元测试：使用 Jest、React Testing Library 进行单元测试
  2. 集成测试：使用 Cypress、Playwright 进行集成测试
  3. 快照测试：使用 Jest 进行快照测试
  4. E2E 测试：使用 Cypress、Playwright 进行 E2E 测试

示例：

javascript
// 单元测试（React Testing Library）
import { render, screen, fireEvent } from '@testing-library/react'
import Button from './Button'

test('renders button with text', () => {
  render(<Button>Click me</Button>)
  expect(screen.getByText('Click me')).toBeInTheDocument()
})

test('calls onClick when clicked', () => {
  const handleClick = jest.fn()
  render(<Button onClick={handleClick}>Click me</Button>)
  
  fireEvent.click(screen.getByText('Click me'))
  expect(handleClick).toHaveBeenCalledTimes(1)
})

// 快照测试
test('matches snapshot', () => {
  const { asFragment } = render(<Button>Click me</Button>)
  expect(asFragment()).toMatchSnapshot()
})

// 集成测试（Cypress）
describe('Button', () => {
  it('should call onClick when clicked', () => {
    cy.mount(<Button onClick={cy.stub().as('onClick')}>Click me</Button>)
    cy.contains('Click me').click()
    cy.get('@onClick').should('have.been.calledOnce')
  })
})
