构建工具 ​

构建工具常用场景 ​

1. 代码转换 ​

场景描述：将现代 JavaScript/TypeScript 语法转换为浏览器兼容的代码

常用工具：

  • Babel：ES6+ 转 ES5
  • TypeScript：类型检查和转换
  • PostCSS：CSS 转换和优化

示例：

javascript
// 源代码
const greet = (name) => `Hello, ${name}!`

// 转换后
var greet = function greet(name) {
  return "Hello, " + name + "!";
}

2. 模块打包 ​

场景描述：将多个模块打包成一个或多个文件，解决模块依赖问题

打包策略：

  • 单文件打包：适合小型应用
  • 代码分割：按路由或功能拆分
  • 按需加载：动态导入

示例：

javascript
// 源代码
import { add } from './math'
import { greet } from './utils'

// 打包后
// bundle.js 包含所有依赖的代码

3. 资源处理 ​

场景描述：处理 CSS、图片、字体等静态资源

处理方式：

  • CSS：CSS Modules、Sass/Less、PostCSS
  • 图片：压缩、格式转换、Base64 编码
  • 字体：字体子集化、格式优化

示例：

javascript
// CSS 处理
{
  test: /\.css$/,
  use: ['style-loader', 'css-loader', 'postcss-loader']
}

// 图片处理
{
  test: /\.(png|jpg|jpeg|gif)$/,
  use: {
    loader: 'url-loader',
    options: {
      limit: 8192 // 小于 8KB 转为 Base64
    }
  }
}

4. 代码优化 ​

场景描述：压缩代码、移除无用代码、优化性能

优化技术：

  • Tree Shaking：移除未使用的代码
  • 代码压缩：减小文件体积
  • 作用域提升：减少函数调用开销

示例：

javascript
// 源代码
export function add(a, b) { return a + b }
export function unused() { console.log('never called') }

// Tree Shaking 后
export function add(a, b) { return a + b }
// unused 函数被移除

5. 开发体验 ​

场景描述：提供热更新、源码映射等开发环境支持

开发工具：

  • 热更新：修改代码自动刷新
  • 源码映射：调试原始源代码
  • 开发服务器：本地开发环境

示例：

javascript
// 热更新配置
devServer: {
  hot: true,
  open: true
}

// 源码映射
devtool: 'source-map'

6. 环境适配 ​

场景描述：根据不同环境（开发、测试、生产）使用不同配置

环境变量：

  • 区分开发和生产环境
  • 配置不同的 API 地址
  • 启用/禁用调试功能

示例：

javascript
// 环境变量
const isProduction = process.env.NODE_ENV === 'production'

// 条件编译
if (isProduction) {
  // 生产环境配置
} else {
  // 开发环境配置
}

7. 性能优化 ​

场景描述：通过代码分割、懒加载等技术提升应用性能

优化策略：

  • 代码分割：按路由或功能拆分
  • 懒加载：按需加载模块
  • 预加载：提前加载关键资源

示例：

javascript
// 路由懒加载
const Home = () => import('./views/Home')
const About = () => import('./views/About')

// 动态导入
const module = await import('./module')

8. 构建流程 ​

场景描述：自动化构建、测试、部署流程

构建工具：

  • Webpack：功能强大，配置灵活
  • Vite：开发体验好，构建速度快
  • Rollup：适合库开发
  • Parcel：零配置

选择建议：

  • 复杂项目：Webpack
  • 新项目：Vite
  • 库开发：Rollup
  • 快速原型：Parcel

Webpack ​

基本配置 ​

javascript
const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const MiniCssExtractPlugin = require('mini-css-extract-plugin')
const { CleanWebpackPlugin } = require('clean-webpack-plugin')

module.exports = {
  // 入口配置：指定应用的入口文件
  entry: './src/index.js',

  // 输出配置：指定打包后的文件输出位置和名称
  output: {
    filename: 'bundle.js',                    // 输出文件名
    path: path.resolve(__dirname, 'dist'),     // 输出目录
    publicPath: '/'                           // 公共路径
  },

  // 模式配置：development 或 production
  mode: 'production',

  // 模块配置：处理不同类型的文件
  module: {
    rules: [
      // JavaScript 文件处理：使用 Babel 转换 ES6+ 代码
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        }
      },
      // CSS 文件处理：提取 CSS 到单独文件
      {
        test: /\.css$/,
        use: [
          MiniCssExtractPlugin.loader,  // 提取 CSS 到单独文件
          'css-loader',               // 解析 CSS
          'postcss-loader'            // CSS 后处理
        ]
      },
      // 图片文件处理：小于 8KB 转为 Base64，否则输出为文件
      {
        test: /\.(png|jpg|jpeg|gif)$/,
        use: {
          loader: 'url-loader',
          options: {
            limit: 8192,              // 小于 8KB 转为 Base64
            name: '[name].[hash:8].[ext]', // 文件名格式
            outputPath: 'images/'       // 输出目录
          }
        }
      }
    ]
  },

  // 插件配置：扩展 Webpack 功能
  plugins: [
    // 生成 HTML 文件，自动引入打包后的 JS 和 CSS
    new HtmlWebpackPlugin({
      template: './src/index.html',    // HTML 模板文件
      filename: 'index.html',         // 输出文件名
      minify: true                   // 压缩 HTML
    }),
    // 提取 CSS 到单独文件
    new MiniCssExtractPlugin({
      filename: 'styles.css'
    }),
    // 清理输出目录
    new CleanWebpackPlugin()
  ],

  // 开发服务器配置
  devServer: {
    contentBase: './dist',           // 静态文件目录
    port: 8080,                    // 端口号
    hot: true,                      // 热更新
    open: true                      // 自动打开浏览器
  },

  // 解析配置：模块解析规则
  resolve: {
    extensions: ['.js', '.jsx', '.json'], // 自动解析的扩展名
    alias: {
      '@': path.resolve(__dirname, 'src') // 路径别名
    }
  }
}

Loader 详解 ​

Loader 用于转换模块的源代码，让 Webpack 能够处理非 JavaScript 文件。

Babel Loader ​

javascript
{
  test: /\.js$/,                    // 匹配 .js 文件
  exclude: /node_modules/,          // 排除 node_modules
  use: {
    loader: 'babel-loader',          // 使用 Babel 转换
    options: {
      presets: ['@babel/preset-env'], // ES6+ 转 ES5
      plugins: ['@babel/plugin-transform-runtime'] // 转换新语法
    }
  }
}

功能说明：

  • 将 ES6+ 语法转换为 ES5
  • 支持 JSX 语法
  • 转换新的 JavaScript 特性

CSS Loader ​

javascript
{
  test: /\.css$/,
  use: [
    'style-loader',       // 将 CSS 注入到 style 标签
    'css-loader',        // 解析 @import 和 url()
    'postcss-loader'     // CSS 后处理（自动添加前缀等）
  ]
}

功能说明：

  • style-loader：将 CSS 注入到 HTML 的 style 标签
  • css-loader：解析 CSS 中的 @import 和 url()
  • postcss-loader：使用 PostCSS 处理 CSS（自动添加浏览器前缀等）

File Loader / URL Loader ​

javascript
{
  test: /\.(png|jpg|jpeg|gif|svg)$/,
  use: {
    loader: 'url-loader',
    options: {
      limit: 8192,              // 小于 8KB 转为 Base64
      name: '[name].[hash:8].[ext]',
      outputPath: 'images/',
      esModule: false            // 兼容旧版本
    }
  }
}

功能说明：

  • 小于 limit 的图片转为 Base64，减少 HTTP 请求
  • 大于 limit 的图片输出为文件
  • 支持图片压缩和格式转换

Plugin 详解 ​

Plugin 用于扩展 Webpack 的功能，在构建过程的特定时机执行。

HtmlWebpackPlugin ​

javascript
new HtmlWebpackPlugin({
  template: './src/index.html',    // HTML 模板路径
  filename: 'index.html',         // 输出文件名
  inject: 'body',                // 注入位置：head 或 body
  minify: {                      // 压缩选项
    removeComments: true,
    collapseWhitespace: true
  },
  chunks: 'all'                   // 包含的 chunks
})

功能说明：

  • 自动生成 HTML 文件
  • 自动引入打包后的 JS 和 CSS
  • 支持压缩和自定义模板

MiniCssExtractPlugin ​

javascript
new MiniCssExtractPlugin({
  filename: 'styles.css',         // 输出文件名
  chunkFilename: '[id].css'       // 异步加载的 CSS 文件名
})

功能说明：

  • 将 CSS 提取到单独文件
  • 支持 CSS 代码分割
  • 减少 JS 文件体积

CleanWebpackPlugin ​

javascript
new CleanWebpackPlugin({
  cleanOnceBeforeBuildPatterns: ['**/*'], // 清理所有文件
  verbose: true                          // 显示清理日志
})

功能说明：

  • 每次构建前清理输出目录
  • 避免旧文件残留

DefinePlugin ​

javascript
new webpack.DefinePlugin({
  'process.env.NODE_ENV': JSON.stringify(process.env.NODE_ENV),
  'API_BASE_URL': JSON.stringify('https://api.example.com')
})

功能说明：

  • 定义全局常量
  • 用于环境变量配置
  • 在编译时替换代码中的变量

优化配置 ​

javascript
module.exports = {
  optimization: {
    // 代码分割：将代码拆分成多个文件
    splitChunks: {
      chunks: 'all',                    // 对所有模块进行分割
      minSize: 20000,                  // 最小分割大小
      maxSize: 0,                      // 最大分割大小
      minChunks: 1,                     // 最小引用次数
      maxAsyncRequests: 30,             // 最大异步请求数
      maxInitialRequests: 30,            // 最大初始请求数
      automaticNameDelimiter: '~',         // 名称分隔符
      cacheGroups: {
        // 第三方库单独打包
        vendor: {
          test: /[\\/]node_modules[\\/]/, // 匹配 node_modules
          name: 'vendor',
          chunks: 'all',
          priority: 10
        },
        // 公共模块单独打包
        common: {
          name: 'common',
          minChunks: 2,                 // 至少被引用 2 次
          chunks: 'initial',
          priority: 5,
          reuseExistingChunk: true
        }
      }
    },

    // 代码压缩
    minimize: true,
    minimizer: [
      new TerserPlugin({
        terserOptions: {
          compress: {
            drop_console: true,          // 移除 console
            drop_debugger: true          // 移除 debugger
          }
        }
      }),
      new CssMinimizerPlugin()
    ],

    // Tree Shaking：移除未使用的代码
    usedExports: true,
    sideEffects: false,

    // 运行时代码优化
    runtimeChunk: {
      name: 'runtime'                  // 提取运行时代码
    }
  }
}

优化说明：

  • splitChunks：代码分割，减少重复代码，提高缓存效率
  • minimize：压缩代码，减小文件体积
  • usedExports：Tree Shaking，移除未使用的代码
  • runtimeChunk：提取运行时代码，避免重复

完整配置示例 ​

javascript
const path = require('path')
const webpack = require('webpack')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const MiniCssExtractPlugin = require('mini-css-extract-plugin')
const { CleanWebpackPlugin } = require('clean-webpack-plugin')
const TerserPlugin = require('terser-webpack-plugin')
const CssMinimizerPlugin = require('css-minimizer-webpack-plugin')

const isProduction = process.env.NODE_ENV === 'production'

module.exports = {
  entry: './src/index.js',
  output: {
    filename: isProduction ? '[name].[contenthash:8].js' : '[name].js',
    path: path.resolve(__dirname, 'dist'),
    publicPath: '/'
  },
  mode: isProduction ? 'production' : 'development',
  devtool: isProduction ? 'source-map' : 'eval-cheap-module-source-map',
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env'],
            cacheDirectory: true          // 启用缓存
          }
        }
      },
      {
        test: /\.css$/,
        use: [
          isProduction ? MiniCssExtractPlugin.loader : 'style-loader',
          'css-loader',
          'postcss-loader'
        ]
      },
      {
        test: /\.(png|jpg|jpeg|gif|svg)$/,
        type: 'asset',
        parser: {
          dataUrlCondition: {
            maxSize: 8 * 1024           // 8KB
          }
        },
        generator: {
          filename: 'images/[name].[hash:8][ext]'
        }
      }
    ]
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html',
      minify: isProduction
    }),
    new MiniCssExtractPlugin({
      filename: isProduction ? '[name].[contenthash:8].css' : '[name].css'
    }),
    new CleanWebpackPlugin(),
    new webpack.DefinePlugin({
      'process.env.NODE_ENV': JSON.stringify(process.env.NODE_ENV)
    })
  ],
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendor',
          chunks: 'all'
        }
      }
    },
    minimize: isProduction,
    minimizer: [
      new TerserPlugin(),
      new CssMinimizerPlugin()
    ]
  },
  devServer: {
    contentBase: './dist',
    port: 8080,
    hot: true,
    open: true
  },
  resolve: {
    extensions: ['.js', '.jsx', '.json'],
    alias: {
      '@': path.resolve(__dirname, 'src')
    }
  }
}

配置说明：

  • 根据环境变量区分开发和生产配置
  • 开发环境：快速构建，不压缩，使用 style-loader
  • 生产环境：压缩代码，提取 CSS，使用 contenthash
  • 支持热更新、源码映射、路径别名等功能

Vite ​

基本配置 ​

javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  // 插件配置
  plugins: [
    react()  // React 支持
  ],

  // 开发服务器配置
  server: {
    port: 3000,                    // 端口号
    open: true,                     // 自动打开浏览器
    host: '0.0.0.0',             // 监听所有地址
    cors: true,                     // 允许跨域
    // 代理配置：解决开发环境跨域问题
    proxy: {
      '/api': {
        target: 'http://localhost:8080',  // 目标服务器
        changeOrigin: true,                 // 改变请求源
        rewrite: path => path.replace(/^\/api/, '') // 重写路径
      }
    }
  },

  // 构建配置
  build: {
    outDir: 'dist',                 // 输出目录
    assetsDir: 'assets',             // 静态资源目录
    sourcemap: false,                // 是否生成 sourcemap
    minify: 'terser',               // 压缩工具：terser 或 esbuild
    // 代码分割配置
    rollupOptions: {
      output: {
        // 手动代码分割
        manualChunks: {
          vendor: ['react', 'react-dom', 'react-router-dom'],  // 第三方库
          utils: ['lodash', 'axios']                          // 工具库
        },
        // 文件名格式
        chunkFileNames: 'js/[name]-[hash].js',
        entryFileNames: 'js/[name]-[hash].js',
        assetFileNames: '[ext]/[name]-[hash].[ext]'
      }
    },
    // 压缩配置
    terserOptions: {
      compress: {
        drop_console: true,          // 移除 console
        drop_debugger: true          // 移除 debugger
      }
    },
    // chunk 大小警告限制
    chunkSizeWarningLimit: 1000
  },

  // 路径别名配置
  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src')  // @ 指向 src 目录
    }
  },

  // CSS 配置
  css: {
    modules: {
      localsConvention: 'camelCase'   // CSS Modules 命名规范
    },
    preprocessorOptions: {
      scss: {
        additionalData: '@import "@/styles/variables.scss";'  // 全局变量
      }
    }
  },

  // 环境变量配置
  define: {
    'process.env.NODE_ENV': JSON.stringify(process.env.NODE_ENV)
  }
})

插件详解 ​

@vitejs/plugin-react ​

完整配置示例 ​

javascript
import { defineConfig, loadEnv } from 'vite'
import react from '@vitejs/plugin-react'
import viteCompression from 'vite-plugin-compression'
import path from 'path'

// 加载环境变量
const env = loadEnv(process.env.NODE_ENV, process.cwd())

export default defineConfig({
  plugins: [
    react({
      jsxImportSource: 'react',
      babel: {
        plugins: ['@babel/plugin-transform-runtime']
      }
    }),
    // 生产环境启用压缩
    env.VITE_NODE_ENV === 'production' && viteCompression({
      algorithm: 'gzip',
      ext: '.gz',
      threshold: 10240
    })
  ],

  server: {
    port: 3000,
    open: true,
    proxy: {
      '/api': {
        target: env.VITE_API_BASE_URL || 'http://localhost:8080',
        changeOrigin: true,
        rewrite: path => path.replace(/^\/api/, '')
      }
    }
  },

  build: {
    outDir: 'dist',
    sourcemap: env.VITE_NODE_ENV === 'development',
    minify: env.VITE_NODE_ENV === 'production' ? 'terser' : false,
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom'],
          router: ['react-router-dom']
        }
      }
    }
  },

  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src'),
      '@components': path.resolve(__dirname, 'src/components'),
      '@utils': path.resolve(__dirname, 'src/utils')
    }
  }
})

配置说明：

  • 根据环境变量动态配置
  • 开发环境：启用 sourcemap，不压缩
  • 生产环境：压缩代码，生成 gzip 文件
  • 支持多个路径别名
  • 自动代理 API 请求

基本配置 ​

javascript
import resolve from '@rollup/plugin-node-resolve'
import commonjs from '@rollup/plugin-commonjs'
import babel from '@rollup/plugin-babel'
import terser from '@rollup/plugin-terser'
import typescript from '@rollup/plugin-typescript'

export default {
  // 入口配置：指定打包的入口文件
  input: 'src/index.js',

  // 输出配置：可以配置多个输出格式
  output: [
    {
      file: 'dist/bundle.js',          // 输出文件路径
      format: 'esm',                  // 输出格式：esm、cjs、umd、iife
      sourcemap: true,                 // 生成 sourcemap
      name: 'MyLibrary',               // UMD/IIFE 格式的全局变量名
      exports: 'named',                // 导出方式：auto、default、named、none
      globals: {
        react: 'React',                // 外部依赖的全局变量名
        'react-dom': 'ReactDOM'
      }
    },
    {
      file: 'dist/bundle.cjs',
      format: 'cjs'                   // 同时输出 CommonJS 格式
    }
  ],

  // 插件配置
  plugins: [
    // 解析 node_modules 中的模块
    resolve({
      browser: true,                   // 浏览器环境
      preferBuiltins: false             // 优先使用内置模块
    }),
    // 转换 CommonJS 模块为 ES6
    commonjs(),
    // 使用 Babel 转换代码
    babel({
      babelHelpers: 'bundled',          // Babel helpers 打包进 bundle
      exclude: 'node_modules/**',      // 排除 node_modules
      extensions: ['.js', '.jsx', '.ts', '.tsx']
    }),
    // TypeScript 支持
    typescript({
      tsconfig: './tsconfig.json',
      declaration: true                // 生成 .d.ts 类型声明文件
    }),
    // 代码压缩
    terser({
      compress: {
        drop_console: true,            // 移除 console
        drop_debugger: true            // 移除 debugger
      }
    })
  ],

  // 外部依赖：不打包到 bundle 中
  external: ['react', 'react-dom', 'lodash'],

  // 监听模式：开发时自动重新构建
  watch: {
    include: 'src/**',
    exclude: 'node_modules/**'
  }
}

完整配置示例 ​

javascript
import resolve from '@rollup/plugin-node-resolve'
import commonjs from '@rollup/plugin-commonjs'
import babel from '@rollup/plugin-babel'
import terser from '@rollup/plugin-terser'
import typescript from '@rollup/plugin-typescript'
import peerDepsExternal from 'rollup-plugin-peer-deps-external'

const isProduction = process.env.NODE_ENV === 'production'

export default {
  input: 'src/index.ts',
  output: [
    {
      file: 'dist/index.js',
      format: 'esm',
      sourcemap: !isProduction
    },
    {
      file: 'dist/index.cjs',
      format: 'cjs',
      sourcemap: !isProduction
    }
  ],
  plugins: [
    peerDepsExternal(),              // 不打包 peer dependencies
    resolve({
      browser: true
    }),
    commonjs(),
    typescript({
      tsconfig: './tsconfig.json',
      declaration: true,
      declarationDir: 'dist'
    }),
    babel({
      babelHelpers: 'bundled',
      exclude: 'node_modules/**'
    }),
    isProduction && terser()          // 生产环境才压缩
  ].filter(Boolean),
  external: ['react', 'react-dom']
}

配置说明：

  • 适合开发库和组件库
  • 输出多种格式（ESM、CJS）
  • 生成 TypeScript 类型声明
  • 不打包 peer dependencies

面试题 ​

1. Webpack 和 Vite 的区别是什么？ ​

答案：

主要区别：

  1. 构建速度:

    • Webpack: 构建速度较慢
    • Vite: 构建速度快

  2. 开发体验:

    • Webpack: 开发体验一般
    • Vite: 开发体验好

  3. 配置复杂度:

    • Webpack: 配置复杂
    • Vite: 配置简单

  4. 使用场景:

    • Webpack: 复杂项目
    • Vite: 简单项目

2. Webpack 的 Loader 和 Plugin 的区别是什么？ ​

答案：

主要区别：

  1. 作用:

    • Loader: 转换模块的源代码
    • Plugin: 扩展 Webpack 功能

  2. 使用方式:

    • Loader: 在 module.rules 中配置
    • Plugin: 在 plugins 中配置

  3. 执行时机:

    • Loader: 在模块加载时执行
    • Plugin: 在构建过程中执行

3. 如何优化 Webpack 构建速度？ ​

答案：

优化 Webpack 构建速度的方法：

  1. 使用缓存:

    javascript
    module.exports = {
 cache: {
   type: 'filesystem'
 }
}

  2. 使用多线程:

    javascript
    module.exports = {
 module: {
   rules: [
     {
       test: /\.js$/,
       use: 'thread-loader'
     }
   ]
 }
}

  3. 使用 DLL:

    javascript
    module.exports = {
 plugins: [
   new webpack.DllPlugin({
     name: 'vendor',
     path: path.join(__dirname, 'vendor-manifest.json')
   })
 ]
}

4. 如何优化 Webpack 打包体积？ ​

答案：

优化 Webpack 打包体积的方法：

  1. 代码分割:

    javascript
    module.exports = {
 optimization: {
   splitChunks: {
     chunks: 'all'
   }
 }
}

  2. Tree Shaking:

    javascript
    module.exports = {
 optimization: {
   usedExports: true,
   sideEffects: false
 }
}

  3. 压缩代码:

    javascript
    module.exports = {
 optimization: {
   minimize: true,
   minimizer: [
     new TerserPlugin(),
     new CssMinimizerPlugin()
   ]
 }
}
