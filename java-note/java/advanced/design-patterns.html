Java 设计模式详解 ​

📚 概述 ​

设计模式是软件设计中反复出现问题的解决方案，它们代表了最佳实践，是众多软件开发人员经过相当长的一段时间的试验和错误总结出来的。设计模式不是代码，而是解决问题的通用模板，可以在多种不同情况下使用。在 Java 开发中，设计模式帮助我们编写更加灵活、可重用、可维护的代码。GoF（Gang of Four）在《设计模式：可复用面向对象软件的基础》一书中总结了 23 种经典设计模式，这些模式分为创建型、结构型和行为型三大类。理解和掌握设计模式对于提高代码质量、增强系统架构能力具有重要意义。

🎯 学习目标 ​

  • 深入理解设计模式的基本概念和设计原则
  • 掌握 GoF 23 种经典设计模式的结构和实现
  • 学会在实际项目中选择和应用合适的设计模式
  • 理解设计模式在 Java 框架中的应用
  • 培养面向对象设计的思维和能力
  • 掌握设计模式的最佳实践和使用场景
  • 了解设计模式的演进和现代应用
  • 能够设计灵活、可扩展的软件架构

        System.out.println("   - 继承关系中的行为一致性");
        System.out.println("   - 保证多态的正确性");
        
        System.out.println("4. 接口隔离原则 (ISP):");
        System.out.println("   - 客户端不应该依赖它不需要的接口");
        System.out.println("   - 接口应该小而专一");
        System.out.println("   - 避免胖接口");
        
        System.out.println("5. 依赖倒置原则 (DIP):");
        System.out.println("   - 高层模块不应该依赖低层模块");
        System.out.println("   - 两者都应该依赖抽象");
        System.out.println("   - 抽象不应该依赖细节");

1. 单例模式 (Singleton Pattern) ​

// 单例模式详细演示
public class SingletonPatternDemo {

    public static void main(String[] args) {
        demonstrateSingletonTypes();
        demonstrateThreadSafety();
        demonstrateSingletonUsage();
        demonstrateSingletonBreaking();
    }

    // 单例模式类型演示
    public static void demonstrateSingletonTypes() {
        System.out.println("=== 单例模式类型演示 ===");

        /*
         * 单例模式的几种实现方式：
         * 1. 饿汉式 (Eager Initialization)
         * 2. 懒汉式 (Lazy Initialization)
         * 3. 双重检查锁定 (Double-Checked Locking)
         * 4. 静态内部类 (Static Inner Class)
         * 5. 枚举单例 (Enum Singleton)
         */

        System.out.println("单例模式实现方式:");

        // 饿汉式单例
        EagerSingleton eager1 = EagerSingleton.getInstance();
        EagerSingleton eager2 = EagerSingleton.getInstance();
        System.out.println("饿汉式单例相同实例: " + (eager1 == eager2));

        // 懒汉式单例
        LazySingleton lazy1 = LazySingleton.getInstance();
        LazySingleton lazy2 = LazySingleton.getInstance();
        System.out.println("懒汉式单例相同实例: " + (lazy1 == lazy2));

        // 双重检查锁定单例
        DoubleCheckedSingleton dc1 = DoubleCheckedSingleton.getInstance();
        DoubleCheckedSingleton dc2 = DoubleCheckedSingleton.getInstance();
        System.out.println("双重检查锁定单例相同实例: " + (dc1 == dc2));

        // 静态内部类单例
        StaticInnerClassSingleton sic1 = StaticInnerClassSingleton.getInstance();
        StaticInnerClassSingleton sic2 = StaticInnerClassSingleton.getInstance();
        System.out.println("静态内部类单例相同实例: " + (sic1 == sic2));

        // 枚举单例
        EnumSingleton enum1 = EnumSingleton.INSTANCE;

// 具体排序策略
class BubbleSortStrategy implements SortStrategy {
    @Override
    public void sort(int[] data) {
        System.out.println("使用冒泡排序");
        // 简化的冒泡排序实现
        for (int i = 0; i < data.length - 1; i++) {
            for (int j = 0; j < data.length - 1 - i; j++) {
                if (data[j] > data[j + 1]) {
                    int temp = data[j];
                    data[j] = data[j + 1];
                    data[j + 1] = temp;
                }
            }
        }
        System.out.println("排序结果: " + java.util.Arrays.toString(data));
    }
}

class QuickSortStrategy implements SortStrategy {
    @Override
    public void sort(int[] data) {
        System.out.println("使用快速排序");
        quickSort(data, 0, data.length - 1);
        System.out.println("排序结果: " + java.util.Arrays.toString(data));
    }

    private void quickSort(int[] arr, int low, int high) {
        if (low < high) {
            int pi = partition(arr, low, high);
            quickSort(arr, low, pi - 1);
            quickSort(arr, pi + 1, high);
        }
    }

    private int partition(int[] arr, int low, int high) {
        int pivot = arr[high];
        int i = (low - 1);
        for (int j = low; j < high; j++) {
            if (arr[j] < pivot) {
                i++;
                int temp = arr[i];
                arr[i] = arr[j];
                arr[j] = temp;
            }
        }
        int temp = arr[i + 1];
        arr[i + 1] = arr[high];
        arr[high] = temp;
        return i + 1;
    }
}

class MergeSortStrategy implements SortStrategy {
    @Override
    public void sort(int[] data) {
        System.out.println("使用归并排序");
        mergeSort(data, 0, data.length - 1);
        System.out.println("排序结果: " + java.util.Arrays.toString(data));
    }

    private void mergeSort(int[] arr, int l, int r) {
        if (l < r) {
            int m = (l + r) / 2;
            mergeSort(arr, l, m);
            mergeSort(arr, m + 1, r);
            merge(arr, l, m, r);
        }
    }

    private void merge(int[] arr, int l, int m, int r) {
        int n1 = m - l + 1;
        int n2 = r - m;
        int[] L = new int[n1];
        int[] R = new int[n2];

        System.arraycopy(arr, l, L, 0, n1);
        System.arraycopy(arr, m + 1, R, 0, n2);

        int i = 0, j = 0, k = l;
        while (i < n1 && j < n2) {
            if (L[i] <= R[j]) {
                arr[k] = L[i];
                i++;
            } else {
                arr[k] = R[j];
                j++;
            }
            k++;
        }

        while (i < n1) {
            arr[k] = L[i];
            i++;
            k++;
        }

        while (j < n2) {
            arr[k] = R[j];
            j++;
            k++;
        }
    }
}

💡 最佳实践总结 ​

选择原则 ​

  1. 根据问题选择模式：不要为了使用模式而使用模式
  2. 理解模式本质：掌握模式解决的核心问题
  3. 考虑维护成本：权衡模式带来的复杂性
  4. 遵循设计原则：模式应该符合基本设计原则

实施建议 ​

  1. 渐进式重构：逐步引入设计模式改善代码
  2. 团队共识：确保团队成员理解所使用的模式
  3. 文档记录：记录模式的使用意图和实现细节
  4. 持续学习：关注模式的演进和新的应用场景

常见误区 ​

  1. 过度设计：不要在简单问题上使用复杂模式
  2. 模式崇拜：不要认为使用模式就是好设计
  3. 生搬硬套：要根据具体情况调整模式实现
  4. 忽略性能：考虑模式对性能的影响

学习路径 ​

  1. 理解问题：先理解模式要解决的问题
  2. 掌握结构：学习模式的类图和交互
  3. 实践应用：在实际项目中应用模式
  4. 总结反思：总结使用经验和教训

  3. 创建型模式关注对象创建，结构型模式关注对象组合，行为型模式关注对象交互
