Java 集合框架详解 ​

📚 概述 ​

Java 集合框架（Java Collections Framework）是 Java 平台提供的一套用于存储和操作对象集合的统一架构。它提供了丰富的数据结构和算法实现，包括列表、集合、映射等，是 Java 编程中最重要和最常用的特性之一。集合框架与泛型紧密结合，提供了类型安全的数据操作。

🎯 学习目标 ​

  • 深入理解 Java 集合框架的整体架构
  • 掌握 List、Set、Map 等核心接口和实现类
  • 熟练使用 ArrayList、LinkedList、HashMap 等常用集合
  • 理解不同集合的性能特点和使用场景
  • 掌握集合的遍历、排序和搜索操作
  • 学会选择合适的集合类型解决实际问题
  • 了解集合框架的设计模式和最佳实践
  • 掌握并发集合和线程安全问题

        java.util.Iterator<String> iterator = list.iterator();
        while (iterator.hasNext()) {
            String element = iterator.next();
            System.out.print(element + " ");
            if ("C".equals(element)) {
                iterator.remove();  // 安全删除
            }
        }
        System.out.println("\n删除 C 后: " + list);
        
        // 列表迭代器
        System.out.println("列表迭代器:");
        java.util.ListIterator<String> listIterator = list.listIterator();
        while (listIterator.hasNext()) {
            String element = listIterator.next();
            if ("B".equals(element)) {
                listIterator.set("B_Modified");  // 修改元素
                listIterator.add("B_Added");     // 添加元素
            }
        }
        System.out.println("修改后: " + list);
        
        // 反向遍历
        System.out.println("反向遍历:");
        listIterator = list.listIterator(list.size());
        while (listIterator.hasPrevious()) {
            System.out.print(listIterator.previous() + " ");
        }
        System.out.println();
        
        System.out.println();
    }

// 撤销重做管理器
class UndoRedoManager<T> {
    private java.util.LinkedList<T> undoStack = new java.util.LinkedList<>();
    private java.util.LinkedList<T> redoStack = new java.util.LinkedList<>();
    private T currentState;

    public void execute(T newState) {
        if (currentState != null) {
            undoStack.push(currentState);
        }
        currentState = newState;
        redoStack.clear();  // 执行新操作后清空重做栈
    }

    public boolean undo() {
        if (!undoStack.isEmpty()) {
            redoStack.push(currentState);
            currentState = undoStack.pop();
            return true;
        }
        return false;
    }

    public boolean redo() {
        if (!redoStack.isEmpty()) {
            undoStack.push(currentState);
            currentState = redoStack.pop();
            return true;
        }
        return false;
    }

    public T getCurrentState() {
        return currentState;
    }
}

class LRUCache<K, V> extends java.util.LinkedHashMap<K, V> {
    private final int capacity;

    public LRUCache(int capacity) {
        super(16, 0.75f, true);  // 访问顺序

    protected boolean removeEldestEntry(java.util.Map.Entry<K, V> eldest) {

// 购物车实现
class ShoppingCart {
    private java.util.Map<String, Integer> items = new java.util.HashMap<>();
    private java.util.Map<String, Double> prices = new java.util.HashMap<>();

    public ShoppingCart() {
        // 初始化商品价格
        prices.put("苹果", 5.0);
        prices.put("香蕉", 3.0);
        prices.put("橙子", 4.0);
    }

    public void addItem(String item, int quantity) {
        items.merge(item, quantity, Integer::sum);
    }

    public java.util.Map<String, Integer> getItems() {
        return new java.util.HashMap<>(items);
    }

    public double getTotalPrice() {
        return items.entrySet().stream()
                   .mapToDouble(entry -> prices.getOrDefault(entry.getKey(), 0.0) * entry.getValue())
                   .sum();
    }
}

// 用户权限管理
class UserPermissionManager {
    private java.util.Map<String, java.util.Set<String>> userPermissions = new java.util.HashMap<>();

    public void addPermission(String user, String permission) {
        userPermissions.computeIfAbsent(user, k -> new java.util.HashSet<>()).add(permission);
    }

    public java.util.Set<String> getUserPermissions(String user) {
        return userPermissions.getOrDefault(user, java.util.Collections.emptySet());
    }

    public boolean hasPermission(String user, String permission) {
        return userPermissions.getOrDefault(user, java.util.Collections.emptySet()).contains(permission);
    }
}

// 最近访问记录跟踪器
class RecentAccessTracker {
    private java.util.LinkedHashSet<String> recentAccess;
    private final int maxSize;

    public RecentAccessTracker(int maxSize) {
        this.maxSize = maxSize;
        this.recentAccess = new java.util.LinkedHashSet<>();
    }

    public void access(String item) {
        if (recentAccess.contains(item)) {
            recentAccess.remove(item);  // 移除旧位置
        } else if (recentAccess.size() >= maxSize) {
            // 移除最老的项目
            String oldest = recentAccess.iterator().next();
            recentAccess.remove(oldest);
        }
        recentAccess.add(item);  // 添加到最新位置
    }

    public java.util.List<String> getRecentAccess() {
        return new java.util.ArrayList<>(recentAccess);
    }
}

💡 最佳实践总结 ​

选择原则 ​

  1. 根据需求选择：明确是否需要有序、唯一性、键值映射
  2. 考虑性能：了解不同实现的时间复杂度
  3. 线程安全：根据并发需求选择合适的实现
  4. 内存使用：考虑内存占用和扩容机制

实践建议 ​

  1. 优先使用接口类型：声明时使用 List、Set、Map 接口
  2. 正确实现 hashCode 和 equals：确保哈希集合的正确性
  3. 合理设置初始容量：避免频繁扩容
  4. 使用泛型：提供类型安全和更好的性能

性能优化 ​

  1. 选择合适的集合类型：根据操作特点选择最优实现
  2. 避免装箱拆箱：使用专门的基本类型集合
  3. 批量操作：使用 addAll、removeAll 等批量方法
  4. 预估大小：设置合理的初始容量

常见错误避免 ​

  1. 不要在遍历时修改集合：使用迭代器的 remove 方法
  2. 注意 null 值处理：某些集合不允许 null 值
  3. 理解浅拷贝和深拷贝：正确处理集合的复制
  4. 避免内存泄漏：及时清理不需要的引用
