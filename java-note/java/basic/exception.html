Java 异常处理机制详解 ​

📚 概述 ​

异常处理（Exception Handling）是 Java 编程中处理程序运行时错误的重要机制。通过异常处理，程序可以在遇到错误时优雅地处理问题，而不是直接崩溃。Java 提供了完整的异常体系和处理机制，包括异常的分类、捕获、抛出和自定义异常等功能。

🎯 学习目标 ​

  • 深入理解 Java 异常体系结构和分类
  • 掌握 try-catch-finally 语句的使用
  • 熟练使用 throws 和 throw 关键字
  • 理解受检异常和非受检异常的区别
  • 掌握 try-with-resources 自动资源管理
  • 学会设计和使用自定义异常
  • 了解异常处理的最佳实践和性能考虑
  • 掌握异常处理在实际项目中的应用

// Java 异常体系演示
public class ExceptionHierarchyDemo {

    public static void main(String[] args) {
        demonstrateExceptionHierarchy();
    }

    public static void demonstrateExceptionHierarchy() {
        System.out.println("=== Java 异常体系结构 ===");

        /*
         * Throwable (所有异常和错误的超类)
         * ├── Error (严重错误，通常不应该被捕获)
         * │   ├── OutOfMemoryError
         * │   ├── StackOverflowError
         * │   ├── VirtualMachineError
         * │   └── ...
         * └── Exception (异常，可以被捕获和处理)
         *     ├── RuntimeException (非受检异常)
         *     │   ├── NullPointerException
         *     │   ├── IllegalArgumentException
         *     │   ├── IndexOutOfBoundsException
         *     │   ├── ArithmeticException
         *     │   └── ...
         *     └── 其他 Exception (受检异常)
         *         ├── IOException
         *         ├── SQLException
         *         ├── ClassNotFoundException
         *         └── ...
         */

        // 演示不同类型的异常
        demonstrateRuntimeExceptions();
        demonstrateCheckedExceptions();
        demonstrateErrors();
    }

    // 运行时异常演示
    private static void demonstrateRuntimeExceptions() {
        System.out.println("\n=== 运行时异常（非受检异常）===");

        try {
            // NullPointerException
            String str = null;
            int length = str.length();
        } catch (NullPointerException e) {
            System.out.println("捕获 NullPointerException: " + e.getMessage());
        }

        try {
            // ArithmeticException
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            System.out.println("捕获 ArithmeticException: " + e.getMessage());
        }

        try {
            // ArrayIndexOutOfBoundsException
            int[] array = {1, 2, 3};
            int value = array[5];
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("捕获 ArrayIndexOutOfBoundsException: " + e.getMessage());
        }

        try {
            // IllegalArgumentException
            Thread.sleep(-1000);
        } catch (IllegalArgumentException e) {
            System.out.println("捕获 IllegalArgumentException: " + e.getMessage());
        } catch (InterruptedException e) {
            System.out.println("捕获 InterruptedException: " + e.getMessage());
        }
    }

    // 受检异常演示
    private static void demonstrateCheckedExceptions() {
        System.out.println("\n=== 受检异常 ===");

        try {
            // IOException (文件操作)
            java.io.FileReader reader = new java.io.FileReader("nonexistent.txt");
        } catch (java.io.FileNotFoundException e) {
            System.out.println("捕获 FileNotFoundException: " + e.getMessage());
        }

        try {
            // ClassNotFoundException
            Class.forName("com.nonexistent.Class");
        } catch (ClassNotFoundException e) {
            System.out.println("捕获 ClassNotFoundException: " + e.getMessage());
        }

        try {
            // InterruptedException
            Thread.sleep(100);
        } catch (InterruptedException e) {
            System.out.println("捕获 InterruptedException: " + e.getMessage());
        }
    }

    // 错误演示（通常不应该捕获）
    private static void demonstrateErrors() {
        System.out.println("\n=== 错误（Error）===");

        try {
            // 模拟 StackOverflowError
            recursiveMethod();
        } catch (StackOverflowError e) {
            System.out.println("捕获 StackOverflowError（通常不建议捕获）");
        }
    }

    private static void recursiveMethod() {
        recursiveMethod(); // 无限递归导致栈溢出
    }
}

2. 受检异常 vs 非受检异常 ​

java
// 受检异常和非受检异常的对比
public class CheckedVsUncheckedDemo {

    // 受检异常：编译时必须处理
    public void checkedExceptionExample() {
        // 必须使用 try-catch 或在方法签名中声明 throws
        try {
            Thread.sleep(1000);  // InterruptedException 是受检异常
            java.io.FileReader reader = new java.io.FileReader("file.txt");  // IOException 是受检异常
        } catch (InterruptedException e) {
            System.out.println("线程被中断: " + e.getMessage());
        } catch (java.io.IOException e) {
            System.out.println("文件操作异常: " + e.getMessage());
        }
    }

    // 或者在方法签名中声明
    public void checkedExceptionWithThrows() throws InterruptedException, java.io.IOException {
        Thread.sleep(1000);
        java.io.FileReader reader = new java.io.FileReader("file.txt");
    }

    // 非受检异常：编译时不强制处理
    public void uncheckedExceptionExample() {
        // 可以选择处理，也可以不处理
        String str = null;

        // 不处理：程序会在运行时抛出异常并终止
        // int length = str.length();  // NullPointerException

        // 选择处理
        try {
            int length = str.length();
        } catch (NullPointerException e) {
            System.out.println("处理空指针异常: " + e.getMessage());
        }
    }

    // 演示异常的传播
    public void exceptionPropagationDemo() {
        System.out.println("=== 异常传播演示 ===");

        try {
            methodA();
        } catch (Exception e) {
            System.out.println("在 main 方法中捕获异常: " + e.getMessage());
            e.printStackTrace();
        }
    }

    private void methodA() throws Exception {
        System.out.println("进入 methodA");
        methodB();
        System.out.println("离开 methodA"); // 这行不会执行
    }

    private void methodB() throws Exception {
        System.out.println("进入 methodB");
        methodC();
        System.out.println("离开 methodB"); // 这行不会执行
    }

    private void methodC() throws Exception {
        System.out.println("进入 methodC");
        throw new Exception("在 methodC 中抛出异常");
    }

    public static void main(String[] args) {
        CheckedVsUncheckedDemo demo = new CheckedVsUncheckedDemo();
        demo.checkedExceptionExample();
        demo.uncheckedExceptionExample();
        demo.exceptionPropagationDemo();
    }
}

🔧 try-catch-finally 详解 ​

1. 基本语法和执行流程 ​

java
// try-catch-finally 详细演示
public class TryCatchFinallyDemo {

    public static void main(String[] args) {
        demonstrateBasicTryCatch();
        demonstrateMultipleCatch();
        demonstrateFinallyBlock();
        demonstrateNestedTryCatch();
    }

    // 基本 try-catch 使用
    public static void demonstrateBasicTryCatch() {
        System.out.println("=== 基本 try-catch 演示 ===");

        try {
            System.out.println("执行可能出错的代码");
            int result = 10 / 0;  // 抛出 ArithmeticException
            System.out.println("这行代码不会执行");
        } catch (ArithmeticException e) {
            System.out.println("捕获算术异常: " + e.getMessage());
            System.out.println("异常类型: " + e.getClass().getSimpleName());
        }

        System.out.println("程序继续执行\n");
    }

    // 多重 catch 块
    public static void demonstrateMultipleCatch() {
        System.out.println("=== 多重 catch 演示 ===");

        String[] testCases = {null, "abc", "123"};

        for (String testCase : testCases) {
            try {
                System.out.println("处理: " + testCase);
                int length = testCase.length();  // 可能抛出 NullPointerException
                int number = Integer.parseInt(testCase);  // 可能抛出 NumberFormatException
                int result = 100 / number;  // 可能抛出 ArithmeticException
                System.out.println("结果: " + result);

            } catch (NullPointerException e) {
                System.out.println("  捕获空指针异常");
            } catch (NumberFormatException e) {
                System.out.println("  捕获数字格式异常: " + e.getMessage());
            } catch (ArithmeticException e) {
                System.out.println("  捕获算术异常: " + e.getMessage());
            } catch (Exception e) {
                // 通用异常处理（应该放在最后）
                System.out.println("  捕获其他异常: " + e.getClass().getSimpleName());
            }
        }
        System.out.println();
    }

    // finally 块演示
    public static void demonstrateFinallyBlock() {
        System.out.println("=== finally 块演示 ===");

        // 情况1：正常执行
        try {
            System.out.println("正常执行的代码");
            return; // 即使有 return，finally 也会执行
        } catch (Exception e) {
            System.out.println("异常处理");
        } finally {
            System.out.println("finally 块总是执行（情况1）");
        }

        // 情况2：异常发生
        try {
            System.out.println("即将抛出异常");
            throw new RuntimeException("测试异常");
        } catch (RuntimeException e) {
            System.out.println("捕获异常: " + e.getMessage());
        } finally {
            System.out.println("finally 块总是执行（情况2）");
        }

        System.out.println();
    }

    // 嵌套 try-catch
    public static void demonstrateNestedTryCatch() {
        System.out.println("=== 嵌套 try-catch 演示 ===");

        try {
            System.out.println("外层 try 块");

            try {
                System.out.println("内层 try 块");
                int result = 10 / 0;
            } catch (ArithmeticException e) {
                System.out.println("内层 catch: " + e.getMessage());
                throw new RuntimeException("重新抛出异常", e);
            } finally {
                System.out.println("内层 finally 块");
            }

        } catch (RuntimeException e) {
            System.out.println("外层 catch: " + e.getMessage());
            System.out.println("原因: " + e.getCause().getMessage());
        } finally {
            System.out.println("外层 finally 块");
        }

        System.out.println();
    }
}

// 多重捕获和异常处理顺序
public class MultiCatchDemo {

    public static void main(String[] args) {
        demonstrateMultiCatch();
        demonstrateCatchOrder();
    }

    // Java 7+ 多重捕获语法
    public static void demonstrateMultiCatch() {
        System.out.println("=== 多重捕获语法演示 ===");

        try {
            // 模拟可能抛出多种异常的操作
            processData("invalid");
        } catch (NumberFormatException | IllegalArgumentException e) {
            // 同时捕获多种异常类型
            System.out.println("捕获输入异常: " + e.getClass().getSimpleName() + " - " + e.getMessage());
        } catch (Exception e) {
            System.out.println("捕获其他异常: " + e.getMessage());
        }

        System.out.println();
    }

    // 异常捕获顺序的重要性
    public static void demonstrateCatchOrder() {
        System.out.println("=== 异常捕获顺序演示 ===");

        try {
            throw new IllegalArgumentException("参数错误");
        } catch (IllegalArgumentException e) {
            // 具体异常应该在前面
            System.out.println("捕获 IllegalArgumentException: " + e.getMessage());
        } catch (RuntimeException e) {
            // 更一般的异常在后面
            System.out.println("捕获 RuntimeException: " + e.getMessage());
        } catch (Exception e) {
            // 最一般的异常在最后
            System.out.println("捕获 Exception: " + e.getMessage());
        }

        // 错误示例（编译错误）：
        /*
        try {
            throw new IllegalArgumentException("参数错误");
        } catch (Exception e) {
            // 如果这个在前面，下面的 catch 块将永远不会被执行
            System.out.println("捕获 Exception: " + e.getMessage());
        } catch (IllegalArgumentException e) {
            // 编译错误：Unreachable catch block
            System.out.println("捕获 IllegalArgumentException: " + e.getMessage());
        }
        */

        System.out.println();
    }

    private static void processData(String data) {
        if (data == null) {
            throw new IllegalArgumentException("数据不能为空");
        }

        if (data.equals("invalid")) {
            throw new NumberFormatException("无效的数字格式");
        }

        // 正常处理逻辑
        System.out.println("处理数据: " + data);
    }
}

🔁 try-with-resources（自动资源管理） ​

1. 基本用法 ​

java
// try-with-resources 详细演示
public class TryWithResourcesDemo {

    public static void main(String[] args) {
        demonstrateBasicTryWithResources();
        demonstrateMultipleResources();
        demonstrateSuppressedExceptions();
        demonstrateCustomResource();
    }

    // 基本 try-with-resources 用法
    public static void demonstrateBasicTryWithResources() {
        System.out.println("=== 基本 try-with-resources 演示 ===");

        // 传统方式（容易出错）
        java.io.FileInputStream fis = null;
        try {
            fis = new java.io.FileInputStream("test.txt");
            // 读取文件操作
        } catch (java.io.IOException e) {
            System.out.println("传统方式 - 文件操作异常: " + e.getMessage());
        } finally {
            // 手动关闭资源
            if (fis != null) {
                try {
                    fis.close();
                } catch (java.io.IOException e) {
                    System.out.println("关闭文件时出错: " + e.getMessage());
                }
            }
        }

        // try-with-resources 方式（推荐）
        try (java.io.FileInputStream fis2 = new java.io.FileInputStream("test.txt")) {
            // 读取文件操作
            System.out.println("使用 try-with-resources 读取文件");
        } catch (java.io.IOException e) {
            System.out.println("try-with-resources - 文件操作异常: " + e.getMessage());
        }
        // 资源会自动关闭，无需手动处理

        System.out.println();
    }

    // 多个资源的管理
    public static void demonstrateMultipleResources() {
        System.out.println("=== 多个资源管理演示 ===");

        try (java.io.FileInputStream input = new java.io.FileInputStream("input.txt");
             java.io.FileOutputStream output = new java.io.FileOutputStream("output.txt");
             java.io.BufferedInputStream bufferedInput = new java.io.BufferedInputStream(input);
             java.io.BufferedOutputStream bufferedOutput = new java.io.BufferedOutputStream(output)) {

            // 文件复制操作
            byte[] buffer = new byte[1024];
            int bytesRead;
            while ((bytesRead = bufferedInput.read(buffer)) != -1) {
                bufferedOutput.write(buffer, 0, bytesRead);
            }

            System.out.println("文件复制完成");

        } catch (java.io.IOException e) {
            System.out.println("文件操作异常: " + e.getMessage());
        }
        // 所有资源都会按照声明的相反顺序自动关闭

        System.out.println();
    }

    // 被抑制的异常
    public static void demonstrateSuppressedExceptions() {
        System.out.println("=== 被抑制异常演示 ===");

        try (ProblematicResource resource = new ProblematicResource()) {
            resource.doSomething();
        } catch (Exception e) {
            System.out.println("主异常: " + e.getMessage());

            // 获取被抑制的异常
            Throwable[] suppressed = e.getSuppressed();
            for (Throwable t : suppressed) {
                System.out.println("被抑制的异常: " + t.getMessage());
            }
        }

        System.out.println();
    }

    // 自定义资源类
    public static void demonstrateCustomResource() {
        System.out.println("=== 自定义资源演示 ===");

        try (CustomResource resource = new CustomResource("MyResource")) {
            resource.useResource();
        } catch (Exception e) {
            System.out.println("使用资源时出错: " + e.getMessage());
        }

        System.out.println();
    }
}

// 自定义资源类
class CustomResource implements AutoCloseable {
    private String name;
    private boolean closed = false;

    public CustomResource(String name) {
        this.name = name;
        System.out.println("创建资源: " + name);
    }

    public void useResource() {
        if (closed) {
            throw new IllegalStateException("资源已关闭");
        }
        System.out.println("使用资源: " + name);
    }

    @Override
    public void close() throws Exception {
        if (!closed) {
            System.out.println("关闭资源: " + name);
            closed = true;
        }
    }
}

// 有问题的资源类（用于演示被抑制异常）
class ProblematicResource implements AutoCloseable {

    public void doSomething() throws Exception {
        throw new Exception("在使用资源时发生异常");
    }

    @Override
    public void close() throws Exception {
        throw new Exception("在关闭资源时发生异常");
    }
}

// throws 关键字详细演示
public class ThrowsDemo {

    public static void main(String[] args) {
        ThrowsDemo demo = new ThrowsDemo();

        // 调用声明了异常的方法
        try {
            demo.methodWithCheckedExceptions();
        } catch (java.io.IOException e) {
            System.out.println("捕获 IOException: " + e.getMessage());
        } catch (java.sql.SQLException e) {
            System.out.println("捕获 SQLException: " + e.getMessage());
        }

        // 调用声明了运行时异常的方法（可选处理）
        try {
            demo.methodWithRuntimeExceptions();
        } catch (IllegalArgumentException e) {
            System.out.println("捕获 IllegalArgumentException: " + e.getMessage());
        }

        demo.demonstrateExceptionPropagation();
    }

    // 声明可能抛出的受检异常
    public void methodWithCheckedExceptions() throws java.io.IOException, java.sql.SQLException {
        System.out.println("=== throws 声明受检异常演示 ===");

        // 模拟可能抛出 IOException 的操作
        if (Math.random() > 0.5) {
            throw new java.io.IOException("模拟 IO 异常");
        }

        // 模拟可能抛出 SQLException 的操作
        if (Math.random() > 0.7) {
            throw new java.sql.SQLException("模拟 SQL 异常");
        }

        System.out.println("方法正常执行完成");
    }

    // 声明运行时异常（可选，通常不需要）
    public void methodWithRuntimeExceptions() throws IllegalArgumentException {
        System.out.println("\n=== throws 声明运行时异常演示 ===");

        if (Math.random() > 0.5) {
            throw new IllegalArgumentException("模拟参数异常");
        }

        System.out.println("方法正常执行完成");
    }

    // 异常传播演示
    public void demonstrateExceptionPropagation() {
        System.out.println("\n=== 异常传播演示 ===");

        try {
            level1Method();
        } catch (CustomException e) {
            System.out.println("在顶层捕获异常: " + e.getMessage());
            System.out.println("异常传播路径:");
            e.printStackTrace();
        }
    }

    private void level1Method() throws CustomException {
        System.out.println("进入 level1Method");
        level2Method();
        System.out.println("离开 level1Method"); // 不会执行
    }

    private void level2Method() throws CustomException {
        System.out.println("进入 level2Method");
        level3Method();
        System.out.println("离开 level2Method"); // 不会执行
    }

    private void level3Method() throws CustomException {
        System.out.println("进入 level3Method");
        throw new CustomException("在 level3Method 中抛出异常");
    }
}

// 自定义异常类
class CustomException extends Exception {
    public CustomException(String message) {
        super(message);
    }

    public CustomException(String message, Throwable cause) {
        super(message, cause);
    }
}

                    throw new Exception("操作失败，已重试 " + maxRetries + " 次", lastException);
                }

            try {
                exceptionMethod();
            } catch (Exception e) {
                // 捕获异常
            }
        }
        long exceptionTime = System.nanoTime() - startTime;

        System.out.printf("正常流程耗时: %.2f ms%n", normalTime / 1_000_000.0);
        System.out.printf("异常流程耗时: %.2f ms%n", exceptionTime / 1_000_000.0);
        System.out.printf("异常流程比正常流程慢 %.1f 倍%n", (double) exceptionTime / normalTime);
    }

    // 避免在循环中使用异常进行流程控制
    private void demonstrateExceptionInLoop() {
        System.out.println("\n--- 循环中的异常处理 ---");

        // 错误做法：使用异常控制循环
        long startTime = System.nanoTime();
        badLoopWithException();
        long badTime = System.nanoTime() - startTime;

        // 正确做法：使用正常的条件判断
        startTime = System.nanoTime();
        goodLoopWithCondition();
        long goodTime = System.nanoTime() - startTime;

        System.out.printf("使用异常的循环耗时: %.2f ms%n", badTime / 1_000_000.0);
        System.out.printf("使用条件的循环耗时: %.2f ms%n", goodTime / 1_000_000.0);
        System.out.printf("使用异常比使用条件慢 %.1f 倍%n", (double) badTime / goodTime);
    }

    // 异常 vs 返回值
    private void demonstrateExceptionVsReturnValue() {
        System.out.println("\n--- 异常 vs 返回值 ---");

        int iterations = 10000;

        // 使用异常表示错误
        long startTime = System.nanoTime();
        for (int i = 0; i < iterations; i++) {
            try {
                parseIntWithException("invalid");
            } catch (NumberFormatException e) {
                // 处理异常
            }
        }
        long exceptionTime = System.nanoTime() - startTime;

        // 使用返回值表示错误
        startTime = System.nanoTime();
        for (int i = 0; i < iterations; i++) {
            ParseResult result = parseIntWithResult("invalid");
            if (!result.isSuccess()) {
                // 处理错误
            }
        }
        long returnValueTime = System.nanoTime() - startTime;

        System.out.printf("使用异常耗时: %.2f ms%n", exceptionTime / 1_000_000.0);
        System.out.printf("使用返回值耗时: %.2f ms%n", returnValueTime / 1_000_000.0);
        System.out.printf("使用异常比返回值慢 %.1f 倍%n", (double) exceptionTime / returnValueTime);
    }

    private void normalMethod() {
        // 正常的方法执行
        int result = 1 + 1;
    }

    private void exceptionMethod() throws Exception {
        throw new Exception("测试异常");
    }

    // 错误的循环方式：使用异常控制流程
    private void badLoopWithException() {
        java.util.List<String> list = java.util.Arrays.asList("1", "2", "3", "4", "5");
        int index = 0;
        try {
            while (true) {
                String item = list.get(index++);
                // 处理项目
            }
        } catch (IndexOutOfBoundsException e) {
            // 使用异常来结束循环（错误做法）
        }
    }

    // 正确的循环方式：使用条件判断
    private void goodLoopWithCondition() {
        java.util.List<String> list = java.util.Arrays.asList("1", "2", "3", "4", "5");
        for (int i = 0; i < list.size(); i++) {
            String item = list.get(i);
            // 处理项目
        }
    }

    private int parseIntWithException(String str) throws NumberFormatException {
        return Integer.parseInt(str);
    }

    private ParseResult parseIntWithResult(String str) {
        try {
            int value = Integer.parseInt(str);
            return new ParseResult(true, value, null);
        } catch (NumberFormatException e) {
            return new ParseResult(false, 0, e.getMessage());
        }
    }
}

// 解析结果类
class ParseResult {
    private final boolean success;
    private final int value;
    private final String errorMessage;

    public ParseResult(boolean success, int value, String errorMessage) {
        this.success = success;
        this.value = value;
        this.errorMessage = errorMessage;
    }

    public boolean isSuccess() {
        return success;
    }

    public int getValue() {
        return value;
    }

    public String getErrorMessage() {
        return errorMessage;
    }
}

💡 最佳实践总结 ​

设计原则 ​

  1. 快速失败：尽早发现和报告错误
  2. 具体异常：使用具体的异常类型而非通用异常
  3. 异常链：保留原始异常信息
  4. 资源管理：使用 try-with-resources 管理资源

实践建议 ​

  1. 不要忽略异常：至少要记录日志
  2. 不要捕获过于宽泛的异常：避免捕获 Exception 或 Throwable
  3. 不要使用异常控制流程：异常应该用于异常情况
  4. 合理设计异常层次：根据业务需求设计异常类

💡 重点总结：

  1. 异常处理是 Java 程序健壮性的重要保障，要合理使用
  2. 受检异常强制处理，非受检异常可选处理，要根据场景选择
  3. try-with-resources 是资源管理的最佳实践，避免资源泄露
  4. 异常链保留了完整的错误信息，有助于问题定位
  5. 自定义异常应该提供足够的上下文信息
  6. 异常处理有性能开销，不应该用于正常的流程控制
  7. 遵循最佳实践可以提高代码质量和可维护性
  • throws 声明方法可能抛出的受检异常；throw 在代码中显式抛出一个异常实例。
  • 重新抛出（rethrow）时保留 cause 以便定位根因：
java
try { doIO(); }
catch (IOException e) { throw new UncheckedIOException("wrap", e); }

🎯 多重捕获与顺序 ​

  • 多重捕获：catch (IOException | SQLException e) 避免重复代码。
  • 捕获顺序：从子类到父类；否则父类先匹配将导致子类分支不可达。

🧱 自定义异常 ​

  • 业务异常建议继承 RuntimeException（视团队规范而定），包含错误码与上下文。
java
public class BizException extends RuntimeException {
    private final String code;
    public BizException(String code, String msg){ super(msg); this.code = code; }
    public String getCode(){ return code; }
}

🔒 最佳实践 ​

  • 不吞异常：至少记录日志或包装并上抛；避免空 catch。
  • 分层处理：在边界层（如控制器/任务调度）统一拦截，内部尽量上抛。
  • 不以异常作流程控制；避免在高频路径频繁构造异常对象（开销较大）。
  • 日志规范：包含上下文（参数、用户、traceId）；避免重复打印同一异常导致日志风暴。
