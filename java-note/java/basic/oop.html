Java 面向对象编程详解 ​

📚 概述 ​

面向对象编程（Object-Oriented Programming，OOP）是 Java 语言的核心编程范式。它通过类和对象来组织代码，强调数据封装、代码重用和模块化设计。掌握 OOP 思想是学好 Java 的关键。

🎯 学习目标 ​

  • 深入理解面向对象编程的核心思想和设计原则
  • 掌握封装、继承、多态三大特性的原理和应用
  • 熟练使用类、对象、接口、抽象类等 OOP 概念
  • 理解访问控制和包的作用机制
  • 学会运用 OOP 思想设计和实现程序
  • 了解 OOP 与其他编程范式的区别和优势

🌟 面向对象编程思想 ​

1. 什么是面向对象编程 ​

面向对象编程是一种编程范式，它将现实世界中的事物抽象为程序中的对象，通过对象之间的交互来完成程序功能。

java
// 面向过程的思维：关注"做什么"
public class ProcessOrientedExample {
    public static void main(String[] args) {
        // 计算圆的面积
        double radius = 5.0;
        double area = 3.14159 * radius * radius;
        System.out.println("圆的面积: " + area);

        // 计算矩形的面积
        double length = 4.0;
        double width = 3.0;
        double rectArea = length * width;
        System.out.println("矩形面积: " + rectArea);
    }
}

// 面向对象的思维：关注"谁来做"
abstract class Shape {
    public abstract double calculateArea();
    public abstract void display();
}

class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }

    @Override
    public void display() {
        System.out.println("圆的面积: " + calculateArea());
    }
}

class Rectangle extends Shape {
    private double length;
    private double width;

    public Rectangle(double length, double width) {
        this.length = length;
        this.width = width;
    }

    @Override
    public double calculateArea() {
        return length * width;
    }

    @Override
    public void display() {
        System.out.println("矩形面积: " + calculateArea());
    }
}

public class ObjectOrientedExample {
    public static void main(String[] args) {
        Shape[] shapes = {
            new Circle(5.0),
            new Rectangle(4.0, 3.0)
        };

        for (Shape shape : shapes) {
            shape.display(); // 多态调用
        }
    }
}

🏗️ 面向对象三大特性 ​

1. 封装（Encapsulation） ​

封装是将数据和操作数据的方法绑定在一起，并隐藏内部实现细节的机制。

java
public class Student {
    // 私有属性，外部无法直接访问
    private String name;
    private int age;
    private double gpa;

    // 构造方法
    public Student(String name, int age) {
        this.name = name;
        setAge(age); // 使用setter进行验证
    }

    // 公共方法提供受控访问
    public String getName() {
        return name;
    }

    public void setName(String name) {
        if (name != null && !name.trim().isEmpty()) {
            this.name = name;
        } else {
            throw new IllegalArgumentException("姓名不能为空");
        }
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age >= 0 && age <= 150) {
            this.age = age;
        } else {
            throw new IllegalArgumentException("年龄必须在0-150之间");
        }
    }

    public double getGpa() {
        return gpa;
    }

    public void setGpa(double gpa) {
        if (gpa >= 0.0 && gpa <= 4.0) {
            this.gpa = gpa;
        } else {
            throw new IllegalArgumentException("GPA必须在0.0-4.0之间");
        }
    }

    // 业务方法
    public String getGradeLevel() {
        if (gpa >= 3.5) return "优秀";
        else if (gpa >= 3.0) return "良好";
        else if (gpa >= 2.0) return "及格";
        else return "不及格";
    }

    @Override
    public String toString() {
        return String.format("Student{name='%s', age=%d, gpa=%.2f, level='%s'}",
                           name, age, gpa, getGradeLevel());
    }
}

// 封装的好处演示
public class EncapsulationDemo {
    public static void main(String[] args) {
        Student student = new Student("张三", 20);

        // 通过公共方法安全地设置数据
        student.setGpa(3.8);
        System.out.println(student);

        // 尝试设置无效数据
        try {
            student.setAge(-5); // 会抛出异常
        } catch (IllegalArgumentException e) {
            System.out.println("错误: " + e.getMessage());
        }

        // 无法直接访问私有属性
        // student.name = ""; // 编译错误
    }
}

2. 继承（Inheritance） ​

继承是子类获得父类属性和方法的机制，实现代码重用和建立类层次结构。

java
// 基类（父类）
class Animal {
    protected String name;
    protected int age;

    public Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void eat() {
        System.out.println(name + " 正在吃东西");
    }

    public void sleep() {
        System.out.println(name + " 正在睡觉");
    }

    public void makeSound() {
        System.out.println(name + " 发出声音");
    }

    public void displayInfo() {
        System.out.println("动物信息: " + name + ", 年龄: " + age);
    }
}

// 派生类（子类）
class Dog extends Animal {
    private String breed;

    public Dog(String name, int age, String breed) {
        super(name, age); // 调用父类构造器
        this.breed = breed;
    }

    // 方法重写（Override）
    @Override
    public void makeSound() {
        System.out.println(name + " 汪汪叫");
    }

    // 新增方法
    public void wagTail() {
        System.out.println(name + " 摇尾巴");
    }

    public void fetch() {
        System.out.println(name + " 去捡球");
    }

    @Override
    public void displayInfo() {
        super.displayInfo(); // 调用父类方法
        System.out.println("品种: " + breed);
    }
}

class Cat extends Animal {
    private boolean isIndoor;

    public Cat(String name, int age, boolean isIndoor) {
        super(name, age);
        this.isIndoor = isIndoor;
    }

    @Override
    public void makeSound() {
        System.out.println(name + " 喵喵叫");
    }

    public void climb() {
        System.out.println(name + " 爬树");
    }

    @Override
    public void displayInfo() {
        super.displayInfo();
        System.out.println("室内猫: " + (isIndoor ? "是" : "否"));
    }
}

// 继承演示
public class InheritanceDemo {
    public static void main(String[] args) {
        Dog dog = new Dog("旺财", 3, "金毛");
        Cat cat = new Cat("咪咪", 2, true);

        // 使用继承的方法
        dog.eat();        // 继承自Animal
        dog.makeSound();  // 重写的方法
        dog.wagTail();    // Dog特有的方法
        dog.displayInfo();

        System.out.println();

        cat.sleep();      // 继承自Animal
        cat.makeSound();  // 重写的方法
        cat.climb();      // Cat特有的方法
        cat.displayInfo();
    }
}

3. 多态（Polymorphism） ​

多态是同一个接口可以有多种不同的实现方式，在运行时根据对象的实际类型来决定调用哪个方法。

java
// 多态的基础：继承和方法重写
abstract class Shape {
    protected String color;

    public Shape(String color) {
        this.color = color;
    }

    // 抽象方法，子类必须实现
    public abstract double calculateArea();
    public abstract double calculatePerimeter();

    // 具体方法，子类可以重写
    public void displayInfo() {
        System.out.printf("形状: %s, 颜色: %s, 面积: %.2f, 周长: %.2f%n",
                         getClass().getSimpleName(), color,
                         calculateArea(), calculatePerimeter());
    }
}

class Circle extends Shape {
    private double radius;

    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }

    @Override
    public double calculatePerimeter() {
        return 2 * Math.PI * radius;
    }
}

class Rectangle extends Shape {
    private double width;
    private double height;

    public Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }

    @Override
    public double calculateArea() {
        return width * height;
    }

    @Override
    public double calculatePerimeter() {
        return 2 * (width + height);
    }
}

// 多态演示
public class PolymorphismDemo {
    public static void main(String[] args) {
        // 多态数组：父类引用指向子类对象
        Shape[] shapes = {
            new Circle("红色", 5.0),
            new Rectangle("蓝色", 4.0, 6.0),
            new Circle("黄色", 3.0)
        };

        // 多态调用：运行时绑定
        System.out.println("=== 所有形状信息 ===");
        for (Shape shape : shapes) {
            shape.displayInfo(); // 多态调用，实际调用子类的方法
        }

        // 计算总面积
        double totalArea = 0;
        for (Shape shape : shapes) {
            totalArea += shape.calculateArea(); // 多态调用
        }
        System.out.printf("%n总面积: %.2f%n", totalArea);

        // 类型检查和转换
        System.out.println("\n=== 类型检查演示 ===");
        for (Shape shape : shapes) {
            if (shape instanceof Circle) {
                System.out.println("找到一个圆形");
            } else if (shape instanceof Rectangle) {
                System.out.println("找到一个矩形");
            }
        }

        // 多态的方法参数
        processShape(new Circle("紫色", 2.5));
        processShape(new Rectangle("橙色", 3.0, 4.0));
    }

    // 多态参数：可以接受任何Shape的子类
    public static void processShape(Shape shape) {
        System.out.println("处理形状: " + shape.getClass().getSimpleName());
        shape.displayInfo();
    }
}

🎭 抽象类与接口 ​

1. 接口（Interface） ​

接口定义了类必须实现的方法契约，支持多重继承，实现了完全的抽象。

java
// 基础接口
interface Drawable {
    void draw();
    void erase();
}

interface Movable {
    void move(int x, int y);
    void stop();
}

interface Resizable {
    void resize(double factor);
    double getSize();
}

// 实现多个接口
class GraphicObject implements Drawable, Movable, Resizable {
    private int x, y;
    private double size;
    private String type;

    public GraphicObject(String type, int x, int y, double size) {
        this.type = type;
        this.x = x;
        this.y = y;
        this.size = size;
    }

    @Override
    public void draw() {
        System.out.printf("绘制%s在位置(%d, %d)，大小: %.1f%n", type, x, y, size);
    }

    @Override
    public void erase() {
        System.out.printf("擦除%s%n", type);
    }

    @Override
    public void move(int newX, int newY) {
        System.out.printf("%s从(%d, %d)移动到(%d, %d)%n", type, x, y, newX, newY);
        this.x = newX;
        this.y = newY;
    }

    @Override
    public void stop() {
        System.out.printf("%s停止移动%n", type);
    }

    @Override
    public void resize(double factor) {
        double oldSize = size;
        size *= factor;
        System.out.printf("%s大小从%.1f调整为%.1f%n", type, oldSize, size);
    }

    @Override
    public double getSize() {
        return size;
    }
}

// 接口的默认方法（Java 8+）
interface Vehicle {
    // 抽象方法
    void start();
    void stop();

    // 默认方法
    default void honk() {
        System.out.println("嘟嘟！");
    }

    // 静态方法
    static void checkLicense() {
        System.out.println("检查驾驶证");
    }
}

class Car implements Vehicle {
    private String brand;

    public Car(String brand) {
        this.brand = brand;
    }

    @Override
    public void start() {
        System.out.println(brand + " 汽车启动");
    }

    @Override
    public void stop() {
        System.out.println(brand + " 汽车停止");
    }

    // 可以重写默认方法
    @Override
    public void honk() {
        System.out.println(brand + " 汽车：滴滴！");
    }
}

2. 抽象类 vs 接口 ​

// 抽象类示例
abstract class Employee {
    protected String name;
    protected String id;
    protected double baseSalary;

    public Employee(String name, String id, double baseSalary) {
        this.name = name;
        this.id = id;
        this.baseSalary = baseSalary;
    }

    // 抽象方法
    public abstract double calculateSalary();
    public abstract String getJobTitle();

    // 具体方法
    public void displayInfo() {
        System.out.printf("员工: %s (ID: %s), 职位: %s, 薪资: %.2f%n",
                         name, id, getJobTitle(), calculateSalary());
    }

    public void work() {
        System.out.println(name + " 正在工作");
    }
}

// 接口示例
interface Payable {
    double calculatePay();
    void processPayment();
}

interface Manageable {
    void manage();
    void delegate(String task);
}

// 同时继承抽象类和实现接口
class Manager extends Employee implements Payable, Manageable {
    private double bonus;
    private int teamSize;

    public Manager(String name, String id, double baseSalary, double bonus, int teamSize) {
        super(name, id, baseSalary);
        this.bonus = bonus;
        this.teamSize = teamSize;
    }

    @Override
    public double calculateSalary() {
        return baseSalary + bonus;
    }

    @Override
    public String getJobTitle() {
        return "经理";
    }

    @Override
    public double calculatePay() {
        return calculateSalary();
    }

    @Override
    public void processPayment() {
        System.out.printf("处理%s的薪资支付: %.2f%n", name, calculatePay());
    }

    @Override
    public void manage() {
        System.out.printf("%s正在管理%d人的团队%n", name, teamSize);
    }

    @Override
    public void delegate(String task) {
        System.out.printf("%s委派任务: %s%n", name, task);
    }
}

🏛️ 面向对象设计原则 ​

1. SOLID 原则 ​

java
// S - 单一职责原则（Single Responsibility Principle）
// 一个类应该只有一个引起它变化的原因

// 违反SRP的例子
class BadUser {
    private String name;
    private String email;

    // 用户数据管理
    public void setName(String name) { this.name = name; }
    public void setEmail(String email) { this.email = email; }

    // 数据验证（不同职责）
    public boolean validateEmail() {
        return email.contains("@");
    }

    // 数据持久化（不同职责）
    public void saveToDatabase() {
        System.out.println("保存用户到数据库");
    }

    // 邮件发送（不同职责）
    public void sendEmail(String message) {
        System.out.println("发送邮件: " + message);
    }
}

// 遵循SRP的例子
class User {
    private String name;
    private String email;

    public User(String name, String email) {
        this.name = name;
        this.email = email;
    }

    // 只负责用户数据管理
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }
}

class UserValidator {
    public boolean validateEmail(String email) {
        return email != null && email.contains("@");
    }

    public boolean validateName(String name) {
        return name != null && !name.trim().isEmpty();
    }
}

class UserRepository {
    public void save(User user) {

    }

    public User findByEmail(String email) {
        // 模拟数据库查询
        return new User("示例用户", email);
    }
}

class EmailService {
    public void sendWelcomeEmail(User user) {
        System.out.println("发送欢迎邮件给: " + user.getEmail());
    }
}

// O - 开闭原则（Open-Closed Principle）
// 对扩展开放，对修改关闭

abstract class DiscountCalculator {

2. 组合优于继承 ​

java
// 继承方式（紧耦合）
class Bird {
    public void fly() {
        System.out.println("鸟在飞");
    }
}

class Duck extends Bird {
    public void swim() {
        System.out.println("鸭子在游泳");
    }

    public void quack() {
        System.out.println("鸭子嘎嘎叫");
    }
}

// 问题：如果有不会飞的鸟（如企鹅），继承就不合适了

// 组合方式（松耦合）
interface FlyBehavior {
    void fly();
}

interface SwimBehavior {
    void swim();
}

interface SoundBehavior {
    void makeSound();
}

class CanFly implements FlyBehavior {
    @Override
    public void fly() {
        System.out.println("用翅膀飞行");
    }
}

class CannotFly implements FlyBehavior {
    @Override
    public void fly() {
        System.out.println("不能飞行");
    }
}

class CanSwim implements SwimBehavior {
    @Override
    public void swim() {
        System.out.println("在水中游泳");
    }
}

class QuackSound implements SoundBehavior {
    @Override
    public void makeSound() {
        System.out.println("嘎嘎叫");
    }
}

class Animal {
    protected FlyBehavior flyBehavior;
    protected SwimBehavior swimBehavior;
    protected SoundBehavior soundBehavior;

    public Animal(FlyBehavior flyBehavior, SwimBehavior swimBehavior, SoundBehavior soundBehavior) {
        this.flyBehavior = flyBehavior;
        this.swimBehavior = swimBehavior;
        this.soundBehavior = soundBehavior;
    }

    public void performFly() {
        if (flyBehavior != null) flyBehavior.fly();
    }

    public void performSwim() {
        if (swimBehavior != null) swimBehavior.swim();
    }

    public void performSound() {
        if (soundBehavior != null) soundBehavior.makeSound();
    }

    // 运行时改变行为
    public void setFlyBehavior(FlyBehavior flyBehavior) {

// 抽象基类
abstract class LibraryItem {
    protected String id;
    protected String title;
    protected boolean isAvailable;

    public LibraryItem(String id, String title) {
        this.id = id;
        this.title = title;
        this.isAvailable = true;
    }

    public abstract void displayInfo();
    public abstract double calculateLateFee(int daysLate);

    public void borrow() {
        if (isAvailable) {
            isAvailable = false;
            System.out.println(title + " 已借出");
        } else {
            System.out.println(title + " 不可借");
        }
    }

    public void returnItem() {
        isAvailable = true;
        System.out.println(title + " 已归还");
    }

    // getter方法
    public String getId() { return id; }
    public String getTitle() { return title; }
    public boolean isAvailable() { return isAvailable; }
}

// 具体实现类
class Book extends LibraryItem {

    private String isbn;

    }

    @Override
    public double calculateLateFee(int daysLate) {

// 接口定义
interface Searchable {
    boolean matches(String keyword);
}

interface Reservable {
    void reserve(String userId);
    void cancelReservation(String userId);
}

// 实现接口
class AdvancedBook extends Book implements Searchable, Reservable {
    private String reservedBy;

    public AdvancedBook(String id, String title, String author, String isbn) {
        super(id, title, author, isbn);
    }

    @Override
    public boolean matches(String keyword) {
        return title.toLowerCase().contains(keyword.toLowerCase()) ||
               author.toLowerCase().contains(keyword.toLowerCase());
    }

    @Override
    public void reserve(String userId) {
        if (!isAvailable && reservedBy == null) {
            reservedBy = userId;
            System.out.println(title + " 已被用户 " + userId + " 预约");
        }
    }

    @Override
    public void cancelReservation(String userId) {
        if (userId.equals(reservedBy)) {
            reservedBy = null;
            System.out.println(title + " 的预约已取消");
        }
    }
}

// 管理类
class Library {
    private List<LibraryItem> items;

    public Library() {
        this.items = new ArrayList<>();
    }

    public void addItem(LibraryItem item) {
        items.add(item);
        System.out.println("添加项目: " + item.getTitle());
    }

    public void displayAllItems() {
        System.out.println("\n=== 图书馆藏品 ===");
        for (LibraryItem item : items) {
            item.displayInfo(); // 多态调用
        }
    }

    public List<LibraryItem> searchItems(String keyword) {
        List<LibraryItem> results = new ArrayList<>();
        for (LibraryItem item : items) {
            if (item instanceof Searchable) {
                Searchable searchableItem = (Searchable) item;
                if (searchableItem.matches(keyword)) {
                    results.add(item);
                }
            }
        }
        return results;
    }
}

学习建议 ​

  1. 循序渐进：先掌握基本概念，再学习高级特性
  2. 多写代码：通过实际编程加深理解

  5. 阅读源码：研究优秀开源项目的设计

常见误区 ​

  1. 过度设计：不要为了使用OOP而强行设计复杂的类层次
  2. 滥用继承：优先考虑组合，继承应该表示真正的"is-a"关系
  3. 忽略封装：不要将所有属性都设为public
  4. 接口污染：接口应该小而专一，避免大而全的接口

面向对象四大特性对比 ​

特性定义关键字/机制主要作用
封装隐藏内部实现细节private, protected, public数据安全、接口稳定
继承子类获得父类特性extends, super代码重用、建立层次
多态同一接口多种实现@Override, 动态绑定灵活性、可扩展性
抽象提取共同特征abstract, interface设计规范、代码组织

类关系强度（从弱到强） ​

  1. 依赖（Dependency）：临时使用关系
  2. 关联（Association）：拥有关系
  3. 聚合（Aggregation）：整体-部分（弱拥有）
  4. 组合（Composition）：整体-部分（强拥有）
  5. 继承（Inheritance）：is-a关系
  6. 实现（Implementation）：接口实现关系

💡 重点总结：

  1. 面向对象编程是一种思维方式，不仅仅是语法特性
  2. 封装、继承、多态是相互配合的，不是孤立的概念
  3. 设计时要考虑可维护性、可扩展性和可重用性
  4. 组合通常比继承更灵活，优先考虑组合
  5. 接口是面向对象设计的重要工具，要善于使用
  6. 遵循设计原则，但不要过度设计

🧩 OOP 与过程式的对比 ​

  • 关注点：过程式强调“步骤/函数”，OOP 强调“对象/协作”。
  • 可维护性：OOP 通过抽象和封装降低耦合，利于演进和复用。
  • 典型迁移路径：先识别领域对象与职责 → 确定类与关系 → 设计接口与协作。

🧠 关键名词与易错点 ​

  • 重载 vs 重写：重载在同一类内参数列表不同；重写在子类中方法签名一致、返回值协变、访问权限不更严格。
  • 向上转型/向下转型：向上安全、向下需 instanceof 检查；多态调用分派依赖运行时实际类型。
  • 组合优先于继承：行为复用优先使用组合，避免不必要的层次耦合。

🧪 对象生命周期与内存 ​

  • 创建：new 分配在堆；局部引用在栈；常量池与元空间（类元数据）分离管理。
  • 可达性与回收：通过 GC Root 可达性判断；finalize 已不推荐，使用 AutoCloseable/try-with-resources。
  • 不可变对象：通过私有字段、无暴露可变引用的 getter、无 setter 实现防御性封装。

🔗 类关系（UML 速记） ​

  • 关联（Association）：A 拥有对 B 的引用。
  • 聚合（Aggregation）：整体-部分的弱拥有，部分可独立存在。
  • 组合（Composition）：整体-部分的强拥有，生命周期一致。
  • 依赖（Dependency）：方法级别的使用关系。
  • 继承（Generalization）：is-a。
  • 实现（Realization）：接口实现关系。
