Cesium 数据源系统学习笔记 ​

概述 ​

Cesium 的数据源系统（DataSource API）是一个强大的数据管理框架，用于加载、处理和可视化各种格式的地理空间数据。与 Entity API 相比，DataSource 更适合处理大规模、结构化的地理数据。

核心概念 ​

1. DataSource（数据源） ​

DataSource 是 Cesium 中管理地理数据的抽象基类，它提供了统一的接口来处理不同格式的数据：

javascript
// 数据源的基本属性
const dataSource = new Cesium.GeoJsonDataSource()
dataSource.name = '我的数据源'
dataSource.show = true  // 控制可见性
dataSource.clustering.enabled = true  // 启用聚合

2. DataSourceCollection（数据源集合） ​

viewer.dataSources 是一个 DataSourceCollection，用于管理场景中的所有数据源：

javascript
// 添加数据源
viewer.dataSources.add(dataSource)

// 获取数据源
const dataSource = viewer.dataSources.get(0)

// 删除数据源
viewer.dataSources.remove(dataSource)

// 清空所有数据源
viewer.dataSources.removeAll()

内置数据源类型 ​

1. GeoJsonDataSource ​

用于加载 GeoJSON 格式的数据：

javascript
const dataSource = new Cesium.GeoJsonDataSource()

// 从 URL 加载
await dataSource.load('/data/countries.geojson')

// 从对象加载
const geoJsonObject = {
  type: 'FeatureCollection',
  features: [
    {
      type: 'Feature',
      geometry: {
        type: 'Point',
        coordinates: [116.4, 39.9]
      },
      properties: {
        name: '北京',
        population: 21540000
      }
    }
  ]
}
await dataSource.load(geoJsonObject)

// 添加到场景
viewer.dataSources.add(dataSource)

2. KmlDataSource ​

用于加载 KML/KMZ 格式的数据：

javascript
const dataSource = new Cesium.KmlDataSource()

// 加载 KML 文件
await dataSource.load('/data/sample.kml', {
  camera: viewer.scene.camera,
  canvas: viewer.scene.canvas
})

viewer.dataSources.add(dataSource)

3. CzmlDataSource ​

用于加载 CZML 格式的时间动态数据：

javascript
const dataSource = new Cesium.CzmlDataSource()

// 加载 CZML 文件
await dataSource.load('/data/satellite.czml')

viewer.dataSources.add(dataSource)

4. CustomDataSource ​

用于创建自定义数据源：

javascript
const dataSource = new Cesium.CustomDataSource('myData')

// 手动添加实体
dataSource.entities.add({
  position: Cesium.Cartesian3.fromDegrees(116.4, 39.9),
  point: {
    pixelSize: 10,
    color: Cesium.Color.RED
  }
})

viewer.dataSources.add(dataSource)

GeoJSON 详解 ​

GeoJSON 格式结构 ​

json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "geometry": {
        "type": "Point",
        "coordinates": [116.4074, 39.9042]
      },
      "properties": {
        "name": "北京",
        "population": 21540000,
        "category": "capital"
      }
    }
  ]
}

几何类型支持 ​

  • Point - 点
  • LineString - 线
  • Polygon - 多边形
  • MultiPoint - 多点
  • MultiLineString - 多线
  • MultiPolygon - 多多边形
  • GeometryCollection - 几何集合

KML 详解 ​

KML 格式特点 ​

  • 基于 XML 的标记语言
  • 支持丰富的样式定义
  • 支持时间动画
  • 支持网络链接
  • 支持 3D 模型

xml

数据样式定制 ​

1. 统一样式设置 ​

javascript
// 加载后统一设置样式
const dataSource = await Cesium.GeoJsonDataSource.load(geoJsonUrl)

// 遍历所有实体设置样式
const entities = dataSource.entities.values
for (let i = 0; i < entities.length; i++) {
  const entity = entities[i]
  
  if (entity.point) {
    entity.point.pixelSize = 20
    entity.point.color = Cesium.Color.YELLOW
  }
  
  if (entity.polyline) {
    entity.polyline.width = 5
    entity.polyline.material = Cesium.Color.RED
  }
  
  if (entity.polygon) {
    entity.polygon.material = Cesium.Color.BLUE.withAlpha(0.5)
    entity.polygon.outline = true
    entity.polygon.outlineColor = Cesium.Color.BLUE
  }
}

2. 基于属性的条件样式 ​

javascript
// 根据属性设置不同样式
const dataSource = await Cesium.GeoJsonDataSource.load(geoJsonUrl)

dataSource.entities.values.forEach(entity => {
  const properties = entity.properties
  
  if (entity.point) {
    // 根据人口数量设置点大小
    const population = properties.population
    if (population > 10000000) {
      entity.point.pixelSize = 25
      entity.point.color = Cesium.Color.RED
    } else if (population > 1000000) {
      entity.point.pixelSize = 20
      entity.point.color = Cesium.Color.ORANGE
    } else {
      entity.point.pixelSize = 15
      entity.point.color = Cesium.Color.YELLOW
    }
  }
})

3. 动态样式回调 ​

javascript
// 使用回调函数实现动态样式
dataSource.entities.values.forEach(entity => {
  if (entity.point) {
    entity.point.color = new Cesium.CallbackProperty(() => {
      // 根据时间或其他条件动态改变颜色
      const time = viewer.clock.currentTime
      const seconds = Cesium.JulianDate.secondsDifference(time, viewer.clock.startTime)
      const hue = (seconds % 360) / 360
      return Cesium.Color.fromHsl(hue, 1.0, 0.5)
    }, false)
  }
})

数据交互与查询 ​

1. 点击事件处理 ​

javascript
// 设置点击事件
const handler = new Cesium.ScreenSpaceEventHandler(viewer.scene.canvas)
handler.setInputAction((click) => {
  const pickedObject = viewer.scene.pick(click.position)
  
  if (pickedObject && pickedObject.id) {
    const entity = pickedObject.id
    const properties = entity.properties
    
    // 显示属性信息
    console.log('实体名称:', entity.name)
    console.log('属性:', properties)
    
    // 创建信息窗口
    viewer.selectedEntity = entity
  }
}, Cesium.ScreenSpaceEventType.LEFT_CLICK)

2. 属性查询 ​

javascript
// 根据属性筛选实体
const filterEntitiesByProperty = (dataSource, propertyName, value) => {
  const filteredEntities = []
  
  dataSource.entities.values.forEach(entity => {
    const properties = entity.properties
    if (properties[propertyName] && properties[propertyName].getValue() === value) {
      filteredEntities.push(entity)
    }
  })
  
  return filteredEntities
}

// 使用示例
const capitalCities = filterEntitiesByProperty(dataSource, 'category', 'capital')

3. 空间查询 ​

javascript
// 根据边界框查询实体
const queryEntitiesInBounds = (dataSource, west, south, east, north) => {
  const results = []
  
  dataSource.entities.values.forEach(entity => {
    if (entity.position) {
      const cartographic = Cesium.Cartographic.fromCartesian(entity.position.getValue())
      const lon = Cesium.Math.toDegrees(cartographic.longitude)
      const lat = Cesium.Math.toDegrees(cartographic.latitude)
      
      if (lon >= west && lon <= east && lat >= south && lat <= north) {
        results.push(entity)
      }
    }
  })
  
  return results
}

性能优化 ​

1. 数据分层加载 ​

javascript
// 根据缩放级别加载不同详细程度的数据
const loadDataByZoom = async () => {
  const height = viewer.camera.positionCartographic.height
  
  let dataUrl
  if (height > 10000000) {
    dataUrl = '/data/world-countries.geojson'  // 低详细度
  } else if (height > 1000000) {
    dataUrl = '/data/provinces.geojson'        // 中详细度
  } else {
    dataUrl = '/data/cities.geojson'           // 高详细度
  }
  
  const dataSource = await Cesium.GeoJsonDataSource.load(dataUrl)
  viewer.dataSources.add(dataSource)
}

2. 大文件加载 ​

javascript
// 分块加载大文件
const loadLargeFile = async (url) => {
  try {
    const response = await fetch(url)
    const reader = response.body.getReader()
    let chunks = []
    
    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      chunks.push(value)
    }
    
    const data = new Uint8Array(chunks.reduce((acc, chunk) => acc + chunk.length, 0))
    let offset = 0
    for (const chunk of chunks) {
      data.set(chunk, offset)
      offset += chunk.length
    }
    
    const text = new TextDecoder().decode(data)
    const geoJson = JSON.parse(text)
    
    return await Cesium.GeoJsonDataSource.load(geoJson)
  } catch (error) {
    console.error('加载失败:', error)
  }
}

最佳实践 ​

1. 数据预处理 ​

  • 简化几何体，减少顶点数量
  • 合并小的相邻要素
  • 移除不必要的属性
  • 使用适当的坐标精度

2. 样式优化 ​

  • 使用简单的颜色而不是复杂材质
  • 避免过多的透明效果
  • 合理设置轮廓宽度
  • 使用距离显示条件

3. 交互优化 ​

  • 限制事件处理频率
  • 使用防抖和节流
  • 合理设置拾取范围
  • 缓存查询结果

总结 ​

Cesium 的数据源系统提供了强大的数据管理能力：

  • GeoJSON - 适合 Web 应用的轻量级格式
  • KML - 适合复杂样式和时间动画
  • CZML - 适合时间序列数据
  • CustomDataSource - 适合自定义数据处理

掌握数据源系统是开发实际 GIS 应用的关键，它让我们能够处理真实世界的地理数据，而不仅仅是手动创建的简单实体。
