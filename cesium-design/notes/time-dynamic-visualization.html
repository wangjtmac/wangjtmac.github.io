时间动态可视化 ​

本文档总结 Cesium 中时间动态数据可视化的核心概念和技术要点。

核心概念 ​

1. 时间系统 ​

Cesium 使用 JulianDate 作为时间表示方式，这是一种连续的时间计数系统。

javascript
// 创建 JulianDate
const now = Cesium.JulianDate.now()
const fromDate = Cesium.JulianDate.fromDate(new Date())
const fromIso = Cesium.JulianDate.fromIso8601('2024-01-01T00:00:00Z')

// 时间运算
const later = Cesium.JulianDate.addSeconds(now, 60, new Cesium.JulianDate())
const diff = Cesium.JulianDate.secondsDifference(later, now) // 60

// 转换为 JavaScript Date
const date = Cesium.JulianDate.toDate(now)

2. Clock 时间控制器 ​

Clock 是 Cesium 的时间控制核心，管理动画的播放状态和时间流逝。

javascript
// 配置 Clock
viewer.clock.startTime = startTime.clone()
viewer.clock.stopTime = stopTime.clone()
viewer.clock.currentTime = startTime.clone()
viewer.clock.clockRange = Cesium.ClockRange.LOOP_STOP
viewer.clock.multiplier = 1 // 播放速度倍数
viewer.clock.shouldAnimate = true // 是否播放

// 时钟范围模式
// - UNBOUNDED: 不限制，时间可以超出范围
// - CLAMPED: 到达终点后停止
// - LOOP_STOP: 循环播放

3. 时间区间 ​

TimeInterval 和 TimeIntervalCollection 用于管理时间区间和相关数据。

javascript
// 创建时间区间
const interval = new Cesium.TimeInterval({
  start: startTime,
  stop: stopTime,
  isStartIncluded: true,
  isStopIncluded: false,
  data: { status: 'active' }
})

// 时间区间集合
const intervals = new Cesium.TimeIntervalCollection()
intervals.addInterval(interval)

// 查询指定时间的区间
const current = intervals.findIntervalContainingDate(currentTime)

动态位置属性 ​

1. SampledPositionProperty ​

用于定义实体的运动轨迹，通过采样点和插值算法实现平滑运动。

javascript
// 创建采样位置属性
const position = new Cesium.SampledPositionProperty()

// 添加采样点
position.addSample(
  Cesium.JulianDate.fromIso8601('2024-01-01T00:00:00Z'),
  Cesium.Cartesian3.fromDegrees(116.4, 39.9, 1000)
)

position.addSample(
  Cesium.JulianDate.fromIso8601('2024-01-01T00:01:00Z'),
  Cesium.Cartesian3.fromDegrees(116.5, 40.0, 1500)
)

// 设置插值算法
position.setInterpolationOptions({
  interpolationDegree: 5,
  interpolationAlgorithm: Cesium.LagrangePolynomialApproximation
})

// 应用到实体
entity.position = position

2. CallbackProperty ​

用于动态计算属性值，适合实时数据更新。

javascript
// 创建回调属性
const position = new Cesium.CallbackProperty((time, result) => {
  // 根据时间计算位置
  const data = getRealtimeData(time)
  return Cesium.Cartesian3.fromDegrees(data.lon, data.lat, data.alt, undefined, result)
}, false)

entity.position = position

3. VelocityOrientationProperty ​

使实体朝向运动方向。

javascript
// 自动朝向运动方向
entity.orientation = new Cesium.VelocityOrientationProperty(position)

轨迹路径 ​

PathGraphics 用于显示实体的运动轨迹。

javascript
entity.path = {
  show: true,
  leadTime: 0,        // 前导时间（显示未来轨迹）
  trailTime: 300,     // 尾迹时间（显示历史轨迹）
  width: 3,
  resolution: 1,      // 采样分辨率（秒）
  material: new Cesium.PolylineGlowMaterialProperty({
    glowPower: 0.3,
    color: Cesium.Color.YELLOW
  })
}

插值算法 ​

Cesium 提供多种插值算法用于平滑动画：

1. 线性插值（LinearApproximation） ​

  • 最简单的插值方法
  • 适合匀速运动
  • 性能最好
javascript
position.setInterpolationOptions({
  interpolationDegree: 1,
  interpolationAlgorithm: Cesium.LinearApproximation
})

2. 拉格朗日插值（LagrangePolynomialApproximation） ​

  • 多项式插值
  • 适合平滑曲线运动
  • 可能产生振荡
javascript
position.setInterpolationOptions({
  interpolationDegree: 5,
  interpolationAlgorithm: Cesium.LagrangePolynomialApproximation
})

3. Hermite 插值（HermitePolynomialApproximation） ​

  • 考虑速度和加速度
  • 最平滑的插值
  • 计算成本较高
javascript
position.setInterpolationOptions({
  interpolationDegree: 3,
  interpolationAlgorithm: Cesium.HermitePolynomialApproximation
})

实时数据更新 ​

方法 1: 动态添加采样点 ​

javascript
// 创建采样位置属性
const position = new Cesium.SampledPositionProperty()

// 实时添加新数据点
function addNewDataPoint(time, lon, lat, alt) {
  const pos = Cesium.Cartesian3.fromDegrees(lon, lat, alt)
  position.addSample(time, pos)
}

// 模拟实时数据流
setInterval(() => {
  const currentTime = viewer.clock.currentTime
  const data = getRealtimeData()
  addNewDataPoint(currentTime, data.lon, data.lat, data.alt)
}, 1000)

方法 2: 使用 CallbackProperty ​

javascript
// 创建回调属性
const position = new Cesium.CallbackProperty((time) => {
  const data = getRealtimeData(time)
  return Cesium.Cartesian3.fromDegrees(data.lon, data.lat, data.alt)
}, false)

entity.position = position

时间轴控制 ​

1. Timeline Widget ​

时间轴组件提供可视化的时间控制界面。

javascript
// 缩放到指定时间范围
viewer.timeline.zoomTo(startTime, stopTime)

// 更新时间轴标记
viewer.timeline.updateFromClock()

2. Animation Widget ​

动画控件提供播放控制按钮。

javascript
// 动画控件会自动与 Clock 同步
// 可以通过 Clock 控制播放状态
viewer.clock.shouldAnimate = true  // 播放
viewer.clock.shouldAnimate = false // 暂停

事件监听 ​

Clock 事件 ​

javascript
// 监听时钟变化（每帧触发）
viewer.clock.onTick.addEventListener((clock) => {
  const currentTime = clock.currentTime
  // 处理时间变化
})

// 监听时钟停止
viewer.clock.onStop.addEventListener(() => {
  console.log('动画已停止')
})

性能优化 ​

1. 采样点优化 ​

  • 减少不必要的采样点
  • 使用合适的采样间隔
  • 定期清理过期数据
javascript
// 限制采样点数量
const maxSamples = 1000
if (position._property._times.length > maxSamples) {
  // 移除最早的采样点
  position._property._times.shift()
  position._property._values.shift()
}

2. 插值算法选择 ​

  • 简单运动使用线性插值
  • 复杂曲线使用拉格朗日插值
  • 平滑运动使用 Hermite 插值

3. 更新频率控制 ​

javascript
// 控制更新频率
let lastUpdateTime = 0
const updateInterval = 100 // ms

viewer.clock.onTick.addEventListener((clock) => {
  const now = Date.now()
  if (now - lastUpdateTime < updateInterval) return
  
  lastUpdateTime = now
  // 执行更新操作
})

最佳实践 ​

  1. 合理设置时间范围 - 避免时间范围过大导致性能问题
  2. 选择合适的插值算法 - 根据运动特性选择插值方法
  3. 控制采样点密度 - 平衡精度和性能
  4. 及时清理过期数据 - 避免内存泄漏
  5. 使用时间区间管理 - 组织复杂的时间数据
  6. 监控性能指标 - 使用浏览器开发工具监控帧率

常见问题 ​

1. 轨迹不平滑 ​

  • 增加采样点密度
  • 使用更高阶的插值算法
  • 检查采样点的时间间隔是否均匀

2. 实体朝向错误 ​

  • 使用 VelocityOrientationProperty
  • 检查速度计算是否正确
  • 确保采样点间隔足够小

3. 性能问题 ​

  • 减少采样点数量
  • 降低更新频率
  • 使用更简单的插值算法
  • 限制同时显示的实体数量

4. 时间同步问题 ​

  • 确保所有时间使用 JulianDate
  • 检查时区设置
  • 验证时间范围配置
