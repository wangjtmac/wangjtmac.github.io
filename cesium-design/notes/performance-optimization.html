性能优化与最佳实践 ​

本文档总结 Cesium 应用开发中的性能优化技术和最佳实践。

性能优化原则 ​

1. 测量优先 ​

在优化之前，先测量性能瓶颈：

  • 使用浏览器开发者工具的 Performance 面板
  • 使用 Cesium Inspector 查看渲染统计
  • 监控 FPS、内存使用、网络请求

2. 渐进式优化 ​

  • 先优化影响最大的部分
  • 避免过早优化
  • 保持代码可读性

3. 权衡取舍 ​

  • 性能 vs 视觉质量
  • 加载速度 vs 运行性能
  • 内存占用 vs 渲染速度

数据加载优化 ​

1. 分层加载（LOD） ​

javascript
// 使用 3D Tiles 的 LOD 机制
const tileset = viewer.scene.primitives.add(
  new Cesium.Cesium3DTileset({
    url: 'path/to/tileset.json',
    maximumScreenSpaceError: 16, // 降低此值提高质量，但降低性能
    maximumMemoryUsage: 512 // 限制内存使用（MB）
  })
);

// 动态调整 LOD
viewer.camera.changed.addEventListener(() => {
  const distance = Cesium.Cartesian3.distance(
    viewer.camera.position,
    tileset.boundingSphere.center
  );
  
  if (distance < 1000) {
    tileset.maximumScreenSpaceError = 8; // 近距离高质量
  } else {
    tileset.maximumScreenSpaceError = 32; // 远距离低质量
  }
});

渲染优化 ​

1. 实体合并 ​

javascript
// 不好的做法：创建大量独立实体
for (let i = 0; i < 10000; i++) {
  viewer.entities.add({
    position: Cesium.Cartesian3.fromDegrees(lon, lat, 0),
    point: { pixelSize: 5, color: Cesium.Color.RED }
  });
}

// 好的做法：使用 PointPrimitiveCollection
const pointCollection = viewer.scene.primitives.add(
  new Cesium.PointPrimitiveCollection()
);

for (let i = 0; i < 10000; i++) {
  pointCollection.add({
    position: Cesium.Cartesian3.fromDegrees(lon, lat, 0),
    pixelSize: 5,
    color: Cesium.Color.RED
  });
}

2. 批处理 ​

javascript
// 使用 Primitive 代替 Entity 以获得更好的性能
const instances = [];
for (let i = 0; i < 1000; i++) {
  instances.push(new Cesium.GeometryInstance({
    geometry: new Cesium.RectangleGeometry({
      rectangle: Cesium.Rectangle.fromDegrees(lon, lat, lon + 0.1, lat + 0.1),
      vertexFormat: Cesium.PerInstanceColorAppearance.VERTEX_FORMAT
    }),
    attributes: {
      color: Cesium.ColorGeometryInstanceAttribute.fromColor(Cesium.Color.RED)
    }
  }));
}

viewer.scene.primitives.add(new Cesium.Primitive({
  geometryInstances: instances,
  appearance: new Cesium.PerInstanceColorAppearance()
}));

3. 使用对象池 ​

javascript
// 重用对象而不是频繁创建销毁
class PositionPool {
  constructor(size) {
    this.pool = [];
    for (let i = 0; i < size; i++) {
      this.pool.push(new Cesium.Cartesian3());
    }
    this.index = 0;
  }
  
  get() {
    const position = this.pool[this.index];
    this.index = (this.index + 1) % this.pool.length;
    return position;
  }
}

const pool = new PositionPool(100);
const position = pool.get();

// 防抖和节流
function debounce(func, wait) {
  let timeout;
  return function(...args) {
    clearTimeout(timeout);
    timeout = setTimeout(() => func.apply(this, args), wait);
  };
}

最佳实践 ​

1. 代码组织 ​

javascript
// 使用模块化
// viewer-manager.js
export class ViewerManager {
  constructor(container) {
    this.viewer = new Cesium.Viewer(container);
  }
  
  destroy() {
    this.viewer.destroy();
  }
}

// entity-manager.js
export class EntityManager {
  constructor(viewer) {
    this.viewer = viewer;
    this.entities = new Map();
  }
  
  addEntity(id, entity) {
    this.entities.set(id, entity);
  }
  
  removeEntity(id) {
    const entity = this.entities.get(id);
    if (entity) {
      this.viewer.entities.remove(entity);
      this.entities.delete(id);
    }
  }
}

3. 性能监控 ​

javascript
// 监控 FPS
let lastTime = performance.now();
let frameCount = 0;

viewer.scene.postRender.addEventListener(() => {
  frameCount++;
  const currentTime = performance.now();
  
  if (currentTime - lastTime >= 1000) {
    const fps = frameCount;
    console.log('FPS:', fps);
    frameCount = 0;
    lastTime = currentTime;
  }
});

// 监控内存
setInterval(() => {
  if (performance.memory) {
    console.log('Memory:', {
      used: (performance.memory.usedJSHeapSize / 1048576).toFixed(2) + ' MB',
      total: (performance.memory.totalJSHeapSize / 1048576).toFixed(2) + ' MB'
    });
  }
}, 5000);

常见性能问题 ​

1. FPS 过低 ​

  • 检查是否渲染了过多实体
  • 降低地形和影像质量
  • 禁用不必要的后期处理效果
  • 使用 Primitive 代替 Entity

2. 内存占用过高 ​

  • 清理不需要的实体和 Primitive
  • 限制 3D Tiles 内存使用
  • 减少纹理大小
  • 使用对象池

3. 加载缓慢 ​

  • 使用 CDN
  • 压缩数据
  • 启用缓存
  • 按需加载

4. 移动端卡顿 ​

  • 降低渲染分辨率
  • 禁用阴影和后期处理
  • 优化触摸事件
  • 减少同时渲染的对象
